# V318 개요

V318은 V317(업로드 UI + 귀속 룰 선택 + 통화/세금)에서 한 단계 더 나아가,
**현장에서 가장 자주 깨지는 2가지**를 해결하는 데 초점을 둡니다.

1) CSV 헤더가 조금만 달라도 적재가 실패한다  
→ **매핑 UI**로 사용자가 직접 컬럼을 지정하고 저장(재사용)

2) 전환 귀속이 데이터마다 다르다(UTM만 있는 경우, 소재ID만 있는 경우 등)  
→ **귀속 우선순위 체인**으로 “가능한 가장 정밀한 키”부터 매칭하되 중복 집계 방지

추가로, 환율/세금 파일을 UI로 교체/업로드할 수 있어 운영 편의성을 올렸습니다.
