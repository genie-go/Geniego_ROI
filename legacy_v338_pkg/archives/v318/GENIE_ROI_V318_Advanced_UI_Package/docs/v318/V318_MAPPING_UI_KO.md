# V318 CSV 매핑 UI 사용법

## 목표
채널 CSV가 ‘형식이 제각각’이어도, 업로드 전에
- date는 어떤 열인지
- campaign_id는 어떤 열인지
- cost/currency/tax는 어떤 열인지
를 사용자가 직접 지정해서 **항상 적재 가능**하게 합니다.

## 흐름
1) UI에서 채널 선택
2) 광고 CSV 파일 선택
3) 브라우저가 CSV 첫 줄(헤더)을 읽어 드롭다운에 표시
4) date/campaign_id/cost… 같은 “표준 필드”에 해당하는 열을 선택
5) [매핑 저장] → templates/v318/custom_mappers.json에 저장
6) [업로드/적재] → 방금 저장한 매핑이 최우선 적용

## 장점
- 헤더 매핑을 코드 수정 없이 운영자가 관리 가능
- 대행사/채널별 CSV가 바뀌어도 즉시 대응
