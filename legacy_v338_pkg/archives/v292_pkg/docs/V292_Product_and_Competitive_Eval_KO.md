# GENIE_ROI V292 — 제품/기능 설명 + 냉정한 점수(10점 만점) + 경쟁사 대비

## 1) 이 플랫폼이 고객에게 제공하는 “서비스”
GENIE_ROI는 **MA(마케팅 자동화) + ROI/어트리뷰션/실험(인크리멘탈리티) + 딜(인플루언서/제휴) 의사결정**을 한 플랫폼에서 연결해,
1) **트리거 기반 Journey로 행동을 자동 실행**하고  
2) 실행 결과를 **광고/CRM/커머스 전 채널 성과로 증명**하며  
3) 다음 예산/딜 결정을 **증분 매출/증분 이익(Incremental) 관점으로 최적화**하는 “성과 검증형 Growth 운영 플랫폼”입니다.

핵심 가치(운영 관점):
- **Speed**: 이벤트 발생 → 워커가 즉시 재진입/중복방지 포함 실행(Always-on)
- **Proof**: 캠페인/실험/어트리뷰션/딜 성과가 하나의 드릴다운으로 이어짐
- **Decision**: ROAS가 아닌 **Lift/Incremental Profit** 기반으로 예산/딜 결정을 지원

## 2) V292에서 업그레이드한 약점(요구사항 반영)
### A. 이벤트 기반 캠페인/Journey 상시 워커 고도화
V291 대비 V292에서 아래가 구현/강화되었습니다.
- **Per-journey cursor**: 이벤트를 “룩백 폴링”이 아니라 **여정별 커서**로 안전하게 증분 처리(중복/누락 리스크↓)
- **재진입 정책**: event / daily / journey(1회) 재진입 키를 안정적으로 생성
- **중복 방지(Idempotency)**: 스텝별 idempotency key를 저장해 **재시도/다중 워커에서도 중복 실행 방지**
- **동시성 안전**: due enrollment는 `FOR UPDATE SKIP LOCKED`로 잠금 처리(워커 수평확장)

> 코드 변경: `services/journey_executor/journey_executor.py` + DB migration `0011_v292_journey_and_drilldown.sql`

### B. 리포팅 UI “드릴다운” 대시보드(하나로 보는 화면)
V292에서는 기존 ROI JSON 뷰 수준을 넘어,
- **Campaign(Spend) ↔ Conversion(Revenue) ↔ Experiment(Lift) ↔ Deal(Incremental)** 을
- 동일 기간(from/to) 기준으로 조회하고
- 캠페인/실험을 클릭하면 우측 상세 패널에서 즉시 확인되는 **master-detail 드릴다운 UI**를 추가했습니다.

> 백엔드 엔드포인트: `GET /v1/roi/drilldown?from=YYYY-MM-DD&to=YYYY-MM-DD`  
> UI 탭: Dashboard에 **Drilldown** 탭 추가

## 3) 마케팅/광고/기타 적용 분야별 “어떻게 쓰이는가”
### (1) 퍼포먼스 마케팅(Ads)
- 채널별 spend/impressions/clicks/conversions 수집 → 캠페인 단위로 ROAS/ROI 확인
- 실험(holdout)을 붙이면 “ROAS가 높아도 리프트가 0”인 함정을 걸러냄
- 예산 업데이트/캠페인 on/off 같은 액션을 Journey 스텝으로 자동화 가능

### (2) CRM/리텐션(Email/Push/SMS/온사이트)
- 이벤트 트리거(장바구니, 재방문, 결제 실패 등) → Journey 자동 실행
- 빈도 제한(Frequency cap)과 결합해 과다 발송 리스크 완화
- 메시지 이벤트(오픈/클릭/바운스) 수집 → 어트리뷰션/세그먼트에 피드백

### (3) 실험/인크리멘탈리티(Incrementality)
- unit(contact/cookie) 단위로 treatment/holdout 할당
- outcome(전환/매출)을 집계해 **lift%**로 성과를 검증
- 딜/캠페인 선택을 “증분” 관점으로 정렬

### (4) 인플루언서/제휴 딜 의사결정
- 인플루언서/상품/콘텐츠 타입 기반 기대 노출/클릭/전환/매출 추정
- V291~V292에서는 **lift_pct, incremental_revenue, incremental_profit**까지 포함해
  “딜을 했을 때 실제로 늘어나는 이익”으로 판단 가능

### (5) 조직 운영(Approval/RBAC/감사로그)
- 예산/딜/대규모 캠페인은 승인 플로우를 거쳐 실행
- 감사로그로 누가/왜 실행했는지 추적
