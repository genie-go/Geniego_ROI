# GENIE ROI / MA Platform V294 — 톱티어 동급 업그레이드(제품 정의 & 냉정 평가)

## 1) 이 플랫폼은 무엇을 제공하나 (한 줄 정의)
**“전 채널 Growth 운영(Orchestration) + 성과 검증(Measurement) + 딜/오퍼/예산 의사결정(Decisioning) + 마켓플레이스(Offer/Settlement/Appstore)”를 하나의 운영 루프로 통합한 ‘검증형 Growth OS’**입니다.  
즉, **자동화로 실행** → **코호트/퍼널/세그먼트/실험/어트리뷰션으로 증명** → **CI(신뢰구간) 기반으로 딜/오퍼/예산 결정을 추천** → **정산/오퍼 운영까지 닫는 루프**를 제공합니다.

---

## 2) V294 업그레이드 포인트

### A. 코호트/퍼널/세그먼트 드릴다운 (Reporting UI의 ‘분석 깊이’ 강화)
- **Cohort Drilldown**: cohort_event(예: signup) 기준 코호트를 만들고, conversion_event(예: purchase)까지 **day offset별 전환율**을 확인
- **Funnel Drilldown**: signup → view_product → add_to_cart → purchase 등의 스텝을 한 화면에서 비교(유니크 컨택 기준)
- **Segment Drilldown**: 세그먼트별 **멤버수/구매자/매출/ROI**를 바로 확인

> 구현 메모: `events`, `segment_members` 기반으로 계산하며, `idx_events_tenant_type_time` 등 인덱스로 기본 성능을 확보했습니다.

### B. CI(신뢰구간) 기반 딜 추천 (의사결정 품질 강화)
- influencer_product_stats에서 clicks/conversions/revenue/cost를 집계하고,
- 전환율은 **Wilson interval(95% CI)**, profit/ROI는 **전환 불확실성 기반의 실용적 CI**로 근사
- **하한(LOW bound)** 중심 규칙:
  - **SCALE**: profit_low > 0 AND roi_low > threshold
  - **STOP**: profit_high < 0
  - 그 외 **HOLD**
- UI에서 profit/ROI의 **CI band**를 함께 노출 → “왜 추천했는지”를 숫자로 설명

### C. Journey 노드형 빌더 UI (톱티어 MA급 UX로 상승)
- 드래그&클릭 중심의 **노드 기반 그래프 편집**(Trigger/Action/Wait/Branch)
- 그래프는 `templates(kind=journey_graph)`로 저장되어 협업/재사용이 가능
- 실행 엔진은 V292+의 안정형 워커(재진입/중복방지/재시도)를 그대로 사용하고, 그래프 → 스텝 변환은 얇게 유지

### D. 전용 마켓플레이스 기능 대폭 강화 (약점 보완)
V294는 “마켓플레이스가 약점”을 **해결 방향으로 전환**했습니다.

- **Catalog**: SKU/가격/카테고리/활성 상태 관리
- **Offers**: 오퍼 코드/규칙 JSON/기간/상태(draft/active/paused)로 오퍼 라이프사이클 운영
- **Settlements**: 정산 레저(총매출/수수료/지급액/상태) 기반으로 파트너 지급/회계 운영의 출발점 제공
- (기반 구축) **Apps**: 앱스토어 목록 구조(리스팅/스코프/퍼블리셔) 테이블 추가 — V295에서 설치/권한/리뷰 흐름을 더 촘촘히 만들 수 있음

## 3) 마케팅/광고/여러 분야 적용(고객 가치)

### (1) 마케팅(Lifecycle/CRM)
- 이벤트 트리거 Journey로 온보딩/리텐션/재구매를 자동화
- Cohort/Segment Drilldown으로 “누가/언제/왜 반응했나”를 즉시 확인
- 승인/감사 로그로 엔터프라이즈 운영 리스크 감소

### (2) 광고(퍼포먼스/리타겟/예산 운영)
- 전 채널 지표를 ROI 드릴다운으로 통합 → 예산 재배분 속도 상승
- 딜(인플루언서/파트너)까지 같은 기준(ROI + CI)으로 비교 가능

### (3) 실험/인크리멘털리티
- 캠페인 성과와 실험 Lift를 한 화면에서 비교(“성과가 진짜인가?”)
- V294는 CI 기반 딜 추천까지 확장되어 **검증 루프의 마지막(결정)**까지 닫음

### (4) 딜/마켓플레이스(제휴/파트너/오퍼)
- Catalog/Offer/Settlement로 “오퍼 운영 → 성과 검증 → 정산”까지 연결
- 앱스토어 기반을 갖춰 파트너 생태계 확장 가능

## 6) 시장 진입 시 예상 효과(예측)

### A. 도입/POC 성공률 상승
- 실행(자동화)과 증명(분석/CI)이 한 제품이라 **POC에서 ‘성과 근거’ 제시가 빨라짐**
- “왜 이 결정을 했는지”를 CI로 설명 가능 → 의사결정자 설득 비용 감소

### B. 랜드 앤 익스팬드(확장) 경로 강화
- CRM → 광고 → 딜 → 마켓플레이스(오퍼/정산)까지 자연스러운 확장 경로
- 고객 내 이해관계자(마케팅/그로스/재무/파트너십)를 한 제품에서 연결

### C. 경쟁 스택 대체 가능성 증가
- 기존 MA + 별도 실험/BI + 정산 툴을 붙인 조합 대비
  - 데이터 불일치/운영 비용/의사결정 지연을 줄이는 “통합 가치”로 교체 동기 강화

### D. 리스크/과제(냉정)
- 앱스토어 **생태계 규모**(파트너 수, 템플릿, 리뷰/배포 자동화)는 시간이 필요
- 엔터프라이즈 레퍼런스 확보를 위해 커넥터 확대/보안 심사 자료/운영 가이드가 중요
