# GENIE ROI / MA Platform V293 — 제품 정의 & 냉정 평가(10점 만점)

## 1) 이 플랫폼은 무엇을 제공하나 (한 줄 정의)
**“전 채널 Growth 운영(Orchestration) + 성과 검증(Measurement) + 딜 의사결정(Decisioning)”을 한 흐름으로 묶는 운영 플랫폼**입니다.  
즉, **자동화로 실행**하고 → **실험/어트리뷰션/딜 성과로 증명**하고 → **다음 예산/딜 결정을 추천**하는 형태입니다.

---

## 2) V293에서의 업그레이드 포인트(요청사항 반영)

### A. Journey/트리거 자동화: 상시 워커의 ‘운영 완성도’ 상승
V292에서 “커서+아이템포턴시+락 기반 병렬 처리”를 넣었다면, V293는 **실패/재시도/운영성**을 올려 톱티어 MA의 기본 기대치에 더 가까워집니다.

- **Controlled Retry (신규)**: 스텝 실행 실패 시 즉시 Journey 실패가 아니라,  
  `journey_step_failures`에 기록 → **지수 백오프 재시도** → 횟수 초과 시만 실패 처리
- **DLQ 성격의 추적(신규)**: status=PENDING/GAVE_UP/RESOLVED 로 장애가 “눈에 보이는 상태”가 됨
- **아이템포턴시와 결합(유지/강화)**: 재시도 상황에서도 중복 발송/중복 콜 최소화

### B. 리포팅 UI 드릴다운: “성과 검증 + 딜 의사결정”을 한 화면으로
V292의 드릴다운은 ‘캠페인/실험/딜 성과를 한 화면’까지였고, V293는 **의사결정에 필요한 형태**로 한 단계 더 갑니다.

- **Channel Breakdown (신규)**: 채널별 Spend vs Revenue(프록시)와 ROI를 표 형태로 표시
- **Deal Decision Pack (신규)**: 딜별 incremental profit/ROI 기반으로  
  SCALE / HOLD / STOP 추천(규칙 기반 프록시) + confidence(프록시) 제공

> 주의: V293의 추천/신뢰도는 “규칙 기반 프록시”이며, 진짜 톱티어는 CI(신뢰구간), 인크리멘털 추정 모델, 멀티터치/실험 설계와 결합이 필요합니다.

## 3) 마케팅/광고/CRM/세일즈 딜에 어떻게 적용되나

### 마케팅 운영(CRM / Lifecycle)
- 신규 가입/장바구니 이탈/재방문/구독갱신 등 **이벤트 트리거 Journey**
- 실패 재시도/중복 방지로 “운영 장애 비용” 감소
- 세그먼트/템플릿/승인(Approvals) 기반으로 “대규모 운영”에 적합

### 광고 운영(퍼포먼스/리타겟/크리에이티브)
- 웹훅/커넥터/템플릿을 통해 채널 실행을 표준화
- 채널별 Spend/Revenue 드릴다운으로 “예산 리밸런싱” 가속

### 실험/인크리멘털리티
- 실험 결과(Lift%)를 캠페인/딜 지표와 같은 페이지에서 비교
- 실제 톱티어는 실험 설계 자동화(샘플사이즈/가드레일), 베이지안/CI까지 필요(현재는 확장 여지)

### 딜(파트너/마켓플레이스/제휴) 의사결정
- 딜 ledger 기반으로 incremental profit/ROI를 묶어 “딜 지속/확장/중단” 결정을 지원
- 단, **전용 마켓플레이스 기능(오퍼/재고/정산/오디언스 패키징)은 여전히 약점**으로 남김

## 4) 기능별 점수(절대평가, 10점 만점) — V293
- Journey/트리거 자동화(상시 워커, 재진입/중복방지/재시도): **9.7**
- 리포팅 & 드릴다운(캠페인/실험/어트리뷰션/딜 통합): **9.6**
- 실험/인크리멘털(기초): **9.2**
- 어트리뷰션(기초/프록시): **9.1**
- 딜 의사결정(Decision Pack, 프록시): **9.3**
- 데이터/거버넌스(승인/감사/권한의 기초): **9.2**
- 커넥터/채널 실행(기초): **9.3**
- 전용 마켓플레이스 기능(오퍼/정산/카탈로그): **7.4 (약점 유지)**

> 총평: “운영 자동화 + 검증 + 의사결정” 흐름은 9.7 근접.  
> 다만 **Marketplace 고도화**와 **실험/어트리뷰션의 통계적 엄밀성(특히 CI/베이지안/모델링)**이 없으면 ‘톱티어’ 완전 동급이라고 보긴 어렵습니다.

## 5) 경쟁사 대비 점수(상대평가, 10점 만점) — 냉정 비교(요약)
기준: **Braze / Iterable / SFMC / HubSpot / Klaviyo** 등 “톱티어 MA” + 일부 CDP/Measurement 툴 관점.

- Journey/오케스트레이션: **9.2**
  - 강점: 워커 운영성(재시도/중복방지) 개선, 간결한 엔진
  - 약점: 노드형 빌더/콘텐츠 협업/대규모 템플릿 관리 UX는 더 필요
- 리포팅 드릴다운: **8.9**
  - 강점: 캠페인/실험/딜을 ‘같은 화면’에서 보는 결합
  - 약점: 멀티터치/코호트/퍼널/세그먼트 드릴다운의 깊이 부족
- 실험/인크리멘털: **8.4**
  - 톱티어는 실험 설계 자동화/CI/가드레일이 기본
- 딜 의사결정: **8.6**
  - Decision pack은 방향성 좋지만, 모델링/신뢰구간/리스크(변동성) 반영이 약함
- 커넥터/마켓플레이스: **7.0**
  - “앱스토어형 마켓플레이스”나 “오퍼/정산/패키징”까지는 격차 큼
