## 3. 기능 ② admin 메뉴 가시성 — raw 절차 (백엔드 명세 선결)

요구: admin이 전체 서비스 메뉴를 메뉴 단위 숨김/표시 → 숨김 메뉴는 일반 로그인 사용자에게 비노출.

### 3-1. raw 확인 (소스 도착 즉시 — `session120_diag_menu_tree.py`)
1. **메뉴 트리 정의 위치**: 라우팅(`react-router` routes), 사이드바/네비 컴포넌트, 메뉴 상수 grep (`menuItems`/`navConfig`/`routes`/`sidebar`). 단일 출처인지 분산인지 확정.
2. **메뉴 식별 체계**: 각 메뉴의 고유 key/id 존재 여부. 없으면 가시성 매핑 키 설계 선행 필요(추측 금지, raw 후 검수자 분기).
3. **권한 분기 현황**: admin vs 일반 사용자 구분 로직 위치(role/auth context), 기존 조건부 렌더 패턴.
4. **PM P-1 재판정**: 이미 부분 구현됐을 가능성 raw 확인 (PM 표기 3회 어긋남 전례 — Phase1 no-op, Phase2 이미완성). "없다" 가정 금지.

### 3-2. 백엔드 명세 의존 항목 (§5 PM 요청에 포함)
- admin 전역 메뉴 설정 조회/수정 엔드포인트 + 스키마
- 권한: admin 전용 인가 방식 (PM apiClient `getJsonAuth`/`requestJsonAuth` 중 무엇 — PM 2절 4번 동일 이슈)
- 기본값: 신규 메뉴 추가 시 기본 노출 여부
- 적용 시점: 로그인 시 1회 vs 실시간 (PM 캐싱 이슈와 동일)

### 3-3. 명세 확보 후 적용 (분기)
- **명세 제공**: GDC 또는 전용 context에 `menuVisibility` 상태 + `getJson('/api/admin/menu-visibility')` 분기 + 로딩/에러 + 메뉴 렌더부에 가시성 필터. PM Phase2 패턴(`_isDemo ? mock : await getJson`) 동일 적용 — 데모 모드 보존(PM 2절 2번).
- **명세 없음**: UI 골격 + 로컬 state mock까지만 선행 산출(영속/배선 제외). 명세 도착 시 결합. 추측 영속 구현 금지(P-5).

## 4. 기능 ③ 사용자별 개인 메뉴 가시성 — raw 절차 (백엔드 명세 선결)

요구: 로그인 사용자가 본인에게 보이는 메뉴 중 개인적으로 숨김/펼침.

### 4-1. raw 확인
- ②의 메뉴 트리 raw 재사용. 추가로 사용자 식별(userId/auth context) raw, 사용자별 설정 저장 위치 후보.

### 4-2. 우선순위 규칙 (명세 필수 — §5 핵심 질문)
- **admin 숨김(전역) > 사용자 숨김(개인) 우선순위 정의**. admin이 숨긴 메뉴를 사용자가 펼칠 수 있는지 = 정책 결정 사항. 추측 금지, PM 명세 회신 항목.
- 충돌 동작: 전역 숨김 ∧ 개인 표시 → 최종 비노출(권장 기본) vs 표시. 명세 확정 전 구현 보류.

### 4-3. 명세 확보 후 적용
- 사용자×메뉴 N:N 설정 조회/수정 엔드포인트. 본인 설정만 수정 가능 인가.
- 적용: 최종 가시성 = `전역가시(admin)` AND `개인가시(user)` 합성. 렌더부 단일 필터 함수로 일원화(분산 조건문 금지, 유지보수성).
- 명세 없으면 ②와 동일하게 UI 골격+mock까지만.

## 5. PM 백엔드 명세 요청 — 보강본 (직전 메일 대체)

직전 작성 메일에 카테고리 서버검증 항목을 더해 단일 요청으로 통합. PM 인계서 2절 게이트 충족용.

**확정 필요 항목 (회신받아야 착수 가능):**

1. 캠페인 판매 상품 카테고리(금융/보험/의료서비스/세무회계/법률/기타서비스)가 **서버 검증·저장 대상인지**. 프론트 상수면 명세 불요(§2 즉시 진행), 서버 enum이면 백엔드 동기 추가 필요.
2. admin 전역 메뉴 가시성: 조회/수정 엔드포인트, 메뉴 키 스키마, 인가 방식.
3. 사용자별 메뉴 가시성: 조회/수정 엔드포인트, 본인 한정 인가.
4. **우선순위 정책**: admin 전역 숨김 vs 사용자 개인 설정 충돌 시 최종 동작.
5. 기본값: 신규 메뉴 기본 노출 여부, 미설정 사용자 기본 동작.
6. 적용 시점: 로그인 1회 캐싱 vs 실시간 반영.
7. 데모/실서비스 분기(`_isDemo`) 하에서 메뉴 설정 동작(PM 2절 2번 정책 계승).
