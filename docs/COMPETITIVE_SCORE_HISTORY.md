# 경쟁 평가 점수 이력 (SCORE HISTORY LEDGER)

> **목적**: 차수별 경쟁 재평가 점수를 **동일 기준으로 누적 비교**하기 위한 정본. 매 재평가 후 이 표를 갱신한다.
>
> ⚠️ **읽기 전 필독**: 점수를 차수 간 단순 비교하지 마라. **채점 방법론이 283차에 바뀌었다**(아래 §1). 방법론이 다른 차수의 점수를 나란히 놓고 "후퇴했다"고 판정하는 것 자체가 오판이다.

### 283차 잔여(다음 차수 이월)
- **CRM PII 필드암호화** — 정직한 보류. 전화번호가 SQL 조회조건(`REPLACE(phone,...)=?`)·발송 경로에 쓰여 그대로 암호화하면 **암호문으로 문자가 발송**된다. 5단계 이행계획(email_hash → phone_e164_hash → decrypt-on-read → 암호문 전환) 수립됨.
- **SMS/Kakao 단독 캠페인 STO** — 큐/워커(cron 드레인) 부재로 defer 시 무음 유실. 큐 신설이 선행돼야 함.
- **발송처리 honest pending 채널** — 국내 3종(coupang/naver/cafe24) 등은 API 스펙 확신 부족으로 추측 구현하지 않음(어댑터 추가 즉시 자동 전송되도록 큐는 보존).
- 쇼핑/카탈로그 광고(DPA·PMax) · 드래그드롭 피벗 · 임베디드 애널리틱스 · MLOps 드리프트 파이프라인.
