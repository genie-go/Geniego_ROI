## 결정

### D-1. 🔴 최우선은 보안이 아니라 **접근 유지**다

| 실패 | 결과 |
|---|---|
| 과도한 권한을 못 지움 | 보안 위험 — 나쁘다 |
| **정상 접근을 지워버림** | **고객이 로그인 못 함 — 즉시 장애** |

> **후자가 먼저다.** 보안 강화가 서비스를 멈추면 **롤백되고, 롤백되면 보안도 사라진다.**
> **살아남지 못하는 개선은 개선이 아니다.**

### D-2. 분류 체계를 재작성하지 않는다 (Golden Rule)

5-1 §50/§51이 **실측 file:line 분류를 완료**했다 →
**1-9는 정본화**할 뿐 재작성하지 않는다. `VALIDATED_LEGACY` / `CONSOLIDATION_REQUIRED` /
`KEEP_SEPARATE_WITH_REASON` / `MIGRATION_REQUIRED` / `NOT_APPLICABLE`.

### D-3. 🔴 LEGACY-GAP-01 — `VALIDATED_LEGACY` 인데 검증된 적이 없다

`tools/guard_headerless_getjson.mjs` = **`VALIDATED_LEGACY`(Lint 기반)** 로 분류 ·
규칙은 **"VALIDATED_LEGACY 는 재사용 강제"**. **그런데 호출처 0이다.**

> **"VALIDATED"가 거짓이다** — 아무도 검증하지 않았고 **파일 존재가 검증을 대체**했다.
> 이 분류는 결국 **"아무것도 하지 않는 파일을 반드시 재사용하라"**는 명령이 된다.
> **"CI 가드"라는 설명도 사실이 아니다 — CI에 없다.**

**★같은 고아 가드가 세 번째 층을 오염시켰다:**
**5-8**(미실행) → **1-8**(보존 대상 = 회귀 검증이 공허하게 참) → **1-9**(`VALIDATED_LEGACY` = 재사용 강제).

> **하나의 미배선 파일이 세 층에서 "보호가 있다"는 신뢰를 만들었다.**
> **오신뢰의 복리다 — 각 층이 앞 층을 근거로 신뢰를 키웠다.**

### D-4. 🔴 LEGACY-GAP-02 — **5-8이 위임을 떨어뜨렸다 (본 세션의 자기 결함)**

5-1이 Permission 범위 Legacy Equivalence를 **5-8로 위임**했다:
`§51:10` *"실효 동작 보존 = **5-8 Legacy Equivalence**"* · `§58:18` *"전 Certification = 5-8"*.

**측정:** `grep -c "Legacy"` on 5-8 산출 3종 → **0 / 1 / 1** (그마저 "다음=1-9" 문구).
→ **5-8은 Lint·Guard·Golden·Certification만 다뤘고 위임받은 Legacy Equivalence를 누락했다.**

> **남의 결함이 아니라 289차가 만든 결함이다.**
> 5-8은 **"Permission 8/8 종결"을 선언했으나 위임 1건을 이행하지 않은 종결**이었다.
> **RP-001과 같은 계열** — 그때는 로드맵을 안 봤고, 이번엔 **위임을 안 봤다.**

**조치: 1-9(06-A 전체 범위)가 흡수한다. 5-8은 소급 수정하지 않는다**(무후퇴 · 이력 보존).

### D-5. 🔴 LEGACY-GAP-03 — stale 값이 **5벌**(코드 1 + §53 4)

```
grep -rln "351" DSAR_AUTHORIZATION_*.md → 4
  (CRITICAL_GAP_POLICY · DUPLICATE_IMPLEMENTATION_AUDIT · EXISTING_IMPLEMENTATION · FUNCTION_REGRESSION_GATE)
+ 원본 UserAuth.php:329 (286차 주석)
```

**5-6이 351→455로 정정했으나 4벌은 그대로다.**

> **정정이 한 곳에만 남으면 정정이 아니다.**
> 다음 사람은 **4:1로 351을 보고, 다수결로 351이 정본이 된다.**
> **틀린 값이 복제 수로 이긴다.**

**1-8 `CorrectionPropagation`의 필요성이 규모로 실증됐다** → **수치는 참조로 두고 복사 금지**.

### D-6. `EquivalenceProof` — 통합 전 증명 필수

Role **3계통** · PEP **100+** 통합은 06-A 최대 과제이자 최대 위험.
**Golden 확보 → 동일 입력·동일 출력 증명 → 그 후 교체.**
**증명 없는 통합 = 286차 rank 맵 붕괴 재현**(가설 아님 · **실측 이력**).

### D-7. 🔴 부정 동등성도 동등성이다

**"되던 게 된다"만이 아니라 "안 되던 게 계속 안 된다"도 검증 대상.**
통합 후 **막혀야 할 요청이 통과하면 그것도 동등성 위반**이다.
**1-8 DENY 우선과 같은 논리** — ALLOW만 보면 권한 통합 실패를 못 잡는다.

### D-8. `DelegationLedger` — D-4의 구조적 해소

```
규칙: 블록 종결 선언 시 → 위임받은 항목 전수 이행 확인
      미이행 위임 있으면 → "종결" 선언 금지
```

**5-8이 "8/8 종결"을 선언할 때 이 원장이 있었다면 Legacy 누락이 잡혔다.**
**사람의 기억으로 위임을 추적하면 떨어진다 — 실제로 떨어졌다.**

### D-9. 🔴 Production Certification = **NOT_CERTIFIED**

PC-1~PC-6 전부 미달 → **인증서 발급 0**.
**실패가 아니다** — 06-A는 전방호환 설계 계약 · 코드변경 0이 원칙이었고 **9블록이 전부 지켰다.**

### D-10. 🔴 의존 사슬 — **9블록의 바닥에 1줄이 있다**

```
Production Certification → EquivalenceProof → Golden → 보존목록 무결성 (FAILED)
                                                          → 팬텀 제거 = guard 배선 (1줄)
                                                          → stale 정정 = 351 (5벌)
```

> **06-A 9블록 · 문서 200+편의 최종 인증이 `pre-commit`의 가드 호출 1줄이 없어서 막힌다.**
> 팬텀이 있으면 Golden이 공허하게 참 → EquivalenceProof 불가 →
> 증명 없이 Role 3계통 통합 → **286차 재현**.

**→ `CertificationBlocker.fix_size` 필드 필수.**
**크기를 안 적으면 "인증 불가"가 "손댈 수 없음"으로 읽힌다. 실제 근원은 1줄이다.**

## 거부한 대안

| 대안 | 거부 사유 |
|---|---|
| 5-8을 소급 수정해 Legacy 추가 | **이력 조작**. 누락은 기록하고 1-9가 흡수 (D-4) |
| 5-8 "종결" 선언을 유지·침묵 | **자기 결함 은폐** |
| `VALIDATED_LEGACY` 분류를 1-9가 정정 | **남의 블록 산출물 무단 수정** — 보고·인계 (1-8 D-10 동일) |
| 351 4벌 즉시 수정 | 동상 — **코드변경 0은 문서에도 적용** |
| 고아 가드를 분류에서 삭제 | **275차 의도 소실**(무후퇴 위반). **배선이 답** |
| 증명 없이 Role 3계통 통합 | **286차 rank 맵 붕괴 — 실측 이력** (D-6) |
| ALLOW만 동등성 검증 | 권한 통합 실패를 못 잡음 (D-7) |
| Rebate 구현부터 착수 | **사슬 바닥(1줄)을 두고 가면 인증 불가 고착** (D-10) |

### 06-A가 실제로 찾아낸 것

**Rebate를 설계했지만 값진 산출은 기존 시스템의 결함 8건이다.**

pre-commit 미강제 · 커버리지 분모 부재 · 자기보고 52↔47 · Lint 2/9 블록 · **5-8 위임 누락**.

> **관통 패턴: "있다고 믿는 것이 없다."**
> **Rebate는 없다고 알려진 채로 없어서 아무도 해치지 않았다.**
> **가드·Maker-Checker·pre-commit·351은 있다고 믿긴 채로 없었다.**
> **1-6의 "위험 = 운영영향 × 오신뢰"가 06-A 전체를 관통한다.**
