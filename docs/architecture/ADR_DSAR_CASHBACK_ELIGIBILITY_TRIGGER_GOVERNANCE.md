## 결정 (핵심)

1. **★실측 정직·Eligibility 엔진 부재·인접 엔진 풍부·재사용**: **Cashback Eligibility/Qualification 엔진은 부재(NOT_APPLICABLE·grep 0)**. 그러나 판단 재료의 **실 인접 엔진이 풍부**: **Idempotency**(pg_settlement/Referral/coupon_redemptions UNIQUE·중복 방지 정본)·**Event 정규화**(EventNorm·raw_vendor_event·PixelTracking)·**Consent/Suppression**(isMarketingSendAllowed·email_suppression·crm_channel_prefs·Compliance·PreferenceCenter·Communication 중심)·**Fraud**(AnomalyDetection·DataTrust)·**Segment**(crm_segments·RuleEngine)·**Order/Payment/Subscription/Referral/Return**(OrderHub/pg/Paddle/referral_signup/ReturnsPortal). **기존 EventNorm/Idempotency/Consent/Fraud/Segment 재사용(중복 엔진 금지)**. 지어내기·NO_DATA/오탐 금지·본 Eligibility=멀티테넌트 고객용 미래 cashback 전방호환 계약.

2. **★Consent/Suppression 핵심 GAP(§4.7·4.8·§37)**: **현행 consent/suppression은 Communication(마케팅 발송) 중심**(isMarketingSendAllowed·email_suppression) → **Financial Entitlement Suppression(cashback 권리 차단) 구분 부재=GAP**. **Marketing Consent 없다고 계약상 cashback 권리 자동 차단 금지**(Processing Purpose·Legal Basis 확인)·**Communication Suppression≠Reward Issuance Suppression**. 이 구분을 신설·현행 Communication 게이트 재사용하되 Financial Entitlement 축 추가.

3. **Eligibility≠Qualification·Trigger 수신≠자격 확정(§4.1·§4.2)**: Eligibility(대상 가능)≠Qualification(조건 충족 진행). Trigger Event 수신≠자격 확정(Identity/Tenant/Rule Version/Source Status/Payment/Return Window/Fraud/Limit/Consent/Budget 검증 필요). Return Window 필요 Rule=기간 완료 전 확정 금지(ReturnsPortal/kr return_fee 인접). Eligibility/Qualification/Trigger/Accrual 별도 Lifecycle.

4. **★Event Time≠Received Time·Idempotency·Dedup(§4.3·§4.4·§18·§24)**: event_at/received_at/validated_at/evaluated_at/qualified_at/processed_at 분리. 동일 Event 재수신 시 중복 자격 생성 금지(Event ID·Idempotency Key·Source Object·Sequence). **Duplicate 삭제 금지→Duplicate Group+제외 근거 기록**. **현행 정본**: pg_settlement UNIQUE·Referral referred_user_id UNIQUE·coupon_redemptions UNIQUE=중복 방지 실 패턴 계승. Out-of-order/Late Event(Wait/Reorder/Reprocess/Manual Review)·**Reprocessing 시 중복 Accrual 차단**.

5. **Historical Context Snapshot·결과만 저장 금지(§4.5·§4.6·§28)**: 현재 Customer State로 과거 Eligibility 재계산 금지→당시 Identity/Tier/Segment/Subscription/Consent/Suppression/Fraud/Rule Version/Limit/Scope/Timezone Snapshot 보존. Eligibility 결과만 저장 금지→각 Condition Actual/Expected Value·Data Timestamp·Freshness·Result·Exclusion Reason 보존. **현행 GAP**: crm_segments segment version 미보존(SEG-H4)·Eligibility Context Snapshot 부재.

6. **Fraud Hold≠Block·Shared Account·Referrer≠Referee(§4.9·§4.10·§35·§38·§39)**: Fraud Hold(재검토 후 Eligible 전환 가능)≠Not Eligible/Block(AnomalyDetection·신호 원문 미복제). Shared Account 한도를 개인별로 자동 계산 금지(Account/Member/Household/Organization Scope 구분·Duplicate Benefit Risk). Referrer≠Referee 독립 자격/한도 평가(referral_signup 실 사례·반전=Critical).

7. **Order/Payment/Refund 상태 반영·Reconciliation(§32·§33·§41)**: Order Cancelled·Payment Failed·Refund Completed·Chargeback를 Eligibility에 반영(OrderHub 취소/환불 상태 재사용). Provider↔Internal Eligibility·Trigger↔Source Object Status·Payment↔Qualification·Refund↔Eligibility·Consent↔Processing Purpose·Suppression↔Reward Entitlement·Fraud↔Hold/Block·Limit Snapshot↔Decision·**Event Reprocessing↔Duplicate Accrual Risk** Reconciliation.

8. **정직·무후퇴·Coverage/Gap**: Cashback Eligibility 엔진 부재=PROVIDER_LIMITATION/NOT_APPLICABLE(NO_DATA/오탐 금지). 실 GAP=①Financial Entitlement Suppression 구분 부재(Communication 중심) ②segment version 미보존(SEG-H4) ③Eligibility Context Snapshot 부재. Critical Gap(잘못된 Customer·Cross-Tenant Event·중복 Eligibility·Payment 실패/취소/환불 Eligible·Return Window 전 확정·Version 없이 평가·Consent/Suppression 오해석·Fraud Block Eligible·Shared Limit 중복·Referrer/Referee 반전·Late Event 중복 Accrual·Provider Drift·Historical Context 없이 재계산) 시 Access Review 차단. EventNorm/isMarketingSendAllowed/email_suppression/AnomalyDetection/crm_segments/OrderHub/Paddle/referral_signup/ReturnsPortal 보존(회귀 0). 기능후퇴 0.
