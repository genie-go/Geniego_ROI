## 결정 (핵심)

1. **기존 커넥터 확장(재구현 금지)·인프라 정직**: channel_credential·AdAdapters·OAuth·connector_sync_log·Connectors/ChannelRegistry/ChannelContract·KrChannel/Kakao/Line·Paddle·SmsMarketing·Webhooks·DataPlatform.data_source 는 **정본 — Canonical SaaS Discovery Foundation 아래 통합**. **★현행 외부 SaaS=커머스채널·광고플랫폼·결제·메시징(ingest/outbound)·DSAR Subject Retrieval 미존재**. **외부 CRM/CDP/Support SaaS/Identity Provider=NOT_APPLICABLE**(GeniegoROI 자체가 CRM/CDP·EnterpriseAuth SSO=inbound·지어내기 금지). Provider별·Request Type별 독립 Discovery Engine/Candidate Store 신설 금지(§110).

2. **Provider 이름 ≠ Account Scope(§3.1·3.2)**: 동일 Provider 다중 Account/Workspace/Store/Property/Business/Ad Account 존재(AdAdapters 키 별칭 meta_ads/meta/facebook·OAuth vs 수동 2경로가 실증) → 모든 Retrieval 에 Provider/Application/Provider Account/Workspace·Store·Property/Tenant/Brand/Legal Entity/Environment/Region/Credential/Connector 고정. Global Search 기본 금지. Production/Sandbox 분리.

3. **Internal Sync ≠ Provider 전체·API 성공 ≠ 완전검색(§3.3·3.4)**: 내부 동기화=Sync Lag/Partial/Field Omission/Deleted 누락/Provider-only Data 가능 → Provider 원본/상태 별도검토. Pagination 완료(next token/has_more/end cursor)·Async Job 완료·Deleted/Archived 포함·Object Coverage·Account Scope 검증 후 완료선언. **첫 페이지 성공을 전체 완료로 표시 금지**.

4. **OAuth Scope/Permission 미신뢰·Credential Vault(§3.5·§19)**: Registry Required Scope vs Token Actual Scope 비교 → Missing·**Excessive** 둘 다 Drift. Discovery Connector=최소 Read/Export 권한(Write/Delete/Admin 분리). Credential 원문 코드/Config/로그 저장 금지(Vault Reference)·**개인 직원 Token 장기사용 금지**·Account 간/Prod-Sandbox 재사용 금지. (channel_credential Crypto·평문노출 회피 정합.)

5. **Provider ID ≠ Person ID·Shared Identifier(§3.6)**: Provider Contact/Customer/User/Subscriber ID=Source Account 종속 External Identifier → Account Binding + Customer Identity Mapping 필수. Email/Phone Normalize·Hash Algorithm/Version. Shared Email/Phone/다중 Record=Candidate·Identity Review(단독확정 금지). Deleted Provider ID 재사용·Merged Alias·Sandbox/Production ID 분리.

6. **Webhook ≠ 현재상태·Raw Response 최소화(§3.7·3.8)**: Webhook=Event Evidence → Signature/Timestamp/Replay 검증·Webhook·API 불일치 시 Reconciliation. Async Result URL=Host Allowlist/Signature/Expiry/Checksum·Download 후 Cleanup. Candidate Store=Reference/Minimal Match/Hash/Evidence(Raw=암호화 임시 Review Store 짧은 TTL).

7. **Deleted/Historical/Manual Portal·Provider Limitation(§3.9)**: Active 만 검색 금지 → Archived/Soft·Hard Deleted/Anonymized/Merged/Suppressed 조사·Provider Deleted Endpoint/Audit Log 명시. Historical Coverage(earliest/window/retention) 기록. API 없는 Provider=Manual Portal Task/Owner/Procedure/Evidence/SLA(누락 금지·전체 Export/개인PC저장/개인Email전달 금지). **Provider 미지원=`PROVIDER_LIMITATION` 명시·`COMPLETE`/`NO_DATA` 로 숨김 금지**.

8. **정직·무후퇴·Coverage/Drift**: Retrieval/Pagination 완료검증/Async/Webhook Correlation/Deleted Discovery/Reconciliation/Coverage/Gap=현행 부재→목표계약. Connector Health=285차 가짜녹색 근절(ok=>false) 정합. Coverage 다차원·Critical Gap 시 Access Review 차단. AdAdapters/Connectors/OAuth/channel_credential/Webhooks 보존(Legacy Equivalence·API Compatibility·Sync/Retry 즉시제거 금지). UNEXPLAINED·LEGACY_WRONG_ACCOUNT_RISK·고객영향 LEGACY_DISCOVERY_GAP→전환차단. 기능후퇴 0.
