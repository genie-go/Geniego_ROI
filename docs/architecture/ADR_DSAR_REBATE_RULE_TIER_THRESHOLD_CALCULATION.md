1. **★실측 정직·Rebate Rule/Tier 엔진 부재·인접 재사용**: **Rebate Rule/Tier/Threshold/Calculation 엔진은 부재(NOT_APPLICABLE·grep 0)**. 실 인접=Rule Store(rule_engine metric·op·threshold+**화이트리스트 422 거부**)·Evaluation Evidence(**rule_engine_log observed↔threshold**)·중첩 DSL(Alerting condition_tree)·Segment DSL(CRM 6 operator·AND 암묵)·계산 가드(PriceOpt mode/beat_by/min·max_price/**원가 플로어**/comp_max_age_hours/channel '\*')·Rate·Threshold(kr_fee_rule platform_fee_rate·free_ship_threshold)·Rounding(round -2/0)·Tier 구간(plan seat_tier=**KEEP_SEPARATE**). 지어내기·NO_DATA/오탐 금지·전방호환 계약.

2. **★DSL 4번째 난립 금지(§0·§4.14)**: 현행 **Rule DSL 3종 병존·operator 집합 불일치 실측** — RuleEngine 5 op(**ne 없음**·평면·:33)·crm_segments **6 op**(:1571)·Alerting **문자열 op ">"**(중첩·:71/387). **Canonical Rebate Rule DSL은 Segmentation Canonical DSL(3-2)의 6 operator 상위집합(14)으로 정합**하고 **기존 3종을 대체하지 않고 참조/확장**(추가만·기존 6종 의미 변경 금지·헌법 Vol4 "Intelligence Layer는 하나·중복엔진 금지"·Golden Rule=Replace 아니라 Extend). 평면 규칙은 **단일 AND 그룹으로 무손실 승격**.

3. **Rule Definition≠Evaluation·Operator 화이트리스트(§6·§8·§4.2·§4.14)**: 정의 저장과 평가 결과를 분리(rule_engine/rule_engine_log 분리 패턴 재사용). **Operator 화이트리스트 강제·임의 표현식/eval/raw SQL 금지** — 현행 정본=RuleEngine이 미등록 metric/op/action을 **422로 거부**(:120-121). 4. **Scope Precedence·Stacking(§9~§12)**: Precedence(7·SPECIFICITY 기본·SKU>CATEGORY>BRAND>'\*'·PriceOpt `channel '*'` 와일드카드 재사용)·**동일 구체성 다중 매치 시 결정 규칙 필수(비결정적 매치 금지)**. **Stacking(8) 미지정 시 기본 NONE(fail-closed·묵시적 중복 적용 금지)·SEQUENTIAL은 순서 명시(순서가 금액을 바꾼다)**·Exclusivity Group 위반=Critical. **no-match 기본 지급 금지(Unknown≠Eligible 계승)**. 타 Instrument(Discount/Cashback/Commission)와의 교차 스태킹 금지(4-5-3-1-2 Boundary 계승).

5. **★Tier≠Threshold·Cumulative≠Incremental(§20~§22a·§4.3·§4.4 — 본 파트 최대 리스크)**: Tier=구간[lower,upper)·Threshold=경계값. **Progression Model(6)**: **CUMULATIVE_RETROACTIVE(전체 물량 소급)≠INCREMENTAL_MARGINAL(초과분만)** — **동일 Tier 표에서 모델만 달라도 금액이 근본적으로 달라진다**(명세 예시: 0~100@2%·100~200@3%·실적 150 → 소급형 4.5 vs 증분형 3.5). **★Progression Model 없는 Tier Schedule 발행 금지(기본값 추정 금지·fail-closed)·계약 해석은 Authorized Reference로 근거화·추정 금지**. **경계 overlap/gap 금지=Critical**(2개 Tier 동시 매치 또는 무매치). **★현행 인접에 Cumulative/Incremental 개념 자체가 부재**(free_ship_threshold=단일 on/off·seat_tier=가격표 조회) → **현행에서 유래 불가한 신설 계약임을 명시(지어내기 아님)**.

6. **Retroactive True-up=Append-only(§22b)**: CUMULATIVE_RETROACTIVE 상위 Tier 달성 시 기 발생 Accrual과의 차액 발생 → **기존 Accrual UPDATE 금지·차액을 신규 Append-only 항목으로 생성**(4-5-3-1-4 §32 Backfill≠Recalculation·OrderHub 역분개/4-5-2-5 Reversal 정합). 하향 clawback=4-5-3-1-9 후속.

7. **Basis≠Method·Gross/Net·VAT·취소제외(§23·§24·§4.5)**: Basis=무엇에 대해(4-5-3-1-2 Calculation Basis 33 **분류축을 실행 바인딩으로 연결·재정의 금지**)·Method=어떻게(12·PERCENTAGE_OF_BASIS 현행 `$gross*$feeRate` 인접·OFFSET_BASED `beat_by` 인접·FORMULA는 화이트리스트만). **★Gross≠Net·VAT 포함/제외·취소/반품 제외 술어(286차 취소제외 2축 SSOT 정합) 미지정 계산 금지**.

8. **Period≠Measurement Window·Proration(§25a·§25b·§27)**: 계산 주기와 실적 집계 창 혼동 금지(rolling/calendar/anniversary·late-arriving=Backfill 계약)·타임존(현행 DEFAULT_TZ 'Asia/Seoul' 정합). **Proration(6) 미지정 금지**(기간 중 Version 전환·중도 가입/탈퇴·Suspension 기간 안분).

9. **Cap·Floor·Rounding·Currency(§25c·§25d·§26·§28a·§4.6)**: Cap(BUDGET_LINKED=Commitment 연동·부족 시 Accrual 금지·4-5-3-1-3 §16·BillingMethod MTD cap 재사용·PriceOpt max_price 인접)·**Floor(PriceOpt 원가 플로어 `$cost/(1-$targetMargin)`·`max($minPrice, round($p,0))`=floor 최종 적용 순서 정본)**·**Rounding(단위·방향·적용 시점·잔차 소유자 명시 필수 — 라인별 반올림 합계≠합계 반올림·현행 round(…,-2)/round(…,0) 인접)**·Currency(FX Stage별·rate version·**혼합 통화 basis 단일 요율 계산 금지**).
