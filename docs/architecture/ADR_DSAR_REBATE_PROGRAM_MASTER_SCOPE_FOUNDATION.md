## 결정 (핵심)

1. **★실측 정직·Rebate 엔진 부재·Scope Registry 실 인프라 풍부·재사용**: **고객 Rebate Program 엔진은 부재(NOT_APPLICABLE·grep 0)**. 그러나 **Scope Registry 실 인프라가 풍부**: Provider(ChannelRegistry·data_source SoT priority)·Vendor/Supplier/Distributor(SupplyChain sc_suppliers↔wms_suppliers·PartnerPortal)·Merchant/Seller(channel_credential·kr_settlement_line)·Product/SKU/Category(Catalog·channel_products·Mapping·catalog_category)·Tenant(auth_tenant)·Currency(fxToKrw)·Environment(GENIE_ENV). **기존 Scope Registry 재사용(중복 신설 금지·§6.10)**. 2. **★Rebate≠Cashback·Program≠Rule(§4·§6.1)**: **Rebate=계약/기간/누적 실적/Claim/증빙/검증/정산 조건 사후 확정·B2B(vendor/manufacturer/distributor/dealer)·Credit/Debit Memo·AP/AR 상계·소급 계산·Sell-in/Sell-out 검증** → Cashback Program Registry 이름만 바꿔 복제 금지. 본 블록=Program Master+Scope Foundation만(Definition/Rule/Tier/Threshold/Calculation/Eligibility/Claim/Accrual/Approval/Settlement/Payment=후속 별도 도메인·중복 구현 금지·Reference Field/확장 지점만).

3. **Program 식별·External ID·Hierarchy(§6.2·§6.3·§9~§11)**: Program 이름만으로 동일 판단 금지(Provider/Account/Tenant/Legal Entity/Contract/Country/Currency/Channel/Merchant/Vendor/Product/Period/Environment 다르면 별도). **External Program ID 같아도 Tenant/Provider Account/Environment 다르면 별도 Canonical Binding**(Source System+Account Binding 복합키).
