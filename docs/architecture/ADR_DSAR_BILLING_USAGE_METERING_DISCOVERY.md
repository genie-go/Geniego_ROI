## 결정 (핵심)

1. **★실측 정직·인프라 부재 명시**: **① 내부 Billing Engine 부재** — SaaS 구독 청구=**Paddle(MoR)**·Paddle 이 Invoice/Tax/Dunning/Collection/Proration/Write-off/Billing Run 소유·GeniegoROI 는 paddle_subscriptions/events reference 만 → **Provider Retrieval·`PROVIDER_LIMITATION`**. **② Usage Metering/Rating/Meter/Aggregation=NOT_APPLICABLE**(SaaS=flat seat/tier 요금제·metered 과금 아님·`ai_usage_quota`=사용 **한도**·과금 아님). **③ 내부 실 recurring billing=광고비 관리형 wallet**(`BillingMethod`+`ad_spend_ledger`·billing_key Toss 청구). **④ Settlement(pg/orderhub/creator)=구독사 commerce payout·SaaS 청구 아님(KEEP_SEPARATE)**. 지어내기 금지. 기존 paddle/BillingMethod/ad_spend_ledger/pg_settlement/ai_usage_quota=정본·확장·Provider별 독립 Candidate Store 금지(§98).

2. **Billing Account ≠ Subscription Customer·Invoice ≠ Payment Receipt(§5.1·5.2)**: Billing Account Owner/Billing Contact/Invoice Recipient/Payment Method Holder/AP Contact/Org Admin/Subscriber 역할 분리(11종). **Invoice Recipient/Billing Contact 라는 이유만으로 전체 Org Subscription/Usage/Seat 데이터 포함 금지**. Invoice=Subscription/Usage/Add-on/Seat/Tax/Discount/Credit/Debit/Proration/Fee/Previous Balance 포함(단순 Receipt 아님·Header 만 검색 금지).

3. **Draft ≠ Final·현재 ≠ 완료(§5.3)**: Preview/Estimate/Draft/Open/Finalized/Paid/Void/Uncollectible/Written-off/Archived/Deleted 구분. Invoice Header 만 검색 금지 → Invoice Item(Subscription/Usage(N/A)/Adjustment 원천)·Status/Delivery History·Deleted/Archived 조사.

4. **Raw Usage ≠ Invoice Amount·Event Time ≠ Ingestion Time(§5.4·5.5·5.6)**: (Usage-based 도입 시) Usage Event→Validation→Dedup→Aggregation→Correction→Rating→Invoice Item 각 단계 추적. Duplicate/Rejected/Reversed/Backfill/Late Usage 별도 상태·Late Usage 반영 Period/Invoice 기록. Usage Subject ≠ Account Owner 자동동일시 금지·API Key/Secret 원문 저장 금지. **현행=NOT_APPLICABLE(metered 과금 없음)**.

5. **Proration 전후 보존·Payment 실패 ≠ Billing 실패(§5.7·5.8)**: Upgrade/Downgrade/Quantity/Seat/Anchor Change Proration=변경 전후 조건/기간/Credit·Debit/Invoice Item 보존(Paddle 소유). Invoice Generation/Finalization Failure·Payment Attempt/Collection/Dunning/Usage Rating/Tax Failure 분리.

6. **Tax Identifier 최소화·Dunning 추적**: Tax=Jurisdiction/Category/Rate/Tokenized Reference 만·**전체 Tax Identifier(사업자/주민번호) Candidate 저장 금지**. Collection Attempt(ad_spend_ledger/Paddle)·Dunning(Notice/Retry/Restriction/Pause/Cancellation·Paddle)·Write-off 추적.

7. **Internal ≠ 완료·Reconciliation(§5.9)**: paddle 내부 reference 만으로 완료선언 금지 → Paddle/Payment/ERP/Accounting/Warehouse/Webhook 비교. **Subscription Quantity vs Billed Quantity·Invoice Total vs Payment·Paused Subscription vs New Invoice·Cancelled Subscription vs Future Invoice·Written-off vs Active Collection=High-risk Mismatch**.

8. **정직·무후퇴·Coverage/Gap**: Billing Account↔Subscription↔Payment Mapping·Contact Role·Reconciliation·Candidate/Coverage/Gap=현행 부재→목표계약. Coverage 다차원·Critical Gap(Invoice Recipient 역할혼동·Quantity 불일치·Cancelled Future Invoice·Provider/Internal Total 불일치) 시 Access Review 차단. paddle/BillingMethod/ad_spend_ledger/pg_settlement/ai_usage_quota 보존(Legacy Equivalence·API Compatibility). UNEXPLAINED·LEGACY_WRONG_ACCOUNT_RISK·고객영향 LEGACY_BILLING_DEFECT·LEGACY_DISCOVERY_GAP·CANONICAL_BILLING_DEFECT·Cross-Tenant→전환차단. 기능후퇴 0.
