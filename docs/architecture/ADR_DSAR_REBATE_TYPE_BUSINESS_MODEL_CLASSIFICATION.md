## 결정 (핵심)

1. **★실측 정직·Rebate Type Taxonomy 부재·분류 인프라 인접·재사용**: **Rebate Type Registry/Taxonomy는 부재(NOT_APPLICABLE·grep 0)**. 실 인접=**Provider Type Mapping 패턴(ChannelRegistry group_type·외부 type→canonical)**·**Accounting Nature(Pnl contra-revenue via coupon/point·marketing/commission expense·COGS)**·**Transaction Flow(SupplyChain sell-in·channel_orders sell-out)**·**Calculation Basis(Mmm/Rollup revenue/margin/volume/growth)**·Instrument Boundary(coupon 할인·cashback 부재·마켓 수수료·역분개·point_discount). **기존 group_type/Pnl/SupplyChain/Mmm 재사용(Type Enum 중복 금지·§42·group_type≠rebate type 오혼입 금지)**. 지어내기·NO_DATA/오탐 금지·본 Registry=멀티테넌트 고객용 미래 rebate 전방호환 계약.

2. **★분류만·Type≠계산 로직·단일 Type 금지(§4.1·§4.2·§4.4·§23~§26)**: 본 블록=Type/Business Model **분류만**(상세 Funding/Lifecycle/Rule/Eligibility/Accrual/Claim/Settlement/Payment=후속 중복 금지). **Type=Classification≠계산식/Threshold/Settlement 추론**(상세 Rule=후속). Rebate=단일 Type 금지→Audience/Sponsor/Business Model/Flow/Timing/Calculation Basis/Claim/Settlement/Accounting/Purpose 다차원 동시 분류. **동일 시점 Primary Type 하나만**·Secondary/Hybrid 별도·Mutual Exclusivity(Hybrid 예외).

3. **Instrument Boundary(§4.5·§22·§34)**: **Rebate≠Discount(결제 전 가격 감소·coupon)≠Cashback(거래 후 Reward 적립·부재)≠Refund(원 거래 반환·역분개)≠Commission(성과 보상·마켓 수수료)≠Marketing Fund(MDF 재원)≠Expense Reimbursement(증빙 실비)**. **모호 시 단일 Type 자동 확정 금지→HYBRID/MANUAL_REVIEW**. Cashback/Commission/Refund를 Rebate로 분류=Critical Misclassification.

4. **Provider Mapping·Confidence·이름만 EXACT 금지(§4.3·§27·§31)**: Provider 명칭을 Canonical Type 직접 사용 금지→Versioned Provider Type Mapping(EXACT/CONFIGURED/RULE_BASED/SEMANTIC/MANUAL/LEGACY_ADAPTER). Confidence=Provider Mapping/Contract/Name/Description/Participant/Sponsor Role/Flow/Claim/Settlement/Accounting/Historical/Manual 종합. **Program 이름만 일치 시 EXACT 부여 금지**. → ChannelRegistry group_type 재사용.

5. **Historical Classification 보존(§4.6·§28)**: Current Classification으로 과거 Program/Transaction Type 덮어쓰기 금지→Type Version/Validity(변경/Deprecated/Replacement/Migration/Reporting Impact 보존).

6. **다차원 Classification(§9~§21)**: Audience(19)·Sponsor(15·Sponsor≠Funding Allocation)·Business Model(18)·Transaction Flow(20·Sell-in/through/out·실 Source Object와 Flow 별도 연결)·Timing(16·Prospective/Retrospective)·Calculation Basis(33)≠Structure(18·Tiered/Retroactive)·Claim Model(19)≠Requirement(21)·Settlement Method(22·Credit Memo/AP·AR Offset)≠Timing(14)·Accounting Nature(21·Reference·법인/국가/ERP 정책)·Commercial Purpose(25). → SupplyChain(Flow)·Pnl(Accounting)·Mmm(Basis) 재사용.

7. **정직·무후퇴·Reconciliation/Critical Misclassification**: Rebate 분류 엔진 부재=PROVIDER_LIMITATION/NOT_APPLICABLE(NO_DATA/오탐 금지). Provider↔Canonical·ERP↔Canonical·Contract↔Program Type·Participant↔Audience·Sponsor↔Business Model·Transaction↔Flow·Claim↔Model·Settlement↔Method·Accounting↔Nature·Current↔Historical·Reporting↔Operational Reconciliation.
