## 결정

### D-1. 🔴 인증서를 발급하지 않는다 — `NOT_READY`

**규칙 계약 81(Static 37 + Guard 44) · 구현 0 · 배선 0.**

> **인증이란 "규칙이 실제로 위반을 막는가"의 확인이다. 구현 0이면 확인할 대상이 없다.**
> 0개를 인증하면 **"인증 통과"라는 문자열만 남고 그 문자열이 가장 위험하다** —
> 5-8의 고아 가드가 정확히 그 형태였다(파일 있음 · 실행 0).

**1-7은 기준을 정의하고 상태를 `NOT_READY`로 보고한다.**
**구현 0은 실패가 아니라 설계 의도**(06-A = 전방호환 설계 계약 · 코드변경 0).
**그것을 "인증됨"으로 포장하지 않는 것이 1-7의 임무다.**

### D-2. 🔴 최대 발견 — 9블록 중 2개만 Lint를 영속했다 (LINT-GAP-01)

```
ls DSAR_*.md | grep -iE "static_lint|runtime_guard"  → 4
  = LIFECYCLE(1-4) 20+21 · AUTHORIZATION_FOUNDATION(5-1) 17+23
```

**1-1 Master/Scope · 1-2 Type · 1-3 Funding · R1~R5 · 5-2~5-7 · 1-6 → Lint 0.**

> **81을 "많다"고 읽으면 안 된다. 81은 2개 블록의 것이고 7개 블록은 0이다.**
> **금전 계약의 근간 3블록(Master/Scope·Type·Funding)에 Lint 규칙이 하나도 없다.**

### D-3. `impl_status` 3단계 — `CONTRACT_ONLY` / `IMPLEMENTED` / **`WIRED`**

**`IMPLEMENTED`와 `WIRED`를 합치면 `guard_headerless_getjson` 사고가 재현된다.**
**"구현됨"은 "동작함"이 아니다.** 파일이 있어도 호출처 0이면 실효 0.

### D-4. 🔴 L-2(고아 규칙) 임계 = 0 · **최시급**

**고아 규칙은 "규칙 없음"보다 나쁘다.** 규칙이 없으면 아무도 믿지 않지만,
**고아 규칙은 "가드가 있다"고 믿게 하면서 0이다.**

**현재 L-2 = 1.** **이 1건이 규칙 81건 계약보다 시급하다** —
**계약은 아무도 오해하지 않지만 고아 가드는 오해를 만든다.**
**1-6의 결론과 동일: 위험은 크기가 아니라 오신뢰에서 나온다.**

### D-5. `CertificationReadiness` 게이트 — 조건부 인증 금지

`READY`가 아니면 **verdict 생성 자체를 막는다.**
**"조건부 인증"·"임시 인증"은 반드시 "인증됨"으로 인용된다.**
데이터 헌법 V3의 `READY`/`WARNING`/`BLOCKED`와 동일 철학 —
**품질 미달 데이터를 AI가 못 쓰게 하듯, 미달 Lint 체계로 인증서를 내지 않는다.**

### D-6. `uncertifiable_blocks` 명시 필드

안 적으면 **"1-7 인증 통과"가 06-A 전체를 덮는 것으로 읽힌다. 실제로는 2/9다.**

### D-7. 규칙 재작성 0 — 참조만 (Golden Rule)

5-8 Canonical 규칙(`BypassListIntegrityRule`·`GuardWiringRule` 등)은 **형식만 다르고 내용은 유효** →
**정규화 편입 대상이지 재작성 대상이 아니다.**
`OrphanRuleDetector` = **5-8 `GuardWiringRule` 동일 정본** · `LintRuleRatchet` = **5-8 R0~R3 동일 정본**.
**재정의 금지.**

### D-8. 미달 항목은 1-6 Gap 원장에 연결 — 별도 원장 신설 금지

**원장이 둘이면 정본이 사라진다.**

### D-9. 중복 대조는 `UNVERIFIED`로 남긴다

1-4와 5-1은 독립 작성이라 규칙이 겹칠 수 있으나 **본 세션은 81 규칙 문면 대조를 하지 않았다.**
→ **"중복 없음"이라 보고하지 않는다.** FP 레지스트리 원칙.

---

## 🔴 D-10. 1-6 D-3의 실증 — 여기서는 분모가 있다

1-6은 **§53 요구 목록 부재로 커버리지 계산 불가**였다(COV-GAP-01).
**Lint는 다르다** — 1-4·5-1이 규칙을 **§53 문서로 영속**했으므로 **분모(81)를 저장소에서 도출**할 수 있다.

> **같은 EPIC 안에서 규칙을 문서로 남긴 블록은 인증 가능하고, 남기지 않은 블록은 인증 불가능하다.**
> **차이는 능력이 아니라 영속 여부다.**
>
> 그리고 D-2가 그 대가를 보여준다 — **7개 블록은 영속하지 않았고, 1-7이 인증할 수 없다.**

## 1-7이 인증하지 못하는 것

1. **규칙의 타당성** — **틀린 규칙도 100% 배선될 수 있다.**
2. **7개 블록** — 영속 안 함 → 범위 밖.
3. **미작성 규칙** — **Lint가 스스로 못 잡는다.** 1-1/1-2/1-3의 Lint 0을 **Lint로 발견할 수 없었고,
   사람이 세야 했다. 1-7이 세었다.**
4. **1-7 자신** — 스펙 미수령 → **1-7의 요구 목록도 저장소에 없다**(COV-GAP-01 소급).
