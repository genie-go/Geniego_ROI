## 결정 (핵심)

1. **★실측 정직·인프라 부재 명시**: **① PAN/CVV/PIN 카드원문 저장 0**(grep=Catalog/AutoCampaign UI "card"뿐) — 결제=`Payment`(Toss PG·billing_key 토큰)+`Paddle`(**Merchant of Record**) → **PCI 대부분 OUT_OF_SCOPE·자체 카드 Vault 없음·PAN/CVV/Detokenization=NOT_APPLICABLE**. **② Marketplace Seller/Vendor/Payout/Commission=NOT_APPLICABLE**(구독사 마켓주문 집계·다면 마켓플레이스 미운영). **③ 결제 도메인 2종**: 구독사 e커머스(channel_orders buyer) vs GeniegoROI 자체 SaaS 청구(Paddle/Toss). 지어내기 금지. 기존 channel_orders/OrderHub/Payment/Paddle/Coupon=정본·확장·Store/Order/Payment별 독립 Registry 금지(§91).

2. **Store ≠ Merchant Account(§3.1)**: Store/Site/Sales Channel/Marketplace/Merchant Account/Payment Gateway Account/Subscription Account 분리. Provider 이름/공통 Business Account 만으로 Scope 확정 금지. Prod/Sandbox 분리(운영 Subject 검색에 Test 데이터 금지).

3. **Order Email ≠ Customer Identity·Billing≠Shipping≠Purchaser(§3.2·3.3·3.6)**: buyer_email=Guest/Shared/Corporate/Gift/Billing/Shipping/Relay/Typo/Forwarding 가능 → 단독 Person 확정 금지. 동일 Order 다중 인물(Purchaser/Recipient/Billing/Shipping/Gift/Payment Method Holder/Seller) → Subject Role 별 Field Scope 분리·Order 전체 자동포함 금지. Shipping Address 를 Purchaser 주소로 자동확정 금지.

4. **Payment Customer ID ≠ Person ID(§3.4)**: Merchant/Region/Currency/Environment/Connected Account 종속 → Provider Account Binding 필수(paddle_customer_id·Toss). Guest-to-known Conversion 추적.

5. **Payment Credential 최소화·PCI 차단(§3.5)**: CVV/PIN/PAN/Secret 을 Candidate/Export/로그 포함 금지(현행 미저장 → 발견=Security Incident 후보). Payment Method=Token Reference(billing_key)/Brand/Last Four/Expiry/Funding/Issuer Country 등 Masked Metadata 만. **DSAR Discovery Service 에 Vault Detokenization/Full PAN/Credential Export 권한 금지**.

6. **Internal ≠ 완전성·Payment 성공 ≠ 완전(§3.8·3.9)**: channel_orders(Internal) 만으로 Commerce 완료선언 금지 → Commerce Provider/OMS/Payment/Subscription/Marketplace/ERP/Shipping/Warehouse/Webhook/Archived/Deleted 검토. Payment API 성공≠완전(Pagination/Merchant/Currency/Historical/Refund/Dispute/Payout/Deleted Customer 확인). Webhook(paddle_events)=현재상태 단독근거 아님.

7. **Order-Payment Graph·Wrong-target 차단**: Order→Invoice→Payment→Charge→Refund/Dispute·Subscription→Invoice→Payment·Order→Shipment(WMS)·Payment Customer→Token(billing_key) 관계 Graph. Shared Email 전체 Order/Last Four 단독 Payment Match/Amount·Date 만 Payment 확정 금지. Wrong Customer/Order/Store/Merchant/Cross-Tenant/PCI Prohibited=Exclusion/Block.

8. **정직·무후퇴·Coverage/Gap/PCI**: Store↔Merchant·Guest Identity·Order↔Payment Graph·Subject Role·PCI Classification·Candidate/Coverage/Gap=현행 부재→목표계약. Coverage 다차원·Critical Gap(PCI Unknown·Wrong Merchant·Guest 확정·Order-Payment 미연결) 시 Access Review 차단. channel_orders/OrderHub/Payment/Paddle/Coupon/PgSettlement 보존(Legacy Equivalence·API Compatibility·PCI Control 제거 금지). UNEXPLAINED·LEGACY_PCI_DEFECT·LEGACY_WRONG_CUSTOMER_RISK·고객영향 LEGACY_DISCOVERY_GAP→전환차단. 기능후퇴 0.
