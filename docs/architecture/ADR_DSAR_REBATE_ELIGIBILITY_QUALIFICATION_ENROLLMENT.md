2. **★Eligibility≠Qualification(§18·§19·§20·§4.1)**: **Eligibility=참여 가능 여부(정적·사전·자격 요건)·Qualification=실적/조건 달성(동적·사후)**. 자격 있어도 실적 미달=미지급·실적 달성해도 자격 없음=미지급 — **둘 다 필요·어느 하나로 다른 하나 추정 금지**. Criterion(18·현행 인접 SUBSCRIPTION/PLAN_TIER=Referral PAID_TIERS·RETENTION=RETAIN_DAYS). Qualification 측정은 **4-5-3-1-5 §21 QUALIFICATION Threshold·§23 Basis Binding 연결(재정의 금지)**.

3. **★Qualification≠Accrual≠Availability·Holdback/Maturity(§20d·§21·§4.2·§4.6 — Referral 정본 승격)**: 조건 달성 판정이 곧 지급 가능이 아니다. **Holdback(8 Type: RETENTION_PERIOD·SELL_THROUGH_CONFIRMATION·RETURN_WINDOW·PAYMENT_RECEIPT·AUDIT_CLEARANCE 등) 통과 후 Available**(Cashback 4-5-2-3 "Pending≠Available·Approval≠Availability" 대칭). **현행 정본**: `usable_from = now + 30일` 발급 즉시 잠금(Referral.php:253). **★Maturity=기간 경과만으로 성숙 아님 — 상태 유지 재검증 필수**(현행: CouponRedeem이 usable_from **+ 피추천 활성 여부** 이중 검증·:250-252). Rebate: **Sell-through 미확인·반품 창 미경과·정산 미수령 상태에서 Available 전환 금지**. Tracker 상태=LOCKED/MATURING/MATURED/**FAILED_PERMANENT**("1개월만 쓰고 탈퇴하면 영구 잠금"=실 정본).

4. **★Disqualification≠Termination≠Clawback(§23·§4.8)**: 미확정 단계 미충족 = **Availability 차단(음수 지급/회수 아님)** — **현행 실 사례: Referral은 발급 쿠폰을 회수하지 않고 usable_from 잠금 유지**. 확정 후 사후 취소만 Recovery/Clawback(**4-5-3-1-9 후속·본 블록 구현 금지**). Reason(12·현행 인접 REVOKED=is_revoked·EXPIRED=valid_until).

5. **★1회 평가 후 영구 신뢰 금지·as-of(§22·§22a·§4.11)**: 평가 시점(6: AT_ENROLLMENT·AT_TRANSACTION·AT_QUALIFICATION·**AT_AVAILABILITY**·AT_CLAIM·PERIODIC). **현행 정본: Referral이 발급 시점(isSubscriber)과 리딤 시점(usable_from+피추천 활성) 두 번 평가**(:155/250-252).
