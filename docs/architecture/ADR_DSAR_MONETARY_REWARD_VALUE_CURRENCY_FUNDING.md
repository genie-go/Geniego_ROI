## 결정 (핵심)

1. **★실측 정직·FX 강한 자산·Money VO/Version GAP 명시**: **FX 변환은 강한 실 자산** — `fxToKrw`/`krwToCurrency`(24 통화·KRW internal base·reporting 통화 변환·`app_setting.fx_rates_krw` DB override·hardcoded defaults·live fetch). 그러나 **①금액 float(§4.10 Float 금지 위반) ②per-transaction FX rate version/timestamp 미보존(§4.4 Historical FX = Critical GAP) ③precision/rounding/zero-decimal(JPY/KRW) Registry 부재**. Cost/Margin(`Pnl` COGS WAC·netProfit)·ROI(`Mmm` frontier·payback)는 **주문/마케팅 지출 단위 인접 REAL**(Reward 단위 아님). **Money VO·Value Breakdown(outbound)·Funding Allocation·Liability/Accounting/Revaluation=부재(신설)**. 지어내기·NO_DATA/오탐 금지·본 Value Model=멀티테넌트 고객용 미래 현금성 보상 Value 전방호환 계약.

2. **기존 Money/FX/COGS/ROI 재사용·중복 금지(§4.10·§44)**: **FX=fxToKrw 단일(확장·rate version/timestamp/type 추가)**·COGS=OrderHub::aggregateCogs SSOT(divergence 방지)·ROI=Mmm 단일·Currency=DECIMAL+currency 컬럼(Enum 난립 없음). **Money VO 신설 시 단일(Decimal/Minor Unit)**·Reward 유형별 독립 환율/Cost/ROI 계산 금지.

3. **Currency 강제·Float 금지·Precision/Rounding(§4.1·§4.9·§4.10·§7·§10·§11)**: 모든 Amount에 Amount·ISO Currency·Precision·Rounding Mode·Value Type·Effective Time·Evidence 강제. **Float/Double 금지→Decimal/Minor Unit**(현행 fxToKrw float=우선 시정 대상). Zero-decimal(JPY/KRW)=일반 통화 처리 금지·Currency Precision Registry(ISO numeric/minor unit/rounding)·각 계산 단계 Rounding 기록.

4. **표시≠지급·Face≠Cost·Funding≠Cost·Liability≠Expense(§4.2·§4.5·§4.6·§4.7·§8·§9·§20)**: Display/Face/Estimated/Accrued/Approved/Available/Payable/Settled/Paid/Funded/Cost/Liability/Accounting Value 분리(최대10달러≠적립7.35≠지급6.8). Face Value≠Cost Basis(무료배송 10,000원≠원가 3,200원). Funding(부담 주체)≠Cost(실제 비용). Liability(미사용 발급)≠Expense(사용/만료 시 회계 상이). **현행 Pnl COGS/fee=Cost Component 데이터원(주문 단위·Reward 단위 신설)**.

5. **★원 통화 보존·Historical FX Version(§4.3·§4.4·§13·§14·§15)**: 원 통화·환산 통화 모두 보존(덮어쓰기 금지). 모든 FX Conversion에 Source/Target·Original/Converted·Rate Source/Type/Timestamp/Version·Rounding·Evidence 기록. **과거 Reward에 당시 FX Rate Version 보존·현재 환율로 자동 재계산 금지**(현행 app_setting single current rate=version 미보존 Critical GAP). Conversion Type(Display/Reward Calc/Funding/Settlement/Payout/Accounting/Tax/Liability/Refund/Reversal/Clawback/Budget/Reporting) 분리. 미지급 Liability/Settlement FX Revaluation·Gain/Loss 별도. 지급 완료 Reward 임의 재평가 금지.

6. **Funding Party·Allocation 합계 검증·Duplicate 금지(§17·§19·§35·§37)**: Funding Party(Platform/Merchant/Brand/Seller/Partner/Marketplace/Campaign Budget)를 Canonical Identity(Part 4-5-1-3) 연결·Funding Agreement(계약 Reference·책임)·Allocation(Party별 %·Face/Payable/Paid/Cost/Liability). **Allocation 총합=100%/전체 금액 일치 검증(Over/Under 100% 차단)·동일 비용 다중 Funding 중복 배정=Critical Gap**. 현행 Reward Funding 부재(Referral platform-funded·마켓 수수료 테넌트 부담 인접).

7. **Negative Amount 의미 명시·Refund/Clawback FX 정책(§4.8·§26·§27)**: 음수=Reversal/Clawback/Refund/FX Loss/Adjustment/Correction 중 명시(부호만으로 추론 금지). Refund/Reversal/Clawback FX=Original Rate/Event Date Rate/Settlement Rate/Contract Rate 중 적용 정책 Version 관리·Transaction별 기록.

8. **정직·무후퇴·Margin/ROI Reference·Coverage/Gap**: Margin Impact(before/after·delta)·Incremental Revenue Reference·ROI Reference=**Versioned Reference Contract**(상세 Attribution Engine 아님·`Mmm`/`AttributionEngine` 연결).
