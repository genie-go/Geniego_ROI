- **근거(실측)**: [`../segmentation/CANONICAL_DSAR_MONETARY_PROVIDER_INVENTORY.md`](../segmentation/CANONICAL_DSAR_MONETARY_PROVIDER_INVENTORY.md) · [`GOVERNANCE`](../segmentation/CANONICAL_DSAR_MONETARY_PROVIDER_GOVERNANCE.md) · `channel_registry`(플랫폼 provider 카탈로그 SSOT·group_type sales/marketing/logistics/**pg**/messaging) · `channel_credential`(AES-256-GCM·마스킹·JIT 복호·**tenant=auth_tenant only** 184차) · `data_source`(source_type/channel/**account**/**credential_id**/**priority**) · `pg_settlement`(Stripe/토스페이먼츠/PayPal/이니시스/KCP/카카오페이/네이버페이/Paddle/Adyen·net/fee·**GENIE_ENV 운영/데모 물리분리**·데모 pending) · `kr_settlement_line`/`kr_fee_rule`/Rollup(마켓 수수료·net_payout·fees_source·이중차감 방지) · `ad_spend_ledger`+billing_key(Toss·광고비 월렛) · Paddle(SaaS MoR) · `influencer_store`(마케팅 데이터) · Referral/free_coupons(비현금) · 고객 cashback/rebate/wallet/store_credit·legal_entity/brand_registry/store_registry/merchant_account 테이블 **부재(grep 0)**.

## 결정 (핵심)

1. **★실측 정직·인프라 견고·엔진 부재 분리**: **Provider/Account/Credential 인프라는 실체가 견고** — Provider Registry(`channel_registry`)·Credential Binding(`channel_credential`·암호화·tenant 격리)·Provider Account(`data_source`·source_account/credential_id/priority)·PG Settlement(`pg_settlement` 9개 provider)·마켓 Commission/Settlement(`kr_settlement_line`·net_payout). 이를 **Canonical Monetary Provider Registry로 확장(중복 신설 금지·기존 SSOT 재사용)**. 그러나 **고객 대상 Cashback/Rebate/Refund Incentive/Wallet/Store Credit/Affiliate provider 엔진=부재(NOT_APPLICABLE)**·**Legal Entity/Brand/Store/Workspace/Merchant Account registry=부재(tenant_id row-level만·NOT_APPLICABLE)**. 지어내기·NO_DATA/오탐 금지·본 Inventory=멀티테넌트 고객용 미래 현금성 보상 기반 전방호환 계약.

2. **KEEP_SEPARATE 도메인 분리(오혼입 차단)**: **광고비 월렛(ad_spend_ledger+billing_key)**=광고 지출(고객 현금성 보상 아님) · **Influencer(influencer_store)**=인플루언서 마케팅 데이터(커미션/정산 아님) · **마켓 적립금/point_discount**=정산 차감(Part 4-3) · **Referral+free_coupons**=비현금 coupon(Part 4-4) · **Paddle**=GeniegoROI 자체 구독 billing(MoR). 5개를 고객 현금성 보상 provider로 매핑 금지(Static Lint/Runtime Guard).

3. **Provider Registry 단일·중복 금지(§22·§23)**: `channel_registry`=과거 4곳 하드코딩(ApiKeys/OmniChannel/AdChannelConnect/ConnectorSync) 대체 SSOT → **Canonical Provider Registry로 확장(Monetary Category/Capability 필드)**·중복 Provider/Account/Credential Registry 생성 금지. Provider별 독립 DSAR Registry/Candidate Store 금지.

4. **Account Scope·격리(§7·§10·§42)**: Account를 Tenant/Workspace/Brand/Store/Merchant/Legal Entity/Region/Country/Environment/Timezone/Currency 연결. **Provider 이름 같아도 다른 Account 통합 금지·동일 External Account ID라도 Region/Environment/Platform Account 다르면 별도 Scope**. 현행 Tenant Scope=REAL(auth_tenant)·Workspace/Brand/Store/Legal Entity Scope Node=부재(향후). **Cross-Tenant Account/Production·Sandbox 혼합=Critical Gap·Runtime Guard 차단**(현행 auth_tenant only·GENIE_ENV 물리분리=실 구현).

5. **Currency·ISO·FX(§3.4)**: Default/Settlement/Payout/Reward/Display/Accounting Currency·FX Source/Version·Precision/Rounding·Multi-currency Ledger·**ISO Currency Code(이름 문자열 금지)**. 현행 PG provider currency(net) REAL·**ISO/FX/Rounding/Multi-currency=GAP(부분)**. Currency Mapping 없어 금액 비교 불가=Critical Gap.

6. **Credential Read-only·Secret 보호(§18·§19·§27)**: **Secret 원문 코드/문서/로그/Evidence 저장 금지(Vault Reference만)**·**Discovery Credential=Read-only 원칙**(ISSUE_REWARD/MODIFY_BALANCE/EXECUTE_PAYOUT/APPROVE_SETTLEMENT/CHANGE_BANK_ACCOUNT/DETOKENIZE/DELETE_FINANCIAL_RECORD 권한 미포함). 현행 channel_credential=AES-256-GCM·마스킹·JIT 복호=Vault Reference 준함(Read-only Discovery Scope 분리 신설).

7. **Source of Truth·다중 Primary 금지(§20)**: Domain별 Primary/Secondary/Warehouse/Replica 확정·**동일 Domain 다중 Primary=Critical Gap**. 현행 실 SoT=pg_settlement(PG net)·kr_settlement_line(마켓 net_payout·실정산>kr_fee_rule estimated)·ad_spend_ledger(광고비)·Paddle(SaaS)·**data_source.source_priority=source 우선순위 실 구현**·fees_source 이중차감 방지=SoT 규율 실 사례. 고객 Cashback/Rebate/Wallet SoT=UNKNOWN/N/A(엔진 부재).

8. **정직·무후퇴·Coverage/Gap**: 고객 현금성 보상 provider 부재=결함 아니라 **`PROVIDER_LIMITATION`/NOT_APPLICABLE 명시**(NO_DATA/오탐 금지). 실 GAP=Legal Entity/Brand/Store/Merchant Mapping 부재·Currency ISO/FX/Rounding 부분·Historical/Archived/Deleted Coverage 명시 프로파일 부재·SANDBOX/TEST 환경 부분. Coverage 다차원·Critical Gap(Cross-Tenant Account·Production/Sandbox 혼합·Currency 없어 비교 불가·SoT 불명확·다중 Primary·Credential 과잉권한·Deleted 검색경로 없음·Tenant Binding 없는 Connector) 시 Access Review 차단. channel_registry/channel_credential/data_source/pg_settlement·`/v423/creds`·`/v426/channels`·`/v427/pg/*`·DataPlatform·BillingMethod API 보존(회귀 0). 기능후퇴 0.
