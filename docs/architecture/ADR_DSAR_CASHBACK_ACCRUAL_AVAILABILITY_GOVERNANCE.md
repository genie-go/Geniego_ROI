## 결정 (핵심)

1. **★실측 정직·Accrual 엔진 부재·lifecycle 실 패턴 풍부·재사용**: **Cashback Accrual/Pending/Hold/Approval/Availability 엔진은 부재(NOT_APPLICABLE·grep 0)**. 그러나 **Accrual lifecycle 실 패턴이 풍부**: **`ad_spend_ledger`**(pending→charging→committed + MTD budget cap + charging 선점 = **Pending/Hold/Budget Reservation/Idempotency 참조 정본**·outbound 광고비·KEEP_SEPARATE)·`Referral`(usable_from/referredRetained/valid_until = pending/hold/expiration)·**Idempotency**(UNIQUE)·Return/Settlement/Fraud(ReturnsPortal/pg/AnomalyDetection)·역분개(OrderHub/CRM)·Point Ledger(4-3 상태 계약). **기존 ad_spend_ledger 예약/선점 패턴·Idempotency·역분개 재사용(중복 엔진 금지)**. 지어내기·NO_DATA/오탐 금지·본 Accrual=멀티테넌트 고객용 미래 cashback 전방호환 계약.

2. **기존 예약/Idempotency/역분개 재사용·중복 금지(§50)**: **Budget Reservation=ad_spend_ledger 단일(MTD cap·charging 선점)·Idempotency=UNIQUE 정본·Return=ReturnsPortal·Fraud=AnomalyDetection·역분개=OrderHub/CRM**. Provider별 독립 Accrual Model 금지. outbound 광고비 ledger↔cashback accrual 오혼입 금지.

3. **Calculation Result≠Accrual·금액 분리(§4.1·§4.2)**: Calculation Result 존재≠Accrual(Eligibility 실패/Duplicate/Budget 부족/Limit 초과/Invalid Currency/Fraud Block/Wrong Account/Manual Review로 미생성). Estimated≠Calculated≠Approved≠Available(10,000≠8,500≠8,000≠7,500·500 환불 조정) 별도 관리. Decimal/Minor Unit·Float 금지(Part 4-5-1-4 계승).

4. **★Append-only Version·중복 Accrual 금지(§4.5·§4.8·§12·§35)**: Amount/상태 변경 시 현재값 덮어쓰기 금지→Accrual Version/Adjustment/Decision History(Append-only·Point Ledger 4-3 정합). 동일 Trigger/Rule/Subject 중복 Accrual 금지(Idempotency Key·Duplicate Group). **Duplicate 삭제 금지→Group/Winner/Excluded 보존**. **현행 정본**: pg/Referral/coupon_redemptions UNIQUE.

5. **Pending≠Hold·Approval≠Availability·Hold≠Not Eligible·Pending≠Available Balance(§4.3·§4.4·§4.6·§4.7)**: Pending(정상 대기)≠Hold(위험/검토/조건 미충족 의도적 중단). 승인≠사용 가능(Return Window/Payment Settlement/Funding Confirmation/Budget Commitment/Fraud Clearance/Scheduled Release 대기). Hold≠Not Eligible(해제 후 승인/사용 가능 전환). **Pending Cashback을 Available Balance 포함 금지**(Customer Portal 표시 O·Available 분리). **현행 인접**: ad_spend_ledger pending/charging·Referral usable_from(schedule)/referredRetained(fraud/retention)·ReturnsPortal/pg settled.

6. **Approval Governance·Availability 조건(§4.10·§19~23)**: Approval Actor≠자동 승인 Source(System/Provider/Rule Engine/Fraud/Finance/Operations/Customer Service/Manual Admin). 다단계 승인(Rule Engine→Provider→Fraud→Operations→Finance→Compliance→Final). Partial Approval·Partial Availability(Released/Remaining Pending).

7. **Refund/Cancellation Adjustment·Reservation(§4.9·§27~34)**: 부분 환불 시 전체 Accrual 무조건 취소 금지(Eligible Amount·Rule Policy 기준 부분 조정). 전액 환불=Pending Cancel/Approved Reject/Available Reversal 예약/Clawback 필요/Service Recovery 예외 구분.
