# 코드 리뷰 체크리스트

CCIS Part004 | 2026-07-22

리뷰어와 작성자가 같은 기준을 쓰기 위한 목록이다.
**전부 확인할 필요는 없다** — 변경이 닿는 영역의 항목만 본다.

---

## 0. 리뷰 이전 (작성자)

- [ ] `make quality` 통과 (신규 위반 0)
- [ ] `cd frontend && npm run build` 통과 — 빌드 성공은 **동작 보증이 아니다**
- [ ] 변경과 무관한 파일이 diff 에 섞이지 않았는가 (자동 포맷·공백 정리 유입 주의)
- [ ] 커밋 메시지에 **무엇을 왜** 바꿨는지, 검증한 것과 **검증하지 못한 것**이 적혀 있는가

---

## 1. 무엇보다 먼저 — 이 변경이 필요한가

- [ ] 같은 기능이 이미 있는가? (신설 전 `grep` 전수 확인 — 중복 기능·중복 메뉴 금지)
- [ ] 기존 것을 확장하는 대신 새로 만든 이유가 있는가
- [ ] 요구 범위를 넘어선 리팩터링이 섞이지 않았는가

---

## 2. 정확성

- [ ] 실패 경로가 실패로 보고되는가 — **`ok => true` 고정 반환 금지**
- [ ] 값을 못 구할 때 `0`/임의값이 아니라 `null` + 사유를 반환하는가
- [ ] 경계값(빈 배열·0건·null·최대치)에서 동작하는가
- [ ] 같은 값이 여러 곳에 있다면 한쪽만 고치지 않았는가 (단일 소스 확인)

---

## 3. 보안 (변경이 데이터·인증·외부입력에 닿으면 필수)

- [ ] **테넌트 격리** — 모든 조회/변경에 `tenant_id` 조건이 있는가. 조인 대상에도 있는가
- [ ] 테넌트를 요청 헤더가 아니라 **세션 토큰에서** 해석하는가
- [ ] 세션 조회에 `hashToken()` 을 썼는가 (raw 비교 금지)
- [ ] SQL 이 Prepared Statement 인가. 식별자 보간이 있다면 **코드 내 리터럴**에서만 오는가
- [ ] 권한 검사가 서버에서 이루어지는가 — UI 숨김은 보안이 아니다
- [ ] 응답·로그에 스택 트레이스·SQL·내부 경로·토큰·PII 가 없는가
- [ ] 외부 링크에 `rel="noreferrer"` 가 있는가
- [ ] 사용자 입력이 파일 경로·명령·URL 로 흘러가지 않는가 (SSRF·경로 순회)

---

## 4. 프론트엔드

- [ ] Hook 규칙 준수 — 조건부 호출 없음, 의존성 배열 정확
- [ ] `eslint-disable` 을 새로 추가하지 않았는가
- [ ] 같은 prop 을 두 번 넘기지 않았는가 (뒤엣것만 남고 앞엣것이 조용히 버려진다)
- [ ] 사용자 문구가 `t()` 를 거치는가. 신규 키를 **15개 로케일 전부**에 넣었는가
- [ ] 새 페이지를 `App.jsx` 에 `lazy()` + `<Route>` 양쪽에 등록했는가
- [ ] 청크 그룹(`manualChunks`)에 페이지를 수동 등록하지 **않았는가** (화이트스크린 재발 방지)
- [ ] 밝은 배경에 흰 글자, 세로 flex 의 우측 잘림 등 알려진 UI 트랩을 만들지 않았는가

---

## 5. 백엔드

- [ ] 새 라우트를 `routes.php` 에 등록했는가 — **핸들러만 만들면 존재하지 않는 API 다**
- [ ] 공개 엔드포인트라면 `index.php` 의 bypass 목록과 `/api` 별칭까지 반영했는가
- [ ] 반복문 안에서 쿼리·외부 API 를 호출하지 않는가 (N+1)
- [ ] 새 스키마를 마이그레이션과 `ensureTables` 중 어느 쪽에 넣을지 명시했는가
- [ ] 빈 `catch {}` 를 새로 만들지 않았는가

---

## 6. 데이터·자동화 (헌법 적용 영역)

- [ ] 미검증·목·샘플 데이터가 운영 분석 경로에 들어가지 않는가
- [ ] 자동 집행이 신뢰도·권한·동기화 상태를 확인한 뒤 이루어지는가
- [ ] AI 추천에 근거와 신뢰도가 함께 표시되는가

---

## 7. 운영 영향

- [ ] 되돌릴 방법이 있는가 (재배포 외의 롤백 경로)
- [ ] 배포 순서 의존이 있는가 (스키마 ↔ 애플리케이션)
- [ ] 대용량 변경이 락을 오래 잡지 않는가

---

## 8. 리뷰어가 하지 말아야 할 것

- 화면 캡처나 로그의 겉모습만 보고 통과시키지 않는다. **읽어서 확인**한다.
- "빌드 통과"를 동작 확인으로 받아들이지 않는다.
- 작성자가 "검증하지 못했다"고 적은 항목을 검증된 것으로 취급하지 않는다.
