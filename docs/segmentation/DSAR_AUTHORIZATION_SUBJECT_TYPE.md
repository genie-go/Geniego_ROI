## Subject Type (21)
USER · CUSTOMER_ADMIN · TENANT_ADMIN · WORKSPACE_ADMIN · ORGANIZATION_MEMBER · EMPLOYEE · CONTRACTOR · PARTNER_USER · VENDOR_USER · MERCHANT_USER · AUDITOR · SUPPORT_AGENT · **SERVICE_ACCOUNT** · **MACHINE_IDENTITY** · **API_CLIENT** · OAUTH_CLIENT · WORKLOAD_IDENTITY · AUTOMATION_AGENT · SYSTEM_PROCESS · EMERGENCY_OPERATOR · OTHER

## 실측 매핑
REAL = USER · TENANT_ADMIN(app_user) · **API_CLIENT**(api_key) · PARTNER_USER(partner_session) · ORGANIZATION_MEMBER(agency 토큰).
부재 = SERVICE_ACCOUNT · MACHINE_IDENTITY · WORKLOAD_IDENTITY · **EMERGENCY_OPERATOR**(Break Glass 부재·5-5).

## 규칙
**Service Account·Machine Identity 는 인간 User 와 별도 Subject Type**(§37) · **Human Login 금지 · Impersonation 금지**. EMERGENCY_OPERATOR 는 **5-5 전용 정책으로만** 활성.
