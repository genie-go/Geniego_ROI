## §0 작업 목적

검증된 Actor Identity Assurance & Authentication Binding 위에 **Authorization Registry Foundation**을 구축한다. 이후 모든 Authorization 기능(Permission·RBAC·ABAC·Authority·SoD·COI·Dual-Control)이 공통 사용할 Canonical Authorization Foundation을 세워, 다음에 일관·재현·감사 가능하게 답한다: 현재 Actor가 해당 Action 수행 가능한가·어떤 Definition/Version/Policy가 적용됐나·SoT는 무엇인가·Allow vs Deny 우선순위·Explicit/Default Deny·입력 Context·Identity/Authentication Snapshot 포함·Decision Slot/Case/Resource/Amount/Currency 결합·Validation↔Commit 사이 변경·유효기간·재사용 범위·다른 Resource/Action 재사용 금지·Policy Version 변경 시 무효화·Cache Version/Context·Ledger 불변저장·Engine 장애 시 Fail Open/Closed·고객설정 비활성 불가·Legacy Role/Permission/Authority Check 통합·이후 Part 확장가능.

**이번 Part 미구현(후속)**: 전체 Permission Engine·RBAC·ABAC·Approval Authority·SoD·COI·Dual Control 상세 Rule. 대신 이후 모든 Authorization 기능의 공통 기반(Registry~Migration Foundation)만 구축.

## §5 핵심 원칙

- **§5.1** Authentication ≠ Authorization(인증성공만으로 Action 허용 금지·Identity 결과를 Subject Input으로).
- **§5.2** **Default Deny**(Policy/Registry/Version 누락·Context 불완전·Engine 장애·Evaluation 오류 시 고위험 Approval 허용 금지).
- **§5.3** **Explicit Deny 우선**(Deny/Permit Override는 Policy Set 수준·Versioned·명시적).
- **§5.4** UI Control ≠ Authorization(버튼숨김/메뉴비활성만으로 구현 금지·모든 중요 Action Server-side 재검증).
- **§5.5** Actor ID 직접신뢰 금지(Body/Query/Header/Client State→Canonical Principal·Effective Actor 사용).
- **§5.6** Resource·Action 명시("접근가능" 모호결과 금지 — Subject/Tenant/Legal Entity/Org Scope/Resource/Version/Action/Context/유효기간).
- **§5.7** Policy Versioned(과거 Decision 현재 Policy 재해석 금지).
- **§5.8** Decision=Immutable Snapshot(현재 Role/Permission/Policy로 과거 Decision 덮어쓰기 금지).
- **§5.9** Allow도 Evidence(Deny만 기록 금지·어떤 Policy/Context로 허용됐는지).
- **§5.10** Validation↔Commit 분리(Validation Allow도 Commit 직전 재검증 Binding/Version 보존).
- **§5.11** Context Reuse 제한(Tenant/Legal Entity/Org/Subject/Effective Actor/Resource/Version/Action/Slot/Case Version/Amount/Currency/Policy Version/Session/Environment 하나라도 다르면 재사용 금지).
- **§5.12** Multi-tenant 격리 강제(Cross-Tenant Role/Permission/Policy/Cache/Decision 재사용 차단).
- **§5.13** **Fail Open 금지**(Engine 장애/Timeout/Cache/Parsing Error/External Failure 시 고위험 허용 금지).
- **§5.14** Override ≠ 일반 Allow(Emergency/Administrative/Break-glass는 별도 Policy/Reason/Approval/Expiration/Evidence/Audit).
- **§5.15** 고객설정 비활성 불가: Default Deny·Tenant/Subject/Resource/Action Binding·Policy Version Recording·Decision Snapshot·Evidence·Commit Binding·Explicit Deny/Fail-closed Enforcement·Cross-tenant Isolation·Raw Policy Bypass Prohibition·Server-side Authorization·Audit Recording.
