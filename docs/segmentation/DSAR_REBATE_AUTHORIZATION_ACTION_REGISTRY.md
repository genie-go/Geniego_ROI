## ★핵심 규칙
- **APPROVE 와 EXECUTE 는 반드시 별개 Action**(APPROVE_PAYOUT ≠ EXECUTE_PAYOUT · APPROVE_SETTLEMENT ≠ EXECUTE_SETTLEMENT) — **동일 Subject 결합 = Critical Gap**(§43). 현행 8동작에 approve/execute 가 **이미 분리**(TeamPermissions.php:39)=재사용 근거.
- IMPERSONATE = **5-4** · BREAK_GLASS = **5-5**(본 블록은 Action 등록만·구현 금지).
- **USE_PROVIDER_CREDENTIAL 은 Secret 조회가 아니다** — Tokenized Runtime Reference 사용(§39).
