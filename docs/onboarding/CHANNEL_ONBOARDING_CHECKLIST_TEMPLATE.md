# 채널별 온보딩 체크리스트 & 테스트 플로우 (템플릿)

> 목적: 채널 API는 국가/계정/권한/버전 차이가 크므로, 도입 시 **반드시 검증**해야 하는 항목을
> 표준화합니다.

## 공통 사전 준비
- [ ] 테넌트 생성/사용자 생성(ADMIN, MANAGER, OPERATOR)
- [ ] 거버넌스 정책 선택(국가/채널 템플릿) 및 retention/masking 확인
- [ ] 샌드박스/테스트 계정 준비(가능한 채널)
- [ ] 레이트리밋 정책 확인(채널별)

## A. Shopee (국가/버전 분기)
### 연결(수집)
- [ ] OAuth/토큰 발급 정상
- [ ] 상품 목록 수집(최소 10개)
- [ ] 주문/재고/가격 스냅샷 수집

### 쓰기(등록/수정/피드)
- [ ] write_endpoints 템플릿 선택(예: SG 2.1)
- [ ] `product_upsert` 드라이런(dry-run) 성공
- [ ] 실 실행(승인→실행) 후 결과 확인
- [ ] 피드 제출 → 상태 조회 → 실패 사유 수집

## B. Qoo10 (JP 중심)
### 연결(수집)
- [ ] 인증/키 세팅
- [ ] 상품/주문/클레임 수집

### 쓰기
- [ ] 피드 제출/상태 조회
- [ ] 대표 오류 케이스(이미지, 필수 필드 누락) 재현 및 콘솔 Top에 노출 확인

## C. Rakuten (JP)
### 연결(수집)
- [ ] 인증/서명
- [ ] 상품/주문 수집

### 쓰기
- [ ] 상품 업데이트
- [ ] 피드 처리 결과 수집

## 테스트 플로우 (운영형)
1) 제안 생성(Propose)
2) 승인(Manager → Legal)
3) 실행(Worker)
4) 실패 유도(의도적 오류) → 롤백 발생 확인
5) 감사로그/콘솔에서 이벤트 확인
