# GeniegoROI 작업 프로세스

## 📋 개요

이 문서는 GeniegoROI 프로젝트의 표준 작업 프로세스를 정의합니다. 모든 에이전트는 이 프로세스를 따라 일관성 있고 효율적으로 작업을 수행합니다.

---

## 🐛 버그 수정 프로세스

### 1단계: 버그 발견 및 보고

**담당**: QA 에이전트 또는 사용자

**절차**:
1. 버그 재현 단계 작성
2. 예상 동작 vs 실제 동작 기록
3. 스크린샷/로그 첨부
4. 환경 정보 기록 (브라우저, OS, 버전)
5. `docs/BUGS_TRACKING.md`에 등록

**버그 리포트 템플릿**:
```markdown
## 버그 제목
[간단명료한 버그 설명]

### 심각도
- [ ] Critical (서비스 중단)
- [ ] High (주요 기능 장애)
- [ ] Medium (부분 기능 장애)
- [ ] Low (UI 오류, 사소한 문제)

### 재현 단계
1. [단계 1]
2. [단계 2]
3. [단계 3]

### 예상 동작
[정상적으로 작동해야 하는 방식]

### 실제 동작
[현재 발생하는 문제]

### 환경
- 브라우저: Chrome 120
- OS: Windows 11
- 버전: v418

### 스크린샷/로그
[첨부 파일]
```

---

### 2단계: 버그 분석 및 우선순위 결정

**담당**: PM 에이전트

**절차**:
1. 버그 심각도 평가 (Critical/High/Medium/Low)
2. 영향 범위 분석 (사용자 수, 기능 중요도)
3. 우선순위 결정 (P0/P1/P2/P3/P4)
4. 담당 에이전트 할당
5. 예상 수정 시간 산정

**우선순위 매트릭스**:
| 심각도 | 영향도 | 우선순위 | 대응 시간 |
|--------|--------|----------|-----------|
| Critical | High | P0 | 즉시 (1시간 이내) |
| Critical | Medium | P1 | 24시간 이내 |
| High | High | P1 | 24시간 이내 |
| High | Medium | P2 | 1주일 이내 |
| Medium | Any | P3 | 2주일 이내 |
| Low | Any | P4 | 백로그 |

### 3단계: 원인 분석

**담당**: 해당 영역 에이전트 (프론트엔드/백엔드)

**절차**:
1. **파일 읽기**: 관련 파일의 현재 코드 확인

3. **재현**: 로컬 환경에서 버그 재현
4. **근본 원인 파악**: 버그의 실제 원인 식별
5. **영향 범위 확인**: 다른 기능에 미치는 영향 분석

**분석 체크리스트**:
- [ ] 버그 재현 성공
- [ ] 근본 원인 파악 완료
- [ ] 영향받는 파일 목록 작성
- [ ] 관련 API/컴포넌트 확인
- [ ] 데이터베이스 영향 확인

**담당**: 해당 영역 에이전트

- [단점 1]

### 5단계: 수정 실행

**담당**: 해당 영역 에이전트

**절차**:
1. **브랜치 생성**: `fix/버그명` 형식
2. **코드 수정**: 승인된 방법으로 수정
3. **주석 추가**: 수정 이유 및 변경 사항 설명
4. **로컬 테스트**: 수정 사항 로컬에서 검증
5. **커밋**: 명확한 커밋 메시지 작성

**커밋 메시지 형식**:
```
fix: [버그 간단 설명]

- 원인: [버그 원인]
- 수정: [수정 내용]
- 영향: [영향받는 기능]

Closes #이슈번호
```

## 🔄 Git 워크플로우

### 브랜치 전략

```
main (운영)
  ├── develop (개발)
  │   ├── feature/기능명
  │   ├── fix/버그명
  │   └── refactor/리팩토링명
  └── hotfix/긴급수정명
```

### 커밋 메시지 규칙

**형식**:
```
<타입>: <제목>

<본문>

<푸터>
```

**타입**:
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅
- `refactor`: 코드 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드/설정 변경

**예시**:
```
feat: 사용자 프로필 페이지 추가

- 프로필 정보 표시 컴포넌트 구현
- 프로필 수정 기능 추가
- i18n 15개 언어 지원

Closes #123
```

### Git 작업 순서

1. **작업 시작**:
```bash
git checkout develop
git pull origin develop
git checkout -b feature/기능명
```

2. **작업 중**:
```bash
git add .
git commit -m "feat: 작업 내용"
git push origin feature/기능명
```

3. **PR 생성**:
- GitHub에서 Pull Request 생성
- 리뷰어 지정 (QA 에이전트)
- 라벨 추가

4. **머지 후**:
```bash
git checkout develop
git pull origin develop
git branch -d feature/기능명
```
