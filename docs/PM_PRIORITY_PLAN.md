### 2. Journey Builder 현황

#### 코드 분석 결과 (frontend/src/pages/JourneyBuilder.jsx)
- **총 라인 수**: 729줄
- **KPI 카드 위치**: 443줄 (Builder 탭)
- **KPI 카드 구조**:
  ```javascript
  <div style={{ display: 'grid', gridTemplateColumns: 'repeat(4, 1fr)', gap: 12 }}>
    {[
      { label: tr(K.totalJourneys), value: stats.total, icon: '🗺️', color: '#4f8ef7' },
      { label: tr(K.activeJourneys), value: stats.active, icon: '🟢', color: '#22c55e' },
      { label: tr(K.totalExecutions), value: stats.totalExec, icon: '🚀', color: '#a855f7' },
      { label: tr(K.avgCompletion), value: stats.avgRate + '%', icon: '📈', color: '#f97316' }
    ].map(({ label, value, icon, color }) => (
      <div key={label} style={{ ...CARD, display: 'flex', alignItems: 'center', gap: 14 }}>
        <div style={{ width: 40, height: 40, borderRadius: 12, background: `${color}12`, display: 'flex', alignItems: 'center', justifyContent: 'center', fontSize: 20 }}>{icon}</div>
        <div>
          <div style={{ fontSize: 11, fontWeight: 700, color: '#94a3b8', letterSpacing: '0.03em' }}>{label}</div>
          <div style={{ fontSize: 26, fontWeight: 900, color, marginTop: 2 }}>{value}</div>
        </div>
      </div>
    ))}
  </div>
  ```

#### 발견된 문제점
1. **CSS 클래스 미사용**: `kpi-card` 클래스가 정의되어 있으나 Journey Builder에서는 인라인 스타일만 사용
2. **반응형 미흡**: `gridTemplateColumns: 'repeat(4, 1fr)'`로 고정되어 모바일에서 레이아웃 깨짐 가능성
3. **일관성 부족**: 다른 페이지들은 `className="kpi-card"`를 사용하는데 Journey Builder만 인라인 스타일 사용
4. **테마 지원 미흡**: 라이트/다크 모드 전환 시 색상 대응 부족

### 3. UI/UX 개선 필요 항목

#### 소비자(최종 사용자) 관점 문제점
1. **복잡한 네비게이션**: 100+ 페이지, 다층 메뉴 구조
2. **전문 용어 과다**: 마케팅 전문가가 아닌 일반 사용자에게 어려움
3. **온보딩 부족**: 처음 사용자를 위한 가이드 부족
4. **모바일 최적화 미흡**: 일부 페이지에서 모바일 반응형 문제
5. **피드백 부족**: 작업 완료/실패 시 사용자 알림 불충분

#### 발견된 구체적 사례
- Journey Builder: 가이드 탭은 있으나 첫 방문 시 자동 표시 안 됨
- CRM: 고급 세그먼트 기능이 초보자에게 복잡함
- Email Marketing: 템플릿 선택 UI가 직관적이지 않음

### 4. 마케팅 대행사용 기능 현황

#### V382 기능명세서 분석 결과
- **대행사 권한 템플릿 정의됨** (docs/V382_FUNCTIONAL_SPEC_KO.md):
  - Editor: 광고 캠페인 초안 생성 (publish 불가)
  - Manager/Legal: 광고/소재/타게팅 승인
  - Admin: 광고 계정 연동은 고객 Admin만 가능

#### 현재 구현 상태
- ✅ **RBAC 기본 구조**: 역할 기반 권한 시스템 존재
- ✅ **승인 워크플로**: Approvals 페이지 구현됨
- ⚠️ **대행사 전용 기능 부족**:
  - 클라이언트별 작업 공간 분리 미흡
  - 대행사 성과 리포트 템플릿 없음
  - 클라이언트 승인 대기 대시보드 부족
  - 화이트라벨 기능 없음

### 우선순위 1: Journey Builder KPI 카드 버그 수정 (긴급)

**심각도**: High  
**예상 작업 시간**: 3시간  
**담당**: 프론트엔드 에이전트

#### 문제점
1. CSS 클래스 미사용으로 테마 전환 시 스타일 깨짐
2. 반응형 레이아웃 미흡 (모바일에서 4열 고정)
3. 다른 페이지와 스타일 일관성 부족

#### 해결 방안
1. **CSS 클래스 적용**:
   - 인라인 스타일을 `className="kpi-card"` 클래스로 교체
   - `styles.css`에 정의된 `.kpi-card` 스타일 활용
   - 테마별 색상 자동 적용 (arctic_white, pearl_office 등)

2. **반응형 개선**:
   ```javascript
   // Before
   gridTemplateColumns: 'repeat(4, 1fr)'
   
   // After
   gridTemplateColumns: 'repeat(auto-fit, minmax(180px, 1fr))'
   ```

3. **일관성 확보**:
   - 다른 페이지(BudgetTracker, PerformanceHub 등)와 동일한 KPI 카드 구조 사용
   - 공통 컴포넌트화 검토 (`components/KPICard.jsx` 활용)

#### 작업 단계
1. ✅ 문제 분석 완료
2. [ ] `JourneyBuilder.jsx` 443줄 KPI 카드 수정
3. [ ] 반응형 테스트 (모바일, 태블릿, 데스크톱)
4. [ ] 테마 전환 테스트 (라이트/다크 모드)
5. [ ] 다른 KPI 카드와 일관성 검증
6. [ ] Git commit & push

### 우선순위 2: 소비자 관점 UI/UX 개선 (중요)

**심각도**: Medium  
**예상 작업 시간**: 12시간  
**담당**: 프론트엔드 에이전트 + PM 에이전트

#### 개선 항목

##### 2.1 온보딩 개선 (4시간)
**목표**: 신규 사용자가 5분 내에 핵심 기능 이해

**작업 내용**:
1. **첫 방문 감지 및 가이드 표시**:
   - localStorage로 첫 방문 여부 확인
   - Journey Builder 첫 방문 시 가이드 탭 자동 표시
   - 주요 페이지별 "처음이신가요?" 배너 추가

2. **인터랙티브 튜토리얼**:
   - Dashboard: 3단계 간단 가이드 (연결 → 데이터 확인 → 액션)
   - CRM: 세그먼트 생성 데모
   - Email Marketing: 첫 캠페인 생성 가이드

3. **빠른 시작 체크리스트**:
   ```
   ✅ 1. 채널 연결하기 (Connectors)
   ⬜ 2. 첫 고객 데이터 가져오기 (CRM)
   ⬜ 3. 이메일 캠페인 만들기 (Email Marketing)
   ⬜ 4. 고객 여정 설계하기 (Journey Builder)
   ```

##### 2.2 용어 단순화 (3시간)
**목표**: 전문 용어를 일반 사용자가 이해할 수 있는 표현으로 변경

**작업 내용**:
1. **i18n 키 추가** (15개 언어):
   - "Attribution" → "광고 효과 분석" (어떤 광고가 매출에 기여했나요?)
   - "Cohort Analysis" → "고객 그룹 분석" (같은 시기 가입한 고객들의 행동)
   - "LTV" → "고객 생애 가치" (한 고객이 평생 얼마나 구매할까요?)
   - "ROAS" → "광고 수익률" (광고비 대비 매출)

2. **툴팁 추가**:
   - 전문 용어 옆에 "?" 아이콘
   - 호버 시 쉬운 설명 표시

3. **예시 데이터 개선**:
   - 실제 비즈니스 시나리오 기반 샘플 데이터
   - "테스트 캠페인" → "신규 가입 환영 이메일"

##### 2.3 모바일 최적화 (3시간)
**목표**: 모바일에서 핵심 기능 사용 가능

**작업 내용**:
1. **반응형 테이블**:
   - 가로 스크롤 대신 카드 레이아웃으로 전환
   - 중요 컬럼만 모바일에 표시

2. **터치 최적화**:
   - 버튼 최소 크기 44x44px
   - 스와이프 제스처 지원 (탭 전환)

3. **모바일 네비게이션**:
   - 하단 탭바 개선 (MobileBottomNav.jsx)
   - 자주 쓰는 기능 빠른 접근

##### 2.4 피드백 강화 (2시간)
**목표**: 사용자 액션에 즉각적인 피드백 제공

**작업 내용**:
1. **Toast 알림 표준화**:
   - 성공: 녹색, 3초 자동 닫힘
   - 오류: 빨간색, 5초 또는 수동 닫기
   - 정보: 파란색, 3초

2. **로딩 상태 개선**:
   - 스켈레톤 UI (빈 화면 대신)
   - 진행률 표시 (데이터 동기화 등)

3. **에러 메시지 개선**:
   - 기술적 오류 → 사용자 친화적 메시지
   - 해결 방법 제시
