# GENIE_ROI 보안/심사 문서 패키지 (템플릿)

> 목적: 국내/글로벌(특히 엔터프라이즈) 도입 과정에서 요구되는 보안/컴플라이언스 심사 자료를
> 빠르게 제출할 수 있도록 만드는 **표준 문서 패키지 템플릿**입니다.

## 1. 제품 개요 (Security Overview)
- 서비스명/버전: GENIE_ROI V371
- 배포 형태: SaaS / Self-hosted(옵션)
- 주요 데이터 흐름: 채널 API → 수집/정규화 → 분석 → 승인/실행 자동화 → 감사로그
- 테넌시: Tenant 기반 논리 분리 + RBAC

## 2. 아키텍처 개요
- API: FastAPI
- Worker: RQ
- DB: PostgreSQL
- Object storage(옵션): S3 호환

### 2.1 네트워크 구성
- (예시) API/Worker/DB 분리, Security Group 규칙

## 3. 인증/권한 (IAM)
- 인증: 이메일/비밀번호 + JWT
- 권한: role 기반(예: ADMIN/MANAGER/SELLER/VIEWER)
- 중요 작업: 승인(4-eyes) 후에만 mutation 실행

### 3.1 RBAC 매트릭스(요약)
- 사용자/테넌트 관리
- 채널 연결/해제
- 데이터 조회/다운로드
- mutation 제안/승인/실행

## 4. 데이터 보호
- 전송구간: TLS 권장
- 저장구간: DB 암호화(옵션), 비밀키는 환경변수/시크릿 매니저
- 민감정보 처리: PII detector + 정책 엔진으로 마스킹/드랍/차단

## 5. 로그/감사
- 감사로그(AuditLog): 승인/실행/롤백/정책 위반 차단 기록
- 보관기간: 국가/채널별 템플릿 기반

## 6. 취약점/보안 운영
- 정기 점검: 의존성 취약점 스캔(SCA)
- 비밀정보 관리: .env 금지(운영), Secret Manager 권장
- 침해대응(IR): 사고 인지 → 격리 → 조사 → 통지 → 재발방지

## 7. 개인정보/규정 준수
- GDPR/CCPA/KR PIPA 대비: 최소수집/목적 제한/보관기간/접근통제
- 데이터 주체 요청(DSR) 처리 프로세스
