Hướng dẫn cấp API

Hướng dẫn thân thiện với người mới về cách tạo ứng dụng trên Meta for Developers và chuẩn bị App ID, App Secret cùng mã thông báo truy cập trang (Page Access Token) cần thiết để kết nối phát trực tiếp của trang Facebook.

1. Những điều cốt lõi cần biết trước

YouTube chỉ cần một API Key là xong, nhưng Facebook (Meta) hoạt động với cấu trúc App ID · App Secret · Access Token.

Quan trọng: Với Meta, việc "sao chép một API Key" là chưa đủ. Thông thường bạn cần theo thứ tự sau: tạo ứng dụng → thêm sản phẩm → thiết lập quyền → cấp Page Access Token → xét duyệt ứng dụng.

2. Những thứ cần chuẩn bị

  • Tài khoản Meta/Facebook
  • Quyền truy cập Meta for Developers
  • Một trang Facebook mà bạn có quyền quản trị
  • URL chính sách quyền riêng tư · URL điều khoản dịch vụ cho dịch vụ thực tế

3. Tạo ứng dụng Meta Developer

Mọi tích hợp Meta API đều bắt đầu từ việc tạo ứng dụng.

1

Truy cập Meta for Developers

Đăng nhập vào Meta for Developers trên trình duyệt của bạn.

developers.facebook.com
2

Ứng dụng của tôi → Tạo ứng dụng

Nhấp vào My Apps → Create App ở phía trên.

My Apps → Create App
3

Chọn loại ứng dụng

Tích hợp nền tảng/thương mại trực tiếp gần với mục đích doanh nghiệp hơn.

Chọn loại Business
4

Nhập tên ứng dụng·email → tạo

Nhập tên ứng dụng và email liên hệ, sau đó tạo ứng dụng.

App Name / Contact Email
Sau khi tạo ứng dụng, hãy kiểm tra App ID và App Secret tại Bảng điều khiển → Cài đặt ứng dụng → Cài đặt cơ bản. (Tuyệt đối không để lộ App Secret ở giao diện người dùng/kho lưu trữ công khai.)

4. Quy trình thiết lập Facebook Live API

1

Thêm sản phẩm

Trên bảng điều khiển ứng dụng, chuẩn bị các cài đặt liên quan đến Facebook Login / Graph API.

Add Product → Facebook Login
2

Kết nối trang + mã thông báo trang

Để phát trực tiếp lên một trang, bạn cần quyền quản trị viên trang và Page Access Token.

Page Access Token
3

Đăng ký các quyền cần thiết (App Review)

Đăng ký các quyền nhằm mục đích đăng nội dung·quản lý trang·tạo video trực tiếp (ví dụ: pages_read_engagement, pages_manage_posts).

Permissions → App Review
4

Cấp·kiểm thử mã thông báo bằng Graph API Explorer

Cấp mã thông báo có chứa các quyền và kiểm thử các lệnh gọi API.

Tools → Graph API Explorer
Ví dụ lệnh gọi tạo Facebook Live
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
Khi tạo thành công, thông tin phát trực tiếp như ID phát sóng·URL luồng·khóa luồng sẽ được trả về. Xem ID trang (page_id) trong thông tin trang.

5. Đăng ký trong GenieGo Integration Hub

1

Sao chép Page Access Token

Sao chép Page Access Token bạn đã cấp (khuyến nghị dùng mã thông báo dài hạn).

access_token (Page token)
2

Xác nhận ID trang

Xác nhận ID (page_id) của trang Facebook mục tiêu.

page_id
3

Đăng ký trong Integration Hub

Trong GenieGo Integration Hub → Facebook Live → [Đăng ký], nhập các giá trị bên dưới rồi lưu.

access_token · page_id
Các khóa đã đăng ký được mã hóa khi lưu trữ bằng AES-256-GCM và được che khi đọc.

6. Các loại Access Token

Loại mã thông báoCông dụngLưu ý
User Access TokenLệnh gọi API dựa trên đăng nhập của người dùngDữ liệu có thể truy cập thay đổi tùy theo phạm vi quyền được cấp.
Page Access TokenĐăng trang · tạo phát trực tiếp của trangCần quyền quản trị viên trang và các quyền liên quan.
Long-Lived TokenMã thông báo dài hạn có thời hạn kéo dàiKhi vận hành cần có logic hết hạn·làm mới.

7. Danh sách kiểm tra

  • Đã tạo ứng dụng Meta Developer.
  • Đã xác nhận App ID và App Secret.
  • Đã kết nối trang Facebook và cấp Page Access Token.
  • Đã sắp xếp·đăng ký các quyền cần thiết (App Review).
  • Đã nhập access_token·page_id vào GenieGo [Đăng ký] và lưu.
  • Đã lưu trữ mã thông báo an toàn trong biến môi trường máy chủ.
