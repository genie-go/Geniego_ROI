Hướng dẫn cấp API Key

Hướng dẫn từng bước thân thiện với người mới về việc cấp khóa API Billing (v2) (Bearer, pdl_live_…) từ Developer Tools trong Paddle Dashboard của bạn và đăng ký trong GenieGo Integration Hub.

1. Trước khi bắt đầu

Paddle là một nền tảng thanh toán Merchant of Record dành cho SaaS và sản phẩm số (Billing v2).

Bắt buộc

Tài khoản Paddle

Việc bán SaaS và sản phẩm số yêu cầu một tài khoản Paddle (Billing v2).

Khóa được cấp

Khóa API Billing

Bạn tạo một khóa API Billing định dạng Bearer (pdl_live_…). Đây không phải là phương thức vendor_id/auth_code Classic cũ.

Lưu ý: Khóa cần cấp là khóa API Billing (v2) (Bearer, pdl_live_…). Đừng nhầm lẫn nó với phương thức vendor_id/auth_code Classic cũ.

2. Các bước cấp

Cấp theo thứ tự dưới đây và đăng ký trong GenieGo.

1

Đăng nhập Paddle Dashboard

Đăng nhập Dashboard bằng tài khoản Paddle đã chuẩn bị.

https://vendors.paddle.com
2

Vào menu API keys

Vào [Developer Tools] > [Authentication] > [API keys].

Developer Tools → Authentication → API keys
3

Tạo khóa API Billing

Tạo khóa API Billing (Bearer, pdl_live_…). Đây không phải là phương thức vendor_id/auth_code Classic cũ.

Khi khóa API Billing bắt đầu bằng pdl_live_… được tạo, nghĩa là nó đã được cấp đúng cách.

3. Thông tin cần đăng ký trong GenieGo

TrườngMô tả
api_keyKhóa API Billing (v2) đã được cấp (Bearer, pdl_live_…, xử lý cẩn thận)
Nhập api_key vào GenieGo Integration Hub [Đăng ký] rồi lưu. Thông tin xác thực được mã hóa khi lưu trữ bằng AES-256-GCM và được che khi đọc.

  • Đã vào Developer Tools > Authentication > API keys.
