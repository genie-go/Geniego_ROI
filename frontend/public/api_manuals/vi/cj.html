1. Trước khi bắt đầu

API chuyên dụng của CJ Logistics là một kênh dựa trên hợp đồng và không được cấp ngay từ bảng điều khiển tự phục vụ.

Bắt buộc

Mã hợp đồng (khách hàng) CJ Logistics

Bạn phải có mã hợp đồng (khách hàng) được cấp thông qua hợp đồng với CJ Logistics. Mã này không được cấp trực tiếp từ một bảng điều khiển riêng.

Khuyến nghị

Smart Parcel (theo dõi hợp nhất)

Nếu bạn chỉ cần theo dõi giao hàng, bạn có thể kết nối tất cả các hãng vận chuyển bằng một t_key duy nhất thông qua kênh [Smart Parcel (theo dõi hợp nhất)].

Lưu ý: Nếu mục tiêu của bạn là theo dõi giao hàng, chúng tôi khuyên bạn nên kết nối qua khóa hợp nhất Smart Parcel (SweetTracker) mà không cần hợp đồng API chuyên dụng.

2. Quy trình nộp đơn cấp

Nộp đơn và đăng ký theo thứ tự dưới đây.

1

Chuẩn bị mã hợp đồng (khách hàng)

Chuẩn bị mã hợp đồng (khách hàng) CJ Logistics của bạn. API chuyên dụng được cung cấp trên cơ sở hợp đồng.

2

Quyết định xem bạn có cần theo dõi hợp nhất không

Nếu bạn chỉ cần theo dõi giao hàng, bạn có thể kết nối tất cả các hãng vận chuyển bằng một t_key duy nhất thông qua kênh [Smart Parcel (theo dõi hợp nhất)] thay vì kênh chuyên dụng của CJ Logistics.

3

Nộp đơn xin API chuyên dụng

Nếu bạn cần API chuyên dụng của CJ Logistics, hãy nhận mã hợp đồng (khách hàng) và khóa API từ đại diện kinh doanh của bạn.

4

Đăng ký trong GenieGo

Nhập api_key và cust_code vào [Đăng ký] của GenieGo Integration Hub rồi lưu.

Khi bạn nhận được khóa API và mã hợp đồng (khách hàng) từ đại diện kinh doanh, nghĩa là bạn đã sẵn sàng để đăng ký.

Danh sách kiểm tra cuối cùng

  • Đã có mã hợp đồng (khách hàng) CJ Logistics.
  • Đã xem xét tích hợp Smart Parcel (theo dõi hợp nhất) nếu chỉ cần theo dõi giao hàng.
  • Đã nhận khóa API và mã hợp đồng (khách hàng) từ đại diện kinh doanh.
  • Đã nhập api_key·cust_code vào [Đăng ký] của GenieGo và lưu.
