L Hướng dẫn cấp API cho người mới

LINE
Hướng dẫn cấp API

Hướng dẫn này tổng hợp quá trình tạo Provider và Messaging API Channel trong LINE Developers Console và cấu hình Channel Access Token, Webhook URL.

Cấu trúc xác thựcChannel Access Token
Tài khoản cần thiếtTài khoản LINE
Kết quả chínhChannel ID / Secret / Token
Mục đíchNhắn tin·Chatbot

1. Khái niệm cấp LINE API

Tích hợp LINE hoạt động bằng cách tạo một Provider và một Channel trong LINE Developers Console, sau đó cấp Channel ID, Channel Secret và Channel Access Token cho Messaging API hoặc LINE Login.

Phân biệt: Việc gửi tin nhắn dùng kênh Messaging API, còn tích hợp đăng nhập dùng kênh LINE Login.

2. Chuẩn bị

  • Tài khoản LINE hoặc tài khoản doanh nghiệp
  • Quyền truy cập LINE Developers Console
  • LINE Official Account
  • Tên dịch vụ và mô tả
  • Webhook URL
  • URL chính sách bảo mật và URL điều khoản

3. Tạo Provider

1

Truy cập LINE Developers

Đăng nhập vào LINE Developers Console.

developers.line.biz
2

Tạo Provider

Tạo một Provider bằng tên công ty hoặc tên dịch vụ.

Create Provider
3

Nhập thông tin cơ bản

Nhập chủ thể vận hành dịch vụ, thông tin liên hệ và thông tin điều khoản.

Provider Settings

4. Tạo kênh Messaging API

1

Tạo kênh mới

Nhấp Create a new channel bên trong Provider.

Create a new channel
2

Chọn Messaging API

Chọn Messaging API làm loại kênh.

Messaging API
3

Nhập thông tin kênh

Nhập tên kênh, mô tả, danh mục, email, v.v.

Channel Information
4

Liên kết LINE Official Account

Liên kết tài khoản chính thức để gửi tin nhắn, hoặc tạo mới.

LINE Official Account
5

Hoàn tất tạo kênh

Sau khi tạo, kiểm tra Channel ID và Channel Secret.

Channel ID / Channel Secret

5. Cấp Channel Access Token

1

Chuyển đến tab Messaging API

Trên màn hình kênh đã tạo, chuyển đến tab Messaging API.

Messaging API tab
2

Tìm mục Channel access token

Kiểm tra khu vực Channel access token ở phía dưới.

Channel access token
3

Nhấp Issue

Nhấp nút Issue hoặc Reissue để cấp mã thông báo.

Issue
4

Lưu trữ an toàn trên máy chủ

Lưu mã thông báo đã cấp vào biến môi trường máy chủ hoặc Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token là khóa bí mật có quyền gửi tin nhắn. Không được để lộ trong HTML, frontend hoặc kho lưu trữ công khai.

6. Cấu hình Webhook

1

Nhập Webhook URL

Nhập URL máy chủ sẽ nhận các sự kiện tin nhắn của người dùng.

Webhook URL
2

Bật Use webhook

Đổi tùy chọn Use webhook thành Enabled.

Use webhook: Enabled
3

Kiểm thử Verify

Gửi yêu cầu kiểm thử từ LINE đến máy chủ của bạn để xác nhận kết nối.

Verify
4

Kiểm tra mã phản hồi

Xác nhận máy chủ trả về 200 OK.

HTTP 200 OK

7. Ví dụ gửi tin nhắn

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Xin chào. Đây là bài kiểm thử tin nhắn LINE."} ] }

8. Danh sách kiểm tra cuối cùng

  • Đã tạo Provider.
  • Đã tạo kênh Messaging API.
  • Đã liên kết LINE Official Account.
  • Đã kiểm tra Channel ID và Channel Secret.
  • Đã cấp Channel Access Token.
  • Đã đăng ký Webhook URL và vượt qua Verify.
  • Đã lưu mã thông báo an toàn trong biến môi trường máy chủ.
