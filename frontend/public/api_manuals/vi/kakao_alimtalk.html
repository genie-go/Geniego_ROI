1. Khái niệm cấp Kakao Alimtalk

Alimtalk là dịch vụ tin nhắn doanh nghiệp gửi các tin nhắn thông tin như đơn hàng, đặt chỗ, vận chuyển và thanh toán qua Kakao Talk. Đây không phải là cấu trúc có thể gửi trực tiếp chỉ với khóa ứng dụng nhà phát triển Kakao thông thường, mà cần xác minh kênh doanh nghiệp và thiết lập đại lý gửi/nền tảng.

Quan trọng: Alimtalk tập trung vào tin nhắn thông tin chứ không phải tin nhắn quảng cáo, và các mẫu cần được xét duyệt trước.

2. Những thứ cần chuẩn bị

  • Tài khoản Kakao Business
  • Kênh Kakao Talk
  • Tài liệu xác minh doanh nghiệp như giấy đăng ký kinh doanh
  • Nội dung mẫu Alimtalk cần gửi
  • Tài khoản đại lý gửi hoặc dịch vụ nhắn tin trên đám mây
  • Chính sách thu thập số điện thoại người nhận và quản lý sự đồng ý

3. Chuẩn bị kênh Kakao Talk

1

Truy cập Kakao Business

Đăng nhập vào trung tâm quản trị Kakao Business.

business.kakao.com
2

Tạo kênh Kakao Talk

Tạo một kênh theo tên thương hiệu hoặc tên dịch vụ của bạn.

Trung tâm quản trị kênh Kakao Talk
3

Tiến hành xác minh doanh nghiệp

Gửi thông tin doanh nghiệp và tài liệu chứng minh để xác minh kênh doanh nghiệp.

Xác minh doanh nghiệp
4

Kiểm tra ID tìm kiếm của kênh

Kiểm tra ID kênh hoặc ID tìm kiếm cần thiết cho cài đặt gửi Alimtalk.

Hồ sơ kênh → ID tìm kiếm
Phải hoàn tất xác minh doanh nghiệp thì mới có thể đăng ký hồ sơ người gửi Alimtalk.

4. Đăng ký hồ sơ người gửi / Sender Profile

1

Chọn nền tảng gửi

Chọn nhà cung cấp Alimtalk như NHN Cloud, Naver Cloud SENS, Infobank, Bizgo.

Messaging Provider
2

Tạo dự án

Tạo dự án Alimtalk trên nền tảng bạn đã chọn.

Project / Service
3

Kết nối kênh Kakao Talk

Đăng ký thông tin kênh Kakao Talk đã được xác minh doanh nghiệp.

Sender Profile
4

Xét duyệt hồ sơ người gửi

Tiến hành xác nhận quyền sở hữu kênh và xét duyệt hồ sơ người gửi.

Profile Review
5

Xác nhận thông tin xác thực API

Kiểm tra App Key, Secret Key, API Key, v.v. do nền tảng cung cấp.

API Key / Secret

5. Đăng ký và phê duyệt mẫu

1

Soạn mẫu

Soạn nội dung thông tin như xác nhận đơn hàng, thông báo vận chuyển.

Template Message
2

Thiết lập biến

Thiết lập các giá trị thay đổi động như tên, số đơn hàng, số vận đơn làm biến.

#{biến}
3

Yêu cầu xét duyệt

Lưu mẫu và gửi yêu cầu xét duyệt tới Kakao.

Request Approval
4

Xác nhận phê duyệt

Xác nhận mã mẫu hoặc ID đã được phê duyệt.

Template Code
Nếu nội dung trước khi xét duyệt khác với nội dung gửi thực tế, việc gửi có thể thất bại hoặc bị xử phạt.

6. Ví dụ luồng gửi qua API

1) Gọi Alimtalk API Endpoint từ máy chủ của bạn
2) Xác thực bằng API Key hoặc Secret
3) Truyền số điện thoại người nhận, mã mẫu và giá trị biến
4) Nhận kết quả gửi
5) Xử lý việc có gửi SMS dự phòng khi thất bại hay không

7. Danh sách kiểm tra cuối cùng

  • Đã tạo kênh Kakao Talk.
  • Đã hoàn tất xác minh doanh nghiệp.
  • Đã chọn nền tảng gửi Alimtalk.
  • Đã đăng ký Sender Profile.
  • Đã xác nhận API Key hoặc Secret.
  • Đã đăng ký mẫu và được phê duyệt.
  • Đã chuẩn bị chính sách gửi thất bại và SMS dự phòng.
