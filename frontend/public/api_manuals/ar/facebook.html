دليل إصدار API

دليل ملائم للمبتدئين حول كيفية إنشاء تطبيق على Meta for Developers وتجهيز App ID وApp Secret ورمز الوصول إلى الصفحة (Page Access Token) اللازمة لربط البث المباشر لصفحة Facebook.

المفهوم الأساسيليس API Key بل Access Token
موقع المطورينdevelopers.facebook.com
واجهة API المستخدمةLive Video API (Graph)
الأذونات المطلوبةمسؤول الصفحة + Page Token

1. أهم ما يجب معرفته أولاً

يكفي في YouTube مفتاح API واحد، لكن Facebook (Meta) يعمل بهيكل App ID · App Secret · Access Token.

هام: في Meta لا ينتهي الأمر بـ "نسخ مفتاح API واحد". عادةً تحتاج إلى الترتيب التالي: إنشاء تطبيق ← إضافة منتج ← ضبط الأذونات ← إصدار Page Access Token ← مراجعة التطبيق.

2. المتطلبات

  • حساب Meta/Facebook
  • صلاحية الوصول إلى Meta for Developers
  • صفحة Facebook لديك فيها صلاحيات المسؤول
  • رابط سياسة الخصوصية · رابط شروط الخدمة للخدمة الفعلية

3. إنشاء تطبيق Meta Developer

كل تكامل مع Meta API يبدأ بإنشاء تطبيق.

1

الدخول إلى Meta for Developers

سجّل الدخول إلى Meta for Developers في متصفحك.

developers.facebook.com
2

تطبيقاتي ← إنشاء تطبيق

انقر على My Apps ← Create App في الأعلى.

My Apps → Create App
3

اختيار نوع التطبيق

تكامل المنصات/التجارة المباشرة أقرب إلى الغرض التجاري.

اختر نوع Business
4

إدخال اسم التطبيق·البريد الإلكتروني ← إنشاء

أدخل اسم التطبيق وبريد التواصل، ثم أنشئ التطبيق.

App Name / Contact Email
بعد إنشاء التطبيق، تحقق من App ID وApp Secret في لوحة التحكم ← إعدادات التطبيق ← الإعدادات الأساسية. (يُمنع منعًا باتًا كشف App Secret في الواجهة الأمامية/المستودع العام.)

4. مسار إعداد Facebook Live API

1

إضافة منتج

في لوحة تحكم التطبيق، جهّز الإعدادات المتعلقة بـ Facebook Login / Graph API.

Add Product → Facebook Login
2

ربط الصفحة + رمز الصفحة

لبث المحتوى المباشر إلى صفحة، تحتاج إلى صلاحيات مسؤول الصفحة و Page Access Token.

Page Access Token
3

التقدم بطلب الأذونات المطلوبة (App Review)

تقدّم بطلب أذونات لأغراض نشر المحتوى·إدارة الصفحات·إنشاء فيديوهات مباشرة (مثل pages_read_engagement وpages_manage_posts).

Permissions → App Review
4

إصدار·اختبار الرمز عبر Graph API Explorer

أصدر رمزًا يتضمن الأذونات واختبر استدعاءات API.

Tools → Graph API Explorer
مثال على استدعاء إنشاء Facebook Live
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
عند الإنشاء بنجاح، تُعاد معلومات البث المباشر مثل معرّف البث·رابط التدفق·مفتاح التدفق. اطّلع على معرّف الصفحة (page_id) في معلومات الصفحة.

5. التسجيل في GenieGo Integration Hub

1

نسخ رمز الوصول إلى الصفحة

انسخ Page Access Token الذي أصدرته (يُوصى برمز طويل الأمد).

access_token (Page token)
2

تأكيد معرّف الصفحة

تأكّد من معرّف (page_id) صفحة Facebook المستهدفة.

page_id
3

التسجيل في Integration Hub

في GenieGo Integration Hub ← Facebook Live ← [تسجيل]، أدخل القيم أدناه واحفظ.

access_token · page_id
تُخزَّن المفاتيح المسجَّلة مشفّرة في حالة السكون باستخدام AES-256-GCM وتُخفى عند القراءة.

6. أنواع Access Token

نوع الرمزالاستخدامملاحظات
User Access Tokenاستدعاءات API المستندة إلى تسجيل دخول المستخدمتختلف البيانات التي يمكن الوصول إليها حسب نطاق الأذونات الممنوحة.
Page Access Tokenنشر الصفحة · إنشاء بث مباشر للصفحةتتطلب صلاحيات مسؤول الصفحة والأذونات ذات الصلة.
Long-Lived Tokenرمز طويل الأمد بمدة صلاحية ممتدةعند التشغيل يلزم منطق انتهاء الصلاحية·التجديد.

7. قائمة التحقق

  • أنشأت تطبيق Meta Developer.
  • تحققت من App ID و App Secret.
  • ربطت صفحة Facebook وأصدرت Page Access Token.
  • نظّمت·قدّمت طلب الأذونات المطلوبة (App Review).
  • أدخلت access_token·page_id في GenieGo [تسجيل] وحفظت.
  • خزّنت الرمز بأمان في متغيرات بيئة الخادم.
