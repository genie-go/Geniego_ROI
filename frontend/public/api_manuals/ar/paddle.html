1. قبل أن تبدأ

‏Paddle هي منصة دفع من نوع Merchant of Record لمنتجات SaaS والمنتجات الرقمية (Billing v2).

مطلوب

حساب Paddle

يتطلب بيع منتجات SaaS والمنتجات الرقمية حساب Paddle (Billing v2).

المفاتيح المُصدَرة

مفتاح API الخاص بـ Billing

تقوم بإنشاء مفتاح API الخاص بـ Billing بصيغة Bearer ‏(pdl_live_…). وهو ليس طريقة vendor_id/auth_code القديمة من Classic.

ملاحظة: المفتاح المطلوب إصداره هو مفتاح API الخاص بـ Billing (الإصدار v2) (Bearer، ‎pdl_live_…). لا تخلط بينه وبين طريقة vendor_id/auth_code القديمة من Classic.

2. خطوات الإصدار

أصدر بالترتيب أدناه وسجّل في GenieGo.

1

تسجيل الدخول إلى لوحة تحكم Paddle

سجّل الدخول إلى لوحة التحكم باستخدام حساب Paddle الذي أعددته.

https://vendors.paddle.com
2

الانتقال إلى قائمة API keys

انتقل إلى [Developer Tools] > [Authentication] > [API keys].

Developer Tools → Authentication → API keys
3

إنشاء مفتاح API الخاص بـ Billing

أنشئ مفتاح API الخاص بـ Billing ‏(Bearer، ‎pdl_live_…). وهو ليس طريقة vendor_id/auth_code القديمة من Classic.

بمجرد إنشاء مفتاح API الخاص بـ Billing يبدأ بـ pdl_live_…، يكون قد تم إصداره بشكل صحيح.

3. المعلومات المطلوب تسجيلها في GenieGo

الحقلالوصف
api_keyمفتاح API الخاص بـ Billing (الإصدار v2) المُصدَر (Bearer، ‎pdl_live_…، تعامل معه بحذر)
أدخل api_key في GenieGo Integration Hub ضمن [تسجيل] واحفظ. تُشفَّر بيانات الاعتماد أثناء التخزين باستخدام AES-256-GCM وتُخفى عند القراءة.
