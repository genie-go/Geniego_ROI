Fulfillment (3PL)
دليل إصدار API Key

دليل لإبرام عقد عُهدة لخدمات التنفيذ اللوجستي 3PL (الإدخال/الإخراج من المستودع) وتسجيل مفتاح 3PL API ومُعرّف المستودع (warehouse ID) الصادرَين في GenieGo Integration Hub. في هذه القناة، يتم استلام بيانات الاعتماد عبر عقد عُهدة وليس عبر الإصدار الذاتي من وحدة التحكم.

1. قبل أن تبدأ

خدمة التنفيذ (3PL) قناة قائمة على العقود، حيث تُصدَر بيانات الاعتماد بعد إبرام عقد عُهدة للإدخال/الإخراج من المستودع.

مطلوب

عقد عُهدة للإدخال/الإخراج من المستودع

لاستخدام خدمة التنفيذ 3PL، يجب أولاً إبرام عقد عُهدة للإدخال/الإخراج من المستودع مع مزوّد خدمة التنفيذ أو فريق التشغيل.

المفاتيح الصادرة

مفتاح 3PL API · مُعرّف المستودع

بعد اكتمال العقد، يتم إصدار مُعرّف المستودع ومفتاح 3PL API لك.

ملاحظة: لا تُصدِر هذه القناة بيانات الاعتماد مباشرةً من وحدة تحكم ذاتية؛ بل تستلم بيانات الاعتماد عبر عقد عُهدة.

2. إجراء طلب الإصدار

قدّم الطلب واستلم وفق الترتيب أدناه، ثم سجّل في GenieGo.

1

متابعة عقد العُهدة

تابِع عقد العُهدة للإدخال/الإخراج من المستودع مع مزوّد خدمة التنفيذ أو فريق التشغيل.

2

إبرام العقد

أبرِم عقد العُهدة للإدخال/الإخراج من المستودع لخدمة التنفيذ 3PL.

3

استلام بيانات الاعتماد

بعد اكتمال العقد، يتم إصدار مُعرّف المستودع·مفتاح 3PL API لك.

4

التسجيل في GenieGo

أدخِل api_key وwarehouse_id الصادرَين في GenieGo [تسجيل] واحفظ.

إذا استلمت كلًّا من مُعرّف المستودع·مفتاح 3PL API من مزوّد خدمة التنفيذ، فهذا يعني أنه قد تم إصدارهما بشكل صحيح.

3. المعلومات المطلوب تسجيلها في GenieGo

الحقلالوصف

warehouse_idمُعرّف المستودع الصادر

4. المشكلات الشائعة

قائمة التحقق النهائية

  • أبرمتُ عقد عُهدة للإدخال/الإخراج من المستودع مع مزوّد خدمة التنفيذ أو فريق التشغيل.
  • استلمتُ مُعرّف المستودع·مفتاح 3PL API بعد اكتمال العقد.
  • تأكّدتُ من api_key·warehouse_id.
  • أدخلتُ api_key·warehouse_id في GenieGo [تسجيل] وحفظتُه.
