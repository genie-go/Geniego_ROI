دليل إصدار API

يلخص هذا الدليل عملية إنشاء Provider و Messaging API Channel في LINE Developers Console وتهيئة Channel Access Token و Webhook URL.

بنية المصادقةChannel Access Token
الحساب المطلوبحساب LINE
النتائج الرئيسيةChannel ID / Secret / Token
الاستخدامالمراسلة · روبوت الدردشة

1. مفهوم إصدار LINE API

يعمل تكامل LINE من خلال إنشاء Provider و Channel في LINE Developers Console، ثم إصدار Channel ID و Channel Secret و Channel Access Token لـ Messaging API أو LINE Login.

التمييز: يُستخدم قناة Messaging API لإرسال الرسائل، وقناة LINE Login لتكامل تسجيل الدخول.

2. المتطلبات

  • حساب LINE أو حساب تجاري
  • إذن الوصول إلى LINE Developers Console
  • LINE Official Account
  • اسم الخدمة والوصف
  • Webhook URL
  • عنوان URL لسياسة الخصوصية وعنوان URL للشروط

3. إنشاء Provider

1

الوصول إلى LINE Developers

سجّل الدخول إلى LINE Developers Console.

developers.line.biz
2

إنشاء Provider

أنشئ Provider باستخدام اسم الشركة أو اسم الخدمة.

Create Provider
3

إدخال المعلومات الأساسية

أدخل جهة تشغيل الخدمة وتفاصيل الاتصال ومعلومات الشروط.

Provider Settings

4. إنشاء قناة Messaging API

1

إنشاء قناة جديدة

انقر فوق Create a new channel داخل Provider.

Create a new channel
2

اختيار Messaging API

اختر Messaging API كنوع للقناة.

Messaging API
3

إدخال معلومات القناة

أدخل اسم القناة والوصف والفئة والبريد الإلكتروني وغير ذلك.

Channel Information
4

ربط LINE Official Account

اربط حسابًا رسميًا لإرسال الرسائل منه، أو أنشئ حسابًا جديدًا.

LINE Official Account
5

إكمال إنشاء القناة

بعد الإنشاء، تحقق من Channel ID و Channel Secret.

Channel ID / Channel Secret

5. إصدار Channel Access Token

1

الانتقال إلى علامة التبويب Messaging API

في شاشة القناة التي أنشأتها، انتقل إلى علامة التبويب Messaging API.

Messaging API tab
2

العثور على عنصر Channel access token

تحقق من منطقة Channel access token في الأسفل.

Channel access token
3

النقر فوق Issue

انقر فوق زر Issue أو Reissue لإصدار الرمز المميز.

Issue
4

التخزين بأمان على الخادم

خزّن الرمز المميز الصادر في متغير بيئة الخادم أو في Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token هو مفتاح سري لديه إذن إرسال الرسائل. يجب عدم كشفه في HTML أو الواجهة الأمامية أو المستودعات العامة.

6. تهيئة Webhook

1

إدخال Webhook URL

أدخل عنوان URL للخادم الذي سيستقبل أحداث رسائل المستخدم.

Webhook URL
2

تفعيل Use webhook

غيّر خيار Use webhook إلى Enabled.

Use webhook: Enabled
3

اختبار Verify

أرسل طلب اختبار من LINE إلى خادمك للتأكد من الاتصال.

Verify
4

التحقق من رمز الاستجابة

تأكد من أن الخادم يُرجع 200 OK.

HTTP 200 OK

7. مثال على إرسال رسالة

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "مرحبًا. هذا اختبار رسالة LINE."} ] }

8. قائمة التحقق النهائية

  • أنشأت Provider.
  • أنشأت قناة Messaging API.
  • ربطت LINE Official Account.
  • تحققت من Channel ID و Channel Secret.
  • أصدرت Channel Access Token.
  • سجّلت Webhook URL واجتزت Verify.
  • خزّنت الرمز المميز بأمان في متغير بيئة الخادم.
