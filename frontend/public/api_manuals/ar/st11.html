دليل إصدار مفتاح API

دليل تفصيلي خطوة بخطوة مناسب للمبتدئين لإصدار مفتاح Open API من مركز 11Street Open API (openapi.11st.co.kr) وتسجيله في GenieGo Integration Hub.

1. قبل أن تبدأ

لا يمكن إصدار مفاتيح Open API الخاصة بـ 11Street إلا من حساب بائع.

مطلوب

حساب بائع 11Street

تحتاج إلى حساب بائع 11Street. سجّل الدخول إلى مركز Open API باستخدام معرّف البائع الخاص بك.

المفاتيح الصادرة

مفتاح Open API · معرّف البائع (Seller ID)

بعد إكمال تسجيل الخدمة وتسجيل عنوان IP للوصول، تحقّق من مفتاح Open API الصادر مع معرّف البائع الخاص بك.

ملاحظة: يجب تسجيل عنوان IP لخادم مسموح له بالوصول إلى API قبل أن تتمكن من استخدام المفتاح.

2. خطوات الإصدار

أصدر بالترتيب أدناه وسجّل في GenieGo.

1

تسجيل الدخول إلى مركز Open API

سجّل الدخول إلى مركز 11Street Open API باستخدام معرّف البائع الخاص بك.

https://openapi.11st.co.kr/openapi/OpenApiFrontMain.tmall
2

تسجيل الخدمة

من القائمة، انقر على [تسجيل·التحقق من الخدمة]، وأدخل بيانات البائع، ووافق على الشروط ثم سجّل.

تسجيل·التحقق من الخدمة
3

تسجيل عنوان IP للوصول

في [معلومات Seller API] > [تعديل]، سجّل عنوان IP للخادم المسموح له بالوصول إلى API.

معلومات Seller API → تعديل
4

نسخ مفتاح API

انسخ مفتاح API الصادر.

بمجرد اكتمال تسجيل الخدمة وتسجيل عنوان IP للوصول، يتم إصدار مفتاح API.

3. المعلومات المطلوب تسجيلها في GenieGo

الحقلالوصف
api_keyمفتاح Open API الصادر
seller_idمعرّف البائع (Seller ID)
أدخله واحفظه في GenieGo Integration Hub [تسجيل]. يتم تشفير بيانات الاعتماد أثناء التخزين باستخدام AES-256-GCM وإخفاؤها عند القراءة.

4. المشكلات الشائعة

٥. قبل تسجيل المنتجات (مهم)

حتى مع وجود مفتاح API صالح، يرفض 11Street تسجيل المنتج ما لم تتوفر البنود الثلاثة التالية.

  • فئة العرض (dispCtgrNo) — حدّدها في GenieGo من [مزامنة الكتالوج] > [تعيين الفئات] أو من لوحة «مطلوب تحديد الفئة والعلامة التجارية»
  • العلامة التجارية — سجّلها في GenieGo من [مزامنة الكتالوج] > [إدارة العلامات التجارية] ثم عيّنها للمنتج (اسم العلامة وحده كافٍ، دون رمز)
  • طريقة البيع — يرسلها GenieGo تلقائيًا كسعر ثابت (01)، ولا حاجة لإدخالها
رسالة الخطأالمعنى الحقيقيالحل
-997 لا توجد معلومات API مسجّلةليست فشلًا في المصادقة. تعني أن عنوان الـ API المستدعى (المسار + الأسلوب) غير موجود لدى 11Street.لا تُعِد إصدار المفتاح. اطلب من مسؤول التكامل التحقق من العنوان.
رمز العلامة التجارية مفقودلم تُعيَّن علامة تجارية للمنتج.سجّل العلامة في [مزامنة الكتالوج] > [إدارة العلامات التجارية] وعيّنها للمنتج.
طريقة البيع غير متاحة عبر APIرمز طريقة البيع ليس ضمن القيم المسموح بها.يرسل GenieGo سعرًا ثابتًا (01) تلقائيًا. لا إجراء مطلوب.
تحقق من هذا قبل إعادة إصدار المفتاح. الخطأ -997 ليس مشكلة مفتاح أو IP أو طلب استخدام API. حتى لو كانت الثلاثة سليمة، فإن العنوان الخاطئ يعيد -997.

قائمة التحقق النهائية

  • سجّلت الدخول إلى مركز Open API بحساب بائع 11Street.
  • أدخلت بيانات البائع ووافقت على الشروط في [تسجيل·التحقق من الخدمة].
  • سجّلت عنوان IP للخادم المسموح به في [معلومات Seller API].
  • تأكدت من مفتاح Open API الصادر ومعرّف البائع.
  • أدخلت api_key·seller_id في GenieGo [تسجيل] وحفظت.
