1. مفهوم إصدار واجهة Yahoo! Japan

بحسب نوع الخدمة، تُصدر واجهات عائلة Yahoo! Japan قيم Client ID وClient Secret وبيانات اعتماد OAuth من Yahoo! JAPAN Developers أو من LY/Yahoo!広告 Developer Center.

مهم: يختلف موقع الإصدار وإجراء المراجعة في "Yahoo! Japan" بحسب الواجهة التي تنوي استخدامها. يجب التمييز بين واجهات التسوّق والإعلانات وتسجيل الدخول والبحث.

2. المتطلبات

  • Yahoo! JAPAN ID أو حساب أعمال
  • تحديد نوع الواجهة المراد استخدامها
  • اسم التطبيق ووصف الخدمة
  • Redirect URI
  • رابط سياسة الخصوصية
  • نطاق التشغيل أو نطاق الاختبار

3. الترتيب الأساسي لتسجيل التطبيق

1

الوصول إلى موقع المطوّرين

سجّل الدخول إلى Yahoo! JAPAN Developers أو إلى Developer Center الخاص بالواجهة المعنية.

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

إنشاء تطبيق

انقر على قائمة إضافة تطبيق جديد.

Create Application / Add Application
3

إدخال معلومات التطبيق

أدخل اسم التطبيق والوصف ومعلومات المسؤول ورابط الموقع الإلكتروني.

Application Information
4

إدخال Redirect URI

أدخل بدقة الرابط الذي سيُعاد إليه بعد مصادقة OAuth.

Redirect URI
5

اختيار الأذونات أو الواجهة

اختر أذونات الواجهة والنطاقات (Scope) التي تنوي استخدامها.

Scope / API Permission
6

إتمام التسجيل

بعد التسجيل، تحقق من Client ID وClient Secret.

Client ID / Client Secret

5. تدفق إصدار رمز OAuth

1) إنشاء رابط مصادقة المستخدم باستخدام Client ID 2) يسجّل المستخدم الدخول بحساب Yahoo!‎ 3) استلام authorization code عند Redirect URI 4) طلب Access Token باستخدام authorization code + Client Secret 5) استدعاء الواجهة باستخدام Access Token 6) التجديد باستخدام Refresh Token عند الحاجة

7. قائمة التحقق النهائية

  • حدّدت نوع واجهة Yahoo! Japan المراد استخدامها.
  • سجّلت الدخول إلى وحدة تحكم المطوّرين الصحيحة.
  • سجّلت التطبيق.
  • تحققت من Client ID وClient Secret.
  • سجّلت Redirect URI بدقة.
  • اخترت النطاق (Scope) المطلوب.
  • أكملت اختبار إصدار Access Token.
