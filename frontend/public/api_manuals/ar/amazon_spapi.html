1. ما هي Amazon SP-API؟

إن Amazon Selling Partner API هي واجهة برمجة تطبيقات لدمج بيانات البائع برمجياً مثل الطلبات والمخزون والمنتجات والشحن والتسويات.

مهم: إن SP-API ليست مجرد مخطط API Key بسيط. وعادةً ما تتطلب معاً تسجيل Amazon Developer وتسجيل التطبيق وLogin with Amazon(LWA) وتفويض OAuth.
★يتطلب التكامل مع GenieGo 5 قيم: refresh_token, client_id, client_secret, marketplace_id, seller_id. (يتم استدعاء SP-API الحالية باستخدام رمز LWA وحده، ويتم التعامل مع توقيع AWS IAM/SigV4 داخلياً بواسطة GenieGo، لذا لا تُدخله في حقول التسجيل.)

2. المتطلبات الأساسية

  • حساب بائع احترافي على Amazon Seller Central أو Vendor Central
  • معلومات الشركة/جهة الاتصال المستخدمة لتسجيل المطور
  • اسم التطبيق ووصف التطبيق والغرض من الاستخدام
  • رابط سياسة الخصوصية
  • OAuth Redirect URL

3. تسجيل المطور

أولاً، سجّل ملف تعريف مطور SP-API الخاص بك في Seller Central.

1

تسجيل الدخول إلى Seller Central

سجّل الدخول إلى Seller Central باستخدام حساب البائع على Amazon.

Seller Central → Apps & Services
2

الانتقال إلى قائمة Develop Apps

من قائمة Apps & Services، انتقل إلى قائمة المطور أو تطوير التطبيقات.

Apps & Services → Develop Apps
3

تعبئة Developer Profile

أدخل معلومات الشركة ومعلومات جهة الاتصال والغرض من استخدام API وخطة حماية البيانات.

Developer Profile
4

الإرسال للمراجعة

أرسل معلومات تسجيل المطور لمراجعة Amazon.

Submit for review
بعد اكتمال الموافقة فقط يمكنك تسجيل تطبيق SP-API للإنتاج واستخدام أذوناته.

4. تسجيل تطبيق SP-API → Client ID / Client Secret

1

إنشاء تطبيق جديد

أنشئ تطبيقاً جديداً في Develop Apps.

Add new app
2

اختيار نوع التطبيق

إذا كنت تربط حسابك فقط فهو Private App؛ وإذا قدمته لبائعين خارجيين فهو Public App.

Private / Public App
3

اختيار أدوار API

اختر الأدوار التي تحتاجها، مثل Orders وCatalog وInventory.

API Roles
4

إدخال OAuth Redirect URL

أدخل عنوان URL الخادم الذي سيعود إليه بعد اكتمال التفويض.

Redirect URI
5

الحفظ والتحقق من معلومات Client

بعد إنشاء التطبيق، تحقق من client_id (LWA Client ID) وclient_secret (Client Secret).

client_id / client_secret

5. تفويض OAuth → إصدار Refresh Token

1) الوصول إلى عنوان URL تفويض OAuth الخاص بالبائع 2) استلام authorization code عند Redirect URL 3) تمرير authorization code إلى LWA Token Endpoint 4) استلام refresh_token / access_token
يُستخدم refresh_token للاستدعاءات طويلة المدى بعد تفويض البائع. تقوم GenieGo تلقائياً بتحديث access_token باستخدام هذا refresh_token لاستدعاء SP-API، لذا تحتاج فقط إلى تسجيل refresh_token دون أي إعداد منفصل لتوقيع AWS.

6. التحقق من Marketplace ID · Seller ID

تحقق من المعرّفين المتبقيين اللازمين لتسجيل GenieGo.

marketplace_id

معرّف السوق المستهدف للبيع

إنه Marketplace ID للبلد الذي تتصل به. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

مطلوب لتسجيل المنتج والبحث عن الطلبات. تحقق منه في معلومات حساب Seller Central الخاص بك. Seller Central → الإعدادات → معلومات الحساب → Merchant Token

7. التسجيل في GenieGo Integration Hub

1

تجهيز القيم الـ 5

جهّز refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 بيانات اعتماد
2

Integration Hub → Amazon SP-API → [تسجيل]

أدخل كل قيمة في حقلها واحفظ (الحقول الفارغة تحتفظ بالقيم الحالية).

refresh_token · client_id · client_secret · marketplace_id · seller_id
يتم تشفير القيم المسجّلة عند التخزين باستخدام AES-256-GCM وإخفاؤها عند القراءة. تبدأ المزامنة التلقائية فور الحفظ.
يجب عدم كشف client_secret·refresh_token مطلقاً في كود الواجهة الأمامية أو HTML أو المستودعات العامة.

8. قائمة التحقق النهائية

  • تم إرسال Developer Profile والحصول على الموافقة عليه.
  • تم إنشاء تطبيق SP-API والتحقق من client_id·client_secret.
  • تم إصدار refresh_token عبر تفويض OAuth.
  • تم التحقق من marketplace_id المراد ربطه.
  • تم التحقق من seller_id (Merchant Token) في Seller Central.
  • تم إدخال القيم الـ 5 (refresh_token·client_id·client_secret·marketplace_id·seller_id) في GenieGo [تسجيل] وحفظها.
