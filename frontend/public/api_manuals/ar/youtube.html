1. الفرق بين API Key و OAuth

هذا هو الجزء الذي يربك الناس أكثر عند البدء.

API Key

للاستعلام عن البيانات العامة

يُستخدم لجلب بيانات YouTube العامة مثل البحث عن الفيديوهات والاطلاع على معلومات القناة واسترجاع قوائم التشغيل وقراءة التعليقات.

OAuth 2.0

للمهام التي تحتاج إلى صلاحيات حسابك

يُستخدم للمهام التي تتطلب موافقة المستخدم، مثل رفع الفيديوهات وإدارة قناتك الخاصة والوصول إلى البيانات الخاصة.

مهم: لرفع الفيديوهات تلقائيًا، لا يكفي API Key وحده؛ بل تلزم مصادقة OAuth 2.0.

2. تفعيل YouTube Data API v3

قبل إنشاء API Key، يجب أولًا تشغيل واجهة API التي تريد استخدامها في مشروعك.

1

الوصول إلى Google Cloud Console

سجّل الدخول إلى Google Cloud Console.

console.cloud.google.com
2

اختيار مشروع أو إنشاؤه

في محدّد المشروع بالأعلى، اختر مشروعًا موجودًا أو أنشئ مشروعًا جديدًا.

مثال: Geniego ROI
3

الانتقال إلى مكتبة API

من القائمة اليسرى، انتقل إلى واجهات برمجة التطبيقات والخدمات ← المكتبة.

واجهات برمجة التطبيقات والخدمات ← المكتبة
4

البحث عن YouTube Data API v3 وتفعيلها

اكتب YouTube Data API v3 في مربع البحث، واختر تلك الواجهة، ثم انقر على زر تفعيل.

YouTube Data API v3 ← تفعيل
إذا رأيت الحالة: مُفعَّلة على الشاشة، فهذا يعني أنه تم تفعيلها بشكل صحيح.

1

الانتقال إلى قائمة بيانات الاعتماد

من القائمة اليسرى، انتقل إلى واجهات برمجة التطبيقات والخدمات ← بيانات الاعتماد.

واجهات برمجة التطبيقات والخدمات ← بيانات الاعتماد

النقر على إنشاء بيانات اعتماد

انقر على زر + إنشاء بيانات اعتماد في الأعلى.

+ إنشاء بيانات اعتماد

مفتاح API

ضمن قيود API، تأكد من اختيار YouTube Data API v3.

انقر على زر إنشاء في الأسفل لتوليد API Key.

5. أمثلة على استخدام API Key

يمكنك الاختبار بوضع المفتاح الذي أنشأته في الجزء YOUR_API_KEY.

مثال البحث عن فيديوhttps://www.googleapis.com/youtube/v3/search?part=snippet&q=korea&type=video&key=YOUR_API_KEY
مثال الاستعلام عن معلومات القناةhttps://www.googleapis.com/youtube/v3/channels?part=snippet&id=CHANNEL_ID&key=YOUR_API_KEY
إذا ظهرت نتيجة بصيغة JSON عند إدخال الرابط أعلاه في شريط العناوين في متصفحك، فهذا يعني أن API Key يعمل بشكل صحيح.

7. قائمة التحقق النهائية

  • اخترت مشروع Google Cloud.
  • فعّلت YouTube Data API v3.
  • أنشأت API Key ضمن بيانات الاعتماد.
  • تأكدت من أن API Key الذي تم إنشاؤه يبدأ بـ AIzaSy.
  • ضبطت قيود API على YouTube Data API v3.
  • تعيين قيود الموقع الإلكتروني أو IP قبل النشر إلى خدمة حقيقية.
