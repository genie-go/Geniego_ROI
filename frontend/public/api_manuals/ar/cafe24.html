دليل إصدار مفتاح API

دليل تفصيلي خطوة بخطوة مناسب للمبتدئين لتسجيل تطبيق في Cafe24 Developer Center لإصدار Client ID وClient Secret، والحصول على الرموز عبر تفويض OAuth 2.0، وتسجيلها في GenieGo Integration Hub.

الوقت المقدّرحوالي 10–15 دقيقة
الحساب المطلوبحساب مشغّل متجر Cafe24
المفاتيح المُصدَرةClient ID · Client Secret · Refresh Token
النتيجةبيانات اعتماد OAuth 2.0

1. قبل أن تبدأ

يمر تكامل Cafe24 بتسجيل المطوّر وتسجيل التطبيق، ويصادق عبر OAuth 2.0.

مطلوب

حساب مشغّل المتجر

تحتاج إلى حساب مشغّل يدير متجر Cafe24. تكمل تسجيل المطوّر في Developer Center باستخدام هذا الحساب.

المفاتيح المُصدَرة

Client ID · Client Secret + الرموز

عند تسجيل تطبيق، يتم إصدار Client ID وClient Secret، وتحصل على access token وrefresh token عبر تفويض OAuth 2.0.

ملاحظة: في GenieGo تسجّل الـ refresh_token الطويل الأمد، ويتم تحديث الـ access token تلقائيًا من خلاله.

2. خطوات تسجيل التطبيق وإصدار الرموز

أصدِر بالترتيب أدناه وسجّل في GenieGo.

1

التسجيل كمطوّر

انتقل إلى Cafe24 Developer Center وسجّل كمطوّر.

developer.cafe24.com
2

تسجيل التطبيق

أنشئ تطبيقًا عبر [التطبيقات] > [تسجيل التطبيق]، وسيتم إصدار Client ID وClient Secret.

التطبيقات → تسجيل التطبيق
3

تفويض OAuth 2.0

أصدِر access token وrefresh token عبر تفويض OAuth 2.0.

4

تحقق من معلوماتك

تحقق من mall_id (معرّف المتجر) وClient ID وClient Secret وrefresh token.

بمجرد تأكيد Client ID·Client Secret وrefresh token جميعًا، فهذا يعني أنه تم إصدارها بشكل صحيح.

3. المعلومات المطلوب تسجيلها في GenieGo

الحقلالوصف
mall_idمعرّف متجر Cafe24
client_idالـ Client ID المُصدَر
client_secretالـ Client Secret المُصدَر (تعامل معه بحذر)
refresh_tokenالـ Refresh Token المُستلَم عبر تفويض OAuth 2.0
أدخِلها واحفظها في GenieGo Integration Hub [تسجيل]، ويتم تشفير بيانات الاعتماد أثناء التخزين باستخدام AES-256-GCM وإخفاؤها عند القراءة.

4. المشكلات الشائعة

المشكلةالسببالحل
فشل التفويضmall_id غير صحيح أو عدم تطابق إعدادات التطبيقأعد التحقق من mall_id ومعلومات Client التي تم تعيينها أثناء تسجيل التطبيق.
خطأ انتهاء صلاحية الرمزإدخال refresh_token بشكل خاطئأعد تسجيل أحدث refresh_token تم الحصول عليه عبر تفويض OAuth 2.0.

قائمة التحقق النهائية

  • سجّلت كمطوّر في Cafe24 Developer Center.
  • أنشأت تطبيقًا عبر [التطبيقات > تسجيل التطبيق] وأصدرت Client ID·Client Secret.
  • أصدرت access token·refresh token عبر تفويض OAuth 2.0.
  • أدخلت mall_id·client_id·client_secret·refresh_token في GenieGo [تسجيل] وحفظتها.
