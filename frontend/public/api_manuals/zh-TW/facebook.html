Facebook Live
API 申請手冊

本手冊手把手指導新手在 Meta for Developers 上建立應用程式，並準備連接 Facebook 粉絲專頁直播所需的 App ID、App Secret 和粉絲專頁存取權杖(Page Access Token)。

核心概念不是 API Key，而是 Access Token
開發網站developers.facebook.com
使用的 APILive Video API (Graph)
所需權限粉絲專頁管理員 + Page Token

1. 首先要了解的核心

YouTube 用一個 API Key 即可，但 Facebook(Meta)採用 App ID · App Secret · Access Token 的結構運作。

重要: Meta 並非「複製一個 API Key」就能完成。通常需要依 建立應用程式 → 新增產品 → 設定權限 → 簽發粉絲專頁存取權杖 → 應用程式審查 的順序進行。

2. 準備事項

  • Meta/Facebook 帳戶
  • Meta for Developers 存取權限
  • 擁有管理權限的 Facebook 粉絲專頁
  • 用於實際服務的隱私權政策 URL · 服務條款 URL

3. 建立 Meta Developer 應用程式

所有 Meta API 整合都從建立應用程式開始。

1

存取 Meta for Developers

在瀏覽器中登入 Meta for Developers。

developers.facebook.com
2

我的應用程式 → 建立應用程式

點擊頂部的 My Apps → Create App。

My Apps → Create App
3

選擇應用程式類型

平台整合/直播電商更接近企業用途。

選擇 Business 類型
4

輸入應用程式名稱·電子郵件 → 建立

輸入應用程式名稱和聯絡電子郵件，然後建立應用程式。

App Name / Contact Email
建立應用程式後，在儀表板 → 應用程式設定 → 基本設定中查看 App ID 和 App Secret。(切勿在前端/公開儲存庫中暴露 App Secret)

4. Facebook Live API 設定流程

1

新增產品

在應用程式儀表板中準備 Facebook Login / Graph API 相關設定。

Add Product → Facebook Login
2

連接粉絲專頁 + 粉絲專頁權杖

若要將直播推送到粉絲專頁，需要粉絲專頁管理員權限和 Page Access Token。

Page Access Token
3

申請所需權限(App Review)

申請用於發布內容·管理粉絲專頁·建立直播影片的權限(例如 pages_read_engagement、pages_manage_posts)。

Permissions → App Review
4

用 Graph API Explorer 簽發並測試權杖

簽發包含權限的權杖並測試 API 呼叫。

Tools → Graph API Explorer
Facebook Live 建立呼叫範例
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
建立成功後會回傳直播 ID·串流 URL·串流金鑰等直播推送資訊。粉絲專頁 ID(page_id)可在粉絲專頁資訊中查看。

5. 註冊到 GenieGo 整合中心

1

複製粉絲專頁存取權杖

複製您簽發的 Page Access Token(建議使用長期權杖)。

access_token (Page token)
2

確認粉絲專頁 ID

確認目標 Facebook 粉絲專頁的 ID(page_id)。

page_id
3

註冊到整合中心

在 GenieGo 整合中心 → Facebook Live → [註冊] 中輸入以下值並儲存。

access_token · page_id
註冊的金鑰將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

6. Access Token 類型

權杖類型用途注意事項
User Access Token基於使用者登入的 API 呼叫可存取的資料因權限授權範圍而異。
Page Access Token粉絲專頁發布·粉絲專頁直播建立需要粉絲專頁管理員權限及相關權限。
Long-Lived Token延長了有效期的長期權杖營運時需要有效期·更新邏輯。

7. 檢查清單

  • 已建立 Meta Developer 應用程式。
  • 已確認 App ID 和 App Secret。
  • 已連接 Facebook 粉絲專頁並簽發 Page Access Token。
  • 已整理並申請所需權限(App Review)。
  • 已在 GenieGo [註冊] 中輸入 access_token·page_id 並儲存。
  • 已將權杖安全地保存在伺服器環境變數中。

8. 常見問題

問題原因解決方法
沒有 API Key 選單Meta 不是單一 API Key 結構依 App ID·App Secret·Access Token 的結構進行。
權限錯誤權杖缺少所需權限在 Graph API Explorer 中新增權限後重新簽發權杖。
粉絲專頁直播建立失敗Page Access Token/管理員權限不足檢查粉絲專頁管理員權限和權杖權限。
