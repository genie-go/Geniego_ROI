1. 開始之前

履約（3PL）是一種合約型渠道，在簽訂入庫/出庫委託合約後簽發憑證。

必需

入庫/出庫委託合約

要使用 3PL 履約，必須先與履約服務商或營運團隊簽訂入庫/出庫委託合約。

簽發金鑰

3PL API 金鑰 · 倉庫 ID

合約完成後，您將取得倉庫 ID 和 3PL API 金鑰。

注意: 本渠道不透過自助主控台直接簽發，而是透過委託合約領取憑證。

2. 申請簽發流程

依以下順序申請並領取後，在 GenieGo 中註冊。

1

推進委託合約

與履約服務商或營運團隊推進入庫/出庫委託合約。

2

簽訂合約

簽訂 3PL 履約入庫/出庫委託合約。

3

領取憑證

合約完成後，您將取得倉庫 ID·3PL API 金鑰。

4

在 GenieGo 中註冊

在 GenieGo 的[註冊]中輸入簽發的 api_key、warehouse_id 並儲存。

如果您已從履約服務商處領取了倉庫 ID·3PL API 金鑰，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
api_key簽發的 3PL API 金鑰
warehouse_id簽發的倉庫 ID
在 GenieGo 整合中心[註冊]中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
未收到憑證入庫/出庫委託合約未完成與履約服務商或營運團隊簽訂委託合約後，再領取憑證。
入庫/出庫整合錯誤倉庫 ID(warehouse_id)填寫錯誤重新確認簽發的倉庫 ID 並準確輸入。

最終檢查清單

  • 已與履約服務商或營運團隊簽訂入庫/出庫委託合約。
  • 合約完成後已領取倉庫 ID·3PL API 金鑰。
  • 已確認 api_key·warehouse_id。
  • 已在 GenieGo 的[註冊]中輸入 api_key·warehouse_id 並儲存。
