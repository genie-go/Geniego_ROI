Gmarket
API Key 申請手冊

本手冊介紹如何從 ESM Trading API(etapi.gmarket.com)簽發 Gmarket·拍賣共用的 API 金鑰，並將其註冊到 GenieGo 整合中心。Gmarket 與拍賣透過 ESM Plus 統一營運，因此一個 API 金鑰即可同時連接兩個渠道。

預計時間含審核等待數日(審核制)
所需帳戶Gmarket·拍賣賣家帳戶 + ESM Plus 主 ID
簽發金鑰API 金鑰 · 賣家 ID
成果ESM Trading API 憑證

1. 開始之前

您必須在 Gmarket 和拍賣兩邊都註冊為賣家，並擁有 ESM Plus 主 ID。

必需

賣家帳戶 + ESM 主 ID

您需要拍賣和 Gmarket 各自的賣家帳戶，以及在 ESM Plus(esmplus.com)中建立的主 ID。

簽發金鑰

API 金鑰 · 賣家 ID

ESM Trading API 審核通過後，請一併確認簽發的 API 金鑰和賣家 ID。

注意: ESM Trading API 為審核制，您的申請可能被拒絕。申請時您必須提交三項資訊:拍賣賣家ID、Gmarket 賣家ID 和 ESM 主ID。

2. 簽發步驟

依以下順序申請並簽發，然後在 GenieGo 中註冊。

1

註冊為賣家並建立 ESM 主 ID

在拍賣和 Gmarket 兩邊都註冊為賣家，然後登入 ESM Plus 並建立主 ID。

esmplus.com
2

申請 ESM Trading API 使用權限

存取 ESM Trading API 並申請 API 使用權限。因為是審核制，所以可能被拒絕。

https://etapi.gmarket.com
3

提交身分識別資訊

申請時提交三項資訊:拍賣賣家ID、Gmarket 賣家ID 和 ESM 主 ID。

4

取得金鑰

審核通過後即會簽發 API 金鑰。確認簽發的 API 金鑰和賣家 ID。

審核完成後簽發 API 金鑰即表示已正確處理。審核相關諮詢請聯絡 etapihelp@gmail.com。

3. 需在 GenieGo 中註冊的資訊

欄位說明
api_key簽發的 API 金鑰
seller_id賣家 ID
在 GenieGo 整合中心 [註冊] 中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
API 使用申請被拒絕ESM Trading API 為審核制確認您提交的資訊並聯絡 etapihelp@gmail.com。
申請資訊不一致拍賣·Gmarket 賣家ID 或 ESM 主ID 填寫錯誤重新確認三項資訊:拍賣賣家ID、Gmarket 賣家ID 和 ESM 主ID。

最終檢查清單

  • 已在拍賣和 Gmarket 兩邊都註冊為賣家。
  • 已在 ESM Plus 中建立主 ID。
  • 已在 etapi.gmarket.com 申請 ESM Trading API 使用權限並獲得審核。
  • 已確認簽發的 api_key 和 seller_id。
  • 已在 GenieGo [註冊] 中輸入 api_key·seller_id 並儲存。
