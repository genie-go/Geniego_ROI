Y! API 申請新手手冊

Yahoo! Japan
API 申請手冊

為接入 Yahoo! Japan 系列服務，在開發者主控台中註冊應用程式並準備 Client ID、Client Secret 和 OAuth Token 的流程。

認證結構以 OAuth 2.0 為中心
所需帳戶Yahoo! JAPAN ID
主要成果Client ID / Secret
注意各 API 主控台不同

1. Yahoo! Japan API 申請概念

Yahoo! Japan 系列 API 會依服務類型，在 Yahoo! JAPAN Developers 或 LY/Yahoo!広告 Developer Center 申請 Client ID、Client Secret 和 OAuth 認證資訊。

重要: 「Yahoo! Japan」會依您要使用的 API 不同，其申請站點和審查流程也不同。需要區分購物、廣告、登入和搜尋 API。

2. 準備物

  • Yahoo! JAPAN ID 或企業帳戶
  • 確定要使用的 API 類型
  • 應用名稱和服務說明
  • Redirect URI
  • 隱私權政策 URL
  • 正式環境網域或測試網域

3. 應用程式註冊基本順序

1

存取開發者站點

登入 Yahoo! JAPAN Developers 或對應 API 的 Developer Center。

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

建立應用程式

點擊新增應用程式選單。

Create Application / Add Application
3

輸入應用資訊

輸入應用名稱、說明、負責人資訊和網站 URL。

Application Information
4

輸入 Redirect URI

準確輸入 OAuth 認證後返回的 URL。

Redirect URI
5

選擇權限或 API

選擇要使用的 API 權限和範圍。

Scope / API Permission
6

完成註冊

註冊後確認 Client ID 和 Client Secret。

Client ID / Client Secret

4. 各服務的申請位置

目的申請/管理位置成果
Yahoo! Japan 登入 / OAuthYahoo! JAPAN DevelopersClient ID, Client Secret
Yahoo!広告 APIYahoo!広告 Developer CenterClient ID, Client Secret, OAuth Token
Yahoo! Shopping APIYahoo! JAPAN Developers 或 Commerce 相關主控台Application ID 或 OAuth 資訊
內部測試各服務的 Sandbox/測試環境測試用認證資訊

5. OAuth 權杖申請流程

1) 使用 Client ID 產生使用者認證 URL 2) 使用者使用 Yahoo! 帳戶登入 3) 在 Redirect URI 接收 authorization code 4) 使用 authorization code + Client Secret 請求 Access Token 5) 使用 Access Token 呼叫 API 6) 必要時使用 Refresh Token 更新

6. 常見問題

問題原因解決方法
Redirect URI 錯誤註冊的 URL 與呼叫 URL 不同連同通訊協定、網域、路徑都輸入一致
無法查看 Client Secret基於安全考量被隱藏查看管理介面的 View 或重新簽發功能
權限不足未選擇所需 Scope在應用設定中新增 Scope 後重新認證
API 呼叫失敗所用 API 與申請的應用類型不符查看各 API 的開發者文件

7. 最終檢查清單

  • 已確定要使用的 Yahoo! Japan API 類型。
  • 已登入正確的開發者主控台。
  • 已註冊應用程式。
  • 已確認 Client ID 和 Client Secret。
  • 已準確註冊 Redirect URI。
  • 已選擇所需 Scope。
  • 已完成 Access Token 申請測試。
