Cafe24
API Key 申請手冊

本手冊手把手指導新手在 Cafe24 開發者中心註冊應用程式以簽發 Client ID·Client Secret，透過 OAuth 2.0 授權取得權杖，並註冊到 GenieGo 整合中心。

預計時間約 10~15 分鐘
所需帳戶Cafe24 商城營運者帳戶
簽發金鑰Client ID · Client Secret · Refresh Token
成果OAuth 2.0 憑證

1. 開始之前

Cafe24 整合需經過開發者註冊和應用程式註冊，並透過 OAuth 2.0 方式進行驗證。

必需

商城營運者帳戶

您需要一個營運 Cafe24 商城的營運者帳戶。使用該帳戶在開發者中心完成開發者註冊。

簽發金鑰

Client ID · Client Secret + 權杖

註冊應用程式時會簽發 Client ID 和 Client Secret，並透過 OAuth 2.0 授權取得 access token 和 refresh token。

注意: 在 GenieGo 中註冊 long-lived 的 refresh_token，access token 將透過它自動更新。

2. 應用程式註冊及權杖簽發步驟

依以下順序簽發並在 GenieGo 中註冊。

1

開發者註冊

存取 Cafe24 開發者中心並註冊為開發者。

developer.cafe24.com
2

應用程式註冊

透過[應用程式] > [應用程式註冊]建立應用程式後，將簽發 Client ID 和 Client Secret。

應用程式 → 應用程式註冊
3

OAuth 2.0 授權

透過 OAuth 2.0 授權簽發 access token 和 refresh token。

4

確認資訊

確認 mall_id(商城 ID)、Client ID、Client Secret、refresh token。

當 Client ID·Client Secret 和 refresh token 全部確認無誤時，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
mall_idCafe24 商城 ID
client_id簽發的 Client ID
client_secret簽發的 Client Secret(請妥善保管)
refresh_token透過 OAuth 2.0 授權取得的 Refresh Token
在 GenieGo 整合中心的[註冊]中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
授權失敗錯誤的 mall_id 或應用程式設定不一致重新確認應用程式註冊時設定的 mall_id 和 Client 資訊。
權杖過期錯誤refresh_token 填寫錯誤重新註冊透過 OAuth 2.0 授權取得的最新 refresh_token。

最終檢查清單

  • 已在 Cafe24 開發者中心註冊為開發者。
  • 已透過[應用程式 > 應用程式註冊]建立應用程式並簽發 Client ID·Client Secret。
  • 已透過 OAuth 2.0 授權簽發 access token·refresh token。
  • 已在 GenieGo 的[註冊]中輸入 mall_id·client_id·client_secret·refresh_token 並儲存。
