1. 什麼是 Amazon SP-API?

Amazon Selling Partner API 是用於透過程式對接訂單、庫存、商品、配送、結算等賣家資料的 API。

重要: SP-API 並非單純的 API Key 方式。通常需要同時進行 Amazon Developer 註冊、應用程式註冊、Login with Amazon(LWA)、OAuth 授權。
★GenieGo 對接需要 5 個值: refresh_token, client_id, client_secret, marketplace_id, seller_id。(目前 SP-API 僅憑 LWA 權杖即可呼叫,AWS IAM/SigV4 簽章由 GenieGo 內部處理,因此無需在註冊欄中填寫。)

2. 準備事項

  • Amazon Seller Central 或 Vendor Central 專業版賣家帳戶
  • 用於開發者註冊的公司/負責人資訊
  • 應用程式名稱、應用程式說明、使用目的
  • 隱私權政策 URL
  • OAuth Redirect URL

3. 開發者註冊

首先在 Seller Central 中註冊 SP-API 開發者資料。

1

登入 Seller Central

使用 Amazon 賣家帳戶登入 Seller Central。

Seller Central → Apps & Services
2

進入 Develop Apps 選單

在 Apps & Services 選單中,進入開發者或應用程式開發選單。

Apps & Services → Develop Apps
3

填寫 Developer Profile

填寫公司資訊、負責人資訊、API 使用目的與資料保護計畫。

Developer Profile
4

提交審核

為通過 Amazon 審核,提交開發者註冊資訊。

Submit for review
只有審核通過後,才能註冊正式營運用的 SP-API 應用程式並使用其權限。

4. 註冊 SP-API 應用程式 → Client ID / Client Secret

1

建立新應用程式

在 Develop Apps 中建立新應用程式。

Add new app
2

選擇應用程式類型

若僅對接自有帳戶則為 Private App;若提供給外部賣家則屬於 Public App。

Private / Public App
3

選擇 API 角色

選擇 Orders、Catalog、Inventory 等所需角色。

API Roles
4

輸入 OAuth Redirect URL

輸入授權完成後返回的伺服器 URL。

Redirect URI
5

儲存後確認 Client 資訊

建立應用程式後,確認 client_id (LWA Client ID)、client_secret (Client Secret)。

client_id / client_secret

5. OAuth 授權 → 發行 Refresh Token

1) 存取賣家 OAuth 授權 URL 2) 在 Redirect URL 接收 authorization code 3) 將 authorization code 傳遞給 LWA Token Endpoint 4) 接收 refresh_token / access_token
refresh_token 用於賣家授權後的長期呼叫。GenieGo 使用該 refresh_token 自動更新 access_token 來呼叫 SP-API,因此無需另外的 AWS 簽章設定,只需註冊 refresh_token 即可。

6. 確認 Marketplace ID · Seller ID

確認 GenieGo 註冊所需的其餘兩個識別碼。

marketplace_id

銷售目標的市場 ID

要對接國家的 Marketplace ID。US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

賣家 ID / Merchant Token

商品登錄、訂單查詢所需。在 Seller Central 帳戶資訊中查看。Seller Central → 設定 → 帳戶資訊 → Merchant Token

7. 在 GenieGo 整合中心註冊

1

準備 5 個值

準備 refresh_token · client_id · client_secret · marketplace_id · seller_id。

5 項憑證
2

整合中心 → Amazon SP-API → [註冊]

在各欄輸入對應的值並儲存(空欄保留原有值)。

refresh_token · client_id · client_secret · marketplace_id · seller_id
註冊的值會 以 AES-256-GCM 加密儲存,查詢時進行遮罩處理。儲存後立即開始自動同步。
client_secret·refresh_token 絕不能暴露在前端程式碼、HTML 或公開儲存庫中。

8. 最終檢查清單

  • 已提交 Developer Profile 並獲得核准。
  • 已建立 SP-API 應用程式並確認 client_id·client_secret。
  • 已透過 OAuth 授權發行 refresh_token。
  • 已確認要對接的 marketplace_id。
  • 已在 Seller Central 確認 seller_id(Merchant Token)。
  • 已在 GenieGo [註冊] 中輸入 5 個值(refresh_token·client_id·client_secret·marketplace_id·seller_id)並儲存。

9. 常見問題

問題原因解決方法
沒有 API Key 選單SP-API 並非單一 API Key 結構依 LWA(client_id·client_secret·refresh_token)結構進行。
商品登錄失敗未註冊 seller_id(Merchant Token)註冊 Seller Central 帳戶資訊中的 Merchant Token。
國家資料錯誤marketplace_id 不一致註冊對接國家正確的 Marketplace ID。
權限錯誤應用程式 API Role 不足或權杖過期確認 API Role 並使用 refresh_token 更新權杖。
