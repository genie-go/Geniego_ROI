1. 了解 Twitch 帳戶結構

與 Instagram 不同，Twitch 沒有單獨選擇企業帳戶/創作者帳戶的結構。

企業官方頻道

用於品牌營運

以公司名或品牌名建立一個一般 Twitch 帳戶，然後像官方頻道一樣營運。適合產品介紹、直播電商與活動直播。

實況主成長階段

Affiliate / Partner

達到直播實績、追蹤者與觀看時數條件後，即可加入 Affiliate 或 Partner 計畫。

核心： 企業也應先建立一般 Twitch 帳戶，然後進行頻道品牌化、兩步驟驗證、OBS 連線以及 Developer Console 應用程式註冊。

2. 準備物品

  • 企業電子郵件地址
  • 品牌名或公司名
  • Twitch 使用者名稱候選
  • 公司標誌與頻道橫幅圖片
  • 公司簡介文案與網站 URL
  • 用於直播的 PC 或筆記型電腦
  • OBS Studio 或串流軟體
  • 用於 API 整合的 OAuth Redirect URL

3. 建立 Twitch 帳戶

1

前往 Twitch

在瀏覽器中前往 Twitch 官方網站。

https://www.twitch.tv
2

點擊註冊

點擊右上角的 Sign Up 或註冊按鈕。

Sign Up
3

使用企業資訊註冊

輸入公司電子郵件、使用者名稱、密碼與出生日期。

Email / Username / Password
4

電子郵件驗證

開啟 Twitch 寄送的驗證郵件並完成 Verify Email。

Verify Email

4. 決定品牌帳戶名稱

Twitch 使用者名稱會成為頻道網址，因此最好選擇簡短易記的名稱。

用途推薦名稱範例說明
企業官方頻道Geniego / OCLKorea與公司名一致，可信度更高。
全球直播GeniegoGlobal / OCLGlobal在考量海外觀眾時較為合適。
直播電商GeniegoLive / GeniegoCommerce銷售直播、購物直播的目的明確。
技術/平台GeniegoTech / OCLTech適合 IT、SaaS、平台介紹類直播。

5. 設定頻道個人資料

1

進入設定選單

點擊右上角的個人資料圖示，然後進入 Settings。

Profile → Settings
2

上傳個人資料圖片

將公司標誌或品牌圖示設為個人資料圖片。

Profile Picture
3

上傳頻道橫幅

上傳包含品牌色與服務訊息的橫幅圖片。

Profile Banner
4

填寫簡介文案

填寫公司簡介、服務說明、聯絡電子郵件與網站。

Bio / Links

6. 設定兩步驟驗證

要在 Twitch 上穩定使用直播與安全設定，2FA 設定非常重要。

1

進入安全選單

在設定中開啟 Security and Privacy 選單。

Settings → Security and Privacy
2

啟用 2FA

啟用 Two-Factor Authentication。

Set Up Two-Factor Authentication
3

連接手機或驗證器應用程式

連接簡訊驗證或 Google Authenticator、Authy 等驗證器應用程式。

Authenticator App
注意： 企業帳戶可能由多人使用，請務必管理好管理員權限與復原電子郵件。

7. 確認串流金鑰並連接 OBS

1

開啟創作者儀表板

從個人資料選單進入 Creator Dashboard。

Creator Dashboard
2

進入串流選單

在設定選單中選擇 Stream 項目。

Settings → Stream
3

確認 Primary Stream Key

複製主串流金鑰。此金鑰必須像密碼一樣妥善保管。

Primary Stream Key
4

連接到 OBS

在 OBS Studio 中服務選擇 Twitch，串流金鑰填入複製的值。

OBS → Settings → Stream → Twitch
串流金鑰一旦外洩，他人就能在您的頻道上直播。切勿公開。

9. 在 Twitch Developer Console 註冊

要建立聊天機器人、觀眾分析、直播狀態查詢與自動化功能，必須註冊 Twitch Developer 應用程式。

1

前往 Twitch Developer

前往 Twitch 開發者網站。

https://dev.twitch.tv
2

進入 Console

在頂部選單點擊 Console 或 Developer Console。

Developer Console
3

選擇 Applications 選單

在左側或頂部選單選擇 Applications。

Applications
4

點擊 Register Your Application

點擊註冊新應用程式的按鈕。

Register Your Application

10. 填寫 Twitch API 應用程式註冊資訊

輸入項目輸入範例說明
NameGeniego Twitch App應用程式名稱。依服務名填寫。
OAuth Redirect URLshttps://yourdomain.com/auth/twitch/callbackOAuth 登入後返回的伺服器位址。
CategoryWebsite Integration / Chat Bot選擇與應用程式用途相符的分類。
Client TypeConfidential適合在伺服器端安全保管 Secret 的一般服務。
註冊完成後會顯示 Client ID，並可透過 New Secret 按鈕產生 Client Secret。

11. 確認 Client ID / Client Secret

1

選擇已註冊的應用程式

在 Developer Console 的 Applications 清單中點擊剛建立的應用程式。

Applications → 選擇應用程式
2

複製 Client ID

複製畫面上顯示的 Client ID 並儲存到伺服器設定中。

Client ID
3

產生 Client Secret

點擊 New Secret 按鈕產生 Client Secret 並安全儲存。

New Secret
Client Secret 通常只能查看一次。產生後請立即儲存到安全的地方。

12. Twitch API 認證結構

Twitch Channel │ ▼ Developer Console │ ▼ Application │ ▼ Client ID / Client Secret │ ▼ OAuth Token │ ▼ Twitch Helix API │ ▼ 頻道資訊、直播狀態、聊天機器人、觀眾分析

13. OAuth 權杖簽發流程

1) 產生 Twitch OAuth 授權 URL 2) 使用者以 Twitch 帳戶登入 3) 在 Redirect URL 接收 authorization code 4) 以 authorization code + Client Secret 請求 Access Token 5) 以 Access Token 呼叫 Twitch API 6) 必要時以 Refresh Token 更新
簡單的公開資料查詢可使用 App Access Token，而需要使用者權限的操作則需要 User Access Token。

14. 企業營運推薦配置

用途推薦配置說明
直播電商Twitch + YouTube Live + Facebook Live 同時串流將同一直播推送到多個頻道以提高觸及率。
客戶溝通Twitch Chat Bot自動化聊天自動回覆、商品連結引導與優惠券提示。
全球營運AI 字幕 + 多語言翻譯支援韓語、英語、日語等多語言觀眾。
成效分析Twitch API + 內部儀表板分析追蹤者、直播狀態與觀眾反應。

15. 最終檢查清單

  • 已使用企業電子郵件建立 Twitch 帳戶。
  • 已設定與品牌名一致的使用者名稱。
  • 已完成電子郵件驗證。
  • 已設定個人資料圖片、橫幅、簡介文案與連結。
  • 已啟用兩步驟驗證。
  • 已在 Creator Dashboard 確認 Stream Key。
  • 已將 OBS Studio 與 Twitch 頻道連接。
  • 已在 Twitch Developer Console 註冊應用程式。
  • 已取得 Client ID 與 Client Secret。
  • 已準備好 OAuth Redirect URL 與權杖簽發流程。
