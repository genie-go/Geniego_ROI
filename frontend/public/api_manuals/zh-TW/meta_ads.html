Meta Ads
API Key 申請手冊

本手冊手把手指導新手在 Meta 企業應用程式中簽發系統使用者存取權杖，並連同廣告帳戶 ID(act_)一起註冊到 GenieGo 整合中心，從而接入 Meta Ads 廣告資料。

預計時間約 10~20 分鐘
所需帳戶Facebook 個人帳戶 · Meta 廣告帳戶
簽發金鑰存取權杖 · 廣告帳戶 ID(act_)
成果Marketing API 憑證

1. 開始之前

Meta Ads 整合需要一個企業帳戶和一個正在投放廣告的廣告帳戶。

必需

Facebook 個人帳戶 · 廣告帳戶

您需要一個 Facebook 個人帳戶和一個正在投放廣告的 Meta 廣告帳戶。

簽發金鑰

存取權杖 · 廣告帳戶 ID(act_)

您將取得系統使用者權杖(存取權杖)以及以 act_ 開頭的數字型廣告帳戶 ID。

注意: 存取您本企業以外的資料需要應用程式審查(App Review)。如果您僅連接本企業內的廣告帳戶，則無需審查即可進行。

2. 簽發步驟

依以下順序簽發並在 GenieGo 中註冊。

1

建立並驗證企業帳戶

在瀏覽器中開啟網站並登入，然後透過[建立企業]建立企業帳戶並完成電子郵件驗證。

business.facebook.com
2

建立應用程式

點擊右上角的[我的應用程式] > [建立應用程式]，選擇「企業」類型並建立應用程式。

developers.facebook.com → 我的應用程式 → 建立應用程式
3

新增 Marketing API

在應用程式儀表板中找到「Marketing API」卡片，點擊[設定]將其作為產品新增。

4

建立系統使用者並指派權限

進入企業設定 > [使用者] > [系統使用者] > [新增]建立系統使用者，然後在[指派資產]中授予廣告帳戶權限。

business.facebook.com/settings → 使用者 → 系統使用者
5

產生存取權杖

在同一畫面點擊[產生權杖]，選擇您的應用程式，勾選權限(ads_read, ads_management),產生權杖並複製其值。

6

查詢廣告帳戶 ID

在廣告管理員中找到以 act_ 開頭的數字型廣告帳戶 ID。

adsmanager.facebook.com
當權杖在勾選了 ads_read 和 ads_management 權限的狀態下產生時，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
access_token簽發的系統使用者存取權杖(請妥善保管)
ad_account_id以 act_ 開頭的廣告帳戶 ID
在 GenieGo 整合中心 > 對應渠道的[註冊]中，貼上存取權杖和廣告帳戶 ID(act_)並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
因權限錯誤無法查詢資料權杖權限(ads_read·ads_management)未勾選，或未指派廣告帳戶權限為系統使用者指派廣告帳戶權限，並在產生權杖時勾選 ads_read·ads_management 權限。
存取其他企業的資料遭拒應用程式審查(App Review)未完成如需存取本企業以外的資料，請申請並完成應用程式審查。
廣告帳戶 ID 格式錯誤缺少 act_ 前綴，或僅輸入了數字在廣告管理員中找到以 act_ 開頭的完整 ID 並輸入。

最終檢查清單

  • 已在 business.facebook.com 建立企業帳戶並完成電子郵件驗證。
  • 已在 developers.facebook.com 建立企業類型應用程式並新增 Marketing API。
  • 已建立系統使用者並指派廣告帳戶權限。
  • 已使用 ads_read·ads_management 權限產生並複製存取權杖。
  • 已找到以 act_ 開頭的廣告帳戶 ID。
  • 已在 GenieGo 的[註冊]中輸入 access_token·ad_account_id 並儲存。
