DHL Express
API Key 申請手冊

本手冊手把手指導新手在 DHL Express 開發者入口網站(developer.dhl.com)上簽發 API Key 和 Secret，並連同 DHL Express 客戶帳號一起註冊到 GenieGo 整合中心。

預計時間約 10~15 分鐘(審核等待另計)
所需帳戶DHL Express 客戶帳戶(帳號)
簽發金鑰API Key · Secret
成果API 憑證 · 帳號

1. 開始之前

DHL Express 憑證需在開發者入口網站上申請，並需要客戶帳號。

準備物

DHL Express 客戶帳號

擁有 DHL Express 客戶帳號可使憑證申請更順暢。在 GenieGo 註冊時也需一併輸入帳號。

簽發金鑰

API Key · Secret

您將在入口網站上透過自己的應用程式(公司名稱)查看 API Key 和 Secret。

注意: 透過 [Get Access] 註冊客戶資訊並申請憑證後，通常會在下一個工作日收到 Test/Production 審核郵件。

2. 簽發步驟

依以下順序簽發並在 GenieGo 中註冊。

1

註冊開發者入口網站

存取 DHL Express 開發者入口網站，使用電子郵件註冊並驗證。

https://developer.dhl.com
2

申請憑證

登入後，在 [Get Access] 部分註冊客戶資訊並申請憑證。

Get Access
3

接收審核郵件

通常會在下一個工作日收到 Test/Production 審核郵件。

4

查看 API Key·Secret

在入口網站上進入自己的應用程式(公司名稱) > [Show Key] 查看 API Key 和 Secret。

自己的應用程式(公司名稱) → Show Key
收到審核郵件並在入口網站上看到 API Key·Secret 時，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
api_key在入口網站上查看的 DHL API Key
account_numberDHL Express 客戶帳號
在 GenieGo 整合中心 [註冊] 中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
收不到審核郵件等待下一個工作日審核處理，或未註冊客戶資訊在 [Get Access] 中確認客戶資訊註冊和憑證申請是否已完成，並依工作日等待。
呼叫時認證錯誤Test/Production 環境混淆，或缺少帳號重新確認已審核環境的 API Key·Secret 和帳號(account_number)。

最終檢查清單

  • 已確認 DHL Express 客戶帳號。
  • 已在 developer.dhl.com 註冊並透過 [Get Access] 申請憑證。
  • 已在下一個工作日收到 Test/Production 審核郵件。
  • 已在入口網站上查看 API Key·Secret。
  • 已在 GenieGo [註冊] 中輸入 api_key·account_number 並儲存。
