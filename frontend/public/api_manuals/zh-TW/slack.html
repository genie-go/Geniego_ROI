Slack Webhook
API Key 申請手冊

本手冊介紹如何在 Slack 中建立應用程式、簽發 Incoming Webhook URL，並將其註冊到 GenieGo 整合中心，從而將通知傳送到您指定的 Slack 頻道。

預計時間約 5 分鐘
所需帳戶Slack workspace 管理權限
簽發金鑰Webhook URL
成果Incoming Webhook 憑證

1. 開始之前

只有擁有 workspace 管理權限才能建立 Slack Webhook。

必需

workspace 管理權限

您需要擁有 Slack workspace 的管理權限，才能建立應用程式並啟用 Webhook。

簽發金鑰

Webhook URL

啟用 Incoming Webhooks 並選擇頻道後，查看生成的 Webhook URL。

注意: Webhook URL 與接收通知的頻道綁定。請在選擇頻道的步驟中指定要接收通知的頻道。

2. 簽發步驟

依以下順序簽發並在 GenieGo 中註冊。

1

建立新應用程式

點擊 [Create New App]。

https://api.slack.com/apps
2

啟用 Incoming Webhooks

啟用 [Incoming Webhooks] 並點擊 [Add New Webhook to Workspace]。

3

選擇頻道 · 生成 URL

選擇頻道後即會生成 Webhook URL。

選擇頻道並生成 Webhook URL 後，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
webhook_url生成的 Incoming Webhook URL(請妥善保管)
在 GenieGo [註冊] 中輸入 webhook_url 並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
無法建立應用程式workspace 管理權限不足請使用擁有 Slack workspace 管理權限的帳戶進行操作。
收不到通知Webhook URL 填寫錯誤或未選擇頻道啟用 Incoming Webhooks，選擇頻道，並重新核對生成的 Webhook URL。

最終檢查清單

  • 已使用 Slack workspace 管理權限進行操作。
  • 已在 api.slack.com/apps 透過 [Create New App] 建立應用程式。
  • 已啟用 Incoming Webhooks 並點擊 [Add New Webhook]。
  • 已選擇頻道並生成、複製 Webhook URL。
  • 已在 GenieGo [註冊] 中輸入 webhook_url 並儲存。
