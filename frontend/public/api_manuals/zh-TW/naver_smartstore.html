Naver Smart Store
API 金鑰申請手冊

本手冊介紹如何在 Naver Commerce API 中心（apicenter.commerce.naver.com）註冊應用程式以取得 Client ID 與 Client Secret，並將其註冊到 GenieGo 整合中心的完整步驟。

預計時間約 10～15 分鐘
所需帳戶具有「綜合管理員」權限的智慧商店帳戶
申請的金鑰Client ID · Client Secret
成果Commerce API 憑證

1. 開始之前

只有智慧商店賣家帳戶的綜合管理員權限才能進行申請。

必需

綜合管理員權限

您必須擁有智慧商店賣家帳戶的「綜合管理員」權限。只有綜合管理員才能申請金鑰。

申請的金鑰

Client ID · Client Secret

註冊應用程式後，將核發應用程式 ID（Client ID）與 Secret（Client Secret）。

提示: 必須註冊將呼叫 API 的伺服器 IP，並定期更新應用程式認證。

2. 申請步驟

依下列順序申請並在 GenieGo 中註冊。

1

存取 Commerce API 中心

使用綜合管理員帳戶登入 Naver Commerce API 中心。

https://apicenter.commerce.naver.com
2

建立帳戶

點選右上角的 [建立帳戶]，輸入開發公司帳戶名稱與故障應對聯絡人，同意條款後註冊。

建立帳戶
3

註冊應用程式

在 [我的商店應用程式] 中註冊應用程式，隨後將核發應用程式 ID 與 Secret。

我的商店應用程式
4

註冊 IP 並更新認證

註冊將呼叫 API 的伺服器 IP，並依指引定期更新應用程式認證。

註冊應用程式並核發應用程式 ID 與 Secret 後，即表示處理正確。

3. 要在 GenieGo 中註冊的資訊

欄位說明
client_id已核發的應用程式 ID（Client ID）
client_secret已核發的 Secret（Client Secret，請妥善保管）
在 GenieGo 整合中心的 [註冊] 中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在檢視時進行遮罩處理。

4. 常見問題

問題原因解決
無法註冊應用程式帳戶沒有綜合管理員權限使用具有綜合管理員權限的帳戶存取 apicenter.commerce.naver.com。
呼叫時認證錯誤呼叫 IP 未註冊或遺漏認證更新註冊呼叫伺服器的 IP 並定期更新應用程式認證。

最終檢查清單

  • 已使用綜合管理員帳戶存取 Commerce API 中心。
  • 已透過 [建立帳戶] 輸入開發公司資訊與故障應對聯絡人並同意條款。
  • 已在 [我的商店應用程式] 中註冊應用程式並取得 ID/Secret。
  • 已註冊呼叫伺服器的 IP 並設定定期認證。
  • 已在 GenieGo [註冊] 中輸入 client_id·client_secret 並儲存。
