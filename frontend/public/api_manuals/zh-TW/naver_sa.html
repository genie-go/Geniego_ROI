Naver Search Ads
API 金鑰申請手冊

本手冊以新手也能直接照做的方式,整理了在 Naver Search Ad 管理工具中申請 API 認證資訊(存取授權、密鑰)和 CUSTOMER_ID,並將其註冊到 GenieGo 整合中心的完整流程。

預計時間約 5–10 分鐘
所需帳戶Naver Search Ad 廣告主帳戶
申請的金鑰存取授權 · 密鑰 · CUSTOMER_ID
成果Search Ad API 認證資訊

1. 開始之前

Naver Search Ad API 僅可透過廣告主帳戶申請。

必需

Search Ad 廣告主帳戶

您需要一個使用 Naver Search Ad 管理工具的廣告主帳戶。使用一般的 Naver 會員帳戶無法申請 API 存取權限。

申請的金鑰

存取授權 · 密鑰 · CUSTOMER_ID

申請 API 存取權限後,確認一併核發的 CUSTOMER_ID、存取授權和密鑰。

注意: 密鑰僅在申請時可見,請在申請後立即妥善保管。

2. API 金鑰申請步驟

請依下列順序申請並在 GenieGo 中註冊。

1

登入 Search Ads

使用廣告主帳戶登入 Naver Search Ad 管理工具。

searchad.naver.com
2

前往 API 使用管理

從頂部的[工具]選單進入[API 使用管理]。

工具 → API 使用管理
3

申請 API 服務

點擊[申請 Naver Search Ad API 服務]按鈕進行申請。

申請 Naver Search Ad API 服務
4

確認核發的資訊

確認核發的 CUSTOMER_ID、存取授權和密鑰。

當 CUSTOMER_ID、存取授權和密鑰全部確認無誤後,即表示已正常核發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
api_key核發的存取授權
api_secret核發的密鑰(請謹慎處理)
customer_id核發的 CUSTOMER_ID
在 GenieGo 整合中心的[註冊]中輸入並儲存後,認證資訊將以 AES-256-GCM 加密儲存,並在讀取時進行遮罩處理。

4. 常見問題

問題原因解決
沒有[API 使用管理]選單並非廣告主帳戶,或權限不足請使用使用 Search Ad 管理工具的廣告主帳戶登入。
發生認證錯誤存取授權或密鑰輸入錯誤請重新核對申請頁面上的存取授權、密鑰和 CUSTOMER_ID。

最終檢查清單

  • 已使用 Naver Search Ad 廣告主帳戶登入。
  • 已在[工具 > API 使用管理]中完成 API 服務申請。
  • 已確認 CUSTOMER_ID、存取授權和密鑰。
  • 已在 GenieGo[註冊]中輸入 api_key、api_secret、customer_id 並儲存。
