LINE
API 申請手冊

本手冊整理了在 LINE Developers Console 中建立 Provider 和 Messaging API Channel，並設定 Channel Access Token、Webhook URL 的過程。

驗證結構Channel Access Token
所需帳戶LINE 帳戶
主要成果Channel ID / Secret / Token
用途訊息推送·聊天機器人

1. LINE API 申請概念

LINE 對接是在 LINE Developers Console 中建立 Provider 和 Channel，然後為 Messaging API 或 LINE Login 申請 Channel ID、Channel Secret、Channel Access Token 的方式。

區分: 發送訊息使用 Messaging API 頻道，登入對接使用 LINE Login 頻道。

2. 準備事項

  • LINE 帳戶或企業帳戶
  • LINE Developers Console 存取權限
  • LINE Official Account
  • 服務名稱和說明
  • Webhook URL
  • 隱私權政策 URL 及條款 URL

3. 建立 Provider

1

存取 LINE Developers

登入 LINE Developers Console。

developers.line.biz
2

建立 Provider

使用公司名稱或服務名稱建立 Provider。

Create Provider
3

填寫基本資訊

填寫服務營運主體、聯絡方式和條款資訊。

Provider Settings

4. 建立 Messaging API 頻道

1

建立新頻道

在 Provider 內點擊 Create a new channel。

Create a new channel
2

選擇 Messaging API

在頻道類型中選擇 Messaging API。

Messaging API
3

填寫頻道資訊

填寫頻道名稱、說明、類別、電子郵件等。

Channel Information
4

連結 LINE Official Account

連結用於發送訊息的官方帳號，或新建一個。

LINE Official Account
5

完成頻道建立

建立後確認 Channel ID 和 Channel Secret。

Channel ID / Channel Secret

5. 申請 Channel Access Token

1

切換到 Messaging API 分頁

在建立的頻道頁面切換到 Messaging API 分頁。

Messaging API tab
2

尋找 Channel access token 項目

確認底部的 Channel access token 區域。

Channel access token
3

點擊 Issue

點擊 Issue 或 Reissue 按鈕申請權杖。

Issue
4

安全儲存到伺服器

將申請到的權杖儲存在伺服器環境變數或 Secret Manager 中。

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token 是擁有訊息發送權限的密鑰。不得在 HTML、前端或公開儲存庫中暴露。

6. 設定 Webhook

1

輸入 Webhook URL

輸入用於接收使用者訊息事件的伺服器 URL。

Webhook URL
2

啟用 Use webhook

將 Webhook 使用選項改為 Enabled。

Use webhook: Enabled
3

Verify 測試

從 LINE 向伺服器發送測試請求以確認連線。

Verify
4

確認回應碼

確認伺服器回傳 200 OK。

HTTP 200 OK

7. 訊息發送範例

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "您好。這是 LINE 訊息測試。"} ] }

8. 最終檢查清單

  • 已建立 Provider。
  • 已建立 Messaging API 頻道。
  • 已連結 LINE Official Account。
  • 已確認 Channel ID 和 Channel Secret。
  • 已申請 Channel Access Token。
  • 已註冊 Webhook URL 並通過 Verify。
  • 已將權杖安全儲存到伺服器環境變數。
