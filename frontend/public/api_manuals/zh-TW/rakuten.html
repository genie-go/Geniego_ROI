Rakuten
API Key 申請手冊

本手冊手把手指導新手在 Rakuten RMS 賣家控制台中簽發 RMS Web Service 認證金鑰(Service Secret·License Key),並註冊到 GenieGo 整合中心。

預計時間約 5~15 分鐘
所需帳戶Rakuten 賣家(樂天市場)帳戶
簽發金鑰Service Secret · License Key
成果RMS Web Service 憑證

1. 開始之前

賣家(樂天市場)金鑰在 RMS Web Service 中簽發。

必需

Rakuten 賣家帳戶

您需要一個 Rakuten 賣家(樂天市場)帳戶。您需要一個可以透過 RMS R-Login(glogin.rms.rakuten.co.jp)登入的帳戶。

簽發金鑰

Service Secret · License Key

您將在 RMS Web Service 中簽發 serviceSecret 和 licenseKey。License Key 有 90 天有效期，可以重新簽發。

注意: 簽發賣家(樂天市場)金鑰的 RMS Web Service(glogin.rms.rakuten.co.jp)與用於聯盟/商品搜尋的 webservice.rakuten.co.jp 不同。請務必在 RMS 一側簽發。

2. 簽發步驟

依以下順序簽發並在 GenieGo 中註冊。

1

登入 RMS 開發者入口網站

使用賣家帳戶登入 Rakuten 開發者入口網站(RMS/Webservice)。

https://glogin.rms.rakuten.co.jp
2

進入 RMS Web Service

透過 RMS R-Login 登入後，進入 RMS Web Service 區域。

RMS R-Login → RMS Web Service
3

簽發 Service Secret · License Key

在 RMS Web Service 中簽發 Service Secret 和 License Key。License Key 有 90 天有效期，到期時請重新簽發。

4

確認金鑰

確認已簽發的 Service Secret 和 License Key，以及您本店的 Shop URL。

當 Service Secret 和 License Key 顯示在簽發畫面上時，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
service_secret簽發的 Service Secret
license_key簽發的 License Key(90 天有效期·可重新簽發，請妥善保管)
shop_url您本店的 Shop URL
在 GenieGo 整合中心 [註冊] 中輸入 service_secret·license_key·shop_url 並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
簽發了錯誤的金鑰在 webservice.rakuten.co.jp(用於聯盟/商品搜尋)中簽發在 RMS R-Login(glogin.rms.rakuten.co.jp)的 RMS Web Service 中重新簽發。
一段時間後認證失敗License Key 90 天到期在 RMS Web Service 中重新簽發 License Key，並在 [註冊] 中重新輸入。

最終檢查清單

  • 已使用 Rakuten 賣家(樂天市場)帳戶登入 RMS R-Login。
  • 已在 RMS Web Service 中簽發 Service Secret·License Key。
  • 已確認本店的 Shop URL。
  • 已在 GenieGo [註冊] 中輸入 service_secret·license_key·shop_url 並儲存。
