1. Kakao Alimtalk 申請概念

Alimtalk 是一項透過 Kakao Talk 發送訂單、預約、配送、付款等資訊類訊息的商務訊息服務。它並非僅憑一般 Kakao 開發者應用程式金鑰即可直接發送的架構，而是需要進行企業頻道認證以及發送代理商/平台設定。

重要: Alimtalk 以資訊類訊息為主，而非廣告類訊息，且範本需要事先審查。

2. 準備物品

  • Kakao Business 帳戶
  • Kakao Talk 頻道
  • 營業執照等企業認證資料
  • 要發送的 Alimtalk 範本文案
  • 發送代理商或雲端訊息服務帳戶
  • 收件人電話號碼的蒐集及同意管理政策

3. 準備 Kakao Talk 頻道

1

登入 Kakao Business

登入 Kakao Business 管理員中心。

business.kakao.com
2

建立 Kakao Talk 頻道

以品牌或服務名稱建立頻道。

Kakao Talk 頻道管理員中心
3

進行企業認證

提交企業資訊和證明資料，進行企業頻道認證。

企業認證
4

確認頻道搜尋 ID

確認 Alimtalk 發送設定所需的頻道 ID 或搜尋 ID。

頻道個人檔案 → 搜尋 ID
只有完成企業認證後，才能進行 Alimtalk 發送設定檔的註冊。

4. 發送設定檔 / Sender Profile 註冊

1

選擇發送平台

選擇 NHN Cloud、Naver Cloud SENS、Infobank、Bizgo 等 Alimtalk 供應商。

Messaging Provider
2

建立專案

在所選平台上建立 Alimtalk 專案。

Project / Service
3

連接 Kakao Talk 頻道

註冊已通過企業認證的 Kakao Talk 頻道資訊。

Sender Profile
4

發送設定檔審查

進行頻道所有權確認和發送設定檔審查。

Profile Review
5

確認 API 憑證

確認平台提供的 App Key、Secret Key、API Key 等。

API Key / Secret

5. 範本註冊與審核

1

撰寫範本

撰寫訂單確認、配送通知等資訊類文案。

Template Message
2

設定變數

將姓名、訂單編號、配送單號等動態變化的值設定為變數。

#{變數}
3

提交審查

儲存範本並提交 Kakao 審查申請。

Request Approval
4

確認審核

確認已審核的範本代碼或 ID。

Template Code
如果審查前的文案與實際發送的文案不一致，可能導致發送失敗或受到處罰。

6. API 發送流程範例

1) 從伺服器呼叫 Alimtalk API Endpoint
2) 使用 API Key 或 Secret 進行認證
3) 傳遞收件人電話號碼、範本代碼、變數值
4) 接收發送結果
5) 處理失敗時是否進行 SMS 替代發送

7. 最終檢查清單

  • 已建立 Kakao Talk 頻道。
  • 已完成企業認證。
  • 已選擇 Alimtalk 發送平台。
  • 已註冊 Sender Profile。
  • 已確認 API Key 或 Secret。
  • 已註冊範本並獲得審核。
  • 已準備發送失敗及替代簡訊政策。
