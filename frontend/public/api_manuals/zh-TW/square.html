Square
API Key 申請手冊

本手冊手把手指導新手在 Square 開發者入口網站中建立應用程式以簽發 Access Token 和 Location ID，並將其註冊到 GenieGo 整合中心。

預計時間約 5~10 分鐘
所需帳戶Square 帳戶
簽發金鑰Access Token · Location ID
成果API 憑證

1. 開始之前

Square API 憑證透過在開發者入口網站中建立應用程式來簽發。

必需

Square 帳戶

您需要一個可以在 developer.squareup.com/apps 建立應用程式的 Square 帳戶。

簽發金鑰

Access Token · Location ID

在 Production 分頁中查看 Access Token，在 [Locations] 中查看 Location ID。

注意: 實際整合請使用 Production 權杖，而非 Sandbox 權杖。

2. 簽發步驟

依以下順序簽發並在 GenieGo 中註冊。

1

建立應用程式

在 Square 開發者入口網站中建立應用程式。

developer.squareup.com/apps
2

查看 Access Token

在 Production 分頁中查看 Access Token。

Production
3

查看 Location ID

在 [Locations] 中查看 Location ID。

Locations
確認了 Production 的 Access Token 和 Location ID 後，即表示已正確簽發。

3. 需在 GenieGo 中註冊的資訊

欄位說明
access_token在 Production 分頁中查看到的 Access Token(請妥善保管)
location_id在 [Locations] 中查看到的 Location ID
在 GenieGo 整合中心的 [註冊] 中輸入 access_token·location_id 並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
認證失敗使用了 Sandbox 權杖請使用在 Production 分頁中查看到的 Access Token。
訂單未同步Location ID 填寫錯誤重新確認在 [Locations] 中查看到的 Location ID。

最終檢查清單

  • 已在 developer.squareup.com/apps 建立應用程式。
  • 已在 Production 分頁中查看 Access Token。
  • 已在 [Locations] 中查看 Location ID。
  • 已在 GenieGo 的 [註冊] 中輸入 access_token·location_id 並儲存。
