TNT Express
API Key 申請手冊

TNT Express 已被 FedEx 收購，使用在 FedEx 開發者入口網站(developer.fedex.com)簽發的 API Key·Secret 以及 FedEx 帳戶編號。本手冊整理了將這些憑證註冊到 GenieGo 整合中心的流程。

預計時間約 5~15 分鐘
所需帳戶FedEx 帳戶(帳戶編號)
簽發金鑰API Key · API Secret
成果API 憑證 · 帳戶編號

1. 開始之前

TNT 已整合到 FedEx，沒有獨立的 TNT 開發者入口網站，使用 FedEx Developer Portal 的憑證。

提示: TNT 已被 FedEx 收購，在 FedEx Developer Portal(developer.fedex.com)簽發憑證(API Key/Secret)。沒有獨立的 TNT 開發者入口網站，並且需要 FedEx 帳戶編號。
準備

FedEx 帳戶編號

需要 FedEx 帳戶編號。帳戶編號在 GenieGo 註冊時一併輸入。

簽發金鑰

API Key · API Secret

在 FedEx 開發者入口網站中建立專案後，會簽發 API Key 和 Secret。

3. 需在 GenieGo 中註冊的資訊

欄位說明
api_key簽發的 API Key (Client ID)
api_secret簽發的 API Secret (Client Secret，請妥善保管)
account_numberFedEx 帳戶編號
在 GenieGo 整合中心的 [註冊] 中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
找不到 TNT 專用開發者入口網站TNT 已整合到 FedEx，沒有獨立入口網站在 developer.fedex.com 建立專案以簽發 API Key/Secret。
呼叫時出現認證錯誤Secret 填寫錯誤，或缺少 FedEx 帳戶編號重新確認 api_secret 和帳戶編號(account_number)。

最終檢查清單

  • 已確認 TNT 使用 FedEx 憑證。
  • 已在 developer.fedex.com 註冊並建立專案。
  • 已確認 API Key·Secret 和 FedEx 帳戶編號。
  • 已在 GenieGo 的 [註冊] 中輸入 api_key·api_secret·account_number 並儲存。
