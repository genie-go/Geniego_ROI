Toss Payments
API Key 申請手冊

本手冊整理了在 Toss Payments 商家管理後台的開發者中心查看用戶端金鑰與密鑰（Secret Key），並將其註冊到 GenieGo 整合中心的流程。

預計時間約 5~10 分鐘
所需帳戶Toss Payments 商家合約 · 商家管理帳戶
簽發金鑰用戶端金鑰 · 密鑰（Secret Key）
成果支付整合憑證

1. 開始之前

Toss Payments 在簽訂商家合約後，可在商家管理後台查看金鑰。

必需

商家合約 · 商家管理帳戶

您必須已完成 Toss Payments 商家合約，並且能夠登入商家管理後台。

簽發金鑰

用戶端金鑰 · 密鑰（Secret Key）

在開發者中心成套查看用戶端金鑰與密鑰（Secret Key），並搭配使用。

注意: 開發者中心區分測試與正式（live）環境，因此請確認實際整合使用的是 live 金鑰。

2. 簽發步驟

依以下順序查看並在 GenieGo 中註冊。

1

登入商家管理後台

登入 Toss Payments 商家管理後台。

https://dashboard.tosspayments.com
2

進入開發者中心

進入左下角的 [開發者中心]。

開發者中心
3

查看金鑰

選擇測試/正式（live）環境，查看用戶端金鑰與密鑰（Secret Key）。兩個金鑰需搭配使用。

當用戶端金鑰與密鑰（Secret Key）皆已確認後，即表示已做好註冊準備。

3. 需在 GenieGo 中註冊的資訊

欄位說明
client_key在開發者中心查看到的用戶端金鑰
secret_key在開發者中心查看到的密鑰（Secret Key，請妥善保管）
在 GenieGo 整合中心 [註冊] 中輸入並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
出現認證錯誤混淆了測試金鑰與 live 金鑰在開發者中心確認測試/正式（live）的區分，並使用與環境相符的金鑰。
沒有開發者中心選單商家合約未完成使用已完成商家合約的商家管理帳戶登入。

最終檢查清單

  • 已完成 Toss Payments 商家合約並登入商家管理後台。
  • 已進入左下角的 [開發者中心]。
  • 已區分測試/正式（live）環境並查看用戶端金鑰 · 密鑰（Secret Key）。
  • 已在 GenieGo [註冊] 中輸入 client_key · secret_key 並儲存。
