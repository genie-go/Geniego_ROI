Google Ads
API Key 申請手冊

本手冊介紹如何在 Google Ads 管理員(MCC)帳戶的 API 中心申請開發人員權杖，在 Google Cloud 中建立 OAuth 用戶端以取得 refresh token，並將全部資訊註冊到 GenieGo 整合中心的完整流程。

預計時間約20~40分鐘(開發人員權杖審核另計)
所需帳戶Google Ads 管理員(MCC)帳戶 · Google 帳戶
申請的金鑰開發人員權杖 · client_id/secret · refresh_token · customer_id
成果Google Ads API 憑證

1. 開始之前

Google Ads 整合同時需要管理員(MCC)帳戶和 Google Cloud 中的 OAuth 設定。

開發人員權杖

MCC API 中心

在 Google Ads 管理員(MCC)帳戶的 API 中心申請開發人員權杖。基本(Basic)存取權需經 Google 審核後核准。

OAuth

Google Cloud 用戶端

在 console.cloud.google.com 建立 OAuth 用戶端(client_id·client_secret)並完成驗證以取得 refresh token。

注意： 開發人員權杖的基本存取權需經 Google 審核後核准，核准前只能呼叫測試帳戶。如果沒有管理員(MCC)帳戶，請先在 ads.google.com 建立。

2. 申請步驟

依以下順序申請並註冊到 GenieGo。

1

登入管理員帳戶 · 進入 API 中心

登入 Google Ads 管理員帳戶後，進入頂部的 [工具與設定] > [設定] > [API 中心]。

工具與設定 → 設定 → API 中心
2

申請開發人員權杖

在 API 中心申請 [開發人員權杖]。基本(Basic)存取權需經 Google 審核後核准，核准前只能呼叫測試帳戶。

3

設定 Cloud 專案 · OAuth 同意畫面

建立專案並設定 [API 和服務] > [OAuth 同意畫面]。

console.cloud.google.com → API 和服務 → OAuth 同意畫面
4

建立 OAuth 用戶端 ID

建立 [憑證] > [建立憑證] > [OAuth 用戶端 ID] 以取得 client_id 和 client_secret(範圍 scope: adwords)。

憑證 → 建立憑證 → OAuth 用戶端 ID
5

申請 refresh token

透過 OAuth 驗證流程申請 refresh token(使用 Google OAuth Playground 或自有驗證)。

6

確認客戶 ID

確認要呼叫的廣告帳戶的10位客戶 ID(customer_id)。

開發人員權杖核准 + client_id·client_secret·refresh_token·customer_id 全部準備就緒後，即表示申請成功。

3. 需在 GenieGo 註冊的資訊

欄位說明
developer_token在 API 中心申請的開發人員權杖
client_idOAuth 用戶端 ID
client_secretOAuth 用戶端密鑰(注意外洩)
refresh_token透過 OAuth 驗證申請的 refresh token(填入 [存取權杖] 欄)
customer_id目標廣告帳戶的10位客戶 ID
在 GenieGo [註冊] 中輸入開發人員權杖、client_id、client_secret、refresh_token、customer_id 並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決
只能呼叫測試帳戶開發人員權杖基本存取權審核未核准請等待在 API 中心申請的開發人員權杖通過 Google 審核核准後，再呼叫正式帳戶。
驗證失敗缺少 scope(adwords)或 refresh_token 填寫錯誤在 OAuth 用戶端中包含 adwords 範圍後重新驗證，並重新確認 refresh_token。
客戶 ID 格式錯誤10位 customer_id 填寫錯誤重新確認目標廣告帳戶的10位客戶 ID 並輸入。

最終檢查清單

  • 已準備 Google Ads 管理員(MCC)帳戶並登入。
  • 已在 API 中心申請開發人員權杖(基本存取權需經 Google 審核)。
  • 已在 console.cloud.google.com 建立 OAuth 同意畫面和 OAuth 用戶端 ID/Secret(scope: adwords)。
  • 已透過 OAuth 驗證申請 refresh token。
  • 已確認目標帳戶的10位 customer_id。
  • 已在 GenieGo [註冊] 中輸入開發人員權杖·client_id/secret·refresh_token·customer_id 並儲存。
