Adyen
API Key 申請手冊

本手冊整理了在 Adyen 商家合約與審查核准後，於 Customer Area 中簽發 API 金鑰(X-API-Key)和 Merchant Account，並註冊到 GenieGo 整合中心的流程。

預計時間審查核准後約 5~15 分鐘
所需資格Adyen 商家合約·審查核准
簽發金鑰API 金鑰(X-API-Key) · Merchant Account
成果API 憑證

1. 開始之前

Adyen 是面向企業的全球支付閘道，在簽發憑證之前必須先完成商家合約與審查核准。

必需

商家合約·審查核准

Adyen 是面向企業的全球 PG，需要簽訂商家合約並通過審查核准。只有核准後才能簽發憑證。

簽發金鑰

X-API-Key · Merchant Account

核准後，於 Customer Area 的 [Developers] > [API credentials] 中產生 API 金鑰(X-API-Key)並確認 Merchant Account 名稱。

前置條件: 在商家合約與審查核准之前無法自助簽發。您必須先與 Adyen 完成商家合約與審查。

2. 申請流程

審查核准後，依以下順序簽發並註冊憑證。

1

商家合約·審查核准

與 Adyen 簽訂商家合約並取得審查核准。

Adyen 業務/合約
2

登入 Customer Area

核准後登入 Customer Area。

https://ca-live.adyen.com
3

簽發 API 憑證

於 [Developers] > [API credentials] 中產生 API 金鑰(X-API-Key)並確認 Merchant Account 名稱。

Developers → API credentials
4

確認起始結算批次號

為進行結算的首次擷取，請於 Customer Area 中確認最新的 settlement batch 編號。

當確認了 API 金鑰(X-API-Key)和 Merchant Account 後，即表示憑證已準備就緒。

3. 需在 GenieGo 中註冊的資訊

欄位說明
api_key簽發的 API 金鑰(X-API-Key，請妥善保管)
merchant_account已確認的 Merchant Account 名稱
batch_start用於首次結算擷取的起始結算批次號(Customer Area 中最新的 settlement batch 編號)
在 GenieGo 整合中心的 [註冊] 中輸入 api_key·merchant_account·batch_start 並儲存後，憑證將以 AES-256-GCM 加密儲存，並在查詢時進行遮罩處理。

4. 常見問題

問題原因解決方法
沒有 API credentials 選單商家合約·審查未核准與 Adyen 完成商家合約·審查後，再存取 Customer Area。
認證失敗X-API-Key 或 Merchant Account 填寫錯誤於 Developers > API credentials 中重新確認簽發的 API 金鑰和 Merchant Account 名稱。

最終檢查清單

  • 已取得 Adyen 商家合約·審查核准。
  • 已登入 Customer Area(ca-live.adyen.com)。
  • 已於 Developers > API credentials 中確認 API 金鑰(X-API-Key)和 Merchant Account。
  • 已確認最新的 settlement batch 編號(batch_start)。
  • 已於 GenieGo 的 [註冊] 中輸入 api_key·merchant_account·batch_start 並儲存。
