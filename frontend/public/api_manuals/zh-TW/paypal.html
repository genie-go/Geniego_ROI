PayPal
API Key 申請手冊

本手冊詳細說明如何在 PayPal 開發者入口網站建立應用程式（App）以申請 Client ID 與 Secret，並將其註冊至 GenieGo 整合中心，即使是初學者也能照著完成。

所需時間約 5~10 分鐘
所需帳戶PayPal 商業帳戶
申請金鑰Client ID · Secret
產出API 憑證

1. 開始之前

PayPal API 憑證是使用商業帳戶在開發者入口網站建立應用程式後申請的。

必需

PayPal 商業帳戶

您需要擁有可登入 developer.paypal.com 的 PayPal 商業帳戶。

申請金鑰

Client ID · Secret

建立 Live 應用程式後會申請到 Client ID 與 Secret。

提示: 進行實際整合時，請在 [Live] 標籤頁而非 Sandbox 中建立應用程式。

2. 申請步驟

依以下順序申請並註冊至 GenieGo。

1

登入開發者入口網站

使用 PayPal 商業帳戶登入後，前往 [Apps & Credentials]。

developer.paypal.com → Apps & Credentials
2

建立 Live 應用程式

在 [Live] 標籤頁中點選 [Create App] 建立應用程式。

Live → Create App
3

確認憑證

確認申請到的 Client ID 與 Secret。

顯示出 Client ID 與 Secret 即表示已正常申請成功。

3. 需在 GenieGo 註冊的資訊

欄位說明
client_id申請到的 Client ID
client_secret申請到的 Secret（注意保密）
在 GenieGo 整合中心的 [註冊] 中輸入 client_id·client_secret 並儲存後，憑證將以 AES-256-GCM 加密儲存，查詢時會進行遮罩處理。

4. 常見問題

問題原因解決
認證失敗使用了 Sandbox 應用程式的憑證請使用在 [Live] 標籤頁中建立的應用程式的 Client ID·Secret。
無法建立應用程式使用一般（個人）帳戶登入請使用 PayPal 商業帳戶登入。

最終檢查清單

  • 已使用 PayPal 商業帳戶登入 developer.paypal.com。
  • 已在 [Apps & Credentials] 中前往 [Live] 標籤頁。
  • 已透過 [Create App] 建立應用程式並確認 Client ID·Secret。
  • 已在 GenieGo [註冊] 中輸入 client_id·client_secret 並儲存。
