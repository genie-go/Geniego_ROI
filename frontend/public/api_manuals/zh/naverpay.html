Naver Pay
API Key 申请手册

Naver Pay 是一种在完成商户注册(签约)后由专员向您交付凭证的渠道。本手册整理了将收到的 Client ID、Client Secret 和 Chain ID 注册到 GenieGo 集成中心的流程。

预计时间因签约与审核周期而异
所需账户Naver Pay 商家中心账户
签发密钥Client ID · Client Secret · Chain ID
成果支付集成凭证

1. 开始之前

Naver Pay 不是自助签发，而是在完成商户签约后交付凭证的方式。

Naver Pay 不是在控制台中直接签发密钥的方式。在商户注册(签约)完成后，由专员向您交付凭证。
重要: 这不是普通的自助签发。在商户注册(签约)完成后，会向您交付合作伙伴ID、Client ID/Secret。

2. 申请签发流程

按以下顺序进行签约，并注册收到的凭证。

1

申请商户签约

在 Naver Pay 商家中心申请并完成商户签约。

2

完成审核与签约

持续进行直到审核与签约完成。周期可能因审核安排而有所不同。

3

接收凭证

审核与签约完成后，由专员向您交付 Client ID、Client Secret 和 Chain ID。

当您从专员处收齐 Client ID、Client Secret 和 Chain ID 后，即表示已准备好进行注册。

3. 需在 GenieGo 中注册的信息

字段说明
client_id收到的 Client ID
client_secret收到的 Client Secret(请妥善保管)
chain_id收到的 Chain ID
在 GenieGo 集成中心的[注册]中输入并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
找不到签发凭证的菜单Naver Pay 不是自助签发方式使用在商户签约完成后由专员交付给您的凭证。
认证失败Client Secret 或 Chain ID 填写错误重新确认专员交付给您的 Client ID、Client Secret 和 Chain ID。

最终检查清单

  • 已在 Naver Pay 商家中心申请并完成商户签约。
  • 审核与签约完成后，已从专员处收到 Client ID、Client Secret 和 Chain ID。
  • 已确认收到的全部三项凭证。
  • 已在 GenieGo 的[注册]中输入 client_id·client_secret·chain_id 并保存。
