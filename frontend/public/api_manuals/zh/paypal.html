PayPal
API Key 申请手册

本手册详细说明了如何在 PayPal 开发者门户中创建应用（App）以申请 Client ID 和 Secret，并将其注册到 GenieGo 集成中心，即使是初学者也能照做完成。

所需时间约 5~10 分钟
所需账户PayPal 商业账户
申请密钥Client ID · Secret
产出API 凭证

1. 开始之前

PayPal API 凭证是使用商业账户在开发者门户中创建应用后申请的。

必需

PayPal 商业账户

您需要拥有可登录 developer.paypal.com 的 PayPal 商业账户。

申请密钥

Client ID · Secret

创建 Live 应用后会申请到 Client ID 和 Secret。

提示: 进行实际集成时，请在 [Live] 标签页而非 Sandbox 中创建应用。

2. 申请步骤

按以下顺序申请并注册到 GenieGo。

1

登录开发者门户

使用 PayPal 商业账户登录后，转到 [Apps & Credentials]。

developer.paypal.com → Apps & Credentials
2

创建 Live 应用

在 [Live] 标签页中点击 [Create App] 创建应用。

Live → Create App
3

确认凭证

确认申请到的 Client ID 和 Secret。

显示出 Client ID 和 Secret 即表示已正常申请成功。

3. 需在 GenieGo 注册的信息

字段说明
client_id申请到的 Client ID
client_secret申请到的 Secret（注意保密）
在 GenieGo 集成中心的 [注册] 中输入 client_id·client_secret 并保存后，凭证将以 AES-256-GCM 加密存储，查询时会进行掩码处理。

4. 常见问题

问题原因解决
认证失败使用了 Sandbox 应用的凭证请使用在 [Live] 标签页中创建的应用的 Client ID·Secret。
无法创建应用使用普通（个人）账户登录请使用 PayPal 商业账户登录。

最终检查清单

  • 已使用 PayPal 商业账户登录 developer.paypal.com。
  • 已在 [Apps & Credentials] 中转到 [Live] 标签页。
  • 已通过 [Create App] 创建应用并确认 Client ID·Secret。
  • 已在 GenieGo [注册] 中输入 client_id·client_secret 并保存。
