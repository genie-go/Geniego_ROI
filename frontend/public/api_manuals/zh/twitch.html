1. 了解 Twitch 账户结构

与 Instagram 不同，Twitch 没有单独选择企业账户/创作者账户的结构。

企业官方频道

用于品牌运营

用公司名或品牌名创建一个普通 Twitch 账户，然后像官方频道一样运营。适合产品介绍、直播电商和活动直播。

主播成长阶段

Affiliate / Partner

满足直播业绩、粉丝和观看时长条件后，即可加入 Affiliate 或 Partner 计划。

核心： 企业也应先创建普通 Twitch 账户，然后进行频道品牌化、两步验证、OBS 连接以及 Developer Console 应用注册。

2. 准备物品

  • 企业邮箱地址
  • 品牌名或公司名
  • Twitch 用户名候选
  • 公司徽标和频道横幅图片
  • 公司简介文案和网站 URL
  • 用于直播的 PC 或笔记本电脑
  • OBS Studio 或推流软件
  • 用于 API 集成的 OAuth Redirect URL

3. 创建 Twitch 账户

1

访问 Twitch

在浏览器中访问 Twitch 官方网站。

https://www.twitch.tv
2

点击注册

点击右上角的 Sign Up 或注册按钮。

Sign Up
3

使用企业信息注册

输入公司邮箱、用户名、密码和出生日期。

Email / Username / Password
4

邮箱验证

打开 Twitch 发送的验证邮件并完成 Verify Email。

Verify Email

4. 确定品牌账户名

Twitch 用户名会成为频道地址，因此最好选择简短易记的名称。

用途推荐名称示例说明
企业官方频道Geniego / OCLKorea与公司名一致，可信度更高。
全球直播GeniegoGlobal / OCLGlobal在考虑海外观众时较为合适。
直播电商GeniegoLive / GeniegoCommerce销售直播、购物直播的目的明确。
技术/平台GeniegoTech / OCLTech适合 IT、SaaS、平台介绍类直播。

5. 设置频道资料

1

进入设置菜单

点击右上角的个人资料图标，然后进入 Settings。

Profile → Settings
2

上传头像

将公司徽标或品牌图标设为头像。

Profile Picture
3

上传频道横幅

上传包含品牌色和服务信息的横幅图片。

Profile Banner
4

填写简介文案

填写公司简介、服务说明、联系邮箱和网站。

Bio / Links

6. 设置两步验证

要在 Twitch 上稳定使用直播和安全设置，2FA 设置非常重要。

1

进入安全菜单

在设置中打开 Security and Privacy 菜单。

Settings → Security and Privacy
2

启用 2FA

启用 Two-Factor Authentication。

Set Up Two-Factor Authentication
3

连接手机或验证器应用

连接短信验证或 Google Authenticator、Authy 等验证器应用。

Authenticator App
注意： 企业账户可能由多人使用，请务必管理好管理员权限和恢复邮箱。

7. 确认推流密钥并连接 OBS

1

打开创作者面板

从个人资料菜单进入 Creator Dashboard。

Creator Dashboard
2

进入推流菜单

在设置菜单中选择 Stream 项。

Settings → Stream
3

确认 Primary Stream Key

复制主推流密钥。此密钥必须像密码一样妥善保管。

Primary Stream Key
4

连接到 OBS

在 OBS Studio 中服务选择 Twitch，推流密钥填入复制的值。

OBS → Settings → Stream → Twitch
推流密钥一旦泄露，他人就能在您的频道上直播。切勿公开。

9. 在 Twitch Developer Console 注册

要构建聊天机器人、观众分析、直播状态查询和自动化功能，必须注册 Twitch Developer 应用。

1

访问 Twitch Developer

访问 Twitch 开发者网站。

https://dev.twitch.tv
2

进入 Console

在顶部菜单点击 Console 或 Developer Console。

Developer Console
3

选择 Applications 菜单

在左侧或顶部菜单选择 Applications。

Applications
4

点击 Register Your Application

点击注册新应用的按钮。

Register Your Application

10. 填写 Twitch API 应用注册信息

输入项输入示例说明
NameGeniego Twitch App应用名称。按服务名填写。
OAuth Redirect URLshttps://yourdomain.com/auth/twitch/callbackOAuth 登录后返回的服务器地址。
CategoryWebsite Integration / Chat Bot选择与应用用途相符的分类。
Client TypeConfidential适合在服务器端安全保管 Secret 的一般服务。
注册完成后会显示 Client ID，并可通过 New Secret 按钮生成 Client Secret。

11. 确认 Client ID / Client Secret

1

选择已注册的应用

在 Developer Console 的 Applications 列表中点击刚创建的应用。

Applications → 选择应用
2

复制 Client ID

复制屏幕上显示的 Client ID 并保存到服务器设置中。

Client ID
3

生成 Client Secret

点击 New Secret 按钮生成 Client Secret 并安全保存。

New Secret
Client Secret 通常只能查看一次。生成后请立即保存到安全的地方。

12. Twitch API 认证结构

Twitch Channel │ ▼ Developer Console │ ▼ Application │ ▼ Client ID / Client Secret │ ▼ OAuth Token │ ▼ Twitch Helix API │ ▼ 频道信息、直播状态、聊天机器人、观众分析

13. OAuth 令牌签发流程

1) 生成 Twitch OAuth 授权 URL 2) 用户用 Twitch 账户登录 3) 在 Redirect URL 接收 authorization code 4) 用 authorization code + Client Secret 请求 Access Token 5) 用 Access Token 调用 Twitch API 6) 必要时用 Refresh Token 刷新
简单的公开数据查询可使用 App Access Token，而需要用户权限的操作则需要 User Access Token。

14. 企业运营推荐配置

用途推荐配置说明
直播电商Twitch + YouTube Live + Facebook Live 同时推流将同一直播推送到多个频道以提高触达率。
客户沟通Twitch Chat Bot自动化聊天自动回复、商品链接引导和优惠券提示。
全球运营AI 字幕 + 多语言翻译支持韩语、英语、日语等多语言观众。
成效分析Twitch API + 内部仪表板分析粉丝、直播状态和观众反应。

15. 最终检查清单

  • 已使用企业邮箱创建 Twitch 账户。
  • 已设置与品牌名一致的用户名。
  • 已完成邮箱验证。
  • 已设置头像、横幅、简介文案和链接。
  • 已启用两步验证。
  • 已在 Creator Dashboard 确认 Stream Key。
  • 已将 OBS Studio 与 Twitch 频道连接。
  • 已在 Twitch Developer Console 注册应用。
  • 已获取 Client ID 和 Client Secret。
  • 已准备好 OAuth Redirect URL 和令牌签发流程。
