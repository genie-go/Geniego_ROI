Google Ads
API Key 申请手册

本手册介绍如何在 Google Ads 管理员(MCC)账户的 API 中心申请开发者令牌，在 Google Cloud 中创建 OAuth 客户端以获取 refresh token，并将全部信息注册到 GenieGo 集成中心的完整流程。

预计时间约20~40分钟(开发者令牌审核另计)
所需账户Google Ads 管理员(MCC)账户 · Google 账户
申请的密钥开发者令牌 · client_id/secret · refresh_token · customer_id
成果Google Ads API 凭据

1. 开始之前

Google Ads 集成同时需要管理员(MCC)账户和 Google Cloud 中的 OAuth 设置。

开发者令牌

MCC API 中心

在 Google Ads 管理员(MCC)账户的 API 中心申请开发者令牌。基础(Basic)访问权限需经 Google 审核后批准。

OAuth

Google Cloud 客户端

在 console.cloud.google.com 创建 OAuth 客户端(client_id·client_secret)并完成认证以获取 refresh token。

注意： 开发者令牌的基础访问权限需经 Google 审核后批准，批准前只能调用测试账户。如果没有管理员(MCC)账户，请先在 ads.google.com 创建。

2. 申请步骤

按以下顺序申请并注册到 GenieGo。

1

登录管理员账户 · 进入 API 中心

登录 Google Ads 管理员账户后，进入顶部的 [工具与设置] > [设置] > [API 中心]。

工具与设置 → 设置 → API 中心
2

申请开发者令牌

在 API 中心申请 [开发者令牌]。基础(Basic)访问权限需经 Google 审核后批准，批准前只能调用测试账户。

3

配置 Cloud 项目 · OAuth 同意屏幕

创建项目并配置 [API 和服务] > [OAuth 同意屏幕]。

console.cloud.google.com → API 和服务 → OAuth 同意屏幕
4

创建 OAuth 客户端 ID

创建 [凭据] > [创建凭据] > [OAuth 客户端 ID] 以获取 client_id 和 client_secret(范围 scope: adwords)。

凭据 → 创建凭据 → OAuth 客户端 ID
5

申请 refresh token

通过 OAuth 认证流程申请 refresh token(使用 Google OAuth Playground 或自有认证)。

6

确认客户 ID

确认要调用的广告账户的10位客户 ID(customer_id)。

开发者令牌获批 + client_id·client_secret·refresh_token·customer_id 全部准备就绪后，即表示申请成功。

3. 需在 GenieGo 注册的信息

字段说明
developer_token在 API 中心申请的开发者令牌
client_idOAuth 客户端 ID
client_secretOAuth 客户端密钥(注意泄露)
refresh_token通过 OAuth 认证申请的 refresh token(填入 [访问令牌] 栏)
customer_id目标广告账户的10位客户 ID
在 GenieGo [注册] 中输入开发者令牌、client_id、client_secret、refresh_token、customer_id 并保存后，凭据将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决
只能调用测试账户开发者令牌基础访问权限审核未获批请等待在 API 中心申请的开发者令牌通过 Google 审核获批后，再调用生产账户。
认证失败缺少 scope(adwords)或 refresh_token 填写错误在 OAuth 客户端中包含 adwords 范围后重新认证，并重新确认 refresh_token。
客户 ID 格式错误10位 customer_id 填写错误重新确认目标广告账户的10位客户 ID 并输入。

最终检查清单

  • 已准备 Google Ads 管理员(MCC)账户并登录。
  • 已在 API 中心申请开发者令牌(基础访问权限需经 Google 审核)。
  • 已在 console.cloud.google.com 创建 OAuth 同意屏幕和 OAuth 客户端 ID/Secret(scope: adwords)。
  • 已通过 OAuth 认证申请 refresh token。
  • 已确认目标账户的10位 customer_id。
  • 已在 GenieGo [注册] 中输入开发者令牌·client_id/secret·refresh_token·customer_id 并保存。
