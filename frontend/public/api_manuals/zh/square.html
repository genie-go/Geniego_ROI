Square
API Key 申请手册

本手册手把手指导新手在 Square 开发者门户中创建应用以签发 Access Token 和 Location ID，并将其注册到 GenieGo 集成中心。

预计时间约 5~10 分钟
所需账户Square 账户
签发密钥Access Token · Location ID
成果API 凭证

1. 开始之前

Square API 凭证通过在开发者门户中创建应用来签发。

必需

Square 账户

您需要一个可以在 developer.squareup.com/apps 创建应用的 Square 账户。

签发密钥

Access Token · Location ID

在 Production 选项卡中查看 Access Token，在 [Locations] 中查看 Location ID。

注意: 实际集成请使用 Production 令牌，而非 Sandbox 令牌。

2. 申请步骤

按以下顺序签发并在 GenieGo 中注册。

1

创建应用

在 Square 开发者门户中创建应用。

developer.squareup.com/apps
2

查看 Access Token

在 Production 选项卡中查看 Access Token。

Production
3

查看 Location ID

在 [Locations] 中查看 Location ID。

Locations
确认了 Production 的 Access Token 和 Location ID 后，即表示已正确签发。

3. 需在 GenieGo 中注册的信息

字段说明
access_token在 Production 选项卡中查看到的 Access Token(请妥善保管)
location_id在 [Locations] 中查看到的 Location ID
在 GenieGo 集成中心的 [注册] 中输入 access_token·location_id 并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
认证失败使用了 Sandbox 令牌请使用在 Production 选项卡中查看到的 Access Token。
订单未同步Location ID 填写错误重新确认在 [Locations] 中查看到的 Location ID。

最终检查清单

  • 已在 developer.squareup.com/apps 创建应用。
  • 已在 Production 选项卡中查看 Access Token。
  • 已在 [Locations] 中查看 Location ID。
  • 已在 GenieGo 的 [注册] 中输入 access_token·location_id 并保存。
