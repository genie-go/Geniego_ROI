Naver Search Ads
API 密钥申请手册

本手册以新手也能直接照做的方式,整理了在 Naver Search Ad 管理工具中申请 API 认证信息(访问许可证、密钥)和 CUSTOMER_ID,并将其注册到 GenieGo 集成中心的全过程。

预计时间约 5–10 分钟
所需账户Naver Search Ad 广告主账户
申请的密钥访问许可证 · 密钥 · CUSTOMER_ID
成果Search Ad API 认证信息

1. 开始之前

Naver Search Ad API 仅可通过广告主账户申请。

必需

Search Ad 广告主账户

您需要一个使用 Naver Search Ad 管理工具的广告主账户。使用普通的 Naver 会员账户无法申请 API 访问权限。

申请的密钥

访问许可证 · 密钥 · CUSTOMER_ID

申请 API 访问权限后,确认一并签发的 CUSTOMER_ID、访问许可证和密钥。

注意: 密钥仅在申请时可见,请在申请后立即妥善保管。

2. API 密钥申请步骤

按以下顺序申请并在 GenieGo 中注册。

1

登录 Search Ads

使用广告主账户登录 Naver Search Ad 管理工具。

searchad.naver.com
2

前往 API 使用管理

从顶部的[工具]菜单进入[API 使用管理]。

工具 → API 使用管理
3

申请 API 服务

点击[申请 Naver Search Ad API 服务]按钮进行申请。

申请 Naver Search Ad API 服务
4

确认签发的信息

确认签发的 CUSTOMER_ID、访问许可证和密钥。

当 CUSTOMER_ID、访问许可证和密钥全部确认无误后,即表示已正常签发。

3. 需在 GenieGo 中注册的信息

字段说明
api_key签发的访问许可证
api_secret签发的密钥(请谨慎处理)
customer_id签发的 CUSTOMER_ID
在 GenieGo 集成中心的[注册]中输入并保存后,认证信息将以 AES-256-GCM 加密存储,并在读取时进行掩码处理。

4. 常见问题

问题原因解决
没有[API 使用管理]菜单并非广告主账户,或权限不足请使用使用 Search Ad 管理工具的广告主账户登录。
出现认证错误访问许可证或密钥输入错误请重新核对申请页面上的访问许可证、密钥和 CUSTOMER_ID。

最终检查清单

  • 已使用 Naver Search Ad 广告主账户登录。
  • 已在[工具 > API 使用管理]中完成 API 服务申请。
  • 已确认 CUSTOMER_ID、访问许可证和密钥。
  • 已在 GenieGo[注册]中输入 api_key、api_secret、customer_id 并保存。
