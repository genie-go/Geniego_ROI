1. 开始之前

韩进快递专用 API 是一种合同型渠道，无法在自助控制台中即时签发。

必需

韩进快递合同(客户)代码

您需要通过与韩进快递签订合同而获得的合同(客户)代码。它并非在单独的控制台中直接签发。

推荐

智能快递(统一追踪)

如果您仅需配送追踪，可在 [智能快递(统一追踪)] 渠道中用一个 t_key 连接所有快递公司。

注意: 如果目的是配送追踪，建议无需专用 API 合同，直接使用智能快递统一密钥进行连接。

2. 申请签发流程

按以下顺序申请并注册。

1

准备合同(客户)代码

准备您的韩进快递合同(客户)代码。专用 API 以合同形式提供。

2

判断统一追踪是否足够

如果您仅需配送追踪，可在 [智能快递(统一追踪)] 渠道中用一个 t_key 连接所有快递公司，而无需使用韩进快递专用渠道。

3

申请专用 API

如果您需要专用 API，请向销售代表获取合同(客户)代码和 API 密钥。

4

在 GenieGo 中注册

在 GenieGo 集成中心的 [注册] 中输入 api_key 和 cust_code 并保存。

当您从销售代表处收到 API 密钥和合同(客户)代码后，即可准备注册。

3. 需在 GenieGo 中注册的信息

字段说明
api_key签发的韩进快递专用 API 密钥
cust_code韩进快递合同(客户)代码
在 GenieGo 集成中心的 [注册] 中输入并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。
