1. Kakao Alimtalk 申请概念

Alimtalk 是一项通过 Kakao Talk 发送订单、预约、配送、支付等信息类消息的商务消息服务。它不是仅凭普通 Kakao 开发者应用密钥即可直接发送的结构，而是需要进行企业频道认证以及发送代理商/平台配置。

重要: Alimtalk 以信息类消息为主，而非广告类消息，且模板需要事先审核。

2. 准备物品

  • Kakao Business 账户
  • Kakao Talk 频道
  • 营业执照等企业认证资料
  • 要发送的 Alimtalk 模板文案
  • 发送代理商或云消息服务账户
  • 收件人电话号码的收集及同意管理政策

3. 准备 Kakao Talk 频道

1

登录 Kakao Business

登录 Kakao Business 管理员中心。

business.kakao.com
2

创建 Kakao Talk 频道

以品牌或服务名称创建频道。

Kakao Talk 频道管理员中心
3

进行企业认证

提交企业信息和证明资料，进行企业频道认证。

企业认证
4

确认频道搜索 ID

确认 Alimtalk 发送设置所需的频道 ID 或搜索 ID。

频道资料 → 搜索 ID
只有完成企业认证后，才能进行 Alimtalk 发送配置文件的注册。

4. 发送配置文件 / Sender Profile 注册

1

选择发送平台

选择 NHN Cloud、Naver Cloud SENS、Infobank、Bizgo 等 Alimtalk 提供商。

Messaging Provider
2

创建项目

在所选平台上创建 Alimtalk 项目。

Project / Service
3

连接 Kakao Talk 频道

注册已通过企业认证的 Kakao Talk 频道信息。

Sender Profile
4

发送配置文件审核

进行频道所有权确认和发送配置文件审核。

Profile Review
5

确认 API 凭证

确认平台提供的 App Key、Secret Key、API Key 等。

API Key / Secret

5. 模板注册与审批

1

撰写模板

撰写订单确认、配送通知等信息类文案。

Template Message
2

设置变量

将姓名、订单编号、配送单号等动态变化的值设置为变量。

#{变量}
3

提交审核

保存模板并提交 Kakao 审核申请。

Request Approval
4

确认审批

确认已审批的模板代码或 ID。

Template Code
如果审核前的文案与实际发送的文案不一致，可能导致发送失败或受到处罚。

6. API 发送流程示例

1) 从服务器调用 Alimtalk API Endpoint
2) 使用 API Key 或 Secret 进行认证
3) 传递收件人电话号码、模板代码、变量值
4) 接收发送结果
5) 处理失败时是否进行 SMS 替代发送

7. 最终检查清单

  • 已创建 Kakao Talk 频道。
  • 已完成企业认证。
  • 已选择 Alimtalk 发送平台。
  • 已注册 Sender Profile。
  • 已确认 API Key 或 Secret。
  • 已注册模板并获得审批。
  • 已准备发送失败及替代短信政策。
