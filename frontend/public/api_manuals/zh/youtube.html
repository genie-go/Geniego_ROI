YouTube API Key
申请手册

这是一份即使是新手也能照着操作的实战指南，整理了在 Google Cloud Console 中启用 YouTube Data API v3 并创建 API Key 的全过程。

预计耗时约 5–10 分钟
所需账号Google 账号
使用的 APIYouTube Data API v3
成果AIzaSy... 格式的 API Key

1. API Key 与 OAuth 的区别

这是刚开始时最容易让人困惑的部分。

API Key

用于查询公开数据

用于获取公开的 YouTube 数据，例如搜索视频、查询频道信息、获取播放列表以及读取评论等。

OAuth 2.0

用于需要你账号权限的操作

用于需要用户授权的操作，例如上传视频、管理自己的频道以及访问私密数据等。

重要: 若要自动上传视频，仅靠 API Key 是不够的，还需要 OAuth 2.0 认证。

2. 启用 YouTube Data API v3

在创建 API Key 之前，必须先在项目中开启你要使用的 API。

1

访问 Google Cloud Console

登录 Google Cloud Console。

console.cloud.google.com
2

选择或创建项目

在顶部的项目选择区域中选择现有项目，或创建一个新项目。

例如：Geniego ROI
3

进入 API 库

在左侧菜单中进入 API 和服务 → 库。

API 和服务 → 库
4

搜索 YouTube Data API v3 并启用

在搜索框中输入 YouTube Data API v3，选择该 API，然后点击启用按钮。

YouTube Data API v3 → 启用
如果屏幕上显示 状态：已启用，说明已正常启用。

3. 创建 API Key

启用 YouTube Data API v3 后，即可在凭据界面创建 API Key。

1

进入凭据菜单

在左侧菜单中进入 API 和服务 → 凭据。

API 和服务 → 凭据
2

点击创建凭据

点击顶部的 + 创建凭据按钮。

+ 创建凭据
3

选择 API 密钥

在弹出的菜单中选择 API 密钥。

API 密钥
4

选择可访问的 API

在 API 限制中确认已选择 YouTube Data API v3。

已选择的 API：YouTube Data API v3
5

点击创建按钮

点击底部的创建按钮即可生成 API Key。

创建
生成的密钥通常以 AIzaSy... 开头。这串字符就是你的 YouTube API Key。

4. 在创建界面应该选择哪些选项？

项目新手推荐设置说明
名称YouTube API Key输入一个以后便于识别的名称。
API 限制YouTube Data API v3限制此密钥仅用于 YouTube API。
服务账号认证不勾选创建 YouTube Data API Key 通常不需要。
应用限制首次测试设为无确认正常运行后，建议设置网站或 IP 地址限制。
部署到正式服务时，请务必设置密钥限制。如果在没有限制的情况下公开，他人可能会使用你的密钥。

5. API Key 使用示例

将生成的密钥填入 YOUR_API_KEY 部分即可进行测试。

视频搜索示例https://www.googleapis.com/youtube/v3/search?part=snippet&q=korea&type=video&key=YOUR_API_KEY
频道信息查询示例https://www.googleapis.com/youtube/v3/channels?part=snippet&id=CHANNEL_ID&key=YOUR_API_KEY
在浏览器地址栏输入上述网址后，如果出现 JSON 格式的结果，说明 API Key 工作正常。

6. 常见问题排查

问题原因解决方法
没有选择任何 APIYouTube Data API v3 尚未启用，或在 API 限制中未选择在 API 库中启用 YouTube Data API v3，然后重新选择。
没有可显示的 API 密钥尚未创建 API Key点击创建凭据 → API 密钥。
403 错误API 限制、配额超出或权限问题检查 API 限制、配额和启用状态。
无法上传仅使用了 API Key视频上传必须使用 OAuth 2.0 认证。

7. 最终检查清单

  • 已选择 Google Cloud 项目。
  • 已启用 YouTube Data API v3。
  • 已在凭据中创建 API Key。
  • 已确认生成的 API Key 以 AIzaSy 开头。
  • 已将 API 限制设置为 YouTube Data API v3。
  • 在部署到正式服务前设置网站或 IP 限制。
