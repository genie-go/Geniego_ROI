API Key 申请手册

邮政快递无需单独申请，通过智能快递(Sweet Tracker)统一追踪密钥进行接入。在 tracking.sweettracker.co.kr 申请 t_key，并将其注册到[智能快递(统一追踪)]渠道，即可接入邮政快递配送追踪。

预计时间约 5~15 分钟
所需账户智能快递(Sweet Tracker)账户
申请密钥智能快递追踪 API 密钥(t_key)
成果统一配送追踪凭证

1. 开始之前

邮政快递无需专用申请，通过智能快递统一追踪密钥进行接入。

方式

智能快递(Sweet Tracker)统一追踪

邮政快递无需单独申请，通过智能快递(Sweet Tracker)统一追踪密钥进行接入。一个 t_key 即可处理所有快递公司的配送追踪。

申请密钥

t_key (智能快递追踪 API 密钥)

这是在 tracking.sweettracker.co.kr 注册后申请的统一追踪密钥。在 GenieGo 中请填入 api_key 字段。

重要: 请勿直接选择邮政快递渠道，而应在[智能快递(统一追踪)]渠道注册 t_key。

2. 申请步骤

按以下顺序申请并在 GenieGo 中注册。

1

注册 Sweet Tracker

注册智能快递(Sweet Tracker)。

tracking.sweettracker.co.kr
2

申请 t_key

在 tracking.sweettracker.co.kr 申请 t_key。

3

选择智能快递渠道

在集成中心选择[智能快递(统一追踪)]渠道，而非邮政快递渠道。

4

在 GenieGo 注册

在 GenieGo 的[注册]中填入 t_key(api_key)，即可接入邮政快递配送追踪。

申请 t_key 并将其注册到[智能快递(统一追踪)]渠道后，邮政快递配送追踪即被接入。

3. 需在 GenieGo 中注册的信息

字段说明
api_key由 Sweet Tracker 申请的 t_key(邮政快递智能快递追踪 API 密钥)
在 GenieGo 集成中心的[注册]中填入并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
无法在邮政快递渠道注册密钥邮政快递无需专用申请，通过统一追踪密钥接入选择[智能快递(统一追踪)]渠道并注册 t_key。
配送追踪未接入t_key 填写错误或未申请重新核对在 tracking.sweettracker.co.kr 申请的 t_key。

最终检查清单

  • 已在 tracking.sweettracker.co.kr 注册。
  • 已申请 t_key(智能快递追踪 API 密钥)。
  • 已在集成中心选择[智能快递(统一追踪)]渠道。
  • 已在 GenieGo 的[注册]中填入 t_key(api_key)并保存。
