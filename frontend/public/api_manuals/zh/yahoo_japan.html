Y! API 申请新手手册

Yahoo! Japan
API 申请手册

为接入 Yahoo! Japan 系列服务，在开发者控制台中注册应用程序并准备 Client ID、Client Secret 和 OAuth Token 的流程。

认证结构以 OAuth 2.0 为中心
所需账户Yahoo! JAPAN ID
主要成果Client ID / Secret
注意各 API 控制台不同

1. Yahoo! Japan API 申请概念

Yahoo! Japan 系列 API 会根据服务类型，在 Yahoo! JAPAN Developers 或 LY/Yahoo!広告 Developer Center 申请 Client ID、Client Secret 和 OAuth 认证信息。

重要: 「Yahoo! Japan」会根据您要使用的 API 不同，其申请站点和审核流程也不同。需要区分购物、广告、登录和搜索 API。

2. 准备物

  • Yahoo! JAPAN ID 或企业账户
  • 确定要使用的 API 类型
  • 应用名称和服务说明
  • Redirect URI
  • 隐私政策 URL
  • 生产域名或测试域名

3. 应用程序注册基本顺序

1

访问开发者站点

登录 Yahoo! JAPAN Developers 或对应 API 的 Developer Center。

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

创建应用程序

点击新增应用程序菜单。

Create Application / Add Application
3

输入应用信息

输入应用名称、说明、负责人信息和网站 URL。

Application Information
4

输入 Redirect URI

准确输入 OAuth 认证后返回的 URL。

Redirect URI
5

选择权限或 API

选择要使用的 API 权限和作用域。

Scope / API Permission
6

完成注册

注册后确认 Client ID 和 Client Secret。

Client ID / Client Secret

4. 各服务的申请位置

目的申请/管理位置成果
Yahoo! Japan 登录 / OAuthYahoo! JAPAN DevelopersClient ID, Client Secret
Yahoo!広告 APIYahoo!広告 Developer CenterClient ID, Client Secret, OAuth Token
Yahoo! Shopping APIYahoo! JAPAN Developers 或 Commerce 相关控制台Application ID 或 OAuth 信息
内部测试各服务的 Sandbox/测试环境测试用认证信息

5. OAuth 令牌申请流程

1) 使用 Client ID 生成用户认证 URL 2) 用户使用 Yahoo! 账户登录 3) 在 Redirect URI 接收 authorization code 4) 使用 authorization code + Client Secret 请求 Access Token 5) 使用 Access Token 调用 API 6) 必要时使用 Refresh Token 刷新

6. 常见问题

问题原因解决方法
Redirect URI 错误注册的 URL 与调用 URL 不同连同协议、域名、路径都输入一致
无法查看 Client Secret出于安全考虑被隐藏查看管理界面的 View 或重新签发功能
权限不足未选择所需 Scope在应用设置中添加 Scope 后重新认证
API 调用失败所用 API 与申请的应用类型不匹配查看各 API 的开发者文档

7. 最终检查清单

  • 已确定要使用的 Yahoo! Japan API 类型。
  • 已登录正确的开发者控制台。
  • 已注册应用程序。
  • 已确认 Client ID 和 Client Secret。
  • 已准确注册 Redirect URI。
  • 已选择所需 Scope。
  • 已完成 Access Token 申请测试。
