Cafe24
API Key 申请手册

本手册手把手指导新手在 Cafe24 开发者中心注册应用以签发 Client ID·Client Secret，通过 OAuth 2.0 授权获取令牌，并注册到 GenieGo 集成中心。

预计时间约 10~15 分钟
所需账户Cafe24 商城运营者账户
签发密钥Client ID · Client Secret · Refresh Token
成果OAuth 2.0 凭证

1. 开始之前

Cafe24 集成需经过开发者注册和应用注册，并通过 OAuth 2.0 方式进行认证。

必需

商城运营者账户

您需要一个运营 Cafe24 商城的运营者账户。使用该账户在开发者中心完成开发者注册。

签发密钥

Client ID · Client Secret + 令牌

注册应用时会签发 Client ID 和 Client Secret，并通过 OAuth 2.0 授权获取 access token 和 refresh token。

注意: 在 GenieGo 中注册 long-lived 的 refresh_token，access token 将通过它自动刷新。

2. 应用注册及令牌签发步骤

按以下顺序签发并在 GenieGo 中注册。

1

开发者注册

访问 Cafe24 开发者中心并注册为开发者。

developer.cafe24.com
2

应用注册

通过[应用] > [应用注册]创建应用后，将签发 Client ID 和 Client Secret。

应用 → 应用注册
3

OAuth 2.0 授权

通过 OAuth 2.0 授权签发 access token 和 refresh token。

4

确认信息

确认 mall_id(商城 ID)、Client ID、Client Secret、refresh token。

当 Client ID·Client Secret 和 refresh token 全部确认无误时，即表示已正确签发。

3. 需在 GenieGo 中注册的信息

字段说明
mall_idCafe24 商城 ID
client_id签发的 Client ID
client_secret签发的 Client Secret(请妥善保管)
refresh_token通过 OAuth 2.0 授权获取的 Refresh Token
在 GenieGo 集成中心的[注册]中输入并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
授权失败错误的 mall_id 或应用设置不一致重新确认应用注册时设置的 mall_id 和 Client 信息。
令牌过期错误refresh_token 填写错误重新注册通过 OAuth 2.0 授权获取的最新 refresh_token。

最终检查清单

  • 已在 Cafe24 开发者中心注册为开发者。
  • 已通过[应用 > 应用注册]创建应用并签发 Client ID·Client Secret。
  • 已通过 OAuth 2.0 授权签发 access token·refresh token。
  • 已在 GenieGo 的[注册]中输入 mall_id·client_id·client_secret·refresh_token 并保存。
