Facebook Live
API 申请手册

本手册手把手指导新手在 Meta for Developers 上创建应用，并准备连接 Facebook 主页直播所需的 App ID、App Secret 和主页访问令牌(Page Access Token)。

核心概念不是 API Key，而是 Access Token
开发网站developers.facebook.com
使用的 APILive Video API (Graph)
所需权限主页管理员 + Page Token

1. 首先要了解的核心

YouTube 用一个 API Key 即可，但 Facebook(Meta)采用 App ID · App Secret · Access Token 的结构运行。

重要: Meta 并非「复制一个 API Key」就能完成。通常需要按 创建应用 → 添加产品 → 设置权限 → 签发主页访问令牌 → 应用审核 的顺序进行。

2. 准备事项

  • Meta/Facebook 账户
  • Meta for Developers 访问权限
  • 拥有管理权限的 Facebook 主页
  • 用于实际服务的隐私政策 URL · 服务条款 URL

3. 创建 Meta Developer 应用

所有 Meta API 集成都从创建应用开始。

1

访问 Meta for Developers

在浏览器中登录 Meta for Developers。

developers.facebook.com
2

我的应用 → 创建应用

点击顶部的 My Apps → Create App。

My Apps → Create App
3

选择应用类型

平台集成/直播电商更接近企业用途。

选择 Business 类型
4

输入应用名称·邮箱 → 创建

输入应用名称和联系邮箱，然后创建应用。

App Name / Contact Email
创建应用后，在仪表板 → 应用设置 → 基本设置中查看 App ID 和 App Secret。(切勿在前端/公开仓库中暴露 App Secret)

4. Facebook Live API 设置流程

1

添加产品

在应用仪表板中准备 Facebook Login / Graph API 相关设置。

Add Product → Facebook Login
2

连接主页 + 主页令牌

若要将直播推送到主页，需要主页管理员权限和 Page Access Token。

Page Access Token
3

申请所需权限(App Review)

申请用于发布内容·管理主页·创建直播视频的权限(例如 pages_read_engagement、pages_manage_posts)。

Permissions → App Review
4

用 Graph API Explorer 签发并测试令牌

签发包含权限的令牌并测试 API 调用。

Tools → Graph API Explorer
Facebook Live 创建调用示例
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
创建成功后会返回直播 ID·推流 URL·推流密钥等直播推送信息。主页 ID(page_id)可在主页信息中查看。

5. 注册到 GenieGo 集成中心

1

复制主页访问令牌

复制您签发的 Page Access Token(建议使用长期令牌)。

access_token (Page token)
2

确认主页 ID

确认目标 Facebook 主页的 ID(page_id)。

page_id
3

注册到集成中心

在 GenieGo 集成中心 → Facebook Live → [注册] 中输入以下值并保存。

access_token · page_id
注册的密钥将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

6. Access Token 类型

令牌类型用途注意事项
User Access Token基于用户登录的 API 调用可访问的数据因权限授权范围而异。
Page Access Token主页发布·主页直播创建需要主页管理员权限及相关权限。
Long-Lived Token延长了有效期的长期令牌运营时需要有效期·刷新逻辑。

7. 检查清单

  • 已创建 Meta Developer 应用。
  • 已确认 App ID 和 App Secret。
  • 已连接 Facebook 主页并签发 Page Access Token。
  • 已整理并申请所需权限(App Review)。
  • 已在 GenieGo [注册] 中输入 access_token·page_id 并保存。
  • 已将令牌安全地保存在服务器环境变量中。

8. 常见问题

问题原因解决方法
没有 API Key 菜单Meta 不是单一 API Key 结构按 App ID·App Secret·Access Token 的结构进行。
权限错误令牌缺少所需权限在 Graph API Explorer 中添加权限后重新签发令牌。
主页直播创建失败Page Access Token/管理员权限不足检查主页管理员权限和令牌权限。
