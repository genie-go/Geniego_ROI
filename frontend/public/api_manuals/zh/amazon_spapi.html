Amazon SP-API
申请手册

整理了将 Amazon 卖家数据与订单、库存、商品、配送系统对接所需的 SP-API 开发者注册、应用创建以及 OAuth·LWA 设置流程。

认证结构LWA(OAuth) — 5项凭据
所需账户Seller/Vendor Central(专业版)
GenieGo 注册值refresh_token·client_id·secret·marketplace_id·seller_id
难度中级以上

1. 什么是 Amazon SP-API?

Amazon Selling Partner API 是用于通过程序对接订单、库存、商品、配送、结算等卖家数据的 API。

重要: SP-API 不是简单的 API Key 方式。通常需要同时进行 Amazon Developer 注册、应用注册、Login with Amazon(LWA)、OAuth 授权。
★GenieGo 对接需要 5 个值: refresh_token, client_id, client_secret, marketplace_id, seller_id。(当前 SP-API 仅凭 LWA 令牌即可调用,AWS IAM/SigV4 签名由 GenieGo 内部处理,因此无需在注册栏中填写。)

2. 准备事项

  • Amazon Seller Central 或 Vendor Central 专业版卖家账户
  • 用于开发者注册的公司/负责人信息
  • 应用名称、应用说明、使用目的
  • 隐私政策 URL
  • OAuth Redirect URL

3. 开发者注册

首先在 Seller Central 中注册 SP-API 开发者资料。

1

登录 Seller Central

使用 Amazon 卖家账户登录 Seller Central。

Seller Central → Apps & Services
2

进入 Develop Apps 菜单

在 Apps & Services 菜单中,进入开发者或应用开发菜单。

Apps & Services → Develop Apps
3

填写 Developer Profile

填写公司信息、负责人信息、API 使用目的和数据保护计划。

Developer Profile
4

提交审核

为通过 Amazon 审核,提交开发者注册信息。

Submit for review
只有审核通过后,才能注册正式运营用的 SP-API 应用并使用其权限。

4. 注册 SP-API 应用 → Client ID / Client Secret

1

创建新应用

在 Develop Apps 中创建新应用。

Add new app
2

选择应用类型

若仅对接自有账户则为 Private App;若提供给外部卖家则属于 Public App。

Private / Public App
3

选择 API 角色

选择 Orders、Catalog、Inventory 等所需角色。

API Roles
4

输入 OAuth Redirect URL

输入授权完成后返回的服务器 URL。

Redirect URI
5

保存后确认 Client 信息

创建应用后,确认 client_id (LWA Client ID)、client_secret (Client Secret)。

client_id / client_secret

5. OAuth 授权 → 发行 Refresh Token

1) 访问卖家 OAuth 授权 URL 2) 在 Redirect URL 接收 authorization code 3) 将 authorization code 传递给 LWA Token Endpoint 4) 接收 refresh_token / access_token
refresh_token 用于卖家授权后的长期调用。GenieGo 使用该 refresh_token 自动刷新 access_token 来调用 SP-API,因此无需单独的 AWS 签名设置,只需注册 refresh_token 即可。

6. 确认 Marketplace ID · Seller ID

确认 GenieGo 注册所需的其余两个标识符。

marketplace_id

销售目标的市场 ID

要对接国家的 Marketplace ID。US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

卖家 ID / Merchant Token

商品登记、订单查询所需。在 Seller Central 账户信息中查看。Seller Central → 设置 → 账户信息 → Merchant Token

7. 在 GenieGo 集成中心注册

1

准备 5 个值

准备 refresh_token · client_id · client_secret · marketplace_id · seller_id。

5 项凭据
2

集成中心 → Amazon SP-API → [注册]

在各栏输入对应的值并保存(空栏保留原有值)。

refresh_token · client_id · client_secret · marketplace_id · seller_id
注册的值会 以 AES-256-GCM 加密存储,查询时进行掩码处理。保存后立即开始自动同步。
client_secret·refresh_token 绝不能暴露在前端代码、HTML 或公开仓库中。

8. 最终检查清单

  • 已提交 Developer Profile 并获得批准。
  • 已创建 SP-API 应用并确认 client_id·client_secret。
  • 已通过 OAuth 授权发行 refresh_token。
  • 已确认要对接的 marketplace_id。
  • 已在 Seller Central 确认 seller_id(Merchant Token)。
  • 已在 GenieGo [注册] 中输入 5 个值(refresh_token·client_id·client_secret·marketplace_id·seller_id)并保存。

9. 常见问题

问题原因解决方法
没有 API Key 菜单SP-API 不是单一 API Key 结构按 LWA(client_id·client_secret·refresh_token)结构进行。
商品登记失败未注册 seller_id(Merchant Token)注册 Seller Central 账户信息中的 Merchant Token。
国家数据错误marketplace_id 不匹配注册对接国家正确的 Marketplace ID。
权限错误应用 API Role 不足或令牌过期确认 API Role 并使用 refresh_token 刷新令牌。
