Klarna
API Key 申请手册

本手册手把手指导新手在 Klarna Merchant Portal 中创建 API 凭证(username·password),并连同区域一起注册到 GenieGo 集成中心。

预计时间约 5~10 分钟
所需账户Klarna 商户账户(BNPL，需签约)
签发密钥API username · password · 区域
成果API 凭证

1. 开始之前

Klarna 凭证只能在商户合约完成签订后才能签发。

必需

Klarna 商户账户

首先需要签订 Klarna(BNPL)商户合约。合约完成后才能在 Merchant Portal 中签发 API 凭证。

签发密钥

API username · password · 区域

在 Merchant Portal 的 Settings > Klarna API credentials 中创建 API username·password,并确认区域(eu/na/oc)。

注意: 区域为 eu / na / oc 之一，注册时需一并输入。

2. 申请步骤

签约后，按以下顺序签发并在 GenieGo 中注册。

1

商户签约

首先完成 Klarna(BNPL)商户合约。合约完成后才能进入下一步。

2

登录 Merchant Portal

登录 Klarna Merchant Portal。

portal.klarna.com
3

前往 API credentials

进入 [Settings] > [Klarna API credentials] 菜单。

Settings → Klarna API credentials
4

创建并确认凭证

创建 API username·password,并确认区域(eu/na/oc)。

如果在 Settings > Klarna API credentials 中创建了 API username·password 并确认了区域，即表示正常。

3. 需在 GenieGo 中注册的信息

字段说明
usernameAPI Username(MID/PID)
passwordAPI Password(请妥善保管)
region区域(eu / na / oc)
在 GenieGo 集成中心的 [注册] 中输入 username·password·region 并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
API credentials 菜单中没有签发项商户合约未完成请先完成 Klarna 商户合约。
认证失败区域(eu/na/oc)不匹配或 password 填写错误重新确认在 Merchant Portal 中确认的区域以及 username·password。

最终检查清单

  • 已完成 Klarna(BNPL)商户合约。
  • 已在 Merchant Portal 的 Settings > Klarna API credentials 中创建凭证。
  • 已确认 API username·password 和区域(eu/na/oc)。
  • 已在 GenieGo [注册] 中输入 username·password·region 并保存。
