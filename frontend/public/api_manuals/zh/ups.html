UPS
API Key 申请手册

本手册整理了在 UPS 开发者门户(developer.ups.com)创建应用以签发 client_id 和 client_secret，并连同 UPS 账号一起注册到 GenieGo 集成中心的流程。

预计时间约 5~15 分钟
所需账户UPS 账户(账号)
签发密钥Client ID · Client Secret
成果OAuth 凭证 · 账号

1. 开始之前

UPS 凭证通过在开发者门户创建应用来签发，并且需要账号。

准备事项

UPS 账号

您需要一个 UPS 账号。账号将在注册到 GenieGo 时一并输入。

签发密钥

Client ID · Client Secret

在开发者门户创建应用后，将签发 client_id 和 client_secret。

2. 申请步骤

按以下顺序签发并在 GenieGo 中注册。

1

注册开发者门户并创建应用

注册 UPS 开发者门户并创建应用。

https://developer.ups.com
2

签发 client_id·client_secret

创建应用后，将签发 client_id 和 client_secret。

创建应用
3

确认账号

确认已签发的 client_id·client_secret 以及 UPS 账号。

创建应用后签发了 client_id·client_secret，即表示正常。

3. 需在 GenieGo 中注册的信息

字段说明
client_id签发的 Client ID
client_secret签发的 Client Secret(请妥善保管)
account_numberUPS 账号
在 GenieGo 集成中心 [注册] 中输入并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
应用中看不到凭证应用创建未完成在 developer.ups.com 创建应用后，将签发 client_id·client_secret。
调用时出现认证错误client_secret 填写错误或账号缺失重新确认 client_secret 和账号(account_number)。

最终检查清单

  • 已确认 UPS 账号。
  • 已注册 developer.ups.com 并创建应用。
  • 已签发 client_id·client_secret。
  • 已在 GenieGo [注册] 中输入 client_id·client_secret·account_number 并保存。
