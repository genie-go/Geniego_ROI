Auction
API Key 申请手册

本手册介绍如何在 ESM Trading API(etapi.gmarket.com)申请 Auction·Gmarket 通用 API 密钥并注册到 GenieGo 集成中心。由于 Auction 与 Gmarket 通过 ESM Plus 统一运营，因此用一个 API 密钥即可同时连接两个渠道。

预计时间含审核等待需数日(需审核)
所需账户Auction·Gmarket 卖家账户 + ESM Plus 主 ID
签发密钥API 密钥 · 卖家 ID
成果ESM Trading API 凭证

1. 开始之前

您必须在 Auction 和 Gmarket 上均已注册为卖家，并拥有 ESM Plus 主 ID。

必需

卖家账户 + ESM 主 ID

您需要分别拥有 Auction 和 Gmarket 的卖家账户，以及在 ESM Plus(esmplus.com)上创建的主 ID。

签发密钥

API 密钥 · 卖家 ID

在 ESM Trading API 审核通过后，一并确认签发的 API 密钥和卖家 ID。

注意: ESM Trading API 采用审核制，申请可能被拒绝。申请时需提交三项内容：Auction 卖家ID、Gmarket 卖家ID 和 ESM 主ID。

2. 申请步骤

按以下顺序申请、签发，并在 GenieGo 中注册。

1

注册为卖家并创建 ESM 主 ID

分别在 Auction 和 Gmarket 注册为卖家，然后登录 ESM Plus 创建主 ID。

esmplus.com
2

申请使用 ESM Trading API

访问 ESM Trading API 并申请 API 使用权限。因采用审核制，申请可能被拒绝。

https://etapi.gmarket.com
3

提交识别信息

申请时需提交三项内容：Auction 卖家ID、Gmarket 卖家ID 和 ESM 主 ID。

4

密钥签发

审核通过后将签发 API 密钥。确认签发的 API 密钥和卖家 ID。

审核完成后签发 API 密钥即表示已正常处理。审核相关咨询请联系 etapihelp@gmail.com。

3. 需在 GenieGo 中注册的信息

字段说明
api_key签发的 API 密钥
seller_id卖家 ID
在 GenieGo 集成中心 [注册] 中输入并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
API 使用申请被拒绝ESM Trading API 采用审核制请确认提交的信息并联系 etapihelp@gmail.com。
申请信息不一致Auction·Gmarket 卖家ID 或 ESM 主ID 填写错误重新确认 Auction 卖家ID、Gmarket 卖家ID 和 ESM 主ID 三项内容。

最终检查清单

  • 已分别在 Auction 和 Gmarket 注册为卖家。
  • 已在 ESM Plus 创建主 ID。
  • 已在 etapi.gmarket.com 申请使用 ESM Trading API 并获得审核通过。
  • 已确认签发的 api_key 和 seller_id。
  • 已在 GenieGo [注册] 中输入 api_key·seller_id 并保存。
