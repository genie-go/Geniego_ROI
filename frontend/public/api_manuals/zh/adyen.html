Adyen
API Key 申请手册

本手册整理了在 Adyen 商户合同与审核获批后，在 Customer Area 中签发 API 密钥(X-API-Key)和 Merchant Account，并注册到 GenieGo 集成中心的流程。

预计时间审核获批后约 5~15 分钟
所需资格Adyen 商户合同·审核获批
签发密钥API 密钥(X-API-Key) · Merchant Account
成果API 凭证

1. 开始之前

Adyen 是面向企业的全球支付网关，在签发凭证之前必须先完成商户合同与审核获批。

必需

商户合同·审核获批

Adyen 是面向企业的全球 PG，需要签订商户合同并通过审核获批。只有获批后才能签发凭证。

签发密钥

X-API-Key · Merchant Account

获批后，在 Customer Area 的 [Developers] > [API credentials] 中生成 API 密钥(X-API-Key)并确认 Merchant Account 名称。

前置条件: 在商户合同与审核获批之前无法自助签发。您必须先与 Adyen 完成商户合同与审核。

2. 申请流程

审核获批后，按以下顺序签发并注册凭证。

1

商户合同·审核获批

与 Adyen 签订商户合同并获得审核获批。

Adyen 销售/合同
2

登录 Customer Area

获批后登录 Customer Area。

https://ca-live.adyen.com
3

签发 API 凭证

在 [Developers] > [API credentials] 中生成 API 密钥(X-API-Key)并确认 Merchant Account 名称。

Developers → API credentials
4

确认起始结算批次号

为进行结算的首次采集，请在 Customer Area 中确认最新的 settlement batch 编号。

当确认了 API 密钥(X-API-Key)和 Merchant Account 后，即表示凭证已准备就绪。

3. 需在 GenieGo 中注册的信息

字段说明
api_key签发的 API 密钥(X-API-Key，请妥善保管)
merchant_account已确认的 Merchant Account 名称
batch_start用于首次结算采集的起始结算批次号(Customer Area 中最新的 settlement batch 编号)
在 GenieGo 集成中心的 [注册] 中输入 api_key·merchant_account·batch_start 并保存后，凭证将以 AES-256-GCM 加密存储，并在查询时进行掩码处理。

4. 常见问题

问题原因解决方法
没有 API credentials 菜单商户合同·审核未获批与 Adyen 完成商户合同·审核后，再访问 Customer Area。
认证失败X-API-Key 或 Merchant Account 填写错误在 Developers > API credentials 中重新确认签发的 API 密钥和 Merchant Account 名称。

最终检查清单

  • 已获得 Adyen 商户合同·审核获批。
  • 已登录 Customer Area(ca-live.adyen.com)。
  • 已在 Developers > API credentials 中确认 API 密钥(X-API-Key)和 Merchant Account。
  • 已确认最新的 settlement batch 编号(batch_start)。
  • 已在 GenieGo 的 [注册] 中输入 api_key·merchant_account·batch_start 并保存。
