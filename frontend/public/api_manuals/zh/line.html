LINE
API 申请手册

本手册整理了在 LINE Developers Console 中创建 Provider 和 Messaging API Channel，并配置 Channel Access Token、Webhook URL 的过程。

认证结构Channel Access Token
所需账户LINE 账户
主要成果Channel ID / Secret / Token
用途消息推送·聊天机器人

1. LINE API 申请概念

LINE 对接是在 LINE Developers Console 中创建 Provider 和 Channel，然后为 Messaging API 或 LINE Login 申请 Channel ID、Channel Secret、Channel Access Token 的方式。

区分: 发送消息使用 Messaging API 渠道，登录对接使用 LINE Login 渠道。

2. 准备事项

  • LINE 账户或企业账户
  • LINE Developers Console 访问权限
  • LINE Official Account
  • 服务名称和说明
  • Webhook URL
  • 隐私政策 URL 及条款 URL

3. 创建 Provider

1

访问 LINE Developers

登录 LINE Developers Console。

developers.line.biz
2

创建 Provider

使用公司名称或服务名称创建 Provider。

Create Provider
3

填写基本信息

填写服务运营主体、联系方式和条款信息。

Provider Settings

4. 创建 Messaging API 渠道

1

创建新渠道

在 Provider 内点击 Create a new channel。

Create a new channel
2

选择 Messaging API

在渠道类型中选择 Messaging API。

Messaging API
3

填写渠道信息

填写渠道名称、说明、类别、邮箱等。

Channel Information
4

关联 LINE Official Account

关联用于发送消息的官方账号，或新建一个。

LINE Official Account
5

完成渠道创建

创建后确认 Channel ID 和 Channel Secret。

Channel ID / Channel Secret

5. 申请 Channel Access Token

1

切换到 Messaging API 标签页

在创建的渠道页面切换到 Messaging API 标签页。

Messaging API tab
2

查找 Channel access token 项

确认底部的 Channel access token 区域。

Channel access token
3

点击 Issue

点击 Issue 或 Reissue 按钮申请令牌。

Issue
4

安全保存到服务器

将申请到的令牌保存在服务器环境变量或 Secret Manager 中。

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token 是拥有消息发送权限的密钥。不得在 HTML、前端或公开仓库中暴露。

6. 配置 Webhook

1

输入 Webhook URL

输入用于接收用户消息事件的服务器 URL。

Webhook URL
2

启用 Use webhook

将 Webhook 使用选项改为 Enabled。

Use webhook: Enabled
3

Verify 测试

从 LINE 向服务器发送测试请求以确认连接。

Verify
4

确认响应码

确认服务器返回 200 OK。

HTTP 200 OK

7. 消息发送示例

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "您好。这是 LINE 消息测试。"} ] }

8. 最终检查清单

  • 已创建 Provider。
  • 已创建 Messaging API 渠道。
  • 已关联 LINE Official Account。
  • 已确认 Channel ID 和 Channel Secret。
  • 已申请 Channel Access Token。
  • 已注册 Webhook URL 并通过 Verify。
  • 已将令牌安全保存到服务器环境变量。
