API 密钥申请手册

本手册介绍如何在 Naver Commerce API 中心（apicenter.commerce.naver.com）注册应用以获取 Client ID 和 Client Secret，并将其注册到 GenieGo 集成中心的完整步骤。

预计时间约 10～15 分钟
所需账户具有“综合管理员”权限的智能商店账户
申请的密钥Client ID · Client Secret
成果Commerce API 凭据

1. 开始之前

只有智能商店卖家账户的综合管理员权限才能进行申请。

必需

综合管理员权限

您必须拥有智能商店卖家账户的“综合管理员”权限。只有综合管理员才能申请密钥。

申请的密钥

Client ID · Client Secret

注册应用后，将颁发应用 ID（Client ID）和 Secret（Client Secret）。

提示: 必须注册将调用 API 的服务器 IP，并定期更新应用认证。

2. 申请步骤

按以下顺序申请并在 GenieGo 中注册。

1

访问 Commerce API 中心

使用综合管理员账户登录 Naver Commerce API 中心。

https://apicenter.commerce.naver.com
2

创建账户

点击右上角的 [创建账户]，输入开发公司账户名和故障应对联系人，同意条款后注册。

创建账户
3

注册应用

在 [我的商店应用] 中注册应用，随后将颁发应用 ID 和 Secret。

我的商店应用
4

注册 IP 并更新认证

注册将调用 API 的服务器 IP，并按指引定期更新应用认证。

注册应用并颁发应用 ID 和 Secret 后，即表示处理正确。

3. 在 GenieGo 中注册的信息

字段说明
client_id已颁发的应用 ID（Client ID）
client_secret已颁发的 Secret（Client Secret，请妥善保管）
在 GenieGo 集成中心的 [注册] 中输入并保存后，凭据将以 AES-256-GCM 加密存储，并在查看时进行掩码处理。

4. 常见问题

问题原因解决
无法注册应用账户没有综合管理员权限使用具有综合管理员权限的账户访问 apicenter.commerce.naver.com。
调用时认证错误调用 IP 未注册或遗漏认证更新注册调用服务器的 IP 并定期更新应用认证。

最终检查清单

  • 已使用综合管理员账户访问 Commerce API 中心。
  • 已通过 [创建账户] 输入开发公司信息和故障应对联系人并同意条款。
  • 已在 [我的商店应用] 中注册应用并获取 ID/Secret。
  • 已注册调用服务器的 IP 并设置定期认证。
  • 已在 GenieGo [注册] 中输入 client_id·client_secret 并保存。
