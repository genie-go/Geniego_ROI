11Street
(11번가) API Key 발급 매뉴얼

11번가 오픈API센터(openapi.11st.co.kr)에서 오픈API 키를 발급받아 GenieGo 연동 허브에 등록하는 과정을 초보자도 그대로 따라 할 수 있도록 정리한 매뉴얼입니다.

소요 시간약 5~15분
필요 계정11번가 셀러(판매자) 계정
발급 키오픈API 키 · 셀러 ID
결과물오픈API 자격증명

1. 시작하기 전에

11번가 오픈API는 셀러(판매자) 계정에서만 발급할 수 있습니다.

필요

11번가 셀러 계정

11번가 셀러(판매자) 계정이 있어야 합니다. 오픈API센터에 셀러 아이디로 로그인합니다.

발급 키

오픈API 키 · 셀러 ID

서비스 등록과 접근 IP 등록을 마치면 발급되는 오픈API 키와 셀러 ID를 함께 확인합니다.

참고: API 접근이 가능한 서버 IP를 등록해야 키를 사용할 수 있습니다.

2. 발급 단계

아래 순서대로 발급하고 GenieGo에 등록합니다.

1

오픈API센터 로그인

셀러 아이디로 11번가 오픈API센터에 로그인합니다.

https://openapi.11st.co.kr/openapi/OpenApiFrontMain.tmall
2

서비스 등록

메뉴에서 [서비스 등록·확인]을 눌러 셀러 정보를 입력하고 약관에 동의해 등록합니다.

서비스 등록·확인
3

접근 IP 등록

[Seller API 정보] > [수정]에서 API 접근이 가능한 서버 IP를 등록합니다.

Seller API 정보 → 수정
4

API Key 복사

발급된 API Key를 복사합니다.

서비스 등록과 접근 IP 등록을 마치면 API Key가 발급됩니다.

3. GenieGo에 등록할 정보

필드설명
api_key발급된 오픈API 키
seller_id셀러 ID
GenieGo 연동 허브 [등록]에 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
API Key가 발급되지 않음서비스 등록 또는 접근 IP 등록 미완료[서비스 등록·확인]과 [Seller API 정보]의 접근 IP 등록을 완료합니다.
호출 시 인증 오류접근 가능 서버 IP 미등록[Seller API 정보] > [수정]에서 접근 가능 서버 IP를 다시 확인합니다.

5. 상품등록 전 준비물 (중요)

API 키가 발급돼도 아래 3가지가 없으면 11번가가 상품등록을 거부합니다.

  • 표시카테고리(dispCtgrNo) — GenieGo [카탈로그 동기화] > [카테고리 매핑] 또는 "카테고리·브랜드 확정 필요" 패널에서 지정
  • 브랜드 — GenieGo [카탈로그 동기화] > [브랜드 관리]에서 등록 후 상품에 지정(브랜드코드 없이 브랜드명만으로 전송됩니다)
  • 판매방식 — GenieGo가 고정가판매(01)로 자동 전송하므로 직접 입력할 필요가 없습니다
오류 메시지진짜 의미해결 방법
-997 등록된 API 정보가 존재하지 않습니다인증 실패가 아닙니다. 호출한 API 주소(경로+방식)가 11번가에 존재하지 않는다는 뜻입니다.키를 재발급하지 마세요. 연동 담당자에게 호출 주소 확인을 요청하세요.
브랜드코드가 없습니다상품에 브랜드가 지정되지 않았습니다.[카탈로그 동기화] > [브랜드 관리]에서 브랜드를 등록하고 상품에 지정하세요.
API로 이용할 수 없는 판매 방식입니다판매방식 코드가 API 허용값이 아닙니다.GenieGo가 고정가판매(01)로 자동 전송합니다. 별도 조치가 필요 없습니다.
키를 재발급하기 전에 확인하세요. -997 오류는 키·IP·API 이용신청 문제가 아닙니다. 이 셋이 모두 정상이어도 호출 주소가 틀리면 -997이 납니다.

최종 체크리스트

  • 11번가 셀러 계정으로 오픈API센터에 로그인했다.
  • [서비스 등록·확인]에서 셀러 정보를 입력하고 약관에 동의했다.
  • [Seller API 정보]에서 접근 가능 서버 IP를 등록했다.
  • 발급된 오픈API 키와 셀러 ID를 확인했다.
  • GenieGo [등록]에 api_key·seller_id를 입력해 저장했다.
