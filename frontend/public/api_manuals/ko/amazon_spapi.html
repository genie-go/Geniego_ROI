Amazon SP-API 발급 매뉴얼

Amazon 판매자 데이터를 주문·재고·상품·배송 시스템과 연동하기 위한 SP-API 개발자 등록, 앱 생성, OAuth·LWA 설정 흐름을 정리했습니다.

인증 구조LWA(OAuth) — 5개 자격증명
필요 계정Seller/Vendor Central(프로)
GenieGo 등록값refresh_token·client_id·secret·marketplace_id·seller_id
난이도중급 이상

1. Amazon SP-API란?

Amazon Selling Partner API는 주문, 재고, 상품, 배송, 정산 등 판매자 데이터를 프로그램에서 연동하기 위한 API입니다.

중요: SP-API는 단순 API Key 방식이 아닙니다. 일반적으로 Amazon Developer 등록, 앱 등록, Login with Amazon(LWA), OAuth 승인이 함께 필요합니다.
★GenieGo 연동에 필요한 값은 5개입니다: refresh_token, client_id, client_secret, marketplace_id, seller_id. (현 SP-API는 LWA 토큰만으로 호출되며, AWS IAM/SigV4 서명은 GenieGo가 내부 처리하므로 등록 칸에 입력하지 않습니다.)

2. 준비물

  • Amazon Seller Central 또는 Vendor Central 프로페셔널 셀러 계정
  • 개발자 등록에 사용할 회사/담당자 정보
  • 앱 이름, 앱 설명, 사용 목적
  • 개인정보처리방침 URL
  • OAuth Redirect URL

3. 개발자 등록

먼저 Seller Central에서 SP-API 개발자 프로필을 등록합니다.

1

Seller Central 로그인

Amazon 판매자 계정으로 Seller Central에 로그인합니다.

Seller Central → Apps & Services
2

Develop Apps 메뉴 이동

앱 및 서비스 메뉴에서 개발자 또는 앱 개발 메뉴로 이동합니다.

Apps & Services → Develop Apps
3

Developer Profile 작성

회사 정보, 담당자 정보, API 사용 목적, 데이터 보호 계획을 입력합니다.

Developer Profile
4

검토 제출

Amazon 검토를 위해 개발자 등록 정보를 제출합니다.

Submit for review
승인이 완료되어야 실제 운영용 SP-API 앱 등록과 권한 사용이 가능합니다.

4. SP-API 앱 등록 → Client ID / Client Secret

1

새 애플리케이션 생성

Develop Apps에서 새 앱을 생성합니다.

Add new app
2

앱 유형 선택

자사 계정만 연동하면 Private App, 외부 판매자에게 제공하면 Public App 성격입니다.

Private / Public App
3

API 역할 선택

Orders, Catalog, Inventory 등 필요한 역할을 선택합니다.

API Roles
4

OAuth Redirect URL 입력

승인 완료 후 돌아올 서버 URL을 입력합니다.

Redirect URI
5

저장 후 Client 정보 확인

앱 생성 후 client_id(LWA Client ID), client_secret(Client Secret)을 확인합니다.

client_id / client_secret

5. OAuth 승인 → Refresh Token 발급

1) 판매자 OAuth 승인 URL 접속
2) Redirect URL로 authorization code 수신
3) authorization code를 LWA Token Endpoint에 전달
4) refresh_token / access_token 수신
refresh_token은 판매자 승인 후 장기 호출에 사용됩니다. GenieGo는 이 refresh_token으로 access_token을 자동 갱신해 SP-API를 호출하므로, 별도의 AWS 서명 설정 없이 refresh_token만 등록하면 됩니다.

6. Marketplace ID · Seller ID 확인

GenieGo 등록에 필요한 나머지 두 식별자를 확인합니다.

marketplace_id

판매 대상 마켓플레이스 ID

연동할 국가의 Marketplace ID입니다.

US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9
seller_id

판매자 ID / Merchant Token

상품 등록·주문 조회에 필요합니다. Seller Central 계정정보에서 확인합니다.

Seller Central → 설정 → 계정 정보 → Merchant Token

7. GenieGo 연동허브에 등록

1

5개 값 준비

refresh_token · client_id · client_secret · marketplace_id · seller_id를 준비합니다.

5개 자격증명
2

연동허브 → Amazon SP-API → [등록]

각 칸에 해당 값을 입력하고 저장합니다(빈 칸은 기존 값 유지).

refresh_token · client_id · client_secret · marketplace_id · seller_id
등록한 값은 AES-256-GCM으로 암호화 저장되며 조회 시 마스킹됩니다. 저장 직후 자동 동기화가 시작됩니다.
client_secret·refresh_token은 절대 프론트엔드 코드, HTML, 공개 저장소에 노출하면 안 됩니다.

8. 최종 체크리스트

  • Developer Profile을 제출하고 승인받았다.
  • SP-API 앱을 생성하고 client_id·client_secret을 확인했다.
  • OAuth 승인으로 refresh_token을 발급받았다.
  • 연동 대상 marketplace_id를 확인했다.
  • Seller Central에서 seller_id(Merchant Token)를 확인했다.
  • GenieGo [등록]에 5개 값(refresh_token·client_id·client_secret·marketplace_id·seller_id)을 입력해 저장했다.

9. 자주 발생하는 문제

문제원인해결 방법
API Key 메뉴가 없음SP-API는 단일 API Key 구조가 아님LWA(client_id·client_secret·refresh_token) 구조로 진행합니다.
상품 등록 실패seller_id(Merchant Token) 미등록Seller Central 계정정보의 Merchant Token을 등록합니다.
잘못된 국가 데이터marketplace_id 불일치연동 국가의 정확한 Marketplace ID를 등록합니다.
권한 오류앱 API Role 부족 또는 토큰 만료API Role을 확인하고 refresh_token으로 토큰을 갱신합니다.
