Amazon SP-API 발급 매뉴얼

Amazon 판매자 데이터를 주문·재고·상품·배송 시스템과 연동하기 위한 SP-API 개발자 등록, 앱 생성, OAuth·LWA·IAM 설정 흐름을 정리했습니다.

인증 구조LWA + OAuth + AWS SigV4
필요 계정Seller/Vendor Central
주요 결과물Client ID, Secret, Token
난이도중급 이상

1. Amazon SP-API란?

Amazon Selling Partner API는 주문, 재고, 상품, 배송, 정산 등 판매자 데이터를 프로그램에서 연동하기 위한 API입니다.

중요: SP-API는 단순 API Key 방식이 아닙니다. 일반적으로 Amazon Developer 등록, 앱 등록, Login with Amazon, IAM, OAuth 승인이 함께 필요합니다.

2. 준비물

  • Amazon Seller Central 또는 Vendor Central 계정
  • 개발자 등록에 사용할 회사/담당자 정보
  • 앱 이름, 앱 설명, 사용 목적
  • 개인정보처리방침 URL
  • AWS 계정 및 IAM 설정 권한
  • OAuth Redirect URL

3. 개발자 등록

먼저 Seller Central에서 SP-API 개발자 프로필을 등록합니다.

1

Seller Central 로그인

Amazon 판매자 계정으로 Seller Central에 로그인합니다.

Seller Central → Apps & Services
2

Develop Apps 메뉴 이동

앱 및 서비스 메뉴에서 개발자 또는 앱 개발 메뉴로 이동합니다.

Apps & Services → Develop Apps
3

Developer Profile 작성

회사 정보, 담당자 정보, API 사용 목적, 데이터 보호 계획을 입력합니다.

Developer Profile
4

검토 제출

Amazon 검토를 위해 개발자 등록 정보를 제출합니다.

Submit for review
승인이 완료되어야 실제 운영용 SP-API 앱 등록과 권한 사용이 가능합니다.

4. SP-API 앱 등록

1

새 애플리케이션 생성

Develop Apps에서 새 앱을 생성합니다.

Add new app
2

앱 유형 선택

자사 계정만 연동하면 Private App, 외부 판매자에게 제공하면 Public App 성격입니다.

Private / Public App
3

API 역할 선택

Orders, Catalog, Inventory 등 필요한 역할을 선택합니다.

API Roles
4

OAuth Redirect URL 입력

승인 완료 후 돌아올 서버 URL을 입력합니다.

Redirect URI
5

저장 후 Client 정보 확인

앱 생성 후 Client ID, Client Secret 또는 관련 자격 정보를 확인합니다.

Client ID / Client Secret

5. AWS IAM 및 LWA 설정

항목의미주의사항
Login with Amazon Client ID판매자 OAuth 승인에 사용하는 클라이언트 ID프론트에 노출해도 되지만 Secret은 서버에 보관
Client Secret토큰 교환에 필요한 비밀값절대 공개 저장소에 올리지 않음
AWS IAM Role/UserSP-API 요청 서명에 필요한 AWS 권한최소 권한 원칙 적용
Refresh Token판매자 승인 후 장기 호출에 사용암호화 저장 필요

6. 토큰 발급 흐름

1) 판매자 OAuth 승인 URL 접속
2) Redirect URL로 authorization code 수신
3) authorization code를 LWA Token Endpoint에 전달
4) Refresh Token / Access Token 수신
5) Access Token + AWS SigV4 서명으로 SP-API 호출
실제 운영에서는 Access Token 만료, Refresh Token 보관, 요청 서명 실패 로그를 반드시 관리해야 합니다.

7. 최종 체크리스트

  • Developer Profile을 제출하고 승인받았다.
  • SP-API 앱을 생성했다.
  • 필요 API Role을 선택했다.
  • LWA Client ID와 Client Secret을 확인했다.
  • OAuth Redirect URL을 등록했다.
  • AWS IAM 권한과 SigV4 서명 구조를 준비했다.
  • Refresh Token을 안전하게 저장한다.
