API Key 발급 매뉴얼

CJ대한통운 전용 API는 계약(고객) 코드 기반으로 발급되며, 영업 담당으로 계약(고객) 코드와 API 키를 발급받아 GenieGo 연동 허브에 등록하는 과정을 정리한 매뉴얼입니다.

1. 시작하기 전에

CJ대한통운 전용 API는 자가 콘솔에서 즉시 발급되지 않는 계약형 채널입니다.

필요

CJ대한통운 계약(고객) 코드

CJ대한통운과의 계약을 통해 발급되는 계약(고객) 코드가 있어야 합니다. 별도의 콘솔에서 직접 발급하는 방식이 아닙니다.

권장

스마트택배(통합 추적)

배송 추적만 필요하다면 [스마트택배(통합 추적)] 채널에 t_key 하나로 전 택배사를 연동할 수 있습니다.

참고: 배송 추적이 목적이라면 전용 API 계약 없이 스마트택배(스윗트래커) 통합 키로 연동하는 방법을 권장합니다.

2. 발급 신청 절차

아래 순서대로 신청·등록합니다.

1

계약(고객) 코드 준비

CJ대한통운 계약(고객) 코드를 준비합니다. 전용 API는 계약형으로 제공됩니다.

2

통합 추적 여부 판단

배송 추적만 필요하면 CJ대한통운 전용 채널 대신 [스마트택배(통합 추적)] 채널에 t_key 하나로 전 택배사를 연동할 수 있습니다.

3

전용 API 발급 신청

CJ대한통운 전용 API가 필요하면 영업 담당으로 계약(고객) 코드와 API 키를 발급받습니다.

4

GenieGo 등록

GenieGo 연동 허브 [등록]에 api_key와 cust_code를 입력해 저장합니다.

영업 담당으로부터 API 키와 계약(고객) 코드를 수령하면 등록 준비가 완료된 것입니다.

3. GenieGo에 등록할 정보

필드설명
api_key발급받은 CJ대한통운 전용 API 키
cust_codeCJ대한통운 계약(고객) 코드
GenieGo 연동 허브 [등록]에 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
콘솔에서 발급 메뉴를 찾을 수 없음CJ대한통운 전용 API는 계약형으로 자가 콘솔 발급이 아님영업 담당으로 계약(고객) 코드와 API 키를 발급받습니다.
배송 추적만 필요한데 절차가 복잡함전용 API 계약이 추적 목적에는 과함[스마트택배(통합 추적)] 채널에 t_key를 등록해 연동합니다.
등록 후 인증 오류api_key 또는 cust_code 오기재발급받은 api_key와 계약(고객) 코드를 다시 확인합니다.

최종 체크리스트

  • CJ대한통운 계약(고객) 코드를 확보했다.
  • 배송 추적만 필요한 경우 스마트택배(통합 추적) 연동을 검토했다.
  • 영업 담당으로 API 키와 계약(고객) 코드를 발급받았다.
  • GenieGo [등록]에 api_key·cust_code를 입력해 저장했다.
