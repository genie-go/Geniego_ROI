1. Kakao 알림톡 발급 개념

알림톡은 주문, 예약, 배송, 결제 등 정보성 메시지를 카카오톡으로 발송하는 비즈메시지 서비스입니다. 일반 카카오 개발자 앱 키만으로 바로 발송하는 구조가 아니라, 비즈니스 채널 인증과 발송 대행사/플랫폼 설정이 필요합니다.

중요: 알림톡은 광고성 메시지가 아니라 정보성 메시지 중심이며, 템플릿 사전 심사가 필요합니다.

2. 준비물

  • 카카오 비즈니스 계정
  • 카카오톡 채널
  • 사업자등록증 등 비즈니스 인증 자료
  • 발송할 알림톡 템플릿 문구
  • 발송 대행사 또는 클라우드 메시징 서비스 계정
  • 수신자 전화번호 수집 및 동의 관리 정책

3. 카카오톡 채널 준비

1

카카오비즈니스 접속

카카오비즈니스 관리자센터에 로그인합니다.

business.kakao.com
2

카카오톡 채널 생성

브랜드 또는 서비스 이름으로 채널을 생성합니다.

카카오톡 채널 관리자센터
3

비즈니스 인증 진행

사업자 정보와 증빙 자료를 제출해 비즈니스 채널 인증을 진행합니다.

비즈니스 인증
4

채널 검색용 ID 확인

알림톡 발송 설정에 필요한 채널 ID 또는 검색용 ID를 확인합니다.

채널 프로필 → 검색용 ID
비즈니스 인증이 완료되어야 알림톡 발송 프로필 등록을 진행할 수 있습니다.

4. 발송 프로필 / Sender Profile 등록

1

발송 플랫폼 선택

NHN Cloud, Naver Cloud SENS, Infobank, Bizgo 등 알림톡 제공사를 선택합니다.

Messaging Provider
2

프로젝트 생성

선택한 플랫폼에서 알림톡 프로젝트를 생성합니다.

Project / Service
3

카카오톡 채널 연결

비즈니스 인증된 카카오톡 채널 정보를 등록합니다.

Sender Profile
4

발신 프로필 검수

채널 소유 확인과 발신 프로필 검수를 진행합니다.

Profile Review
5

API 인증 정보 확인

플랫폼에서 제공하는 App Key, Secret Key, API Key 등을 확인합니다.

API Key / Secret

5. 템플릿 등록 및 승인

1

템플릿 작성

주문 확인, 배송 안내 등 정보성 문구를 작성합니다.

Template Message
2

변수 설정

이름, 주문번호, 배송번호 등 동적으로 바뀌는 값을 변수로 설정합니다.

#{'{'}변수{'}'}
3

검수 요청

템플릿을 저장하고 카카오 검수 요청을 진행합니다.

Request Approval
4

승인 확인

승인된 템플릿 코드 또는 ID를 확인합니다.

Template Code
검수 전 문구와 실제 발송 문구가 다르면 발송 실패 또는 제재가 발생할 수 있습니다.

6. API 발송 흐름 예시

1) 서버에서 알림톡 API Endpoint 호출
2) API Key 또는 Secret으로 인증
3) 수신자 전화번호, 템플릿 코드, 변수값 전달
4) 발송 결과 수신
5) 실패 시 SMS 대체 발송 여부 처리

  • 비즈니스 인증을 완료했다.
  • 알림톡 발송 플랫폼을 선택했다.
  • Sender Profile을 등록했다.
  • API Key 또는 Secret을 확인했다.
  • 템플릿을 등록하고 승인받았다.
  • 발송 실패 및 대체문자 정책을 준비했다.
