EMS (우체국 국제특송)
API Key 발급 매뉴얼

우체국 국제특송(EMS)은 계약(고객) 코드 기반의 계약형 서비스입니다. EMS 계약 후 발급받은 우체국 API 키와 고객(계약) 코드를 GenieGo 연동 허브에 등록하는 절차를 정리한 매뉴얼입니다.

소요 시간계약 처리 기간에 따라 상이
필요 계정우체국 EMS 계약(고객 코드)
발급 키우체국 API 키 · 고객(계약) 코드
결과물API 자격증명 · 계약 코드

1. 시작하기 전에

EMS는 자가 콘솔 발급이 아닌 계약(고객) 코드 기반의 계약형 서비스입니다.

안내: 우체국 국제특송(EMS)은 계약(고객) 코드 기반입니다. 자격증명을 자체 발급하는 방식이 아니라 EMS 계약을 진행한 뒤 고객(계약) 코드와 우체국 API 키를 받는 절차로 진행됩니다.
준비물

우체국 EMS 계약(고객 코드)

우체국 국제특송(EMS) 계약(고객) 코드가 필요합니다. 계약을 통해서만 발급됩니다.

수령 자격증명

우체국 API 키 · 고객(계약) 코드

계약 완료 후 고객(계약) 코드와 우체국 API 키를 발급받습니다.

2. 발급 신청 절차

계약형 서비스이므로 아래 순서로 계약·자격증명 수령 후 등록합니다.

1

EMS 계약 진행

인터넷우체국 또는 계약창구에서 EMS 계약을 진행합니다.

2

고객 코드·API 키 수령

계약 완료 후 고객(계약) 코드와 우체국 API 키를 발급받습니다.

3

GenieGo에 등록

수령한 자격증명을 GenieGo [등록]에 입력해 저장합니다.

계약형 서비스이므로 계약 처리 상태에 따라 자격증명 수령까지 시간이 소요될 수 있습니다.

3. GenieGo에 등록할 정보

필드설명
api_key발급받은 우체국 API 키
cust_code고객(계약) 코드
GenieGo 연동 허브 [등록]에 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
자격증명을 발급받을 수 없음EMS 계약 미완료인터넷우체국 또는 계약창구에서 EMS 계약을 먼저 완료합니다.
등록 후 인증 오류API 키 또는 고객(계약) 코드 오기재발급받은 api_key와 cust_code를 다시 확인합니다.

최종 체크리스트

  • 우체국 국제특송(EMS) 계약을 진행했다.
  • 고객(계약) 코드와 우체국 API 키를 발급받았다.
  • 발급받은 자격증명을 확인했다.
  • GenieGo [등록]에 api_key·cust_code를 입력해 저장했다.
