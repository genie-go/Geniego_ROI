DHL Express
API Key 발급 매뉴얼

DHL Express 개발자 포털(developer.dhl.com)에서 API Key와 Secret을 발급하고, DHL Express 고객 계정번호와 함께 GenieGo 연동 허브에 등록하는 과정을 초보자도 그대로 따라 할 수 있도록 정리한 매뉴얼입니다.

소요 시간약 10~15분(승인 대기 별도)
필요 계정DHL Express 고객 계정(계정번호)
발급 키API Key · Secret
결과물API 자격증명 · 계정번호

1. 시작하기 전에

DHL Express 자격증명은 개발자 포털에서 신청하며, 고객 계정번호가 필요합니다.

준비물

DHL Express 고객 계정번호

DHL Express 고객 계정번호가 있으면 자격증명 신청이 수월합니다. 계정번호는 GenieGo 등록 시에도 함께 입력합니다.

발급 키

API Key · Secret

포털에서 본인 앱(회사명)을 통해 API Key와 Secret을 확인하게 됩니다.

참고: [Get Access]로 고객 정보를 등록하고 자격증명을 신청하면, 보통 영업일 다음 날 Test/Production 승인 메일을 받습니다.

2. 발급 단계

아래 순서대로 발급하고 GenieGo에 등록합니다.

1

개발자 포털 가입

DHL Express 개발자 포털에 접속해 이메일로 가입하고 인증합니다.

https://developer.dhl.com
2

자격증명 신청

로그인 후 [Get Access] 섹션에서 고객 정보를 등록하고 자격증명을 신청합니다.

Get Access
3

승인 메일 수신

보통 영업일 다음 날 Test/Production 승인 메일을 받습니다.

4

API Key·Secret 확인

포털에서 본인 앱(회사명) > [Show Key]로 API Key와 Secret을 확인합니다.

본인 앱(회사명) → Show Key
승인 메일을 받고 포털에서 API Key·Secret이 보이면 정상 발급된 것입니다.

3. GenieGo에 등록할 정보

필드설명
api_key포털에서 확인한 DHL API Key
account_numberDHL Express 고객 계정번호
GenieGo 연동 허브 [등록]에 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
승인 메일이 오지 않음영업일 익일 승인 처리 대기 또는 고객 정보 미등록[Get Access]에서 고객 정보 등록·자격증명 신청이 완료됐는지 확인하고 영업일 기준으로 기다립니다.
호출 시 인증 오류Test/Production 환경 혼동 또는 계정번호 누락승인된 환경의 API Key·Secret과 계정번호(account_number)를 다시 확인합니다.

최종 체크리스트

  • DHL Express 고객 계정번호를 확인했다.
  • developer.dhl.com에 가입하고 [Get Access]로 자격증명을 신청했다.
  • 영업일 익일 Test/Production 승인 메일을 받았다.
  • 포털에서 API Key·Secret을 확인했다.
  • GenieGo [등록]에 api_key·account_number를 입력해 저장했다.
