Google Ads
API Key 발급 매뉴얼

Google Ads 관리자(MCC) 계정의 API 센터에서 개발자 토큰을 발급하고, Google Cloud에서 OAuth 클라이언트를 만들어 refresh token을 획득한 뒤 GenieGo 연동 허브에 등록하는 과정을 정리한 매뉴얼입니다.

소요 시간약 20~40분(개발자 토큰 심사 별도)
필요 계정Google Ads 관리자(MCC) 계정 · Google 계정
발급 키개발자 토큰 · client_id/secret · refresh_token · customer_id
결과물Google Ads API 자격증명

1. 시작하기 전에

Google Ads 연동은 관리자(MCC) 계정과 Google Cloud의 OAuth 설정이 모두 필요합니다.

개발자 토큰

MCC API 센터

Google Ads 관리자(MCC) 계정의 API 센터에서 개발자 토큰을 신청합니다. 기본(Basic) 액세스는 Google 심사를 거쳐 승인됩니다.

OAuth

Google Cloud 클라이언트

console.cloud.google.com에서 OAuth 클라이언트(client_id·client_secret)를 만들고 인증해 refresh token을 획득합니다.

참고: 개발자 토큰의 기본 액세스는 Google 심사를 거쳐 승인되며, 승인 전에는 테스트 계정만 호출됩니다. 관리자(MCC) 계정이 없으면 ads.google.com에서 먼저 만드세요.

2. 발급 단계

아래 순서대로 발급하고 GenieGo에 등록합니다.

1

관리자 계정 로그인 · API 센터 이동

Google Ads 관리자 계정에 로그인한 뒤 상단 [도구 및 설정] > [설정] > [API 센터]로 이동합니다.

도구 및 설정 → 설정 → API 센터
2

개발자 토큰 신청

API 센터에서 [개발자 토큰]을 신청합니다. 기본(Basic) 액세스는 Google 심사를 거쳐 승인되며, 승인 전에는 테스트 계정만 호출됩니다.

3

Cloud 프로젝트 · OAuth 동의 화면 구성

프로젝트를 만들고 [API 및 서비스] > [OAuth 동의 화면]을 구성합니다.

console.cloud.google.com → API 및 서비스 → OAuth 동의 화면
4

OAuth 클라이언트 ID 생성

[사용자 인증 정보] > [사용자 인증 정보 만들기] > [OAuth 클라이언트 ID]를 만들어 client_id와 client_secret을 발급받습니다(범위 scope: adwords).

사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID
5

refresh token 발급

OAuth 인증 절차로 refresh token을 발급받습니다(Google OAuth Playground 또는 자체 인증 사용).

6

고객 ID 확인

호출 대상 광고 계정의 10자리 고객 ID(customer_id)를 확인합니다.

개발자 토큰 승인 + client_id·client_secret·refresh_token·customer_id가 모두 준비되면 정상 발급된 것입니다.

3. GenieGo에 등록할 정보

필드설명
developer_tokenAPI 센터에서 발급된 개발자 토큰
client_idOAuth 클라이언트 ID
client_secretOAuth 클라이언트 시크릿(노출 주의)
refresh_tokenOAuth 인증으로 발급된 refresh token([액세스 토큰] 칸에 입력)
customer_id대상 광고 계정의 10자리 고객 ID
GenieGo [등록]에 개발자 토큰, client_id, client_secret, refresh_token, customer_id를 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
테스트 계정만 호출됨개발자 토큰 기본 액세스 심사 미승인API 센터에서 신청한 개발자 토큰의 Google 심사가 승인될 때까지 기다린 뒤 운영 계정을 호출합니다.
인증 실패scope(adwords) 누락 또는 refresh_token 오기재OAuth 클라이언트에 adwords 범위를 포함해 다시 인증하고 refresh_token을 재확인합니다.
고객 ID 형식 오류10자리 customer_id 오기재대상 광고 계정의 10자리 고객 ID를 다시 확인해 입력합니다.

최종 체크리스트

  • Google Ads 관리자(MCC) 계정을 준비하고 로그인했다.
  • API 센터에서 개발자 토큰을 신청했다(기본 액세스는 Google 심사).
  • console.cloud.google.com에서 OAuth 동의 화면과 OAuth 클라이언트 ID/Secret을 만들었다(scope: adwords).
  • OAuth 인증으로 refresh token을 발급받았다.
  • 대상 계정의 10자리 customer_id를 확인했다.
  • GenieGo [등록]에 개발자 토큰·client_id/secret·refresh_token·customer_id를 입력해 저장했다.
