API Key 발급 매뉴얼

TikTok for Business 개발자 포털에서 앱을 만들어 액세스 토큰을 발급하고, 광고주 ID(advertiser_id)와 함께 GenieGo 연동 허브에 등록해 TikTok Business 광고 데이터를 연동하는 과정을 정리한 매뉴얼입니다.

소요 시간약 10~20분(운영 전환 심사 별도)
필요 계정TikTok For Business 광고 계정
발급 키액세스 토큰 · 광고주 ID
결과물TikTok Business API 자격증명

1. 시작하기 전에

TikTok Business 연동에는 광고 계정과 개발자(조직) 등록이 필요합니다.

필요

TikTok For Business 광고 계정

광고를 집행 중인 TikTok For Business 광고 계정이 있어야 합니다.

발급 키

액세스 토큰 · 광고주 ID

앱의 App ID·Secret으로 광고주 인가를 거쳐 액세스 토큰을 얻고, 연결된 advertiser_id를 확인합니다.

참고: 신규 앱은 샌드박스 상태로 시작하며, 운영(Production)으로 전환할 때 심사가 필요합니다.

2. 발급 단계

아래 순서대로 발급하고 GenieGo에 등록합니다.

1

개발자 등록

TikTok for Business 개발자 포털에 로그인하고 개발자(조직) 등록을 완료합니다.

developers.tiktok.com
2

앱 만들기

[My Apps] > [Create an App]을 눌러 앱 이름과 용도 설명을 입력하고 앱을 만듭니다.

My Apps → Create an App
3

App ID · Secret 확인

앱 상세 페이지에서 App ID와 Secret을 확인합니다.

4

광고주 인가

광고주 인가(authorization) 링크로 광고 계정 접근을 승인하면 auth_code가 발급됩니다.

5

액세스 토큰 교환

auth_code를 access_token으로 교환합니다(앱이 자동 처리하거나 포털 가이드 참고).

6

광고주 ID 확인

연결된 advertiser_id(광고주 ID)를 확인합니다.

광고주 인가 후 auth_code가 access_token으로 교환되고 advertiser_id가 확인되면 정상 발급된 것입니다.

3. GenieGo에 등록할 정보

필드설명
access_tokenauth_code를 교환해 얻은 액세스 토큰(노출 주의)
advertiser_id연결된 광고주 ID
GenieGo [등록]에 access_token과 advertiser_id를 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
운영 데이터 호출 안 됨신규 앱이 샌드박스 상태운영(Production) 전환 심사를 신청·완료한 뒤 호출합니다.
광고주 인가 실패광고 계정 접근 미승인 또는 잘못된 인가 링크광고주 인가 링크로 광고 계정 접근을 다시 승인해 auth_code를 발급받습니다.
토큰 교환 실패auth_code 만료 또는 재사용광고주 인가를 다시 진행해 새 auth_code로 access_token을 교환합니다.

최종 체크리스트

  • developers.tiktok.com에서 개발자(조직) 등록을 완료했다.
  • My Apps > Create an App으로 앱을 만들고 용도 설명을 입력했다.
  • 앱 상세에서 App ID·Secret을 확인했다.
  • 광고주 인가로 auth_code를 받아 access_token으로 교환했다.
  • 연결된 advertiser_id를 확인했다.
  • GenieGo [등록]에 access_token·advertiser_id를 입력해 저장했다.
