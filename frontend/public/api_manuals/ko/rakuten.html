Rakuten
API Key 발급 매뉴얼

Rakuten RMS 판매자 콘솔에서 RMS Web Service 인증키(Service Secret·License Key)를 발급하고 GenieGo 연동 허브에 등록하는 과정을 초보자도 그대로 따라 할 수 있도록 정리한 매뉴얼입니다.

소요 시간약 5~15분
필요 계정Rakuten 판매자(이치바) 계정
발급 키Service Secret · License Key
결과물RMS Web Service 자격증명

1. 시작하기 전에

판매자(이치바)용 키는 RMS Web Service에서 발급합니다.

필요

Rakuten 판매자 계정

Rakuten 판매자(이치바) 계정이 있어야 합니다. RMS R-Login(glogin.rms.rakuten.co.jp)으로 로그인할 수 있는 계정이 필요합니다.

발급 키

Service Secret · License Key

RMS Web Service에서 serviceSecret과 licenseKey를 발급받습니다. License Key는 90일 만료가 있으며 재발급이 가능합니다.

주의: 판매자(이치바) 키를 발급하는 RMS Web Service(glogin.rms.rakuten.co.jp)는 제휴/상품검색용 webservice.rakuten.co.jp와 다릅니다. 반드시 RMS 쪽에서 발급하세요.

2. 발급 단계

아래 순서대로 발급하고 GenieGo에 등록합니다.

1

RMS 개발자 포털 로그인

Rakuten 개발자 포털(RMS/Webservice)에 판매자 계정으로 로그인합니다.

https://glogin.rms.rakuten.co.jp
2

RMS Web Service 이동

RMS R-Login으로 로그인한 뒤 RMS Web Service 영역으로 이동합니다.

RMS R-Login → RMS Web Service
3

Service Secret · License Key 발급

RMS Web Service에서 Service Secret과 License Key를 발급받습니다. License Key는 90일 만료가 있어 만료 시 재발급합니다.

4

키 확인

발급된 Service Secret과 License Key, 그리고 본인 상점의 Shop URL을 함께 확인합니다.

Service Secret과 License Key가 발급 화면에 표시되면 정상 발급된 것입니다.

3. GenieGo에 등록할 정보

필드설명
service_secret발급된 Service Secret
license_key발급된 License Key(90일 만료·재발급 가능, 노출 주의)
shop_url본인 상점의 Shop URL
GenieGo 연동 허브 [등록]에 service_secret·license_key·shop_url을 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
엉뚱한 키가 발급됨webservice.rakuten.co.jp(제휴/상품검색용)에서 발급RMS R-Login(glogin.rms.rakuten.co.jp)의 RMS Web Service에서 다시 발급합니다.
일정 기간 후 인증 실패License Key 90일 만료RMS Web Service에서 License Key를 재발급하고 [등록]에 다시 입력합니다.

최종 체크리스트

  • Rakuten 판매자(이치바) 계정으로 RMS R-Login에 로그인했다.
  • RMS Web Service에서 Service Secret·License Key를 발급했다.
  • 본인 상점의 Shop URL을 확인했다.
  • GenieGo [등록]에 service_secret·license_key·shop_url을 입력해 저장했다.
