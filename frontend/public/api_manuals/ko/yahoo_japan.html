1. Yahoo! Japan API 발급 개념

Yahoo! Japan 계열 API는 서비스 종류에 따라 Yahoo! JAPAN Developers 또는 LY/Yahoo!広告 Developer Center에서 Client ID, Client Secret, OAuth 인증 정보를 발급받습니다.

중요: “Yahoo! Japan”은 사용하려는 API가 무엇인지에 따라 발급 사이트와 심사 절차가 달라집니다. 쇼핑, 광고, 로그인, 검색 API를 구분해야 합니다.

2. 준비물

  • Yahoo! JAPAN ID 또는 비즈니스 계정
  • 사용하려는 API 종류 결정
  • 앱 이름과 서비스 설명
  • Redirect URI
  • 개인정보처리방침 URL
  • 운영 도메인 또는 테스트 도메인

3. 애플리케이션 등록 기본 순서

1

개발자 사이트 접속

Yahoo! JAPAN Developers 또는 해당 API의 Developer Center에 로그인합니다.

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

애플리케이션 생성

새 애플리케이션 추가 메뉴를 클릭합니다.

Create Application / Add Application
3

앱 정보 입력

앱 이름, 설명, 담당자 정보, 웹사이트 URL을 입력합니다.

Application Information
4

Redirect URI 입력

OAuth 인증 후 돌아올 URL을 정확히 입력합니다.

Redirect URI
5

권한 또는 API 선택

사용하려는 API 권한과 스코프를 선택합니다.

Scope / API Permission
6

등록 완료

등록 후 Client ID와 Client Secret을 확인합니다.

Client ID / Client Secret

5. OAuth 토큰 발급 흐름

1) Client ID로 사용자 인증 URL 생성
2) 사용자가 Yahoo! 계정으로 로그인
3) Redirect URI로 authorization code 수신
4) authorization code + Client Secret으로 Access Token 요청
5) Access Token으로 API 호출
6) 필요 시 Refresh Token으로 갱신
