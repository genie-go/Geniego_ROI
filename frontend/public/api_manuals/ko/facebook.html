f Facebook Live API Beginner Manual

Facebook Live
API 발급 매뉴얼

Meta for Developers에서 앱을 만들고 Facebook 페이지 라이브 연동에 필요한 App ID, App Secret, 페이지 액세스 토큰(Page Access Token)을 준비하는 과정을 초보자도 따라 할 수 있도록 정리했습니다.

핵심 개념API Key가 아니라 Access Token
개발 사이트developers.facebook.com
사용 APILive Video API (Graph)
필요 권한페이지 관리자 + Page Token

1. 먼저 알아야 할 핵심

YouTube는 API Key 한 개로 끝나지만, Facebook(Meta)은 App ID · App Secret · Access Token 구조로 동작합니다.

중요: Meta는 “API Key 하나 복사”로 끝나지 않습니다. 보통 앱 생성 → 제품 추가 → 권한 설정 → 페이지 액세스 토큰 발급 → 앱 검수 순서가 필요합니다.

2. 준비물

  • Meta/Facebook 계정
  • Meta for Developers 접속 권한
  • 관리 권한이 있는 Facebook 페이지
  • 실제 서비스용 개인정보처리방침 URL · 서비스 약관 URL

3. Meta Developer 앱 만들기

모든 Meta API 연동은 앱을 만드는 것에서 시작합니다.

1

Meta for Developers 접속

브라우저에서 Meta for Developers에 로그인합니다.

developers.facebook.com
2

내 앱 → 앱 만들기

상단 My Apps → Create App을 클릭합니다.

My Apps → Create App
3

앱 유형 선택

플랫폼 연동/라이브커머스는 비즈니스 목적에 가깝습니다.

Business 유형 선택
4

앱 이름·이메일 입력 → 생성

앱 이름, 연락 이메일을 입력하고 앱을 생성합니다.

App Name / Contact Email
앱 생성 후 대시보드 → 앱 설정 → 기본 설정에서 App ID와 App Secret을 확인합니다. (App Secret은 프론트엔드/공개 저장소에 절대 노출 금지)

4. Facebook Live API 설정 흐름

1

제품 추가

앱 대시보드에서 Facebook Login / Graph API 관련 설정을 준비합니다.

Add Product → Facebook Login
2

페이지 연결 + 페이지 토큰

라이브를 페이지에 송출하려면 페이지 관리자 권한과 Page Access Token이 필요합니다.

Page Access Token
3

필요 권한 신청(App Review)

콘텐츠 게시·페이지 관리·라이브 비디오 생성 목적의 권한(예: pages_read_engagement, pages_manage_posts)을 신청합니다.

Permissions → App Review
4

Graph API Explorer로 토큰 발급·테스트

권한이 포함된 토큰을 발급하고 API 호출을 테스트합니다.

Tools → Graph API Explorer

Facebook Live 생성 호출 예시

POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
정상 생성되면 방송 ID·스트림 URL·스트림 키 등 라이브 송출 정보가 반환됩니다. 페이지 ID(page_id)는 페이지 정보에서 확인합니다.

5. GenieGo 연동허브에 등록

1

페이지 액세스 토큰 복사

발급한 Page Access Token을 복사합니다(장기 토큰 권장).

access_token (Page token)
2

페이지 ID 확인

대상 Facebook 페이지의 ID(page_id)를 확인합니다.

page_id
3

연동허브에 등록

GenieGo 연동허브 → Facebook Live → [등록]에 아래 값을 입력해 저장합니다.

access_token · page_id
등록한 키는 AES-256-GCM으로 암호화 저장되며 조회 시 마스킹됩니다.

6. Access Token 종류

토큰 종류용도주의사항
User Access Token사용자 로그인 기반 API 호출권한 승인 범위에 따라 접근 데이터가 달라집니다.
Page Access Token페이지 게시·페이지 라이브 생성페이지 관리자 권한과 관련 권한이 필요합니다.
Long-Lived Token만료 시간을 늘린 장기 토큰운영 시 만료·갱신 로직이 필요합니다.

7. 체크리스트

  • Meta Developer 앱을 생성했다.
  • App ID와 App Secret을 확인했다.
  • Facebook 페이지를 연결하고 Page Access Token을 발급했다.
  • 필요 권한(App Review)을 정리·신청했다.
  • GenieGo [등록]에 access_token·page_id를 입력해 저장했다.
  • 토큰을 서버 환경변수에 안전하게 보관했다.

8. 자주 발생하는 문제

문제원인해결 방법
API Key 메뉴가 없음Meta는 단일 API Key 구조가 아님App ID·App Secret·Access Token 구조로 진행합니다.
권한 오류토큰에 필요한 권한 없음Graph API Explorer에서 권한 추가 후 토큰을 재발급합니다.
페이지 라이브 생성 실패Page Access Token/관리자 권한 부족페이지 관리자 권한과 토큰 권한을 확인합니다.
