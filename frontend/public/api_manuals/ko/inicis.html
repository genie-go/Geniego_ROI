KG이니시스
API Key 발급 매뉴얼

KG이니시스 상점관리자에서 INIAPI Key와 웹표준 사인키(SignKey)를 발급하고, 상점 ID(MID)와 함께 GenieGo 연동 허브에 등록하는 과정을 정리한 매뉴얼입니다.

소요 시간약 5~15분
필요 계정KG이니시스 가맹점 계약·상점관리자 계정
발급 키MID · API 키 · 사인키(SignKey)
결과물결제 연동 자격증명

1. 시작하기 전에

KG이니시스는 가맹점 계약이 완료된 후에만 인증키를 발급할 수 있습니다.

필요

가맹점 계약·상점관리자 계정

KG이니시스 가맹점 계약이 체결되어 있어야 하며, 상점관리자 계정으로 로그인할 수 있어야 합니다.

발급 키

INIAPI Key · 웹표준 사인키

상점관리자에서 INIAPI Key를 생성하고, 웹결제 signkey 생성으로 웹표준 사인키를 함께 발급합니다.

참고: 등록에는 상점 ID(MID)·사인키(SignKey)·API 키 세 가지가 필요합니다.

2. 발급 단계

아래 순서대로 발급하고 GenieGo에 등록합니다.

1

상점관리자 로그인

이니시스 상점관리자에 로그인합니다.

https://iniweb.inicis.com
2

부가정보 메뉴 이동

[상점정보] > [계약정보] > [부가정보]로 이동합니다.

상점정보 → 계약정보 → 부가정보
3

INIAPI Key 생성

INIAPI Key를 생성합니다.

4

웹표준 사인키 생성

[웹결제 signkey 생성]을 눌러 웹표준 사인키(SignKey)를 발급합니다.

웹결제 signkey 생성
INIAPI Key와 웹표준 사인키가 모두 생성되면 정상 발급된 것입니다.

3. GenieGo에 등록할 정보

필드설명
MID상점 ID(MID)
sign_key발급된 웹표준 사인키(SignKey, 노출 주의)
api_key발급된 INIAPI Key
GenieGo 연동 허브 [등록]에 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
부가정보 메뉴가 보이지 않음가맹점 계약 미완료 또는 권한 없는 계정가맹점 계약을 완료한 상점관리자 계정으로 로그인합니다.
결제 인증 실패사인키 또는 API 키 오기재상점정보 > 계약정보 > 부가정보에서 발급한 사인키·INIAPI Key를 다시 확인합니다.

최종 체크리스트

  • KG이니시스 가맹점 계약을 완료하고 상점관리자에 로그인했다.
  • 상점정보 > 계약정보 > 부가정보에서 INIAPI Key를 생성했다.
  • [웹결제 signkey 생성]으로 웹표준 사인키를 발급했다.
  • GenieGo [등록]에 MID·sign_key·api_key를 입력해 저장했다.
