API Key 발급 매뉴얼

Adyen 가맹 계약·심사 승인 후 Customer Area에서 API 키(X-API-Key)와 Merchant Account를 발급해 GenieGo 연동 허브에 등록하는 과정을 정리한 매뉴얼입니다.

소요 시간심사 승인 후 약 5~15분
필요 자격Adyen 가맹 계약·심사 승인
발급 키API 키(X-API-Key) · Merchant Account
결과물API 자격증명

1. 시작하기 전에

Adyen은 엔터프라이즈 글로벌 PG로, 자격증명 발급 전에 가맹 계약·심사 승인이 선행되어야 합니다.

필요

가맹 계약·심사 승인

Adyen은 엔터프라이즈 글로벌 PG로 가맹 계약과 심사 승인이 필요합니다. 승인 후에야 자격증명을 발급할 수 있습니다.

발급 키

X-API-Key · Merchant Account

승인 후 Customer Area의 [Developers] > [API credentials]에서 API 키(X-API-Key)를 생성하고 Merchant Account명을 확인합니다.

선행 조건: 가맹 계약·심사 승인 전에는 자가 발급이 불가능합니다. 먼저 Adyen과 가맹 계약·심사를 완료해야 합니다.

2. 발급 신청 절차

심사 승인 후 아래 순서대로 자격증명을 발급·등록합니다.

1

가맹 계약·심사 승인

Adyen과 가맹 계약을 체결하고 심사 승인을 받습니다.

Adyen 영업/계약
2

Customer Area 로그인

승인 후 Customer Area에 로그인합니다.

https://ca-live.adyen.com
3

API 자격증명 발급

[Developers] > [API credentials]에서 API 키(X-API-Key)를 생성하고 Merchant Account명을 확인합니다.

Developers → API credentials
4

시작 정산배치 번호 확인

정산 첫 수집을 위해 Customer Area에서 최근 settlement batch 번호를 확인합니다.

API 키(X-API-Key)와 Merchant Account가 확인되면 자격증명 준비가 완료된 것입니다.

3. GenieGo에 등록할 정보

필드설명
api_key발급된 API 키(X-API-Key, 노출 주의)
merchant_account확인한 Merchant Account명
batch_start정산 첫 수집용 시작 정산배치 번호(Customer Area의 최근 settlement batch 번호)
GenieGo 연동 허브 [등록]에 api_key·merchant_account·batch_start를 입력해 저장하면 자격증명은 AES-256-GCM으로 암호화 저장되고 조회 시 마스킹됩니다.

4. 자주 발생하는 문제

문제원인해결
API credentials 메뉴가 없음가맹 계약·심사 미승인Adyen과 가맹 계약·심사를 완료한 뒤 Customer Area에 접속합니다.
인증 실패X-API-Key 또는 Merchant Account 오기재Developers > API credentials에서 발급한 API 키와 Merchant Account명을 다시 확인합니다.

최종 체크리스트

  • Adyen 가맹 계약·심사 승인을 받았다.
  • Customer Area(ca-live.adyen.com)에 로그인했다.
  • Developers > API credentials에서 API 키(X-API-Key)와 Merchant Account를 확인했다.
  • 최근 settlement batch 번호(batch_start)를 확인했다.
  • GenieGo [등록]에 api_key·merchant_account·batch_start를 입력해 저장했다.
