LINE API 발급 매뉴얼

용도메시징·챗봇

1. LINE API 발급 개념

LINE 연동은 LINE Developers Console에서 Provider와 Channel을 만들고, Messaging API 또는 LINE Login용 Channel ID, Channel Secret, Channel Access Token을 발급받는 방식입니다.

구분: 메시지 발송은 Messaging API 채널, 로그인 연동은 LINE Login 채널을 사용합니다.

2. 준비물

  • LINE 계정 또는 비즈니스 계정
  • LINE Developers Console 접속 권한
  • LINE Official Account
  • 서비스 이름과 설명
  • Webhook URL
  • 개인정보처리방침 URL 및 약관 URL

3. Provider 생성

1

LINE Developers 접속

LINE Developers Console에 로그인합니다.

developers.line.biz
2

Provider 생성

회사명 또는 서비스명으로 Provider를 만듭니다.

Create Provider
3

기본 정보 입력

서비스 운영 주체, 연락처, 약관 정보를 입력합니다.

Provider Settings

4. Messaging API 채널 만들기

1

새 채널 생성

Provider 안에서 Create a new channel을 클릭합니다.

Create a new channel
2

Messaging API 선택

채널 유형에서 Messaging API를 선택합니다.

Messaging API
3

채널 정보 입력

채널 이름, 설명, 카테고리, 이메일 등을 입력합니다.

Channel Information
4

LINE Official Account 연결

메시지를 보낼 공식 계정을 연결하거나 새로 생성합니다.

LINE Official Account
5

채널 생성 완료

생성 후 Channel ID와 Channel Secret을 확인합니다.

Channel ID / Channel Secret

5. Channel Access Token 발급

1

Messaging API 탭 이동

생성한 채널 화면에서 Messaging API 탭으로 이동합니다.

Messaging API tab
2

Channel access token 항목 찾기

하단의 Channel access token 영역을 확인합니다.

Channel access token
3

Issue 클릭

Issue 또는 Reissue 버튼을 눌러 토큰을 발급합니다.

Issue
4

서버에 안전하게 저장

발급된 토큰은 서버 환경변수나 Secret Manager에 저장합니다.

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token은 메시지 발송 권한을 가진 비밀키입니다. HTML, 프론트엔드, 공개 저장소에 노출하면 안 됩니다.

6. Webhook 설정

1

Webhook URL 입력

사용자 메시지 이벤트를 받을 서버 URL을 입력합니다.

Webhook URL
2

Use webhook 활성화

Webhook 사용 옵션을 Enabled로 변경합니다.

Use webhook: Enabled
3

Verify 테스트

LINE에서 서버로 테스트 요청을 보내 연결을 확인합니다.

Verify
4

응답 코드 확인

서버가 200 OK를 반환하는지 확인합니다.

HTTP 200 OK

7. 메시지 발송 예시

POST https://api.line.me/v2/bot/message/push
Authorization: Bearer {CHANNEL_ACCESS_TOKEN}
Content-Type: application/json

{
  "to": "USER_ID",
  "messages": [
    {"type": "text", "text": "안녕하세요. LINE 메시지 테스트입니다."}
  ]
}

8. 최종 체크리스트

  • Provider를 생성했다.
  • Messaging API 채널을 생성했다.
  • LINE Official Account를 연결했다.
  • Channel ID와 Channel Secret을 확인했다.
  • Channel Access Token을 발급했다.
  • Webhook URL을 등록하고 Verify를 통과했다.
  • 토큰을 서버 환경변수에 안전하게 저장했다.
