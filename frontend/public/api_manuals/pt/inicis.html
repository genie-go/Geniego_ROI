KG Inicis
Manual de emissão de API Key

Um guia que resume o processo de emitir sua INIAPI Key e a chave de assinatura padrão web (SignKey) no admin de comerciante da KG Inicis e registrá-las junto com seu ID de loja (MID) no GenieGo Integration Hub.

Tempo estimadoCerca de 5–15 min
Conta necessáriaContrato de comerciante da KG Inicis · Conta de admin de comerciante
Chaves emitidasMID · Chave de API · Chave de assinatura (SignKey)
ResultadoCredenciais de integração de pagamento

1. Antes de começar

A KG Inicis só pode emitir chaves de autenticação após a conclusão do contrato de comerciante.

Obrigatório

Contrato de comerciante · Conta de admin de comerciante

Você deve ter um contrato de comerciante da KG Inicis assinado e poder fazer login com uma conta de admin de comerciante.

Chaves emitidas

INIAPI Key · Chave de assinatura padrão web

Crie uma INIAPI Key no admin de comerciante e emita ao mesmo tempo a chave de assinatura padrão web por meio da geração de signkey de pagamento web.

Nota: O registro exige três itens: ID de loja (MID), chave de assinatura (SignKey) e chave de API.

2. Etapas de emissão

Emita na ordem abaixo e registre no GenieGo.

1

Fazer login no admin de comerciante

Faça login no admin de comerciante da Inicis.

https://iniweb.inicis.com
2

Ir ao menu Informações adicionais

Vá para [Informações da loja] > [Informações do contrato] > [Informações adicionais].

Informações da loja → Informações do contrato → Informações adicionais
3

Criar uma INIAPI Key

Crie uma INIAPI Key.

4

Criar a chave de assinatura padrão web

Clique em [Geração de signkey de pagamento web] para emitir a chave de assinatura padrão web (SignKey).

Geração de signkey de pagamento web
Quando tanto a INIAPI Key quanto a chave de assinatura padrão web forem criadas, a emissão foi concluída corretamente.

3. Informações para registrar no GenieGo

CampoDescrição
MIDID de loja (MID)
sign_keyA chave de assinatura padrão web emitida (SignKey, manuseie com cuidado)
api_keyA INIAPI Key emitida
Insira e salve no GenieGo Integration Hub [Registrar] e as credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
O menu Informações adicionais não está visívelContrato de comerciante não concluído, ou conta sem permissõesFaça login com uma conta de admin de comerciante que tenha um contrato de comerciante concluído.
A autenticação do pagamento falhaChave de assinatura ou chave de API inserida incorretamenteVerifique novamente a chave de assinatura·INIAPI Key emitida em Informações da loja > Informações do contrato > Informações adicionais.

Lista de verificação final

  • Concluí o contrato de comerciante da KG Inicis e fiz login no admin de comerciante.
  • Criei uma INIAPI Key em Informações da loja > Informações do contrato > Informações adicionais.
  • Emiti a chave de assinatura padrão web por meio de [Geração de signkey de pagamento web].
  • Inseri MID·sign_key·api_key no GenieGo [Registrar] e salvei.
