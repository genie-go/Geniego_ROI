Manual de emissão da API Key

Um guia passo a passo para iniciantes sobre como emitir uma chave de API Billing (v2) (Bearer, pdl_live_…) nas Developer Tools do seu Paddle Dashboard e registrá-la no GenieGo Integration Hub.

Tempo estimadoCerca de 5–10 min
Conta necessáriaConta Paddle
Chaves emitidasChave de API Billing (pdl_live_…)
ResultadoCredenciais de API

1. Antes de começar

O Paddle é uma plataforma de pagamento do tipo Merchant of Record para SaaS e produtos digitais (Billing v2).

Obrigatório

Conta Paddle

Vender SaaS e produtos digitais exige uma conta Paddle (Billing v2).

Chaves emitidas

Chave de API Billing

Você gera uma chave de API Billing no formato Bearer (pdl_live_…). Não é o antigo método Classic vendor_id/auth_code.

Observação: A chave a ser emitida é uma chave de API Billing (v2) (Bearer, pdl_live_…). Não a confunda com o antigo método Classic vendor_id/auth_code.

2. Etapas de emissão

Emita na ordem abaixo e registre no GenieGo.

1

Faça login no Paddle Dashboard

Faça login no Dashboard com a conta Paddle que você preparou.

https://vendors.paddle.com
2

Vá ao menu API keys

Vá em [Developer Tools] > [Authentication] > [API keys].

Developer Tools → Authentication → API keys
3

Crie uma chave de API Billing

Crie uma chave de API Billing (Bearer, pdl_live_…). Não é o antigo método Classic vendor_id/auth_code.

Quando uma chave de API Billing que começa com pdl_live_… for criada, ela foi emitida corretamente.

3. Informações para registrar no GenieGo

CampoDescrição
api_keyA chave de API Billing (v2) emitida (Bearer, pdl_live_…, manuseie com cuidado)
Insira api_key no GenieGo Integration Hub [Registrar] e salve. As credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Falha na autenticaçãoUso do antigo método Classic vendor_id/auth_codeUse a chave de API Billing (v2) (pdl_live_…) emitida em Developer Tools > Authentication > API keys.
Formato de chave incorretoUso de uma chave de teste/antigaVerifique se é uma chave de API Billing no formato pdl_live_….

Lista de verificação final

  • Fiz login no Dashboard com uma conta Paddle.
  • Acessei Developer Tools > Authentication > API keys.
  • Criei uma chave de API Billing (v2) (Bearer, pdl_live_…).
  • Inseri api_key no GenieGo [Registrar] e salvei.
