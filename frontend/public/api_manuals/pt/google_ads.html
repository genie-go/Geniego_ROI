Google Ads
Manual de emissão de API Key

Um guia que o orienta na emissão de um developer token no Centro de API da sua conta de administrador (MCC) do Google Ads, na criação de um cliente OAuth no Google Cloud para obter um refresh token e no registro de tudo no GenieGo Integration Hub.

Tempo estimadoCerca de 20–40 min (análise do developer token à parte)
Conta necessáriaConta de administrador (MCC) do Google Ads · Conta Google
Chaves emitidasdeveloper token · client_id/secret · refresh_token · customer_id
ResultadoCredenciais da Google Ads API

1. Antes de começar

A integração do Google Ads requer tanto uma conta de administrador (MCC) quanto a configuração de OAuth no Google Cloud.

developer token

Centro de API da MCC

Solicite um developer token no Centro de API da sua conta de administrador (MCC) do Google Ads. O acesso básico (Basic) é aprovado após uma análise do Google.

OAuth

Cliente do Google Cloud

Em console.cloud.google.com, crie um cliente OAuth (client_id·client_secret) e autentique-se para obter um refresh token.

Observação: O acesso básico ao developer token é aprovado após uma análise do Google e, antes da aprovação, apenas contas de teste podem ser chamadas. Se você não tiver uma conta de administrador (MCC), crie uma primeiro em ads.google.com.

2. Etapas de emissão

Emita na ordem abaixo e registre no GenieGo.

1

Faça login na conta de administrador · Vá para o Centro de API

Faça login na sua conta de administrador do Google Ads e, em seguida, vá para [Ferramentas e configurações] > [Configuração] > [Centro de API] na parte superior.

Ferramentas e configurações → Configuração → Centro de API
2

Solicite um developer token

Solicite um [developer token] no Centro de API. O acesso básico (Basic) é aprovado após uma análise do Google e, antes da aprovação, apenas contas de teste podem ser chamadas.

3

Configure um projeto do Cloud · tela de consentimento OAuth

Crie um projeto e configure [APIs e serviços] > [Tela de consentimento OAuth].

console.cloud.google.com → APIs e serviços → Tela de consentimento OAuth
4

Crie um ID de cliente OAuth

Crie [Credenciais] > [Criar credenciais] > [ID de cliente OAuth] para obter um client_id e um client_secret (scope: adwords).

Credenciais → Criar credenciais → ID de cliente OAuth
5

Emita um refresh token

Obtenha um refresh token por meio do processo de autenticação OAuth (use o Google OAuth Playground ou sua própria autenticação).

6

Encontre o customer ID

Encontre o customer ID de 10 dígitos (customer_id) da conta de anúncios que você vai chamar.

Quando o developer token estiver aprovado e client_id·client_secret·refresh_token·customer_id estiverem todos prontos, a emissão está concluída.

3. Informações para registrar no GenieGo

CampoDescrição
developer_tokenO developer token emitido no Centro de API
client_idID de cliente OAuth
client_secretSegredo de cliente OAuth (manuseie com cuidado)
refresh_tokenO refresh token emitido via autenticação OAuth (digite-o no campo [Token de acesso])
customer_idO customer ID de 10 dígitos da conta de anúncios de destino
Digite o developer token, client_id, client_secret, refresh_token e customer_id no GenieGo [Registrar] e salve. As credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Apenas contas de teste podem ser chamadasAnálise de acesso básico do developer token não aprovadaAguarde até que a análise do Google do developer token que você solicitou no Centro de API seja aprovada e, em seguida, chame contas de produção.
Falha na autenticaçãoScope (adwords) ausente ou refresh_token incorretoAutentique-se novamente com o scope adwords incluído no cliente OAuth e verifique novamente o refresh_token.
Erro de formato do customer IDcustomer_id de 10 dígitos incorretoVerifique novamente o customer ID de 10 dígitos da conta de anúncios de destino e digite-o.

Lista de verificação final

  • Preparei uma conta de administrador (MCC) do Google Ads e fiz login.
  • Solicitei um developer token no Centro de API (o acesso básico passa por uma análise do Google).
  • Criei uma tela de consentimento OAuth e um ID/Segredo de cliente OAuth em console.cloud.google.com (scope: adwords).
  • Obtive um refresh token via autenticação OAuth.
  • Encontrei o customer_id de 10 dígitos da conta de destino.
  • Digitei o developer token·client_id/secret·refresh_token·customer_id no GenieGo [Registrar] e salvei.
