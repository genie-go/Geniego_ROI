Naver Search Ads
Manual de emissão da chave de API

Um guia passo a passo para iniciantes sobre como emitir as credenciais de API (licença de acesso, chave secreta) e o seu CUSTOMER_ID a partir da ferramenta de gestão do Naver Search Ad e registá-las no GenieGo Integration Hub.

Tempo estimadoCerca de 5–10 min
Conta necessáriaConta de anunciante do Naver Search Ad
Chaves emitidasLicença de acesso · Chave secreta · CUSTOMER_ID
ResultadoCredenciais da Search Ad API

1. Antes de começar

A API do Naver Search Ad só pode ser solicitada a partir de uma conta de anunciante.

Obrigatório

Conta de anunciante do Search Ad

Precisa de uma conta de anunciante que use a ferramenta de gestão do Naver Search Ad. Não é possível solicitar acesso à API com uma conta de membro comum do Naver.

Chaves emitidas

Licença de acesso · Chave secreta · CUSTOMER_ID

Confirme o CUSTOMER_ID, a licença de acesso e a chave secreta emitidos em conjunto após solicitar o acesso à API.

Nota: A chave secreta só pode ser visualizada no momento da emissão, por isso guarde-a com segurança logo após a emissão.

2. Passos para emitir a chave de API

Emita na ordem abaixo e registe no GenieGo.

1

Iniciar sessão no Search Ads

Inicie sessão na ferramenta de gestão do Naver Search Ad com a sua conta de anunciante.

searchad.naver.com
2

Ir para a gestão de uso da API

A partir do menu superior [Ferramentas], vá para [Gestão de Uso da API].

Ferramentas → Gestão de Uso da API
3

Solicitar o serviço de API

Clique no botão [Solicitar o serviço Naver Search Ad API] para solicitar.

Solicitar o serviço Naver Search Ad API
4

Verificar as informações emitidas

Confirme o CUSTOMER_ID, a licença de acesso e a chave secreta emitidos.

Assim que o CUSTOMER_ID, a licença de acesso e a chave secreta forem todos confirmados, foram emitidos corretamente.

3. Informações a registar no GenieGo

CampoDescrição
api_keyA licença de acesso emitida
api_secretA chave secreta emitida (manuseie com cuidado)
customer_idO CUSTOMER_ID emitido
Insira-as e guarde-as em [Registar] no GenieGo Integration Hub; as credenciais são encriptadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Sem menu [Gestão de Uso da API]Não é uma conta de anunciante ou permissões insuficientesInicie sessão com uma conta de anunciante que use a ferramenta de gestão do Search Ad.
Ocorre um erro de autenticaçãoLicença de acesso ou chave secreta inserida incorretamenteVerifique novamente a licença de acesso, a chave secreta e o CUSTOMER_ID no ecrã de emissão.

Lista de verificação final

  • Iniciei sessão com uma conta de anunciante do Naver Search Ad.
  • Concluí o pedido do serviço de API em [Ferramentas > Gestão de Uso da API].
  • Confirmei o CUSTOMER_ID, a licença de acesso e a chave secreta.
  • Inseri api_key·api_secret·customer_id em GenieGo [Registar] e guardei.
