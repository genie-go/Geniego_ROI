Manual de Emissão de API Key

Um guia passo a passo amigável para iniciantes sobre como solicitar as chaves da Open API do godomall no NHN Commerce Developer Center, receber sua partner key e API key, e registrá-las no GenieGo Integration Hub.

Tempo estimadoCerca de 10–20 min (incluindo espera de aprovação)
Conta necessáriaConta de operador de loja godomall (NHN Commerce)
Chaves emitidaspartner key · API key
ResultadoCredenciais da Open API

1. Antes de começar

A Open API do godomall é solicitada no NHN Commerce Developer Center após o registro e a aprovação do desenvolvedor.

Obrigatório

Conta de operador de loja

Você precisa de uma conta de operador que administre uma loja godomall (NHN Commerce). Use esta conta para concluir o registro de membro e desenvolvedor no Developer Center.

Chaves emitidas

partner key · API key

Assim que sua solicitação de emissão de chaves for aprovada, as chaves da Open API (partner key / API key) são emitidas.

Observação: O registro do desenvolvedor e a solicitação de emissão de chaves passam por um processo de aprovação, portanto a aprovação pode levar algum tempo.

2. Etapas de emissão da chave da Open API

Solicite e emita na ordem abaixo e registre no GenieGo.

1

Registrar-se e obter aprovação como desenvolvedor

Registre-se como membro e desenvolvedor no NHN Commerce Developer Center e obtenha a aprovação.

devcenter.nhn-commerce.com
2

Solicitar a emissão das chaves

Em [Open API] > [Solicitar emissão de chaves], preencha o formulário de solicitação e envie-o.

Open API → Solicitar emissão de chaves
3

Verificar as chaves

Verifique as chaves da Open API emitidas (partner key / API key).

Assim que a partner key e a API key são emitidas, a solicitação foi enviada e aprovada corretamente.

3. Informações para registrar no GenieGo

CampoDescrição
partner_keyA partner key emitida
api_keyA API key emitida (manuseie com cuidado)
Insira e salve os valores no GenieGo Integration Hub [Registrar]. As credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Não é possível solicitar a emissão de chavesRegistro do desenvolvedor / aprovação não concluídosConclua primeiro o registro de membro e desenvolvedor e a aprovação no Developer Center.
As chaves não são emitidas após a solicitaçãoAprovação pendenteAguarde até que o processo de aprovação seja concluído e verifique novamente.

Lista de verificação final

  • Registrou-se como membro e desenvolvedor no NHN Commerce Developer Center e obteve a aprovação.
  • Preencheu e enviou o formulário de solicitação em [Open API > Solicitar emissão de chaves].
  • Verificou a partner key e a API key emitidas.
  • Inseriu partner_key·api_key no GenieGo [Registrar] e salvou.
