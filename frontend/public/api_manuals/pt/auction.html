Auction
Manual de emissão de API Key

Um guia que explica como obter uma chave de API compartilhada para Auction·Gmarket a partir da ESM Trading API (etapi.gmarket.com) e registrá-la no GenieGo Integration Hub. Como Auction e Gmarket são operados em conjunto através do ESM Plus, uma única chave de API conecta ambos os canais.

Tempo estimadoVários dias incl. espera de aprovação (requer aprovação)
Conta necessáriaConta de vendedor Auction·Gmarket + ID mestre do ESM Plus
Chaves emitidasChave de API · ID do vendedor
ResultadoCredenciais da ESM Trading API

1. Antes de começar

Você deve estar registrado como vendedor tanto no Auction quanto no Gmarket e possuir um ID mestre do ESM Plus.

Obrigatório

Conta de vendedor + ID mestre do ESM

Você precisa de uma conta de vendedor para cada um, Auction e Gmarket, além de um ID mestre criado no ESM Plus (esmplus.com).

Chaves emitidas

Chave de API · ID do vendedor

Após a aprovação da ESM Trading API, confirme a chave de API emitida junto com o ID do vendedor.

Nota: A ESM Trading API requer aprovação, e as solicitações podem ser rejeitadas. Ao solicitar, você deve enviar três itens: seu ID de vendedor do Auction, ID de vendedor do Gmarket e ID mestre do ESM.

2. Etapas de emissão

Solicite e obtenha a chave na ordem abaixo e, em seguida, registre-a no GenieGo.

1

Registrar-se como vendedor e criar um ID mestre do ESM

Registre-se como vendedor tanto no Auction quanto no Gmarket, depois faça login no ESM Plus e crie um ID mestre.

esmplus.com
2

Solicitar o uso da ESM Trading API

Acesse a ESM Trading API e solicite o acesso à API. Como requer aprovação, a solicitação pode ser rejeitada.

https://etapi.gmarket.com
3

Enviar informações de identificação

Ao solicitar, envie três itens: seu ID de vendedor do Auction, ID de vendedor do Gmarket e ID mestre do ESM.

4

Emissão da chave

Uma vez aprovada, uma chave de API é emitida. Confirme a chave de API emitida e o ID do vendedor.

Quando a chave de API é emitida após a conclusão da aprovação, o processo foi bem-sucedido. Para dúvidas sobre a aprovação, entre em contato com etapihelp@gmail.com.

3. Informações para registrar no GenieGo

CampoDescrição
api_keyA chave de API emitida
seller_idID do vendedor
Insira os valores no GenieGo Integration Hub [Registrar] e salve. As credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Solicitação de uso da API rejeitadaA ESM Trading API requer aprovaçãoVerifique as informações enviadas e entre em contato com etapihelp@gmail.com.
Informações da solicitação divergentesID de vendedor do Auction·Gmarket ou ID mestre do ESM inseridos incorretamenteVerifique novamente os três itens: ID de vendedor do Auction, ID de vendedor do Gmarket e ID mestre do ESM.

Lista de verificação final

  • Registrado como vendedor tanto no Auction quanto no Gmarket.
  • Criado um ID mestre no ESM Plus.
  • Solicitado o acesso à ESM Trading API em etapi.gmarket.com e obtida a aprovação.
  • Confirmados o api_key e o seller_id emitidos.
  • Inserido api_key·seller_id no GenieGo [Registrar] e salvo.
