CJ Logistics International
Manual de emissão da API Key

Um manual que resume o processo de assinatura de um contrato CJ Logistics International Express e de registro da chave de API e do código de cliente (contrato) emitidos no GenieGo Integration Hub. Este canal recebe as credenciais por meio de um contrato comercial, em vez de emissão por console de autoatendimento.

Tempo estimadoVaria conforme o andamento do contrato
Conta necessáriaContrato CJ Logistics International Express
Chaves emitidasChave de API · Código de cliente (contrato)
ResultadoCredenciais da API do International Express

1. Antes de começar

O CJ Logistics International Express é um canal baseado em contrato, no qual o código de cliente (contrato) é emitido após um contrato comercial.

Obrigatório

Contrato do International Express

Para usar o CJ Logistics International Express, é preciso primeiro firmar um contrato com um representante comercial. Sem contrato, o código de cliente (contrato) não pode ser emitido.

Chaves emitidas

Chave de API · Código de cliente (contrato)

Após a conclusão do contrato, você recebe o código de cliente (contrato) e a chave de API.

Observação: Este canal não é emitido diretamente em um console de autoatendimento; em vez disso, as credenciais são recebidas por meio de um contrato comercial.

2. Processo de solicitação de emissão

Solicite e receba na ordem abaixo e, em seguida, registre no GenieGo.

1

Tramitar o contrato com o representante comercial

Tramite um contrato com o representante comercial do CJ Logistics International Express.

2

Firmar o contrato

Firme o contrato de uso do International Express.

3

Receber as credenciais

Após a conclusão do contrato, você recebe o código de cliente (contrato) e a chave de API.

4

Registrar no GenieGo

Insira o api_key e o cust_code emitidos no GenieGo [Registrar] e salve.

Se você recebeu tanto o código de cliente (contrato) quanto a chave de API do representante comercial, eles foram emitidos corretamente.

3. Informações para registrar no GenieGo

CampoDescrição
api_keyA chave de API do International Express emitida
cust_codeO código de cliente (contrato) emitido
Ao inseri-los e salvá-los no GenieGo Integration Hub [Registrar], as credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Código de cliente (contrato) não recebidoContrato do International Express não concluídoTramite um contrato com o representante comercial do CJ Logistics International Express e, em seguida, solicite a emissão das credenciais.
Erro de integração do International ExpressCódigo de cliente (contrato) (cust_code) inserido incorretamenteVerifique novamente o código de cliente (contrato) emitido e insira-o corretamente.

  • Recebi o código de cliente (contrato) e a chave de API após a conclusão do contrato.
  • Verifiquei api_key·cust_code.
