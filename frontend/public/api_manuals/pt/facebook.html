Facebook Live
Manual de Emissão de API

Um guia amigável para iniciantes sobre como criar um app no Meta for Developers e preparar o App ID, o App Secret e o token de acesso de página (Page Access Token) necessários para conectar a transmissão ao vivo de uma página do Facebook.

Conceito centralNão é uma API Key, mas um Access Token
Site de desenvolvedoresdevelopers.facebook.com
API utilizadaLive Video API (Graph)
Permissões necessáriasAdministrador da página + Page Token

1. O essencial a saber primeiro

No YouTube basta uma única API Key, mas o Facebook (Meta) funciona com uma estrutura de App ID · App Secret · Access Token.

Importante: Com a Meta não basta "copiar uma única API Key". Normalmente você precisa da seguinte ordem: criar um app → adicionar um produto → configurar permissões → emitir um Page Access Token → revisão do app.

2. Pré-requisitos

  • Uma conta Meta/Facebook
  • Acesso ao Meta for Developers
  • Uma página do Facebook na qual você tenha direitos de administrador
  • Uma URL de política de privacidade · uma URL de termos de serviço para o serviço real

3. Criar um app do Meta Developer

Toda integração com a API da Meta começa com a criação de um app.

1

Acessar o Meta for Developers

Faça login no Meta for Developers no seu navegador.

developers.facebook.com
2

Meus apps → Criar app

Clique em My Apps → Create App na parte superior.

My Apps → Create App
3

Escolher o tipo de app

A integração de plataformas/live commerce está mais próxima de um propósito empresarial.

Selecionar o tipo Business
4

Inserir nome do app·e-mail → criar

Insira o nome do app e um e-mail de contato e crie o app.

App Name / Contact Email
Após criar o app, verifique o App ID e o App Secret em Painel → Configurações do app → Configurações básicas. (Nunca exponha o App Secret no frontend ou em um repositório público.)

4. Fluxo de configuração da API do Facebook Live

1

Adicionar um produto

No painel do app, prepare as configurações relacionadas ao Facebook Login / Graph API.

Add Product → Facebook Login
2

Conectar uma página + token de página

Para transmitir ao vivo em uma página, você precisa de direitos de administrador da página e de um Page Access Token.

Page Access Token
3

Solicitar as permissões necessárias (App Review)

Solicite permissões destinadas a publicar conteúdo·gerenciar páginas·criar vídeos ao vivo (por exemplo, pages_read_engagement, pages_manage_posts).

Permissions → App Review
4

Emitir·testar o token com o Graph API Explorer

Emita um token que inclua as permissões e teste as chamadas à API.

Tools → Graph API Explorer
Exemplo de chamada de criação do Facebook Live
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
Em caso de criação bem-sucedida, são retornadas informações de transmissão ao vivo, como ID da transmissão·URL do stream·chave do stream. Consulte o ID da página (page_id) nas informações da página.

5. Registrar no GenieGo Integration Hub

1

Copiar o Page Access Token

Copie o Page Access Token que você emitiu (recomenda-se um token de longa duração).

access_token (Page token)
2

Confirmar o ID da página

Confirme o ID (page_id) da página do Facebook de destino.

page_id
3

Registrar no Integration Hub

No GenieGo Integration Hub → Facebook Live → [Registrar], insira os valores abaixo e salve.

access_token · page_id
As chaves registradas são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

6. Tipos de Access Token

Tipo de tokenUsoObservações
User Access TokenChamadas de API baseadas no login do usuárioOs dados acessíveis variam conforme o escopo das permissões concedidas.
Page Access TokenPublicação na página · criação de transmissões ao vivo da páginaSão necessários direitos de administrador da página e as permissões relacionadas.
Long-Lived TokenUm token de longa duração com expiração estendidaEm produção, é necessária lógica de expiração·renovação.

7. Lista de verificação

  • Criei um app do Meta Developer.
  • Confirmei o App ID e o App Secret.
  • Conectei uma página do Facebook e emiti um Page Access Token.
  • Organizei·solicitei as permissões necessárias (App Review).
  • Inseri access_token·page_id no GenieGo [Registrar] e salvei.
  • Armazenei o token com segurança nas variáveis de ambiente do servidor.

8. Problemas frequentes

ProblemaCausaSolução
Não há menu de API KeyA Meta não usa uma estrutura de API Key únicoProssiga com a estrutura App ID·App Secret·Access Token.
Erro de permissãoO token não tem as permissões necessáriasAdicione as permissões no Graph API Explorer e reemita o token.
Falha na criação da transmissão ao vivo da páginaPage Access Token/direitos de administrador insuficientesVerifique os direitos de administrador da página e as permissões do token.
