Manual de emissão da API Key

Um guia passo a passo, adequado para iniciantes, para emitir uma chave de Open API no Centro de Open API da 11Street (openapi.11st.co.kr) e registrá-la no GenieGo Integration Hub.

Tempo estimadoCerca de 5–15 min
Conta necessáriaConta de vendedor da 11Street
Chaves emitidasChave de Open API · Seller ID
ResultadoCredenciais de Open API

1. Antes de começar

As chaves de Open API da 11Street só podem ser emitidas a partir de uma conta de vendedor.

Obrigatório

Conta de vendedor da 11Street

Você precisa de uma conta de vendedor da 11Street. Faça login no Centro de Open API com seu Seller ID.

Chaves emitidas

Chave de Open API · Seller ID

Após concluir o registro do serviço e o registro do IP de acesso, verifique a chave de Open API emitida junto com seu Seller ID.

Nota: Você precisa registrar um IP de servidor autorizado a acessar a API antes de poder usar a chave.

2. Etapas de emissão

Emita na ordem abaixo e registre no GenieGo.

1

Faça login no Centro de Open API

Faça login no Centro de Open API da 11Street com seu Seller ID.

https://openapi.11st.co.kr/openapi/OpenApiFrontMain.tmall
2

Registre o serviço

No menu, clique em [Registro·Verificação do serviço], insira os dados do vendedor, aceite os termos e registre.

Registro·Verificação do serviço
3

Registre o IP de acesso

Em [Informações da Seller API] > [Editar], registre o IP do servidor autorizado a acessar a API.

Informações da Seller API → Editar
4

Copie a API Key

Copie a API Key emitida.

Assim que o registro do serviço e o registro do IP de acesso forem concluídos, a API Key é emitida.

3. Informações para registrar no GenieGo

CampoDescrição
api_keyA chave de Open API emitida
seller_idSeller ID
Insira e salve no GenieGo Integration Hub [Registrar]. As credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
A API Key não é emitidaRegistro do serviço ou registro do IP de acesso não concluídoConclua [Registro·Verificação do serviço] e o registro do IP de acesso em [Informações da Seller API].
Erro de autenticação nas chamadasIP de servidor autorizado não registradoVerifique novamente o IP do servidor autorizado em [Informações da Seller API] > [Editar].

5. Antes de registrar produtos (importante)

Mesmo com a chave de API válida, a 11Street recusa o registro se faltar qualquer um dos três itens abaixo.

  • Categoria de exibição (dispCtgrNo) — defina em GenieGo [Sincronização de catálogo] > [Mapeamento de categorias] ou no painel "Categoria e marca pendentes"
  • Marca — cadastre em GenieGo [Sincronização de catálogo] > [Gestão de marcas] e atribua ao produto (basta o nome, sem código)
  • Método de venda — o GenieGo envia preço fixo (01) automaticamente; não é preciso preencher
Mensagem de erroO que realmente significaComo resolver
-997 Nenhuma informação de API registradaNÃO é falha de autenticação. A URL de API chamada (caminho + método) não existe na 11Street.Não reemita a chave. Peça a verificação da URL chamada.
Código de marca ausenteO produto não tem marca definida.Cadastre a marca em [Sincronização de catálogo] > [Gestão de marcas] e atribua ao produto.
Método de venda indisponível via APIO código do método de venda não é permitido pela API.O GenieGo envia preço fixo (01) automaticamente. Nenhuma ação necessária.
Verifique isto antes de reemitir a chave. O erro -997 não é problema de chave, IP ou solicitação de API. Mesmo com os três válidos, uma URL errada retorna -997.

Lista de verificação final

  • Fiz login no Centro de Open API com uma conta de vendedor da 11Street.
  • Inseri os dados do vendedor e aceitei os termos em [Registro·Verificação do serviço].
  • Registrei o IP do servidor autorizado em [Informações da Seller API].
  • Confirmei a chave de Open API emitida e o Seller ID.
  • Inseri api_key·seller_id no GenieGo [Registrar] e salvei.
