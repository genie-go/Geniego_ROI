Manual de emissão de API

Este guia resume o processo de criação de um Provider e de um Messaging API Channel no LINE Developers Console e de configuração do Channel Access Token e da Webhook URL.

UsoMensagens · Chatbot

1. Conceito de emissão da API do LINE

A integração do LINE funciona criando um Provider e um Channel no LINE Developers Console e, em seguida, emitindo o Channel ID, o Channel Secret e o Channel Access Token para a Messaging API ou o LINE Login.

Distinção: O envio de mensagens usa um canal Messaging API, e a integração de login usa um canal LINE Login.

2. Pré-requisitos

  • Uma conta LINE ou uma conta empresarial
  • Permissão de acesso ao LINE Developers Console
  • Uma LINE Official Account
  • Nome do serviço e descrição
  • Webhook URL
  • URL da política de privacidade e URL dos termos

3. Criar um Provider

1

Acessar o LINE Developers

Faça login no LINE Developers Console.

developers.line.biz
2

Criar um Provider

Crie um Provider usando o nome da sua empresa ou do seu serviço.

Create Provider
3

Inserir as informações básicas

Insira o operador do serviço, os dados de contato e as informações dos termos.

Provider Settings

4. Criar um canal Messaging API

1

Criar um novo canal

Clique em Create a new channel dentro do Provider.

Create a new channel
2

Selecionar Messaging API

Escolha Messaging API como o tipo de canal.

Messaging API
3

Inserir as informações do canal

Insira o nome do canal, a descrição, a categoria, o e-mail, etc.

Channel Information
4

Vincular uma LINE Official Account

Vincule uma conta oficial a partir da qual enviar mensagens, ou crie uma nova.

LINE Official Account
5

Concluir a criação do canal

Após a criação, verifique o Channel ID e o Channel Secret.

Channel ID / Channel Secret

5. Emitir o Channel Access Token

1

Ir para a aba Messaging API

Na tela do canal que você criou, vá para a aba Messaging API.

Messaging API tab
2

Encontrar o item Channel access token

Localize a área Channel access token na parte inferior.

Channel access token
3

Clicar em Issue

Clique no botão Issue ou Reissue para emitir o token.

Issue
4

Armazenar com segurança no servidor

Armazene o token emitido em uma variável de ambiente do servidor ou no Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
O Channel Access Token é uma chave secreta com permissão para enviar mensagens. Ele não deve ser exposto em HTML, no frontend ou em repositórios públicos.

6. Configurar o Webhook

1

Inserir a Webhook URL

Insira a URL do servidor que receberá os eventos de mensagens dos usuários.

Webhook URL
2

Ativar Use webhook

Altere a opção Use webhook para Enabled.

Use webhook: Enabled
3

Teste Verify

Envie uma solicitação de teste do LINE para o seu servidor para confirmar a conexão.

Verify
4

Verificar o código de resposta

Confirme que o servidor retorna 200 OK.

HTTP 200 OK

7. Exemplo de envio de mensagem

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Olá. Este é um teste de mensagem do LINE."} ] }

8. Lista de verificação final

  • Criei um Provider.
  • Criei um canal Messaging API.
  • Vinculei uma LINE Official Account.
  • Verifiquei o Channel ID e o Channel Secret.
  • Emiti o Channel Access Token.
  • Registrei a Webhook URL e passei no Verify.
  • Armazenei o token com segurança em uma variável de ambiente do servidor.
