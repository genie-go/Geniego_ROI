🛍️ Guia para iniciantes

TikTok Shop
Manual de emissão de API Key

Um guia passo a passo amigável para iniciantes para criar um app (App Key·App Secret·Service ID) no TikTok Shop Partner Center, emitir um access_token por meio da autorização da loja e registrá-lo no GenieGo Integration Hub.

Tempo estimadoCerca de 10–15 min
Conta necessáriaConta de vendedor do TikTok Shop
Chaves emitidasApp Key · App Secret · Access Token
ResultadoCredenciais de API

1. Antes de começar

Um token só é emitido depois de criar um app no Partner Center e autorizar uma loja.

Obrigatório

Conta de vendedor do TikTok Shop

Você precisa de uma conta de vendedor do TikTok Shop e deve se registrar como desenvolvedor em partner.tiktokshop.com (Partner Center).

Chaves emitidas

App Key · App Secret · Access Token

Criar o app fornece App Key·App Secret·Service ID, e você obtém um access_token por meio da autorização da loja.

Observação: É necessária uma revisão do app ao mudar para produção.

2. Etapas de emissão

Emita na ordem abaixo e registre no GenieGo.

1

Registrar-se como desenvolvedor no Partner Center

Registre-se como desenvolvedor em partner.tiktokshop.com (Partner Center).

https://partner.tiktokshop.com
2

Criar um app

Crie um app via [App & Service] > [Create App] e verifique o App Key, App Secret e Service ID.

App & Service → Create App
3

Ativar permissões de API

Nas configurações do app, ative as permissões de API necessárias (Shop Authorized Information, Product, Order).

4

Autorização da loja → emitir access_token

Em [Get shop authorization], selecione a Região (Region) para autorizar a loja do vendedor, e um access_token é emitido.

Get shop authorization
5

Verificar as chaves

Verifique o App Key·App Secret e o access_token emitido. Se necessário, verifique também o Shop Cipher.

Quando o access_token é emitido após a autorização da loja, ele foi processado corretamente.

3. Informações para registrar no GenieGo

CampoDescrição
app_keyO App Key emitido
app_secretO App Secret emitido (manuseie com cuidado)
access_tokenO token de acesso emitido por meio da autorização da loja
shop_cipherShop Cipher (opcional)
Insira app_key·app_secret·access_token em [Registrar] do GenieGo Integration Hub e salve. As credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
O access_token não é emitidoAutorização da loja (Get shop authorization) não concluída, ou Região (Region) não selecionadaEm [Get shop authorization], selecione a Região e autorize a loja do vendedor.
Erro de permissãoPermissões de API necessárias não ativadasNas configurações do app, ative as permissões Shop Authorized Information·Product·Order.
Chamadas de produção negadasNão passou na revisão de mudança para produçãoConclua a revisão do app necessária para mudar para produção.

Lista de verificação final

  • Registrei-me como desenvolvedor em partner.tiktokshop.com.
  • Verifiquei o App Key·App Secret·Service ID via App & Service > Create App.
  • Ativei as permissões de API necessárias (Shop/Product/Order).
  • Emiti um access_token por meio da autorização da loja (Get shop authorization).
  • Inseri app_key·app_secret·access_token em GenieGo [Registrar] e salvei.
