Manual de emissão de API

O procedimento para registrar um aplicativo no console do desenvolvedor e preparar o Client ID, o Client Secret e o OAuth Token para conectar os serviços da família Yahoo! Japan.

Estrutura de autenticaçãoCentrada em OAuth 2.0
Conta necessáriaYahoo! JAPAN ID
Resultado principalClient ID / Secret
AtençãoO console difere por API

1. Conceito de emissão da API do Yahoo! Japan

Dependendo do tipo de serviço, as APIs da família Yahoo! Japan emitem o Client ID, o Client Secret e as credenciais OAuth a partir do Yahoo! JAPAN Developers ou do LY/Yahoo!広告 Developer Center.

Importante: No "Yahoo! Japan", o site de emissão e o processo de análise variam conforme a API que você pretende usar. Você precisa distinguir entre as APIs de Compras, Publicidade, Login e Pesquisa.

2. Pré-requisitos

  • Yahoo! JAPAN ID ou conta empresarial
  • Decidir o tipo de API a ser usada
  • Nome do aplicativo e descrição do serviço
  • Redirect URI
  • URL da política de privacidade
  • Domínio de produção ou domínio de teste

3. Ordem básica de registro do aplicativo

1

Acessar o site do desenvolvedor

Faça login no Yahoo! JAPAN Developers ou no Developer Center da API correspondente.

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

Criar um aplicativo

Clique no menu para adicionar um novo aplicativo.

Create Application / Add Application
3

Inserir as informações do aplicativo

Insira o nome do aplicativo, a descrição, as informações do responsável e a URL do site.

Application Information
4

Inserir o Redirect URI

Insira com exatidão a URL para a qual retornar após a autenticação OAuth.

Redirect URI
5

Selecionar permissões ou API

Selecione as permissões de API e os escopos (scope) que você pretende usar.

Scope / API Permission
6

Concluir o registro

Após o registro, verifique o Client ID e o Client Secret.

Client ID / Client Secret

5. Fluxo de emissão do token OAuth

1) Gerar a URL de autenticação do usuário com o Client ID 2) O usuário faz login com sua conta Yahoo! 3) Receber o authorization code no Redirect URI 4) Solicitar um Access Token com o authorization code + Client Secret 5) Chamar a API com o Access Token 6) Renovar com o Refresh Token quando necessário

6. Problemas frequentes

ProblemaCausaSolução
Erro de Redirect URIA URL registrada difere da URL de chamadaInsira de forma idêntica, incluindo protocolo, domínio e caminho
Não é possível ver o Client SecretOculto por segurançaVerifique a função View ou de reemissão na tela de gestão
Permissões insuficientesO Scope necessário não foi selecionadoAdicione o Scope nas configurações do aplicativo e autentique novamente
Falha na chamada da APIIncompatibilidade entre a API usada e o tipo de aplicativo emitidoConsulte a documentação para desenvolvedores de cada API

7. Lista de verificação final

  • Confirmei o tipo de API do Yahoo! Japan a ser usada.
  • Fiz login no console do desenvolvedor correto.
  • Registrei o aplicativo.
  • Verifiquei o Client ID e o Client Secret.
  • Registrei o Redirect URI com exatidão.
  • Selecionei o Scope necessário.
  • Concluí o teste de emissão do Access Token.
