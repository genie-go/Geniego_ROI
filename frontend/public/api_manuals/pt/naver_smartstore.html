Naver Smart Store
Manual de emissão de chave de API

Um guia passo a passo para registrar um aplicativo no Naver Commerce API Center (apicenter.commerce.naver.com) a fim de obter um Client ID e um Client Secret e, em seguida, registrá-los no GenieGo Integration Hub.

Tempo estimadoCerca de 10–15 min
Conta necessáriaConta da Smart Store com permissão de "Gerente integrado"
Chaves emitidasClient ID · Client Secret
ResultadoCredenciais da Commerce API

1. Antes de começar

A emissão só é possível com a permissão de Gerente integrado em uma conta de vendedor da Smart Store.

Obrigatório

Permissão de Gerente integrado

Você precisa ter a permissão de "Gerente integrado" em uma conta de vendedor da Smart Store. Apenas um Gerente integrado pode emitir as chaves.

Chaves emitidas

Client ID · Client Secret

Ao registrar um aplicativo, um ID de aplicativo (Client ID) e um Secret (Client Secret) são emitidos.

Observação: É obrigatório registrar o IP do servidor que chamará a API e renovar periodicamente a autenticação do aplicativo.

2. Etapas de emissão

Emita na ordem abaixo e registre no GenieGo.

1

Acessar o Commerce API Center

Faça login no Naver Commerce API Center com sua conta de Gerente integrado.

https://apicenter.commerce.naver.com
2

Criar uma conta

Clique em [Criar conta] no canto superior direito, informe o nome da conta da empresa desenvolvedora e um contato de resposta a incidentes, aceite os termos e cadastre-se.

Criar conta
3

Registrar um aplicativo

Em [Meus aplicativos da loja], registre um aplicativo; em seguida, um ID de aplicativo e um Secret são emitidos.

Meus aplicativos da loja
4

Registrar IP e renovar a autenticação

Registre o IP do servidor que chamará a API e renove periodicamente a autenticação do aplicativo conforme orientado.

Após o aplicativo ser registrado e o ID de aplicativo e o Secret serem emitidos, o processamento foi feito corretamente.

3. Informações para registrar no GenieGo

CampoDescrição
client_idO ID de aplicativo emitido (Client ID)
client_secretO Secret emitido (Client Secret, manuseie com cuidado)
Insira-os em [Registrar] do GenieGo Integration Hub e salve; as credenciais são criptografadas em repouso com AES-256-GCM e mascaradas na leitura.

4. Problemas comuns

ProblemaCausaSolução
Não é possível registrar um aplicativoA conta não tem permissão de Gerente integradoAcesse apicenter.commerce.naver.com com uma conta que tenha permissão de Gerente integrado.
Erro de autenticação na chamadaIP do chamador não registrado ou renovação de autenticação não realizadaRegistre o IP do servidor chamador e renove periodicamente a autenticação do aplicativo.

Lista de verificação final

  • Acessei o Commerce API Center com uma conta de Gerente integrado.
  • Informei os dados da empresa desenvolvedora e um contato de resposta a incidentes via [Criar conta] e aceitei os termos.
  • Registrei um aplicativo em [Meus aplicativos da loja] e obtive o ID/Secret.
  • Registrei o IP do servidor chamador e configurei a autenticação periódica.
  • Inseri client_id·client_secret no GenieGo [Registrar] e salvei.
