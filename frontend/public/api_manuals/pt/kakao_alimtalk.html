Manual de Emissão de API

Este manual reúne os passos práticos desde a verificação comercial do canal do Kakao Talk, o registro do perfil de remetente e a aprovação do modelo do Alimtalk até a confirmação das suas credenciais de API.

1. Conceito de emissão do Kakao Alimtalk

O Alimtalk é um serviço de mensagens empresariais que envia mensagens informativas como pedidos, reservas, envios e pagamentos pelo Kakao Talk. Não é uma estrutura que permite enviar diretamente apenas com uma chave de aplicativo de desenvolvedor Kakao comum; é necessária a verificação comercial do canal e a configuração de uma agência de envio/plataforma.

Importante: O Alimtalk é focado em mensagens informativas e não promocionais, e os modelos exigem análise prévia.

2. Pré-requisitos

  • Conta do Kakao Business
  • Canal do Kakao Talk
  • Documentos de verificação comercial, como um certificado de registro comercial
  • O texto do modelo do Alimtalk a ser enviado
  • Uma conta de agência de envio ou de serviço de mensagens na nuvem
  • Uma política de coleta de números de telefone dos destinatários e gestão de consentimento

3. Preparação do canal do Kakao Talk

1

Acessar o Kakao Business

Faça login no centro de administração do Kakao Business.

business.kakao.com
2

Criar um canal do Kakao Talk

Crie um canal com o nome da sua marca ou serviço.

Centro de Administração de Canais do Kakao Talk
3

Realizar a verificação comercial

Envie as informações da sua empresa e os documentos comprovativos para verificar o canal comercial.

Verificação comercial
4

Verificar o ID de pesquisa do canal

Verifique o ID do canal ou o ID de pesquisa necessário para as configurações de envio do Alimtalk.

Perfil do canal → ID de pesquisa
A verificação comercial deve ser concluída antes de você poder registrar um perfil de remetente do Alimtalk.

4. Registro do perfil de remetente / Sender Profile

1

Escolher uma plataforma de envio

Selecione um provedor de Alimtalk como NHN Cloud, Naver Cloud SENS, Infobank ou Bizgo.

Messaging Provider
2

Criar um projeto

Crie um projeto do Alimtalk na plataforma que você escolheu.

Project / Service
3

Conectar o canal do Kakao Talk

Registre as informações do seu canal do Kakao Talk verificado comercialmente.

Sender Profile
4

Análise do perfil de remetente

Realize a confirmação da propriedade do canal e a análise do perfil de remetente.

Profile Review
5

Confirmar as credenciais de API

Verifique o App Key, Secret Key, API Key, etc. fornecidos pela plataforma.

API Key / Secret

5. Registro e aprovação de modelos

1

Escrever o modelo

Escreva textos informativos como confirmações de pedido e avisos de envio.

Template Message
2

Definir variáveis

Defina como variáveis os valores que mudam dinamicamente, como nome, número do pedido e número de rastreamento.

#{variável}
3

Solicitar a análise

Salve o modelo e envie a solicitação de análise do Kakao.

Request Approval
4

Confirmar a aprovação

Confirme o código do modelo ou o ID aprovado.

Template Code
Se o texto antes da análise diferir do texto realmente enviado, o envio pode falhar ou podem ocorrer sanções.

6. Exemplo de fluxo de envio por API

1) Chame o Alimtalk API Endpoint a partir do seu servidor
2) Autentique-se com a API Key ou o Secret
3) Passe o número de telefone do destinatário, o código do modelo e os valores das variáveis
4) Receba o resultado do envio
5) Trate se deve enviar um SMS alternativo em caso de falha

7. Lista de verificação final

  • Criei um canal do Kakao Talk.
  • Concluí a verificação comercial.
  • Selecionei uma plataforma de envio do Alimtalk.
  • Registrei um Sender Profile.
  • Confirmei a API Key ou o Secret.
  • Registrei o modelo e obtive aprovação.
  • Preparei uma política de falha de envio e SMS alternativo.
