Amazon SP-API
Manual de emissão

Um guia sobre o fluxo de registro de desenvolvedor da SP-API, criação do aplicativo e configuração de OAuth·LWA para conectar seus dados de vendedor da Amazon com sistemas de pedidos, estoque, produtos e envio.

Estrutura de autenticaçãoLWA(OAuth) — 5 credenciais
Conta necessáriaSeller/Vendor Central (Professional)
Valores do GenieGorefresh_token·client_id·secret·marketplace_id·seller_id
DificuldadeIntermediário ou superior

1. O que é a Amazon SP-API?

A Amazon Selling Partner API é uma API para integrar programaticamente dados do vendedor, como pedidos, estoque, produtos, envio e liquidações.

Importante: A SP-API não é um esquema simples de API Key. Geralmente exige em conjunto o registro de Amazon Developer, o registro do aplicativo, Login with Amazon(LWA) e a autorização OAuth.
★A integração com o GenieGo requer 5 valores: refresh_token, client_id, client_secret, marketplace_id, seller_id. (A SP-API atual é chamada apenas com o token LWA, e a assinatura AWS IAM/SigV4 é tratada internamente pelo GenieGo, portanto você não a insere nos campos de registro.)

2. Pré-requisitos

  • Uma conta de vendedor Professional da Amazon Seller Central ou Vendor Central
  • Informações da empresa/contato a usar para o registro de desenvolvedor
  • Nome do aplicativo, descrição do aplicativo e uso pretendido
  • URL da política de privacidade
  • OAuth Redirect URL

3. Registro de desenvolvedor

Primeiro, registre seu perfil de desenvolvedor da SP-API no Seller Central.

1

Faça login no Seller Central

Faça login no Seller Central com sua conta de vendedor da Amazon.

Seller Central → Apps & Services
2

Vá ao menu Develop Apps

No menu Apps & Services, vá ao menu de desenvolvedor ou de desenvolvimento de aplicativos.

Apps & Services → Develop Apps
3

Preencha o Developer Profile

Insira as informações da empresa, as informações de contato, a finalidade de uso da API e o plano de proteção de dados.

Developer Profile
4

Enviar para revisão

Envie suas informações de registro de desenvolvedor para a revisão da Amazon.

Submit for review
Somente após a conclusão da aprovação você pode registrar um aplicativo SP-API de produção e usar suas permissões.

4. Registrar um aplicativo SP-API → Client ID / Client Secret

1

Crie um novo aplicativo

Crie um novo aplicativo em Develop Apps.

Add new app
2

Escolha o tipo de aplicativo

Se você conectar apenas sua própria conta, é um Private App; se você o fornecer a vendedores externos, é um Public App.

Private / Public App
3

Selecione as funções de API

Selecione as funções de que precisa, como Orders, Catalog e Inventory.

API Roles
4

Insira a OAuth Redirect URL

Insira a URL do servidor para a qual retornar após a conclusão da autorização.

Redirect URI
5

Salve e verifique as informações de Client

Após criar o aplicativo, verifique o client_id (LWA Client ID) e o client_secret (Client Secret).

client_id / client_secret

5. Autorização OAuth → Emitir um Refresh Token

1) Acesse a URL de autorização OAuth do vendedor 2) Receba o authorization code na Redirect URL 3) Passe o authorization code ao LWA Token Endpoint 4) Receba o refresh_token / access_token
O refresh_token é usado para chamadas de longo prazo após a autorização do vendedor. O GenieGo atualiza automaticamente o access_token usando este refresh_token para chamar a SP-API, portanto você só precisa registrar o refresh_token sem nenhuma configuração de assinatura AWS separada.

6. Verifique o Marketplace ID · Seller ID

Verifique os dois identificadores restantes necessários para o registro no GenieGo.

marketplace_id

ID do marketplace de destino

O Marketplace ID do país que você está conectando. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Necessário para o registro de produtos e a consulta de pedidos. Verifique-o nas informações da sua conta do Seller Central. Seller Central → Configurações → Informações da conta → Merchant Token

7. Registrar no GenieGo Integration Hub

1

Prepare os 5 valores

Prepare refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 credenciais
2

Integration Hub → Amazon SP-API → [Registrar]

Insira cada valor em seu campo e salve (os campos em branco mantêm os valores existentes).

refresh_token · client_id · client_secret · marketplace_id · seller_id
Os valores registrados são armazenados criptografados com AES-256-GCM e mascarados na leitura. A sincronização automática começa imediatamente após salvar.
O client_secret·refresh_token nunca deve ser exposto em código frontend, HTML ou repositórios públicos.

8. Lista de verificação final

  • Enviei o Developer Profile e o tive aprovado.
  • Criei o aplicativo SP-API e verifiquei o client_id·client_secret.
  • Emiti um refresh_token via autorização OAuth.
  • Verifiquei o marketplace_id a conectar.
  • Verifiquei o seller_id (Merchant Token) no Seller Central.
  • Inseri os 5 valores (refresh_token·client_id·client_secret·marketplace_id·seller_id) no GenieGo [Registrar] e os salvei.

9. Problemas comuns

ProblemaCausaSolução
Não há menu de API KeyA SP-API não é uma estrutura de API Key únicaProssiga com a estrutura LWA (client_id·client_secret·refresh_token).
O registro de produtos falhaseller_id (Merchant Token) não registradoRegistre o Merchant Token a partir das informações da sua conta do Seller Central.
Dados de país erradosmarketplace_id não correspondeRegistre o Marketplace ID correto para o país que você está conectando.
Erro de permissãoAPI Role do aplicativo insuficiente ou token expiradoVerifique o API Role e atualize o token com o refresh_token.
