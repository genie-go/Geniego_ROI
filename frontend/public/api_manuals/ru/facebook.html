Facebook Live
Руководство по выпуску API

Понятное для новичков руководство по созданию приложения на Meta for Developers и подготовке App ID, App Secret и токена доступа страницы (Page Access Token), необходимых для подключения прямой трансляции страницы Facebook.

Ключевая концепцияНе API Key, а Access Token
Сайт для разработчиковdevelopers.facebook.com
Используемый APILive Video API (Graph)
Необходимые разрешенияАдминистратор страницы + Page Token

1. Главное, что нужно знать сначала

В YouTube достаточно одного API Key, но Facebook (Meta) работает по структуре App ID · App Secret · Access Token.

Важно: В Meta дело не ограничивается «копированием одного API Key». Обычно требуется следующий порядок: создать приложение → добавить продукт → настроить разрешения → выпустить Page Access Token → проверка приложения.

2. Что потребуется

  • Учётная запись Meta/Facebook
  • Доступ к Meta for Developers
  • Страница Facebook, на которой у вас есть права администратора
  • URL политики конфиденциальности · URL условий обслуживания для реального сервиса

3. Создание приложения Meta Developer

Любая интеграция с API Meta начинается с создания приложения.

1

Войдите в Meta for Developers

Войдите в Meta for Developers в вашем браузере.

developers.facebook.com
2

Мои приложения → Создать приложение

Нажмите My Apps → Create App вверху.

My Apps → Create App
3

Выберите тип приложения

Интеграция платформ/прямая коммерция ближе к бизнес-цели.

Выберите тип Business
4

Введите имя приложения·email → создать

Введите имя приложения и контактный email, затем создайте приложение.

App Name / Contact Email
После создания приложения проверьте App ID и App Secret в разделе Панель → Настройки приложения → Основные настройки. (Никогда не раскрывайте App Secret во фронтенде/публичном репозитории.)

4. Процесс настройки Facebook Live API

1

Добавьте продукт

На панели приложения подготовьте настройки, связанные с Facebook Login / Graph API.

Add Product → Facebook Login
2

Подключите страницу + токен страницы

Чтобы транслировать прямой эфир на страницу, нужны права администратора страницы и Page Access Token.

Page Access Token
3

Запросите необходимые разрешения (App Review)

Запросите разрешения для публикации контента·управления страницами·создания прямых трансляций (например, pages_read_engagement, pages_manage_posts).

Permissions → App Review
4

Выпустите·протестируйте токен через Graph API Explorer

Выпустите токен с включёнными разрешениями и протестируйте вызовы API.

Tools → Graph API Explorer
Пример вызова создания Facebook Live
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
При успешном создании возвращается информация о прямой трансляции: ID трансляции·URL потока·ключ потока. ID страницы (page_id) смотрите в информации о странице.

5. Регистрация в GenieGo Integration Hub

1

Скопируйте Page Access Token

Скопируйте выпущенный вами Page Access Token (рекомендуется долгоживущий токен).

access_token (Page token)
2

Подтвердите ID страницы

Подтвердите ID (page_id) целевой страницы Facebook.

page_id
3

Зарегистрируйте в Integration Hub

В GenieGo Integration Hub → Facebook Live → [Зарегистрировать] введите указанные ниже значения и сохраните.

access_token · page_id
Зарегистрированные ключи шифруются при хранении с помощью AES-256-GCM и маскируются при чтении.

6. Типы Access Token

Тип токенаНазначениеПримечания
User Access TokenВызовы API на основе входа пользователяДоступные данные зависят от объёма предоставленных разрешений.
Page Access TokenПубликация на странице · создание прямых трансляций страницыТребуются права администратора страницы и соответствующие разрешения.
Long-Lived TokenДолгоживущий токен с увеличенным сроком действияПри эксплуатации нужна логика истечения·обновления.

7. Контрольный список

  • Создал приложение Meta Developer.
  • Подтвердил App ID и App Secret.
  • Подключил страницу Facebook и выпустил Page Access Token.
  • Систематизировал·запросил необходимые разрешения (App Review).
  • Ввёл access_token·page_id в GenieGo [Зарегистрировать] и сохранил.
  • Надёжно сохранил токен в переменных окружения сервера.

8. Часто возникающие проблемы

ПроблемаПричинаРешение
Нет меню API KeyMeta не использует структуру с единым API KeyДействуйте по структуре App ID·App Secret·Access Token.
Ошибка разрешенийУ токена нет необходимых разрешенийДобавьте разрешения в Graph API Explorer и заново выпустите токен.
Не удаётся создать прямую трансляцию страницыНедостаточно Page Access Token/прав администратораПроверьте права администратора страницы и разрешения токена.
