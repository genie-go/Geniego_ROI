Руководство по выпуску API Key

Руководство, описывающее, как выпустить API-ключ (X-API-Key) и Merchant Account в Customer Area после одобрения торгового договора и проверки Adyen и зарегистрировать их в GenieGo Integration Hub.

Примерное времяОколо 5–15 мин после одобрения
Необходимое правоТорговый договор Adyen · одобрение проверки
Выпускаемые ключиAPI-ключ (X-API-Key) · Merchant Account
РезультатУчётные данные API

1. Прежде чем начать

Adyen — это глобальный платёжный шлюз корпоративного уровня, поэтому перед выпуском учётных данных сначала необходимы торговый договор и одобрение проверки.

Требуется

Торговый договор · одобрение проверки

Adyen — глобальный PG корпоративного уровня, который требует торгового договора и одобрения проверки. Выпустить учётные данные можно только после одобрения.

Выпускаемые ключи

X-API-Key · Merchant Account

После одобрения создайте API-ключ (X-API-Key) в разделе [Developers] > [API credentials] в Customer Area и подтвердите имя своей Merchant Account.

Предварительное условие: Самостоятельный выпуск невозможен до одобрения торгового договора и проверки. Сначала необходимо завершить торговый договор и проверку с Adyen.

2. Процедура подачи заявки на выпуск

После одобрения проверки выпустите и зарегистрируйте учётные данные в указанном ниже порядке.

1

Торговый договор · одобрение проверки

Заключите торговый договор с Adyen и получите одобрение проверки.

Продажи/договоры Adyen
2

Войти в Customer Area

После одобрения войдите в Customer Area.

https://ca-live.adyen.com
3

Выпустить учётные данные API

В разделе [Developers] > [API credentials] создайте API-ключ (X-API-Key) и подтвердите имя своей Merchant Account.

Developers → API credentials
4

Подтвердить начальный номер расчётного пакета

Для первого сбора расчётов найдите самый последний номер settlement batch в Customer Area.

Как только API-ключ (X-API-Key) и Merchant Account подтверждены, ваши учётные данные готовы.

3. Информация для регистрации в GenieGo

ПолеОписание
api_keyВыпущенный API-ключ (X-API-Key, обращайтесь осторожно)
merchant_accountПодтверждённое имя вашей Merchant Account
batch_startНачальный номер расчётного пакета для первого сбора расчётов (самый последний номер settlement batch в Customer Area)
Введите api_key·merchant_account·batch_start в [Регистрация] в GenieGo Integration Hub и сохраните. Учётные данные шифруются при хранении с помощью AES-256-GCM и маскируются при чтении.
