Руководство по выпуску API Key

Удобное для новичков пошаговое руководство по выпуску Consumer Key и Consumer Secret в настройках REST API панели администратора WooCommerce и их регистрации в GenieGo Integration Hub.

Примерное времяОколо 5–10 мин
Требуемая учётная записьПрава администратора WordPress/WooCommerce
Выпускаемые ключиConsumer Key · Consumer Secret
РезультатУчётные данные REST API

1. Прежде чем начать

Ключи REST API можно выпустить только с правами администратора.

Требуется

Права администратора WooCommerce

Вам нужны права администратора WordPress/WooCommerce. С правами обычного пользователя выпустить ключи REST API невозможно.

Выпускаемые ключи

Consumer Key · Consumer Secret

При добавлении ключа в настройках REST API выпускаются Consumer Key и Consumer Secret.

2. Шаги выпуска

Выпустите в указанном ниже порядке и зарегистрируйте в GenieGo.

1

Перейдите к настройкам REST API

В WooCommerce перейдите в [Настройки] > [Дополнительно] > [REST API].

WooCommerce → Настройки → Дополнительно → REST API
2

Добавьте ключ

Нажмите [Добавить ключ], введите описание и выберите разрешения (Чтение/Запись), затем создайте.

Добавить ключ
3

Скопируйте ключи

Скопируйте выпущенные Consumer Key и Consumer Secret.

4

Проверьте информацию

Проверьте скопированные Consumer Key·Consumer Secret и собственный store URL.

Если на экране выпуска отображаются Consumer Key и Consumer Secret, ключи выпущены правильно.
Внимание: Consumer Secret часто отображается полностью только сразу после выпуска, поэтому обязательно безопасно скопируйте его в этот момент.

3. Информация для регистрации в GenieGo

ПолеОписание
site_urlstore URL
consumer_keyВыпущенный Consumer Key
consumer_secretВыпущенный Consumer Secret (обращайтесь осторожно)
Введите site_url·consumer_key·consumer_secret в GenieGo Integration Hub [Регистрация] и сохраните. Учётные данные шифруются при хранении с помощью AES-256-GCM и маскируются при чтении.

4. Частые проблемы

ПроблемаПричинаРешение
Нет меню REST APIНедостаточно прав администратораВойдите под учётной записью с правами администратора и проверьте Настройки > Дополнительно > REST API.
Операции записи не выполняютсяПри создании ключа выбрано разрешение «только чтение»Перевыпустите с разрешениями «Чтение/Запись» при [Добавить ключ].

Финальный чек-лист

  • Вошёл с правами администратора WordPress/WooCommerce.
  • Добавил ключ в Настройки > Дополнительно > REST API.
  • Выбрал разрешения (Чтение/Запись) и скопировал Consumer Key·Consumer Secret.
  • Ввёл site_url·consumer_key·consumer_secret в GenieGo [Регистрация] и сохранил.
