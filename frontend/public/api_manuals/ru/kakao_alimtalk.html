Это руководство объединяет практические шаги — от бизнес-верификации канала Kakao Talk, регистрации профиля отправителя и одобрения шаблона Alimtalk до подтверждения ваших учётных данных API.

НазначениеИнформационные сообщения

1. Концепция выпуска Kakao Alimtalk

Alimtalk — это сервис бизнес-сообщений, который отправляет информационные сообщения, такие как заказы, бронирования, доставка и платежи, через Kakao Talk. Это не структура, позволяющая отправлять напрямую только с помощью обычного ключа приложения разработчика Kakao; требуется бизнес-верификация канала и настройка агентства отправки/платформы.

Важно: Alimtalk ориентирован на информационные сообщения, а не на рекламные, и шаблоны требуют предварительной проверки.

2. Что потребуется

  • Аккаунт Kakao Business
  • Канал Kakao Talk
  • Документы для бизнес-верификации, например свидетельство о регистрации бизнеса
  • Текст шаблона Alimtalk для отправки
  • Аккаунт агентства отправки или облачного сервиса обмена сообщениями
  • Политика сбора номеров телефонов получателей и управления согласием

3. Подготовка канала Kakao Talk

1

Доступ к Kakao Business

Войдите в центр администрирования Kakao Business.

business.kakao.com
2

Создание канала Kakao Talk

Создайте канал под именем вашего бренда или сервиса.

Центр администрирования каналов Kakao Talk
3

Проведение бизнес-верификации

Отправьте сведения о компании и подтверждающие документы для верификации бизнес-канала.

Бизнес-верификация
4

Проверка поискового ID канала

Проверьте ID канала или поисковый ID, необходимый для настроек отправки Alimtalk.

Профиль канала → Поисковый ID
Бизнес-верификация должна быть завершена, прежде чем вы сможете зарегистрировать профиль отправителя Alimtalk.

4. Регистрация профиля отправителя / Sender Profile

1

Выбор платформы отправки

Выберите провайдера Alimtalk, например NHN Cloud, Naver Cloud SENS, Infobank или Bizgo.

Messaging Provider
2

Создание проекта

Создайте проект Alimtalk на выбранной вами платформе.

Project / Service
3

Подключение канала Kakao Talk

Зарегистрируйте сведения о вашем бизнес-верифицированном канале Kakao Talk.

Sender Profile
4

Проверка профиля отправителя

Выполните подтверждение владения каналом и проверку профиля отправителя.

Profile Review
5

Подтверждение учётных данных API

Проверьте App Key, Secret Key, API Key и другие данные, предоставленные платформой.

API Key / Secret

5. Регистрация и одобрение шаблонов

1

Написание шаблона

Напишите информационные тексты, например подтверждения заказа и уведомления о доставке.

Template Message
2

Настройка переменных

Задайте как переменные динамически меняющиеся значения, такие как имя, номер заказа и номер отправления.

#{переменная}
3

Запрос проверки

Сохраните шаблон и отправьте запрос на проверку Kakao.

Request Approval
4

Подтверждение одобрения

Подтвердите одобренный код шаблона или ID.

Template Code
Если текст до проверки отличается от фактически отправленного текста, отправка может завершиться неудачей или могут быть применены санкции.

6. Пример процесса отправки через API

1) Вызовите Alimtalk API Endpoint с вашего сервера
2) Выполните аутентификацию с помощью API Key или Secret
3) Передайте номер телефона получателя, код шаблона и значения переменных
4) Получите результат отправки
5) Обработайте, отправлять ли резервное SMS при неудаче

7. Финальный контрольный список

  • Создан канал Kakao Talk.
  • Завершена бизнес-верификация.
  • Выбрана платформа отправки Alimtalk.
  • Зарегистрирован Sender Profile.
  • Подтверждён API Key или Secret.
  • Шаблон зарегистрирован и одобрен.
  • Подготовлена политика неудачной отправки и резервных SMS.
