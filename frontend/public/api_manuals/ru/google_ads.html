Руководство по выпуску API Key

Руководство, которое проведёт вас через выпуск developer token в Центре API вашего управляющего (MCC) аккаунта Google Ads, создание OAuth-клиента в Google Cloud для получения refresh token и регистрацию всего в GenieGo Integration Hub.

Примерное времяОколо 20–40 мин (проверка developer token отдельно)
Требуемый аккаунтУправляющий (MCC) аккаунт Google Ads · аккаунт Google
Выпускаемые ключиdeveloper token · client_id/secret · refresh_token · customer_id
РезультатУчётные данные Google Ads API

1. Перед началом

Интеграция Google Ads требует как управляющего (MCC) аккаунта, так и настройки OAuth в Google Cloud.

developer token

Центр API MCC

Подайте заявку на developer token в Центре API вашего управляющего (MCC) аккаунта Google Ads. Базовый (Basic) доступ утверждается после проверки Google.

OAuth

Клиент Google Cloud

На console.cloud.google.com создайте OAuth-клиент (client_id·client_secret) и пройдите аутентификацию, чтобы получить refresh token.

Примечание: Базовый доступ к developer token утверждается после проверки Google, и до утверждения можно вызывать только тестовые аккаунты. Если у вас нет управляющего (MCC) аккаунта, сначала создайте его на ads.google.com.

2. Шаги выпуска

Выпустите в указанном ниже порядке и зарегистрируйте в GenieGo.

1

Войдите в управляющий аккаунт · Перейдите в Центр API

Войдите в свой управляющий аккаунт Google Ads, затем перейдите в [Инструменты и настройки] > [Настройка] > [Центр API] вверху.

Инструменты и настройки → Настройка → Центр API
2

Подайте заявку на developer token

Подайте заявку на [developer token] в Центре API. Базовый (Basic) доступ утверждается после проверки Google, и до утверждения можно вызывать только тестовые аккаунты.

3

Настройте проект Cloud · экран согласия OAuth

Создайте проект и настройте [API и сервисы] > [Экран согласия OAuth].

console.cloud.google.com → API и сервисы → Экран согласия OAuth
4

Создайте идентификатор клиента OAuth

Создайте [Учётные данные] > [Создать учётные данные] > [Идентификатор клиента OAuth], чтобы получить client_id и client_secret (scope: adwords).

Учётные данные → Создать учётные данные → Идентификатор клиента OAuth
5

Выпустите refresh token

Получите refresh token через процесс аутентификации OAuth (используйте Google OAuth Playground или собственную аутентификацию).

6

Найдите customer ID

Найдите 10-значный customer ID (customer_id) рекламного аккаунта, который вы будете вызывать.

Как только developer token будет утверждён, а client_id·client_secret·refresh_token·customer_id будут готовы, выпуск завершён.

3. Информация для регистрации в GenieGo

ПолеОписание
developer_tokendeveloper token, выпущенный в Центре API
client_idИдентификатор клиента OAuth
client_secretСекрет клиента OAuth (обращайтесь осторожно)
refresh_tokenrefresh token, выпущенный через аутентификацию OAuth (введите в поле [Токен доступа])
customer_id10-значный customer ID целевого рекламного аккаунта
Введите developer token, client_id, client_secret, refresh_token и customer_id в GenieGo [Регистрация] и сохраните. Учётные данные шифруются при хранении с помощью AES-256-GCM и маскируются при чтении.

4. Частые проблемы

ПроблемаПричинаРешение
Можно вызывать только тестовые аккаунтыПроверка базового доступа developer token не утвержденаПодождите, пока проверка Google для developer token, на который вы подали заявку в Центре API, не будет утверждена, затем вызывайте рабочие аккаунты.
Сбой аутентификацииОтсутствует scope (adwords) или неверный refresh_tokenПройдите аутентификацию заново с включённым scope adwords в OAuth-клиенте и перепроверьте refresh_token.
Ошибка формата customer IDНеверный 10-значный customer_idПерепроверьте 10-значный customer ID целевого рекламного аккаунта и введите его.
