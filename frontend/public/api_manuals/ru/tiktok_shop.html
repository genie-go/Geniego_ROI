Руководство по выпуску API Key

Пошаговое руководство для начинающих по созданию приложения (App Key·App Secret·Service ID) в TikTok Shop Partner Center, выпуску access_token через авторизацию магазина и его регистрации в GenieGo Integration Hub.

Примерное времяОколо 10–15 мин
Требуется аккаунтАккаунт продавца TikTok Shop
Выпускаемые ключиApp Key · App Secret · Access Token
РезультатУчётные данные API

1. Перед началом

Токен выпускается только после создания приложения в Partner Center и авторизации магазина.

Требуется

Аккаунт продавца TikTok Shop

Вам нужен аккаунт продавца TikTok Shop, и вы должны зарегистрироваться как разработчик на partner.tiktokshop.com (Partner Center).

Выпускаемые ключи

App Key · App Secret · Access Token

При создании приложения вы получаете App Key·App Secret·Service ID, а access_token получаете через авторизацию магазина.

Примечание: При переходе в рабочую среду требуется проверка приложения.

2. Шаги выпуска

Выпустите в указанном ниже порядке и зарегистрируйте в GenieGo.

1

Регистрация как разработчик в Partner Center

Зарегистрируйтесь как разработчик на partner.tiktokshop.com (Partner Center).

https://partner.tiktokshop.com
2

Создание приложения

Создайте приложение через [App & Service] > [Create App] и проверьте App Key, App Secret, Service ID.

App & Service → Create App
3

Включение разрешений API

В настройках приложения включите необходимые разрешения API (Shop Authorized Information, Product, Order).

4

Авторизация магазина → выпуск access_token

В [Get shop authorization] выберите регион (Region), чтобы авторизовать магазин продавца, и будет выпущен access_token.

Get shop authorization
5

Проверка ключей

Проверьте App Key·App Secret и выпущенный access_token. При необходимости проверьте также Shop Cipher.

Как только access_token выпущен после авторизации магазина, значит он обработан правильно.

3. Информация для регистрации в GenieGo

ПолеОписание
app_keyВыпущенный App Key
app_secretВыпущенный App Secret (обращайтесь осторожно)
access_tokenТокен доступа, выпущенный через авторизацию магазина
shop_cipherShop Cipher (необязательно)
Введите app_key·app_secret·access_token в [Регистрация] в GenieGo Integration Hub и сохраните. Учётные данные шифруются при хранении с помощью AES-256-GCM и маскируются при чтении.

4. Частые проблемы

ПроблемаПричинаРешение
access_token не выпускаетсяАвторизация магазина (Get shop authorization) не завершена или не выбран регион (Region)В [Get shop authorization] выберите регион и авторизуйте магазин продавца.
Ошибка разрешенийНеобходимые разрешения API не включеныВ настройках приложения включите разрешения Shop Authorized Information·Product·Order.
Вызовы в рабочей среде отклоненыНе пройдена проверка перехода в рабочую средуЗавершите проверку приложения, необходимую для перехода в рабочую среду.
