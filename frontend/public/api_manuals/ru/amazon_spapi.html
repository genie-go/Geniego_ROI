1. Что такое Amazon SP-API?

Amazon Selling Partner API — это API для программной интеграции данных продавца, таких как заказы, склад, товары, доставка и расчёты.

Важно: SP-API — это не простая схема API Key. Обычно она одновременно требует регистрации Amazon Developer, регистрации приложения, Login with Amazon(LWA) и авторизации OAuth.
★Для интеграции с GenieGo требуется 5 значений: refresh_token, client_id, client_secret, marketplace_id, seller_id. (Текущая SP-API вызывается только с токеном LWA, а подпись AWS IAM/SigV4 обрабатывается GenieGo внутренне, поэтому вы не вводите её в поля регистрации.)

2. Что нужно подготовить

  • Аккаунт продавца Professional в Amazon Seller Central или Vendor Central
  • Сведения о компании/контактном лице для регистрации разработчика
  • Название приложения, описание приложения и цель использования
  • URL политики конфиденциальности
  • OAuth Redirect URL

3. Регистрация разработчика

Сначала зарегистрируйте профиль разработчика SP-API в Seller Central.

1

Войдите в Seller Central

Войдите в Seller Central с помощью своего аккаунта продавца Amazon.

Seller Central → Apps & Services
2

Перейдите в меню Develop Apps

В меню Apps & Services перейдите в меню разработчика или разработки приложений.

Apps & Services → Develop Apps
3

Заполните Developer Profile

Введите сведения о компании, контактную информацию, цель использования API и план защиты данных.

Developer Profile
4

Отправить на проверку

Отправьте сведения о регистрации разработчика на проверку Amazon.

Submit for review
Только после завершения утверждения вы сможете зарегистрировать рабочее приложение SP-API и использовать его разрешения.

4. Регистрация приложения SP-API → Client ID / Client Secret

1

Создайте новое приложение

Создайте новое приложение в Develop Apps.

Add new app
2

Выберите тип приложения

Если вы подключаете только свой аккаунт — это Private App; если предоставляете его внешним продавцам — это Public App.

Private / Public App
3

Выберите роли API

Выберите нужные роли, такие как Orders, Catalog и Inventory.

API Roles
4

Введите OAuth Redirect URL

Введите URL сервера, на который нужно вернуться после завершения авторизации.

Redirect URI
5

Сохраните и проверьте сведения Client

После создания приложения проверьте client_id (LWA Client ID) и client_secret (Client Secret).

client_id / client_secret

5. Авторизация OAuth → Выпуск Refresh Token

1) Перейдите по URL авторизации OAuth продавца 2) Получите authorization code на Redirect URL 3) Передайте authorization code в LWA Token Endpoint 4) Получите refresh_token / access_token
refresh_token используется для долгосрочных вызовов после авторизации продавца. GenieGo автоматически обновляет access_token с помощью этого refresh_token для вызова SP-API, поэтому вам нужно лишь зарегистрировать refresh_token без отдельной настройки подписи AWS.

6. Проверка Marketplace ID · Seller ID

Проверьте два оставшихся идентификатора, необходимых для регистрации в GenieGo.

marketplace_id

Целевой идентификатор маркетплейса

Marketplace ID страны, которую вы подключаете. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Необходим для регистрации товаров и поиска заказов. Проверьте его в сведениях своего аккаунта Seller Central. Seller Central → Настройки → Сведения об аккаунте → Merchant Token

7. Регистрация в GenieGo Integration Hub

1

Подготовьте 5 значений

Подготовьте refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 учётных данных
2

Integration Hub → Amazon SP-API → [Зарегистрировать]

Введите каждое значение в его поле и сохраните (пустые поля сохраняют существующие значения).

refresh_token · client_id · client_secret · marketplace_id · seller_id
Зарегистрированные значения хранятся в зашифрованном виде с AES-256-GCM и маскируются при чтении. Автоматическая синхронизация начинается сразу после сохранения.
client_secret·refresh_token никогда не должны раскрываться во фронтенд-коде, HTML или публичных репозиториях.

8. Итоговый контрольный список

  • Отправил Developer Profile и получил его утверждение.
  • Создал приложение SP-API и проверил client_id·client_secret.
  • Выпустил refresh_token через авторизацию OAuth.
  • Проверил marketplace_id для подключения.
  • Проверил seller_id (Merchant Token) в Seller Central.
  • Ввёл 5 значений (refresh_token·client_id·client_secret·marketplace_id·seller_id) в GenieGo [Зарегистрировать] и сохранил их.

9. Часто возникающие проблемы

ПроблемаПричинаРешение
Нет меню API KeySP-API не является единой структурой API KeyДействуйте по структуре LWA (client_id·client_secret·refresh_token).
Не удаётся зарегистрировать товарseller_id (Merchant Token) не зарегистрированЗарегистрируйте Merchant Token из сведений своего аккаунта Seller Central.
Неверные данные странынесоответствие marketplace_idЗарегистрируйте правильный Marketplace ID для страны, которую вы подключаете.
Ошибка разрешенийНедостаточно API Role приложения или истёк срок действия токенаПроверьте API Role и обновите токен с помощью refresh_token.
