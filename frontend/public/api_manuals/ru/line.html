Руководство по выпуску API

В этом руководстве описан процесс создания Provider и Messaging API Channel в LINE Developers Console, а также настройки Channel Access Token и Webhook URL.

1. Концепция выпуска LINE API

Интеграция LINE работает за счёт создания Provider и Channel в LINE Developers Console и последующего выпуска Channel ID, Channel Secret и Channel Access Token для Messaging API или LINE Login.

Различие: Для отправки сообщений используется канал Messaging API, а для интеграции входа — канал LINE Login.

2. Что потребуется

  • Учётная запись LINE или бизнес-аккаунт
  • Право доступа к LINE Developers Console
  • LINE Official Account
  • Название сервиса и описание
  • Webhook URL
  • URL политики конфиденциальности и URL условий

3. Создание Provider

1

Вход в LINE Developers

Войдите в LINE Developers Console.

developers.line.biz
2

Создание Provider

Создайте Provider, используя название компании или сервиса.

Create Provider
3

Ввод основной информации

Введите оператора сервиса, контактные данные и информацию об условиях.

Provider Settings

4. Создание канала Messaging API

1

Создание нового канала

Внутри Provider нажмите Create a new channel.

Create a new channel
2

Выбор Messaging API

Выберите Messaging API в качестве типа канала.

Messaging API
3

Ввод информации о канале

Введите имя канала, описание, категорию, эл. почту и т. д.

Channel Information
4

Привязка LINE Official Account

Привяжите официальный аккаунт, с которого будут отправляться сообщения, или создайте новый.

LINE Official Account
5

Завершение создания канала

После создания проверьте Channel ID и Channel Secret.

Channel ID / Channel Secret

5. Выпуск Channel Access Token

1

Переход на вкладку Messaging API

На экране созданного канала перейдите на вкладку Messaging API.

Messaging API tab
2

Поиск элемента Channel access token

Проверьте область Channel access token внизу.

Channel access token
3

Нажатие Issue

Нажмите кнопку Issue или Reissue, чтобы выпустить токен.

Issue
4

Безопасное хранение на сервере

Сохраните выпущенный токен в переменной окружения сервера или в Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token — это секретный ключ с правом отправки сообщений. Его нельзя раскрывать в HTML, во фронтенде или в публичных репозиториях.

6. Настройка Webhook

1

Ввод Webhook URL

Введите URL сервера, который будет получать события сообщений пользователей.

Webhook URL
2

Включение Use webhook

Измените параметр Use webhook на Enabled.

Use webhook: Enabled
3

Тест Verify

Отправьте тестовый запрос из LINE на ваш сервер, чтобы подтвердить соединение.

Verify
4

Проверка кода ответа

Убедитесь, что сервер возвращает 200 OK.

HTTP 200 OK

7. Пример отправки сообщения

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Здравствуйте. Это тест сообщения LINE."} ] }

8. Итоговый контрольный список

  • Создан Provider.
  • Создан канал Messaging API.
  • Привязан LINE Official Account.
  • Проверены Channel ID и Channel Secret.
  • Выпущен Channel Access Token.
  • Зарегистрирован Webhook URL и пройден Verify.
  • Токен безопасно сохранён в переменной окружения сервера.
