Руководство по выпуску API

Процедура регистрации приложения в консоли разработчика и подготовки Client ID, Client Secret и OAuth Token для подключения сервисов семейства Yahoo! Japan.

Структура аутентификацииНа основе OAuth 2.0
Требуемая учётная записьYahoo! JAPAN ID
Основной результатClient ID / Secret
ВниманиеКонсоль различается для каждого API

1. Концепция выпуска API Yahoo! Japan

В зависимости от типа сервиса API семейства Yahoo! Japan выпускают Client ID, Client Secret и учётные данные OAuth через Yahoo! JAPAN Developers или LY/Yahoo!広告 Developer Center.

Важно: Для «Yahoo! Japan» сайт выпуска и процедура проверки различаются в зависимости от того, какой API вы собираетесь использовать. Необходимо различать API Покупок, Рекламы, Входа и Поиска.

2. Необходимое

  • Yahoo! JAPAN ID или бизнес-аккаунт
  • Определить тип используемого API
  • Название приложения и описание сервиса
  • Redirect URI
  • URL политики конфиденциальности
  • Рабочий домен или тестовый домен

3. Базовый порядок регистрации приложения

1

Перейти на сайт разработчика

Войдите в Yahoo! JAPAN Developers или в Developer Center соответствующего API.

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

Создать приложение

Нажмите меню добавления нового приложения.

Create Application / Add Application
3

Ввести информацию о приложении

Введите название приложения, описание, сведения об ответственном лице и URL веб-сайта.

Application Information
4

Ввести Redirect URI

Точно введите URL, на который произойдёт возврат после аутентификации OAuth.

Redirect URI
5

Выбрать разрешения или API

Выберите разрешения API и области (scope), которые вы собираетесь использовать.

Scope / API Permission
6

Завершить регистрацию

После регистрации проверьте Client ID и Client Secret.

Client ID / Client Secret

5. Процесс выпуска токена OAuth

1) Сформировать URL аутентификации пользователя с помощью Client ID 2) Пользователь входит со своей учётной записью Yahoo! 3) Получить authorization code на Redirect URI 4) Запросить Access Token с помощью authorization code + Client Secret 5) Вызвать API с помощью Access Token 6) При необходимости обновить с помощью Refresh Token
