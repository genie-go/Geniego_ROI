Anleitung zur API-Key-Ausstellung

Eine einsteigerfreundliche Schritt-für-Schritt-Anleitung zum Ausstellen eines Consumer Key und Consumer Secret in den REST-API-Einstellungen Ihres WooCommerce-Admins und zum Registrieren im GenieGo Integration Hub.

Geschätzte ZeitEtwa 5–10 Min.
Benötigtes KontoWordPress/WooCommerce-Administratorrechte
Ausgestellte SchlüsselConsumer Key · Consumer Secret
ErgebnisREST-API-Anmeldedaten

1. Bevor Sie beginnen

REST-API-Schlüssel können nur mit Administratorrechten ausgestellt werden.

Erforderlich

WooCommerce-Administratorrechte

Sie benötigen WordPress/WooCommerce-Administratorrechte. Mit normalen Benutzerrechten können keine REST-API-Schlüssel ausgestellt werden.

Ausgestellte Schlüssel

Consumer Key · Consumer Secret

Wenn Sie in den REST-API-Einstellungen einen Schlüssel hinzufügen, werden ein Consumer Key und ein Consumer Secret ausgestellt.

2. Ausstellungsschritte

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Zu den REST-API-Einstellungen wechseln

Gehen Sie in WooCommerce zu [Einstellungen] > [Erweitert] > [REST API].

WooCommerce → Einstellungen → Erweitert → REST API
2

Schlüssel hinzufügen

Klicken Sie auf [Schlüssel hinzufügen], geben Sie eine Beschreibung ein, wählen Sie die Berechtigungen (Lesen/Schreiben) und generieren Sie.

Schlüssel hinzufügen
3

Schlüssel kopieren

Kopieren Sie den ausgestellten Consumer Key und Consumer Secret.

4

Informationen prüfen

Prüfen Sie den kopierten Consumer Key, Consumer Secret und Ihre eigene store URL.

Wenn der Consumer Key und das Consumer Secret auf dem Ausstellungsbildschirm angezeigt werden, wurden die Schlüssel korrekt ausgestellt.
Achtung: Das Consumer Secret wird oft nur unmittelbar nach der Ausstellung vollständig angezeigt, kopieren Sie es daher in diesem Moment sicher.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
site_urlstore URL
consumer_keyDer ausgestellte Consumer Key
consumer_secretDas ausgestellte Consumer Secret (vertraulich behandeln)
Geben Sie site_url·consumer_key·consumer_secret im GenieGo Integration Hub [Registrieren] ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Kein REST-API-MenüUnzureichende AdministratorrechteMelden Sie sich mit einem Konto mit Administratorrechten an und prüfen Sie Einstellungen > Erweitert > REST API.
Schreibvorgänge schlagen fehlBeim Erstellen des Schlüssels wurde die schreibgeschützte Berechtigung gewähltStellen Sie beim [Schlüssel hinzufügen] mit Lese-/Schreibberechtigungen erneut aus.

Abschließende Checkliste

  • Mit WordPress/WooCommerce-Administratorrechten angemeldet.
  • Unter Einstellungen > Erweitert > REST API einen Schlüssel hinzugefügt.
  • Die Berechtigungen (Lesen/Schreiben) ausgewählt und den Consumer Key·Consumer Secret kopiert.
  • site_url·consumer_key·consumer_secret in GenieGo [Registrieren] eingegeben und gespeichert.
