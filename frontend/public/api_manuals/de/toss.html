Toss Payments
Anleitung zur API-Key-Ausstellung

Eine Anleitung, die beschreibt, wie Sie Ihren Client Key und Secret Key im Developer Center der Toss-Payments-Händlerverwaltung nachschlagen und im GenieGo Integration Hub registrieren.

Geschätzte ZeitEtwa 5–10 Min.
Benötigtes KontoToss-Payments-Händlervertrag · Händlerverwaltungskonto
Ausgestellte SchlüsselClient Key · Secret Key
ErgebnisAnmeldedaten für die Zahlungsintegration

1. Bevor Sie beginnen

Bei Toss Payments können Sie Ihre Schlüssel nach Abschluss eines Händlervertrags in der Händlerverwaltung nachschlagen.

Erforderlich

Händlervertrag · Händlerverwaltungskonto

Sie müssen einen Toss-Payments-Händlervertrag abgeschlossen haben und sich in der Händlerverwaltung anmelden können.

Ausgestellte Schlüssel

Client Key · Secret Key

Schlagen Sie Client Key und Secret Key im Developer Center als Satz nach und verwenden Sie beide gemeinsam.

Hinweis: Das Developer Center unterscheidet zwischen Test und Live, stellen Sie daher sicher, dass Sie für die tatsächliche Integration die Live-Schlüssel verwenden.

2. Ausstellungsschritte

Schlagen Sie sie in der unten angegebenen Reihenfolge nach und registrieren Sie sie in GenieGo.

1

Bei der Händlerverwaltung anmelden

Melden Sie sich bei der Toss-Payments-Händlerverwaltung an.

https://dashboard.tosspayments.com
2

Zum Developer Center wechseln

Wechseln Sie unten links zu [Developer Center].

Developer Center
3

Schlüssel nachschlagen

Wählen Sie Test/Live und schlagen Sie Client Key und Secret Key nach. Die beiden Schlüssel werden als Satz gemeinsam verwendet.

Sobald Sie sowohl den Client Key als auch den Secret Key bestätigt haben, sind Sie bereit zur Registrierung.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
client_keyDer im Developer Center gefundene Client Key
secret_keyDer im Developer Center gefundene Secret Key (vertraulich behandeln)
Geben Sie sie im GenieGo Integration Hub [Registrieren] ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Authentifizierungsfehler tritt aufVerwechslung von Test- und Live-SchlüsselÜberprüfen Sie im Developer Center die Unterscheidung zwischen Test/Live und verwenden Sie den zur Umgebung passenden Schlüssel.
Kein Developer-Center-MenüHändlervertrag nicht abgeschlossenMelden Sie sich mit einem Händlerverwaltungskonto an, dessen Händlervertrag abgeschlossen ist.

Abschließende Checkliste

  • Einen Toss-Payments-Händlervertrag abgeschlossen und bei der Händlerverwaltung angemeldet.
  • Unten links zu [Developer Center] gewechselt.
  • Test/Live unterschieden und Client Key sowie Secret Key nachgeschlagen.
  • client_key·secret_key in GenieGo [Registrieren] eingegeben und gespeichert.
