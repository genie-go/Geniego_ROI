Anleitung zur API-Key-Ausstellung

Eine einsteigerfreundliche Schritt-für-Schritt-Anleitung zum Ausstellen eines Billing-(v2)-API-Keys (Bearer, pdl_live_…) über die Developer Tools in Ihrem Paddle Dashboard und zur Registrierung im GenieGo Integration Hub.

Geschätzte ZeitEtwa 5–10 Min.
Benötigtes KontoPaddle-Konto
Ausgestellte SchlüsselBilling-API-Key (pdl_live_…)
ErgebnisAPI-Anmeldedaten

1. Bevor Sie beginnen

Paddle ist eine Merchant-of-Record-Zahlungsplattform für SaaS und digitale Produkte (Billing v2).

Erforderlich

Paddle-Konto

Für den Verkauf von SaaS und digitalen Produkten benötigen Sie ein Paddle-Konto (Billing v2).

Ausgestellte Schlüssel

Billing-API-Key

Sie generieren einen Billing-API-Key im Bearer-Format (pdl_live_…). Es ist nicht die alte Classic-Methode mit vendor_id/auth_code.

Hinweis: Der auszustellende Schlüssel ist ein Billing-(v2)-API-Key (Bearer, pdl_live_…). Verwechseln Sie ihn nicht mit der alten Classic-Methode vendor_id/auth_code.

2. Ausstellungsschritte

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Beim Paddle Dashboard anmelden

Melden Sie sich mit Ihrem vorbereiteten Paddle-Konto beim Dashboard an.

https://vendors.paddle.com
2

Zum Menü API keys wechseln

Gehen Sie zu [Developer Tools] > [Authentication] > [API keys].

Developer Tools → Authentication → API keys
3

Billing-API-Key erstellen

Erstellen Sie einen Billing-API-Key (Bearer, pdl_live_…). Es ist nicht die alte Classic-Methode vendor_id/auth_code.

Sobald ein Billing-API-Key erstellt wurde, der mit pdl_live_… beginnt, wurde er korrekt ausgestellt.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
api_keyDer ausgestellte Billing-(v2)-API-Key (Bearer, pdl_live_…, vertraulich behandeln)
Geben Sie api_key im GenieGo Integration Hub unter [Registrieren] ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
AuthentifizierungsfehlerVerwendung der alten Classic-Methode vendor_id/auth_codeVerwenden Sie den unter Developer Tools > Authentication > API keys ausgestellten Billing-(v2)-API-Key (pdl_live_…).
Falsches SchlüsselformatVerwendung eines Test-/älteren SchlüsselsPrüfen Sie, ob es sich um einen Billing-API-Key im Format pdl_live_… handelt.

Abschließende Checkliste

  • Mit einem Paddle-Konto beim Dashboard angemeldet.
  • Zu Developer Tools > Authentication > API keys navigiert.
  • Einen Billing-(v2)-API-Key (Bearer, pdl_live_…) erstellt.
  • api_key in GenieGo [Registrieren] eingegeben und gespeichert.
