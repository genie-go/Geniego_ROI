Anleitung zur API-Key-Ausstellung

Eine einsteigerfreundliche Schritt-für-Schritt-Anleitung zum Registrieren einer App im Cafe24 Developer Center, um eine Client ID und ein Client Secret auszustellen, Token über OAuth 2.0 zu erhalten und sie im GenieGo Integration Hub zu registrieren.

Geschätzte ZeitEtwa 10–15 Min.
Benötigtes KontoCafe24-Shop-Betreiberkonto
Ausgestellte SchlüsselClient ID · Client Secret · Refresh Token
ErgebnisOAuth-2.0-Anmeldedaten

1. Bevor Sie beginnen

Die Cafe24-Integration durchläuft die Entwicklerregistrierung und die App-Registrierung und authentifiziert sich über OAuth 2.0.

Erforderlich

Shop-Betreiberkonto

Sie benötigen ein Betreiberkonto, das einen Cafe24-Shop betreibt. Mit diesem Konto führen Sie die Entwicklerregistrierung im Developer Center durch.

Ausgestellte Schlüssel

Client ID · Client Secret + Token

Bei der App-Registrierung werden eine Client ID und ein Client Secret ausgestellt, und Sie erhalten über OAuth 2.0 ein Access Token und ein Refresh Token.

Hinweis: In GenieGo registrieren Sie das langlebige refresh_token, und das Access Token wird darüber automatisch aktualisiert.

2. Schritte zur App-Registrierung und Token-Ausstellung

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Als Entwickler registrieren

Gehen Sie zum Cafe24 Developer Center und registrieren Sie sich als Entwickler.

developer.cafe24.com
2

App registrieren

Erstellen Sie eine App über [Apps] > [App registrieren]; dabei werden eine Client ID und ein Client Secret ausgestellt.

Apps → App registrieren
3

OAuth-2.0-Autorisierung

Stellen Sie über die OAuth-2.0-Autorisierung ein Access Token und ein Refresh Token aus.

4

Informationen prüfen

Prüfen Sie mall_id (Mall-ID), Client ID, Client Secret und Refresh Token.

Sobald Client ID, Client Secret und Refresh Token alle bestätigt sind, wurden sie korrekt ausgestellt.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
mall_idCafe24-Shop-ID
client_idDie ausgestellte Client ID
client_secretDas ausgestellte Client Secret (vertraulich behandeln)
refresh_tokenDas über die OAuth-2.0-Autorisierung erhaltene Refresh Token
Geben Sie sie im GenieGo Integration Hub unter [Registrieren] ein und speichern Sie; die Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
AutorisierungsfehlerFalsche mall_id oder Nichtübereinstimmung der App-EinstellungenÜberprüfen Sie die bei der App-Registrierung festgelegte mall_id und die Client-Informationen erneut.
Token-AblauffehlerFalsch eingegebenes refresh_tokenRegistrieren Sie das neueste über die OAuth-2.0-Autorisierung erhaltene refresh_token erneut.

Abschließende Checkliste

  • Im Cafe24 Developer Center als Entwickler registriert.
  • Über [Apps > App registrieren] eine App erstellt und eine Client ID und ein Client Secret ausgestellt.
  • Über die OAuth-2.0-Autorisierung ein Access Token und Refresh Token ausgestellt.
  • mall_id·client_id·client_secret·refresh_token in GenieGo [Registrieren] eingegeben und gespeichert.
