Slack Webhook
Anleitung zur API-Key-Ausstellung

Eine Anleitung zum Erstellen einer App in Slack, zum Ausstellen einer Incoming-Webhook-URL und zum Registrieren im GenieGo Integration Hub, damit Benachrichtigungen an den von Ihnen festgelegten Slack-Kanal gesendet werden.

Geschätzte ZeitEtwa 5 Min.
Benötigtes KontoSlack-Workspace-Adminrechte
Ausgestellte SchlüsselWebhook URL
ErgebnisIncoming-Webhook-Anmeldedaten

1. Bevor Sie beginnen

Ein Slack-Webhook kann nur mit Workspace-Adminrechten erstellt werden.

Erforderlich

Workspace-Adminrechte

Sie benötigen Adminrechte für den Slack-Workspace, um eine App zu erstellen und den Webhook zu aktivieren.

Ausgestellte Schlüssel

Webhook URL

Nachdem Sie Incoming Webhooks aktiviert und einen Kanal ausgewählt haben, prüfen Sie die generierte Webhook-URL.

Hinweis: Die Webhook-URL ist mit dem Kanal verknüpft, an den Benachrichtigungen gesendet werden. Legen Sie im Schritt zur Kanalauswahl den Kanal fest, der Benachrichtigungen erhalten soll.

2. Ausstellungsschritte

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Neue App erstellen

Klicken Sie auf [Create New App].

https://api.slack.com/apps
2

Incoming Webhooks aktivieren

Aktivieren Sie [Incoming Webhooks] und klicken Sie auf [Add New Webhook to Workspace].

3

Kanal auswählen · URL generieren

Durch Auswahl eines Kanals wird die Webhook-URL generiert.

Sobald Sie einen Kanal ausgewählt haben und die Webhook-URL generiert wurde, wurde sie korrekt ausgestellt.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
webhook_urlDie generierte Incoming-Webhook-URL (vertraulich behandeln)
Geben Sie die webhook_url in GenieGo [Registrieren] ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
App kann nicht erstellt werdenUnzureichende Workspace-AdminrechteFahren Sie mit einem Konto fort, das Slack-Workspace-Adminrechte hat.
Benachrichtigungen kommen nicht anFalsche Webhook-URL oder kein Kanal ausgewähltAktivieren Sie Incoming Webhooks, wählen Sie einen Kanal aus und prüfen Sie die generierte Webhook-URL erneut.

Abschließende Checkliste

  • Mit Slack-Workspace-Adminrechten vorgegangen.
  • Eine App unter api.slack.com/apps über [Create New App] erstellt.
  • Incoming Webhooks aktiviert und auf [Add New Webhook] geklickt.
  • Einen Kanal ausgewählt und die Webhook-URL generiert und kopiert.
  • Die webhook_url in GenieGo [Registrieren] eingegeben und gespeichert.
