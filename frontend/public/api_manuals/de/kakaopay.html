Anleitung zur API-Key-Ausstellung

Eine Anleitung, die Sie durch das Erstellen einer Anwendung im KakaoPay Developer Center (developers.kakaopay.com) zum Ausstellen eines Händlercodes (CID) und eines Secret Key sowie deren Registrierung im GenieGo Integration Hub führt.

Geschätzte ZeitEtwa 5–15 Min.
Benötigtes KontoKakaoPay-Händlervertrag · Developer-Center-Konto
Ausgestellte SchlüsselHändlercode (CID) · Secret Key
ErgebnisZahlungsintegrations-Anmeldedaten

1. Bevor Sie beginnen

KakaoPay stellt nach Abschluss eines Händlervertrags die Anmeldedaten im Developer Center aus.

Erforderlich

Händlervertrag · Developer-Center-Konto

Ihr KakaoPay-Händlervertrag muss abgeschlossen sein, und Sie müssen sich im Developer Center (developers.kakaopay.com) registrieren und anmelden können.

Ausgestellte Schlüssel

CID · Secret Key

Wenn Sie eine Anwendung erstellen, werden ein Händlercode (CID) und ein Secret Key ausgestellt.

Hinweis: Beim Aufruf der KakaoPay-API verwendet der Authentifizierungs-Header das Format Authorization: SECRET_KEY.

2. Ausstellungsschritte

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Im Developer Center registrieren und anmelden

Registrieren Sie sich im KakaoPay Developer Center und melden Sie sich an.

https://developers.kakaopay.com
2

Anwendung erstellen

Erstellen Sie eine Anwendung, um den Händlercode (CID) und den Secret Key auszustellen oder zu überprüfen.

3

Authentifizierungs-Header prüfen

Der ausgestellte Secret Key wird im Authentifizierungs-Header im Format Authorization: SECRET_KEY verwendet.

Authorization: SECRET_KEY ${secret_key}
Sobald der Händlercode (CID) und der Secret Key ausgestellt sind, ist alles korrekt vorbereitet.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
cidDer ausgestellte Händlercode (CID)
secret_keyDer ausgestellte Secret Key (vertraulich behandeln, Authorization: SECRET_KEY)
Geben Sie sie im GenieGo Integration Hub [Registrieren] ein und speichern Sie; die Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Fehler im Authentifizierungs-HeaderDas Format des Authorization-Headers ist falschPrüfen Sie, ob der Secret Key im Format Authorization: SECRET_KEY übergeben wird.
Anwendung kann nicht erstellt werdenHändlervertrag nicht abgeschlossenSchließen Sie den Händlervertrag ab und erstellen Sie dann die Anwendung im Developer Center.

Abschließende Checkliste

  • Den KakaoPay-Händlervertrag abgeschlossen und sich im Developer Center (developers.kakaopay.com) registriert.
  • Eine Anwendung erstellt und den Händlercode (CID) und den Secret Key ausgestellt.
  • Das Authentifizierungs-Header-Format Authorization: SECRET_KEY überprüft.
  • cid·secret_key in GenieGo [Registrieren] eingegeben und gespeichert.
