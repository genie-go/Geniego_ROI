Anleitung zur API-Key-Ausstellung

Eine einsteigerfreundliche Schritt-für-Schritt-Anleitung zum Ausstellen eines Production-Keysets (DevID·AppID·CertID) im eBay-Entwicklerprogramm, zum Generieren eines OAuth-Benutzertokens und zum Registrieren im GenieGo Integration Hub.

Geschätzte ZeitEtwa 10–20 Min.
Benötigtes KontoeBay-Konto
Ausgestellte SchlüsselDevID · AppID · CertID + OAuth-Token
ErgebnisOAuth-Zugriffstoken

1. Bevor Sie beginnen

Das Ausstellen eines eBay-Keysets erfordert die Anmeldung im Entwicklerprogramm.

Erforderlich

eBay-Konto

Um mit der Ausstellung fortzufahren, benötigen Sie ein eBay-Konto. Verwenden Sie dieses Konto, um sich unter developer.ebay.com im Entwicklerprogramm anzumelden.

Ausgestellte Schlüssel

DevID · AppID · CertID

Beim Erstellen eines Production-Keysets werden eine DevID, AppID und CertID ausgestellt. Anschließend müssen Sie außerdem ein OAuth-Benutzertoken generieren, um es bei GenieGo zu registrieren.

Hinweis: Bei einem Production-Keyset kann der Hinweis "Your Keyset is currently disabled" angezeigt werden. In diesem Fall wird es erst aktiviert, nachdem Sie das Verfahren zum Abonnieren/Ablehnen der Benachrichtigung über die Löschung von Marketplace-Konten abgeschlossen haben.

2. Ausstellungsschritte

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Im Entwicklerprogramm anmelden

Gehen Sie zu developer.ebay.com und melden Sie sich mit Ihrem eBay-Konto im Entwicklerprogramm an.

developer.ebay.com
2

Production-Keyset erstellen

Klicken Sie auf der Seite [Application Keys] im Abschnitt Production auf [Create a keyset].

https://developer.ebay.com/my/keys
3

Verfahren zur Keyset-Aktivierung

Wenn der Hinweis "Your Keyset is currently disabled" angezeigt wird, klicken Sie auf den Link, um das Verfahren zum Abonnieren/Ablehnen der Benachrichtigung über die Löschung von Marketplace-Konten abzuschließen.

Application Keys → Production
4

Schlüssel überprüfen und OAuth-Token generieren

Überprüfen Sie die ausgestellte DevID, AppID und CertID und generieren Sie ein OAuth-Benutzertoken.

Sobald DevID·AppID·CertID ausgestellt und ein OAuth-Benutzertoken generiert wurde, ist die Ausstellung abgeschlossen.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
OAuth-ZugriffstokenDas von Ihnen generierte OAuth-Benutzertoken (vertraulich behandeln)
Geben Sie im GenieGo Integration Hub unter [Registrieren] Ihr OAuth-Zugriffstoken ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Keyset befindet sich im deaktivierten ZustandVerfahren zum Abonnieren/Ablehnen der Benachrichtigung über die Löschung von Marketplace-Konten nicht abgeschlossenKlicken Sie auf den Hinweis-Link "Your Keyset is currently disabled" und schließen Sie dieses Verfahren ab.
OAuth-Token kann nicht ausgestellt werdenProduction-Keyset (DevID·AppID·CertID) noch nicht ausgestelltErstellen Sie unter [Application Keys] zuerst das Production-Keyset und generieren Sie dann das OAuth-Token.

Abschließende Checkliste

  • Mit dem eBay-Konto unter developer.ebay.com im Entwicklerprogramm angemeldet.
  • DevID·AppID·CertID über Production [Create a keyset] ausgestellt.
  • Bei deaktiviertem Keyset das Verfahren zum Abonnieren/Ablehnen der Benachrichtigung über die Löschung von Marketplace-Konten abgeschlossen.
  • Ein OAuth-Benutzertoken generiert und das OAuth-Zugriffstoken in GenieGo [Registrieren] eingegeben und gespeichert.
