Naver Search Ads
Anleitung zur Ausstellung des API-Schlüssels

Eine einsteigerfreundliche Schritt-für-Schritt-Anleitung zum Ausstellen von API-Anmeldedaten (Zugriffslizenz, geheimer Schlüssel) und Ihrer CUSTOMER_ID im Naver Search Ad-Verwaltungstool sowie zur Registrierung im GenieGo Integration Hub.

Geschätzte ZeitEtwa 5–10 Min.
Benötigtes KontoNaver Search Ad Werbekundenkonto
Ausgestellte SchlüsselZugriffslizenz · Geheimer Schlüssel · CUSTOMER_ID
ErgebnisSearch Ad API-Anmeldedaten

1. Bevor Sie beginnen

Die Naver Search Ad API kann nur über ein Werbekundenkonto beantragt werden.

Erforderlich

Search Ad Werbekundenkonto

Sie benötigen ein Werbekundenkonto, das das Naver Search Ad-Verwaltungstool verwendet. Mit einem gewöhnlichen Naver-Mitgliedskonto können Sie keinen API-Zugriff beantragen.

Ausgestellte Schlüssel

Zugriffslizenz · Geheimer Schlüssel · CUSTOMER_ID

Bestätigen Sie die CUSTOMER_ID, die Zugriffslizenz und den geheimen Schlüssel, die zusammen ausgestellt werden, nachdem Sie den API-Zugriff beantragt haben.

Hinweis: Der geheime Schlüssel kann nur im Moment der Ausstellung eingesehen werden. Bewahren Sie ihn daher direkt nach der Ausstellung sicher auf.

2. Schritte zur Ausstellung des API-Schlüssels

Stellen Sie ihn in der folgenden Reihenfolge aus und registrieren Sie ihn in GenieGo.

1

Bei Search Ads anmelden

Melden Sie sich mit Ihrem Werbekundenkonto beim Naver Search Ad-Verwaltungstool an.

searchad.naver.com
2

Zur API-Nutzungsverwaltung gehen

Gehen Sie über das obere Menü [Tools] zu [API-Nutzungsverwaltung].

Tools → API-Nutzungsverwaltung
3

API-Dienst beantragen

Klicken Sie auf die Schaltfläche [Naver Search Ad API-Dienst beantragen], um den Antrag zu stellen.

Naver Search Ad API-Dienst beantragen
4

Ausgestellte Informationen prüfen

Bestätigen Sie die ausgestellte CUSTOMER_ID, Zugriffslizenz und den geheimen Schlüssel.

Sobald CUSTOMER_ID, Zugriffslizenz und geheimer Schlüssel alle bestätigt sind, wurden sie korrekt ausgestellt.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
api_keyDie ausgestellte Zugriffslizenz
api_secretDer ausgestellte geheime Schlüssel (mit Sorgfalt behandeln)
customer_idDie ausgestellte CUSTOMER_ID
Geben Sie sie unter [Registrieren] im GenieGo Integration Hub ein und speichern Sie sie; die Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Kein Menü [API-Nutzungsverwaltung]Kein Werbekundenkonto oder unzureichende BerechtigungenMelden Sie sich mit einem Werbekundenkonto an, das das Search Ad-Verwaltungstool verwendet.
Authentifizierungsfehler tritt aufZugriffslizenz oder geheimer Schlüssel falsch eingegebenÜberprüfen Sie die Zugriffslizenz, den geheimen Schlüssel und die CUSTOMER_ID auf dem Ausstellungsbildschirm erneut.

Abschließende Checkliste

  • Mit einem Naver Search Ad Werbekundenkonto angemeldet.
  • Den API-Dienst-Antrag unter [Tools > API-Nutzungsverwaltung] abgeschlossen.
  • CUSTOMER_ID, Zugriffslizenz und geheimen Schlüssel bestätigt.
  • api_key·api_secret·customer_id unter GenieGo [Registrieren] eingegeben und gespeichert.
