Naver Smart Store
Handbuch zur API-Schlüsselausstellung

Eine Schritt-für-Schritt-Anleitung zum Registrieren einer Anwendung im Naver Commerce API Center (apicenter.commerce.naver.com), um eine Client ID und ein Client Secret zu erhalten und diese anschließend im GenieGo Integration Hub zu registrieren.

Geschätzte ZeitEtwa 10–15 Min.
Benötigtes KontoSmart-Store-Konto mit der Berechtigung "Integrierter Manager"
Ausgestellte SchlüsselClient ID · Client Secret
ErgebnisCommerce-API-Anmeldedaten

1. Bevor Sie beginnen

Die Ausstellung ist nur mit der Berechtigung "Integrierter Manager" auf einem Smart-Store-Verkäuferkonto möglich.

Erforderlich

Berechtigung "Integrierter Manager"

Sie benötigen die Berechtigung "Integrierter Manager" auf einem Smart-Store-Verkäuferkonto. Nur ein integrierter Manager kann die Schlüssel ausstellen.

Ausgestellte Schlüssel

Client ID · Client Secret

Wenn Sie eine Anwendung registrieren, werden eine Anwendungs-ID (Client ID) und ein Secret (Client Secret) ausgestellt.

Hinweis: Die Registrierung der Server-IP, die die API aufruft, sowie die regelmäßige Erneuerung der Anwendungsauthentifizierung sind erforderlich.

2. Ausstellungsschritte

Stellen Sie in der untenstehenden Reihenfolge aus und registrieren Sie in GenieGo.

1

Auf das Commerce API Center zugreifen

Melden Sie sich mit Ihrem Konto als integrierter Manager beim Naver Commerce API Center an.

https://apicenter.commerce.naver.com
2

Konto erstellen

Klicken Sie oben rechts auf [Konto erstellen], geben Sie den Kontonamen des Entwicklerunternehmens und einen Kontakt für die Störungsbehebung ein, stimmen Sie den Bedingungen zu und registrieren Sie sich.

Konto erstellen
3

Eine Anwendung registrieren

Registrieren Sie unter [Meine Store-Anwendungen] eine Anwendung; anschließend werden eine Anwendungs-ID und ein Secret ausgestellt.

Meine Store-Anwendungen
4

IP registrieren und Authentifizierung erneuern

Registrieren Sie die Server-IP, die die API aufruft, und erneuern Sie die Anwendungsauthentifizierung wie angeleitet regelmäßig.

Sobald die Anwendung registriert ist und die Anwendungs-ID und das Secret ausgestellt wurden, wurde sie korrekt verarbeitet.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
client_idDie ausgestellte Anwendungs-ID (Client ID)
client_secretDas ausgestellte Secret (Client Secret, mit Sorgfalt behandeln)
Geben Sie sie im GenieGo Integration Hub unter [Registrieren] ein und speichern Sie; Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Anwendung kann nicht registriert werdenKonto verfügt nicht über die Berechtigung "Integrierter Manager"Greifen Sie mit einem Konto, das über die Berechtigung "Integrierter Manager" verfügt, auf apicenter.commerce.naver.com zu.
Authentifizierungsfehler beim AufrufAufrufer-IP nicht registriert oder Authentifizierungserneuerung versäumtRegistrieren Sie die IP des aufrufenden Servers und erneuern Sie die Anwendungsauthentifizierung regelmäßig.

Abschließende Checkliste

  • Mit einem Konto als integrierter Manager auf das Commerce API Center zugegriffen.
  • Über [Konto erstellen] Entwicklerunternehmensinformationen und einen Kontakt für die Störungsbehebung eingegeben und den Bedingungen zugestimmt.
  • Eine Anwendung unter [Meine Store-Anwendungen] registriert und die ID/das Secret erhalten.
  • Die IP des aufrufenden Servers registriert und eine regelmäßige Authentifizierung eingerichtet.
  • client_id·client_secret in GenieGo [Registrieren] eingegeben und gespeichert.
