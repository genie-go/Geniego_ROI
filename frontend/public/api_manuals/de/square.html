Anleitung zur API-Key-Ausstellung

Eine einsteigerfreundliche Schritt-für-Schritt-Anleitung zum Erstellen einer App im Square Developer Portal, um ein Access Token und eine Location ID auszustellen und diese im GenieGo Integration Hub zu registrieren.

Geschätzte ZeitEtwa 5–10 Min.
Benötigtes KontoSquare-Konto
Ausgestellte SchlüsselAccess Token · Location ID
ErgebnisAPI-Anmeldedaten

1. Bevor Sie beginnen

Square-API-Anmeldedaten werden durch das Erstellen einer App im Developer Portal ausgestellt.

Erforderlich

Square-Konto

Sie benötigen ein Square-Konto, mit dem Sie unter developer.squareup.com/apps eine App erstellen können.

Ausgestellte Schlüssel

Access Token · Location ID

Finden Sie das Access Token auf dem Tab Production und die Location ID unter [Locations].

Hinweis: Für die echte Integration verwenden Sie das Production-Token, nicht das Sandbox-Token.

2. Ausstellungsschritte

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

App erstellen

Erstellen Sie eine App im Square Developer Portal.

developer.squareup.com/apps
2

Access Token finden

Finden Sie das Access Token auf dem Tab Production.

Production
3

Location ID finden

Finden Sie die Location ID unter [Locations].

Locations
Sobald Sie das Production Access Token und die Location ID bestätigt haben, wurden sie korrekt ausgestellt.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
access_tokenDas auf dem Tab Production gefundene Access Token (vertraulich behandeln)
location_idDie unter [Locations] gefundene Location ID
Geben Sie access_token·location_id im GenieGo Integration Hub [Registrieren] ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Authentifizierung fehlgeschlagenEin Sandbox-Token wird verwendetVerwenden Sie das auf dem Tab Production gefundene Access Token.
Bestellungen werden nicht synchronisiertFalsche Location IDÜberprüfen Sie die unter [Locations] gefundene Location ID erneut.

Abschließende Checkliste

  • Eine App unter developer.squareup.com/apps erstellt.
  • Das Access Token auf dem Tab Production gefunden.
  • Die Location ID unter [Locations] gefunden.
  • access_token·location_id in GenieGo [Registrieren] eingegeben und gespeichert.
