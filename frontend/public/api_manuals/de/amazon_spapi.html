1. Was ist die Amazon SP-API?

Die Amazon Selling Partner API ist eine API zur programmatischen Integration von Verkäuferdaten wie Bestellungen, Bestand, Produkten, Versand und Abrechnungen.

Wichtig: Die SP-API ist kein einfaches API-Key-Verfahren. In der Regel sind gemeinsam Amazon-Developer-Registrierung, App-Registrierung, Login with Amazon(LWA) und OAuth-Autorisierung erforderlich.
★Für die GenieGo-Integration sind 5 Werte erforderlich: refresh_token, client_id, client_secret, marketplace_id, seller_id. (Die aktuelle SP-API wird allein mit dem LWA-Token aufgerufen, und die AWS-IAM/SigV4-Signierung wird intern von GenieGo verarbeitet, sodass Sie sie nicht in die Registrierungsfelder eingeben.)

2. Voraussetzungen

  • Ein Amazon-Seller-Central- oder Vendor-Central-Konto als Professional-Verkäufer
  • Firmen-/Kontaktinformationen für die Entwicklerregistrierung
  • App-Name, App-Beschreibung und Verwendungszweck
  • URL der Datenschutzrichtlinie
  • OAuth Redirect URL

3. Entwicklerregistrierung

Registrieren Sie zunächst Ihr SP-API-Entwicklerprofil in Seller Central.

1

Bei Seller Central anmelden

Melden Sie sich mit Ihrem Amazon-Verkäuferkonto bei Seller Central an.

Seller Central → Apps & Services
2

Zum Menü Develop Apps gehen

Gehen Sie im Menü Apps & Services zum Entwickler- oder App-Entwicklungsmenü.

Apps & Services → Develop Apps
3

Developer Profile ausfüllen

Geben Sie Firmeninformationen, Kontaktinformationen, API-Verwendungszweck und Datenschutzplan ein.

Developer Profile
4

Zur Prüfung einreichen

Reichen Sie Ihre Entwicklerregistrierungsdaten zur Prüfung durch Amazon ein.

Submit for review
Erst nach abgeschlossener Genehmigung können Sie eine produktive SP-API-App registrieren und deren Berechtigungen nutzen.

4. SP-API-App registrieren → Client ID / Client Secret

1

Neue Anwendung erstellen

Erstellen Sie in Develop Apps eine neue App.

Add new app
2

App-Typ wählen

Wenn Sie nur Ihr eigenes Konto verbinden, ist es eine Private App; wenn Sie sie externen Verkäufern bereitstellen, ist es eine Public App.

Private / Public App
3

API-Rollen auswählen

Wählen Sie die benötigten Rollen wie Orders, Catalog und Inventory aus.

API Roles
4

OAuth Redirect URL eingeben

Geben Sie die Server-URL ein, zu der nach abgeschlossener Autorisierung zurückgekehrt werden soll.

Redirect URI
5

Speichern und Client-Infos prüfen

Prüfen Sie nach dem Erstellen der App die client_id (LWA Client ID) und das client_secret (Client Secret).

client_id / client_secret

5. OAuth-Autorisierung → Refresh Token ausstellen

1) Auf die OAuth-Autorisierungs-URL des Verkäufers zugreifen 2) Den authorization code an der Redirect URL empfangen 3) Den authorization code an den LWA Token Endpoint übergeben 4) Das refresh_token / access_token empfangen
Das refresh_token wird für langfristige Aufrufe nach der Verkäuferautorisierung verwendet. GenieGo aktualisiert mit diesem refresh_token automatisch das access_token, um die SP-API aufzurufen, sodass Sie ohne separate AWS-Signierungseinrichtung lediglich das refresh_token registrieren müssen.

6. Marketplace ID · Seller ID prüfen

Prüfen Sie die beiden verbleibenden Kennungen, die für die GenieGo-Registrierung benötigt werden.

marketplace_id

Ziel-Marketplace-ID

Die Marketplace ID des Landes, das Sie verbinden. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Erforderlich für die Produktregistrierung und Bestellabfrage. Prüfen Sie ihn in Ihren Seller-Central-Kontoinformationen. Seller Central → Einstellungen → Kontoinformationen → Merchant Token

7. Im GenieGo Integration Hub registrieren

1

Die 5 Werte vorbereiten

Bereiten Sie refresh_token · client_id · client_secret · marketplace_id · seller_id vor.

5 Anmeldedaten
2

Integration Hub → Amazon SP-API → [Registrieren]

Geben Sie jeden Wert in das entsprechende Feld ein und speichern Sie (leere Felder behalten die vorhandenen Werte).

refresh_token · client_id · client_secret · marketplace_id · seller_id
Die registrierten Werte werden mit AES-256-GCM verschlüsselt gespeichert und beim Abruf maskiert. Die automatische Synchronisierung beginnt unmittelbar nach dem Speichern.
Das client_secret·refresh_token darf niemals in Frontend-Code, HTML oder öffentlichen Repositories offengelegt werden.

8. Abschließende Checkliste

  • Das Developer Profile eingereicht und genehmigt bekommen.
  • Die SP-API-App erstellt und die client_id·client_secret geprüft.
  • Ein refresh_token per OAuth-Autorisierung ausgestellt.
  • Die zu verbindende marketplace_id geprüft.
  • Die seller_id (Merchant Token) in Seller Central geprüft.
  • Die 5 Werte (refresh_token·client_id·client_secret·marketplace_id·seller_id) in GenieGo [Registrieren] eingegeben und gespeichert.

9. Häufige Probleme

ProblemUrsacheLösung
Es gibt kein API-Key-MenüDie SP-API ist keine einzelne API-Key-StrukturGehen Sie mit der LWA-Struktur (client_id·client_secret·refresh_token) vor.
Produktregistrierung schlägt fehlseller_id (Merchant Token) nicht registriertRegistrieren Sie den Merchant Token aus Ihren Seller-Central-Kontoinformationen.
Falsche Länderdatenmarketplace_id stimmt nicht übereinRegistrieren Sie die korrekte Marketplace ID für das Land, das Sie verbinden.
BerechtigungsfehlerUnzureichende App-API-Rolle oder abgelaufenes TokenPrüfen Sie die API Role und aktualisieren Sie das Token mit dem refresh_token.
