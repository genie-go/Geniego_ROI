Facebook Live
API-Ausstellungsanleitung

Eine einsteigerfreundliche Anleitung zum Erstellen einer App auf Meta for Developers und zum Vorbereiten der App ID, des App Secret und des Page Access Token, die für die Verbindung des Facebook-Seiten-Livestreams erforderlich sind.

KernkonzeptKein API Key, sondern ein Access Token
Entwicklerseitedevelopers.facebook.com
Verwendete APILive Video API (Graph)
Erforderliche BerechtigungenSeiten-Admin + Page Token

1. Das Wichtigste vorab

Bei YouTube reicht ein einziger API Key, aber Facebook (Meta) arbeitet mit einer Struktur aus App ID · App Secret · Access Token.

Wichtig: Bei Meta ist es nicht mit dem "Kopieren eines einzigen API Key" getan. Üblicherweise ist folgende Reihenfolge erforderlich: App erstellen → Produkt hinzufügen → Berechtigungen festlegen → Page Access Token ausstellen → App-Überprüfung.

2. Voraussetzungen

  • Ein Meta/Facebook-Konto
  • Zugang zu Meta for Developers
  • Eine Facebook-Seite, für die Sie Admin-Rechte haben
  • Eine Datenschutzerklärungs-URL · Nutzungsbedingungen-URL für den Live-Dienst

3. Eine Meta-Developer-App erstellen

Jede Meta-API-Integration beginnt mit dem Erstellen einer App.

1

Meta for Developers öffnen

Melden Sie sich in Ihrem Browser bei Meta for Developers an.

developers.facebook.com
2

Meine Apps → App erstellen

Klicken Sie oben auf My Apps → Create App.

My Apps → Create App
3

App-Typ auswählen

Plattformintegration/Live-Commerce kommt einem geschäftlichen Zweck näher.

Typ Business auswählen
4

App-Name·E-Mail eingeben → erstellen

Geben Sie den App-Namen und eine Kontakt-E-Mail ein und erstellen Sie die App.

App Name / Contact Email
Prüfen Sie nach dem Erstellen der App unter Dashboard → App-Einstellungen → Grundeinstellungen die App ID und das App Secret. (Das App Secret darf niemals im Frontend oder in einem öffentlichen Repository offengelegt werden.)

4. Ablauf der Facebook-Live-API-Einrichtung

1

Produkt hinzufügen

Bereiten Sie im App-Dashboard die Einstellungen rund um Facebook Login / Graph API vor.

Add Product → Facebook Login
2

Seite verbinden + Page Token

Um einen Livestream auf eine Seite zu übertragen, benötigen Sie Seiten-Admin-Rechte und ein Page Access Token.

Page Access Token
3

Erforderliche Berechtigungen beantragen (App Review)

Beantragen Sie Berechtigungen zum Veröffentlichen von Inhalten, Verwalten von Seiten und Erstellen von Live-Videos (z. B. pages_read_engagement, pages_manage_posts).

Permissions → App Review
4

Token mit Graph API Explorer ausstellen·testen

Stellen Sie ein Token aus, das die Berechtigungen enthält, und testen Sie die API-Aufrufe.

Tools → Graph API Explorer
Beispiel für einen Facebook-Live-Erstellungsaufruf
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
Bei erfolgreicher Erstellung werden Live-Übertragungsinformationen wie Broadcast-ID·Stream-URL·Stream-Key zurückgegeben. Die Seiten-ID (page_id) finden Sie in den Seiteninformationen.

5. Im GenieGo Integration Hub registrieren

1

Page Access Token kopieren

Kopieren Sie das ausgestellte Page Access Token (ein langlebiges Token wird empfohlen).

access_token (Page token)
2

Seiten-ID bestätigen

Bestätigen Sie die ID (page_id) der Ziel-Facebook-Seite.

page_id
3

Im Integration Hub registrieren

Geben Sie im GenieGo Integration Hub → Facebook Live → [Registrieren] die folgenden Werte ein und speichern Sie.

access_token · page_id
Registrierte Schlüssel werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

6. Arten von Access Tokens

Token-TypVerwendungHinweise
User Access TokenAPI-Aufrufe auf Basis der NutzeranmeldungDie zugänglichen Daten hängen vom Umfang der genehmigten Berechtigungen ab.
Page Access TokenSeitenveröffentlichung · Erstellung von Seiten-LivestreamsSeiten-Admin-Rechte und die zugehörigen Berechtigungen sind erforderlich.
Long-Lived TokenEin langlebiges Token mit verlängerter AblaufzeitIm Betrieb ist eine Ablauf-/Erneuerungslogik erforderlich.

7. Checkliste

  • Eine Meta-Developer-App erstellt.
  • Die App ID und das App Secret bestätigt.
  • Eine Facebook-Seite verbunden und ein Page Access Token ausgestellt.
  • Die erforderlichen Berechtigungen (App Review) zusammengestellt·beantragt.
  • access_token·page_id in GenieGo [Registrieren] eingegeben und gespeichert.
  • Das Token sicher in den Server-Umgebungsvariablen gespeichert.

8. Häufige Probleme

ProblemUrsacheLösung
