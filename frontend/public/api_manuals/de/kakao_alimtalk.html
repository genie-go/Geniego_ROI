Anleitung zur API-Ausstellung

Diese Anleitung führt Sie durch die praktischen Schritte von der Geschäftsverifizierung des Kakao-Talk-Kanals über die Registrierung des Absenderprofils und die Genehmigung der Alimtalk-Vorlage bis hin zur Bestätigung Ihrer API-Anmeldedaten.

KernstrukturKanalverifizierung + Vorlagengenehmigung
Benötigtes KontoKakao Business
Wichtigste ErgebnisseSender Profile / API Key
VerwendungInformationsnachrichten

1. Konzept der Kakao-Alimtalk-Ausstellung

Alimtalk ist ein Business-Messaging-Dienst, der Informationsnachrichten wie Bestellungen, Reservierungen, Versand und Zahlungen über Kakao Talk sendet. Es ist nicht möglich, allein mit einem regulären Kakao-Entwickler-App-Schlüssel direkt zu senden; eine Geschäftskanalverifizierung sowie die Einrichtung einer Versandagentur/Plattform sind erforderlich.

Wichtig: Alimtalk ist auf Informationsnachrichten ausgerichtet, nicht auf Werbenachrichten, und Vorlagen erfordern eine vorherige Prüfung.

2. Voraussetzungen

  • Kakao-Business-Konto
  • Kakao-Talk-Kanal
  • Geschäftsverifizierungsunterlagen wie ein Gewerbeanmeldungsnachweis
  • Zu sendender Alimtalk-Vorlagentext
  • Konto einer Versandagentur oder eines Cloud-Messaging-Dienstes
  • Richtlinie zur Erfassung von Empfängertelefonnummern und zur Einwilligungsverwaltung

3. Vorbereitung des Kakao-Talk-Kanals

1

Auf Kakao Business zugreifen

Melden Sie sich beim Kakao-Business-Admin-Center an.

business.kakao.com
2

Kakao-Talk-Kanal erstellen

Erstellen Sie einen Kanal unter Ihrem Marken- oder Dienstnamen.

Kakao-Talk-Kanal-Admin-Center
3

Geschäftsverifizierung durchführen

Reichen Sie Ihre Geschäftsinformationen und Nachweise ein, um den Geschäftskanal zu verifizieren.

Geschäftsverifizierung
4

Such-ID des Kanals prüfen

Prüfen Sie die für die Alimtalk-Sendeeinstellungen erforderliche Kanal-ID oder Such-ID.

Kanalprofil → Such-ID
Die Geschäftsverifizierung muss abgeschlossen sein, bevor Sie ein Alimtalk-Absenderprofil registrieren können.

4. Absenderprofil / Sender Profile registrieren

1

Sendeplattform auswählen

Wählen Sie einen Alimtalk-Anbieter wie NHN Cloud, Naver Cloud SENS, Infobank oder Bizgo.

Messaging Provider
2

Projekt erstellen

Erstellen Sie auf der ausgewählten Plattform ein Alimtalk-Projekt.

Project / Service
3

Kakao-Talk-Kanal verbinden

Registrieren Sie die Informationen Ihres geschäftsverifizierten Kakao-Talk-Kanals.

Sender Profile
4

Absenderprofil-Prüfung

Führen Sie die Bestätigung des Kanalbesitzes und die Prüfung des Absenderprofils durch.

Profile Review
5

API-Anmeldedaten bestätigen

Prüfen Sie den von der Plattform bereitgestellten App Key, Secret Key, API Key usw.

API Key / Secret

5. Vorlagen registrieren und genehmigen

1

Vorlage verfassen

Verfassen Sie Informationstexte wie Bestellbestätigungen und Versandhinweise.

Template Message
2

Variablen festlegen

Legen Sie dynamisch wechselnde Werte wie Name, Bestellnummer und Sendungsnummer als Variablen fest.

#{Variable}
3

Prüfung anfordern

Speichern Sie die Vorlage und reichen Sie sie zur Kakao-Prüfung ein.

Request Approval
4

Genehmigung bestätigen

Bestätigen Sie den genehmigten Vorlagencode oder die ID.

Template Code
Wenn sich der Text vor der Prüfung vom tatsächlich gesendeten Text unterscheidet, kann der Versand fehlschlagen oder es können Sanktionen verhängt werden.

6. Beispiel für den API-Sendeablauf

1) Rufen Sie den Alimtalk-API-Endpoint von Ihrem Server aus auf
2) Authentifizieren Sie sich mit dem API Key oder Secret
3) Übergeben Sie die Empfängertelefonnummer, den Vorlagencode und die Variablenwerte
4) Empfangen Sie das Sendeergebnis
5) Behandeln Sie, ob bei einem Fehler ein SMS-Fallback gesendet wird

7. Abschließende Checkliste

  • Einen Kakao-Talk-Kanal erstellt.
  • Die Geschäftsverifizierung abgeschlossen.
  • Eine Alimtalk-Sendeplattform ausgewählt.
  • Ein Sender Profile registriert.
  • Den API Key oder Secret bestätigt.
  • Die Vorlage registriert und genehmigen lassen.
  • Eine Richtlinie für Sendefehler und Fallback-SMS vorbereitet.
