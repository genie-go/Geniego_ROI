Google Ads
Anleitung zur API-Key-Ausstellung

Eine Anleitung, die Sie durch die Ausstellung eines Entwickler-Tokens im API-Center Ihres Google Ads-Verwaltungskontos (MCC), die Erstellung eines OAuth-Clients in Google Cloud zur Beschaffung eines refresh token und die Registrierung aller Daten im GenieGo Integration Hub führt.

Geschätzte ZeitEtwa 20–40 Min. (Prüfung des Entwickler-Tokens separat)
Benötigtes KontoGoogle Ads-Verwaltungskonto (MCC) · Google-Konto
Ausgestellte SchlüsselEntwickler-Token · client_id/secret · refresh_token · customer_id
ErgebnisGoogle Ads API-Anmeldedaten

1. Bevor Sie beginnen

Die Google Ads-Integration erfordert sowohl ein Verwaltungskonto (MCC) als auch eine OAuth-Einrichtung in Google Cloud.

Entwickler-Token

MCC API-Center

Beantragen Sie ein Entwickler-Token im API-Center Ihres Google Ads-Verwaltungskontos (MCC). Der Basiszugriff (Basic) wird nach einer Google-Prüfung genehmigt.

OAuth

Google Cloud-Client

Erstellen Sie auf console.cloud.google.com einen OAuth-Client (client_id·client_secret) und authentifizieren Sie sich, um ein refresh token zu erhalten.

Hinweis: Der Basiszugriff für das Entwickler-Token wird nach einer Google-Prüfung genehmigt, und vor der Genehmigung können nur Testkonten aufgerufen werden. Wenn Sie kein Verwaltungskonto (MCC) haben, erstellen Sie zuerst eines unter ads.google.com.

2. Schritte zur Ausstellung

Stellen Sie in der unten angegebenen Reihenfolge aus und registrieren Sie in GenieGo.

1

Beim Verwaltungskonto anmelden · Zum API-Center wechseln

Melden Sie sich bei Ihrem Google Ads-Verwaltungskonto an und gehen Sie dann oben zu [Tools und Einstellungen] > [Einrichtung] > [API-Center].

Tools und Einstellungen → Einrichtung → API-Center
2

Entwickler-Token beantragen

Beantragen Sie ein [Entwickler-Token] im API-Center. Der Basiszugriff (Basic) wird nach einer Google-Prüfung genehmigt, und vor der Genehmigung können nur Testkonten aufgerufen werden.

3

Cloud-Projekt einrichten · OAuth-Zustimmungsbildschirm

Erstellen Sie ein Projekt und konfigurieren Sie [APIs und Dienste] > [OAuth-Zustimmungsbildschirm].

console.cloud.google.com → APIs und Dienste → OAuth-Zustimmungsbildschirm
4

OAuth-Client-ID erstellen

Erstellen Sie [Anmeldedaten] > [Anmeldedaten erstellen] > [OAuth-Client-ID], um eine client_id und ein client_secret zu erhalten (Scope: adwords).

Anmeldedaten → Anmeldedaten erstellen → OAuth-Client-ID
5

Refresh token ausstellen

Erhalten Sie ein refresh token über den OAuth-Authentifizierungsprozess (verwenden Sie Google OAuth Playground oder Ihre eigene Authentifizierung).

6

Kundennummer ermitteln

Ermitteln Sie die 10-stellige Kundennummer (customer_id) des Werbekontos, das Sie aufrufen möchten.

Sobald das Entwickler-Token genehmigt ist und client_id·client_secret·refresh_token·customer_id alle bereit sind, ist die Ausstellung abgeschlossen.

3. In GenieGo zu registrierende Informationen

FeldBeschreibung
developer_tokenDas im API-Center ausgestellte Entwickler-Token
client_idOAuth-Client-ID
client_secretOAuth-Client-Secret (mit Vorsicht behandeln)
refresh_tokenDas per OAuth-Authentifizierung ausgestellte refresh token (im Feld [Zugriffstoken] eingeben)
customer_idDie 10-stellige Kundennummer des Zielwerbekontos
Geben Sie das Entwickler-Token, client_id, client_secret, refresh_token und customer_id in GenieGo [Registrieren] ein und speichern Sie. Anmeldedaten werden im Ruhezustand mit AES-256-GCM verschlüsselt und beim Lesen maskiert.

4. Häufige Probleme

ProblemUrsacheLösung
Nur Testkonten können aufgerufen werdenPrüfung des Basiszugriffs für das Entwickler-Token nicht genehmigtWarten Sie, bis die Google-Prüfung des im API-Center beantragten Entwickler-Tokens genehmigt ist, und rufen Sie dann Produktionskonten auf.
Authentifizierung schlägt fehlFehlender Scope (adwords) oder falsches refresh_tokenAuthentifizieren Sie sich erneut mit dem im OAuth-Client enthaltenen adwords-Scope und überprüfen Sie das refresh_token erneut.
Fehler im Kundennummern-FormatFalsche 10-stellige customer_idÜberprüfen Sie die 10-stellige Kundennummer des Zielwerbekontos erneut und geben Sie sie ein.

Abschließende Checkliste

  • Ein Google Ads-Verwaltungskonto (MCC) vorbereitet und angemeldet.
  • Ein Entwickler-Token im API-Center beantragt (Basiszugriff durchläuft eine Google-Prüfung).
  • Einen OAuth-Zustimmungsbildschirm und eine OAuth-Client-ID/Secret auf console.cloud.google.com erstellt (Scope: adwords).
  • Ein refresh token per OAuth-Authentifizierung erhalten.
  • Die 10-stellige customer_id des Zielkontos ermittelt.
  • Das Entwickler-Token·client_id/secret·refresh_token·customer_id in GenieGo [Registrieren] eingegeben und gespeichert.
