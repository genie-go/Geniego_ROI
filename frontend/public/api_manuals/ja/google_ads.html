Google Ads
API Key 発行マニュアル

Google Ads 管理者(MCC)アカウントの API センターで開発者トークンを発行し、Google Cloud で OAuth クライアントを作成して refresh token を取得した後、GenieGo 連携ハブに登録する手順をまとめたマニュアルです。

所要時間約20〜40分(開発者トークン審査は別途)
必要なアカウントGoogle Ads 管理者(MCC)アカウント · Google アカウント
発行するキー開発者トークン · client_id/secret · refresh_token · customer_id
成果物Google Ads API 認証情報

1. はじめる前に

Google Ads 連携には、管理者(MCC)アカウントと Google Cloud での OAuth 設定の両方が必要です。

開発者トークン

MCC API センター

Google Ads 管理者(MCC)アカウントの API センターで開発者トークンを申請します。基本(Basic)アクセスは Google の審査を経て承認されます。

OAuth

Google Cloud クライアント

console.cloud.google.com で OAuth クライアント(client_id·client_secret)を作成し、認証して refresh token を取得します。

注意: 開発者トークンの基本アクセスは Google の審査を経て承認され、承認前はテストアカウントのみ呼び出せます。管理者(MCC)アカウントがない場合は、まず ads.google.com で作成してください。

2. 発行手順

以下の順序で発行し、GenieGo に登録します。

1

管理者アカウントにログイン · API センターへ移動

Google Ads 管理者アカウントにログインした後、上部の [ツールと設定] > [設定] > [API センター] に移動します。

ツールと設定 → 設定 → API センター
2

開発者トークンを申請

API センターで [開発者トークン] を申請します。基本(Basic)アクセスは Google の審査を経て承認され、承認前はテストアカウントのみ呼び出せます。

3

Cloud プロジェクト · OAuth 同意画面の構成

プロジェクトを作成し、[API とサービス] > [OAuth 同意画面] を構成します。

console.cloud.google.com → API とサービス → OAuth 同意画面
4

OAuth クライアント ID を作成

[認証情報] > [認証情報を作成] > [OAuth クライアント ID] を作成して client_id と client_secret を発行します(スコープ scope: adwords)。

認証情報 → 認証情報を作成 → OAuth クライアント ID
5

refresh token を発行

OAuth 認証手順で refresh token を発行します(Google OAuth Playground または独自の認証を使用)。

6

顧客 ID を確認

呼び出し対象の広告アカウントの10桁の顧客 ID(customer_id)を確認します。

開発者トークンの承認 + client_id·client_secret·refresh_token·customer_id がすべて揃えば、正常に発行されています。

3. GenieGo に登録する情報

フィールド説明
developer_tokenAPI センターで発行された開発者トークン
client_idOAuth クライアント ID
client_secretOAuth クライアントシークレット(漏えい注意)
refresh_tokenOAuth 認証で発行された refresh token([アクセストークン] 欄に入力)
customer_id対象広告アカウントの10桁の顧客 ID
GenieGo [登録] に開発者トークン、client_id、client_secret、refresh_token、customer_id を入力して保存すると、認証情報は AES-256-GCM で暗号化保存され、照会時にマスキングされます。

4. よく発生する問題

問題原因解決
テストアカウントのみ呼び出せる開発者トークンの基本アクセス審査が未承認API センターで申請した開発者トークンの Google 審査が承認されるまで待ってから、運用アカウントを呼び出します。
認証失敗scope(adwords)の欠落または refresh_token の誤記OAuth クライアントに adwords スコープを含めて再認証し、refresh_token を再確認します。
顧客 ID 形式エラー10桁の customer_id の誤記対象広告アカウントの10桁の顧客 ID を再確認して入力します。

最終チェックリスト

  • Google Ads 管理者(MCC)アカウントを準備し、ログインした。
  • API センターで開発者トークンを申請した(基本アクセスは Google の審査)。
  • console.cloud.google.com で OAuth 同意画面と OAuth クライアント ID/Secret を作成した(scope: adwords)。
  • OAuth 認証で refresh token を発行した。
  • 対象アカウントの10桁の customer_id を確認した。
  • GenieGo [登録] に開発者トークン·client_id/secret·refresh_token·customer_id を入力して保存した。
