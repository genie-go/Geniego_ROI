1. Yahoo! Japan API 発行の概念

Yahoo! Japan 系の API は、サービスの種類に応じて Yahoo! JAPAN Developers または LY/Yahoo!広告 Developer Center で Client ID、Client Secret、OAuth 認証情報を発行します。

重要: 「Yahoo! Japan」は、利用する API が何であるかによって発行サイトと審査手続きが異なります。ショッピング、広告、ログイン、検索 API を区別する必要があります。

2. 準備物

  • Yahoo! JAPAN ID またはビジネスアカウント
  • 利用する API の種類を決定
  • アプリ名とサービス説明
  • Redirect URI
  • プライバシーポリシー URL
  • 運用ドメインまたはテストドメイン

3. アプリケーション登録の基本手順

1

開発者サイトにアクセス

Yahoo! JAPAN Developers または該当 API の Developer Center にログインします。

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

アプリケーションを作成

新規アプリケーション追加メニューをクリックします。

Create Application / Add Application
3

アプリ情報を入力

アプリ名、説明、担当者情報、ウェブサイト URL を入力します。

Application Information
4

Redirect URI を入力

OAuth 認証後に戻る URL を正確に入力します。

Redirect URI
5

権限または API を選択

利用する API の権限とスコープを選択します。

Scope / API Permission
6

登録完了

登録後、Client ID と Client Secret を確認します。

Client ID / Client Secret

4. サービス別の発行場所

目的発行/管理場所成果物
Yahoo! Japan ログイン / OAuthYahoo! JAPAN DevelopersClient ID, Client Secret
Yahoo!広告 APIYahoo!広告 Developer CenterClient ID, Client Secret, OAuth Token
Yahoo! Shopping APIYahoo! JAPAN Developers または Commerce 関連コンソールApplication ID または OAuth 情報
社内テスト各サービスの Sandbox/テスト環境テスト用認証情報

5. OAuth トークン発行フロー

1) Client ID でユーザー認証 URL を生成 2) ユーザーが Yahoo! アカウントでログイン 3) Redirect URI で authorization code を受信 4) authorization code + Client Secret で Access Token をリクエスト 5) Access Token で API を呼び出し 6) 必要に応じて Refresh Token で更新

6. よくある問題

問題原因解決方法
Redirect URI エラー登録した URL と呼び出し URL が異なるプロトコル、ドメイン、パスまで同一に入力
Client Secret を確認できないセキュリティ上、非表示処理管理画面の View または再発行機能を確認
権限不足必要な Scope が未選択アプリ設定で Scope を追加してから再認証
API 呼び出し失敗利用 API と発行したアプリ種別の不一致API ごとの開発者ドキュメントを確認

7. 最終チェックリスト

  • 利用する Yahoo! Japan API の種類を確定した。
  • 正しい開発者コンソールにログインした。
  • アプリケーションを登録した。
  • Client ID と Client Secret を確認した。
  • Redirect URI を正確に登録した。
  • 必要な Scope を選択した。
  • Access Token 発行テストを完了した。
