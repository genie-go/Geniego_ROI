1. はじめる前に

フルフィルメント(3PL)は、入出庫委託契約の後に認証情報が発行される契約型チャネルです。

必須

入出庫委託契約

3PL フルフィルメントを利用するには、フルフィルメント事業者または運用チームとの入出庫委託契約が先行して必要です。

発行するキー

3PL API キー · 倉庫 ID

契約完了後、倉庫 ID と 3PL API キーが発行されます。

注意: このチャネルは、セルフコンソールで直接発行する方式ではなく、委託契約を通じて認証情報を受領する方式です。

2. 発行申請手順

以下の順序で申請・受領してから GenieGo に登録します。

1

委託契約の進行

フルフィルメント事業者または運用チームと入出庫委託契約を進めます。

2

契約締結

3PL フルフィルメントの入出庫委託契約を締結します。

3

認証情報の受領

契約完了後、倉庫 ID·3PL API キーが発行されます。

4

GenieGo に登録

発行された api_key、warehouse_id を GenieGo の [登録] に入力して保存します。

フルフィルメント事業者から倉庫 ID·3PL API キーをすべて受領していれば、正常に発行されています。

3. GenieGo に登録する情報

フィールド説明
api_key発行された 3PL API キー
warehouse_id発行された倉庫 ID
GenieGo 連携ハブの [登録] に入力して保存すると、認証情報は AES-256-GCM で暗号化して保存され、閲覧時にはマスキングされます。

4. よくある問題

問題原因解決方法
認証情報を受け取れない入出庫委託契約が未完了フルフィルメント事業者または運用チームと委託契約を締結したうえで、認証情報を発行してもらいます。
入出庫連携エラー倉庫 ID(warehouse_id)の記載ミス発行された倉庫 ID を再度確認し、正確に入力します。

最終チェックリスト

  • フルフィルメント事業者または運用チームと入出庫委託契約を締結した。
  • 契約完了後、倉庫 ID·3PL API キーを発行してもらった。
  • api_key·warehouse_id を確認した。
  • GenieGo の [登録] に api_key·warehouse_id を入力して保存した。
