1. Twitch アカウント構成の理解

Twitch は Instagram のようにビジネスアカウント/クリエイターアカウントを別途選択する構成ではありません。

企業公式チャネル

ブランド運営用

会社名またはブランド名で一般の Twitch アカウントを作成し、公式チャネルのように運営します。製品紹介、ライブコマース、イベント配信に適しています。

ストリーマー成長段階

Affiliate / Partner

配信実績、フォロワー、視聴時間の条件を満たすと、Affiliate または Partner プログラムに参加できます。

ポイント: 企業もまず一般の Twitch アカウントを作成し、チャネルのブランディングと2段階認証、OBS 接続、Developer Console のアプリ登録を進めればよいです。

2. 準備物

  • 企業用メールアドレス
  • ブランド名または会社名
  • Twitch ユーザー名の候補
  • 会社ロゴとチャネルバナー画像
  • 会社紹介文とウェブサイト URL
  • 配信用 PC またはノートパソコン
  • OBS Studio またはストリーミングソフトウェア
  • API 連携用 OAuth Redirect URL

3. Twitch アカウントの作成

1

Twitch にアクセス

ブラウザで Twitch 公式サイトにアクセスします。

https://www.twitch.tv
2

会員登録をクリック

右上の Sign Up または会員登録ボタンをクリックします。

Sign Up
3

企業情報で登録

会社メール、ユーザー名、パスワード、生年月日を入力します。

Email / Username / Password
4

メール認証

Twitch から送られた認証メールを開き、Verify Email を完了します。

Verify Email

4. ブランドアカウント名を決める

Twitch のユーザー名はチャネルのアドレスになるため、短く覚えやすい名前が良いです。

目的おすすめの名前の例説明
企業公式チャネルGeniego / OCLKorea会社名と一致して信頼度が高くなります。
グローバル配信GeniegoGlobal / OCLGlobal海外の視聴者を考慮する際に適しています。
ライブコマースGeniegoLive / GeniegoCommerce販売配信、ショッピング配信の目的が明確です。
技術/プラットフォームGeniegoTech / OCLTechIT、SaaS、プラットフォーム紹介配信に適しています。

5. チャネルプロフィールの設定

1

設定メニューへ移動

プロフィール画像を登録

ブランドカラーとサービスメッセージを含むバナー画像を登録します。

Profile Banner

会社紹介、サービス説明、問い合わせメール、ウェブサイトを入力します。

Bio / Links

6. 2段階認証の設定

Twitch で配信とセキュリティ設定を安定して使用するには、2FA 設定が非常に重要です。

1

セキュリティメニューへ移動

設定で Security and Privacy メニューを開きます。

Settings → Security and Privacy
2

2FA を有効化

Two-Factor Authentication を有効化します。

Set Up Two-Factor Authentication
3

携帯電話または認証アプリを接続

SMS 認証または Google Authenticator、Authy のような認証アプリを接続します。

Authenticator App
注意: 企業アカウントは複数人で使用できるため、管理者権限と復旧用メールを必ず管理してください。

7. ストリームキーの確認と OBS 接続

1

クリエイターダッシュボードを開く

プロフィールメニューから Creator Dashboard に移動します。

Creator Dashboard
2

ストリームメニューへ移動

設定メニューで Stream 項目を選択します。

Settings → Stream
3

Primary Stream Key を確認

基本のストリームキーをコピーします。このキーはパスワードのように保管する必要があります。

Primary Stream Key
4

OBS に接続

OBS Studio でサービスは Twitch、ストリームキーはコピーした値を入力します。

OBS → Settings → Stream → Twitch
ストリームキーが漏洩すると、他人があなたのチャネルで配信できてしまいます。絶対に公開しないでください。

9. Twitch Developer Console への登録

チャットボット、視聴者分析、配信状態の照会、自動化機能を作るには、Twitch Developer アプリを登録する必要があります。

1

Twitch Developer にアクセス

Twitch 開発者サイトにアクセスします。

https://dev.twitch.tv
2

Console へ移動

上部メニューで Console または Developer Console をクリックします。

Developer Console
3

Applications メニューを選択

左側または上部メニューで Applications を選択します。

Applications
4

Register Your Application をクリック

新しいアプリケーション登録ボタンをクリックします。

Register Your Application

10. Twitch API アプリ登録情報の入力

入力項目入力例説明
NameGeniego Twitch Appアプリ名です。サービス名を基準に入力します。
OAuth Redirect URLshttps://yourdomain.com/auth/twitch/callbackOAuth ログイン後に戻るサーバーアドレスです。
CategoryWebsite Integration / Chat Botアプリの用途に合ったカテゴリを選択します。
Client TypeConfidentialサーバーで Secret を安全に保管する一般的なサービスに適しています。
登録が完了すると Client ID が表示され、New Secret ボタンで Client Secret を生成できます。

11. Client ID / Client Secret の確認

1

登録したアプリを選択

Developer Console の Applications リストで先ほど作成したアプリをクリックします。

Applications → アプリを選択
2

Client ID をコピー

画面に表示された Client ID をコピーしてサーバー設定に保存します。

Client ID
3

Client Secret を生成

New Secret ボタンを押して Client Secret を生成し、安全に保存します。

New Secret
Client Secret は一度しか確認できない場合が多いです。生成したらすぐに安全な場所に保存してください。

12. Twitch API 認証構造

Twitch Channel │ ▼ Developer Console │ ▼ Application │ ▼ Client ID / Client Secret │ ▼ OAuth Token │ ▼ Twitch Helix API │ ▼ チャネル情報、配信状態、チャットボット、視聴者分析

13. OAuth トークン発行フロー

1) Twitch OAuth 認証 URL を生成 2) ユーザーが Twitch アカウントでログイン 3) Redirect URL で authorization code を受信 4) authorization code + Client Secret で Access Token を要求 5) Access Token で Twitch API を呼び出し 6) 必要に応じて Refresh Token で更新
単純な公開データの照会は App Access Token で可能で、ユーザー権限が必要な操作には User Access Token が必要です。
