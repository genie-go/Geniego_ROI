Facebook Live
API 発行マニュアル

Meta for Developers でアプリを作成し、Facebook ページのライブ連携に必要な App ID、App Secret、ページアクセストークン(Page Access Token)を準備する手順を、初心者でもそのまま実行できるようにまとめました。

核心概念API Key ではなく Access Token
開発サイトdevelopers.facebook.com
使用 APILive Video API (Graph)
必要な権限ページ管理者 + Page Token

1. まず知っておくべき核心

YouTube は API Key 一つで完結しますが、Facebook(Meta)は App ID · App Secret · Access Token の構造で動作します。

重要: Meta は「API Key を一つコピーする」だけでは完結しません。通常は アプリ作成 → 製品追加 → 権限設定 → ページアクセストークン発行 → アプリ審査 の順序が必要です。

2. 準備するもの

  • Meta/Facebook アカウント
  • Meta for Developers へのアクセス権限
  • 管理権限を持つ Facebook ページ
  • 実サービス用のプライバシーポリシー URL · サービス利用規約 URL

3. Meta Developer アプリを作成

すべての Meta API 連携はアプリを作成することから始まります。

1

Meta for Developers にアクセス

ブラウザで Meta for Developers にログインします。

developers.facebook.com
2

マイアプリ → アプリを作成

上部の My Apps → Create App をクリックします。

My Apps → Create App
3

アプリの種類を選択

プラットフォーム連携/ライブコマースはビジネス目的に近いです。

Business タイプを選択
4

アプリ名·メール入力 → 作成

アプリ名、連絡先メールを入力してアプリを作成します。

App Name / Contact Email
アプリ作成後、ダッシュボード → アプリ設定 → 基本設定で App ID と App Secret を確認します。(App Secret はフロントエンド/公開リポジトリに絶対に公開しないでください)

4. Facebook Live API 設定の流れ

1

製品を追加

アプリのダッシュボードで Facebook Login / Graph API 関連の設定を準備します。

Add Product → Facebook Login
2

ページ連携 + ページトークン

ライブをページに配信するには、ページ管理者権限と Page Access Token が必要です。

Page Access Token
3

必要な権限を申請(App Review)

コンテンツ投稿·ページ管理·ライブビデオ作成を目的とする権限(例: pages_read_engagement, pages_manage_posts)を申請します。

Permissions → App Review
4

Graph API Explorer でトークン発行·テスト

権限を含むトークンを発行し、API 呼び出しをテストします。

Tools → Graph API Explorer
Facebook Live 作成呼び出しの例
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
正常に作成されると、配信 ID·ストリーム URL·ストリームキーなどのライブ配信情報が返されます。ページ ID(page_id)はページ情報で確認します。

5. GenieGo 連携ハブに登録

1

ページアクセストークンをコピー

発行した Page Access Token をコピーします(長期トークン推奨)。

access_token (Page token)
2

ページ ID を確認

対象の Facebook ページの ID(page_id)を確認します。

page_id
3

連携ハブに登録

GenieGo 連携ハブ → Facebook Live → [登録] に以下の値を入力して保存します。

access_token · page_id
登録したキーは AES-256-GCM で暗号化して保存され、閲覧時にはマスキングされます。

6. Access Token の種類

トークンの種類用途注意事項
User Access Tokenユーザーログインに基づく API 呼び出し権限の承認範囲によってアクセスできるデータが変わります。
Page Access Tokenページ投稿·ページライブ作成ページ管理者権限と関連権限が必要です。
Long-Lived Token有効期限を延ばした長期トークン運用時には有効期限·更新ロジックが必要です。

7. チェックリスト

  • Meta Developer アプリを作成した。
  • App ID と App Secret を確認した。
  • Facebook ページを連携し、Page Access Token を発行した。
  • 必要な権限(App Review)を整理·申請した。
  • GenieGo の [登録] に access_token·page_id を入力して保存した。
  • トークンをサーバーの環境変数に安全に保管した。

8. よくある問題

問題原因解決方法
API Key メニューがないMeta は単一の API Key 構造ではないApp ID·App Secret·Access Token の構造で進めます。
権限エラートークンに必要な権限がないGraph API Explorer で権限を追加した後、トークンを再発行します。
ページライブの作成に失敗Page Access Token/管理者権限が不足ページ管理者権限とトークンの権限を確認します。
