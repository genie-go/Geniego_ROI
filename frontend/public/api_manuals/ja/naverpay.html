API Key 発行マニュアル

ネイバーペイは、加盟店登録(契約)の完了後に担当者から認証情報を受け取るチャネルです。受け取った Client ID・Client Secret・Chain ID を GenieGo 連携ハブに登録する手順をまとめたマニュアルです。

所要時間契約・審査期間によって異なる
必要なアカウントネイバーペイ ビジネスセンター アカウント
発行するキーClient ID · Client Secret · Chain ID
成果物決済連携の認証情報

1. はじめる前に

ネイバーペイは自己発行ではなく、加盟契約の完了後に認証情報が渡される方式です。

ネイバーペイは、コンソールで直接キーを発行する方式ではありません。 加盟店登録(契約)が完了した後、担当者から認証情報を受け取ります。
重要: 通常の自己発行ではありません。加盟店登録(契約)の完了後に、パートナーID・Client ID/Secret が渡されます。

2. 発行申請の手順

以下の順序で契約を進め、受け取った認証情報を登録します。

1

加盟契約の申請

ネイバーペイ ビジネスセンターで加盟契約を申請して完了します。

2

審査・契約の完了

審査と契約が完了するまで進めます。期間は審査スケジュールによって変わる場合があります。

3

認証情報の受領

審査・契約の完了後、担当者から Client ID・Client Secret・Chain ID を受け取ります。

担当者から Client ID・Client Secret・Chain ID をすべて受け取れば、登録の準備が完了したことになります。

3. GenieGo に登録する情報

フィールド説明
client_id受け取った Client ID
client_secret受け取った Client Secret(取り扱い注意)
chain_id受け取った Chain ID
GenieGo 連携ハブの [登録] に入力して保存すると、認証情報は AES-256-GCM で暗号化して保存され、閲覧時にはマスキングされます。

4. よくある問題

問題原因解決方法
認証情報を発行するメニューが見つからないネイバーペイは自己発行方式ではない加盟契約の完了後に担当者から受け取った認証情報を使用します。
認証失敗Client Secret または Chain ID の記載ミス担当者から受け取った Client ID・Client Secret・Chain ID を再度確認します。

最終チェックリスト

  • ネイバーペイ ビジネスセンターで加盟契約を申請・完了した。
  • 審査・契約の完了後、担当者から Client ID・Client Secret・Chain ID を受け取った。
  • 受け取った3つの認証情報をすべて確認した。
  • GenieGo の [登録] に client_id·client_secret·chain_id を入力して保存した。
