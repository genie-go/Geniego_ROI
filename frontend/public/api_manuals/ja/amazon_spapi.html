1. Amazon SP-APIとは?

Amazon Selling Partner APIは、注文、在庫、商品、配送、精算などの出品者データをプログラムから連携するためのAPIです。

重要: SP-APIは単純なAPI Key方式ではありません。一般的に Amazon Developer登録、アプリ登録、Login with Amazon(LWA)、OAuth承認 が併せて必要です。
★GenieGo連携に必要な値は5つです: refresh_token, client_id, client_secret, marketplace_id, seller_id。(現在のSP-APIはLWAトークンのみで呼び出され、AWS IAM/SigV4署名はGenieGoが内部で処理するため、登録欄には入力しません。)

2. 準備するもの

  • Amazon Seller Central または Vendor Central のプロフェッショナル出品者アカウント
  • 開発者登録に使用する会社/担当者情報
  • アプリ名、アプリ説明、利用目的
  • プライバシーポリシーURL
  • OAuth Redirect URL

3. 開発者登録

まずSeller CentralでSP-API開発者プロフィールを登録します。

1

Seller Centralにログイン

Amazon出品者アカウントでSeller Centralにログインします。

Seller Central → Apps & Services
2

Develop Appsメニューへ移動

Apps & Servicesメニューから開発者またはアプリ開発メニューへ移動します。

Apps & Services → Develop Apps
3

Developer Profileを記入

会社情報、担当者情報、API利用目的、データ保護計画を入力します。

Developer Profile
4

審査に提出

Amazonの審査のために開発者登録情報を提出します。

Submit for review
承認が完了して初めて、本番運用用のSP-APIアプリ登録と権限の利用が可能になります。

4. SP-APIアプリ登録 → Client ID / Client Secret

1

新規アプリケーション作成

Develop Appsで新しいアプリを作成します。

Add new app
2

アプリ種別の選択

自社アカウントのみ連携する場合はPrivate App、外部出品者に提供する場合はPublic Appの性質になります。

Private / Public App
3

API役割の選択

Orders、Catalog、Inventoryなど必要な役割を選択します。

API Roles
4

OAuth Redirect URLの入力

承認完了後に戻るサーバーURLを入力します。

Redirect URI
5

保存後にClient情報を確認

アプリ作成後、client_id (LWA Client ID)、client_secret (Client Secret)を確認します。

client_id / client_secret

5. OAuth承認 → Refresh Token発行

1) 出品者OAuth承認URLにアクセス 2) Redirect URLでauthorization codeを受信 3) authorization codeをLWA Token Endpointに渡す 4) refresh_token / access_tokenを受信
refresh_tokenは出品者承認後の長期呼び出しに使用されます。GenieGoはこのrefresh_tokenでaccess_tokenを自動更新してSP-APIを呼び出すため、別途のAWS署名設定なしにrefresh_tokenだけを登録すれば十分です。

6. Marketplace ID · Seller IDの確認

GenieGo登録に必要な残り2つの識別子を確認します。

marketplace_id

販売対象のマーケットプレイスID

連携する国のMarketplace IDです。US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

出品者ID / Merchant Token

商品登録・注文照会に必要です。Seller Centralのアカウント情報で確認します。Seller Central → 設定 → アカウント情報 → Merchant Token

7. GenieGo 連携ハブに登録

1

5つの値を準備

refresh_token · client_id · client_secret · marketplace_id · seller_idを準備します。

5つの認証情報
2

連携ハブ → Amazon SP-API → [登録]

各欄に該当する値を入力して保存します(空欄は既存の値を維持)。

refresh_token · client_id · client_secret · marketplace_id · seller_id
登録した値は AES-256-GCMで暗号化保存 され、照会時にマスキングされます。保存直後に自動同期が開始されます。
client_secret·refresh_tokenは絶対にフロントエンドコード、HTML、公開リポジトリに公開してはいけません。

8. 最終チェックリスト

  • Developer Profileを提出し承認を受けた。
  • SP-APIアプリを作成しclient_id·client_secretを確認した。
  • OAuth承認でrefresh_tokenを発行した。
  • 連携対象のmarketplace_idを確認した。
  • Seller Centralでseller_id(Merchant Token)を確認した。
  • GenieGo [登録]に5つの値(refresh_token·client_id·client_secret·marketplace_id·seller_id)を入力して保存した。

9. よくある問題

問題原因解決方法
API KeyメニューがないSP-APIは単一のAPI Key構造ではないLWA(client_id·client_secret·refresh_token)構造で進めます。
商品登録に失敗seller_id(Merchant Token)が未登録Seller Centralのアカウント情報のMerchant Tokenを登録します。
国データが誤っているmarketplace_idの不一致連携国の正確なMarketplace IDを登録します。
権限エラーアプリのAPI Role不足またはトークン期限切れAPI Roleを確認し、refresh_tokenでトークンを更新します。
