Meta Ads
API Key 発行マニュアル

Meta ビジネスアプリでシステムユーザーのアクセストークンを発行し、広告アカウント ID(act_)とともに GenieGo 連携ハブに登録して Meta Ads の広告データを連携する手順を、初心者でもそのまま実行できるようにまとめたマニュアルです。

所要時間約 10〜20 分
必要なアカウントFacebook 個人アカウント · Meta 広告アカウント
発行するキーアクセストークン · 広告アカウント ID(act_)
成果物Marketing API 認証情報

1. はじめる前に

Meta Ads 連携には、ビジネスアカウントと、広告を実際に配信中の広告アカウントが必要です。

必須

Facebook 個人アカウント · 広告アカウント

Facebook 個人アカウントと、実際に広告を配信中の Meta 広告アカウントが必要です。

発行するキー

アクセストークン · 広告アカウント ID(act_)

システムユーザートークン(アクセストークン)と、act_ で始まる数値型の広告アカウント ID を確認します。

注意: 自社ビジネス以外のデータにアクセスするにはアプリ審査(App Review)が必要です。自社ビジネスの広告アカウントのみを連携する場合は、審査なしで進められます。

2. 発行手順

以下の順序で発行し、GenieGo に登録します。

1

ビジネスアカウントの作成・認証

ブラウザでサイトにアクセスしてログインし、[ビジネスを作成]からビジネスアカウントを作成してメール認証を完了します。

business.facebook.com
2

アプリを作成

右上の [マイアプリ] > [アプリを作成] をクリックし、種類は「ビジネス」を選択してアプリを作成します。

developers.facebook.com → マイアプリ → アプリを作成
3

Marketing API を追加

アプリのダッシュボードで「Marketing API」カードを探し、[設定]をクリックして製品として追加します。

4

システムユーザーの作成・権限割り当て

ビジネス設定 > [ユーザー] > [システムユーザー] > [追加]でシステムユーザーを作成し、[アセットを割り当て]で広告アカウントへの権限を付与します。

business.facebook.com/settings → ユーザー → システムユーザー
5

アクセストークンを生成

同じ画面で [トークンを生成] をクリックしてアプリを選択し、権限(ads_read, ads_management)をチェックしてからトークンを生成し、その値をコピーします。

6

広告アカウント ID を確認

広告マネージャで act_ で始まる数値型の広告アカウント ID を確認します。

adsmanager.facebook.com
ads_read、ads_management の権限がチェックされた状態でトークンが生成されれば、正常に発行されています。

3. GenieGo に登録する情報

フィールド説明
access_token発行されたシステムユーザーのアクセストークン(取り扱い注意)
ad_account_idact_ で始まる広告アカウント ID
GenieGo 連携ハブ > 該当チャネルの [登録] にアクセストークンと広告アカウント ID(act_)を貼り付けて保存すると、認証情報は AES-256-GCM で暗号化して保存され、閲覧時にはマスキングされます。

4. よくある問題

問題原因解決方法
権限エラーでデータを取得できないトークン権限(ads_read·ads_management)が未チェック、または広告アカウント権限が未割り当てシステムユーザーに広告アカウント権限を割り当て、トークン生成時に ads_read·ads_management の権限をチェックします。
他社ビジネスのデータへのアクセスが拒否されるアプリ審査(App Review)が未完了自社ビジネス以外のデータにアクセスするには、アプリ審査を申請・完了します。
広告アカウント ID の形式エラーact_ 接頭辞の欠落、または数字のみ入力広告マネージャで act_ で始まる ID 全体を確認して入力します。

最終チェックリスト

  • business.facebook.com でビジネスアカウントを作成し、メール認証を完了した。
  • developers.facebook.com でビジネス種類のアプリを作成し、Marketing API を追加した。
  • システムユーザーを作成し、広告アカウント権限を割り当てた。
  • ads_read·ads_management 権限でアクセストークンを生成・コピーした。
  • act_ で始まる広告アカウント ID を確認した。
  • GenieGo の [登録] に access_token·ad_account_id を入力して保存した。
