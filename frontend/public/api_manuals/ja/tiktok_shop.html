TikTok Shop
API Key 発行マニュアル

TikTok Shop Partner Center でアプリ(App Key·App Secret·Service ID)を作成し、ショップ認可で access_token を発行して GenieGo 連携ハブに登録する手順を、初心者でもそのまま実行できるようにまとめたマニュアルです。

所要時間約 10〜15 分
必要なアカウントTikTok Shop 販売者アカウント
発行するキーApp Key · App Secret · Access Token
成果物API 認証情報

1. はじめる前に

Partner Center でアプリを作成し、ショップを認可してはじめてトークンが発行されます。

必須

TikTok Shop 販売者アカウント

TikTok Shop 販売者アカウントが必要で、partner.tiktokshop.com(Partner Center)に開発者として登録する必要があります。

発行するキー

App Key · App Secret · Access Token

アプリ作成で App Key·App Secret·Service ID を確認し、ショップ認可を経て access_token を発行します。

注意: 本番環境へ移行する際はアプリ審査が必要です。

2. 発行手順

以下の順序で発行し、GenieGo に登録します。

1

Partner Center に開発者登録

partner.tiktokshop.com(Partner Center)に開発者として登録します。

https://partner.tiktokshop.com
2

アプリ作成

[App & Service] > [Create App] でアプリを作成し、App Key、App Secret、Service ID を確認します。

App & Service → Create App
3

API 権限の有効化

アプリ設定で必要な API 権限(Shop Authorized Information、Product、Order)を有効化します。

4

ショップ認可 → access_token 発行

[Get shop authorization] で地域(Region)を選択して販売者ショップを認可すると、access_token が発行されます。

Get shop authorization
5

キーの確認

App Key·App Secret と発行された access_token を確認します。必要に応じて Shop Cipher も確認します。

ショップ認可後に access_token が発行されれば、正常に処理されています。

3. GenieGo に登録する情報

フィールド説明
app_key発行された App Key
app_secret発行された App Secret(取り扱い注意)
access_tokenショップ認可で発行されたアクセストークン
shop_cipherShop Cipher(任意)
GenieGo 連携ハブの [登録] に app_key·app_secret·access_token を入力して保存すると、認証情報は AES-256-GCM で暗号化して保存され、閲覧時にはマスキングされます。

4. よくある問題

問題原因解決方法
access_token が発行されないショップ認可(Get shop authorization)が未完了、または地域(Region)が未選択[Get shop authorization] で地域を選択して販売者ショップを認可します。
権限エラー必要な API 権限が未有効化アプリ設定で Shop Authorized Information·Product·Order の権限を有効化します。
本番呼び出しが拒否される本番移行審査に未通過本番移行に必要なアプリ審査を完了します。

最終チェックリスト

  • partner.tiktokshop.com に開発者として登録した。
  • App & Service > Create App で App Key·App Secret·Service ID を確認した。
  • 必要な API 権限(Shop/Product/Order)を有効化した。
  • ショップ認可(Get shop authorization)で access_token を発行した。
  • GenieGo の [登録] に app_key·app_secret·access_token を入力して保存した。
