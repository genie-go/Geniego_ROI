1. Kakao アルリムトークの発行概念

アルリムトークは、注文・予約・配送・決済などの情報系メッセージをカカオトークで送信するビジネスメッセージサービスです。通常のカカオ開発者アプリキーだけで直接送信できる仕組みではなく、ビジネスチャンネル認証と送信代行会社/プラットフォームの設定が必要です。

重要: アルリムトークは広告系メッセージではなく情報系メッセージが中心であり、テンプレートの事前審査が必要です。

2. 準備物

  • カカオビジネスアカウント
  • カカオトークチャンネル
  • 事業者登録証などのビジネス認証資料
  • 送信するアルリムトークのテンプレート文言
  • 送信代行会社またはクラウドメッセージングサービスのアカウント
  • 受信者電話番号の収集および同意管理ポリシー

3. カカオトークチャンネルの準備

1

カカオビジネスにアクセス

カカオビジネス管理者センターにログインします。

business.kakao.com
2

カカオトークチャンネルの作成

ブランドまたはサービス名でチャンネルを作成します。

カカオトークチャンネル管理者センター
3

ビジネス認証の実施

事業者情報と証明資料を提出してビジネスチャンネル認証を行います。

ビジネス認証
4

チャンネル検索用 ID の確認

アルリムトーク送信設定に必要なチャンネル ID または検索用 ID を確認します。

チャンネルプロフィール → 検索用 ID
ビジネス認証が完了して初めて、アルリムトークの送信プロフィール登録を進められます。

4. 送信プロフィール / Sender Profile 登録

1

送信プラットフォームの選択

NHN Cloud、Naver Cloud SENS、Infobank、Bizgo などのアルリムトーク提供会社を選択します。

Messaging Provider
2

プロジェクトの作成

選択したプラットフォームでアルリムトークプロジェクトを作成します。

Project / Service
3

カカオトークチャンネルの連携

ビジネス認証されたカカオトークチャンネル情報を登録します。

Sender Profile
4

送信プロフィールの検収

チャンネル所有確認と送信プロフィールの検収を行います。

Profile Review
5

API 認証情報の確認

プラットフォームが提供する App Key、Secret Key、API Key などを確認します。

API Key / Secret

5. テンプレート登録および承認

1

テンプレートの作成

注文確認、配送案内などの情報系文言を作成します。

Template Message
2

変数の設定

氏名、注文番号、配送番号など動的に変わる値を変数として設定します。

#{変数}
3

検収依頼

テンプレートを保存し、カカオの検収依頼を行います。

Request Approval
4

承認の確認

承認されたテンプレートコードまたは ID を確認します。

Template Code
検収前の文言と実際の送信文言が異なると、送信失敗または制裁が発生する可能性があります。

6. API 送信フローの例

1) サーバーからアルリムトーク API Endpoint を呼び出し
2) API Key または Secret で認証
3) 受信者電話番号、テンプレートコード、変数値を渡す
4) 送信結果を受信
5) 失敗時に SMS 代替送信を行うかどうかを処理

7. 最終チェックリスト

  • カカオトークチャンネルを作成した。
  • ビジネス認証を完了した。
  • アルリムトーク送信プラットフォームを選択した。
  • Sender Profile を登録した。
  • API Key または Secret を確認した。
  • テンプレートを登録して承認を受けた。
  • 送信失敗および代替メッセージのポリシーを準備した。
