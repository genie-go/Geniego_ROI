Naver Search Ads
API キー発行マニュアル

Naver Search Ad 管理ツールで API 認証情報(アクセスライセンス・シークレットキー)と CUSTOMER_ID を発行し、GenieGo 連携ハブに登録するまでの流れを、初心者でもそのまま実行できるようにまとめたマニュアルです。

所要時間約5〜10分
必要なアカウントNaver Search Ad 広告主アカウント
発行されるキーアクセスライセンス · シークレットキー · CUSTOMER_ID
成果物Search Ad API 認証情報

1. はじめる前に

Naver Search Ad API は広告主アカウントからのみ申請できます。

必須

Search Ad 広告主アカウント

Naver Search Ad 管理ツールを使用する広告主アカウントが必要です。一般の Naver 会員アカウントでは API 利用を申請できません。

発行されるキー

アクセスライセンス · シークレットキー · CUSTOMER_ID

API 利用を申請した後にまとめて発行される CUSTOMER_ID、アクセスライセンス、シークレットキーを確認します。

注意: シークレットキーは発行時にのみ確認できるため、発行後すぐに安全に保管してください。

2. API キー発行手順

以下の順序で発行し、GenieGo に登録します。

1

Search Ads にサインイン

広告主アカウントで Naver Search Ad 管理ツールにサインインします。

searchad.naver.com
2

API 利用管理に移動

上部の[ツール]メニューから[API 利用管理]に移動します。

ツール → API 利用管理
3

API サービスを申請

[Naver Search Ad API サービス申請]ボタンをクリックして申請します。

Naver Search Ad API サービス申請
4

発行された情報を確認

発行された CUSTOMER_ID、アクセスライセンス、シークレットキーを確認します。

CUSTOMER_ID・アクセスライセンス・シークレットキーがすべて確認できれば、正常に発行されています。

3. GenieGo に登録する情報

フィールド説明
api_key発行されたアクセスライセンス
api_secret発行されたシークレットキー(取り扱い注意)
customer_id発行された CUSTOMER_ID
GenieGo 連携ハブの[登録]に入力して保存すると、認証情報は AES-256-GCM で暗号化保存され、参照時にはマスキングされます。

4. よくある問題

問題原因解決
[API 利用管理]メニューがない広告主アカウントではない、または権限不足Search Ad 管理ツールを使用する広告主アカウントでサインインします。
認証エラーが発生するアクセスライセンスまたはシークレットキーの入力ミス発行画面のアクセスライセンス、シークレットキー、CUSTOMER_ID を再確認します。

最終チェックリスト

  • Naver Search Ad 広告主アカウントでサインインした。
  • [ツール > API 利用管理]で API サービス申請を完了した。
  • CUSTOMER_ID・アクセスライセンス・シークレットキーを確認した。
  • GenieGo[登録]に api_key・api_secret・customer_id を入力して保存した。
