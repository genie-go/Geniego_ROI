1. LINE API 発行の概念

LINE 連携は、LINE Developers Console で Provider と Channel を作成し、Messaging API または LINE Login 用の Channel ID、Channel Secret、Channel Access Token を発行する方式です。

区分: メッセージ送信は Messaging API チャネル、ログイン連携は LINE Login チャネルを使用します。

2. 準備するもの

  • LINE アカウントまたはビジネスアカウント
  • LINE Developers Console へのアクセス権限
  • LINE Official Account
  • サービス名と説明
  • Webhook URL
  • プライバシーポリシー URL および利用規約 URL

3. Provider の作成

1

LINE Developers にアクセス

LINE Developers Console にログインします。

developers.line.biz
2

Provider を作成

会社名またはサービス名で Provider を作成します。

Create Provider
3

基本情報を入力

サービス運営主体、連絡先、規約情報を入力します。

Provider Settings

4. Messaging API チャネルの作成

1

新規チャネルを作成

Provider 内で Create a new channel をクリックします。

Create a new channel
2

Messaging API を選択

チャネルの種類で Messaging API を選択します。

Messaging API
3

チャネル情報を入力

チャネル名、説明、カテゴリ、メールなどを入力します。

Channel Information
4

LINE Official Account を連携

メッセージを送信する公式アカウントを連携するか、新規に作成します。

LINE Official Account
5

チャネル作成完了

作成後に Channel ID と Channel Secret を確認します。

Channel ID / Channel Secret

5. Channel Access Token の発行

1

Messaging API タブへ移動

作成したチャネル画面で Messaging API タブに移動します。

Messaging API tab
2

Channel access token 項目を探す

下部の Channel access token 領域を確認します。

Channel access token
3

Issue をクリック

Issue または Reissue ボタンを押してトークンを発行します。

Issue
4

サーバーに安全に保存

発行されたトークンはサーバーの環境変数や Secret Manager に保存します。

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token はメッセージ送信権限を持つ秘密鍵です。HTML、フロントエンド、公開リポジトリに公開してはいけません。

6. Webhook の設定

1

Webhook URL を入力

ユーザーのメッセージイベントを受け取るサーバー URL を入力します。

Webhook URL
2

Use webhook を有効化

Webhook 使用オプションを Enabled に変更します。

Use webhook: Enabled
3

Verify テスト

LINE からサーバーへテストリクエストを送り、接続を確認します。

Verify
4

応答コードを確認

サーバーが 200 OK を返すか確認します。

HTTP 200 OK

7. メッセージ送信の例

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "こんにちは。LINE メッセージのテストです。"} ] }

8. 最終チェックリスト

  • Provider を作成した。
  • Messaging API チャネルを作成した。
  • LINE Official Account を連携した。
  • Channel ID と Channel Secret を確認した。
  • Channel Access Token を発行した。
  • Webhook URL を登録し、Verify を通過した。
  • トークンをサーバーの環境変数に安全に保存した。
