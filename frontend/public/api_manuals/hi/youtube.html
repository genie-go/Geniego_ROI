1. API Key और OAuth में अंतर

शुरुआत करते समय लोग सबसे ज़्यादा इसी हिस्से को लेकर भ्रमित होते हैं।

API Key

सार्वजनिक डेटा क्वेरी करने के लिए

सार्वजनिक YouTube डेटा प्राप्त करने के लिए उपयोग किया जाता है, जैसे वीडियो खोजना, चैनल की जानकारी देखना, प्लेलिस्ट प्राप्त करना और टिप्पणियाँ पढ़ना।

OAuth 2.0

उन कार्यों के लिए जिनमें आपके खाते की अनुमतियाँ चाहिए

उन कार्यों के लिए उपयोग किया जाता है जिनमें उपयोगकर्ता की स्वीकृति आवश्यक होती है, जैसे वीडियो अपलोड करना, अपना चैनल प्रबंधित करना और निजी डेटा तक पहुँच।

महत्वपूर्ण: वीडियो स्वचालित रूप से अपलोड करने के लिए केवल API Key पर्याप्त नहीं है; OAuth 2.0 प्रमाणीकरण आवश्यक है।

2. YouTube Data API v3 सक्षम करें

API Key बनाने से पहले, आपको पहले अपने प्रोजेक्ट में वह API चालू करनी होगी जिसका आप उपयोग करना चाहते हैं।

1

Google Cloud Console तक पहुँचें

Google Cloud Console में साइन इन करें।

console.cloud.google.com
2

प्रोजेक्ट चुनें या बनाएँ

ऊपर दिए प्रोजेक्ट चयनकर्ता में, कोई मौजूदा प्रोजेक्ट चुनें या नया प्रोजेक्ट बनाएँ।

उदा., Geniego ROI
3

API लाइब्रेरी पर जाएँ

बाएँ मेन्यू में, API और सेवाएँ → लाइब्रेरी पर जाएँ।

API और सेवाएँ → लाइब्रेरी
4

YouTube Data API v3 खोजें और सक्षम करें

खोज बॉक्स में YouTube Data API v3 टाइप करें, उस API को चुनें और सक्षम करें बटन पर क्लिक करें।

YouTube Data API v3 → सक्षम करें
यदि स्क्रीन पर स्थिति: सक्षम दिखाई देता है, तो यह सही ढंग से सक्षम हो चुकी है।

3. एक API Key बनाएँ

YouTube Data API v3 सक्षम हो जाने पर, आप क्रेडेंशियल स्क्रीन पर एक API Key बना सकते हैं।

1

क्रेडेंशियल मेन्यू पर जाएँ

बाएँ मेन्यू में, API और सेवाएँ → क्रेडेंशियल पर जाएँ।

API और सेवाएँ → क्रेडेंशियल
2

क्रेडेंशियल बनाएँ पर क्लिक करें

ऊपर दिए + क्रेडेंशियल बनाएँ बटन पर क्लिक करें।

+ क्रेडेंशियल बनाएँ
3

API कुंजी चुनें

दिखाई देने वाले मेन्यू में, API कुंजी चुनें।

API कुंजी
4

सुलभ API चुनें

API प्रतिबंध के अंतर्गत, पुष्टि करें कि YouTube Data API v3 चयनित है।

चयनित API: YouTube Data API v3
5

बनाएँ बटन पर क्लिक करें

API Key उत्पन्न करने के लिए नीचे दिए बनाएँ बटन पर क्लिक करें।

4. निर्माण स्क्रीन पर आपको कौन-से विकल्प चुनने चाहिए?

आइटमशुरुआती लोगों के लिए अनुशंसित सेटिंगविवरण
नामYouTube API Keyऐसा नाम दर्ज करें जिसे बाद में पहचानना आसान हो।
API प्रतिबंधYouTube Data API v3इस कुंजी को प्रतिबंधित करें ताकि यह केवल YouTube API के साथ ही उपयोग हो।
सेवा खाता प्रमाणीकरणअनचेक रहने देंYouTube Data API Key बनाने के लिए आमतौर पर आवश्यक नहीं।
एप्लिकेशन प्रतिबंधप्रारंभिक परीक्षण के लिए कोई नहींयह सही ढंग से काम करता है इसकी पुष्टि के बाद, हम वेबसाइट या IP पते के प्रतिबंध सेट करने की सलाह देते हैं।
किसी वास्तविक सेवा में परिनियोजित करते समय, कुंजी प्रतिबंध अवश्य सेट करें। यदि इसे बिना प्रतिबंध के उजागर किया जाता है, तो अन्य लोग आपकी कुंजी का उपयोग कर सकते हैं।

5. API Key उपयोग के उदाहरण

आप अपनी उत्पन्न की गई कुंजी को YOUR_API_KEY भाग में रखकर परीक्षण कर सकते हैं।

वीडियो खोज उदाहरणhttps://www.googleapis.com/youtube/v3/search?part=snippet&q=korea&type=video&key=YOUR_API_KEY
चैनल जानकारी देखने का उदाहरणhttps://www.googleapis.com/youtube/v3/channels?part=snippet&id=CHANNEL_ID&key=YOUR_API_KEY
यदि अपने ब्राउज़र के पता बार में ऊपर दिया URL दर्ज करने पर JSON प्रारूप में परिणाम दिखाई देता है, तो API Key सही ढंग से काम कर रही है।

6. सामान्य समस्याओं का समाधान

समस्याकारणसमाधान

प्रदर्शित करने के लिए कोई API कुंजी नहींआपने अभी तक कोई API Key नहीं बनाई हैक्रेडेंशियल बनाएँ → API कुंजी पर क्लिक करें।
403 त्रुटिAPI प्रतिबंध, कोटा पार हो जाना, या अनुमति संबंधी समस्याएँAPI प्रतिबंध, कोटा और सक्षम स्थिति की जाँच करें।
अपलोड काम नहीं कर रहेकेवल API Key का उपयोगवीडियो अपलोड के लिए OAuth 2.0 प्रमाणीकरण का उपयोग करना आवश्यक है।
