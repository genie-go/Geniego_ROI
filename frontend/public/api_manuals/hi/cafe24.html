API Key जारी करने की नियमावली

Cafe24 Developer Center में एक ऐप पंजीकृत करके Client ID·Client Secret जारी करने, OAuth 2.0 प्राधिकरण के माध्यम से टोकन प्राप्त करने और उन्हें GenieGo Integration Hub में पंजीकृत करने के लिए शुरुआती-अनुकूल चरण-दर-चरण मार्गदर्शिका।

अनुमानित समयलगभग 10–15 मिनट
आवश्यक खाताCafe24 स्टोर संचालक खाता
जारी की गई कुंजियाँClient ID · Client Secret · Refresh Token
परिणामOAuth 2.0 क्रेडेंशियल

1. शुरू करने से पहले

Cafe24 एकीकरण डेवलपर पंजीकरण और ऐप पंजीकरण से होकर गुजरता है, और OAuth 2.0 के माध्यम से प्रमाणित करता है।

आवश्यक

स्टोर संचालक खाता

आपको एक संचालक खाता चाहिए जो Cafe24 स्टोर चलाता हो। इस खाते से आप Developer Center में डेवलपर पंजीकरण पूरा करते हैं।

जारी की गई कुंजियाँ

Client ID · Client Secret + टोकन

ऐप पंजीकृत करने पर एक Client ID और Client Secret जारी होते हैं, और आप OAuth 2.0 प्राधिकरण के माध्यम से एक access token और refresh token प्राप्त करते हैं।

नोट: GenieGo में आप long-lived refresh_token पंजीकृत करते हैं, और access token उसके माध्यम से स्वचालित रूप से ताज़ा हो जाता है।

2. ऐप पंजीकरण और टोकन जारी करने के चरण

नीचे दिए गए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

डेवलपर के रूप में पंजीकरण करें

Cafe24 Developer Center पर जाएँ और डेवलपर के रूप में पंजीकरण करें।

developer.cafe24.com
2

ऐप पंजीकृत करें

[ऐप्स] > [ऐप पंजीकृत करें] के माध्यम से एक ऐप बनाएँ, और एक Client ID और Client Secret जारी होते हैं।

ऐप्स → ऐप पंजीकृत करें
3

OAuth 2.0 प्राधिकरण

OAuth 2.0 प्राधिकरण के माध्यम से एक access token और refresh token जारी करें।

4

अपनी जानकारी जाँचें

mall_id (मॉल ID), Client ID, Client Secret और refresh token जाँचें।

जब Client ID·Client Secret और refresh token सभी पुष्ट हो जाएँ, तो वे सही ढंग से जारी हो गए हैं।

3. GenieGo में पंजीकृत करने योग्य जानकारी

फ़ील्डविवरण
mall_idCafe24 स्टोर ID
client_idजारी की गई Client ID
client_secretजारी किया गया Client Secret (सावधानी से संभालें)
refresh_tokenOAuth 2.0 प्राधिकरण के माध्यम से प्राप्त Refresh Token
इन्हें GenieGo Integration Hub [पंजीकृत करें] में दर्ज करें और सहेजें, और क्रेडेंशियल AES-256-GCM के साथ संग्रहण के समय एन्क्रिप्ट किए जाते हैं और पढ़ते समय मास्क किए जाते हैं।

4. सामान्य समस्याएँ

समस्याकारणसमाधान
प्राधिकरण विफलतागलत mall_id या ऐप सेटिंग्स में बेमेलऐप पंजीकरण के दौरान निर्धारित mall_id और Client जानकारी को फिर से जाँचें।
टोकन समाप्ति त्रुटिrefresh_token गलत दर्ज किया गयाOAuth 2.0 प्राधिकरण के माध्यम से प्राप्त नवीनतम refresh_token को फिर से पंजीकृत करें।

अंतिम जाँच सूची

  • Cafe24 Developer Center में डेवलपर के रूप में पंजीकरण किया।
  • [ऐप्स > ऐप पंजीकृत करें] के माध्यम से एक ऐप बनाया और एक Client ID·Client Secret जारी किया।
  • OAuth 2.0 प्राधिकरण के माध्यम से एक access token·refresh token जारी किया।
  • GenieGo [पंजीकृत करें] में mall_id·client_id·client_secret·refresh_token दर्ज किया और सहेजा।
