1. Amazon SP-API क्या है?

Amazon Selling Partner API ऑर्डर, इन्वेंट्री, उत्पाद, शिपिंग और निपटान जैसे विक्रेता डेटा को प्रोग्रामेटिक रूप से एकीकृत करने के लिए एक API है।

महत्वपूर्ण: SP-API कोई साधारण API Key योजना नहीं है। इसके लिए आम तौर पर एक साथ Amazon Developer पंजीकरण, ऐप पंजीकरण, Login with Amazon(LWA) और OAuth प्राधिकरण की आवश्यकता होती है।
★GenieGo एकीकरण के लिए 5 मान आवश्यक हैं: refresh_token, client_id, client_secret, marketplace_id, seller_id। (वर्तमान SP-API केवल LWA टोकन के साथ कॉल की जाती है, और AWS IAM/SigV4 हस्ताक्षर GenieGo द्वारा आंतरिक रूप से संभाला जाता है, इसलिए आप इसे पंजीकरण फ़ील्ड में दर्ज नहीं करते।)

2. पूर्वापेक्षाएँ

  • एक Amazon Seller Central या Vendor Central Professional विक्रेता खाता
  • डेवलपर पंजीकरण के लिए उपयोग की जाने वाली कंपनी/संपर्क जानकारी
  • ऐप का नाम, ऐप का विवरण और इच्छित उपयोग
  • गोपनीयता नीति URL
  • OAuth Redirect URL

3. डेवलपर पंजीकरण

सबसे पहले, Seller Central में अपनी SP-API डेवलपर प्रोफ़ाइल पंजीकृत करें।

1

Seller Central में साइन इन करें

अपने Amazon विक्रेता खाते से Seller Central में साइन इन करें।

Seller Central → Apps & Services
2

Develop Apps मेनू पर जाएँ

Apps & Services मेनू से, डेवलपर या ऐप डेवलपमेंट मेनू पर जाएँ।

Apps & Services → Develop Apps
3

Developer Profile भरें

अपनी कंपनी की जानकारी, संपर्क जानकारी, API उपयोग का उद्देश्य और डेटा सुरक्षा योजना दर्ज करें।

Developer Profile
4

समीक्षा के लिए सबमिट करें

Amazon की समीक्षा के लिए अपनी डेवलपर पंजीकरण जानकारी सबमिट करें।

Submit for review
अनुमोदन पूरा होने के बाद ही आप एक प्रोडक्शन SP-API ऐप पंजीकृत कर सकते हैं और उसकी अनुमतियों का उपयोग कर सकते हैं।

4. एक SP-API ऐप पंजीकृत करें → Client ID / Client Secret

1

एक नया एप्लिकेशन बनाएँ

Develop Apps में एक नया ऐप बनाएँ।

Add new app
2

ऐप प्रकार चुनें

यदि आप केवल अपना खाता जोड़ते हैं तो यह Private App है; यदि आप इसे बाहरी विक्रेताओं को प्रदान करते हैं तो यह Public App है।

Private / Public App
3

API भूमिकाएँ चुनें

Orders, Catalog और Inventory जैसी आवश्यक भूमिकाएँ चुनें।

API Roles
4

OAuth Redirect URL दर्ज करें

प्राधिकरण पूरा होने के बाद लौटने के लिए सर्वर URL दर्ज करें।

Redirect URI
5

सहेजें और Client जानकारी जाँचें

ऐप बनाने के बाद, client_id (LWA Client ID) और client_secret (Client Secret) की जाँच करें।

client_id / client_secret

5. OAuth प्राधिकरण → Refresh Token जारी करें

1) विक्रेता OAuth प्राधिकरण URL तक पहुँचें 2) Redirect URL पर authorization code प्राप्त करें 3) authorization code को LWA Token Endpoint पर भेजें 4) refresh_token / access_token प्राप्त करें
refresh_token का उपयोग विक्रेता प्राधिकरण के बाद दीर्घकालिक कॉल के लिए किया जाता है। GenieGo SP-API को कॉल करने के लिए इस refresh_token का उपयोग करके access_token को स्वचालित रूप से रीफ़्रेश करता है, इसलिए आपको किसी अलग AWS हस्ताक्षर सेटअप के बिना केवल refresh_token पंजीकृत करने की आवश्यकता है।

6. Marketplace ID · Seller ID जाँचें

GenieGo पंजीकरण के लिए आवश्यक शेष दो पहचानकर्ताओं की जाँच करें।

marketplace_id

लक्ष्य मार्केटप्लेस ID

जिस देश को आप जोड़ रहे हैं उसका Marketplace ID। US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

उत्पाद पंजीकरण और ऑर्डर खोज के लिए आवश्यक। इसे अपनी Seller Central खाता जानकारी में जाँचें। Seller Central → सेटिंग्स → खाता जानकारी → Merchant Token

1

5 मान तैयार करें

refresh_token · client_id · client_secret · marketplace_id · seller_id तैयार करें।

5 क्रेडेंशियल
2

Integration Hub → Amazon SP-API → [पंजीकृत करें]

प्रत्येक मान को उसके फ़ील्ड में दर्ज करें और सहेजें (खाली फ़ील्ड मौजूदा मान बनाए रखते हैं)।

refresh_token · client_id · client_secret · marketplace_id · seller_id
पंजीकृत मान AES-256-GCM के साथ एन्क्रिप्ट करके संग्रहीत किए जाते हैं और पढ़ते समय मास्क किए जाते हैं। सहेजने के तुरंत बाद स्वचालित सिंक शुरू हो जाती है।
client_secret·refresh_token को कभी भी फ्रंटएंड कोड, HTML या सार्वजनिक रिपॉजिटरी में उजागर नहीं किया जाना चाहिए।

8. अंतिम जाँच सूची

  • Developer Profile सबमिट किया और उसे अनुमोदित करवाया।
  • SP-API ऐप बनाया और client_id·client_secret की जाँच की।
  • OAuth प्राधिकरण के माध्यम से refresh_token जारी किया।
  • जोड़ने के लिए marketplace_id की जाँच की।
  • Seller Central में seller_id (Merchant Token) की जाँच की।
  • GenieGo [पंजीकृत करें] में 5 मान (refresh_token·client_id·client_secret·marketplace_id·seller_id) दर्ज किए और सहेजे।
