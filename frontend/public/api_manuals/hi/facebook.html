API जारी करने की मार्गदर्शिका

Meta for Developers पर एक ऐप बनाने और Facebook पेज लाइव कनेक्शन के लिए आवश्यक App ID, App Secret तथा पेज एक्सेस टोकन (Page Access Token) तैयार करने की, शुरुआती लोगों के लिए अनुकूल मार्गदर्शिका।

मुख्य अवधारणाAPI Key नहीं, बल्कि Access Token
डेवलपर साइटdevelopers.facebook.com

1. सबसे पहले जानने योग्य मुख्य बातें

YouTube में एक ही API Key से काम चल जाता है, लेकिन Facebook (Meta) App ID · App Secret · Access Token संरचना के साथ काम करता है।

महत्वपूर्ण: Meta में "एक API Key कॉपी करना" से काम पूरा नहीं होता। आमतौर पर इस क्रम की आवश्यकता होती है: ऐप बनाना → उत्पाद जोड़ना → अनुमतियाँ सेट करना → Page Access Token जारी करना → ऐप समीक्षा।

2. आवश्यक चीज़ें

  • एक Meta/Facebook खाता
  • Meta for Developers तक पहुँच
  • एक Facebook पेज जिस पर आपके पास एडमिन अधिकार हों
  • वास्तविक सेवा के लिए गोपनीयता नीति URL · सेवा की शर्तें URL

3. Meta Developer ऐप बनाना

हर Meta API एकीकरण एक ऐप बनाने से शुरू होता है।

1

Meta for Developers पर पहुँचें

अपने ब्राउज़र में Meta for Developers में साइन इन करें।

developers.facebook.com
2

मेरे ऐप → ऐप बनाएँ

ऊपर My Apps → Create App पर क्लिक करें।

My Apps → Create App
3

ऐप प्रकार चुनें

प्लेटफ़ॉर्म एकीकरण/लाइव कॉमर्स व्यावसायिक उद्देश्य के अधिक निकट है।

Business प्रकार चुनें
4

ऐप नाम·ईमेल दर्ज करें → बनाएँ

ऐप का नाम और संपर्क ईमेल दर्ज करें, फिर ऐप बनाएँ।

App Name / Contact Email
ऐप बनाने के बाद, डैशबोर्ड → ऐप सेटिंग्स → मूल सेटिंग्स में App ID और App Secret देखें। (App Secret को फ्रंटएंड/सार्वजनिक रिपॉज़िटरी में कभी उजागर न करें।)

4. Facebook Live API सेटअप प्रवाह

1

उत्पाद जोड़ें

ऐप डैशबोर्ड में Facebook Login / Graph API से संबंधित सेटिंग्स तैयार करें।

Add Product → Facebook Login
2

पेज कनेक्ट करें + पेज टोकन

किसी पेज पर लाइव प्रसारण करने के लिए, आपको पेज एडमिन अधिकार और Page Access Token की आवश्यकता होती है।

Page Access Token
3

आवश्यक अनुमतियों के लिए आवेदन करें (App Review)

सामग्री प्रकाशित करने·पेज प्रबंधित करने·लाइव वीडियो बनाने के उद्देश्य से अनुमतियों (जैसे pages_read_engagement, pages_manage_posts) के लिए आवेदन करें।

Permissions → App Review
4

Graph API Explorer से टोकन जारी·परीक्षण करें

अनुमतियों सहित एक टोकन जारी करें और API कॉल का परीक्षण करें।

Tools → Graph API Explorer
Facebook Live निर्माण कॉल का उदाहरण
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
सफलतापूर्वक बनने पर, प्रसारण ID·स्ट्रीम URL·स्ट्रीम कुंजी जैसी लाइव प्रसारण जानकारी लौटाई जाती है। पेज ID (page_id) पेज जानकारी में देखें।

5. GenieGo Integration Hub में पंजीकृत करें

1

Page Access Token कॉपी करें

आपके द्वारा जारी किया गया Page Access Token कॉपी करें (दीर्घकालिक टोकन अनुशंसित)।

access_token (Page token)
2

पेज ID की पुष्टि करें

लक्षित Facebook पेज की ID (page_id) की पुष्टि करें।

page_id
3

Integration Hub में पंजीकृत करें

GenieGo Integration Hub → Facebook Live → [पंजीकृत करें] में नीचे दिए गए मान दर्ज करें और सहेजें।

access_token · page_id
पंजीकृत कुंजियाँ AES-256-GCM के साथ विश्राम में एन्क्रिप्ट की जाती हैं और पढ़ते समय मास्क की जाती हैं।

6. Access Token के प्रकार

टोकन प्रकारउपयोगध्यान देने योग्य बातें
User Access Tokenउपयोगकर्ता लॉगिन पर आधारित API कॉलजिन डेटा तक पहुँच संभव है वे प्रदान की गई अनुमतियों के दायरे के अनुसार बदलते हैं।
Page Access Tokenपेज प्रकाशन · पेज लाइव निर्माणपेज एडमिन अधिकार और संबंधित अनुमतियाँ आवश्यक हैं।
Long-Lived Tokenविस्तारित समाप्ति वाला दीर्घकालिक टोकनसंचालन के समय समाप्ति·नवीनीकरण लॉजिक आवश्यक है।

7. चेकलिस्ट

  • Meta Developer ऐप बनाया।
  • App ID और App Secret की पुष्टि की।
  • Facebook पेज कनेक्ट किया और Page Access Token जारी किया।
  • आवश्यक अनुमतियाँ (App Review) व्यवस्थित·आवेदित कीं।
  • GenieGo [पंजीकृत करें] में access_token·page_id दर्ज किया और सहेजा।
  • टोकन को सर्वर पर्यावरण चर में सुरक्षित रूप से संग्रहीत किया।
