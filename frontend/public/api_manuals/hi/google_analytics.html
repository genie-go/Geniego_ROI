API Key जारी करने की नियमावली

आपकी Google Analytics 4 डेटा स्ट्रीम से मेज़रमेंट ID (G-) और Measurement Protocol API Secret जारी करने और उन्हें GenieGo Integration Hub में पंजीकृत करने की, शुरुआती-अनुकूल चरण-दर-चरण मार्गदर्शिका।

1. शुरू करने से पहले

GA4 मेज़रमेंट ID और API Secret केवल प्रॉपर्टी प्रबंधन अनुमति वाले खाते से ही जारी किए जा सकते हैं।

आवश्यक

GA4 प्रॉपर्टी प्रबंधन अनुमति

आपके पास Google Analytics 4 प्रॉपर्टी को प्रबंधित करने की अनुमति होनी चाहिए। केवल देखने की अनुमति से API Secret नहीं बनाया जा सकता।

जारी की गई कुंजियाँ

मेज़रमेंट ID (G-) · API Secret

वेब स्ट्रीम की मेज़रमेंट ID (G- से शुरू) और Measurement Protocol API Secret दोनों की जाँच करें।

2. जारी करने के चरण

नीचे दिए गए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

डेटा स्ट्रीम पर जाएँ

Google Analytics में [Admin] > [Data Streams] पर जाएँ।

analytics.google.com → Admin → Data Streams
2

मेज़रमेंट ID ढूँढें

वेब स्ट्रीम चुनें और मेज़रमेंट ID (G- से शुरू) ढूँढें।

3

API Secret बनाएँ

उसी स्क्रीन पर, एक [Measurement Protocol API secret] बनाएँ।

जब मेज़रमेंट ID (G-) और API Secret दोनों की पुष्टि हो जाए, तो वे सही ढंग से जारी हो गए हैं।

3. GenieGo में पंजीकृत करने योग्य जानकारी

फ़ील्डविवरण
measurement_idमेज़रमेंट ID (G- से शुरू)
api_secretबनाया गया Measurement Protocol API Secret (सावधानी से संभालें)
इन्हें GenieGo Integration Hub के [Register] में दर्ज करें और सहेजें। क्रेडेंशियल को संग्रहण के समय AES-256-GCM से एन्क्रिप्ट किया जाता है और पढ़ते समय मास्क किया जाता है।

4. सामान्य समस्याएँ

समस्याकारणसमाधान
API Secret नहीं बनाया जा सकताप्रॉपर्टी प्रबंधन अनुमति अपर्याप्तGA4 प्रॉपर्टी प्रबंधन अनुमति वाले खाते से साइन इन करें।
डेटा एकत्र नहीं हो रहामेज़रमेंट ID या API Secret गलत दर्जवेब स्ट्रीम की मेज़रमेंट ID (G-) और API Secret को फिर से जाँचें।

अंतिम जाँच-सूची

  • GA4 प्रॉपर्टी प्रबंधन अनुमति वाले खाते से साइन इन किया।
  • [Admin > Data Streams] में वेब स्ट्रीम की मेज़रमेंट ID (G-) ढूँढी।
  • Measurement Protocol API Secret बनाया।
  • GenieGo [Register] में measurement_id·api_secret दर्ज करके सहेजा।
