1. Yahoo! Japan API जारी करने की अवधारणा

सेवा के प्रकार के आधार पर, Yahoo! Japan परिवार की API, Client ID, Client Secret और OAuth प्रमाण-पत्र Yahoo! JAPAN Developers या LY/Yahoo!広告 Developer Center से जारी करती हैं।

महत्वपूर्ण: "Yahoo! Japan" में, आप जिस API का उपयोग करना चाहते हैं उसके आधार पर जारी करने की साइट और समीक्षा प्रक्रिया अलग-अलग होती है। आपको Shopping, Advertising, Login और Search API के बीच अंतर करना होगा।

2. आवश्यक सामग्री

  • Yahoo! JAPAN ID या व्यावसायिक खाता
  • उपयोग की जाने वाली API का प्रकार तय करें
  • ऐप का नाम और सेवा विवरण
  • Redirect URI
  • गोपनीयता नीति URL
  • उत्पादन डोमेन या परीक्षण डोमेन

3. एप्लिकेशन पंजीकरण का बुनियादी क्रम

1

डेवलपर साइट तक पहुँचें

Yahoo! JAPAN Developers या संबंधित API के Developer Center में साइन इन करें।

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

एप्लिकेशन बनाएँ

नया एप्लिकेशन जोड़ने वाले मेनू पर क्लिक करें।

Create Application / Add Application
3

ऐप जानकारी दर्ज करें

ऐप का नाम, विवरण, ज़िम्मेदार व्यक्ति की जानकारी और वेबसाइट URL दर्ज करें।

Application Information
4

Redirect URI दर्ज करें

OAuth प्रमाणीकरण के बाद लौटने वाला URL ठीक-ठीक दर्ज करें।

Redirect URI
5

अनुमतियाँ या API चुनें

जिस API की अनुमतियाँ और स्कोप का उपयोग करना है उन्हें चुनें।

Scope / API Permission
6

पंजीकरण पूरा करें

पंजीकरण के बाद Client ID और Client Secret की जाँच करें।

Client ID / Client Secret

5. OAuth टोकन जारी करने का प्रवाह

1) Client ID से उपयोगकर्ता प्रमाणीकरण URL बनाएँ 2) उपयोगकर्ता अपने Yahoo! खाते से लॉगिन करता है 3) Redirect URI पर authorization code प्राप्त करें 4) authorization code + Client Secret से Access Token का अनुरोध करें 5) Access Token से API को कॉल करें 6) आवश्यकता होने पर Refresh Token से नवीनीकरण करें

6. अक्सर आने वाली समस्याएँ

समस्याकारणसमाधान
Redirect URI त्रुटिपंजीकृत URL और कॉल किए गए URL में अंतरप्रोटोकॉल, डोमेन और पथ तक समान दर्ज करें
Client Secret नहीं देखा जा सकतासुरक्षा कारणों से छिपा हुआप्रबंधन स्क्रीन में View या पुनः जारी करने का फ़ंक्शन जाँचें
अपर्याप्त अनुमतियाँआवश्यक Scope नहीं चुना गयाऐप सेटिंग्स में Scope जोड़ें और फिर से प्रमाणित करें
API कॉल विफलउपयोग की गई API और जारी किए गए ऐप प्रकार में बेमेलप्रत्येक API के डेवलपर दस्तावेज़ जाँचें

7. अंतिम जाँच सूची

  • उपयोग की जाने वाली Yahoo! Japan API का प्रकार तय कर लिया है।
  • सही डेवलपर कंसोल में साइन इन कर लिया है।
  • एप्लिकेशन पंजीकृत कर लिया है।
  • Client ID और Client Secret की जाँच कर ली है।
  • Redirect URI सही-सही पंजीकृत कर लिया है।
  • आवश्यक Scope चुन लिया है।
  • Access Token जारी करने का परीक्षण पूरा कर लिया है।
