1. शुरू करने से पहले

Fulfillment (3PL) एक अनुबंध-आधारित चैनल है, जहाँ इनबाउंड/आउटबाउंड कंसाइनमेंट अनुबंध के बाद क्रेडेंशियल जारी किए जाते हैं।

आवश्यक

इनबाउंड/आउटबाउंड कंसाइनमेंट अनुबंध

3PL फ़ुलफ़िलमेंट का उपयोग करने के लिए, सबसे पहले किसी फ़ुलफ़िलमेंट प्रदाता या ऑपरेशन टीम के साथ इनबाउंड/आउटबाउंड कंसाइनमेंट अनुबंध होना चाहिए।

जारी की गई कुंजियाँ

3PL API key · warehouse ID

अनुबंध पूरा होने के बाद, आपको warehouse ID और 3PL API key जारी की जाती है।

नोट: यह चैनल सेल्फ-सर्विस कंसोल से सीधे क्रेडेंशियल जारी नहीं करता; इसके बजाय, आप कंसाइनमेंट अनुबंध के माध्यम से क्रेडेंशियल प्राप्त करते हैं।

2. जारी करने के अनुरोध की प्रक्रिया

नीचे दिए गए क्रम में अनुरोध करें और प्राप्त करें, फिर GenieGo में पंजीकृत करें।

1

कंसाइनमेंट अनुबंध आगे बढ़ाएँ

किसी फ़ुलफ़िलमेंट प्रदाता या ऑपरेशन टीम के साथ इनबाउंड/आउटबाउंड कंसाइनमेंट अनुबंध आगे बढ़ाएँ।

2

अनुबंध पर हस्ताक्षर करें

3PL फ़ुलफ़िलमेंट इनबाउंड/आउटबाउंड कंसाइनमेंट अनुबंध पर हस्ताक्षर करें।

3

क्रेडेंशियल प्राप्त करें

अनुबंध पूरा होने के बाद, आपको warehouse ID·3PL API key जारी की जाती है।

4

GenieGo में पंजीकृत करें

जारी की गई api_key, warehouse_id को GenieGo [पंजीकरण] में दर्ज करें और सहेजें।

यदि आपको फ़ुलफ़िलमेंट प्रदाता से warehouse ID·3PL API key दोनों प्राप्त हो गए हैं, तो वे सही ढंग से जारी हुए हैं।

3. GenieGo में पंजीकृत की जाने वाली जानकारी

फ़ील्डविवरण
api_keyजारी की गई 3PL API key
warehouse_idजारी की गई warehouse ID
जब आप इन्हें GenieGo Integration Hub [पंजीकरण] में दर्ज करके सहेजते हैं, तो क्रेडेंशियल AES-256-GCM के साथ संग्रहण के समय एन्क्रिप्ट किए जाते हैं और पढ़ते समय मास्क किए जाते हैं।

4. सामान्य समस्याएँ

समस्याकारणसमाधान
क्रेडेंशियल प्राप्त नहीं हुएइनबाउंड/आउटबाउंड कंसाइनमेंट अनुबंध पूरा नहीं हुआकिसी फ़ुलफ़िलमेंट प्रदाता या ऑपरेशन टीम के साथ कंसाइनमेंट अनुबंध पर हस्ताक्षर करें, फिर क्रेडेंशियल जारी करवाएँ।
इनबाउंड/आउटबाउंड एकीकरण त्रुटिwarehouse ID (warehouse_id) गलत दर्ज की गईजारी की गई warehouse ID को फिर से जाँचें और सही ढंग से दर्ज करें।

अंतिम चेकलिस्ट

  • अनुबंध पूरा होने के बाद warehouse ID·3PL API key प्राप्त की।
  • api_key·warehouse_id की पुष्टि की।
  • GenieGo [पंजीकरण] में api_key·warehouse_id दर्ज करके सहेजा।
