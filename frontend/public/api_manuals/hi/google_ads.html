1. शुरू करने से पहले

Google Ads एकीकरण के लिए मैनेजर (MCC) खाते और Google Cloud में OAuth सेटअप दोनों की आवश्यकता होती है।

developer token

MCC API Center

अपने Google Ads मैनेजर (MCC) खाते के API Center में developer token के लिए आवेदन करें। बेसिक (Basic) एक्सेस Google समीक्षा के बाद स्वीकृत होता है।

OAuth

Google Cloud client

console.cloud.google.com पर, एक OAuth client (client_id·client_secret) बनाएँ और refresh token प्राप्त करने के लिए प्रमाणित करें।

नोट: developer token के लिए बेसिक एक्सेस Google समीक्षा के बाद स्वीकृत होता है, और स्वीकृति से पहले केवल परीक्षण खातों को ही कॉल किया जा सकता है। यदि आपके पास मैनेजर (MCC) खाता नहीं है, तो पहले ads.google.com पर एक बनाएँ।

2. जारी करने के चरण

नीचे दिए गए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

मैनेजर खाते में साइन इन करें · API Center पर जाएँ

अपने Google Ads मैनेजर खाते में साइन इन करें, फिर शीर्ष पर [टूल और सेटिंग] > [सेटअप] > [API Center] पर जाएँ।

टूल और सेटिंग → सेटअप → API Center
2

developer token के लिए आवेदन करें

API Center में [developer token] के लिए आवेदन करें। बेसिक (Basic) एक्सेस Google समीक्षा के बाद स्वीकृत होता है, और स्वीकृति से पहले केवल परीक्षण खातों को ही कॉल किया जा सकता है।

3

Cloud प्रोजेक्ट सेट करें · OAuth सहमति स्क्रीन

एक प्रोजेक्ट बनाएँ और [API और सेवाएँ] > [OAuth सहमति स्क्रीन] कॉन्फ़िगर करें।

console.cloud.google.com → API और सेवाएँ → OAuth सहमति स्क्रीन
4

OAuth client ID बनाएँ

client_id और client_secret प्राप्त करने के लिए [क्रेडेंशियल] > [क्रेडेंशियल बनाएँ] > [OAuth client ID] बनाएँ (scope: adwords)।

क्रेडेंशियल → क्रेडेंशियल बनाएँ → OAuth client ID
5

refresh token जारी करें

OAuth प्रमाणीकरण प्रक्रिया के माध्यम से refresh token प्राप्त करें (Google OAuth Playground या अपने स्वयं के प्रमाणीकरण का उपयोग करें)।

6

customer ID खोजें

उस विज्ञापन खाते का 10-अंकीय customer ID (customer_id) खोजें जिसे आप कॉल करेंगे।

जब developer token स्वीकृत हो जाए और client_id·client_secret·refresh_token·customer_id सभी तैयार हों, तो जारी करना पूर्ण हो जाता है।

3. GenieGo में पंजीकृत करने योग्य जानकारी

फ़ील्डविवरण
developer_tokenAPI Center में जारी किया गया developer token
client_idOAuth client ID
client_secretOAuth client secret (सावधानी से संभालें)
refresh_tokenOAuth प्रमाणीकरण के माध्यम से जारी किया गया refresh token ([access token] फ़ील्ड में दर्ज करें)
customer_idलक्षित विज्ञापन खाते का 10-अंकीय customer ID
GenieGo [पंजीकरण] में developer token, client_id, client_secret, refresh_token और customer_id दर्ज करें और सहेजें। क्रेडेंशियल AES-256-GCM के साथ संग्रहण में एन्क्रिप्ट किए जाते हैं और पढ़ते समय मास्क किए जाते हैं।

4. सामान्य समस्याएँ

समस्याकारणसमाधान
केवल परीक्षण खातों को कॉल किया जा सकता हैdeveloper token बेसिक एक्सेस समीक्षा स्वीकृत नहीं हुईजब तक API Center में आपके द्वारा आवेदन किए गए developer token की Google समीक्षा स्वीकृत न हो जाए तब तक प्रतीक्षा करें, फिर उत्पादन खातों को कॉल करें।
प्रमाणीकरण विफलscope (adwords) गायब या गलत refresh_tokenOAuth client में adwords scope शामिल करके फिर से प्रमाणित करें, और refresh_token को फिर से जाँचें।
customer ID प्रारूप त्रुटिगलत 10-अंकीय customer_idलक्षित विज्ञापन खाते का 10-अंकीय customer ID फिर से जाँचें और दर्ज करें।

अंतिम जाँच-सूची

  • Google Ads मैनेजर (MCC) खाता तैयार किया और साइन इन किया।
  • API Center में developer token के लिए आवेदन किया (बेसिक एक्सेस Google समीक्षा से होकर गुजरता है)।
  • console.cloud.google.com पर OAuth सहमति स्क्रीन और OAuth client ID/Secret बनाया (scope: adwords)।
  • OAuth प्रमाणीकरण के माध्यम से refresh token प्राप्त किया।
  • लक्षित खाते का 10-अंकीय customer_id खोजा।
  • GenieGo [पंजीकरण] में developer token·client_id/secret·refresh_token·customer_id दर्ज किया और सहेजा।
