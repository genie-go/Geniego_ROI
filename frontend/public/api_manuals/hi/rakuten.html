1. शुरू करने से पहले

विक्रेता (Ichiba) कुंजियाँ RMS Web Service से जारी की जाती हैं।

आवश्यक

Rakuten विक्रेता खाता

आपको एक Rakuten विक्रेता (Ichiba) खाता चाहिए। आपको एक ऐसा खाता चाहिए जो RMS R-Login (glogin.rms.rakuten.co.jp) के माध्यम से साइन इन कर सके।

जारी की गई कुंजियाँ

Service Secret · License Key

आपको RMS Web Service से serviceSecret और licenseKey प्राप्त होंगे। License Key 90 दिनों के बाद समाप्त हो जाती है और इसे फिर से जारी किया जा सकता है।

सावधानी: विक्रेता (Ichiba) कुंजियाँ जारी करने वाला RMS Web Service (glogin.rms.rakuten.co.jp) एफिलिएट/उत्पाद खोज के लिए बने webservice.rakuten.co.jp से भिन्न है। अपनी कुंजियाँ अवश्य RMS की ओर से जारी करें।

2. जारी करने के चरण

नीचे दिए गए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

RMS डेवलपर पोर्टल में साइन इन करें

अपने विक्रेता खाते से Rakuten डेवलपर पोर्टल (RMS/Webservice) में साइन इन करें।

https://glogin.rms.rakuten.co.jp
2

RMS Web Service पर जाएँ

RMS R-Login के माध्यम से साइन इन करने के बाद, RMS Web Service क्षेत्र पर जाएँ।

RMS R-Login → RMS Web Service
3

Service Secret · License Key जारी करें

RMS Web Service से Service Secret और License Key जारी करें। License Key 90 दिनों के बाद समाप्त हो जाती है, इसलिए समाप्त होने पर इसे फिर से जारी करें।

4

कुंजियाँ सत्यापित करें

जारी किए गए Service Secret और License Key, साथ ही अपनी दुकान का Shop URL सत्यापित करें।

जब Service Secret और License Key जारी करने की स्क्रीन पर दिखाई दें, तो वे सही ढंग से जारी हो गए हैं।

3. GenieGo में पंजीकृत करने योग्य जानकारी

फ़ील्डविवरण
service_secretजारी किया गया Service Secret
license_keyजारी की गई License Key (90 दिनों में समाप्त·फिर से जारी करने योग्य, सावधानी से संभालें)
shop_urlआपकी दुकान का Shop URL
GenieGo Integration Hub [पंजीकृत करें] में service_secret·license_key·shop_url दर्ज करें और सहेजें। क्रेडेंशियल भंडारण के समय AES-256-GCM से एन्क्रिप्ट किए जाते हैं और पढ़ने पर मास्क किए जाते हैं।
