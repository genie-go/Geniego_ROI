Checkout.com
API Key जारी करने की नियमावली

Checkout.com Dashboard में Live परिवेश की API कुंजियाँ (Secret Key और Public Key) खोजने और उन्हें GenieGo Integration Hub में पंजीकृत करने की एक शुरुआती-अनुकूल चरण-दर-चरण नियमावली।

अनुमानित समयलगभग 5–10 मिनट
आवश्यक खाताCheckout.com मर्चेंट खाता (अनुबंध·समीक्षा अनुमोदन आवश्यक)
जारी कुंजियाँSecret Key (sk_) · Public Key (pk_)
परिणामLive API क्रेडेंशियल

1. शुरू करने से पहले

Checkout.com कुंजियाँ केवल आपके मर्चेंट अनुबंध·समीक्षा के अनुमोदित होने के बाद ही जारी की जा सकती हैं।

आवश्यक

Checkout.com मर्चेंट अनुबंध

सबसे पहले Checkout.com मर्चेंट अनुबंध·समीक्षा अनुमोदन आवश्यक है। अनुमोदन पूरा होने के बाद ही Live कुंजियाँ जारी की जा सकती हैं।

जारी कुंजियाँ

Secret Key · Public Key

Dashboard में Developers > Keys के अंतर्गत, Live परिवेश की Secret Key (sk_) और Public Key (pk_) खोजें।

नोट: Secret Key sk_ से और Public Key pk_ से शुरू होती है। सुनिश्चित करें कि आप Live परिवेश की कुंजियाँ उपयोग करें।

2. जारी करने के अनुरोध की प्रक्रिया

अनुबंध·समीक्षा अनुमोदन के बाद, नीचे दिए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

मर्चेंट अनुबंध·समीक्षा अनुमोदन

पहले Checkout.com मर्चेंट अनुबंध·समीक्षा अनुमोदन पूरा करें। अनुमोदन पूरा होने के बाद ही आप अगले चरण पर जा सकते हैं।

2

Dashboard में लॉग इन करें

Checkout.com Dashboard में लॉग इन करें।

dashboard.checkout.com
3

Keys पर जाएँ

[Developers] > [Keys] मेनू पर जाएँ।

Developers → Keys
4

कुंजियाँ जाँचें

Live परिवेश की Secret Key (sk_) और Public Key (pk_) जाँचें।

यदि Developers > Keys के अंतर्गत Live की Secret Key (sk_) और Public Key (pk_) दिखाई देती हैं, तो वे सही ढंग से जारी की गई हैं।

3. GenieGo में पंजीकृत करने योग्य जानकारी

फ़ील्डविवरण
secret_keyLive Secret Key (sk_ से शुरू, सावधानी से संभालें)
public_keyLive Public Key (pk_ से शुरू)
GenieGo Integration Hub [पंजीकरण] में secret_key·public_key दर्ज करें और सहेजें। क्रेडेंशियल विश्राम अवस्था में AES-256-GCM से एन्क्रिप्ट किए जाते हैं और पढ़ते समय मास्क किए जाते हैं।

4. सामान्य समस्याएँ

समस्याकारणसमाधान
Keys मेनू में कोई Live कुंजी नहींमर्चेंट अनुबंध·समीक्षा अनुमोदित नहींपहले Checkout.com मर्चेंट अनुबंध·समीक्षा अनुमोदन पूरा करें।
प्रमाणीकरण विफलTest कुंजी का उपयोग या Secret Key गलत दर्ज करनाLive परिवेश की sk_ Secret Key और pk_ Public Key फिर से जाँचें।

अंतिम जाँच सूची

  • Checkout.com मर्चेंट अनुबंध·समीक्षा अनुमोदित हो गई है।
  • Dashboard के Developers > Keys में Live कुंजियाँ जाँच लीं।
  • Secret Key (sk_)·Public Key (pk_) की पुष्टि कर ली।
  • GenieGo [पंजीकरण] में secret_key·public_key दर्ज करके सहेज लिया।
