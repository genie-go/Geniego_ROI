1. LINE API जारी करने की अवधारणा

LINE एकीकरण LINE Developers Console में एक Provider और एक Channel बनाकर, फिर Messaging API या LINE Login के लिए Channel ID, Channel Secret और Channel Access Token जारी करके काम करता है।

अंतर: संदेश भेजने के लिए Messaging API चैनल का और लॉगिन एकीकरण के लिए LINE Login चैनल का उपयोग करें।

2. आवश्यक चीज़ें

  • एक LINE खाता या व्यावसायिक खाता
  • LINE Developers Console तक पहुँच की अनुमति
  • एक LINE Official Account
  • सेवा का नाम और विवरण
  • Webhook URL
  • गोपनीयता नीति URL और शर्तें URL

3. Provider बनाएँ

1

LINE Developers तक पहुँचें

LINE Developers Console में साइन इन करें।

developers.line.biz
2

Provider बनाएँ

अपनी कंपनी या सेवा के नाम का उपयोग करके एक Provider बनाएँ।

Create Provider
3

बुनियादी जानकारी दर्ज करें

सेवा संचालक, संपर्क विवरण और शर्तों की जानकारी दर्ज करें।

Provider Settings

4. Messaging API चैनल बनाएँ

1

नया चैनल बनाएँ

Provider के अंदर Create a new channel पर क्लिक करें।

Create a new channel
2

Messaging API चुनें

चैनल प्रकार के रूप में Messaging API चुनें।

Messaging API
3

चैनल जानकारी दर्ज करें

चैनल का नाम, विवरण, श्रेणी, ईमेल आदि दर्ज करें।

Channel Information
4

LINE Official Account लिंक करें

संदेश भेजने के लिए एक आधिकारिक खाता लिंक करें, या नया बनाएँ।

LINE Official Account
5

चैनल निर्माण पूरा करें

बनाने के बाद Channel ID और Channel Secret की जाँच करें।

Channel ID / Channel Secret

5. Channel Access Token जारी करें

1

Messaging API टैब पर जाएँ

अपने बनाए गए चैनल की स्क्रीन पर, Messaging API टैब पर जाएँ।

Messaging API tab
2

Channel access token आइटम खोजें

नीचे दिए गए Channel access token क्षेत्र की जाँच करें।

Channel access token
3

Issue पर क्लिक करें

टोकन जारी करने के लिए Issue या Reissue बटन पर क्लिक करें।

Issue
4

सर्वर पर सुरक्षित रूप से संग्रहीत करें

जारी किए गए टोकन को सर्वर पर्यावरण चर या Secret Manager में संग्रहीत करें।

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token संदेश भेजने की अनुमति वाली एक गुप्त कुंजी है। इसे HTML, फ्रंटएंड या सार्वजनिक रिपॉज़िटरी में उजागर नहीं किया जाना चाहिए।

6. Webhook कॉन्फ़िगर करें

1

Webhook URL दर्ज करें

वह सर्वर URL दर्ज करें जो उपयोगकर्ता संदेश इवेंट प्राप्त करेगा।

Webhook URL
2

Use webhook सक्षम करें

Use webhook विकल्प को Enabled में बदलें।

Use webhook: Enabled
3

Verify परीक्षण

कनेक्शन की पुष्टि के लिए LINE से अपने सर्वर पर एक परीक्षण अनुरोध भेजें।

Verify
4

प्रतिक्रिया कोड जाँचें

पुष्टि करें कि सर्वर 200 OK लौटाता है।

HTTP 200 OK

7. संदेश भेजने का उदाहरण

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "नमस्ते। यह एक LINE संदेश परीक्षण है।"} ] }

8. अंतिम जाँच सूची

  • एक Provider बनाया।
  • एक Messaging API चैनल बनाया।
  • एक LINE Official Account लिंक किया।
  • Channel ID और Channel Secret की जाँच की।
  • Channel Access Token जारी किया।
  • Webhook URL पंजीकृत किया और Verify पास किया।
  • टोकन को सर्वर पर्यावरण चर में सुरक्षित रूप से संग्रहीत किया।
