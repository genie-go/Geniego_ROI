Naver Smart Store
API कुंजी जारी करने की मार्गदर्शिका

Naver Commerce API Center (apicenter.commerce.naver.com) में एक एप्लिकेशन पंजीकृत करके Client ID और Client Secret प्राप्त करने और फिर उन्हें GenieGo Integration Hub में पंजीकृत करने की चरण-दर-चरण मार्गदर्शिका।

परिणामCommerce API क्रेडेंशियल

1. शुरू करने से पहले

जारी करना केवल Smart Store विक्रेता खाते पर एकीकृत प्रबंधक अनुमति के साथ ही संभव है।

आवश्यक

एकीकृत प्रबंधक अनुमति

आपके पास Smart Store विक्रेता खाते पर "एकीकृत प्रबंधक" अनुमति होनी चाहिए। केवल एकीकृत प्रबंधक ही कुंजियाँ जारी कर सकता है।

जारी की गई कुंजियाँ

Client ID · Client Secret

जब आप कोई एप्लिकेशन पंजीकृत करते हैं, तो एक एप्लिकेशन ID (Client ID) और एक Secret (Client Secret) जारी किया जाता है।

नोट: उस सर्वर IP को पंजीकृत करना जो API को कॉल करेगा और एप्लिकेशन प्रमाणीकरण को समय-समय पर नवीनीकृत करना अनिवार्य है।

2. जारी करने के चरण

नीचे दिए गए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

Commerce API Center तक पहुँचें

अपने एकीकृत प्रबंधक खाते से Naver Commerce API Center में साइन इन करें।

https://apicenter.commerce.naver.com
2

खाता बनाएँ

ऊपर दाईं ओर [खाता बनाएँ] पर क्लिक करें, डेवलपर कंपनी का खाता नाम और घटना-प्रतिक्रिया संपर्क दर्ज करें, शर्तों से सहमत हों और साइन अप करें।

खाता बनाएँ
3

एक एप्लिकेशन पंजीकृत करें

[मेरे स्टोर एप्लिकेशन] के अंतर्गत एक एप्लिकेशन पंजीकृत करें; फिर एक एप्लिकेशन ID और Secret जारी किया जाता है।

मेरे स्टोर एप्लिकेशन
4

IP पंजीकृत करें और प्रमाणीकरण नवीनीकृत करें

उस सर्वर IP को पंजीकृत करें जो API को कॉल करेगा, और मार्गदर्शन के अनुसार एप्लिकेशन प्रमाणीकरण को समय-समय पर नवीनीकृत करें।

एक बार एप्लिकेशन पंजीकृत हो जाने और एप्लिकेशन ID तथा Secret जारी हो जाने पर, इसे सही ढंग से संसाधित कर दिया गया है।

3. GenieGo में पंजीकृत करने की जानकारी

फ़ील्डविवरण
client_idजारी की गई एप्लिकेशन ID (Client ID)
client_secretजारी किया गया Secret (Client Secret, सावधानी से संभालें)
उन्हें GenieGo Integration Hub के [पंजीकरण] में दर्ज करें और सहेजें; क्रेडेंशियल को AES-256-GCM के साथ भंडारण के समय एन्क्रिप्ट किया जाता है और पढ़ते समय मास्क किया जाता है।

4. आम समस्याएँ

समस्याकारणसमाधान
एप्लिकेशन पंजीकृत नहीं किया जा सकताखाते में एकीकृत प्रबंधक अनुमति नहीं हैएकीकृत प्रबंधक अनुमति वाले खाते से apicenter.commerce.naver.com तक पहुँचें।
कॉल पर प्रमाणीकरण त्रुटिकॉलर IP पंजीकृत नहीं है या प्रमाणीकरण नवीनीकरण छूट गयाकॉलर सर्वर IP पंजीकृत करें और एप्लिकेशन प्रमाणीकरण को समय-समय पर नवीनीकृत करें।

अंतिम जाँच सूची

  • एकीकृत प्रबंधक खाते से Commerce API Center तक पहुँचा।
  • [खाता बनाएँ] के माध्यम से डेवलपर कंपनी की जानकारी और घटना-प्रतिक्रिया संपर्क दर्ज किया और शर्तों से सहमत हुआ।
  • [मेरे स्टोर एप्लिकेशन] के अंतर्गत एक एप्लिकेशन पंजीकृत किया और ID/Secret प्राप्त किया।
  • कॉलर सर्वर IP पंजीकृत किया और समय-समय पर प्रमाणीकरण सेट किया।
  • GenieGo [पंजीकरण] में client_id·client_secret दर्ज किया और सहेजा।
