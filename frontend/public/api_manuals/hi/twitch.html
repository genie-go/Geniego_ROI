1. Twitch खाता संरचना को समझना

Instagram के विपरीत, Twitch में अलग से व्यवसाय/क्रिएटर खाता चुनने की संरचना नहीं है।

कंपनी का आधिकारिक चैनल

ब्रांड संचालन के लिए

अपनी कंपनी या ब्रांड के नाम से एक सामान्य Twitch खाता बनाएँ और उसे एक आधिकारिक चैनल की तरह संचालित करें। यह उत्पाद प्रदर्शन, लाइव-कॉमर्स और इवेंट प्रसारण के लिए उपयुक्त है।

स्ट्रीमर विकास स्तर

Affiliate / Partner

प्रसारण प्रदर्शन, फ़ॉलोअर्स और देखने के समय की शर्तें पूरी करने पर, आप Affiliate या Partner प्रोग्राम में शामिल हो सकते हैं।

मुख्य बात: कंपनियों को भी पहले एक सामान्य Twitch खाता बनाना चाहिए, फिर चैनल ब्रांडिंग, दो-कारक प्रमाणीकरण, OBS कनेक्शन और Developer Console में ऐप पंजीकरण करना चाहिए।

  • व्यावसायिक ईमेल पता

  • कंपनी का लोगो और चैनल बैनर छवि

4. ब्रांड खाता नाम चुनना

Twitch उपयोगकर्ता नाम आपके चैनल का पता बन जाता है, इसलिए छोटा और याद रखने में आसान नाम सबसे अच्छा है।

उद्देश्यअनुशंसित नाम के उदाहरणविवरण

वैश्विक प्रसारणGeniegoGlobal / OCLGlobalविदेशी दर्शकों पर विचार करते समय उपयुक्त।

11. Client ID / Client Secret जाँचना

1

पंजीकृत ऐप चुनें

Developer Console की Applications सूची में अभी बनाए गए ऐप पर क्लिक करें।

Applications → ऐप चुनें
2

Client ID कॉपी करें

स्क्रीन पर दिखाई गई Client ID कॉपी करें और इसे अपनी सर्वर सेटिंग्स में सहेजें।

Client ID
3

Client Secret जनरेट करें

Client Secret जनरेट करने के लिए New Secret बटन पर क्लिक करें और इसे सुरक्षित रूप से संग्रहीत करें।

New Secret
Client Secret अक्सर केवल एक बार ही देखा जा सकता है। जनरेट करने के तुरंत बाद इसे सुरक्षित स्थान पर सहेजें।

12. Twitch API प्रमाणीकरण संरचना

Twitch Channel │ ▼ Developer Console │ ▼ Application │ ▼ Client ID / Client Secret │ ▼ OAuth Token │ ▼ Twitch Helix API │ ▼ चैनल जानकारी, प्रसारण स्थिति, चैट बॉट, दर्शक विश्लेषण

13. OAuth टोकन जारी करने का प्रवाह

1) Twitch OAuth प्राधिकरण URL जनरेट करें 2) उपयोगकर्ता अपने Twitch खाते से लॉगिन करता है 3) Redirect URL पर authorization code प्राप्त करें 4) authorization code + Client Secret के साथ Access Token का अनुरोध करें 5) Access Token के साथ Twitch API को कॉल करें 6) आवश्यकता पड़ने पर Refresh Token से रिफ़्रेश करें
साधारण सार्वजनिक डेटा क्वेरी App Access Token से संभव है, जबकि उपयोगकर्ता अनुमतियों की आवश्यकता वाले कार्यों के लिए User Access Token की आवश्यकता होती है।

14. व्यावसायिक संचालन के लिए अनुशंसित कॉन्फ़िगरेशन

उद्देश्यअनुशंसित कॉन्फ़िगरेशनविवरण
लाइव-कॉमर्सTwitch + YouTube Live + Facebook Live पर एक साथ प्रसारणपहुँच बढ़ाने के लिए एक ही प्रसारण को कई चैनलों पर भेजें।
ग्राहक संवादTwitch Chat Botचैट स्वतः-उत्तर, उत्पाद लिंक मार्गदर्शन और कूपन सूचना को स्वचालित करें।
वैश्विक संचालनAI उपशीर्षक + बहुभाषी अनुवादकोरियाई, अंग्रेज़ी, जापानी आदि बहुभाषी दर्शकों का समर्थन करें।
प्रदर्शन विश्लेषणTwitch API + आंतरिक डैशबोर्डफ़ॉलोअर्स, प्रसारण स्थिति और दर्शक प्रतिक्रियाओं का विश्लेषण करें।
