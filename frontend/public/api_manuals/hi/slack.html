1. शुरू करने से पहले

Slack Webhook केवल workspace व्यवस्थापक अनुमति के साथ ही बनाया जा सकता है।

आवश्यक

workspace व्यवस्थापक अनुमति

ऐप बनाने और Webhook सक्षम करने के लिए आपको Slack workspace की व्यवस्थापक अनुमति चाहिए।

जारी की गई कुंजियाँ

Webhook URL

Incoming Webhooks सक्षम करने और चैनल चुनने के बाद, जनरेट हुए Webhook URL की जाँच करें।

नोट: Webhook URL उस चैनल से जुड़ा होता है जहाँ सूचनाएं भेजी जाती हैं। चैनल चयन चरण में वह चैनल निर्दिष्ट करें जिसे सूचनाएं प्राप्त करनी हैं।

2. जारी करने के चरण

नीचे दिए क्रम में जारी करें और GenieGo में पंजीकृत करें।

1

नया ऐप बनाएं

[Create New App] पर क्लिक करें।

https://api.slack.com/apps
2

Incoming Webhooks सक्षम करें

[Incoming Webhooks] सक्षम करें और [Add New Webhook to Workspace] पर क्लिक करें।

3

चैनल चुनें · URL जनरेट करें

चैनल चुनने पर Webhook URL जनरेट हो जाता है।

चैनल चुनकर Webhook URL जनरेट हो जाने पर, यह सही ढंग से जारी हो गया है।

3. GenieGo में पंजीकृत करने की जानकारी

फ़ील्डविवरण
webhook_urlजनरेट किया गया Incoming Webhook URL (सावधानी से संभालें)
GenieGo [पंजीकरण] में webhook_url दर्ज करें और सहेजें। क्रेडेंशियल भंडारण के समय AES-256-GCM से एन्क्रिप्ट किए जाते हैं और पढ़ते समय मास्क किए जाते हैं।

4. आम समस्याएँ

समस्याकारणसमाधान
ऐप नहीं बना सकतेworkspace व्यवस्थापक अनुमति अपर्याप्तऐसे खाते से आगे बढ़ें जिसके पास Slack workspace व्यवस्थापक अनुमति हो।
सूचनाएं नहीं आतींगलत Webhook URL या चैनल चयन छूट गयाIncoming Webhooks सक्षम करें, चैनल चुनें और जनरेट किए गए Webhook URL की फिर से जाँच करें।

अंतिम जाँच सूची

  • Slack workspace व्यवस्थापक अनुमति के साथ आगे बढ़ा।
  • api.slack.com/apps पर [Create New App] से ऐप बनाया।
  • Incoming Webhooks सक्षम किया और [Add New Webhook] पर क्लिक किया।
  • चैनल चुना और Webhook URL जनरेट कर कॉपी किया।
  • GenieGo [पंजीकरण] में webhook_url दर्ज कर सहेजा।
