คู่มือการออก API Key

คู่มือสำหรับการออก API key ที่ใช้ร่วมกันของ Gmarket·Auction จาก ESM Trading API (etapi.gmarket.com) และการลงทะเบียนใน GenieGo Integration Hub เนื่องจาก Gmarket และ Auction ดำเนินการร่วมกันผ่าน ESM Plus จึงสามารถใช้ API key เพียงตัวเดียวเชื่อมต่อทั้งสองช่องทางได้พร้อมกัน

1. ก่อนเริ่มต้น

คุณต้องลงทะเบียนเป็นผู้ขายทั้งบน Gmarket และ Auction และมี ESM Plus master ID

จำเป็น

บัญชีผู้ขาย + ESM master ID

คุณต้องมีบัญชีผู้ขายทั้งของ Auction และ Gmarket รวมถึง master ID ที่สร้างใน ESM Plus (esmplus.com)

คีย์ที่ออกให้

API key · Seller ID

หลังจากได้รับการอนุมัติจาก ESM Trading API ให้ตรวจสอบ API key และ seller ID ที่ออกให้พร้อมกัน

หมายเหตุ: ESM Trading API ต้องผ่านการอนุมัติ และคำขอของคุณอาจถูกปฏิเสธได้ เมื่อยื่นคำขอ คุณต้องส่งข้อมูลสามรายการ ได้แก่ Auction seller ID, Gmarket seller ID และ ESM master ID ของคุณ

รับคีย์

เมื่อได้รับการอนุมัติ ระบบจะออก API key ให้ตรวจสอบ API key และ seller ID ที่ออกให้

เมื่อคำขอของคุณได้รับการอนุมัติและ API key ถูกออกให้แล้ว แสดงว่าดำเนินการเรียบร้อยถูกต้อง สำหรับการสอบถามเกี่ยวกับการอนุมัติ โปรดติดต่อ etapihelp@gmail.com

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
คำขอเข้าถึง API ถูกปฏิเสธESM Trading API ต้องผ่านการอนุมัติตรวจสอบข้อมูลที่คุณส่งและติดต่อ etapihelp@gmail.com
ข้อมูลคำขอไม่ตรงกันกรอก Auction·Gmarket seller ID หรือ ESM master ID ไม่ถูกต้องตรวจสอบข้อมูลทั้งสามรายการอีกครั้ง ได้แก่ Auction seller ID, Gmarket seller ID และ ESM master ID ของคุณ

รายการตรวจสอบขั้นสุดท้าย

  • ลงทะเบียนเป็นผู้ขายทั้งบน Auction และ Gmarket แล้ว
  • สร้าง master ID ใน ESM Plus แล้ว
  • ยื่นคำขอและได้รับการอนุมัติการเข้าถึง ESM Trading API ที่ etapi.gmarket.com แล้ว
  • ตรวจสอบ api_key และ seller_id ที่ออกให้แล้ว
  • กรอก api_key·seller_id ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
