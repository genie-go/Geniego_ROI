1. ก่อนเริ่มต้น

การเชื่อมต่อ Google Ads ต้องใช้ทั้งบัญชีผู้ดูแล (MCC) และการตั้งค่า OAuth ใน Google Cloud

developer token

MCC API Center

ขอ developer token ใน API Center ของบัญชีผู้ดูแล (MCC) ของ Google Ads สิทธิ์การเข้าถึงพื้นฐาน (Basic) จะได้รับอนุมัติหลังจากการตรวจสอบของ Google

OAuth

Google Cloud client

ที่ console.cloud.google.com สร้าง OAuth client (client_id·client_secret) และยืนยันตัวตนเพื่อรับ refresh token

หมายเหตุ: สิทธิ์การเข้าถึงพื้นฐานของ developer token จะได้รับอนุมัติหลังจากการตรวจสอบของ Google และก่อนได้รับอนุมัติจะเรียกได้เฉพาะบัญชีทดสอบเท่านั้น หากคุณยังไม่มีบัญชีผู้ดูแล (MCC) ให้สร้างก่อนที่ ads.google.com

2. ขั้นตอนการออก

ออกตามลำดับด้านล่างและลงทะเบียนใน GenieGo

1

เข้าสู่ระบบบัญชีผู้ดูแล · ไปที่ API Center

เข้าสู่ระบบบัญชีผู้ดูแล Google Ads ของคุณ จากนั้นไปที่ [เครื่องมือและการตั้งค่า] > [การตั้งค่า] > [API Center] ที่ด้านบน

เครื่องมือและการตั้งค่า → การตั้งค่า → API Center
2

ขอ developer token

ขอ [developer token] ใน API Center สิทธิ์การเข้าถึงพื้นฐาน (Basic) จะได้รับอนุมัติหลังจากการตรวจสอบของ Google และก่อนได้รับอนุมัติจะเรียกได้เฉพาะบัญชีทดสอบเท่านั้น

3

ตั้งค่าโปรเจกต์ Cloud · หน้าจอยินยอม OAuth

สร้างโปรเจกต์และกำหนดค่า [API และบริการ] > [หน้าจอยินยอม OAuth]

console.cloud.google.com → API และบริการ → หน้าจอยินยอม OAuth
4

สร้าง OAuth client ID

สร้าง [ข้อมูลรับรอง] > [สร้างข้อมูลรับรอง] > [OAuth client ID] เพื่อรับ client_id และ client_secret (scope: adwords)

ข้อมูลรับรอง → สร้างข้อมูลรับรอง → OAuth client ID
5

ออก refresh token

รับ refresh token ผ่านขั้นตอนการยืนยันตัวตน OAuth (ใช้ Google OAuth Playground หรือการยืนยันตัวตนของคุณเอง)

6

ค้นหา customer ID

ค้นหา customer ID 10 หลัก (customer_id) ของบัญชีโฆษณาที่คุณจะเรียก

เมื่อ developer token ได้รับอนุมัติและ client_id·client_secret·refresh_token·customer_id พร้อมทั้งหมดแล้ว การออกก็เสร็จสมบูรณ์

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
developer_tokendeveloper token ที่ออกใน API Center
client_idOAuth client ID
client_secretOAuth client secret (ระวังการรั่วไหล)
refresh_tokenrefresh token ที่ออกผ่านการยืนยันตัวตน OAuth (กรอกในช่อง [access token])
customer_idcustomer ID 10 หลักของบัญชีโฆษณาเป้าหมาย
กรอก developer token, client_id, client_secret, refresh_token และ customer_id ใน GenieGo [ลงทะเบียน] แล้วบันทึก ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และถูกปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้
เรียกได้เฉพาะบัญชีทดสอบการตรวจสอบสิทธิ์การเข้าถึงพื้นฐานของ developer token ยังไม่ได้รับอนุมัติรอจนกว่าการตรวจสอบของ Google สำหรับ developer token ที่คุณขอใน API Center จะได้รับอนุมัติ จากนั้นจึงเรียกบัญชีที่ใช้งานจริง
การยืนยันตัวตนล้มเหลวไม่มี scope (adwords) หรือ refresh_token ไม่ถูกต้องยืนยันตัวตนอีกครั้งโดยรวม scope adwords ไว้ใน OAuth client และตรวจสอบ refresh_token อีกครั้ง
รูปแบบ customer ID ผิดcustomer_id 10 หลักไม่ถูกต้องตรวจสอบ customer ID 10 หลักของบัญชีโฆษณาเป้าหมายอีกครั้งแล้วกรอก

รายการตรวจสอบขั้นสุดท้าย

  • เตรียมบัญชีผู้ดูแล (MCC) ของ Google Ads และเข้าสู่ระบบแล้ว
  • ขอ developer token ใน API Center แล้ว (สิทธิ์การเข้าถึงพื้นฐานผ่านการตรวจสอบของ Google)
  • สร้างหน้าจอยินยอม OAuth และ OAuth client ID/Secret ที่ console.cloud.google.com แล้ว (scope: adwords)
  • ได้รับ refresh token ผ่านการยืนยันตัวตน OAuth แล้ว
  • ค้นหา customer_id 10 หลักของบัญชีเป้าหมายแล้ว
  • กรอก developer token·client_id/secret·refresh_token·customer_id ใน GenieGo [ลงทะเบียน] แล้วบันทึกแล้ว
