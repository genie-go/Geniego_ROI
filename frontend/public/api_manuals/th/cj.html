1. ก่อนเริ่มต้น

API เฉพาะของ CJ Logistics เป็นช่องทางแบบสัญญา และไม่สามารถออกได้ทันทีจากคอนโซลแบบบริการตนเอง

จำเป็น

รหัสสัญญา (ลูกค้า) ของ CJ Logistics

คุณต้องมีรหัสสัญญา (ลูกค้า) ที่ออกผ่านสัญญากับ CJ Logistics โดยไม่ใช่การออกโดยตรงจากคอนโซลแยกต่างหาก

แนะนำ

Smart Parcel (การติดตามแบบรวม)

หากคุณต้องการเพียงการติดตามการจัดส่ง คุณสามารถเชื่อมต่อบริษัทขนส่งทั้งหมดด้วย t_key เพียงตัวเดียวผ่านช่องทาง [Smart Parcel (การติดตามแบบรวม)]

หมายเหตุ: หากเป้าหมายของคุณคือการติดตามการจัดส่ง เราขอแนะนำให้เชื่อมต่อผ่านคีย์รวม Smart Parcel (SweetTracker) โดยไม่ต้องมีสัญญา API เฉพาะ

2. ขั้นตอนการยื่นขอออก

ยื่นขอและลงทะเบียนตามลำดับด้านล่าง

1

เตรียมรหัสสัญญา (ลูกค้า)

เตรียมรหัสสัญญา (ลูกค้า) ของ CJ Logistics ของคุณ API เฉพาะให้บริการแบบสัญญา

2

ตัดสินใจว่าต้องการการติดตามแบบรวมหรือไม่

หากคุณต้องการเพียงการติดตามการจัดส่ง คุณสามารถเชื่อมต่อบริษัทขนส่งทั้งหมดด้วย t_key เพียงตัวเดียวผ่านช่องทาง [Smart Parcel (การติดตามแบบรวม)] แทนช่องทางเฉพาะของ CJ Logistics

3

ยื่นขอออก API เฉพาะ

หากคุณต้องการ API เฉพาะของ CJ Logistics โปรดรับรหัสสัญญา (ลูกค้า) และคีย์ API จากตัวแทนฝ่ายขายของคุณ

4

ลงทะเบียนใน GenieGo

ป้อน api_key และ cust_code ใน [ลงทะเบียน] ของ GenieGo Integration Hub แล้วบันทึก

เมื่อคุณได้รับคีย์ API และรหัสสัญญา (ลูกค้า) จากตัวแทนฝ่ายขาย แสดงว่าพร้อมที่จะลงทะเบียนแล้ว

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
api_keyคีย์ API เฉพาะของ CJ Logistics ที่ออกแล้ว
cust_codeรหัสสัญญา (ลูกค้า) ของ CJ Logistics
เมื่อคุณป้อนและบันทึกใน [ลงทะเบียน] ของ GenieGo Integration Hub ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่พบเมนูการออกในคอนโซลAPI เฉพาะของ CJ Logistics เป็นแบบสัญญา ไม่ใช่การออกจากคอนโซลแบบบริการตนเองรับรหัสสัญญา (ลูกค้า) และคีย์ API จากตัวแทนฝ่ายขายของคุณ
ต้องการเพียงการติดตามการจัดส่งแต่ขั้นตอนซับซ้อนสัญญา API เฉพาะนั้นเกินความจำเป็นสำหรับวัตถุประสงค์การติดตามลงทะเบียน t_key ในช่องทาง [Smart Parcel (การติดตามแบบรวม)] เพื่อเชื่อมต่อ
เกิดข้อผิดพลาดด้านการยืนยันตัวตนหลังลงทะเบียนป้อน api_key หรือ cust_code ไม่ถูกต้องตรวจสอบ api_key ที่ออกและรหัสสัญญา (ลูกค้า) อีกครั้ง

รายการตรวจสอบขั้นสุดท้าย

  • ได้รับรหัสสัญญา (ลูกค้า) ของ CJ Logistics แล้ว
  • ได้ประเมินการเชื่อมต่อ Smart Parcel (การติดตามแบบรวม) แล้ว หากต้องการเพียงการติดตามการจัดส่ง
  • ได้รับคีย์ API และรหัสสัญญา (ลูกค้า) จากตัวแทนฝ่ายขายแล้ว
  • ป้อน api_key·cust_code ใน [ลงทะเบียน] ของ GenieGo และบันทึกแล้ว
