ผลลัพธ์ข้อมูลรับรอง API

1. ก่อนเริ่มต้น

ข้อมูลรับรอง API ของ PayPal จะออกให้โดยการสร้างแอปใน Developer Portal ด้วยบัญชีธุรกิจ

จำเป็น

บัญชีธุรกิจ PayPal

คุณต้องมีบัญชีธุรกิจ PayPal ที่สามารถเข้าสู่ระบบ developer.paypal.com ได้

คีย์ที่ออก

Client ID · Secret

การสร้างแอป Live จะออก Client ID และ Secret ให้

หมายเหตุ: สำหรับการเชื่อมต่อจริง ให้สร้างแอปในแท็บ [Live] ไม่ใช่ Sandbox

2. ขั้นตอนการออก

ออกตามลำดับด้านล่างและลงทะเบียนใน GenieGo

1

เข้าสู่ระบบ Developer Portal

เข้าสู่ระบบด้วยบัญชีธุรกิจ PayPal ของคุณ จากนั้นไปที่ [Apps & Credentials]

developer.paypal.com → Apps & Credentials
2

สร้างแอป Live

ในแท็บ [Live] คลิก [Create App] เพื่อสร้างแอป

Live → Create App
3

ตรวจสอบข้อมูลรับรอง

ตรวจสอบ Client ID และ Secret ที่ออกให้

เมื่อแสดง Client ID และ Secret แล้ว แสดงว่าออกให้เรียบร้อยถูกต้อง

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
client_idClient ID ที่ออกให้
client_secretSecret ที่ออกให้ (ระวังการเปิดเผย)
ในศูนย์เชื่อมต่อ GenieGo [ลงทะเบียน] ให้ป้อน client_id·client_secret แล้วบันทึก ข้อมูลรับรองจะถูกเข้ารหัสจัดเก็บด้วย AES-256-GCM และถูกปิดบังเมื่อเรียกดู

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้
การยืนยันตัวตนล้มเหลวใช้ข้อมูลรับรองจากแอป Sandboxใช้ Client ID·Secret จากแอปที่สร้างในแท็บ [Live]
ไม่สามารถสร้างแอปได้เข้าสู่ระบบด้วยบัญชีทั่วไป (ส่วนบุคคล)เข้าสู่ระบบด้วยบัญชีธุรกิจ PayPal

รายการตรวจสอบขั้นสุดท้าย

  • เข้าสู่ระบบ developer.paypal.com ด้วยบัญชีธุรกิจ PayPal แล้ว
  • ไปที่แท็บ [Live] ใน [Apps & Credentials] แล้ว
  • สร้างแอปผ่าน [Create App] และตรวจสอบ Client ID·Secret แล้ว
  • ป้อน client_id·client_secret ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
