1. ความแตกต่างระหว่าง API Key กับ OAuth

นี่คือส่วนที่คนสับสนมากที่สุดเมื่อเริ่มต้นใช้งาน

API Key

สำหรับการเรียกดูข้อมูลสาธารณะ

ใช้สำหรับดึงข้อมูล YouTube ที่เป็นสาธารณะ เช่น การค้นหาวิดีโอ การดูข้อมูลช่อง การดึงเพลย์ลิสต์ และการอ่านความคิดเห็น

OAuth 2.0

สำหรับงานที่ต้องใช้สิทธิ์ของบัญชีคุณ

ใช้สำหรับงานที่ต้องได้รับการอนุมัติจากผู้ใช้ เช่น การอัปโหลดวิดีโอ การจัดการช่องของคุณเอง และการเข้าถึงข้อมูลส่วนตัว

สำคัญ: หากต้องการอัปโหลดวิดีโออัตโนมัติ การใช้ API Key เพียงอย่างเดียวไม่เพียงพอ จำเป็นต้องมีการยืนยันตัวตนแบบ OAuth 2.0

2. เปิดใช้งาน YouTube Data API v3

ก่อนสร้าง API Key คุณต้องเปิดใช้งาน API ที่ต้องการในโปรเจกต์ของคุณก่อน

1

เข้าสู่ Google Cloud Console

ลงชื่อเข้าใช้ Google Cloud Console

console.cloud.google.com
2

เลือกหรือสร้างโปรเจกต์

ในส่วนตัวเลือกโปรเจกต์ด้านบน ให้เลือกโปรเจกต์ที่มีอยู่หรือสร้างโปรเจกต์ใหม่

เช่น Geniego ROI
3

ไปที่ไลบรารี API

ในเมนูด้านซ้าย ไปที่ API และบริการ → ไลบรารี

API และบริการ → ไลบรารี
4

ค้นหา YouTube Data API v3 แล้วเปิดใช้งาน

พิมพ์ YouTube Data API v3 ในช่องค้นหา เลือก API นั้น แล้วคลิกปุ่มเปิดใช้งาน

YouTube Data API v3 → เปิดใช้งาน
หากคุณเห็น สถานะ: เปิดใช้งานแล้ว บนหน้าจอ แสดงว่าเปิดใช้งานเรียบร้อยแล้ว

3. สร้าง API Key

เมื่อเปิดใช้งาน YouTube Data API v3 แล้ว คุณสามารถสร้าง API Key ได้ในหน้าข้อมูลรับรอง

1

ไปที่เมนูข้อมูลรับรอง

ในเมนูด้านซ้าย ไปที่ API และบริการ → ข้อมูลรับรอง

API และบริการ → ข้อมูลรับรอง
2

คลิกสร้างข้อมูลรับรอง

คลิกปุ่ม + สร้างข้อมูลรับรอง ที่ด้านบน

+ สร้างข้อมูลรับรอง
3

เลือกคีย์ API

ในเมนูที่ปรากฏ ให้เลือกคีย์ API

คีย์ API
4

เลือก API ที่เข้าถึงได้

ในส่วนข้อจำกัด API ให้ยืนยันว่าได้เลือก YouTube Data API v3 แล้ว

API ที่เลือก: YouTube Data API v3
5

คลิกปุ่มสร้าง

คลิกปุ่มสร้างที่ด้านล่างเพื่อสร้าง API Key

4. ควรเลือกตัวเลือกใดในหน้าจอการสร้าง?

รายการการตั้งค่าที่แนะนำสำหรับผู้เริ่มต้นคำอธิบาย
ชื่อYouTube API Keyป้อนชื่อที่จดจำได้ง่ายในภายหลัง
ข้อจำกัด APIYouTube Data API v3จำกัดคีย์นี้ให้ใช้เฉพาะกับ YouTube API เท่านั้น
การยืนยันตัวตนบัญชีบริการไม่ต้องเลือกโดยทั่วไปไม่จำเป็นสำหรับการสร้าง YouTube Data API Key
ข้อจำกัดแอปพลิเคชันไม่ตั้งสำหรับการทดสอบครั้งแรกหลังจากยืนยันว่าทำงานได้ถูกต้องแล้ว เราแนะนำให้ตั้งค่าข้อจำกัดเว็บไซต์หรือที่อยู่ IP
เมื่อนำไปใช้งานในบริการจริง ให้ตั้งค่าข้อจำกัดของคีย์เสมอ หากเปิดเผยโดยไม่มีข้อจำกัด ผู้อื่นจะสามารถใช้คีย์ของคุณได้

5. ตัวอย่างการใช้งาน API Key

คุณสามารถทดสอบได้โดยใส่คีย์ที่สร้างขึ้นในส่วน YOUR_API_KEY

ตัวอย่างการค้นหาวิดีโอhttps://www.googleapis.com/youtube/v3/search?part=snippet&q=korea&type=video&key=YOUR_API_KEY
ตัวอย่างการดูข้อมูลช่องhttps://www.googleapis.com/youtube/v3/channels?part=snippet&id=CHANNEL_ID&key=YOUR_API_KEY
หากผลลัพธ์ในรูปแบบ JSON ปรากฏขึ้นเมื่อคุณป้อน URL ด้านบนในแถบที่อยู่ของเบราว์เซอร์ แสดงว่า API Key ทำงานได้อย่างถูกต้อง

6. การแก้ไขปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่ได้เลือก APIยังไม่ได้เปิดใช้งาน YouTube Data API v3 หรือไม่ได้เลือกในข้อจำกัด APIเปิดใช้งาน YouTube Data API v3 ในไลบรารี API แล้วเลือกอีกครั้ง
ไม่มีคีย์ API ที่จะแสดงคุณยังไม่ได้สร้าง API Keyคลิกสร้างข้อมูลรับรอง → คีย์ API
ข้อผิดพลาด 403ข้อจำกัด API โควตาเกิน หรือปัญหาสิทธิ์ตรวจสอบข้อจำกัด API โควตา และสถานะการเปิดใช้งาน
อัปโหลดไม่ได้ใช้เฉพาะ API Keyการอัปโหลดวิดีโอต้องใช้การยืนยันตัวตนแบบ OAuth 2.0

7. รายการตรวจสอบขั้นสุดท้าย

  • เลือกโปรเจกต์ Google Cloud แล้ว
  • เปิดใช้งาน YouTube Data API v3 แล้ว
  • สร้าง API Key ในข้อมูลรับรองแล้ว
  • ยืนยันแล้วว่า API Key ที่สร้างขึ้นขึ้นต้นด้วย AIzaSy
  • ตั้งค่าข้อจำกัด API เป็น YouTube Data API v3 แล้ว
  • ตั้งค่าข้อจำกัดเว็บไซต์หรือ IP ก่อนนำไปใช้งานในบริการจริง
