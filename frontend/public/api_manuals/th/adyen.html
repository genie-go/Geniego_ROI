คู่มือที่สรุปขั้นตอนการออกคีย์ API (X-API-Key) และ Merchant Account ใน Customer Area หลังจากสัญญาร้านค้า Adyen และการตรวจสอบได้รับอนุมัติ และลงทะเบียนใน GenieGo Integration Hub

เวลาโดยประมาณประมาณ 5–15 นาทีหลังได้รับอนุมัติ
คุณสมบัติที่ต้องมีสัญญาร้านค้า Adyen · การตรวจสอบอนุมัติ
คีย์ที่ออกคีย์ API (X-API-Key) · Merchant Account
ผลลัพธ์ข้อมูลรับรอง API

1. ก่อนเริ่มต้น

Adyen เป็นเกตเวย์การชำระเงินระดับองค์กรทั่วโลก ดังนั้นจึงต้องมีสัญญาร้านค้าและการตรวจสอบอนุมัติก่อนจึงจะออกข้อมูลรับรองได้

จำเป็น

สัญญาร้านค้า · การตรวจสอบอนุมัติ

Adyen เป็น PG ระดับองค์กรทั่วโลกที่ต้องมีสัญญาร้านค้าและการตรวจสอบอนุมัติ คุณจะออกข้อมูลรับรองได้ก็ต่อเมื่อได้รับอนุมัติแล้วเท่านั้น

คีย์ที่ออก

X-API-Key · Merchant Account

หลังได้รับอนุมัติ ให้สร้างคีย์ API (X-API-Key) ใน [Developers] > [API credentials] ของ Customer Area และยืนยันชื่อ Merchant Account

เงื่อนไขเบื้องต้น: ไม่สามารถออกด้วยตนเองได้ก่อนที่สัญญาร้านค้าและการตรวจสอบจะได้รับอนุมัติ คุณต้องทำสัญญาร้านค้าและการตรวจสอบกับ Adyen ให้เสร็จสิ้นก่อน

2. ขั้นตอนการยื่นขอออก

หลังการตรวจสอบอนุมัติ ให้ออกและลงทะเบียนข้อมูลรับรองตามลำดับด้านล่าง

1

สัญญาร้านค้า · การตรวจสอบอนุมัติ

ทำสัญญาร้านค้ากับ Adyen และรับการตรวจสอบอนุมัติ

ฝ่ายขาย/สัญญา Adyen
2

ลงชื่อเข้าใช้ Customer Area

หลังได้รับอนุมัติ ให้ลงชื่อเข้าใช้ Customer Area

https://ca-live.adyen.com
3

ออกข้อมูลรับรอง API

ใน [Developers] > [API credentials] ให้สร้างคีย์ API (X-API-Key) และยืนยันชื่อ Merchant Account

Developers → API credentials
4

ยืนยันหมายเลขแบตช์การชำระบัญชีเริ่มต้น

สำหรับการเก็บข้อมูลการชำระบัญชีครั้งแรก ให้ค้นหาหมายเลข settlement batch ล่าสุดใน Customer Area

เมื่อยืนยันคีย์ API (X-API-Key) และ Merchant Account แล้ว แสดงว่าข้อมูลรับรองพร้อมใช้งาน

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
api_keyคีย์ API ที่ออกแล้ว (X-API-Key จัดการด้วยความระมัดระวัง)
merchant_accountชื่อ Merchant Account ที่ยืนยันแล้ว
batch_startหมายเลขแบตช์การชำระบัญชีเริ่มต้นสำหรับการเก็บข้อมูลการชำระบัญชีครั้งแรก (หมายเลข settlement batch ล่าสุดใน Customer Area)
ป้อน api_key·merchant_account·batch_start ใน [ลงทะเบียน] ของ GenieGo Integration Hub แล้วบันทึก ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่มีเมนู API credentialsสัญญาร้านค้า · การตรวจสอบยังไม่ได้รับอนุมัติทำสัญญาร้านค้า · การตรวจสอบกับ Adyen ให้เสร็จสิ้น จากนั้นจึงเข้าใช้ Customer Area
การยืนยันตัวตนล้มเหลวป้อน X-API-Key หรือ Merchant Account ไม่ถูกต้องตรวจสอบคีย์ API ที่ออกและชื่อ Merchant Account อีกครั้งใน Developers > API credentials

รายการตรวจสอบขั้นสุดท้าย

  • ได้รับสัญญาร้านค้า · การตรวจสอบอนุมัติจาก Adyen แล้ว
  • ลงชื่อเข้าใช้ Customer Area (ca-live.adyen.com) แล้ว
  • ยืนยันคีย์ API (X-API-Key) และ Merchant Account ใน Developers > API credentials แล้ว
  • ยืนยันหมายเลข settlement batch ล่าสุด (batch_start) แล้ว
  • ป้อน api_key·merchant_account·batch_start ใน [ลงทะเบียน] ของ GenieGo และบันทึกแล้ว
