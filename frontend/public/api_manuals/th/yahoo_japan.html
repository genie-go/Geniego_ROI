1. แนวคิดการออก API ของ Yahoo! Japan

API ในเครือ Yahoo! Japan จะออก Client ID, Client Secret และข้อมูลรับรอง OAuth จาก Yahoo! JAPAN Developers หรือ LY/Yahoo!広告 Developer Center ขึ้นอยู่กับประเภทของบริการ

สำคัญ: "Yahoo! Japan" จะมีไซต์การออกและขั้นตอนการตรวจสอบที่แตกต่างกันขึ้นอยู่กับ API ที่คุณต้องการใช้ คุณต้องแยกแยะ API สำหรับการช้อปปิ้ง โฆษณา การเข้าสู่ระบบ และการค้นหา

2. สิ่งที่ต้องเตรียม

  • Yahoo! JAPAN ID หรือบัญชีธุรกิจ
  • ตัดสินใจประเภทของ API ที่จะใช้
  • ชื่อแอปและคำอธิบายบริการ
  • Redirect URI
  • URL นโยบายความเป็นส่วนตัว
  • โดเมนใช้งานจริงหรือโดเมนทดสอบ

3. ลำดับพื้นฐานในการลงทะเบียนแอปพลิเคชัน

1

เข้าถึงไซต์นักพัฒนา

ลงชื่อเข้าใช้ Yahoo! JAPAN Developers หรือ Developer Center ของ API ที่เกี่ยวข้อง

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

สร้างแอปพลิเคชัน

คลิกเมนูเพิ่มแอปพลิเคชันใหม่

Create Application / Add Application
3

กรอกข้อมูลแอป

กรอกชื่อแอป คำอธิบาย ข้อมูลผู้รับผิดชอบ และ URL ของเว็บไซต์

Application Information
4

กรอก Redirect URI

กรอก URL ที่จะกลับมาหลังการยืนยันตัวตน OAuth ให้ถูกต้อง

Redirect URI
5

เลือกสิทธิ์หรือ API

เลือกสิทธิ์ API และขอบเขตที่คุณต้องการใช้

Scope / API Permission
6

ลงทะเบียนเสร็จสมบูรณ์

หลังลงทะเบียน ให้ตรวจสอบ Client ID และ Client Secret

Client ID / Client Secret

วัตถุประสงค์ตำแหน่งการออก/จัดการผลลัพธ์

การทดสอบภายในสภาพแวดล้อม Sandbox/ทดสอบของแต่ละบริการข้อมูลรับรองสำหรับทดสอบ

5. ขั้นตอนการออกโทเค็น OAuth

1) สร้าง URL การยืนยันตัวตนผู้ใช้ด้วย Client ID 2) ผู้ใช้เข้าสู่ระบบด้วยบัญชี Yahoo! 3) รับ authorization code ที่ Redirect URI 4) ขอ Access Token ด้วย authorization code + Client Secret 5) เรียก API ด้วย Access Token 6) รีเฟรชด้วย Refresh Token เมื่อจำเป็น

6. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ข้อผิดพลาด Redirect URIURL ที่ลงทะเบียนต่างจาก URL ที่เรียกกรอกให้เหมือนกันทั้งโปรโตคอล โดเมน และเส้นทาง
ดู Client Secret ไม่ได้ถูกซ่อนเพื่อความปลอดภัยตรวจสอบฟังก์ชัน View หรือออกใหม่ในหน้าจัดการ
สิทธิ์ไม่เพียงพอไม่ได้เลือก Scope ที่จำเป็นเพิ่ม Scope ในการตั้งค่าแอปแล้วยืนยันตัวตนใหม่
เรียก API ไม่สำเร็จAPI ที่ใช้ไม่ตรงกับประเภทแอปที่ออกตรวจสอบเอกสารนักพัฒนาของแต่ละ API

7. รายการตรวจสอบขั้นสุดท้าย

  • ยืนยันประเภท API ของ Yahoo! Japan ที่จะใช้แล้ว
  • ลงชื่อเข้าใช้คอนโซลนักพัฒนาที่ถูกต้องแล้ว
  • ลงทะเบียนแอปพลิเคชันแล้ว
  • ตรวจสอบ Client ID และ Client Secret แล้ว
  • ลงทะเบียน Redirect URI อย่างถูกต้องแล้ว
  • เลือก Scope ที่จำเป็นแล้ว
  • ทำการทดสอบการออก Access Token เสร็จแล้ว
