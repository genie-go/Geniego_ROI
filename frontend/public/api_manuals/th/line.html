คู่มือการออก API

คู่มือนี้สรุปขั้นตอนการสร้าง Provider และ Messaging API Channel ใน LINE Developers Console และการตั้งค่า Channel Access Token, Webhook URL

บัญชีที่ต้องใช้บัญชี LINE

การใช้งานการส่งข้อความ·แชทบอท

1. แนวคิดการออก LINE API

การเชื่อมต่อ LINE คือการสร้าง Provider และ Channel ใน LINE Developers Console จากนั้นออก Channel ID, Channel Secret, Channel Access Token สำหรับ Messaging API หรือ LINE Login

การแบ่งประเภท: การส่งข้อความใช้ช่องทาง Messaging API และการเชื่อมต่อการเข้าสู่ระบบใช้ช่องทาง LINE Login

2. สิ่งที่ต้องเตรียม

  • บัญชี LINE หรือบัญชีธุรกิจ
  • สิทธิ์การเข้าถึง LINE Developers Console
  • LINE Official Account
  • ชื่อบริการและคำอธิบาย
  • Webhook URL
  • URL นโยบายความเป็นส่วนตัวและ URL ข้อกำหนด

1

เข้าถึง LINE Developers

เข้าสู่ระบบ LINE Developers Console

developers.line.biz

Create Provider

กรอกผู้ดำเนินการบริการ ข้อมูลติดต่อ และข้อมูลข้อกำหนด

Provider Settings

5. ออก Channel Access Token

1

ไปที่แท็บ Messaging API

ในหน้าจอช่องทางที่สร้างไว้ ให้ไปที่แท็บ Messaging API

Messaging API tab
2

ค้นหารายการ Channel access token

ตรวจสอบส่วน Channel access token ที่ด้านล่าง

Channel access token
3

คลิก Issue

คลิกปุ่ม Issue หรือ Reissue เพื่อออกโทเค็น

Issue
4

จัดเก็บบนเซิร์ฟเวอร์อย่างปลอดภัย

จัดเก็บโทเค็นที่ออกแล้วในตัวแปรสภาพแวดล้อมของเซิร์ฟเวอร์หรือ Secret Manager

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token เป็นกุญแจลับที่มีสิทธิ์ส่งข้อความ ห้ามเปิดเผยใน HTML ฟรอนต์เอนด์ หรือที่เก็บโค้ดสาธารณะ

6. ตั้งค่า Webhook

1

ป้อน Webhook URL

ป้อน URL ของเซิร์ฟเวอร์ที่จะรับเหตุการณ์ข้อความของผู้ใช้

Webhook URL
2

เปิดใช้งาน Use webhook

เปลี่ยนตัวเลือก Use webhook เป็น Enabled

Use webhook: Enabled
3

ทดสอบ Verify

ส่งคำขอทดสอบจาก LINE ไปยังเซิร์ฟเวอร์ของคุณเพื่อยืนยันการเชื่อมต่อ

Verify
4

ตรวจสอบรหัสการตอบกลับ

ตรวจสอบว่าเซิร์ฟเวอร์ส่งคืน 200 OK

HTTP 200 OK

7. ตัวอย่างการส่งข้อความ

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "สวัสดี นี่คือการทดสอบข้อความ LINE"} ] }

8. รายการตรวจสอบขั้นสุดท้าย

  • สร้าง Provider แล้ว
  • สร้างช่องทาง Messaging API แล้ว
  • เชื่อมต่อ LINE Official Account แล้ว
  • ตรวจสอบ Channel ID และ Channel Secret แล้ว
  • ออก Channel Access Token แล้ว
  • ลงทะเบียน Webhook URL และผ่าน Verify แล้ว
  • จัดเก็บโทเค็นในตัวแปรสภาพแวดล้อมของเซิร์ฟเวอร์อย่างปลอดภัยแล้ว
