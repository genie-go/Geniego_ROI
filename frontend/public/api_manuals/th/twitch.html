1. ทำความเข้าใจโครงสร้างบัญชี Twitch

Twitch ไม่มีโครงสร้างให้เลือกบัญชีธุรกิจ/บัญชีครีเอเตอร์แยกต่างหากเหมือน Instagram

ช่องทางการขององค์กร

สำหรับการดำเนินงานแบรนด์

สร้างบัญชี Twitch ทั่วไปด้วยชื่อบริษัทหรือชื่อแบรนด์ แล้วดำเนินการเหมือนช่องทางการ เหมาะกับการแนะนำสินค้า ไลฟ์คอมเมิร์ซ และการถ่ายทอดสดอีเวนต์

ระดับการเติบโตของสตรีมเมอร์

Affiliate / Partner

เมื่อคุณตรงตามเงื่อนไขด้านผลงานการถ่ายทอดสด ผู้ติดตาม และเวลารับชม คุณสามารถเข้าร่วมโปรแกรม Affiliate หรือ Partner ได้

ประเด็นสำคัญ: องค์กรก็ควรสร้างบัญชี Twitch ทั่วไปก่อน แล้วจึงดำเนินการสร้างแบรนด์ช่อง การยืนยันตัวตนสองขั้นตอน การเชื่อมต่อ OBS และการลงทะเบียนแอปใน Developer Console

2. สิ่งที่ต้องเตรียม

  • ที่อยู่อีเมลสำหรับองค์กร
  • ชื่อแบรนด์หรือชื่อบริษัท
  • ชื่อผู้ใช้ Twitch ที่เป็นตัวเลือก
  • โลโก้บริษัทและรูปภาพแบนเนอร์ช่อง
  • ข้อความแนะนำบริษัทและ URL เว็บไซต์
  • PC หรือโน้ตบุ๊กสำหรับถ่ายทอดสด
  • OBS Studio หรือซอฟต์แวร์สตรีมมิง
  • OAuth Redirect URL สำหรับการเชื่อมต่อ API

4. การตั้งชื่อบัญชีแบรนด์

ชื่อผู้ใช้ Twitch จะกลายเป็นที่อยู่ของช่อง จึงควรใช้ชื่อที่สั้นและจำง่าย

วัตถุประสงค์ตัวอย่างชื่อที่แนะนำคำอธิบาย
ช่องทางการขององค์กรGeniego / OCLKoreaตรงกับชื่อบริษัทจึงน่าเชื่อถือสูง
การถ่ายทอดสดระดับโลกGeniegoGlobal / OCLGlobalเหมาะเมื่อต้องคำนึงถึงผู้ชมต่างประเทศ
ไลฟ์คอมเมิร์ซGeniegoLive / GeniegoCommerceวัตถุประสงค์การถ่ายทอดสดเพื่อขายหรือช้อปปิงชัดเจน
เทคโนโลยี/แพลตฟอร์มGeniegoTech / OCLTechเหมาะกับการถ่ายทอดสดแนะนำ IT, SaaS และแพลตฟอร์ม

12. โครงสร้างการยืนยันตัวตน Twitch API

Twitch Channel │ ▼ Developer Console │ ▼ Application │ ▼ Client ID / Client Secret │ ▼ OAuth Token │ ▼ Twitch Helix API │ ▼ ข้อมูลช่อง สถานะการถ่ายทอดสด แชตบอต การวิเคราะห์ผู้ชม

13. ขั้นตอนการออกโทเค็น OAuth

1) สร้าง URL การให้สิทธิ์ Twitch OAuth 2) ผู้ใช้เข้าสู่ระบบด้วยบัญชี Twitch 3) รับ authorization code ที่ Redirect URL 4) ขอ Access Token ด้วย authorization code + Client Secret 5) เรียก Twitch API ด้วย Access Token 6) รีเฟรชด้วย Refresh Token เมื่อจำเป็น
การดึงข้อมูลสาธารณะอย่างง่ายทำได้ด้วย App Access Token ส่วนการดำเนินการที่ต้องใช้สิทธิ์ผู้ใช้จำเป็นต้องใช้ User Access Token

14. การกำหนดค่าที่แนะนำสำหรับการดำเนินงานองค์กร

วัตถุประสงค์การกำหนดค่าที่แนะนำคำอธิบาย
ไลฟ์คอมเมิร์ซถ่ายทอดสดพร้อมกันบน Twitch + YouTube Live + Facebook Liveส่งการถ่ายทอดสดเดียวกันไปยังหลายช่องเพื่อเพิ่มการเข้าถึง
การสื่อสารกับลูกค้าTwitch Chat Botทำให้การตอบแชตอัตโนมัติ การแนะนำลิงก์สินค้า และการแจ้งคูปองเป็นอัตโนมัติ
การดำเนินงานระดับโลกคำบรรยาย AI + การแปลหลายภาษารองรับผู้ชมหลายภาษาทั้งภาษาเกาหลี อังกฤษ ญี่ปุ่น และอื่นๆ
การวิเคราะห์ผลงานTwitch API + แดชบอร์ดภายในวิเคราะห์ผู้ติดตาม สถานะการถ่ายทอดสด และปฏิกิริยาของผู้ชม

15. รายการตรวจสอบขั้นสุดท้าย

  • สร้างบัญชี Twitch ด้วยอีเมลองค์กรแล้ว
  • ตั้งชื่อผู้ใช้ที่ตรงกับชื่อแบรนด์แล้ว
  • ยืนยันอีเมลเสร็จแล้ว
  • ตั้งค่ารูปโปรไฟล์ แบนเนอร์ ข้อความแนะนำ และลิงก์แล้ว
  • เปิดใช้งานการยืนยันตัวตนสองขั้นตอนแล้ว
  • ตรวจสอบ Stream Key ใน Creator Dashboard แล้ว
  • เชื่อมต่อ OBS Studio กับช่อง Twitch แล้ว
  • ลงทะเบียนแอปใน Twitch Developer Console แล้ว
  • ได้รับ Client ID และ Client Secret แล้ว
  • เตรียม OAuth Redirect URL และขั้นตอนการออกโทเค็นแล้ว
