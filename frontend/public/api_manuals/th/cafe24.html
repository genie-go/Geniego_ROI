คู่มือทีละขั้นตอนที่เป็นมิตรกับผู้เริ่มต้น สำหรับการลงทะเบียนแอปใน Cafe24 Developer Center เพื่อออก Client ID·Client Secret รับโทเค็นผ่านการอนุญาต OAuth 2.0 และลงทะเบียนใน GenieGo Integration Hub

เวลาโดยประมาณประมาณ 10–15 นาที
บัญชีที่ต้องใช้บัญชีผู้ดูแลร้านค้า Cafe24
คีย์ที่ออกClient ID · Client Secret · Refresh Token
ผลลัพธ์ข้อมูลรับรอง OAuth 2.0

1. ก่อนเริ่มต้น

การเชื่อมต่อ Cafe24 ต้องผ่านการลงทะเบียนนักพัฒนาและการลงทะเบียนแอป และยืนยันตัวตนด้วยวิธี OAuth 2.0

จำเป็น

บัญชีผู้ดูแลร้านค้า

คุณต้องมีบัญชีผู้ดูแลที่บริหารร้านค้า Cafe24 ใช้บัญชีนี้เพื่อดำเนินการลงทะเบียนนักพัฒนาใน Developer Center

คีย์ที่ออก

Client ID · Client Secret + โทเค็น

เมื่อลงทะเบียนแอป จะออก Client ID และ Client Secret และคุณจะได้รับ access token และ refresh token ผ่านการอนุญาต OAuth 2.0

หมายเหตุ: ใน GenieGo คุณจะลงทะเบียน refresh_token ที่เป็นแบบ long-lived และ access token จะถูกรีเฟรชโดยอัตโนมัติผ่านโทเค็นนั้น

2. ขั้นตอนการลงทะเบียนแอปและการออกโทเค็น

ออกตามลำดับด้านล่างและลงทะเบียนใน GenieGo

1

ลงทะเบียนนักพัฒนา

เข้าสู่ Cafe24 Developer Center และลงทะเบียนเป็นนักพัฒนา

developer.cafe24.com
2

ลงทะเบียนแอป

สร้างแอปผ่าน [แอป] > [ลงทะเบียนแอป] แล้วจะออก Client ID และ Client Secret

แอป → ลงทะเบียนแอป
3

การอนุญาต OAuth 2.0

ออก access token และ refresh token ผ่านการอนุญาต OAuth 2.0

4

ตรวจสอบข้อมูล

ตรวจสอบ mall_id (ID ร้านค้า), Client ID, Client Secret และ refresh token

เมื่อตรวจสอบ Client ID·Client Secret และ refresh token ครบถ้วนแล้ว แสดงว่าออกอย่างถูกต้อง

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
mall_idID ร้านค้า Cafe24
client_idClient ID ที่ออกแล้ว
client_secretClient Secret ที่ออกแล้ว (จัดการด้วยความระมัดระวัง)
refresh_tokenRefresh Token ที่ได้รับผ่านการอนุญาต OAuth 2.0
ป้อนและบันทึกใน GenieGo Integration Hub [ลงทะเบียน] แล้วข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
การอนุญาตล้มเหลวmall_id ไม่ถูกต้อง หรือการตั้งค่าแอปไม่ตรงกันตรวจสอบ mall_id และข้อมูล Client ที่ตั้งไว้ตอนลงทะเบียนแอปอีกครั้ง
ข้อผิดพลาดโทเค็นหมดอายุป้อน refresh_token ผิดลงทะเบียน refresh_token ล่าสุดที่ได้รับผ่านการอนุญาต OAuth 2.0 อีกครั้ง

รายการตรวจสอบขั้นสุดท้าย

  • ลงทะเบียนเป็นนักพัฒนาใน Cafe24 Developer Center แล้ว
  • สร้างแอปผ่าน [แอป > ลงทะเบียนแอป] และออก Client ID·Client Secret แล้ว
  • ออก access token·refresh token ผ่านการอนุญาต OAuth 2.0 แล้ว
  • ป้อน mall_id·client_id·client_secret·refresh_token ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
