คู่มือการออก API Key

คู่มือทีละขั้นตอนสำหรับการลงทะเบียนแอปพลิเคชันใน Naver Commerce API Center (apicenter.commerce.naver.com) เพื่อรับ Client ID และ Client Secret จากนั้นลงทะเบียนไว้ใน GenieGo Integration Hub

1. ก่อนเริ่มต้น

การออกคีย์ทำได้เฉพาะด้วยสิทธิ์ผู้จัดการแบบรวมในบัญชีผู้ขาย Smart Store เท่านั้น

จำเป็น

สิทธิ์ผู้จัดการแบบรวม

คุณต้องมีสิทธิ์ "ผู้จัดการแบบรวม" ในบัญชีผู้ขาย Smart Store เฉพาะผู้จัดการแบบรวมเท่านั้นที่สามารถออกคีย์ได้

คีย์ที่ออกให้

Client ID · Client Secret

เมื่อคุณลงทะเบียนแอปพลิเคชัน ระบบจะออกรหัสแอปพลิเคชัน (Client ID) และ Secret (Client Secret) ให้

หมายเหตุ: จำเป็นต้องลงทะเบียน IP ของเซิร์ฟเวอร์ที่จะเรียก API และต่ออายุการยืนยันตัวตนของแอปพลิเคชันเป็นระยะ

2. ขั้นตอนการออกคีย์

ออกคีย์ตามลำดับด้านล่างและลงทะเบียนใน GenieGo

1

เข้าสู่ Commerce API Center

ลงชื่อเข้าใช้ Naver Commerce API Center ด้วยบัญชีผู้จัดการแบบรวมของคุณ

https://apicenter.commerce.naver.com
2

สร้างบัญชี

คลิก [สร้างบัญชี] ที่มุมขวาบน กรอกชื่อบัญชีบริษัทผู้พัฒนาและผู้ติดต่อสำหรับการรับมือเหตุขัดข้อง ยอมรับข้อกำหนด แล้วสมัครสมาชิก

สร้างบัญชี
3

ลงทะเบียนแอปพลิเคชัน

ที่ [แอปพลิเคชันร้านค้าของฉัน] ลงทะเบียนแอปพลิเคชัน จากนั้นระบบจะออกรหัสแอปพลิเคชันและ Secret

แอปพลิเคชันร้านค้าของฉัน
4

ลงทะเบียน IP และต่ออายุการยืนยันตัวตน

ลงทะเบียน IP ของเซิร์ฟเวอร์ที่จะเรียก API และต่ออายุการยืนยันตัวตนของแอปพลิเคชันเป็นระยะตามคำแนะนำ

เมื่อลงทะเบียนแอปพลิเคชันและออกรหัสแอปพลิเคชันและ Secret แล้ว ถือว่าดำเนินการถูกต้อง

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
client_idรหัสแอปพลิเคชันที่ออกให้ (Client ID)
client_secretSecret ที่ออกให้ (Client Secret, โปรดจัดการด้วยความระมัดระวัง)
กรอกข้อมูลใน [ลงทะเบียน] ของ GenieGo Integration Hub แล้วบันทึก ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และถูกปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่สามารถลงทะเบียนแอปพลิเคชันได้บัญชีไม่มีสิทธิ์ผู้จัดการแบบรวมเข้าใช้ apicenter.commerce.naver.com ด้วยบัญชีที่มีสิทธิ์ผู้จัดการแบบรวม
เกิดข้อผิดพลาดการยืนยันตัวตนเมื่อเรียกใช้ไม่ได้ลงทะเบียน IP ผู้เรียกหรือลืมต่ออายุการยืนยันตัวตนลงทะเบียน IP ของเซิร์ฟเวอร์ผู้เรียกและต่ออายุการยืนยันตัวตนของแอปพลิเคชันเป็นระยะ

รายการตรวจสอบขั้นสุดท้าย

  • เข้าใช้ Commerce API Center ด้วยบัญชีผู้จัดการแบบรวมแล้ว
  • กรอกข้อมูลบริษัทผู้พัฒนาและผู้ติดต่อสำหรับการรับมือเหตุขัดข้องผ่าน [สร้างบัญชี] และยอมรับข้อกำหนดแล้ว
  • ลงทะเบียนแอปพลิเคชันที่ [แอปพลิเคชันร้านค้าของฉัน] และได้รับ ID/Secret แล้ว
  • ลงทะเบียน IP ของเซิร์ฟเวอร์ผู้เรียกและตั้งค่าการยืนยันตัวตนเป็นระยะแล้ว
  • กรอก client_id·client_secret ใน [ลงทะเบียน] ของ GenieGo และบันทึกแล้ว
