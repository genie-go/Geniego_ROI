1. ก่อนเริ่มต้น

API เฉพาะของ Hanjin Logistics เป็นช่องทางแบบสัญญา ซึ่งไม่ได้ออกทันทีจากคอนโซลแบบบริการตนเอง

จำเป็น

รหัสสัญญา (ลูกค้า) ของ Hanjin Logistics

คุณต้องมีรหัสสัญญา (ลูกค้า) ที่ออกผ่านการทำสัญญากับ Hanjin Logistics ไม่ใช่วิธีการออกโดยตรงจากคอนโซลแยกต่างหาก

แนะนำ

Smart Delivery (การติดตามแบบรวม)

หากคุณต้องการเพียงการติดตามการจัดส่ง คุณสามารถเชื่อมต่อบริษัทขนส่งทั้งหมดด้วย t_key เพียงตัวเดียวบนช่องทาง [Smart Delivery (การติดตามแบบรวม)]

หมายเหตุ: หากเป้าหมายของคุณคือการติดตามการจัดส่ง เราแนะนำให้เชื่อมต่อผ่านคีย์รวมของ Smart Delivery โดยไม่ต้องทำสัญญา API เฉพาะ

2. ขั้นตอนการยื่นขอออก

ยื่นขอและลงทะเบียนตามลำดับด้านล่าง

1

เตรียมรหัสสัญญา (ลูกค้า)

เตรียมรหัสสัญญา (ลูกค้า) ของ Hanjin Logistics ของคุณ API เฉพาะจะให้บริการในรูปแบบสัญญา

2

พิจารณาว่าการติดตามแบบรวมเพียงพอหรือไม่

หากคุณต้องการเพียงการติดตามการจัดส่ง คุณสามารถเชื่อมต่อบริษัทขนส่งทั้งหมดด้วย t_key เพียงตัวเดียวบนช่องทาง [Smart Delivery (การติดตามแบบรวม)] แทนช่องทางเฉพาะของ Hanjin Logistics

3

ยื่นขอ API เฉพาะ

หากคุณต้องการ API เฉพาะ โปรดรับรหัสสัญญา (ลูกค้า) และคีย์ API จากตัวแทนฝ่ายขายของคุณ

4

ลงทะเบียนใน GenieGo

ป้อน api_key และ cust_code ใน [ลงทะเบียน] ของ GenieGo Integration Hub แล้วบันทึก

เมื่อคุณได้รับคีย์ API และรหัสสัญญา (ลูกค้า) จากตัวแทนฝ่ายขายแล้ว แสดงว่าคุณพร้อมที่จะลงทะเบียน

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
api_keyคีย์ API เฉพาะของ Hanjin Logistics ที่ออกแล้ว
cust_codeรหัสสัญญา (ลูกค้า) ของ Hanjin Logistics
ป้อนค่าใน [ลงทะเบียน] ของ GenieGo Integration Hub แล้วบันทึก ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่พบเมนูการออกในคอนโซลAPI เฉพาะของ Hanjin Logistics เป็นแบบสัญญา ไม่ใช่การออกผ่านคอนโซลแบบบริการตนเองรับรหัสสัญญา (ลูกค้า) และคีย์ API จากตัวแทนฝ่ายขายของคุณ
ต้องการเพียงการติดตามการจัดส่งแต่ขั้นตอนซับซ้อนสัญญา API เฉพาะมากเกินไปสำหรับวัตถุประสงค์การติดตามลงทะเบียน t_key บนช่องทาง [Smart Delivery (การติดตามแบบรวม)] เพื่อเชื่อมต่อ
เกิดข้อผิดพลาดการรับรองหลังลงทะเบียนป้อน api_key หรือ cust_code ไม่ถูกต้องตรวจสอบ api_key และรหัสสัญญา (ลูกค้า) ที่ออกแล้วอีกครั้ง

รายการตรวจสอบขั้นสุดท้าย

  • ได้รหัสสัญญา (ลูกค้า) ของ Hanjin Logistics แล้ว
  • หากต้องการเพียงการติดตามการจัดส่ง ได้พิจารณาการเชื่อมต่อ Smart Delivery (การติดตามแบบรวม) แล้ว
  • ได้รับคีย์ API และรหัสสัญญา (ลูกค้า) จากตัวแทนฝ่ายขายแล้ว
  • ป้อน api_key·cust_code ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
