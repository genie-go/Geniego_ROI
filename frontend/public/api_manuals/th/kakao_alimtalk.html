1. แนวคิดการออก Kakao Alimtalk

Alimtalk เป็นบริการข้อความธุรกิจที่ส่งข้อความเชิงข้อมูล เช่น คำสั่งซื้อ การจอง การจัดส่ง และการชำระเงิน ผ่าน Kakao Talk ไม่ใช่โครงสร้างที่ส่งได้ทันทีด้วยคีย์แอปนักพัฒนา Kakao ทั่วไปเพียงอย่างเดียว แต่ต้องมีการยืนยันช่องธุรกิจและการตั้งค่าตัวแทนส่ง/แพลตฟอร์ม

สำคัญ: Alimtalk เน้นข้อความเชิงข้อมูลมากกว่าข้อความโฆษณา และเทมเพลตจำเป็นต้องผ่านการตรวจสอบล่วงหน้า

2. สิ่งที่ต้องเตรียม

  • บัญชี Kakao Business
  • ช่อง Kakao Talk
  • เอกสารยืนยันธุรกิจ เช่น ใบทะเบียนพาณิชย์
  • ข้อความเทมเพลต Alimtalk ที่จะส่ง
  • บัญชีตัวแทนส่งหรือบริการส่งข้อความบนคลาวด์
  • นโยบายการเก็บหมายเลขโทรศัพท์ผู้รับและการจัดการความยินยอม

3. การเตรียมช่อง Kakao Talk

1

เข้าสู่ Kakao Business

ลงชื่อเข้าใช้ศูนย์ผู้ดูแล Kakao Business

business.kakao.com
2

สร้างช่อง Kakao Talk

สร้างช่องด้วยชื่อแบรนด์หรือชื่อบริการ

ศูนย์ผู้ดูแลช่อง Kakao Talk
3

ดำเนินการยืนยันธุรกิจ

ส่งข้อมูลธุรกิจและเอกสารหลักฐานเพื่อยืนยันช่องธุรกิจ

การยืนยันธุรกิจ
4

ตรวจสอบ ID สำหรับค้นหาช่อง

ตรวจสอบ ID ช่องหรือ ID สำหรับค้นหาที่จำเป็นสำหรับการตั้งค่าการส่ง Alimtalk

โปรไฟล์ช่อง → ID สำหรับค้นหา
ต้องยืนยันธุรกิจให้เสร็จสมบูรณ์ก่อน จึงจะสามารถลงทะเบียนโปรไฟล์ผู้ส่ง Alimtalk ได้

4. การลงทะเบียนโปรไฟล์ผู้ส่ง / Sender Profile

1

เลือกแพลตฟอร์มการส่ง

เลือกผู้ให้บริการ Alimtalk เช่น NHN Cloud, Naver Cloud SENS, Infobank, Bizgo

Messaging Provider
2

สร้างโปรเจกต์

สร้างโปรเจกต์ Alimtalk บนแพลตฟอร์มที่คุณเลือก

Project / Service
3

เชื่อมต่อช่อง Kakao Talk

ลงทะเบียนข้อมูลช่อง Kakao Talk ที่ผ่านการยืนยันธุรกิจแล้ว

Sender Profile
4

ตรวจสอบโปรไฟล์ผู้ส่ง

ดำเนินการยืนยันความเป็นเจ้าของช่องและตรวจสอบโปรไฟล์ผู้ส่ง

Profile Review
5

ยืนยันข้อมูลรับรอง API

ตรวจสอบ App Key, Secret Key, API Key ฯลฯ ที่แพลตฟอร์มจัดให้

API Key / Secret

5. การลงทะเบียนและอนุมัติเทมเพลต

1

เขียนเทมเพลต

เขียนข้อความเชิงข้อมูล เช่น การยืนยันคำสั่งซื้อ การแจ้งการจัดส่ง

Template Message
2

ตั้งค่าตัวแปร

ตั้งค่าค่าที่เปลี่ยนแปลงแบบไดนามิก เช่น ชื่อ หมายเลขคำสั่งซื้อ หมายเลขการจัดส่ง เป็นตัวแปร

#{ตัวแปร}
3

ขอการตรวจสอบ

บันทึกเทมเพลตและส่งคำขอตรวจสอบไปยัง Kakao

Request Approval
4

ยืนยันการอนุมัติ

ยืนยันรหัสเทมเพลตหรือ ID ที่ได้รับการอนุมัติ

Template Code
หากข้อความก่อนการตรวจสอบแตกต่างจากข้อความที่ส่งจริง อาจทำให้การส่งล้มเหลวหรือถูกลงโทษได้

6. ตัวอย่างขั้นตอนการส่งผ่าน API

1) เรียก Alimtalk API Endpoint จากเซิร์ฟเวอร์
2) ยืนยันตัวตนด้วย API Key หรือ Secret
3) ส่งหมายเลขโทรศัพท์ผู้รับ รหัสเทมเพลต และค่าตัวแปร
4) รับผลการส่ง
5) จัดการว่าจะส่ง SMS สำรองหรือไม่เมื่อล้มเหลว

7. รายการตรวจสอบขั้นสุดท้าย

  • สร้างช่อง Kakao Talk แล้ว
  • ทำการยืนยันธุรกิจเสร็จสมบูรณ์แล้ว
  • เลือกแพลตฟอร์มการส่ง Alimtalk แล้ว
  • ลงทะเบียน Sender Profile แล้ว
  • ยืนยัน API Key หรือ Secret แล้ว
  • ลงทะเบียนเทมเพลตและได้รับการอนุมัติแล้ว
  • เตรียมนโยบายการส่งล้มเหลวและข้อความสำรองแล้ว
