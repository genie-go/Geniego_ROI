1. ก่อนเริ่มต้น

Fulfillment (3PL) เป็นช่องทางแบบสัญญา ซึ่งจะออกข้อมูลรับรองหลังจากทำสัญญารับฝากจัดการคลังสินค้า (เข้า/ออก)

จำเป็น

สัญญารับฝากจัดการคลังสินค้า (เข้า/ออก)

หากต้องการใช้ 3PL fulfillment จะต้องมีสัญญารับฝากจัดการคลังสินค้า (เข้า/ออก) กับผู้ให้บริการ fulfillment หรือทีมปฏิบัติการก่อน

คีย์ที่ออก

3PL API key · warehouse ID

หลังจากทำสัญญาเสร็จสมบูรณ์ คุณจะได้รับ warehouse ID และ 3PL API key

หมายเหตุ: ช่องทางนี้ไม่ได้ออกข้อมูลรับรองโดยตรงจากคอนโซลด้วยตนเอง แต่เป็นการรับข้อมูลรับรองผ่านสัญญารับฝากจัดการ

2. ขั้นตอนการยื่นขอออก

ยื่นขอและรับตามลำดับด้านล่าง จากนั้นลงทะเบียนใน GenieGo

1

ดำเนินการสัญญารับฝากจัดการ

ดำเนินการสัญญารับฝากจัดการคลังสินค้า (เข้า/ออก) กับผู้ให้บริการ fulfillment หรือทีมปฏิบัติการ

2

ลงนามในสัญญา

ลงนามในสัญญารับฝากจัดการคลังสินค้า (เข้า/ออก) แบบ 3PL fulfillment

3

รับข้อมูลรับรอง

หลังจากทำสัญญาเสร็จสมบูรณ์ คุณจะได้รับ warehouse ID·3PL API key

4

ลงทะเบียนใน GenieGo

ป้อน api_key, warehouse_id ที่ออกให้ใน GenieGo [ลงทะเบียน] แล้วบันทึก

หากคุณได้รับทั้ง warehouse ID·3PL API key จากผู้ให้บริการ fulfillment แล้ว แสดงว่าออกอย่างถูกต้อง

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
api_key3PL API key ที่ออกให้
warehouse_idwarehouse ID ที่ออกให้
เมื่อป้อนและบันทึกใน GenieGo Integration Hub [ลงทะเบียน] ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่ได้รับข้อมูลรับรองสัญญารับฝากจัดการคลังสินค้า (เข้า/ออก) ยังไม่เสร็จสมบูรณ์ลงนามในสัญญารับฝากจัดการกับผู้ให้บริการ fulfillment หรือทีมปฏิบัติการ จากนั้นจึงรับข้อมูลรับรอง
ข้อผิดพลาดการเชื่อมต่อเข้า/ออกคลังสินค้าป้อน warehouse ID (warehouse_id) ผิดตรวจสอบ warehouse ID ที่ออกให้อีกครั้งและป้อนให้ถูกต้อง

รายการตรวจสอบขั้นสุดท้าย

  • ลงนามในสัญญารับฝากจัดการคลังสินค้า (เข้า/ออก) กับผู้ให้บริการ fulfillment หรือทีมปฏิบัติการแล้ว
  • ได้รับ warehouse ID·3PL API key หลังจากทำสัญญาเสร็จสมบูรณ์แล้ว
  • ยืนยัน api_key·warehouse_id แล้ว
  • ป้อน api_key·warehouse_id ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
