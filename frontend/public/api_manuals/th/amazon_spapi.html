1. Amazon SP-API คืออะไร?

Amazon Selling Partner API คือ API สำหรับเชื่อมต่อข้อมูลผู้ขาย เช่น คำสั่งซื้อ สินค้าคงคลัง สินค้า การจัดส่ง และการชำระบัญชี ผ่านโปรแกรม

สำคัญ: SP-API ไม่ใช่รูปแบบ API Key อย่างง่าย โดยทั่วไปต้องใช้ การลงทะเบียน Amazon Developer การลงทะเบียนแอป Login with Amazon(LWA) และการอนุญาต OAuth ร่วมกัน
★การเชื่อมต่อ GenieGo ต้องใช้ค่า 5 รายการ: refresh_token, client_id, client_secret, marketplace_id, seller_id (SP-API ปัจจุบันเรียกใช้ด้วยโทเค็น LWA เพียงอย่างเดียว และการลงนาม AWS IAM/SigV4 จัดการภายในโดย GenieGo จึงไม่ต้องกรอกในช่องลงทะเบียน)

2. สิ่งที่ต้องเตรียม

  • บัญชีผู้ขายระดับ Professional ของ Amazon Seller Central หรือ Vendor Central
  • ข้อมูลบริษัท/ผู้ติดต่อที่จะใช้ลงทะเบียนนักพัฒนา
  • ชื่อแอป คำอธิบายแอป และวัตถุประสงค์การใช้งาน
  • URL นโยบายความเป็นส่วนตัว
  • OAuth Redirect URL

3. การลงทะเบียนนักพัฒนา

อันดับแรก ลงทะเบียนโปรไฟล์นักพัฒนา SP-API ใน Seller Central

1

เข้าสู่ระบบ Seller Central

เข้าสู่ระบบ Seller Central ด้วยบัญชีผู้ขาย Amazon ของคุณ

Seller Central → Apps & Services
2

ไปที่เมนู Develop Apps

จากเมนู Apps & Services ไปที่เมนูนักพัฒนาหรือการพัฒนาแอป

Apps & Services → Develop Apps
3

กรอก Developer Profile

กรอกข้อมูลบริษัท ข้อมูลผู้ติดต่อ วัตถุประสงค์การใช้ API และแผนการปกป้องข้อมูล

Developer Profile
4

ส่งเพื่อตรวจสอบ

ส่งข้อมูลการลงทะเบียนนักพัฒนาเพื่อให้ Amazon ตรวจสอบ

Submit for review
หลังจากได้รับการอนุมัติเรียบร้อยแล้วเท่านั้น จึงจะสามารถลงทะเบียนแอป SP-API สำหรับใช้งานจริงและใช้สิทธิ์ของแอปได้

4. ลงทะเบียนแอป SP-API → Client ID / Client Secret

1

สร้างแอปพลิเคชันใหม่

สร้างแอปใหม่ใน Develop Apps

Add new app
2

เลือกประเภทแอป

หากเชื่อมต่อเฉพาะบัญชีของคุณเองจะเป็น Private App; หากให้บริการแก่ผู้ขายภายนอกจะเป็น Public App

Private / Public App
3

เลือกบทบาท API

เลือกบทบาทที่ต้องการ เช่น Orders, Catalog และ Inventory

API Roles
4

กรอก OAuth Redirect URL

กรอก URL ของเซิร์ฟเวอร์ที่จะกลับมาหลังจากการอนุญาตเสร็จสิ้น

Redirect URI
5

บันทึกแล้วตรวจสอบข้อมูล Client

หลังจากสร้างแอป ให้ตรวจสอบ client_id (LWA Client ID) และ client_secret (Client Secret)

client_id / client_secret

5. การอนุญาต OAuth → ออก Refresh Token

1) เข้าถึง URL การอนุญาต OAuth ของผู้ขาย 2) รับ authorization code ที่ Redirect URL 3) ส่ง authorization code ไปยัง LWA Token Endpoint 4) รับ refresh_token / access_token
refresh_token ใช้สำหรับการเรียกใช้งานระยะยาวหลังจากผู้ขายอนุญาต GenieGo จะรีเฟรช access_token โดยอัตโนมัติด้วย refresh_token นี้เพื่อเรียก SP-API ดังนั้นคุณเพียงลงทะเบียน refresh_token โดยไม่ต้องตั้งค่าการลงนาม AWS แยกต่างหาก

6. ตรวจสอบ Marketplace ID · Seller ID

ตรวจสอบตัวระบุที่เหลืออีกสองรายการที่จำเป็นสำหรับการลงทะเบียน GenieGo

marketplace_id

Marketplace ID เป้าหมายการขาย

Marketplace ID ของประเทศที่คุณกำลังเชื่อมต่อ US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

จำเป็นสำหรับการลงทะเบียนสินค้าและการค้นหาคำสั่งซื้อ ตรวจสอบได้ในข้อมูลบัญชี Seller Central ของคุณ Seller Central → การตั้งค่า → ข้อมูลบัญชี → Merchant Token

7. ลงทะเบียนใน GenieGo Integration Hub

1

เตรียมค่า 5 รายการ

เตรียม refresh_token · client_id · client_secret · marketplace_id · seller_id

ข้อมูลรับรอง 5 รายการ
2

Integration Hub → Amazon SP-API → [ลงทะเบียน]

กรอกค่าแต่ละรายการในช่องของมันแล้วบันทึก (ช่องที่ว่างจะคงค่าเดิมไว้)

refresh_token · client_id · client_secret · marketplace_id · seller_id
ค่าที่ลงทะเบียนจะถูก เข้ารหัสจัดเก็บด้วย AES-256-GCM และปิดบังเมื่อมีการเรียกดู การซิงค์อัตโนมัติจะเริ่มทันทีหลังบันทึก
ห้ามเปิดเผย client_secret·refresh_token ในโค้ดฟรอนต์เอนด์ HTML หรือที่เก็บโค้ดสาธารณะโดยเด็ดขาด

8. รายการตรวจสอบขั้นสุดท้าย

  • ส่ง Developer Profile และได้รับการอนุมัติแล้ว
  • สร้างแอป SP-API และตรวจสอบ client_id·client_secret แล้ว
  • ออก refresh_token ผ่านการอนุญาต OAuth แล้ว
  • ตรวจสอบ marketplace_id ที่จะเชื่อมต่อแล้ว
  • ตรวจสอบ seller_id (Merchant Token) ใน Seller Central แล้ว
  • กรอกค่า 5 รายการ (refresh_token·client_id·client_secret·marketplace_id·seller_id) ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว

9. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่มีเมนู API KeySP-API ไม่ใช่โครงสร้าง API Key เดี่ยวดำเนินการด้วยโครงสร้าง LWA (client_id·client_secret·refresh_token)
ลงทะเบียนสินค้าล้มเหลวยังไม่ได้ลงทะเบียน seller_id (Merchant Token)ลงทะเบียน Merchant Token จากข้อมูลบัญชี Seller Central ของคุณ
ข้อมูลประเทศผิดmarketplace_id ไม่ตรงกันลงทะเบียน Marketplace ID ที่ถูกต้องสำหรับประเทศที่คุณเชื่อมต่อ
ข้อผิดพลาดด้านสิทธิ์API Role ของแอปไม่เพียงพอหรือโทเค็นหมดอายุตรวจสอบ API Role และรีเฟรชโทเค็นด้วย refresh_token
