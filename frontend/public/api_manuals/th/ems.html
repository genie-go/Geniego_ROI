Korea Post International Express (EMS) เป็นบริการแบบสัญญาที่อ้างอิงจากรหัสสัญญา (ลูกค้า) คู่มือนี้สรุปขั้นตอนการลงทะเบียนคีย์ API ของไปรษณีย์เกาหลีและรหัสลูกค้า (สัญญา) ที่คุณได้รับหลังทำสัญญา EMS ไว้ใน GenieGo Integration Hub

1. ก่อนเริ่มต้น

EMS ไม่ใช่การออกผ่านคอนโซลแบบบริการตนเอง แต่เป็นบริการแบบสัญญาที่อ้างอิงจากรหัสสัญญา (ลูกค้า)

หมายเหตุ: Korea Post International Express (EMS) อ้างอิงจากรหัสสัญญา (ลูกค้า) ไม่ใช่วิธีที่คุณออกข้อมูลรับรองด้วยตนเอง แต่เป็นขั้นตอนที่คุณทำสัญญา EMS ก่อน แล้วจึงรับรหัสลูกค้า (สัญญา) และคีย์ API ของไปรษณีย์
สิ่งที่ต้องเตรียม

สัญญา EMS ของไปรษณีย์ (รหัสลูกค้า)

คุณต้องมีรหัสสัญญา (ลูกค้า) ของ Korea Post International Express (EMS) ซึ่งออกได้ผ่านสัญญาเท่านั้น

ข้อมูลรับรองที่ได้รับ

คีย์ API ของไปรษณีย์ · รหัสลูกค้า (สัญญา)

หลังจากทำสัญญาเสร็จสิ้น คุณจะได้รับรหัสลูกค้า (สัญญา) และคีย์ API ของไปรษณีย์

2. ขั้นตอนการยื่นขอออก

เนื่องจากเป็นบริการแบบสัญญา ให้ทำสัญญา รับข้อมูลรับรอง แล้วลงทะเบียนตามลำดับด้านล่าง

1

ทำสัญญา EMS

ทำสัญญา EMS ผ่านไปรษณีย์ออนไลน์หรือเคาน์เตอร์ทำสัญญา

2

รับรหัสลูกค้าและคีย์ API

หลังจากทำสัญญาเสร็จสิ้น คุณจะได้รับรหัสลูกค้า (สัญญา) และคีย์ API ของไปรษณีย์

3

ลงทะเบียนใน GenieGo

ป้อนข้อมูลรับรองที่ได้รับลงใน GenieGo [ลงทะเบียน] แล้วบันทึก

เนื่องจากเป็นบริการแบบสัญญา การรับข้อมูลรับรองอาจใช้เวลาขึ้นอยู่กับสถานะการดำเนินการสัญญา

3. ข้อมูลที่ต้องลงทะเบียนใน GenieGo

ฟิลด์คำอธิบาย
api_keyคีย์ API ของไปรษณีย์ที่ออกแล้ว
cust_codeรหัสลูกค้า (สัญญา)
เมื่อป้อนและบันทึกใน GenieGo Integration Hub [ลงทะเบียน] ข้อมูลรับรองจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

4. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่สามารถออกข้อมูลรับรองได้สัญญา EMS ยังไม่เสร็จสมบูรณ์ทำสัญญา EMS ให้เสร็จก่อนผ่านไปรษณีย์ออนไลน์หรือเคาน์เตอร์ทำสัญญา
เกิดข้อผิดพลาดการยืนยันตัวตนหลังลงทะเบียนป้อนคีย์ API หรือรหัสลูกค้า (สัญญา) ผิดตรวจสอบ api_key และ cust_code ที่ออกแล้วอีกครั้ง

รายการตรวจสอบขั้นสุดท้าย

  • ได้รับรหัสลูกค้า (สัญญา) และคีย์ API ของไปรษณีย์แล้ว
  • ตรวจสอบข้อมูลรับรองที่ออกแล้ว
  • ป้อน api_key·cust_code ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
