Facebook Live
คู่มือการออก API

คู่มือที่เป็นมิตรกับผู้เริ่มต้น สำหรับการสร้างแอปบน Meta for Developers และเตรียม App ID, App Secret และโทเค็นการเข้าถึงเพจ (Page Access Token) ที่จำเป็นสำหรับการเชื่อมต่อการถ่ายทอดสดของเพจ Facebook

แนวคิดหลักไม่ใช่ API Key แต่เป็น Access Token
เว็บไซต์นักพัฒนาdevelopers.facebook.com
API ที่ใช้Live Video API (Graph)
สิทธิ์ที่ต้องใช้ผู้ดูแลเพจ + Page Token

1. สิ่งสำคัญที่ต้องรู้ก่อน

YouTube ใช้ API Key เพียงตัวเดียวก็จบ แต่ Facebook (Meta) ทำงานด้วยโครงสร้าง App ID · App Secret · Access Token

สำคัญ: สำหรับ Meta แค่ "คัดลอก API Key ตัวเดียว" ยังไม่จบ โดยทั่วไปต้องทำตามลำดับ สร้างแอป → เพิ่มผลิตภัณฑ์ → ตั้งค่าสิทธิ์ → ออก Page Access Token → ตรวจสอบแอป

2. สิ่งที่ต้องเตรียม

  • บัญชี Meta/Facebook
  • สิทธิ์การเข้าถึง Meta for Developers
  • เพจ Facebook ที่คุณมีสิทธิ์ผู้ดูแล
  • URL นโยบายความเป็นส่วนตัว · URL ข้อกำหนดในการให้บริการ สำหรับบริการจริง

3. สร้างแอป Meta Developer

การเชื่อมต่อ Meta API ทั้งหมดเริ่มต้นจากการสร้างแอป

1

เข้าถึง Meta for Developers

ลงชื่อเข้าใช้ Meta for Developers ในเบราว์เซอร์ของคุณ

developers.facebook.com
2

แอปของฉัน → สร้างแอป

คลิก My Apps → Create App ที่ด้านบน

My Apps → Create App
3

เลือกประเภทแอป

การเชื่อมต่อแพลตฟอร์ม/ไลฟ์คอมเมิร์ซใกล้เคียงกับวัตถุประสงค์ทางธุรกิจมากกว่า

เลือกประเภท Business
4

ป้อนชื่อแอป·อีเมล → สร้าง

ป้อนชื่อแอปและอีเมลติดต่อ แล้วสร้างแอป

App Name / Contact Email
หลังจากสร้างแอปแล้ว ดู App ID และ App Secret ที่ แดชบอร์ด → การตั้งค่าแอป → การตั้งค่าพื้นฐาน (ห้ามเปิดเผย App Secret ในฝั่งหน้าบ้าน/ที่เก็บโค้ดสาธารณะโดยเด็ดขาด)

4. ขั้นตอนการตั้งค่า Facebook Live API

1

เพิ่มผลิตภัณฑ์

บนแดชบอร์ดของแอป เตรียมการตั้งค่าที่เกี่ยวข้องกับ Facebook Login / Graph API

Add Product → Facebook Login
2

เชื่อมต่อเพจ + โทเค็นเพจ

หากต้องการถ่ายทอดสดไปยังเพจ คุณต้องมีสิทธิ์ผู้ดูแลเพจและ Page Access Token

Page Access Token
3

ขอสิทธิ์ที่จำเป็น (App Review)

ขอสิทธิ์สำหรับการเผยแพร่เนื้อหา·จัดการเพจ·สร้างวิดีโอถ่ายทอดสด (เช่น pages_read_engagement, pages_manage_posts)

Permissions → App Review
4

ออกและทดสอบโทเค็นด้วย Graph API Explorer

ออกโทเค็นที่มีสิทธิ์รวมอยู่และทดสอบการเรียก API

Tools → Graph API Explorer
ตัวอย่างการเรียกสร้าง Facebook Live
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
เมื่อสร้างสำเร็จ ระบบจะส่งคืนข้อมูลการถ่ายทอดสด เช่น ID การถ่ายทอด·URL สตรีม·คีย์สตรีม ดู ID เพจ (page_id) ได้ในข้อมูลเพจ

5. ลงทะเบียนใน GenieGo Integration Hub

1

คัดลอกโทเค็นการเข้าถึงเพจ

คัดลอก Page Access Token ที่คุณออก (แนะนำโทเค็นระยะยาว)

access_token (Page token)
2

ยืนยัน ID เพจ

ยืนยัน ID (page_id) ของเพจ Facebook เป้าหมาย

page_id
3

ลงทะเบียนใน Integration Hub

ใน GenieGo Integration Hub → Facebook Live → [ลงทะเบียน] ป้อนค่าด้านล่างแล้วบันทึก

access_token · page_id
คีย์ที่ลงทะเบียนจะถูกเข้ารหัสขณะจัดเก็บด้วย AES-256-GCM และปิดบังเมื่ออ่าน

6. ประเภทของ Access Token

ประเภทโทเค็นการใช้งานข้อควรระวัง
User Access Tokenการเรียก API ตามการเข้าสู่ระบบของผู้ใช้ข้อมูลที่เข้าถึงได้จะแตกต่างกันตามขอบเขตสิทธิ์ที่อนุมัติ
Page Access Tokenการเผยแพร่เพจ·การสร้างไลฟ์ของเพจต้องมีสิทธิ์ผู้ดูแลเพจและสิทธิ์ที่เกี่ยวข้อง
Long-Lived Tokenโทเค็นระยะยาวที่ขยายเวลาหมดอายุเมื่อใช้งานจริงต้องมีตรรกะการหมดอายุ·การต่ออายุ

7. รายการตรวจสอบ

  • สร้างแอป Meta Developer แล้ว
  • ยืนยัน App ID และ App Secret แล้ว
  • เชื่อมต่อเพจ Facebook และออก Page Access Token แล้ว
  • จัดระเบียบและขอสิทธิ์ที่จำเป็น (App Review) แล้ว
  • ป้อน access_token·page_id ใน GenieGo [ลงทะเบียน] และบันทึกแล้ว
  • จัดเก็บโทเค็นอย่างปลอดภัยในตัวแปรสภาพแวดล้อมของเซิร์ฟเวอร์แล้ว

8. ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้ไข
ไม่มีเมนู API KeyMeta ไม่ได้ใช้โครงสร้าง API Key เดียวดำเนินการด้วยโครงสร้าง App ID·App Secret·Access Token
ข้อผิดพลาดด้านสิทธิ์โทเค็นไม่มีสิทธิ์ที่จำเป็นเพิ่มสิทธิ์ใน Graph API Explorer แล้วออกโทเค็นใหม่
สร้างไลฟ์ของเพจไม่สำเร็จPage Access Token/สิทธิ์ผู้ดูแลไม่เพียงพอตรวจสอบสิทธิ์ผู้ดูแลเพจและสิทธิ์ของโทเค็น
