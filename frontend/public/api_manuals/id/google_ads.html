Google Ads
Panduan Penerbitan API Key

Panduan yang memandu Anda melalui proses penerbitan developer token di Pusat API akun pengelola (MCC) Google Ads Anda, membuat OAuth client di Google Cloud untuk memperoleh refresh token, dan mendaftarkan semuanya di GenieGo Integration Hub.

Perkiraan waktuSekitar 20–40 mnt (peninjauan developer token terpisah)
Akun yang diperlukanAkun pengelola (MCC) Google Ads · Akun Google
Kunci yang diterbitkandeveloper token · client_id/secret · refresh_token · customer_id
HasilKredensial Google Ads API

1. Sebelum Anda Mulai

Integrasi Google Ads memerlukan akun pengelola (MCC) sekaligus penyiapan OAuth di Google Cloud.

developer token

Pusat API MCC

Ajukan developer token di Pusat API akun pengelola (MCC) Google Ads Anda. Akses dasar (Basic) disetujui setelah peninjauan Google.

OAuth

Google Cloud client

Di console.cloud.google.com, buat OAuth client (client_id·client_secret) dan lakukan autentikasi untuk memperoleh refresh token.

Catatan: Akses dasar untuk developer token disetujui setelah peninjauan Google, dan sebelum disetujui hanya akun pengujian yang dapat dipanggil. Jika Anda tidak memiliki akun pengelola (MCC), buat satu terlebih dahulu di ads.google.com.

2. Langkah Penerbitan

Terbitkan sesuai urutan di bawah ini dan daftarkan di GenieGo.

1

Masuk ke akun pengelola · Buka Pusat API

Masuk ke akun pengelola Google Ads Anda, lalu buka [Alat & Setelan] > [Penyiapan] > [Pusat API] di bagian atas.

Alat & Setelan → Penyiapan → Pusat API
2

Ajukan developer token

Ajukan [developer token] di Pusat API. Akses dasar (Basic) disetujui setelah peninjauan Google, dan sebelum disetujui hanya akun pengujian yang dapat dipanggil.

3

Siapkan proyek Cloud · layar persetujuan OAuth

Buat proyek dan konfigurasikan [API & Layanan] > [Layar persetujuan OAuth].

console.cloud.google.com → API & Layanan → Layar persetujuan OAuth
4

Buat OAuth client ID

Buat [Kredensial] > [Buat kredensial] > [OAuth client ID] untuk memperoleh client_id dan client_secret (scope: adwords).

Kredensial → Buat kredensial → OAuth client ID
5

Terbitkan refresh token

Peroleh refresh token melalui proses autentikasi OAuth (gunakan Google OAuth Playground atau autentikasi Anda sendiri).

6

Temukan customer ID

Temukan customer ID 10 digit (customer_id) dari akun iklan yang akan Anda panggil.

Setelah developer token disetujui dan client_id·client_secret·refresh_token·customer_id semuanya siap, penerbitan selesai.

3. Informasi yang Didaftarkan di GenieGo

BidangDeskripsi
developer_tokendeveloper token yang diterbitkan di Pusat API
client_idOAuth client ID
client_secretOAuth client secret (tangani dengan hati-hati)
refresh_tokenrefresh token yang diterbitkan melalui autentikasi OAuth (masukkan di kolom [access token])
customer_idcustomer ID 10 digit dari akun iklan target
Masukkan developer token, client_id, client_secret, refresh_token, dan customer_id di GenieGo [Daftar] lalu simpan. Kredensial dienkripsi saat disimpan dengan AES-256-GCM dan disamarkan saat dibaca.

4. Masalah Umum

MasalahPenyebabSolusi
Hanya akun pengujian yang dapat dipanggilPeninjauan akses dasar developer token belum disetujuiTunggu hingga peninjauan Google atas developer token yang Anda ajukan di Pusat API disetujui, lalu panggil akun produksi.
Autentikasi gagalScope (adwords) hilang atau refresh_token salahLakukan autentikasi ulang dengan scope adwords disertakan dalam OAuth client, dan periksa kembali refresh_token.
Kesalahan format customer IDcustomer_id 10 digit salahPeriksa kembali customer ID 10 digit dari akun iklan target dan masukkan.

Daftar Periksa Akhir

  • Menyiapkan akun pengelola (MCC) Google Ads dan masuk.
  • Mengajukan developer token di Pusat API (akses dasar melalui peninjauan Google).
  • Membuat layar persetujuan OAuth dan OAuth client ID/Secret di console.cloud.google.com (scope: adwords).
  • Memperoleh refresh token melalui autentikasi OAuth.
  • Menemukan customer_id 10 digit dari akun target.
  • Memasukkan developer token·client_id/secret·refresh_token·customer_id di GenieGo [Daftar] dan menyimpannya.
