LINE
Panduan Penerbitan API

Panduan ini merangkum proses pembuatan Provider dan Messaging API Channel di LINE Developers Console serta konfigurasi Channel Access Token dan Webhook URL.

Struktur autentikasiChannel Access Token
Akun yang dibutuhkanAkun LINE
Hasil utamaChannel ID / Secret / Token
KegunaanPesan·Chatbot

1. Konsep Penerbitan LINE API

Integrasi LINE bekerja dengan membuat Provider dan Channel di LINE Developers Console, lalu menerbitkan Channel ID, Channel Secret, dan Channel Access Token untuk Messaging API atau LINE Login.

Pembedaan: Pengiriman pesan menggunakan kanal Messaging API, dan integrasi login menggunakan kanal LINE Login.

2. Yang Perlu Disiapkan

  • Akun LINE atau akun bisnis
  • Izin akses ke LINE Developers Console
  • LINE Official Account
  • Nama layanan dan deskripsi
  • Webhook URL
  • URL kebijakan privasi dan URL ketentuan

3. Membuat Provider

1

Akses LINE Developers

Masuk ke LINE Developers Console.

developers.line.biz
2

Buat Provider

Buat Provider menggunakan nama perusahaan atau nama layanan.

Create Provider
3

Masukkan informasi dasar

Masukkan penyelenggara layanan, detail kontak, dan informasi ketentuan.

Provider Settings

4. Membuat Kanal Messaging API

1

Buat kanal baru

Klik Create a new channel di dalam Provider.

Create a new channel
2

Pilih Messaging API

Pilih Messaging API sebagai jenis kanal.

Messaging API
3

Masukkan informasi kanal

Masukkan nama kanal, deskripsi, kategori, email, dan sebagainya.

Channel Information
4

Tautkan LINE Official Account

Tautkan akun resmi untuk mengirim pesan, atau buat yang baru.

LINE Official Account
5

Selesaikan pembuatan kanal

Setelah dibuat, periksa Channel ID dan Channel Secret.

Channel ID / Channel Secret

5. Menerbitkan Channel Access Token

1

Buka tab Messaging API

Pada layar kanal yang Anda buat, pindah ke tab Messaging API.

Messaging API tab
2

Temukan item Channel access token

Periksa area Channel access token di bagian bawah.

Channel access token
3

Klik Issue

Klik tombol Issue atau Reissue untuk menerbitkan token.

Issue
4

Simpan dengan aman di server

Simpan token yang diterbitkan di variabel lingkungan server atau Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
Channel Access Token adalah kunci rahasia dengan izin untuk mengirim pesan. Tidak boleh diekspos di HTML, frontend, atau repositori publik.

6. Mengonfigurasi Webhook

1

Masukkan Webhook URL

Masukkan URL server yang akan menerima event pesan pengguna.

Webhook URL
2

Aktifkan Use webhook

Ubah opsi Use webhook menjadi Enabled.

Use webhook: Enabled
3

Uji Verify

Kirim permintaan uji dari LINE ke server Anda untuk mengonfirmasi koneksi.

Verify
4

Periksa kode respons

Konfirmasikan bahwa server mengembalikan 200 OK.

HTTP 200 OK

7. Contoh Pengiriman Pesan

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Halo. Ini adalah uji pesan LINE."} ] }

8. Daftar Periksa Akhir

  • Membuat Provider.
  • Membuat kanal Messaging API.
  • Menautkan LINE Official Account.
  • Memeriksa Channel ID dan Channel Secret.
  • Menerbitkan Channel Access Token.
  • Mendaftarkan Webhook URL dan lulus Verify.
  • Menyimpan token dengan aman di variabel lingkungan server.
