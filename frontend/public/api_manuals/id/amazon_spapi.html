Amazon SP-API
Manual Penerbitan

Panduan tentang alur registrasi developer SP-API, pembuatan aplikasi, dan penyiapan OAuth·LWA untuk menghubungkan data penjual Amazon Anda dengan sistem pesanan, inventaris, produk, dan pengiriman.

Struktur autentikasiLWA(OAuth) — 5 kredensial
Akun yang dibutuhkanSeller/Vendor Central (Professional)
Nilai GenieGorefresh_token·client_id·secret·marketplace_id·seller_id
Tingkat kesulitanMenengah ke atas

1. Apa itu Amazon SP-API?

Amazon Selling Partner API adalah API untuk mengintegrasikan data penjual secara terprogram seperti pesanan, inventaris, produk, pengiriman, dan penyelesaian.

Penting: SP-API bukan skema API Key sederhana. Umumnya memerlukan registrasi Amazon Developer, registrasi aplikasi, Login with Amazon(LWA), dan otorisasi OAuth secara bersamaan.
★Integrasi GenieGo memerlukan 5 nilai: refresh_token, client_id, client_secret, marketplace_id, seller_id. (SP-API saat ini dipanggil hanya dengan token LWA, dan penandatanganan AWS IAM/SigV4 ditangani secara internal oleh GenieGo, sehingga Anda tidak memasukkannya di kolom registrasi.)

2. Prasyarat

  • Akun penjual Professional Amazon Seller Central atau Vendor Central
  • Informasi perusahaan/kontak untuk registrasi developer
  • Nama aplikasi, deskripsi aplikasi, dan tujuan penggunaan
  • URL kebijakan privasi
  • OAuth Redirect URL

3. Registrasi Developer

Pertama, daftarkan profil developer SP-API Anda di Seller Central.

1

Masuk ke Seller Central

Masuk ke Seller Central dengan akun penjual Amazon Anda.

Seller Central → Apps & Services
2

Buka menu Develop Apps

Dari menu Apps & Services, buka menu developer atau pengembangan aplikasi.

Apps & Services → Develop Apps
3

Isi Developer Profile

Masukkan informasi perusahaan, informasi kontak, tujuan penggunaan API, dan rencana perlindungan data.

Developer Profile
4

Kirim untuk ditinjau

Kirim informasi registrasi developer Anda untuk ditinjau Amazon.

Submit for review
Hanya setelah persetujuan selesai, Anda dapat mendaftarkan aplikasi SP-API produksi dan menggunakan izinnya.

4. Daftarkan Aplikasi SP-API → Client ID / Client Secret

1

Buat aplikasi baru

Buat aplikasi baru di Develop Apps.

Add new app
2

Pilih jenis aplikasi

Jika Anda hanya menghubungkan akun sendiri, itu adalah Private App; jika Anda menyediakannya untuk penjual eksternal, itu adalah Public App.

Private / Public App
3

Pilih peran API

Pilih peran yang Anda butuhkan, seperti Orders, Catalog, dan Inventory.

API Roles
4

Masukkan OAuth Redirect URL

Masukkan URL server tempat kembali setelah otorisasi selesai.

Redirect URI
5

Simpan dan periksa info Client

Setelah membuat aplikasi, periksa client_id (LWA Client ID) dan client_secret (Client Secret).

client_id / client_secret

5. Otorisasi OAuth → Terbitkan Refresh Token

1) Akses URL otorisasi OAuth penjual 2) Terima authorization code di Redirect URL 3) Teruskan authorization code ke LWA Token Endpoint 4) Terima refresh_token / access_token
refresh_token digunakan untuk panggilan jangka panjang setelah otorisasi penjual. GenieGo secara otomatis menyegarkan access_token menggunakan refresh_token ini untuk memanggil SP-API, jadi Anda hanya perlu mendaftarkan refresh_token tanpa penyiapan penandatanganan AWS terpisah.

6. Periksa Marketplace ID · Seller ID

Periksa dua pengenal yang tersisa yang dibutuhkan untuk registrasi GenieGo.

marketplace_id

Marketplace ID target penjualan

Marketplace ID negara yang Anda hubungkan. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Dibutuhkan untuk registrasi produk dan pencarian pesanan. Periksa di informasi akun Seller Central Anda. Seller Central → Pengaturan → Info Akun → Merchant Token

7. Daftarkan di GenieGo Integration Hub

1

Siapkan 5 nilai

Siapkan refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 kredensial
2

Integration Hub → Amazon SP-API → [Daftar]

Masukkan setiap nilai di kolomnya dan simpan (kolom kosong mempertahankan nilai yang ada).

refresh_token · client_id · client_secret · marketplace_id · seller_id
Nilai yang terdaftar dienkripsi saat disimpan dengan AES-256-GCM dan disamarkan saat dibaca. Sinkronisasi otomatis dimulai segera setelah menyimpan.
client_secret·refresh_token tidak boleh diekspos di kode frontend, HTML, atau repositori publik.

8. Daftar Periksa Akhir

  • Telah mengirimkan Developer Profile dan disetujui.
  • Telah membuat aplikasi SP-API dan memeriksa client_id·client_secret.
  • Telah menerbitkan refresh_token melalui otorisasi OAuth.
  • Telah memeriksa marketplace_id untuk dihubungkan.
  • Telah memeriksa seller_id (Merchant Token) di Seller Central.
  • Telah memasukkan 5 nilai (refresh_token·client_id·client_secret·marketplace_id·seller_id) di GenieGo [Daftar] dan menyimpannya.
