Rakuten
Panduan Penerbitan API Key

Panduan langkah demi langkah yang ramah pemula untuk menerbitkan kredensial RMS Web Service (Service Secret dan License Key) dari konsol penjual Rakuten RMS dan mendaftarkannya di GenieGo Integration Hub.

Perkiraan waktuSekitar 5–15 menit
Akun yang dibutuhkanAkun penjual Rakuten (Ichiba)
Kunci yang diterbitkanService Secret · License Key
HasilKredensial RMS Web Service

1. Sebelum Anda Mulai

Kunci penjual (Ichiba) diterbitkan dari RMS Web Service.

Diperlukan

Akun penjual Rakuten

Anda memerlukan akun penjual Rakuten (Ichiba). Anda memerlukan akun yang dapat masuk melalui RMS R-Login (glogin.rms.rakuten.co.jp).

Kunci yang diterbitkan

Service Secret · License Key

Anda akan memperoleh serviceSecret dan licenseKey dari RMS Web Service. License Key kedaluwarsa setelah 90 hari dan dapat diterbitkan ulang.

Perhatian: RMS Web Service (glogin.rms.rakuten.co.jp) yang menerbitkan kunci penjual (Ichiba) berbeda dari webservice.rakuten.co.jp, yang ditujukan untuk afiliasi/pencarian produk. Pastikan untuk menerbitkan kunci Anda di sisi RMS.

2. Langkah Penerbitan

Terbitkan sesuai urutan di bawah dan daftarkan di GenieGo.

1

Masuk ke portal pengembang RMS

Masuk ke portal pengembang Rakuten (RMS/Webservice) dengan akun penjual Anda.

https://glogin.rms.rakuten.co.jp
2

Buka RMS Web Service

Setelah masuk melalui RMS R-Login, navigasikan ke area RMS Web Service.

RMS R-Login → RMS Web Service
3

Terbitkan Service Secret · License Key

Dari RMS Web Service, terbitkan Service Secret dan License Key. License Key kedaluwarsa setelah 90 hari, jadi terbitkan ulang saat kedaluwarsa.

4

Verifikasi kunci

Verifikasi Service Secret dan License Key yang diterbitkan, beserta Shop URL toko Anda.

Setelah Service Secret dan License Key muncul di layar penerbitan, berarti keduanya telah diterbitkan dengan benar.

3. Informasi yang Harus Didaftarkan di GenieGo

BidangDeskripsi
service_secretService Secret yang diterbitkan
license_keyLicense Key yang diterbitkan (kedaluwarsa dalam 90 hari·dapat diterbitkan ulang, tangani dengan hati-hati)
shop_urlShop URL toko Anda
Di GenieGo Integration Hub [Daftar], masukkan service_secret·license_key·shop_url lalu simpan. Kredensial dienkripsi saat disimpan dengan AES-256-GCM dan disamarkan saat dibaca.

4. Masalah Umum

MasalahPenyebabSolusi
Kunci yang salah diterbitkanDiterbitkan dari webservice.rakuten.co.jp (untuk afiliasi/pencarian produk)Terbitkan ulang dari RMS Web Service di bawah RMS R-Login (glogin.rms.rakuten.co.jp).
Autentikasi gagal setelah beberapa waktuLicense Key kedaluwarsa setelah 90 hariTerbitkan ulang License Key dari RMS Web Service dan masukkan kembali di [Daftar].

Daftar Periksa Akhir

  • Masuk ke RMS R-Login dengan akun penjual Rakuten (Ichiba).
  • Menerbitkan Service Secret·License Key dari RMS Web Service.
  • Memverifikasi Shop URL toko Anda.
  • Memasukkan service_secret·license_key·shop_url di GenieGo [Daftar] dan menyimpan.
