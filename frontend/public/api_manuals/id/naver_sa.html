Naver Search Ads
Panduan Penerbitan API Key

Panduan langkah demi langkah yang ramah pemula untuk menerbitkan kredensial API (lisensi akses, kunci rahasia) dan CUSTOMER_ID Anda dari alat pengelolaan Naver Search Ad serta mendaftarkannya di GenieGo Integration Hub.

Perkiraan waktuSekitar 5–10 mnt
Akun yang diperlukanAkun pengiklan Naver Search Ad
Kunci yang diterbitkanLisensi akses · Kunci rahasia · CUSTOMER_ID
HasilKredensial Search Ad API

1. Sebelum Anda Mulai

Naver Search Ad API hanya dapat diminta dari akun pengiklan.

Wajib

Akun pengiklan Search Ad

Anda memerlukan akun pengiklan yang menggunakan alat pengelolaan Naver Search Ad. Anda tidak dapat mengajukan akses API dengan akun anggota Naver biasa.

Kunci yang diterbitkan

Lisensi akses · Kunci rahasia · CUSTOMER_ID

Konfirmasikan CUSTOMER_ID, lisensi akses, dan kunci rahasia yang diterbitkan bersamaan setelah Anda mengajukan akses API.

Catatan: Kunci rahasia hanya dapat dilihat pada saat penerbitan, jadi simpanlah dengan aman segera setelah diterbitkan.

2. Langkah Penerbitan API Key

Terbitkan dalam urutan di bawah ini dan daftarkan di GenieGo.

1

Masuk ke Search Ads

Masuk ke alat pengelolaan Naver Search Ad dengan akun pengiklan Anda.

searchad.naver.com
2

Buka pengelolaan penggunaan API

Dari menu [Alat] di bagian atas, buka [Pengelolaan Penggunaan API].

Alat → Pengelolaan Penggunaan API
3

Ajukan layanan API

Klik tombol [Ajukan Layanan Naver Search Ad API] untuk mengajukan.

Ajukan Layanan Naver Search Ad API
4

Periksa informasi yang diterbitkan

Konfirmasikan CUSTOMER_ID, lisensi akses, dan kunci rahasia yang diterbitkan.

Setelah CUSTOMER_ID, lisensi akses, dan kunci rahasia semuanya dikonfirmasi, berarti telah diterbitkan dengan benar.

3. Informasi yang Didaftarkan di GenieGo

BidangDeskripsi
api_keyLisensi akses yang diterbitkan
api_secretKunci rahasia yang diterbitkan (tangani dengan hati-hati)
customer_idCUSTOMER_ID yang diterbitkan
Masukkan dan simpan di [Daftar] GenieGo Integration Hub, dan kredensial dienkripsi saat disimpan dengan AES-256-GCM dan disamarkan saat dibaca.

4. Masalah Umum

MasalahPenyebabSolusi
Tidak ada menu [Pengelolaan Penggunaan API]Bukan akun pengiklan, atau izin tidak mencukupiMasuk dengan akun pengiklan yang menggunakan alat pengelolaan Search Ad.
Terjadi kesalahan autentikasiLisensi akses atau kunci rahasia dimasukkan dengan salahPeriksa kembali lisensi akses, kunci rahasia, dan CUSTOMER_ID dari layar penerbitan.

Daftar Periksa Akhir

  • Telah masuk dengan akun pengiklan Naver Search Ad.
  • Telah menyelesaikan pengajuan layanan API di [Alat > Pengelolaan Penggunaan API].
  • Telah mengonfirmasi CUSTOMER_ID, lisensi akses, dan kunci rahasia.
  • Telah memasukkan api_key·api_secret·customer_id di GenieGo [Daftar] dan menyimpannya.
