Manual de Emisión de API

Este manual reúne los pasos prácticos desde la verificación comercial del canal de Kakao Talk, el registro del perfil de remitente y la aprobación de la plantilla de Alimtalk hasta la confirmación de tus credenciales de API.

1. Concepto de emisión de Kakao Alimtalk

Alimtalk es un servicio de mensajería empresarial que envía mensajes informativos como pedidos, reservas, envíos y pagos a través de Kakao Talk. No es una estructura que permita enviar directamente solo con una clave de aplicación de desarrollador de Kakao común, sino que requiere la verificación comercial del canal y la configuración de una agencia de envío/plataforma.

Importante: Alimtalk se centra en mensajes informativos en lugar de promocionales, y las plantillas requieren una revisión previa.

2. Requisitos

  • Cuenta de Kakao Business
  • Canal de Kakao Talk
  • Documentos de verificación comercial, como un certificado de registro mercantil
  • El texto de la plantilla de Alimtalk que vas a enviar
  • Una cuenta de agencia de envío o de servicio de mensajería en la nube
  • Una política de recopilación de números de teléfono de destinatarios y gestión del consentimiento

3. Preparación del canal de Kakao Talk

1

Acceder a Kakao Business

Inicia sesión en el centro de administración de Kakao Business.

business.kakao.com
2

Crear un canal de Kakao Talk

Crea un canal con el nombre de tu marca o servicio.

Centro de Administración de Canales de Kakao Talk
3

Realizar la verificación comercial

Envía la información de tu empresa y los documentos justificativos para verificar el canal comercial.

Verificación comercial
4

Comprobar el ID de búsqueda del canal

Comprueba el ID del canal o el ID de búsqueda necesarios para la configuración de envío de Alimtalk.

Perfil del canal → ID de búsqueda
La verificación comercial debe completarse antes de poder registrar un perfil de remitente de Alimtalk.

4. Registro del perfil de remitente / Sender Profile

1

Elegir una plataforma de envío

Selecciona un proveedor de Alimtalk como NHN Cloud, Naver Cloud SENS, Infobank o Bizgo.

Messaging Provider
2

Crear un proyecto

Crea un proyecto de Alimtalk en la plataforma que elegiste.

Project / Service
3

Conectar el canal de Kakao Talk

Registra la información de tu canal de Kakao Talk verificado comercialmente.

Sender Profile
4

Revisión del perfil de remitente

Realiza la confirmación de la propiedad del canal y la revisión del perfil de remitente.

Profile Review
5

Confirmar las credenciales de API

Comprueba el App Key, Secret Key, API Key, etc. que proporciona la plataforma.

API Key / Secret

5. Registro y aprobación de plantillas

1

Redactar la plantilla

Redacta textos informativos como confirmaciones de pedido y avisos de envío.

Template Message
2

Definir variables

Define como variables los valores que cambian dinámicamente, como el nombre, el número de pedido y el número de envío.

#{variable}
3

Solicitar la revisión

Guarda la plantilla y envía la solicitud de revisión de Kakao.

Request Approval
4

Confirmar la aprobación

Confirma el código de plantilla o el ID aprobado.

Template Code
Si el texto antes de la revisión difiere del texto realmente enviado, el envío puede fallar o pueden aplicarse sanciones.

6. Ejemplo de flujo de envío por API

1) Llama al Alimtalk API Endpoint desde tu servidor
2) Autentícate con la API Key o el Secret
3) Pasa el número de teléfono del destinatario, el código de plantilla y los valores de las variables
4) Recibe el resultado del envío
5) Gestiona si enviar un SMS de respaldo en caso de fallo

7. Lista de verificación final

  • Creé un canal de Kakao Talk.
  • Completé la verificación comercial.
  • Seleccioné una plataforma de envío de Alimtalk.
  • Registré un Sender Profile.
  • Confirmé la API Key o el Secret.
  • Registré la plantilla y obtuve su aprobación.
  • Preparé una política de fallos de envío y SMS de respaldo.
