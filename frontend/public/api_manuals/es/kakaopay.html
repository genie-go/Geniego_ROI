KakaoPay
Manual de emisión de API Key

Una guía que le explica cómo crear una aplicación en el Centro de Desarrolladores de KakaoPay (developers.kakaopay.com) para emitir un código de comercio (CID) y una Secret Key, y luego registrarlos en el GenieGo Integration Hub.

Tiempo estimadoUnos 5–15 min
Cuenta necesariaContrato de comercio KakaoPay · Cuenta del Centro de Desarrolladores
Claves emitidasCódigo de comercio (CID) · Secret Key
ResultadoCredenciales de integración de pagos

1. Antes de empezar

KakaoPay emite las credenciales en el Centro de Desarrolladores una vez que existe un contrato de comercio.

Requerido

Contrato de comercio · Cuenta del Centro de Desarrolladores

Su contrato de comercio de KakaoPay debe estar completo, y debe poder registrarse e iniciar sesión en el Centro de Desarrolladores (developers.kakaopay.com).

Claves emitidas

CID · Secret Key

Cuando crea una aplicación, se emiten un código de comercio (CID) y una Secret Key.

Nota: Al llamar a la API de KakaoPay, el encabezado de autenticación usa el formato Authorization: SECRET_KEY.

2. Pasos de emisión

Emita en el orden siguiente y registre en GenieGo.

1

Registrarse e iniciar sesión en el Centro de Desarrolladores

Regístrese en el Centro de Desarrolladores de KakaoPay e inicie sesión.

https://developers.kakaopay.com
2

Crear una aplicación

Cree una aplicación para emitir o comprobar el código de comercio (CID) y la Secret Key.

3

Comprobar el encabezado de autenticación

La Secret Key emitida se usa en el encabezado de autenticación con el formato Authorization: SECRET_KEY.

Authorization: SECRET_KEY ${secret_key}
Una vez emitidos el código de comercio (CID) y la Secret Key, todo está correctamente preparado.

3. Información para registrar en GenieGo

CampoDescripción
cidEl código de comercio (CID) emitido
secret_keyLa Secret Key emitida (manéjela con cuidado, Authorization: SECRET_KEY)
Introdúzcalos y guárdelos en el GenieGo Integration Hub [Registrar], y las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas comunes

ProblemaCausaSolución
Error en el encabezado de autenticaciónEl formato del encabezado Authorization es incorrectoCompruebe que la Secret Key se pasa con el formato Authorization: SECRET_KEY.
No se puede crear la aplicaciónContrato de comercio no completadoComplete el contrato de comercio y luego cree la aplicación en el Centro de Desarrolladores.

Lista de verificación final

  • Completé el contrato de comercio de KakaoPay y me registré en el Centro de Desarrolladores (developers.kakaopay.com).
  • Creé una aplicación y emití el código de comercio (CID) y la Secret Key.
  • Verifiqué el formato del encabezado de autenticación Authorization: SECRET_KEY.
  • Introduje cid·secret_key en GenieGo [Registrar] y guardé.
