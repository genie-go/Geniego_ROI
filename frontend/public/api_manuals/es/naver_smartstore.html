Naver Smart Store
Manual de emisión de claves de API

Una guía paso a paso para registrar una aplicación en el Naver Commerce API Center (apicenter.commerce.naver.com) a fin de obtener un Client ID y un Client Secret, y luego registrarlos en el GenieGo Integration Hub.

Tiempo estimadoUnos 10–15 min
Cuenta necesariaCuenta de Smart Store con permiso de "Gestor integrado"
Claves emitidasClient ID · Client Secret
ResultadoCredenciales de Commerce API

1. Antes de empezar

La emisión solo es posible con el permiso de Gestor integrado en una cuenta de vendedor de Smart Store.

Obligatorio

Permiso de Gestor integrado

Debe tener el permiso de "Gestor integrado" en una cuenta de vendedor de Smart Store. Solo un Gestor integrado puede emitir las claves.

Claves emitidas

Client ID · Client Secret

Cuando registra una aplicación, se emiten un ID de aplicación (Client ID) y un Secret (Client Secret).

Nota: Es obligatorio registrar la IP del servidor que llamará a la API y renovar periódicamente la autenticación de la aplicación.

2. Pasos de emisión

Emita en el orden siguiente y regístrelo en GenieGo.

1

Acceder al Commerce API Center

Inicie sesión en el Naver Commerce API Center con su cuenta de Gestor integrado.

https://apicenter.commerce.naver.com
2

Crear una cuenta

Haga clic en [Crear cuenta] en la parte superior derecha, introduzca el nombre de cuenta de la empresa desarrolladora y un contacto de respuesta a incidentes, acepte los términos y regístrese.

Crear cuenta
3

Registrar una aplicación

En [Mis aplicaciones de tienda], registre una aplicación; a continuación se emiten un ID de aplicación y un Secret.

Mis aplicaciones de tienda
4

Registrar la IP y renovar la autenticación

Registre la IP del servidor que llamará a la API y renueve periódicamente la autenticación de la aplicación según las indicaciones.

Una vez registrada la aplicación y emitidos el ID de aplicación y el Secret, se ha procesado correctamente.

3. Información para registrar en GenieGo

CampoDescripción
client_idEl ID de aplicación emitido (Client ID)
client_secretEl Secret emitido (Client Secret, manéjelo con cuidado)
Introdúzcalos en [Registrar] del GenieGo Integration Hub y guarde; las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
No se puede registrar una aplicaciónLa cuenta no tiene permiso de Gestor integradoAcceda a apicenter.commerce.naver.com con una cuenta que tenga permiso de Gestor integrado.
Error de autenticación en la llamadaIP del llamante no registrada o renovación de autenticación omitidaRegistre la IP del servidor llamante y renueve periódicamente la autenticación de la aplicación.

Lista de verificación final

  • Accedí al Commerce API Center con una cuenta de Gestor integrado.
  • Introduje la información de la empresa desarrolladora y un contacto de respuesta a incidentes mediante [Crear cuenta] y acepté los términos.
  • Registré una aplicación en [Mis aplicaciones de tienda] y obtuve el ID/Secret.
  • Registré la IP del servidor llamante y configuré la autenticación periódica.
  • Introduje client_id·client_secret en GenieGo [Registrar] y guardé.
