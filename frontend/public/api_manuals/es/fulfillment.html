Fulfillment (3PL)
Manual de emisión de API Key

Una guía para firmar un contrato de consignación de fulfillment 3PL (entrada/salida de almacén) y registrar la 3PL API key y el warehouse ID emitidos en el GenieGo Integration Hub. En este canal, las credenciales se reciben mediante un contrato de consignación, no por emisión desde una consola de autoservicio.

Tiempo estimadoVaría según el avance del contrato
Cuenta necesariaContrato de consignación de fulfillment 3PL
Claves emitidas3PL API key · warehouse ID
ResultadoCredenciales de fulfillment API

1. Antes de comenzar

Fulfillment (3PL) es un canal basado en contrato en el que las credenciales se emiten tras un contrato de consignación de entrada/salida de almacén.

Requerido

Contrato de consignación de entrada/salida de almacén

Para usar fulfillment 3PL, primero debe existir un contrato de consignación de entrada/salida de almacén con un proveedor de fulfillment o el equipo de operaciones.

Claves emitidas

3PL API key · warehouse ID

Tras completarse el contrato, se le emite un warehouse ID y una 3PL API key.

Nota: Este canal no emite las credenciales directamente desde una consola de autoservicio; en su lugar, recibe las credenciales mediante un contrato de consignación.

2. Procedimiento de solicitud de emisión

Solicite y reciba en el orden siguiente y luego registre en GenieGo.

1

Avanzar con el contrato de consignación

Avance con un contrato de consignación de entrada/salida de almacén con un proveedor de fulfillment o el equipo de operaciones.

2

Firmar el contrato

Firme el contrato de consignación de entrada/salida de almacén de fulfillment 3PL.

3

Recibir las credenciales

Tras completarse el contrato, se le emite un warehouse ID·3PL API key.

4

Registrar en GenieGo

Introduzca el api_key y el warehouse_id emitidos en GenieGo [Registrar] y guarde.

Si ha recibido tanto el warehouse ID·3PL API key del proveedor de fulfillment, se han emitido correctamente.

3. Información para registrar en GenieGo

CampoDescripción
api_keyLa 3PL API key emitida
warehouse_idEl warehouse ID emitido
Cuando los introduce y guarda en el GenieGo Integration Hub [Registrar], las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
No se reciben las credencialesContrato de consignación de entrada/salida de almacén no completadoFirme un contrato de consignación con un proveedor de fulfillment o el equipo de operaciones y luego haga emitir las credenciales.
Error de integración de entrada/salida de almacénwarehouse ID (warehouse_id) introducido incorrectamenteVuelva a comprobar el warehouse ID emitido e introdúzcalo correctamente.

Lista de verificación final

  • Firmé un contrato de consignación de entrada/salida de almacén con un proveedor de fulfillment o el equipo de operaciones.
  • Recibí el warehouse ID·3PL API key tras completarse el contrato.
  • Confirmé api_key·warehouse_id.
  • Introduje api_key·warehouse_id en GenieGo [Registrar] y guardé.
