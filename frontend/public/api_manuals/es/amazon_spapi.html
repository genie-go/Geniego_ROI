Amazon SP-API
Manual de emisión

Una guía sobre el flujo de registro de desarrollador de SP-API, creación de la aplicación y configuración de OAuth·LWA para conectar tus datos de vendedor de Amazon con los sistemas de pedidos, inventario, productos y envíos.

Estructura de autenticaciónLWA(OAuth) — 5 credenciales
Cuenta necesariaSeller/Vendor Central (Professional)
Valores de GenieGorefresh_token·client_id·secret·marketplace_id·seller_id
DificultadIntermedia o superior

1. ¿Qué es la Amazon SP-API?

La Amazon Selling Partner API es una API para integrar mediante programación los datos del vendedor, como pedidos, inventario, productos, envíos y liquidaciones.

Importante: La SP-API no es un esquema simple de API Key. Generalmente requiere en conjunto el registro de Amazon Developer, el registro de la aplicación, Login with Amazon(LWA) y la autorización OAuth.
★La integración con GenieGo requiere 5 valores: refresh_token, client_id, client_secret, marketplace_id, seller_id. (La SP-API actual se invoca solo con el token LWA, y la firma AWS IAM/SigV4 la gestiona internamente GenieGo, por lo que no la introduces en los campos de registro.)

2. Requisitos previos

  • Una cuenta de vendedor Professional de Amazon Seller Central o Vendor Central
  • Información de la empresa/contacto para el registro de desarrollador
  • Nombre de la aplicación, descripción de la aplicación y uso previsto
  • URL de la política de privacidad
  • OAuth Redirect URL

3. Registro de desarrollador

Primero, registra tu perfil de desarrollador de SP-API en Seller Central.

1

Inicia sesión en Seller Central

Inicia sesión en Seller Central con tu cuenta de vendedor de Amazon.

Seller Central → Apps & Services
2

Ve al menú Develop Apps

Desde el menú Apps & Services, ve al menú de desarrollador o de desarrollo de aplicaciones.

Apps & Services → Develop Apps
3

Completa el Developer Profile

Introduce la información de la empresa, la información de contacto, el propósito de uso de la API y el plan de protección de datos.

Developer Profile
4

Enviar para revisión

Envía tu información de registro de desarrollador para la revisión de Amazon.

Submit for review
Solo después de completar la aprobación podrás registrar una aplicación SP-API de producción y usar sus permisos.

4. Registrar una aplicación SP-API → Client ID / Client Secret

1

Crea una nueva aplicación

Crea una nueva aplicación en Develop Apps.

Add new app
2

Elige el tipo de aplicación

Si conectas solo tu propia cuenta, es una Private App; si la proporcionas a vendedores externos, es una Public App.

Private / Public App
3

Selecciona los roles de API

Selecciona los roles que necesitas, como Orders, Catalog e Inventory.

API Roles
4

Introduce la OAuth Redirect URL

Introduce la URL del servidor a la que volver después de completar la autorización.

Redirect URI
5

Guarda y comprueba la info de Client

Tras crear la aplicación, comprueba el client_id (LWA Client ID) y el client_secret (Client Secret).

client_id / client_secret

5. Autorización OAuth → Emitir un Refresh Token

1) Accede a la URL de autorización OAuth del vendedor 2) Recibe el authorization code en la Redirect URL 3) Pasa el authorization code al LWA Token Endpoint 4) Recibe el refresh_token / access_token
El refresh_token se usa para llamadas a largo plazo tras la autorización del vendedor. GenieGo actualiza automáticamente el access_token usando este refresh_token para llamar a la SP-API, por lo que solo necesitas registrar el refresh_token sin ninguna configuración de firma de AWS por separado.

6. Comprueba el Marketplace ID · Seller ID

Comprueba los dos identificadores restantes necesarios para el registro en GenieGo.

marketplace_id

ID del marketplace de destino

El Marketplace ID del país que estás conectando. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Necesario para el registro de productos y la consulta de pedidos. Compruébalo en la información de tu cuenta de Seller Central. Seller Central → Configuración → Información de la cuenta → Merchant Token

7. Registrar en el GenieGo Integration Hub

1

Prepara los 5 valores

Prepara refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 credenciales
2

Integration Hub → Amazon SP-API → [Registrar]

Introduce cada valor en su campo y guarda (los campos en blanco mantienen los valores existentes).

refresh_token · client_id · client_secret · marketplace_id · seller_id
Los valores registrados se almacenan cifrados con AES-256-GCM y se enmascaran al consultarlos. La sincronización automática comienza inmediatamente después de guardar.
El client_secret·refresh_token nunca debe exponerse en el código del frontend, HTML ni repositorios públicos.

8. Lista de verificación final

  • Envié el Developer Profile y obtuve su aprobación.
  • Creé la aplicación SP-API y comprobé el client_id·client_secret.
  • Emití un refresh_token mediante la autorización OAuth.
  • Comprobé el marketplace_id a conectar.
  • Comprobé el seller_id (Merchant Token) en Seller Central.
  • Introduje los 5 valores (refresh_token·client_id·client_secret·marketplace_id·seller_id) en GenieGo [Registrar] y los guardé.

9. Problemas frecuentes

ProblemaCausaSolución
No hay menú de API KeyLa SP-API no es una estructura de API Key únicaProcede con la estructura LWA (client_id·client_secret·refresh_token).
El registro de productos fallaseller_id (Merchant Token) no registradoRegistra el Merchant Token desde la información de tu cuenta de Seller Central.
Datos de país incorrectosmarketplace_id no coincideRegistra el Marketplace ID correcto para el país que estás conectando.
Error de permisosAPI Role de la aplicación insuficiente o token caducadoComprueba el API Role y actualiza el token con el refresh_token.
