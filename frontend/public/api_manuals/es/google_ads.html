Google Ads
Manual de emisión de API Key

Una guía que le explica cómo emitir un developer token en el Centro de API de su cuenta de administrador (MCC) de Google Ads, crear un cliente OAuth en Google Cloud para obtener un refresh token y registrar todo en el GenieGo Integration Hub.

Tiempo estimadoUnos 20–40 min (revisión del developer token aparte)
Cuenta necesariaCuenta de administrador (MCC) de Google Ads · Cuenta de Google
Claves emitidasdeveloper token · client_id/secret · refresh_token · customer_id
ResultadoCredenciales de Google Ads API

1. Antes de empezar

La integración de Google Ads requiere tanto una cuenta de administrador (MCC) como la configuración de OAuth en Google Cloud.

developer token

Centro de API de MCC

Solicite un developer token en el Centro de API de su cuenta de administrador (MCC) de Google Ads. El acceso básico (Basic) se aprueba tras una revisión de Google.

OAuth

Cliente de Google Cloud

En console.cloud.google.com, cree un cliente OAuth (client_id·client_secret) y autentíquese para obtener un refresh token.

Nota: El acceso básico del developer token se aprueba tras una revisión de Google, y antes de la aprobación solo se pueden llamar cuentas de prueba. Si no tiene una cuenta de administrador (MCC), cree una primero en ads.google.com.

2. Pasos de emisión

Emita en el orden siguiente y regístrelo en GenieGo.

1

Inicie sesión en la cuenta de administrador · Vaya al Centro de API

Inicie sesión en su cuenta de administrador de Google Ads y luego vaya a [Herramientas y configuración] > [Configuración] > [Centro de API] en la parte superior.

Herramientas y configuración → Configuración → Centro de API
2

Solicite un developer token

Solicite un [developer token] en el Centro de API. El acceso básico (Basic) se aprueba tras una revisión de Google, y antes de la aprobación solo se pueden llamar cuentas de prueba.

3

Configure un proyecto de Cloud · pantalla de consentimiento de OAuth

Cree un proyecto y configure [API y servicios] > [Pantalla de consentimiento de OAuth].

console.cloud.google.com → API y servicios → Pantalla de consentimiento de OAuth
4

Cree un ID de cliente de OAuth

Cree [Credenciales] > [Crear credenciales] > [ID de cliente de OAuth] para obtener un client_id y un client_secret (scope: adwords).

Credenciales → Crear credenciales → ID de cliente de OAuth
5

Emita un refresh token

Obtenga un refresh token mediante el proceso de autenticación de OAuth (use Google OAuth Playground o su propia autenticación).

6

Encuentre el ID de cliente

Encuentre el ID de cliente de 10 dígitos (customer_id) de la cuenta de anuncios que va a llamar.

Una vez que el developer token esté aprobado y client_id·client_secret·refresh_token·customer_id estén todos listos, la emisión está completa.

3. Información para registrar en GenieGo

CampoDescripción
developer_tokenEl developer token emitido en el Centro de API
client_idID de cliente de OAuth
client_secretSecreto de cliente de OAuth (manéjelo con cuidado)
refresh_tokenEl refresh token emitido mediante autenticación de OAuth (introdúzcalo en el campo [Token de acceso])
customer_idEl ID de cliente de 10 dígitos de la cuenta de anuncios de destino
Introduzca el developer token, client_id, client_secret, refresh_token y customer_id en GenieGo [Registrar] y guarde. Las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
Solo se pueden llamar cuentas de pruebaRevisión del acceso básico del developer token no aprobadaEspere hasta que se apruebe la revisión de Google del developer token que solicitó en el Centro de API y luego llame a las cuentas de producción.
Falla la autenticaciónFalta el scope (adwords) o el refresh_token es incorrectoVuelva a autenticarse con el scope adwords incluido en el cliente OAuth y vuelva a verificar el refresh_token.
Error de formato del ID de clientecustomer_id de 10 dígitos incorrectoVuelva a verificar el ID de cliente de 10 dígitos de la cuenta de anuncios de destino e introdúzcalo.

Lista de verificación final

  • Preparé una cuenta de administrador (MCC) de Google Ads e inicié sesión.
  • Solicité un developer token en el Centro de API (el acceso básico pasa por una revisión de Google).
  • Creé una pantalla de consentimiento de OAuth y un ID/Secreto de cliente de OAuth en console.cloud.google.com (scope: adwords).
  • Obtuve un refresh token mediante autenticación de OAuth.
  • Encontré el customer_id de 10 dígitos de la cuenta de destino.
  • Introduje el developer token·client_id/secret·refresh_token·customer_id en GenieGo [Registrar] y lo guardé.
