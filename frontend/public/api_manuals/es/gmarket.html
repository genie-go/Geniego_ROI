Gmarket
Manual de emisión de API Key

Una guía para emitir la clave API compartida de Gmarket·Auction desde la ESM Trading API (etapi.gmarket.com) y registrarla en el GenieGo Integration Hub. Gmarket y Auction se operan conjuntamente a través de ESM Plus, por lo que una única clave API conecta ambos canales a la vez.

Tiempo estimadoVarios días incl. espera de aprobación (sujeto a aprobación)
Cuenta necesariaCuentas de vendedor de Gmarket·Auction + ID maestro de ESM Plus
Claves emitidasClave API · ID de vendedor
ResultadoCredenciales de ESM Trading API

1. Antes de empezar

Debes estar registrado como vendedor tanto en Gmarket como en Auction y tener un ID maestro de ESM Plus.

Requerido

Cuentas de vendedor + ID maestro de ESM

Necesitas cuentas de vendedor tanto para Auction como para Gmarket, además de un ID maestro creado en ESM Plus (esmplus.com).

Claves emitidas

Clave API · ID de vendedor

Tras la aprobación de la ESM Trading API, comprueba juntos la clave API emitida y el ID de vendedor.

Nota: La ESM Trading API está sujeta a aprobación y tu solicitud puede ser rechazada. Al solicitarla, debes enviar tres elementos: tu ID de vendedor de Auction, tu ID de vendedor de Gmarket y tu ID maestro de ESM.

2. Pasos de emisión

Solicita y emite en el orden siguiente, y luego regístrala en GenieGo.

1

Regístrate como vendedor y crea un ID maestro de ESM

Regístrate como vendedor tanto en Auction como en Gmarket, luego inicia sesión en ESM Plus y crea un ID maestro.

esmplus.com
2

Solicita el acceso a la ESM Trading API

Ve a la ESM Trading API y solicita el acceso a la API. Está sujeta a aprobación, por lo que puede ser rechazada.

https://etapi.gmarket.com
3

Envía los datos de identificación

Al solicitarla, envía tres elementos: tu ID de vendedor de Auction, tu ID de vendedor de Gmarket y tu ID maestro de ESM.

4

Recibe la clave

Una vez aprobada, se emite una clave API. Comprueba la clave API emitida y el ID de vendedor.

Una vez que tu solicitud sea aprobada y se emita la clave API, se habrá procesado correctamente. Para consultas relacionadas con la aprobación, contacta con etapihelp@gmail.com.

3. Información para registrar en GenieGo

CampoDescripción
api_keyLa clave API emitida
seller_idID de vendedor
Introdúcelos y guárdalos en el GenieGo Integration Hub [Registrar]. Las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
Solicitud de acceso a la API rechazadaLa ESM Trading API está sujeta a aprobaciónVerifica la información que enviaste y contacta con etapihelp@gmail.com.
Discrepancia en los datos de la solicitudID de vendedor de Auction·Gmarket o ID maestro de ESM introducido incorrectamenteVuelve a comprobar los tres elementos: tu ID de vendedor de Auction, tu ID de vendedor de Gmarket y tu ID maestro de ESM.

Lista de verificación final

  • Te registraste como vendedor tanto en Auction como en Gmarket.
  • Creaste un ID maestro en ESM Plus.
  • Solicitaste y recibiste la aprobación para el acceso a la ESM Trading API en etapi.gmarket.com.
  • Comprobaste el api_key y el seller_id emitidos.
  • Introdujiste api_key·seller_id en GenieGo [Registrar] y los guardaste.
