Manual de emisión de API

El procedimiento para registrar una aplicación en la consola de desarrolladores y preparar el Client ID, el Client Secret y el OAuth Token para conectar los servicios de la familia Yahoo! Japan.

Estructura de autenticaciónCentrada en OAuth 2.0
Cuenta necesariaYahoo! JAPAN ID
Resultado principalClient ID / Secret
PrecauciónLa consola difiere según la API

1. Concepto de emisión de la API de Yahoo! Japan

Según el tipo de servicio, las API de la familia Yahoo! Japan emiten el Client ID, el Client Secret y las credenciales de OAuth desde Yahoo! JAPAN Developers o desde el LY/Yahoo!広告 Developer Center.

Importante: En "Yahoo! Japan", el sitio de emisión y el proceso de revisión varían según la API que desee utilizar. Debe distinguir entre las API de Compras, Publicidad, Inicio de sesión y Búsqueda.

2. Requisitos previos

  • Yahoo! JAPAN ID o cuenta de empresa
  • Decidir el tipo de API que se va a usar
  • Nombre de la aplicación y descripción del servicio
  • Redirect URI
  • URL de la política de privacidad
  • Dominio de producción o dominio de pruebas

3. Orden básico de registro de la aplicación

1

Acceder al sitio de desarrolladores

Inicie sesión en Yahoo! JAPAN Developers o en el Developer Center de la API correspondiente.

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

Crear una aplicación

Haga clic en el menú para agregar una nueva aplicación.

Create Application / Add Application
3

Introducir la información de la aplicación

Introduzca el nombre de la aplicación, la descripción, la información del responsable y la URL del sitio web.

Application Information
4

Introducir la Redirect URI

Introduzca con exactitud la URL a la que se regresará tras la autenticación OAuth.

Redirect URI
5

Seleccionar permisos o API

Seleccione los permisos de API y los ámbitos (scope) que desea utilizar.

Scope / API Permission
6

Completar el registro

Tras el registro, compruebe el Client ID y el Client Secret.

Client ID / Client Secret

5. Flujo de emisión del token OAuth

1) Generar la URL de autenticación del usuario con el Client ID 2) El usuario inicia sesión con su cuenta de Yahoo! 3) Recibir el authorization code en la Redirect URI 4) Solicitar un Access Token con el authorization code + Client Secret 5) Llamar a la API con el Access Token 6) Renovar con el Refresh Token cuando sea necesario

6. Problemas frecuentes

ProblemaCausaSolución
Error de Redirect URILa URL registrada difiere de la URL de llamadaIntrodúzcala de forma idéntica, incluidos el protocolo, el dominio y la ruta

Permisos insuficientesNo se seleccionó el Scope necesarioAgregue el Scope en la configuración de la aplicación y vuelva a autenticarse
Fallo en la llamada a la APIDiscrepancia entre la API usada y el tipo de aplicación emitidaConsulte la documentación para desarrolladores de cada API

7. Lista de verificación final

  • He confirmado el tipo de API de Yahoo! Japan que voy a usar.
  • He iniciado sesión en la consola de desarrolladores correcta.
  • He registrado la aplicación.
  • He comprobado el Client ID y el Client Secret.
  • He registrado la Redirect URI con exactitud.
  • He seleccionado el Scope necesario.
  • He completado la prueba de emisión del Access Token.
