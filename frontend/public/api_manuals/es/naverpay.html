Naver Pay
Manual de emisión de API Key

Naver Pay es un canal en el que un representante le entrega las credenciales tras completar el registro de comercio (contrato). Este manual explica el proceso de registrar el Client ID, el Client Secret y el Chain ID que reciba en el GenieGo Integration Hub.

Tiempo estimadoVaría según el período de contrato y revisión
Cuenta necesariaCuenta de Naver Pay Business Center
Claves emitidasClient ID · Client Secret · Chain ID
ResultadoCredenciales de integración de pagos

1. Antes de empezar

Con Naver Pay no emite las claves usted mismo; las credenciales se entregan tras completar el contrato de comercio.

Naver Pay no es un método en el que emita claves directamente en una consola. Una vez completado el registro de comercio (contrato), un representante le entrega las credenciales.
Importante: No se trata de una emisión propia habitual. Tras completar el registro de comercio (contrato), se le entregan el Partner ID y el Client ID/Secret.

2. Proceso de solicitud de emisión

Realice el contrato en el orden siguiente y registre las credenciales que reciba.

1

Solicitar el contrato de comercio

Solicite y complete el contrato de comercio en el Naver Pay Business Center.

2

Completar la revisión y el contrato

Continúe hasta que se completen la revisión y el contrato. La duración puede variar según el calendario de revisión.

3

Recibir las credenciales

Tras completar la revisión y el contrato, un representante le entrega el Client ID, el Client Secret y el Chain ID.

Una vez que haya recibido del representante el Client ID, el Client Secret y el Chain ID completos, estará listo para registrarse.

3. Información para registrar en GenieGo

CampoDescripción
client_idEl Client ID que recibió
client_secretEl Client Secret que recibió (manéjelo con cuidado)
chain_idEl Chain ID que recibió
Introdúzcalos en [Registrar] del GenieGo Integration Hub y guarde; las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
No se encuentra un menú para emitir credencialesNaver Pay no es un método de emisión propiaUse las credenciales que le entregó un representante tras completar el contrato de comercio.
Error de autenticaciónClient Secret o Chain ID introducidos incorrectamenteVuelva a comprobar el Client ID, el Client Secret y el Chain ID entregados por su representante.

Lista de verificación final

  • Solicité y completé el contrato de comercio en el Naver Pay Business Center.
  • Tras completar la revisión y el contrato, recibí el Client ID, el Client Secret y el Chain ID de un representante.
  • Verifiqué las tres credenciales recibidas.
  • Introduje client_id·client_secret·chain_id en GenieGo [Registrar] y guardé.
