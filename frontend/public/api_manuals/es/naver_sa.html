Naver Search Ads
Manual de emisión de la clave de API

Una guía paso a paso para principiantes sobre cómo emitir las credenciales de API (licencia de acceso, clave secreta) y tu CUSTOMER_ID desde la herramienta de gestión de Naver Search Ad y registrarlas en el GenieGo Integration Hub.

Tiempo estimadoUnos 5–10 min
Cuenta necesariaCuenta de anunciante de Naver Search Ad
Claves emitidasLicencia de acceso · Clave secreta · CUSTOMER_ID
ResultadoCredenciales de Search Ad API

1. Antes de comenzar

La API de Naver Search Ad solo se puede solicitar desde una cuenta de anunciante.

Obligatorio

Cuenta de anunciante de Search Ad

Necesitas una cuenta de anunciante que utilice la herramienta de gestión de Naver Search Ad. No puedes solicitar acceso a la API con una cuenta de miembro común de Naver.

Claves emitidas

Licencia de acceso · Clave secreta · CUSTOMER_ID

Confirma el CUSTOMER_ID, la licencia de acceso y la clave secreta que se emiten en conjunto después de solicitar el acceso a la API.

Nota: La clave secreta solo se puede ver en el momento de la emisión, así que guárdala de forma segura justo después de emitirla.

2. Pasos para emitir la clave de API

Emítela en el orden siguiente y regístrala en GenieGo.

1

Iniciar sesión en Search Ads

Inicia sesión en la herramienta de gestión de Naver Search Ad con tu cuenta de anunciante.

searchad.naver.com
2

Ir a la gestión de uso de la API

Desde el menú superior [Herramientas], ve a [Gestión de uso de la API].

Herramientas → Gestión de uso de la API
3

Solicitar el servicio de API

Haz clic en el botón [Solicitar el servicio de Naver Search Ad API] para solicitarlo.

Solicitar el servicio de Naver Search Ad API
4

Verificar la información emitida

Confirma el CUSTOMER_ID, la licencia de acceso y la clave secreta emitidos.

Una vez confirmados el CUSTOMER_ID, la licencia de acceso y la clave secreta, se han emitido correctamente.

3. Información a registrar en GenieGo

CampoDescripción
api_keyLa licencia de acceso emitida
api_secretLa clave secreta emitida (manéjala con cuidado)
customer_idEl CUSTOMER_ID emitido
Ingrésalas y guárdalas en [Registrar] del GenieGo Integration Hub; las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
No aparece el menú [Gestión de uso de la API]No es una cuenta de anunciante o permisos insuficientesInicia sesión con una cuenta de anunciante que use la herramienta de gestión de Search Ad.
Se produce un error de autenticaciónLicencia de acceso o clave secreta ingresada incorrectamenteVuelve a verificar la licencia de acceso, la clave secreta y el CUSTOMER_ID en la pantalla de emisión.

Lista de verificación final

  • Inicié sesión con una cuenta de anunciante de Naver Search Ad.
  • Completé la solicitud del servicio de API en [Herramientas > Gestión de uso de la API].
  • Confirmé el CUSTOMER_ID, la licencia de acceso y la clave secreta.
  • Ingresé api_key·api_secret·customer_id en GenieGo [Registrar] y los guardé.
