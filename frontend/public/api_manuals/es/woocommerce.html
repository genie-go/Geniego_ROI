Manual de emisión de API Key

Una guía paso a paso para principiantes sobre cómo emitir una Consumer Key y un Consumer Secret desde la configuración de REST API en tu administrador de WooCommerce y registrarlos en el GenieGo Integration Hub.

Tiempo estimadoUnos 5–10 min
Cuenta necesariaPrivilegios de administrador de WordPress/WooCommerce
Claves emitidasConsumer Key · Consumer Secret
ResultadoCredenciales de REST API

1. Antes de empezar

Las claves de REST API solo se pueden emitir con privilegios de administrador.

Requerido

Privilegios de administrador de WooCommerce

Necesitas privilegios de administrador de WordPress/WooCommerce. No se pueden emitir claves de REST API con privilegios de usuario normal.

Claves emitidas

Consumer Key · Consumer Secret

Al agregar una clave en la configuración de REST API, se emiten una Consumer Key y un Consumer Secret.

2. Pasos de emisión

Emite en el orden siguiente y regístralos en GenieGo.

1

Ir a la configuración de REST API

En WooCommerce, ve a [Ajustes] > [Avanzado] > [REST API].

WooCommerce → Ajustes → Avanzado → REST API
2

Agregar una clave

Haz clic en [Agregar clave], introduce una descripción y selecciona los permisos (Lectura/Escritura), luego genérala.

Agregar clave
3

Copiar las claves

Copia la Consumer Key y el Consumer Secret emitidos.

4

Confirmar la información

Confirma la Consumer Key·Consumer Secret copiados y tu propia store URL.

Si la Consumer Key y el Consumer Secret se muestran en la pantalla de emisión, las claves se han emitido correctamente.
Precaución: El Consumer Secret a menudo se muestra completo solo justo después de la emisión, así que asegúrate de copiarlo de forma segura en ese momento.

3. Información para registrar en GenieGo

CampoDescripción
site_urlstore URL
consumer_keyLa Consumer Key emitida
consumer_secretEl Consumer Secret emitido (manéjalo con cuidado)
Introduce site_url·consumer_key·consumer_secret en el GenieGo Integration Hub [Registrar] y guarda. Las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas comunes

ProblemaCausaSolución
No aparece el menú de REST APIPrivilegios de administrador insuficientesInicia sesión con una cuenta que tenga privilegios de administrador y comprueba Ajustes > Avanzado > REST API.
Fallan las operaciones de escrituraSe seleccionó el permiso de solo lectura al crear la claveVuelve a emitirla con permisos de Lectura/Escritura al [Agregar clave].

Lista de verificación final

  • Inicié sesión con privilegios de administrador de WordPress/WooCommerce.
  • Agregué una clave en Ajustes > Avanzado > REST API.
  • Seleccioné los permisos (Lectura/Escritura) y copié la Consumer Key·Consumer Secret.
  • Introduje site_url·consumer_key·consumer_secret en GenieGo [Registrar] y guardé.
