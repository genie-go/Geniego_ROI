Facebook Live
Manual de Emisión de API

Una guía apta para principiantes sobre cómo crear una app en Meta for Developers y preparar el App ID, el App Secret y el token de acceso de página (Page Access Token) necesarios para conectar la transmisión en vivo de una página de Facebook.

Concepto claveNo es un API Key, sino un Access Token
Sitio de desarrolladoresdevelopers.facebook.com
API utilizadaLive Video API (Graph)
Permisos necesariosAdministrador de página + Page Token

1. Lo esencial que debe saber primero

Con YouTube basta un único API Key, pero Facebook (Meta) funciona con una estructura de App ID · App Secret · Access Token.

Importante: Con Meta no basta con "copiar un único API Key". Por lo general se necesita el siguiente orden: crear una app → agregar un producto → configurar permisos → emitir un Page Access Token → revisión de la app.

2. Requisitos

  • Una cuenta de Meta/Facebook
  • Acceso a Meta for Developers
  • Una página de Facebook en la que tengas permisos de administrador
  • URL de política de privacidad · URL de términos del servicio para el servicio real

3. Crear una app de Meta Developer

Toda integración con la API de Meta comienza con la creación de una app.

1

Acceder a Meta for Developers

Inicia sesión en Meta for Developers en tu navegador.

developers.facebook.com
2

Mis apps → Crear app

Haz clic en My Apps → Create App en la parte superior.

My Apps → Create App
3

Elegir el tipo de app

La integración de plataformas/comercio en vivo se aproxima más a un propósito empresarial.

Seleccionar el tipo Business
4

Introducir nombre de la app·correo → crear

Introduce el nombre de la app y un correo de contacto, y crea la app.

App Name / Contact Email
Tras crear la app, consulta el App ID y el App Secret en Panel → Configuración de la app → Configuración básica. (Nunca expongas el App Secret en el frontend ni en un repositorio público.)

4. Flujo de configuración de la API de Facebook Live

1

Agregar un producto

En el panel de la app, prepara la configuración relacionada con Facebook Login / Graph API.

Add Product → Facebook Login
2

Conectar una página + token de página

Para transmitir en vivo a una página, necesitas permisos de administrador de página y un Page Access Token.

Page Access Token
3

Solicitar los permisos necesarios (App Review)

Solicita permisos destinados a publicar contenido·administrar páginas·crear videos en vivo (p. ej., pages_read_engagement, pages_manage_posts).

Permissions → App Review
4

Emitir·probar el token con Graph API Explorer

Emite un token que incluya los permisos y prueba las llamadas a la API.

Tools → Graph API Explorer
Ejemplo de llamada de creación de Facebook Live
POST /{page-id}/live_videos
access_token={PAGE_ACCESS_TOKEN}
Si se crea correctamente, se devuelve la información de transmisión en vivo, como el ID de la transmisión·la URL del stream·la clave del stream. Consulta el ID de página (page_id) en la información de la página.

5. Registrar en el GenieGo Integration Hub

1

Copiar el Page Access Token

Copia el Page Access Token que emitiste (se recomienda un token de larga duración).

access_token (Page token)
2

Confirmar el ID de página

Confirma el ID (page_id) de la página de Facebook de destino.

page_id
3

Registrar en el Integration Hub

En GenieGo Integration Hub → Facebook Live → [Registrar], introduce los valores siguientes y guarda.

access_token · page_id
Las claves registradas se cifran en reposo con AES-256-GCM y se enmascaran al consultarlas.

6. Tipos de Access Token

Tipo de tokenUsoNotas
User Access TokenLlamadas a la API basadas en el inicio de sesión del usuarioLos datos accesibles varían según el alcance de los permisos otorgados.
Page Access TokenPublicación en página · creación de transmisiones en vivo de páginaSe requieren permisos de administrador de página y los permisos relacionados.
Long-Lived TokenUn token de larga duración con caducidad ampliadaEn producción se necesita lógica de caducidad·renovación.

7. Lista de verificación

  • Creé una app de Meta Developer.
  • Confirmé el App ID y el App Secret.
  • Conecté una página de Facebook y emití un Page Access Token.
  • Organicé·solicité los permisos necesarios (App Review).
  • Introduje access_token·page_id en GenieGo [Registrar] y guardé.
  • Almacené el token de forma segura en las variables de entorno del servidor.

8. Problemas frecuentes

ProblemaCausaSolución
No hay menú de API KeyMeta no usa una estructura de API Key únicoProcede con la estructura App ID·App Secret·Access Token.
Error de permisosEl token carece de los permisos necesariosAgrega los permisos en Graph API Explorer y vuelve a emitir el token.
Falla la creación de la transmisión en vivo de la páginaPage Access Token/permisos de administrador insuficientesVerifica los permisos de administrador de página y los permisos del token.
