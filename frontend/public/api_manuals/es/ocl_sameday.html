GenieGo Same-Day Delivery
Manual de emisión de API Key

Un manual que explica cómo registrar la clave API de Entrega el Mismo Día y el ID de comerciante—emitidos tras contratar el servicio GenieGo Entrega el Mismo Día—en el GenieGo Integration Hub. Para este canal, las credenciales se reciben mediante un contrato con el equipo de operaciones en lugar de emitirse por cuenta propia desde una consola.

ResultadoCredenciales de la API de Entrega el Mismo Día

1. Antes de empezar

GenieGo Entrega el Mismo Día es un canal basado en contrato, donde las credenciales se emiten tras un contrato de servicio con el equipo de operaciones.

Requerido

Contrato del servicio de Entrega el Mismo Día

Para usar GenieGo Entrega el Mismo Día, primero debe existir un contrato de uso del servicio con el equipo de operaciones. Sin contrato, no se pueden emitir las credenciales.

Claves emitidas

Clave API · ID de comerciante · Región de servicio

Tras completar el contrato, el equipo de operaciones le emite el ID de comerciante y la clave API junto con una región de servicio.

Nota: Este canal no se emite por cuenta propia desde una consola; en cambio, las credenciales se reciben mediante un contrato con el equipo de operaciones.

2. Procedimiento de solicitud de emisión

Solicite y reciba en el orden siguiente, luego registre en GenieGo.

1

Solicitar Entrega el Mismo Día

Solicite al equipo de operaciones usar GenieGo Entrega el Mismo Día y avance con el contrato.

2

Firmar el contrato

Firme el contrato de uso del servicio de Entrega el Mismo Día con el equipo de operaciones.

3

Recibir las credenciales

Tras completar el contrato, se le emiten el ID de comerciante, la clave API y la región de servicio.

4

Registrar en GenieGo

Introduzca el api_key, merchant_id y region emitidos en GenieGo [Registrar] y guarde.

Si ha recibido el ID de comerciante, la clave API y la región de servicio en su totalidad del equipo de operaciones, la emisión está completa.

3. Información para registrar en GenieGo

CampoDescripción
api_keyLa clave API de GenieGo Entrega el Mismo Día emitida
merchant_idEl ID de comerciante emitido
regionRegión de servicio (p. ej., Área Metropolitana de la Capital)
Una vez introducidas y guardadas en el GenieGo Integration Hub [Registrar], las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerse.

4. Problemas frecuentes

ProblemaCausaSolución
No se reciben las credencialesEl contrato del servicio de Entrega el Mismo Día no se ha completadoSolicite y contrate el uso de Entrega el Mismo Día con el equipo de operaciones, luego reciba las credenciales.
Error de procesamiento de región tras el registroRegión de servicio (region) no introducida o introducida incorrectamenteVuelva a comprobar la región de servicio emitida e introdúzcala correctamente.

Lista de verificación final

  • Solicité y contraté el uso de GenieGo Entrega el Mismo Día con el equipo de operaciones.
  • Recibí el ID de comerciante, la clave API y la región de servicio tras completar el contrato.
  • Confirmé api_key, merchant_id y region.
  • Introduje api_key, merchant_id y region en GenieGo [Registrar] y guardé.
