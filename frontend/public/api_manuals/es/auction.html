Auction
Manual de emisión de API Key

Una guía que explica cómo obtener una clave de API compartida para Auction·Gmarket desde la ESM Trading API (etapi.gmarket.com) y registrarla en el GenieGo Integration Hub. Dado que Auction y Gmarket se operan conjuntamente a través de ESM Plus, una sola clave de API conecta ambos canales.

Tiempo estimadoVarios días incl. espera de aprobación (requiere aprobación)
Cuenta necesariaCuenta de vendedor de Auction·Gmarket + ID maestro de ESM Plus
Claves emitidasClave de API · ID de vendedor
ResultadoCredenciales de ESM Trading API

1. Antes de comenzar

Debe estar registrado como vendedor tanto en Auction como en Gmarket y tener un ID maestro de ESM Plus.

Requerido

Cuenta de vendedor + ID maestro de ESM

Necesita una cuenta de vendedor para cada uno de Auction y Gmarket, además de un ID maestro creado en ESM Plus (esmplus.com).

Claves emitidas

Clave de API · ID de vendedor

Tras la aprobación de la ESM Trading API, confirme la clave de API emitida junto con el ID de vendedor.

Nota: La ESM Trading API requiere aprobación y las solicitudes pueden ser rechazadas. Al solicitarla, debe enviar tres elementos: su ID de vendedor de Auction, su ID de vendedor de Gmarket y su ID maestro de ESM.

2. Pasos de emisión

Solicite y obtenga la clave en el orden indicado a continuación y luego regístrela en GenieGo.

1

Registrarse como vendedor y crear un ID maestro de ESM

Regístrese como vendedor tanto en Auction como en Gmarket, luego inicie sesión en ESM Plus y cree un ID maestro.

esmplus.com
2

Solicitar el uso de la ESM Trading API

Vaya a la ESM Trading API y solicite el acceso a la API. Como requiere aprobación, la solicitud puede ser rechazada.

https://etapi.gmarket.com
3

Enviar información de identificación

Al solicitar, envíe tres elementos: su ID de vendedor de Auction, su ID de vendedor de Gmarket y su ID maestro de ESM.

4

Emisión de la clave

Una vez aprobada, se emite una clave de API. Confirme la clave de API emitida y el ID de vendedor.

Una vez que se emite la clave de API tras completarse la aprobación, el proceso ha tenido éxito. Para consultas sobre la aprobación, contacte con etapihelp@gmail.com.

3. Información para registrar en GenieGo

CampoDescripción
api_keyLa clave de API emitida
seller_idID de vendedor
Introduzca los valores en el GenieGo Integration Hub [Registrar] y guarde. Las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
Solicitud de uso de la API rechazadaLa ESM Trading API requiere aprobaciónVerifique la información enviada y contacte con etapihelp@gmail.com.
La información de la solicitud no coincideID de vendedor de Auction·Gmarket o ID maestro de ESM introducidos incorrectamenteVuelva a verificar los tres elementos: ID de vendedor de Auction, ID de vendedor de Gmarket e ID maestro de ESM.

Lista de verificación final

  • Registrado como vendedor tanto en Auction como en Gmarket.
  • Creado un ID maestro en ESM Plus.
  • Solicitado el acceso a la ESM Trading API en etapi.gmarket.com y obtenida la aprobación.
  • Confirmados el api_key y el seller_id emitidos.
  • Introducido api_key·seller_id en GenieGo [Registrar] y guardado.
