Manual de Emisión de API Key

Una guía paso a paso para principiantes sobre cómo solicitar las claves de Open API de godomall en el NHN Commerce Developer Center, recibir tu partner key y API key, y registrarlas en el GenieGo Integration Hub.

Tiempo estimadoUnos 10–20 min (incluida la espera de aprobación)
Cuenta necesariaCuenta de operador de tienda godomall (NHN Commerce)
Claves emitidaspartner key · API key
ResultadoCredenciales de Open API

1. Antes de empezar

La Open API de godomall se solicita en el NHN Commerce Developer Center tras el registro y la aprobación del desarrollador.

Requerido

Cuenta de operador de tienda

Necesitas una cuenta de operador que gestione una tienda godomall (NHN Commerce). Usa esta cuenta para completar el registro de miembro y desarrollador en el Developer Center.

Claves emitidas

partner key · API key

Una vez que se apruebe tu solicitud de emisión de claves, se emiten las claves de Open API (partner key / API key).

Nota: El registro del desarrollador y la solicitud de emisión de claves pasan por un proceso de aprobación, por lo que la aprobación puede tardar algún tiempo.

2. Pasos para la emisión de la clave de Open API

Solicita y emite en el orden siguiente, luego registra en GenieGo.

1

Registrarse y obtener la aprobación como desarrollador

Regístrate como miembro y desarrollador en el NHN Commerce Developer Center y obtén la aprobación.

devcenter.nhn-commerce.com
2

Solicitar la emisión de claves

En [Open API] > [Solicitar emisión de claves], completa el formulario de solicitud y envíalo.

Open API → Solicitar emisión de claves
3

Verificar las claves

Verifica las claves de Open API emitidas (partner key / API key).

Una vez que se emiten la partner key y la API key, la solicitud se ha enviado y aprobado correctamente.

3. Información para registrar en GenieGo

CampoDescripción
partner_keyLa partner key emitida
api_keyLa API key emitida (manéjala con cuidado)
Introduce y guarda los valores en el GenieGo Integration Hub [Registrar]. Las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerlas.

4. Problemas frecuentes

ProblemaCausaSolución
No se puede solicitar la emisión de clavesRegistro del desarrollador / aprobación no completadosCompleta primero el registro de miembro y desarrollador y la aprobación en el Developer Center.
Las claves no se emiten tras la solicitudAprobación pendienteEspera a que se complete el proceso de aprobación y luego vuelve a comprobarlo.

Lista de verificación final

  • Te registraste como miembro y desarrollador en el NHN Commerce Developer Center y obtuviste la aprobación.
  • Completaste y enviaste el formulario de solicitud en [Open API > Solicitar emisión de claves].
  • Verificaste la partner key y la API key emitidas.
  • Introdujiste partner_key·api_key en GenieGo [Registrar] y guardaste.
