KG Inicis
Manual de emisión de API Key

Una guía que resume el proceso de emitir tu INIAPI Key y la clave de firma estándar web (SignKey) en el administrador de comercio de KG Inicis y registrarlas junto con tu ID de tienda (MID) en el GenieGo Integration Hub.

Tiempo estimadoUnos 5–15 min
Cuenta necesariaContrato de comercio de KG Inicis · Cuenta de administrador de comercio
Claves emitidasMID · Clave API · Clave de firma (SignKey)
ResultadoCredenciales de integración de pagos

1. Antes de empezar

KG Inicis solo puede emitir claves de autenticación una vez completado el contrato de comercio.

Requerido

Contrato de comercio · Cuenta de administrador de comercio

Debes tener un contrato de comercio de KG Inicis firmado y poder iniciar sesión con una cuenta de administrador de comercio.

Claves emitidas

INIAPI Key · Clave de firma estándar web

Crea una INIAPI Key en el administrador de comercio y emite a la vez la clave de firma estándar web mediante la generación de signkey de pago web.

Nota: El registro requiere tres elementos: ID de tienda (MID), clave de firma (SignKey) y clave API.

2. Pasos de emisión

Emite en el orden siguiente y registra en GenieGo.

1

Iniciar sesión en el administrador de comercio

Inicia sesión en el administrador de comercio de Inicis.

https://iniweb.inicis.com
2

Ir al menú Información adicional

Ve a [Información de la tienda] > [Información del contrato] > [Información adicional].

Información de la tienda → Información del contrato → Información adicional
3

Crear una INIAPI Key

Crea una INIAPI Key.

4

Crear la clave de firma estándar web

Haz clic en [Generación de signkey de pago web] para emitir la clave de firma estándar web (SignKey).

Generación de signkey de pago web
Una vez creadas tanto la INIAPI Key como la clave de firma estándar web, la emisión se ha completado correctamente.

3. Información para registrar en GenieGo

CampoDescripción
MIDID de tienda (MID)
sign_keyLa clave de firma estándar web emitida (SignKey, manéjala con cuidado)
api_keyLa INIAPI Key emitida
Introduce y guarda en el GenieGo Integration Hub [Registrar] y las credenciales se cifran en reposo con AES-256-GCM y se enmascaran al leerse.

4. Problemas comunes

ProblemaCausaSolución
El menú Información adicional no es visibleContrato de comercio no completado, o una cuenta sin permisosInicia sesión con una cuenta de administrador de comercio que tenga un contrato de comercio completado.
Falla la autenticación del pagoClave de firma o clave API introducida incorrectamenteVuelve a comprobar la clave de firma·INIAPI Key emitida en Información de la tienda > Información del contrato > Información adicional.

Lista de verificación final

  • Completé el contrato de comercio de KG Inicis e inicié sesión en el administrador de comercio.
  • Creé una INIAPI Key en Información de la tienda > Información del contrato > Información adicional.
  • Emití la clave de firma estándar web mediante [Generación de signkey de pago web].
  • Introduje MID·sign_key·api_key en GenieGo [Registrar] y guardé.
