Manual de emisión de API

Esta guía resume el proceso de crear un Provider y un Messaging API Channel en la LINE Developers Console y de configurar el Channel Access Token y la Webhook URL.

1. Concepto de emisión de la API de LINE

La integración de LINE funciona creando un Provider y un Channel en la LINE Developers Console y luego emitiendo el Channel ID, el Channel Secret y el Channel Access Token para la Messaging API o LINE Login.

Distinción: El envío de mensajes utiliza un canal Messaging API, y la integración de inicio de sesión utiliza un canal LINE Login.

2. Requisitos previos

  • Una cuenta de LINE o una cuenta empresarial
  • Permiso de acceso a la LINE Developers Console
  • Una LINE Official Account
  • Nombre del servicio y descripción
  • Webhook URL
  • URL de la política de privacidad y URL de los términos

3. Crear un Provider

1

Acceder a LINE Developers

Inicie sesión en la LINE Developers Console.

developers.line.biz
2

Crear un Provider

Cree un Provider usando el nombre de su empresa o de su servicio.

Create Provider
3

Introducir la información básica

Introduzca el operador del servicio, los datos de contacto y la información de los términos.

Provider Settings

4. Crear un canal Messaging API

1

Crear un nuevo canal

Haga clic en Create a new channel dentro del Provider.

Create a new channel
2

Seleccionar Messaging API

Elija Messaging API como tipo de canal.

Messaging API
3

Introducir la información del canal

Introduzca el nombre del canal, la descripción, la categoría, el correo electrónico, etc.

Channel Information
4

Vincular una LINE Official Account

Vincule una cuenta oficial desde la que enviar mensajes, o cree una nueva.

LINE Official Account
5

Finalizar la creación del canal

Tras la creación, verifique el Channel ID y el Channel Secret.

Channel ID / Channel Secret

5. Emitir el Channel Access Token

1

Ir a la pestaña Messaging API

En la pantalla del canal que creó, vaya a la pestaña Messaging API.

Messaging API tab
2

Buscar el elemento Channel access token

Localice el área Channel access token en la parte inferior.

Channel access token
3

Hacer clic en Issue

Haga clic en el botón Issue o Reissue para emitir el token.

Issue
4

Almacenarlo de forma segura en el servidor

Almacene el token emitido en una variable de entorno del servidor o en Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
El Channel Access Token es una clave secreta con permiso para enviar mensajes. No debe exponerse en HTML, en el frontend ni en repositorios públicos.

6. Configurar el Webhook

1

Introducir la Webhook URL

Introduzca la URL del servidor que recibirá los eventos de mensajes de los usuarios.

Webhook URL
2

Activar Use webhook

Cambie la opción Use webhook a Enabled.

Use webhook: Enabled
3

Prueba Verify

Envíe una solicitud de prueba desde LINE a su servidor para confirmar la conexión.

Verify
4

Comprobar el código de respuesta

Confirme que el servidor devuelve 200 OK.

HTTP 200 OK

7. Ejemplo de envío de mensaje

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Hola. Esta es una prueba de mensaje de LINE."} ] }

8. Lista de verificación final

  • Creé un Provider.
  • Creé un canal Messaging API.
  • Vinculé una LINE Official Account.
  • Verifiqué el Channel ID y el Channel Secret.
  • Emití el Channel Access Token.
  • Registré la Webhook URL y pasé Verify.
  • Almacené el token de forma segura en una variable de entorno del servidor.
