KG Inicis
Manuel d'émission de clé API

Un guide qui résume le processus d'émission de votre INIAPI Key et de votre clé de signature standard web (SignKey) dans l'administration marchand de KG Inicis, puis de leur enregistrement avec votre ID de boutique (MID) dans le GenieGo Integration Hub.

Temps estiméEnviron 5 à 15 min
Compte requisContrat marchand KG Inicis · Compte d'administration marchand
Clés émisesMID · Clé API · Clé de signature (SignKey)
RésultatIdentifiants d'intégration de paiement

1. Avant de commencer

KG Inicis ne peut émettre les clés d'authentification qu'une fois le contrat marchand finalisé.

Requis

Contrat marchand · Compte d'administration marchand

Vous devez avoir un contrat marchand KG Inicis signé et pouvoir vous connecter avec un compte d'administration marchand.

Clés émises

INIAPI Key · Clé de signature standard web

Créez une INIAPI Key dans l'administration marchand et émettez en même temps la clé de signature standard web via la génération de signkey de paiement web.

Remarque : L'enregistrement nécessite trois éléments : l'ID de boutique (MID), la clé de signature (SignKey) et la clé API.

2. Étapes d'émission

Émettez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

Se connecter à l'administration marchand

Connectez-vous à l'administration marchand Inicis.

https://iniweb.inicis.com
2

Aller au menu Informations complémentaires

Allez dans [Informations de la boutique] > [Informations du contrat] > [Informations complémentaires].

Informations de la boutique → Informations du contrat → Informations complémentaires
3

Créer une INIAPI Key

Créez une INIAPI Key.

4

Créer la clé de signature standard web

Cliquez sur [Génération de signkey de paiement web] pour émettre la clé de signature standard web (SignKey).

Génération de signkey de paiement web
Une fois que l'INIAPI Key et la clé de signature standard web sont toutes deux créées, l'émission est correctement terminée.

3. Informations à enregistrer dans GenieGo

ChampDescription
MIDID de boutique (MID)
sign_keyLa clé de signature standard web émise (SignKey, à manipuler avec précaution)
api_keyL'INIAPI Key émise
Saisissez et enregistrez dans le GenieGo Integration Hub [Enregistrer] ; les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes courants

ProblèmeCauseSolution
Le menu Informations complémentaires n'est pas visibleContrat marchand non finalisé, ou compte sans autorisationsConnectez-vous avec un compte d'administration marchand disposant d'un contrat marchand finalisé.
L'authentification du paiement échoueClé de signature ou clé API mal saisieVérifiez à nouveau la clé de signature·INIAPI Key émise dans Informations de la boutique > Informations du contrat > Informations complémentaires.

Liste de contrôle finale

  • Finalisé le contrat marchand KG Inicis et connecté à l'administration marchand.
  • Créé une INIAPI Key dans Informations de la boutique > Informations du contrat > Informations complémentaires.
  • Émis la clé de signature standard web via [Génération de signkey de paiement web].
  • Saisi MID·sign_key·api_key dans GenieGo [Enregistrer] et enregistré.
