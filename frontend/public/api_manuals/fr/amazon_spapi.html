1. Qu'est-ce que l'Amazon SP-API ?

L'Amazon Selling Partner API est une API permettant d'intégrer par programmation les données vendeur telles que les commandes, le stock, les produits, l'expédition et les règlements.

Important : La SP-API n'est pas un simple schéma d'API Key. Elle nécessite généralement ensemble l'enregistrement Amazon Developer, l'enregistrement de l'application, Login with Amazon(LWA) et l'autorisation OAuth.
★L'intégration GenieGo nécessite 5 valeurs : refresh_token, client_id, client_secret, marketplace_id, seller_id. (La SP-API actuelle est appelée avec le seul token LWA, et la signature AWS IAM/SigV4 est gérée en interne par GenieGo, vous ne la saisissez donc pas dans les champs d'enregistrement.)

2. Prérequis

  • Un compte vendeur Professional Amazon Seller Central ou Vendor Central
  • Les informations entreprise/contact à utiliser pour l'enregistrement développeur
  • Le nom de l'application, la description de l'application et l'usage prévu
  • L'URL de la politique de confidentialité
  • OAuth Redirect URL

3. Enregistrement développeur

D'abord, enregistrez votre profil développeur SP-API dans Seller Central.

1

Connectez-vous à Seller Central

Connectez-vous à Seller Central avec votre compte vendeur Amazon.

Seller Central → Apps & Services
2

Accédez au menu Develop Apps

Depuis le menu Apps & Services, accédez au menu développeur ou de développement d'applications.

Apps & Services → Develop Apps
3

Remplissez le Developer Profile

Saisissez les informations de l'entreprise, les informations de contact, l'objet d'utilisation de l'API et le plan de protection des données.

Developer Profile
4

Soumettre pour examen

Soumettez vos informations d'enregistrement développeur pour l'examen d'Amazon.

Submit for review
Ce n'est qu'une fois l'approbation terminée que vous pouvez enregistrer une application SP-API de production et utiliser ses autorisations.

4. Enregistrer une application SP-API → Client ID / Client Secret

1

Créez une nouvelle application

Créez une nouvelle application dans Develop Apps.

Add new app
2

Choisissez le type d'application

Si vous connectez uniquement votre propre compte, c'est une Private App ; si vous la fournissez à des vendeurs externes, c'est une Public App.

Private / Public App
3

Sélectionnez les rôles API

Sélectionnez les rôles dont vous avez besoin, tels que Orders, Catalog et Inventory.

API Roles
4

Saisissez l'OAuth Redirect URL

Saisissez l'URL du serveur vers laquelle revenir une fois l'autorisation terminée.

Redirect URI
5

Enregistrez et vérifiez les infos Client

Après avoir créé l'application, vérifiez le client_id (LWA Client ID) et le client_secret (Client Secret).

client_id / client_secret

5. Autorisation OAuth → Émettre un Refresh Token

1) Accédez à l'URL d'autorisation OAuth du vendeur 2) Recevez l'authorization code à la Redirect URL 3) Transmettez l'authorization code au LWA Token Endpoint 4) Recevez le refresh_token / access_token
Le refresh_token est utilisé pour les appels à long terme après l'autorisation du vendeur. GenieGo rafraîchit automatiquement l'access_token à l'aide de ce refresh_token pour appeler la SP-API, vous n'avez donc qu'à enregistrer le refresh_token sans configuration de signature AWS distincte.

6. Vérifiez le Marketplace ID · Seller ID

Vérifiez les deux identifiants restants nécessaires à l'enregistrement GenieGo.

marketplace_id

ID de la marketplace cible

Le Marketplace ID du pays que vous connectez. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Nécessaire pour l'enregistrement des produits et la consultation des commandes. Vérifiez-le dans les informations de votre compte Seller Central. Seller Central → Paramètres → Informations du compte → Merchant Token

7. Enregistrer dans le GenieGo Integration Hub

1

Préparez les 5 valeurs

Préparez refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 identifiants
2

Integration Hub → Amazon SP-API → [Enregistrer]

Saisissez chaque valeur dans son champ et enregistrez (les champs vides conservent les valeurs existantes).

refresh_token · client_id · client_secret · marketplace_id · seller_id
Les valeurs enregistrées sont chiffrées au repos avec AES-256-GCM et masquées à la lecture. La synchronisation automatique démarre immédiatement après l'enregistrement.
Le client_secret·refresh_token ne doit jamais être exposé dans le code frontend, le HTML ou des dépôts publics.

8. Liste de vérification finale

  • J'ai soumis le Developer Profile et l'ai fait approuver.
  • J'ai créé l'application SP-API et vérifié le client_id·client_secret.
  • J'ai émis un refresh_token via l'autorisation OAuth.
  • J'ai vérifié le marketplace_id à connecter.
  • J'ai vérifié le seller_id (Merchant Token) dans Seller Central.
  • J'ai saisi les 5 valeurs (refresh_token·client_id·client_secret·marketplace_id·seller_id) dans GenieGo [Enregistrer] et les ai enregistrées.

9. Problèmes fréquents

ProblèmeCauseSolution
Il n'y a pas de menu API KeyLa SP-API n'est pas une structure d'API Key uniqueProcédez avec la structure LWA (client_id·client_secret·refresh_token).
L'enregistrement des produits échoueseller_id (Merchant Token) non enregistréEnregistrez le Merchant Token depuis les informations de votre compte Seller Central.
Données de pays incorrectesmarketplace_id ne correspond pasEnregistrez le Marketplace ID correct pour le pays que vous connectez.
Erreur d'autorisationAPI Role de l'application insuffisant ou token expiréVérifiez l'API Role et rafraîchissez le token avec le refresh_token.
