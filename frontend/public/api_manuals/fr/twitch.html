Manuel d'ouverture de chaîne d'entreprise et d'émission d'API

Un guide pratique et accessible aux débutants destiné aux entreprises, marques, live-commerce et exploitants de plateformes IT/logistique pour créer une chaîne Twitch officielle et réaliser la configuration de diffusion, la connexion OBS et l'émission de la Twitch Developer API.

1. Comprendre la structure du compte Twitch

Contrairement à Instagram, Twitch ne propose pas de structure permettant de choisir séparément un compte entreprise/créateur.

Chaîne officielle de l'entreprise

Pour l'exploitation de la marque

Créez un compte Twitch ordinaire au nom de votre entreprise ou de votre marque, puis exploitez-le comme une chaîne officielle. Idéal pour les présentations de produits, le live-commerce et les diffusions d'événements.

Niveau de croissance du streamer

Affiliate / Partner

Une fois les conditions de performance de diffusion, de followers et de temps de visionnage remplies, vous pouvez rejoindre le programme Affiliate ou Partner.

Point clé : Les entreprises doivent elles aussi d'abord créer un compte Twitch ordinaire, puis s'occuper de l'image de marque de la chaîne, de l'authentification à deux facteurs, de la connexion OBS et de l'enregistrement de l'application dans la Developer Console.

2. Ce dont vous avez besoin

  • Adresse e-mail professionnelle
  • Nom de marque ou nom d'entreprise
  • Nom d'utilisateur Twitch envisagé
  • Logo de l'entreprise et image de bannière de la chaîne
  • Texte de présentation de l'entreprise et URL du site web
  • PC ou ordinateur portable pour diffuser
  • OBS Studio ou logiciel de streaming
  • OAuth Redirect URL pour l'intégration de l'API

3. Créer un compte Twitch

1

Ouvrir Twitch

Ouvrez le site officiel de Twitch dans votre navigateur.

https://www.twitch.tv
2

Cliquer sur S'inscrire

Cliquez sur le bouton Sign Up ou S'inscrire en haut à droite.

Sign Up
3

S'inscrire avec les informations de l'entreprise

Saisissez l'e-mail de l'entreprise, le nom d'utilisateur, le mot de passe et la date de naissance.

Email / Username / Password
4

Vérifier l'e-mail

Ouvrez l'e-mail de vérification envoyé par Twitch et terminez Verify Email.

Verify Email

4. Choisir le nom du compte de marque

Le nom d'utilisateur Twitch devient l'adresse de la chaîne, il est donc préférable de choisir un nom court et facile à retenir.

ObjectifExemples de noms recommandésDescription
Chaîne officielle de l'entrepriseGeniego / OCLKoreaCorrespond au nom de l'entreprise, ce qui inspire une grande confiance.
Diffusion mondialeGeniegoGlobal / OCLGlobalAdapté lorsque vous tenez compte des spectateurs étrangers.
Live-commerceGeniegoLive / GeniegoCommerceTransmet clairement un objectif de diffusion de vente ou de shopping.
Technologie/plateformeGeniegoTech / OCLTechAdapté aux diffusions de présentation IT, SaaS et plateforme.

5. Configurer le profil de la chaîne

1

Aller dans le menu Paramètres

Cliquez sur l'icône de profil en haut à droite, puis allez dans Settings.

Profile → Settings
2

Téléverser une image de profil

Définissez le logo de l'entreprise ou l'icône de la marque comme image de profil.

Profile Picture
3

Téléverser une bannière de chaîne

Téléversez une image de bannière portant les couleurs de votre marque et votre message de service.

Profile Banner
4

Saisir le texte de bio

Saisissez la présentation de l'entreprise, la description du service, l'e-mail de contact et le site web.

Bio / Links

6. Configurer l'authentification à deux facteurs

La configuration de la 2FA est très importante pour utiliser de manière fiable les paramètres de diffusion et de sécurité sur Twitch.

1

Aller dans le menu de sécurité

Ouvrez le menu Security and Privacy dans les paramètres.

Settings → Security and Privacy
2

Activer la 2FA

Activez Two-Factor Authentication.

Set Up Two-Factor Authentication
3

Connecter un téléphone ou une application d'authentification

Connectez la vérification par SMS ou une application d'authentification telle que Google Authenticator ou Authy.

Authenticator App
Attention : Les comptes d'entreprise peuvent être utilisés par plusieurs personnes, veillez donc à gérer les autorisations d'administrateur et l'e-mail de récupération.

7. Vérifier la Stream Key et connecter OBS

1

Ouvrir le Creator Dashboard

Accédez au Creator Dashboard depuis le menu de profil.

Creator Dashboard
2

Aller dans le menu Stream

Sélectionnez l'élément Stream dans le menu des paramètres.

Settings → Stream
3

Vérifier la Primary Stream Key

Copiez la clé de stream principale. Cette clé doit être conservée comme un mot de passe.

Primary Stream Key
4

Connecter dans OBS

Dans OBS Studio, réglez le service sur Twitch et saisissez la valeur copiée comme clé de stream.

OBS → Settings → Stream → Twitch
Si votre clé de stream est exposée, quelqu'un d'autre peut diffuser sur votre chaîne. Ne la rendez jamais publique.

9. S'enregistrer sur la Twitch Developer Console

Pour créer des chat bots, des analyses de spectateurs, des vérifications de l'état de la diffusion et des fonctions d'automatisation, vous devez enregistrer une application Twitch Developer.

1

Ouvrir Twitch Developer

Ouvrez le site développeurs de Twitch.

https://dev.twitch.tv
2

Aller à la Console

Cliquez sur Console ou Developer Console dans le menu supérieur.

Developer Console
3

Sélectionner le menu Applications

Sélectionnez Applications dans le menu de gauche ou supérieur.

Applications
4

Cliquer sur Register Your Application

Cliquez sur le bouton pour enregistrer une nouvelle application.

Register Your Application

10. Saisir les informations d'enregistrement de l'application Twitch API

Champ de saisieExemple de saisieDescription
NameGeniego Twitch AppLe nom de l'application. Saisissez-le en fonction du nom de votre service.
OAuth Redirect URLshttps://yourdomain.com/auth/twitch/callbackL'adresse du serveur vers laquelle revenir après la connexion OAuth.
CategoryWebsite Integration / Chat BotSélectionnez la catégorie qui correspond à l'objectif de votre application.

Une fois l'enregistrement terminé, le Client ID s'affiche et vous pouvez générer un Client Secret avec le bouton New Secret.

11. Vérifier le Client ID / Client Secret

1

Sélectionner l'application enregistrée

Cliquez sur l'application que vous venez de créer dans la liste Applications de la Developer Console.

Applications → sélectionner l'application
2

Copier le Client ID

Copiez le Client ID affiché à l'écran et enregistrez-le dans les paramètres de votre serveur.

Client ID
3

Générer le Client Secret

Cliquez sur le bouton New Secret pour générer un Client Secret et conservez-le en lieu sûr.

New Secret
Le Client Secret ne peut souvent être consulté qu'une seule fois. Enregistrez-le dans un endroit sûr immédiatement après l'avoir généré.

12. Structure d'authentification de la Twitch API

Twitch Channel │ ▼ Developer Console │ ▼ Application │ ▼ Client ID / Client Secret │ ▼ OAuth Token │ ▼ Twitch Helix API │ ▼ Infos de la chaîne, état de la diffusion, chat bot, analyse des spectateurs

13. Flux d'émission du token OAuth

1) Générer l'URL d'autorisation Twitch OAuth 2) L'utilisateur se connecte avec son compte Twitch 3) Recevoir l'authorization code à la Redirect URL 4) Demander un Access Token avec l'authorization code + Client Secret 5) Appeler la Twitch API avec l'Access Token 6) Renouveler avec le Refresh Token si nécessaire
Les requêtes simples de données publiques sont possibles avec un App Access Token, tandis que les opérations nécessitant des autorisations utilisateur requièrent un User Access Token.

15. Liste de vérification finale

  • J'ai créé un compte Twitch avec un e-mail professionnel.
  • J'ai défini un nom d'utilisateur correspondant au nom de la marque.
  • J'ai terminé la vérification de l'e-mail.
  • J'ai configuré l'image de profil, la bannière, le texte de bio et les liens.
  • J'ai activé l'authentification à deux facteurs.
  • J'ai vérifié la Stream Key dans le Creator Dashboard.
  • J'ai connecté OBS Studio à la chaîne Twitch.
  • J'ai enregistré l'application dans la Twitch Developer Console.
  • J'ai obtenu le Client ID et le Client Secret.
  • J'ai préparé l'OAuth Redirect URL et le flux d'émission du token.
