Naver Smart Store
Manuel de délivrance de clé API

Un guide étape par étape pour enregistrer une application dans le Naver Commerce API Center (apicenter.commerce.naver.com) afin d'obtenir un Client ID et un Client Secret, puis de les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 10–15 min
Compte requisCompte Smart Store avec l'autorisation "Gestionnaire intégré"
Clés délivréesClient ID · Client Secret
RésultatIdentifiants Commerce API

1. Avant de commencer

La délivrance n'est possible qu'avec l'autorisation Gestionnaire intégré sur un compte vendeur Smart Store.

Requis

Autorisation Gestionnaire intégré

Vous devez disposer de l'autorisation "Gestionnaire intégré" sur un compte vendeur Smart Store. Seul un Gestionnaire intégré peut délivrer les clés.

Clés délivrées

Client ID · Client Secret

Lorsque vous enregistrez une application, un ID d'application (Client ID) et un Secret (Client Secret) sont délivrés.

Remarque : L'enregistrement de l'adresse IP du serveur qui appellera l'API et le renouvellement périodique de l'authentification de l'application sont obligatoires.

2. Étapes de délivrance

Délivrez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

Accéder au Commerce API Center

Connectez-vous au Naver Commerce API Center avec votre compte Gestionnaire intégré.

https://apicenter.commerce.naver.com
2

Créer un compte

Cliquez sur [Créer un compte] en haut à droite, saisissez le nom de compte de l'entreprise de développement et un contact pour la gestion des incidents, acceptez les conditions et inscrivez-vous.

Créer un compte
3

Enregistrer une application

Sous [Mes applications de boutique], enregistrez une application ; un ID d'application et un Secret sont alors délivrés.

Mes applications de boutique
4

Enregistrer l'IP et renouveler l'authentification

Enregistrez l'adresse IP du serveur qui appellera l'API, et renouvelez périodiquement l'authentification de l'application comme indiqué.

Une fois l'application enregistrée et l'ID d'application ainsi que le Secret délivrés, le traitement est correct.

3. Informations à enregistrer dans GenieGo

ChampDescription
client_idL'ID d'application délivré (Client ID)
client_secretLe Secret délivré (Client Secret, à manipuler avec précaution)
Saisissez-les dans [Enregistrer] du GenieGo Integration Hub et enregistrez ; les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes courants

ProblèmeCauseSolution
Impossible d'enregistrer une applicationLe compte n'a pas l'autorisation Gestionnaire intégréAccédez à apicenter.commerce.naver.com avec un compte disposant de l'autorisation Gestionnaire intégré.
Erreur d'authentification lors de l'appelIP de l'appelant non enregistrée ou renouvellement de l'authentification omisEnregistrez l'IP du serveur appelant et renouvelez périodiquement l'authentification de l'application.

Liste de contrôle finale

  • Accédé au Commerce API Center avec un compte Gestionnaire intégré.
  • Saisi les informations de l'entreprise de développement et un contact pour la gestion des incidents via [Créer un compte] et accepté les conditions.
  • Enregistré une application sous [Mes applications de boutique] et obtenu l'ID/le Secret.
  • Enregistré l'IP du serveur appelant et configuré une authentification périodique.
  • Saisi client_id·client_secret dans GenieGo [Enregistrer] et enregistré.
