Manuel d'émission de la clé API

Un guide pas à pas convivial pour les débutants expliquant comment demander les clés Open API de godomall dans le NHN Commerce Developer Center, recevoir votre partner key et votre API key, et les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 10–20 min (attente d'approbation incluse)
Compte requisCompte opérateur de boutique godomall (NHN Commerce)
Clés émisespartner key · API key
RésultatIdentifiants Open API

1. Avant de commencer

L'Open API de godomall se demande dans le NHN Commerce Developer Center après l'inscription et l'approbation du développeur.

Requis

Compte opérateur de boutique

Vous avez besoin d'un compte opérateur qui gère une boutique godomall (NHN Commerce). Utilisez ce compte pour effectuer l'inscription en tant que membre et développeur dans le Developer Center.

Clés émises

partner key · API key

Une fois votre demande d'émission de clés approuvée, les clés Open API (partner key / API key) sont émises.

Remarque : L'inscription du développeur et la demande d'émission de clés passent par un processus d'approbation, l'approbation peut donc prendre un certain temps.

2. Étapes d'émission de la clé Open API

Demandez et émettez dans l'ordre ci-dessous, puis enregistrez dans GenieGo.

1

S'inscrire et obtenir l'approbation en tant que développeur

Inscrivez-vous en tant que membre et développeur dans le NHN Commerce Developer Center et obtenez l'approbation.

devcenter.nhn-commerce.com
2

Demander l'émission des clés

Dans [Open API] > [Demander l'émission des clés], remplissez le formulaire de demande et soumettez-le.

Open API → Demander l'émission des clés
3

Vérifier les clés

Vérifiez les clés Open API émises (partner key / API key).

Une fois que la partner key et l'API key sont émises, la demande a été correctement soumise et approuvée.

3. Informations à enregistrer dans GenieGo

ChampDescription
partner_keyLa partner key émise
api_keyL'API key émise (à manipuler avec précaution)
Saisissez et enregistrez les valeurs dans le GenieGo Integration Hub [Enregistrer]. Les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
Impossible de demander l'émission des clésInscription du développeur / approbation non terminéesTerminez d'abord l'inscription en tant que membre et développeur et l'approbation dans le Developer Center.
Les clés ne sont pas émises après la demandeEn attente d'approbationAttendez que le processus d'approbation soit terminé, puis vérifiez à nouveau.

Liste de vérification finale

  • Inscrit en tant que membre et développeur dans le NHN Commerce Developer Center et approbation obtenue.
  • Formulaire de demande rempli et soumis dans [Open API > Demander l'émission des clés].
  • partner key et API key émises vérifiées.
  • partner_key·api_key saisis dans GenieGo [Enregistrer] et enregistrés.
