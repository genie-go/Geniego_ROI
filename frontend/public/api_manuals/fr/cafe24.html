Cafe24
Manuel d'émission de la clé API

Un guide étape par étape adapté aux débutants pour enregistrer une application dans le Cafe24 Developer Center afin d'émettre un Client ID et un Client Secret, obtenir des jetons via l'autorisation OAuth 2.0 et les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 10–15 min
Compte requisCompte d'opérateur de boutique Cafe24
Clés émisesClient ID · Client Secret · Refresh Token
RésultatIdentifiants OAuth 2.0

1. Avant de commencer

L'intégration de Cafe24 passe par l'enregistrement du développeur et l'enregistrement de l'application, et s'authentifie via OAuth 2.0.

Requis

Compte d'opérateur de boutique

Vous avez besoin d'un compte d'opérateur qui gère une boutique Cafe24. Avec ce compte, vous effectuez l'enregistrement du développeur dans le Developer Center.

Clés émises

Client ID · Client Secret + jetons

Lors de l'enregistrement d'une application, un Client ID et un Client Secret sont émis, et vous obtenez un access token et un refresh token via l'autorisation OAuth 2.0.

Remarque : Dans GenieGo, vous enregistrez le refresh_token à longue durée de vie, et l'access token est automatiquement actualisé via celui-ci.

2. Étapes d'enregistrement de l'application et d'émission des jetons

Émettez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

S'enregistrer en tant que développeur

Accédez au Cafe24 Developer Center et enregistrez-vous en tant que développeur.

developer.cafe24.com
2

Enregistrer une application

Créez une application via [Applications] > [Enregistrer une application], et un Client ID et un Client Secret sont émis.

Applications → Enregistrer une application
3

Autorisation OAuth 2.0

Émettez un access token et un refresh token via l'autorisation OAuth 2.0.

4

Vérifier vos informations

Vérifiez mall_id (ID de la boutique), Client ID, Client Secret et refresh token.

Une fois le Client ID, le Client Secret et le refresh token tous confirmés, ils ont été émis correctement.

3. Informations à enregistrer dans GenieGo

ChampDescription
mall_idID de la boutique Cafe24
client_idLe Client ID émis
client_secretLe Client Secret émis (à manipuler avec précaution)
refresh_tokenLe Refresh Token obtenu via l'autorisation OAuth 2.0
Saisissez-les et enregistrez-les dans le GenieGo Integration Hub [Enregistrer] ; les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
Échec d'autorisationmall_id incorrect ou non-concordance des paramètres de l'applicationRevérifiez le mall_id et les informations Client définis lors de l'enregistrement de l'application.
Erreur d'expiration du jetonrefresh_token saisi incorrectementRéenregistrez le refresh_token le plus récent obtenu via l'autorisation OAuth 2.0.

Liste de vérification finale

  • Enregistré en tant que développeur dans le Cafe24 Developer Center.
  • Créé une application via [Applications > Enregistrer une application] et émis un Client ID et un Client Secret.
  • Obtenu un access token et un refresh token via l'autorisation OAuth 2.0.
  • Saisi mall_id·client_id·client_secret·refresh_token dans GenieGo [Enregistrer] et enregistré.
