Fulfillment (3PL)
Manuel d'émission de l'API Key

Un guide pour signer un contrat de consignation de fulfillment 3PL (entrée/sortie d'entrepôt) et enregistrer la 3PL API key et le warehouse ID émis dans le GenieGo Integration Hub. Pour ce canal, les identifiants sont reçus via un contrat de consignation et non émis depuis une console en libre-service.

1. Avant de commencer

Fulfillment (3PL) est un canal basé sur un contrat, où les identifiants sont émis après un contrat de consignation d'entrée/sortie d'entrepôt.

Requis

Contrat de consignation d'entrée/sortie d'entrepôt

Pour utiliser le fulfillment 3PL, un contrat de consignation d'entrée/sortie d'entrepôt avec un prestataire de fulfillment ou l'équipe d'exploitation doit d'abord être en place.

Clés émises

3PL API key · warehouse ID

Une fois le contrat conclu, un warehouse ID et une 3PL API key vous sont émis.

Remarque : Ce canal n'émet pas les identifiants directement depuis une console en libre-service ; vous recevez plutôt les identifiants via un contrat de consignation.

2. Procédure de demande d'émission

Faites la demande et recevez dans l'ordre ci-dessous, puis enregistrez dans GenieGo.

1

Faire avancer le contrat de consignation

Faites avancer un contrat de consignation d'entrée/sortie d'entrepôt avec un prestataire de fulfillment ou l'équipe d'exploitation.

2

Signer le contrat

Signez le contrat de consignation d'entrée/sortie d'entrepôt de fulfillment 3PL.

3

Recevoir les identifiants

Une fois le contrat conclu, un warehouse ID·3PL API key vous sont émis.

4

Enregistrer dans GenieGo

Saisissez l'api_key et le warehouse_id émis dans GenieGo [Enregistrer] et enregistrez.

Si vous avez reçu à la fois le warehouse ID·3PL API key du prestataire de fulfillment, ils ont été émis correctement.

3. Informations à enregistrer dans GenieGo

ChampDescription
api_keyLa 3PL API key émise
warehouse_idLe warehouse ID émis
Lorsque vous les saisissez et les enregistrez dans le GenieGo Integration Hub [Enregistrer], les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

Liste de vérification finale

  • Signé un contrat de consignation d'entrée/sortie d'entrepôt avec un prestataire de fulfillment ou l'équipe d'exploitation.
  • Reçu le warehouse ID·3PL API key une fois le contrat conclu.
  • Confirmé api_key·warehouse_id.
  • Saisi api_key·warehouse_id dans GenieGo [Enregistrer] et enregistré.
