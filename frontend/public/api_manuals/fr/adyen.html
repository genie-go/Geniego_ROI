Manuel d'émission de clé API

Un guide expliquant comment émettre une clé API (X-API-Key) et un Merchant Account dans le Customer Area après l'approbation du contrat marchand et de l'examen Adyen, puis les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 5–15 min après l'approbation
Éligibilité requiseContrat marchand Adyen · approbation de l'examen
Clés émisesClé API (X-API-Key) · Merchant Account
RésultatIdentifiants API

1. Avant de commencer

Adyen est une passerelle de paiement mondiale d'entreprise ; un contrat marchand et l'approbation de l'examen doivent donc être obtenus avant de pouvoir émettre des identifiants.

Requis

Contrat marchand · approbation de l'examen

Adyen est un PG mondial d'entreprise qui exige un contrat marchand et l'approbation de l'examen. Vous ne pouvez émettre des identifiants qu'après approbation.

Clés émises

X-API-Key · Merchant Account

Après approbation, générez une clé API (X-API-Key) dans [Developers] > [API credentials] du Customer Area et confirmez le nom de votre Merchant Account.

Condition préalable : L'auto-émission n'est pas possible avant l'approbation du contrat marchand et de l'examen. Vous devez d'abord finaliser le contrat marchand et l'examen avec Adyen.

2. Procédure de demande d'émission

Après l'approbation de l'examen, émettez et enregistrez les identifiants dans l'ordre ci-dessous.

1

Contrat marchand · approbation de l'examen

Signez un contrat marchand avec Adyen et obtenez l'approbation de l'examen.

Ventes/contrats Adyen
2

Se connecter au Customer Area

Après approbation, connectez-vous au Customer Area.

https://ca-live.adyen.com
3

Émettre les identifiants API

Dans [Developers] > [API credentials], générez une clé API (X-API-Key) et confirmez le nom de votre Merchant Account.

Developers → API credentials
4

Confirmer le numéro de lot de règlement initial

Pour la première collecte de règlement, recherchez le numéro de settlement batch le plus récent dans le Customer Area.

Une fois la clé API (X-API-Key) et le Merchant Account confirmés, vos identifiants sont prêts.

3. Informations à enregistrer dans GenieGo

ChampDescription
api_keyLa clé API émise (X-API-Key, à manipuler avec précaution)
merchant_accountLe nom confirmé de votre Merchant Account
batch_startNuméro de lot de règlement initial pour la première collecte de règlement (le numéro de settlement batch le plus récent dans le Customer Area)
Saisissez api_key·merchant_account·batch_start dans [Enregistrer] du GenieGo Integration Hub et enregistrez. Les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
Pas de menu API credentialsContrat marchand · examen non approuvésFinalisez le contrat marchand · l'examen avec Adyen, puis accédez au Customer Area.
Échec d'authentificationX-API-Key ou Merchant Account saisis incorrectementVérifiez à nouveau la clé API que vous avez émise et le nom du Merchant Account dans Developers > API credentials.

Liste de vérification finale

  • Obtenu le contrat marchand · l'approbation de l'examen Adyen.
  • Connecté au Customer Area (ca-live.adyen.com).
  • Confirmé la clé API (X-API-Key) et le Merchant Account dans Developers > API credentials.
  • Confirmé le numéro de settlement batch le plus récent (batch_start).
  • Saisi api_key·merchant_account·batch_start dans GenieGo [Enregistrer] et enregistré.
