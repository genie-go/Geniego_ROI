Manuel d’émission d’API

La procédure pour enregistrer une application dans la console développeur et préparer le Client ID, le Client Secret et l'OAuth Token afin de connecter les services de la famille Yahoo! Japan.

Structure d'authentificationCentrée sur OAuth 2.0
Compte requisYahoo! JAPAN ID
Résultat principalClient ID / Secret
AttentionLa console diffère selon l'API

1. Concept d'émission de l'API Yahoo! Japan

Selon le type de service, les API de la famille Yahoo! Japan émettent le Client ID, le Client Secret et les identifiants OAuth depuis Yahoo! JAPAN Developers ou depuis le LY/Yahoo!広告 Developer Center.

Important : Pour « Yahoo! Japan », le site d'émission et la procédure d'examen varient selon l'API que vous comptez utiliser. Vous devez distinguer les API Shopping, Publicité, Connexion et Recherche.

2. Prérequis

  • Yahoo! JAPAN ID ou compte professionnel
  • Décider du type d'API à utiliser
  • Nom de l'application et description du service
  • Redirect URI
  • URL de la politique de confidentialité
  • Domaine de production ou domaine de test

3. Ordre de base d'enregistrement de l'application

1

Accéder au site développeur

Connectez-vous à Yahoo! JAPAN Developers ou au Developer Center de l'API concernée.

developer.yahoo.co.jp / ads-developers.yahoo.co.jp
2

Créer une application

Cliquez sur le menu d'ajout d'une nouvelle application.

Create Application / Add Application
3

Saisir les informations de l'application

Saisissez le nom de l'application, la description, les informations du responsable et l'URL du site web.

Application Information
4

Saisir le Redirect URI

Saisissez exactement l'URL de retour après l'authentification OAuth.

Redirect URI
5

Sélectionner les autorisations ou l'API

Sélectionnez les autorisations d'API et les portées (scope) que vous comptez utiliser.

Scope / API Permission
6

Terminer l'enregistrement

Après l'enregistrement, vérifiez le Client ID et le Client Secret.

Client ID / Client Secret

5. Flux d'émission du token OAuth

1) Générer l'URL d'authentification de l'utilisateur avec le Client ID 2) L'utilisateur se connecte avec son compte Yahoo! 3) Recevoir l'authorization code sur le Redirect URI 4) Demander un Access Token avec l'authorization code + Client Secret 5) Appeler l'API avec l'Access Token 6) Renouveler avec le Refresh Token si nécessaire

6. Problèmes fréquents

ProblèmeCauseSolution
Erreur de Redirect URIL'URL enregistrée diffère de l'URL d'appelSaisissez-la à l'identique, jusqu'au protocole, au domaine et au chemin
Impossible d'afficher le Client SecretMasqué pour des raisons de sécuritéVérifiez la fonction View ou de réémission sur l'écran de gestion
Autorisations insuffisantesLe Scope requis n'est pas sélectionnéAjoutez le Scope dans les paramètres de l'application puis ré-authentifiez-vous
Échec de l'appel d'APIIncohérence entre l'API utilisée et le type d'application émisConsultez la documentation développeur de chaque API

7. Liste de contrôle finale

  • J'ai confirmé le type d'API Yahoo! Japan à utiliser.
  • Je me suis connecté à la bonne console développeur.
  • J'ai enregistré l'application.
  • J'ai vérifié le Client ID et le Client Secret.
  • J'ai enregistré le Redirect URI avec exactitude.
  • J'ai sélectionné le Scope requis.
  • J'ai terminé le test d'émission de l'Access Token.
