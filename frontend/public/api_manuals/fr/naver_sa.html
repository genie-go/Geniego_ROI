🟩 Beginner Guide

Naver Search Ads
Manuel d'émission de la clé API

Un guide étape par étape destiné aux débutants pour émettre les identifiants d'API (licence d'accès, clé secrète) et votre CUSTOMER_ID depuis l'outil de gestion Naver Search Ad et les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 5–10 min
Compte requisCompte annonceur Naver Search Ad
Clés émisesLicence d'accès · Clé secrète · CUSTOMER_ID
RésultatIdentifiants Search Ad API

1. Avant de commencer

L'API Naver Search Ad ne peut être demandée que depuis un compte annonceur.

Requis

Compte annonceur Search Ad

Vous avez besoin d'un compte annonceur utilisant l'outil de gestion Naver Search Ad. Vous ne pouvez pas demander l'accès à l'API avec un compte de membre Naver ordinaire.

Clés émises

Licence d'accès · Clé secrète · CUSTOMER_ID

Confirmez le CUSTOMER_ID, la licence d'accès et la clé secrète émis ensemble après votre demande d'accès à l'API.

Remarque : La clé secrète ne peut être consultée qu'au moment de l'émission, conservez-la donc en lieu sûr juste après son émission.

2. Étapes d'émission de la clé API

Émettez-la dans l'ordre ci-dessous et enregistrez-la dans GenieGo.

1

Se connecter à Search Ads

Connectez-vous à l'outil de gestion Naver Search Ad avec votre compte annonceur.

searchad.naver.com
2

Accéder à la gestion de l'utilisation de l'API

Depuis le menu supérieur [Outils], accédez à [Gestion de l'utilisation de l'API].

Outils → Gestion de l'utilisation de l'API
3

Demander le service API

Cliquez sur le bouton [Demander le service Naver Search Ad API] pour faire la demande.

Demander le service Naver Search Ad API
4

Vérifier les informations émises

Confirmez le CUSTOMER_ID, la licence d'accès et la clé secrète émis.

Une fois le CUSTOMER_ID, la licence d'accès et la clé secrète tous confirmés, ils ont été émis correctement.

3. Informations à enregistrer dans GenieGo

ChampDescription
api_keyLa licence d'accès émise
api_secretLa clé secrète émise (à manipuler avec précaution)
customer_idLe CUSTOMER_ID émis
Saisissez-les et enregistrez-les dans [Enregistrer] du GenieGo Integration Hub ; les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
Pas de menu [Gestion de l'utilisation de l'API]Pas un compte annonceur, ou autorisations insuffisantesConnectez-vous avec un compte annonceur utilisant l'outil de gestion Search Ad.
Une erreur d'authentification survientLicence d'accès ou clé secrète saisie incorrectementVérifiez à nouveau la licence d'accès, la clé secrète et le CUSTOMER_ID sur l'écran d'émission.

Liste de contrôle finale

  • Connecté avec un compte annonceur Naver Search Ad.
  • Demande de service API terminée sous [Outils > Gestion de l'utilisation de l'API].
  • CUSTOMER_ID, licence d'accès et clé secrète confirmés.
  • api_key·api_secret·customer_id saisis dans GenieGo [Enregistrer] et enregistrés.
