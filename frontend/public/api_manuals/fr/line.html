L Manuel d'émission d'API pour débutants

LINE
Manuel d'émission d'API

Ce guide résume le processus de création d'un Provider et d'un Messaging API Channel dans la LINE Developers Console, ainsi que la configuration du Channel Access Token et de la Webhook URL.

Structure d'authentificationChannel Access Token

UtilisationMessagerie · Chatbot

1. Concept d'émission de l'API LINE

L'intégration de LINE consiste à créer un Provider et un Channel dans la LINE Developers Console, puis à émettre le Channel ID, le Channel Secret et le Channel Access Token pour la Messaging API ou LINE Login.

Distinction : L'envoi de messages utilise un canal Messaging API, et l'intégration de connexion utilise un canal LINE Login.

2. Prérequis

  • Un compte LINE ou un compte professionnel
  • Une autorisation d'accès à la LINE Developers Console
  • Un LINE Official Account
  • Le nom du service et sa description
  • Webhook URL
  • URL de la politique de confidentialité et URL des conditions

3. Créer un Provider

1

Accéder à LINE Developers

Connectez-vous à la LINE Developers Console.

developers.line.biz
2

Créer un Provider

Créez un Provider en utilisant le nom de votre entreprise ou de votre service.

Create Provider
3

Saisir les informations de base

Saisissez l'exploitant du service, les coordonnées et les informations relatives aux conditions.

Provider Settings

4. Créer un canal Messaging API

1

Créer un nouveau canal

Cliquez sur Create a new channel à l'intérieur du Provider.

Create a new channel
2

Sélectionner Messaging API

Choisissez Messaging API comme type de canal.

Messaging API
3

Saisir les informations du canal

Saisissez le nom du canal, la description, la catégorie, l'e-mail, etc.

Channel Information
4

Associer un LINE Official Account

Associez un compte officiel à partir duquel envoyer des messages, ou créez-en un nouveau.

LINE Official Account
5

Terminer la création du canal

Après la création, vérifiez le Channel ID et le Channel Secret.

Channel ID / Channel Secret

5. Émettre le Channel Access Token

1

Aller à l'onglet Messaging API

Sur l'écran du canal que vous avez créé, accédez à l'onglet Messaging API.

Messaging API tab
2

Trouver l'élément Channel access token

Repérez la zone Channel access token en bas.

Channel access token
3

Cliquer sur Issue

Cliquez sur le bouton Issue ou Reissue pour émettre le token.

Issue
4

Le stocker en toute sécurité sur le serveur

Stockez le token émis dans une variable d'environnement du serveur ou dans Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
Le Channel Access Token est une clé secrète dotée de l'autorisation d'envoyer des messages. Il ne doit pas être exposé dans le HTML, le frontend ou les dépôts publics.

6. Configurer le Webhook

1

Saisir la Webhook URL

Saisissez l'URL du serveur qui recevra les événements de messages des utilisateurs.

Webhook URL
2

Activer Use webhook

Réglez l'option Use webhook sur Enabled.

Use webhook: Enabled
3

Test Verify

Envoyez une requête de test depuis LINE vers votre serveur pour confirmer la connexion.

Verify
4

Vérifier le code de réponse

Confirmez que le serveur renvoie 200 OK.

HTTP 200 OK

7. Exemple d'envoi de message

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Bonjour. Ceci est un test de message LINE."} ] }

8. Liste de contrôle finale

  • Créé un Provider.
  • Créé un canal Messaging API.
  • Associé un LINE Official Account.
  • Vérifié le Channel ID et le Channel Secret.
  • Émis le Channel Access Token.
  • Enregistré la Webhook URL et réussi le test Verify.
  • Stocké le token en toute sécurité dans une variable d'environnement du serveur.
