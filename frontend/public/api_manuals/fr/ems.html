EMS (Korea Post)
Manuel de délivrance de la clé API

Korea Post International Express (EMS) est un service sous contrat reposant sur un code de contrat (client). Ce manuel décrit la procédure d'enregistrement de la clé API Korea Post et du code client (contrat) que vous recevez après la signature d'un contrat EMS dans le GenieGo Integration Hub.

Durée estiméeVariable selon le délai de traitement du contrat
Compte requisContrat EMS Korea Post (code client)
Clés délivréesClé API Korea Post · Code client (contrat)
RésultatIdentifiants API · Code de contrat

1. Avant de commencer

EMS n'est pas une délivrance en libre-service via une console, mais un service sous contrat reposant sur un code de contrat (client).

Remarque : Korea Post International Express (EMS) repose sur un code de contrat (client). Ce n'est pas une méthode où vous délivrez vous-même les identifiants, mais une procédure où vous signez un contrat EMS puis recevez un code client (contrat) et une clé API Korea Post.
Prérequis

Contrat EMS Korea Post (code client)

Vous avez besoin d'un code de contrat (client) Korea Post International Express (EMS). Il n'est délivré que par contrat.

Identifiants reçus

Clé API Korea Post · Code client (contrat)

Une fois le contrat conclu, vous recevez un code client (contrat) et une clé API Korea Post.

2. Procédure de demande de délivrance

Comme il s'agit d'un service sous contrat, signez le contrat, recevez vos identifiants, puis enregistrez-les dans l'ordre ci-dessous.

1

Signer le contrat EMS

Signez un contrat EMS via le Bureau de poste en ligne ou un guichet de contrats.

2

Recevoir le code client et la clé API

Une fois le contrat conclu, vous recevez un code client (contrat) et une clé API Korea Post.

3

Enregistrer dans GenieGo

Saisissez les identifiants reçus dans GenieGo [Enregistrer] et enregistrez.

Comme il s'agit d'un service sous contrat, la réception de vos identifiants peut prendre un certain temps selon l'état de traitement du contrat.

3. Informations à enregistrer dans GenieGo

ChampDescription
api_keyLa clé API Korea Post délivrée
cust_codeCode client (contrat)
Lorsque vous les saisissez et les enregistrez dans GenieGo Integration Hub [Enregistrer], les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
Impossible de délivrer les identifiantsContrat EMS non concluConcluez d'abord le contrat EMS via le Bureau de poste en ligne ou un guichet de contrats.
Erreur d'authentification après l'enregistrementClé API ou code client (contrat) saisis incorrectementVérifiez à nouveau l'api_key et le cust_code délivrés.

Liste de contrôle finale

  • Signé un contrat Korea Post International Express (EMS).
  • Reçu le code client (contrat) et la clé API Korea Post.
  • Vérifié les identifiants délivrés.
  • Saisi api_key·cust_code dans GenieGo [Enregistrer] et enregistré.
