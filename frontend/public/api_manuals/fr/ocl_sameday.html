GenieGo Same-Day Delivery
Manuel d'émission de la clé API

Un manuel expliquant comment enregistrer la clé API de Livraison le Jour Même et l'ID marchand—émis après la signature d'un contrat pour le service GenieGo Livraison le Jour Même—dans le GenieGo Integration Hub. Pour ce canal, les identifiants sont reçus via un contrat avec l'équipe d'exploitation plutôt qu'émis soi-même depuis une console.

1. Avant de commencer

GenieGo Livraison le Jour Même est un canal basé sur un contrat, où les identifiants sont émis après un contrat de service avec l'équipe d'exploitation.

Requis

Contrat de service Livraison le Jour Même

Pour utiliser GenieGo Livraison le Jour Même, un contrat d'utilisation du service avec l'équipe d'exploitation doit d'abord être en place. Sans contrat, les identifiants ne peuvent pas être émis.

Clés émises

Clé API · ID marchand · Région de service

Une fois le contrat finalisé, l'équipe d'exploitation vous émet l'ID marchand et la clé API ainsi qu'une région de service.

Remarque : Ce canal n'est pas émis soi-même depuis une console ; les identifiants sont plutôt reçus via un contrat avec l'équipe d'exploitation.

2. Procédure de demande d'émission

Demandez et recevez dans l'ordre ci-dessous, puis enregistrez dans GenieGo.

1

Demander la Livraison le Jour Même

Demandez à l'équipe d'exploitation d'utiliser GenieGo Livraison le Jour Même et avancez avec le contrat.

2

Signer le contrat

Signez le contrat d'utilisation du service Livraison le Jour Même avec l'équipe d'exploitation.

3

Recevoir les identifiants

Une fois le contrat finalisé, l'ID marchand, la clé API et la région de service vous sont émis.

4

Enregistrer dans GenieGo

Saisissez les api_key, merchant_id et region émis dans GenieGo [Enregistrer] et enregistrez.

Si vous avez reçu l'ID marchand, la clé API et la région de service entièrement de l'équipe d'exploitation, l'émission est terminée.

3. Informations à enregistrer dans GenieGo

ChampDescription
api_keyLa clé API GenieGo Livraison le Jour Même émise
merchant_idL'ID marchand émis
regionRégion de service (p. ex. Région de la capitale)
Une fois saisis et enregistrés dans le GenieGo Integration Hub [Enregistrer], les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

Liste de contrôle finale

  • Reçu l'ID marchand, la clé API et la région de service après la finalisation du contrat.
  • Confirmé api_key, merchant_id et region.
  • Saisi api_key, merchant_id et region dans GenieGo [Enregistrer] et enregistré.
