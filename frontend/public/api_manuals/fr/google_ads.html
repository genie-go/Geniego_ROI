Google Ads
Manuel de délivrance de la clé API

Un guide qui vous accompagne dans la délivrance d'un developer token dans le Centre d'API de votre compte administrateur (MCC) Google Ads, la création d'un client OAuth dans Google Cloud pour obtenir un refresh token, et l'enregistrement de l'ensemble dans le GenieGo Integration Hub.

Temps estiméEnviron 20–40 min (examen du developer token séparé)
Compte requisCompte administrateur (MCC) Google Ads · Compte Google
Clés délivréesdeveloper token · client_id/secret · refresh_token · customer_id
RésultatIdentifiants Google Ads API

1. Avant de commencer

L'intégration de Google Ads nécessite à la fois un compte administrateur (MCC) et une configuration OAuth dans Google Cloud.

developer token

Centre d'API MCC

Demandez un developer token dans le Centre d'API de votre compte administrateur (MCC) Google Ads. L'accès de base (Basic) est approuvé après un examen de Google.

OAuth

Client Google Cloud

Sur console.cloud.google.com, créez un client OAuth (client_id·client_secret) et authentifiez-vous pour obtenir un refresh token.

Remarque : L'accès de base au developer token est approuvé après un examen de Google, et avant l'approbation, seuls les comptes de test peuvent être appelés. Si vous n'avez pas de compte administrateur (MCC), créez-en un d'abord sur ads.google.com.

2. Étapes de délivrance

Délivrez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

Connectez-vous au compte administrateur · Accédez au Centre d'API

Connectez-vous à votre compte administrateur Google Ads, puis accédez à [Outils et paramètres] > [Configuration] > [Centre d'API] en haut.

Outils et paramètres → Configuration → Centre d'API
2

Demandez un developer token

Demandez un [developer token] dans le Centre d'API. L'accès de base (Basic) est approuvé après un examen de Google, et avant l'approbation, seuls les comptes de test peuvent être appelés.

3

Configurez un projet Cloud · écran de consentement OAuth

Créez un projet et configurez [API et services] > [Écran de consentement OAuth].

console.cloud.google.com → API et services → Écran de consentement OAuth
4

Créez un ID client OAuth

Créez [Identifiants] > [Créer des identifiants] > [ID client OAuth] pour obtenir un client_id et un client_secret (scope : adwords).

Identifiants → Créer des identifiants → ID client OAuth
5

Délivrez un refresh token

Obtenez un refresh token via le processus d'authentification OAuth (utilisez Google OAuth Playground ou votre propre authentification).

6

Trouvez l'ID client

Trouvez l'ID client à 10 chiffres (customer_id) du compte publicitaire que vous allez appeler.

Une fois le developer token approuvé et client_id·client_secret·refresh_token·customer_id tous prêts, la délivrance est terminée.

3. Informations à enregistrer dans GenieGo

ChampDescription
developer_tokenLe developer token délivré dans le Centre d'API
client_idID client OAuth
client_secretSecret client OAuth (à manipuler avec précaution)
refresh_tokenLe refresh token délivré via l'authentification OAuth (à saisir dans le champ [Jeton d'accès])
customer_idL'ID client à 10 chiffres du compte publicitaire cible
Saisissez le developer token, client_id, client_secret, refresh_token et customer_id dans GenieGo [Enregistrer] et enregistrez. Les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
Seuls les comptes de test peuvent être appelésExamen de l'accès de base du developer token non approuvéAttendez que l'examen Google du developer token que vous avez demandé dans le Centre d'API soit approuvé, puis appelez les comptes de production.
Échec de l'authentificationScope (adwords) manquant ou refresh_token incorrectAuthentifiez-vous à nouveau avec le scope adwords inclus dans le client OAuth, et vérifiez à nouveau le refresh_token.
Erreur de format de l'ID clientcustomer_id à 10 chiffres incorrectVérifiez à nouveau l'ID client à 10 chiffres du compte publicitaire cible et saisissez-le.

Liste de contrôle finale

  • Préparé un compte administrateur (MCC) Google Ads et connecté.
  • Demandé un developer token dans le Centre d'API (l'accès de base passe par un examen de Google).
  • Créé un écran de consentement OAuth et un ID/Secret client OAuth sur console.cloud.google.com (scope : adwords).
  • Obtenu un refresh token via l'authentification OAuth.
  • Trouvé le customer_id à 10 chiffres du compte cible.
  • Saisi le developer token·client_id/secret·refresh_token·customer_id dans GenieGo [Enregistrer] et enregistré.
