🛒 Guide du débutant

Rakuten
Manuel d'émission de clé API

Un guide pas à pas convivial pour les débutants sur l'émission des identifiants du RMS Web Service (Service Secret et License Key) depuis la console vendeur Rakuten RMS et leur enregistrement dans le GenieGo Integration Hub.

Temps estiméEnviron 5–15 min
Compte nécessaireCompte vendeur Rakuten (Ichiba)
Clés émisesService Secret · License Key
RésultatIdentifiants du RMS Web Service

1. Avant de commencer

Les clés vendeur (Ichiba) sont émises depuis le RMS Web Service.

Requis

Compte vendeur Rakuten

Vous avez besoin d'un compte vendeur Rakuten (Ichiba). Vous avez besoin d'un compte capable de se connecter via RMS R-Login (glogin.rms.rakuten.co.jp).

Clés émises

Service Secret · License Key

Vous obtiendrez un serviceSecret et une licenseKey depuis le RMS Web Service. La License Key expire au bout de 90 jours et peut être réémise.

Attention : Le RMS Web Service (glogin.rms.rakuten.co.jp) qui émet les clés vendeur (Ichiba) est différent de webservice.rakuten.co.jp, destiné à l'affiliation/la recherche de produits. Veillez à émettre vos clés du côté RMS.

2. Étapes d'émission

Émettez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

Connectez-vous au portail développeur RMS

Connectez-vous au portail développeur Rakuten (RMS/Webservice) avec votre compte vendeur.

https://glogin.rms.rakuten.co.jp
2

Accédez au RMS Web Service

Après vous être connecté via RMS R-Login, accédez à la zone RMS Web Service.

RMS R-Login → RMS Web Service
3

Émettez le Service Secret · License Key

Depuis le RMS Web Service, émettez le Service Secret et la License Key. La License Key expire au bout de 90 jours, alors réémettez-la à son expiration.

4

Vérifiez les clés

Vérifiez le Service Secret et la License Key émis, ainsi que la Shop URL de votre boutique.

Dès que le Service Secret et la License Key apparaissent sur l'écran d'émission, ils ont été émis correctement.

3. Informations à enregistrer dans GenieGo

ChampDescription
service_secretLe Service Secret émis
license_keyLa License Key émise (expire au bout de 90 jours·réémettable, à manipuler avec soin)
shop_urlLa Shop URL de votre boutique
Dans le GenieGo Integration Hub [Enregistrer], saisissez service_secret·license_key·shop_url puis enregistrez. Les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes fréquents

ProblèmeCauseSolution
De mauvaises clés ont été émisesÉmises depuis webservice.rakuten.co.jp (pour l'affiliation/la recherche de produits)Réémettez-les depuis le RMS Web Service sous RMS R-Login (glogin.rms.rakuten.co.jp).
L'authentification échoue au bout d'un certain tempsLa License Key a expiré au bout de 90 joursRéémettez la License Key depuis le RMS Web Service et saisissez-la à nouveau dans [Enregistrer].

Liste de contrôle finale

  • Je me suis connecté à RMS R-Login avec un compte vendeur Rakuten (Ichiba).
  • J'ai émis le Service Secret·License Key depuis le RMS Web Service.
  • J'ai vérifié la Shop URL de ma boutique.
  • J'ai saisi service_secret·license_key·shop_url dans GenieGo [Enregistrer] et enregistré.
