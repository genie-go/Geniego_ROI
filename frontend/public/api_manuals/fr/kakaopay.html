💛 Guide du débutant

KakaoPay
Manuel de délivrance de la clé API

Un guide qui vous explique comment créer une application dans le Centre des développeurs KakaoPay (developers.kakaopay.com) pour délivrer un code marchand (CID) et une Secret Key, puis les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 5–15 min
Compte requisContrat marchand KakaoPay · Compte du Centre des développeurs
Clés délivréesCode marchand (CID) · Secret Key
RésultatIdentifiants d'intégration de paiement

1. Avant de commencer

KakaoPay délivre les identifiants dans le Centre des développeurs une fois qu'un contrat marchand est en place.

Requis

Contrat marchand · Compte du Centre des développeurs

Votre contrat marchand KakaoPay doit être finalisé, et vous devez pouvoir vous inscrire et vous connecter au Centre des développeurs (developers.kakaopay.com).

Clés délivrées

CID · Secret Key

Lorsque vous créez une application, un code marchand (CID) et une Secret Key sont délivrés.

Remarque : Lors de l'appel de l'API KakaoPay, l'en-tête d'authentification utilise le format Authorization: SECRET_KEY.

2. Étapes de délivrance

Délivrez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

S'inscrire et se connecter au Centre des développeurs

Inscrivez-vous au Centre des développeurs KakaoPay et connectez-vous.

https://developers.kakaopay.com
2

Créer une application

Créez une application pour délivrer ou vérifier le code marchand (CID) et la Secret Key.

3

Vérifier l'en-tête d'authentification

La Secret Key délivrée est utilisée dans l'en-tête d'authentification au format Authorization: SECRET_KEY.

Authorization: SECRET_KEY ${secret_key}
Une fois le code marchand (CID) et la Secret Key délivrés, tout est correctement préparé.

3. Informations à enregistrer dans GenieGo

ChampDescription
cidLe code marchand (CID) délivré
secret_keyLa Secret Key délivrée (à manipuler avec précaution, Authorization: SECRET_KEY)
Saisissez-les et enregistrez-les dans le GenieGo Integration Hub [Enregistrer] ; les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes courants

ProblèmeCauseSolution
Erreur d'en-tête d'authentificationLe format de l'en-tête Authorization est incorrectVérifiez que la Secret Key est transmise au format Authorization: SECRET_KEY.
Impossible de créer l'applicationContrat marchand non finaliséFinalisez le contrat marchand, puis créez l'application dans le Centre des développeurs.

Liste de contrôle finale

  • J'ai finalisé le contrat marchand KakaoPay et je me suis inscrit au Centre des développeurs (developers.kakaopay.com).
  • J'ai créé une application et délivré le code marchand (CID) et la Secret Key.
  • J'ai vérifié le format de l'en-tête d'authentification Authorization: SECRET_KEY.
  • J'ai saisi cid·secret_key dans GenieGo [Enregistrer] et enregistré.
