Manuel de délivrance de la clé API

Un guide pas à pas pour débutants expliquant comment délivrer une clé API Billing (v2) (Bearer, pdl_live_…) depuis les Developer Tools de votre Paddle Dashboard et l'enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 5–10 min
Compte requisCompte Paddle
Clés délivréesClé API Billing (pdl_live_…)
RésultatIdentifiants API

1. Avant de commencer

Paddle est une plateforme de paiement de type Merchant of Record pour les produits SaaS et numériques (Billing v2).

Requis

Compte Paddle

La vente de produits SaaS et numériques nécessite un compte Paddle (Billing v2).

Clés délivrées

Clé API Billing

Vous générez une clé API Billing au format Bearer (pdl_live_…). Ce n'est pas l'ancienne méthode Classic vendor_id/auth_code.

Remarque : La clé à délivrer est une clé API Billing (v2) (Bearer, pdl_live_…). Ne la confondez pas avec l'ancienne méthode Classic vendor_id/auth_code.

2. Étapes de délivrance

Délivrez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

Connectez-vous au Paddle Dashboard

Connectez-vous au Dashboard avec le compte Paddle que vous avez préparé.

https://vendors.paddle.com
2

Accédez au menu API keys

Accédez à [Developer Tools] > [Authentication] > [API keys].

Developer Tools → Authentication → API keys
3

Créez une clé API Billing

Créez une clé API Billing (Bearer, pdl_live_…). Ce n'est pas l'ancienne méthode Classic vendor_id/auth_code.

Une fois qu'une clé API Billing commençant par pdl_live_… est créée, elle a été délivrée correctement.

3. Informations à enregistrer dans GenieGo

ChampDescription
api_keyLa clé API Billing (v2) délivrée (Bearer, pdl_live_…, à manipuler avec précaution)
Saisissez api_key dans le GenieGo Integration Hub [Enregistrer] et enregistrez. Les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.
