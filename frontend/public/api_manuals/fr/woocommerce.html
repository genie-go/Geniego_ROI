🟣 Guide du débutant

WooCommerce
Manuel d'émission de clé API

Un guide étape par étape adapté aux débutants pour émettre une Consumer Key et un Consumer Secret depuis les paramètres REST API de votre administration WooCommerce et les enregistrer dans le GenieGo Integration Hub.

Temps estiméEnviron 5–10 min
Compte requisDroits d'administrateur WordPress/WooCommerce
Clés émisesConsumer Key · Consumer Secret
RésultatIdentifiants REST API

1. Avant de commencer

Les clés REST API ne peuvent être émises qu'avec des droits d'administrateur.

Requis

Droits d'administrateur WooCommerce

Vous avez besoin de droits d'administrateur WordPress/WooCommerce. Les clés REST API ne peuvent pas être émises avec des droits d'utilisateur ordinaire.

Clés émises

Consumer Key · Consumer Secret

L'ajout d'une clé dans les paramètres REST API émet une Consumer Key et un Consumer Secret.

2. Étapes d'émission

Émettez dans l'ordre ci-dessous et enregistrez dans GenieGo.

1

Aller aux paramètres REST API

Dans WooCommerce, allez dans [Réglages] > [Avancé] > [REST API].

WooCommerce → Réglages → Avancé → REST API
2

Ajouter une clé

Cliquez sur [Ajouter une clé], saisissez une description et sélectionnez les autorisations (Lecture/Écriture), puis générez.

Ajouter une clé
3

Copier les clés

Copiez la Consumer Key et le Consumer Secret émis.

4

Confirmer les informations

Confirmez la Consumer Key·Consumer Secret copiés et votre propre store URL.

Si la Consumer Key et le Consumer Secret s'affichent sur l'écran d'émission, les clés ont été émises correctement.
Attention : Le Consumer Secret n'est souvent affiché en entier que juste après l'émission, veillez donc à le copier en toute sécurité à ce moment-là.

3. Informations à enregistrer dans GenieGo

ChampDescription
site_urlstore URL
consumer_keyLa Consumer Key émise
consumer_secretLe Consumer Secret émis (à manipuler avec précaution)
Saisissez site_url·consumer_key·consumer_secret dans le GenieGo Integration Hub [Enregistrer] et enregistrez. Les identifiants sont chiffrés au repos avec AES-256-GCM et masqués à la lecture.

4. Problèmes courants

ProblèmeCauseSolution
Pas de menu REST APIDroits d'administrateur insuffisantsConnectez-vous avec un compte disposant de droits d'administrateur et vérifiez Réglages > Avancé > REST API.
Les opérations d'écriture échouentAutorisation en lecture seule sélectionnée lors de la création de la cléRéémettez avec des autorisations Lecture/Écriture lors de l'[Ajouter une clé].

Liste de vérification finale

  • Connecté avec des droits d'administrateur WordPress/WooCommerce.
  • Ajouté une clé dans Réglages > Avancé > REST API.
  • Sélectionné les autorisations (Lecture/Écriture) et copié la Consumer Key·Consumer Secret.
  • Saisi site_url·consumer_key·consumer_secret dans GenieGo [Enregistrer] et enregistré.
