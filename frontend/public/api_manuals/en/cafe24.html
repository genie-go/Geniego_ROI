API Key Issuance Manual

A beginner-friendly, step-by-step guide to registering an app in the Cafe24 Developer Center to issue a Client ID and Client Secret, obtaining tokens via OAuth 2.0 authorization, and registering them in the GenieGo Integration Hub.

Est. timeAbout 10–15 min
Account neededCafe24 store operator account
Keys issuedClient ID · Client Secret · Refresh Token
ResultOAuth 2.0 credentials

1. Before You Begin

Cafe24 integration goes through developer registration and app registration, and authenticates via OAuth 2.0.

Required

Store operator account

You need an operator account that runs a Cafe24 store. You complete developer registration in the Developer Center with this account.

Keys issued

Client ID · Client Secret + tokens

When you register an app, a Client ID and Client Secret are issued, and you obtain an access token and a refresh token through OAuth 2.0 authorization.

Note: You register the long-lived refresh_token in GenieGo, and the access token is automatically refreshed through it.

2. App Registration and Token Issuance Steps

Issue in the order below and register in GenieGo.

1

Register as a developer

Go to the Cafe24 Developer Center and register as a developer.

developer.cafe24.com
2

Register an app

Create an app via [Apps] > [Register App], and a Client ID and Client Secret are issued.

Apps → Register App
3

OAuth 2.0 authorization

Obtain an access token and a refresh token through OAuth 2.0 authorization.

4

Check your information

Check your mall_id (mall ID), Client ID, Client Secret, and refresh token.

Once the Client ID, Client Secret, and refresh token are all confirmed, they have been issued correctly.

3. Information to Register in GenieGo

FieldDescription
mall_idCafe24 store ID
client_idThe issued Client ID
client_secretThe issued Client Secret (handle with care)
refresh_tokenThe Refresh Token obtained via OAuth 2.0 authorization
Enter and save them in the GenieGo Integration Hub [Register], and the credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
Authorization failureIncorrect mall_id or app settings mismatchRecheck the mall_id and Client information set during app registration.
Token expiration errorIncorrectly entered refresh_tokenRe-register the latest refresh_token obtained via OAuth 2.0 authorization.

Final Checklist

  • Registered as a developer in the Cafe24 Developer Center.
  • Created an app via [Apps > Register App] and issued a Client ID and Client Secret.
  • Obtained an access token and refresh token via OAuth 2.0 authorization.
  • Entered mall_id·client_id·client_secret·refresh_token in GenieGo [Register] and saved.
