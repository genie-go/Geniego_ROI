1. Yahoo! Japan API Issuance Concept

Depending on the service type, Yahoo! Japan family APIs issue the Client ID, Client Secret, and OAuth credentials from Yahoo! JAPAN Developers or the LY/Yahoo!広告 Developer Center.

Important: For "Yahoo! Japan," the issuance site and review process vary depending on which API you intend to use. You must distinguish between the Shopping, Advertising, Login, and Search APIs.

2. Prerequisites

  • Yahoo! JAPAN ID or business account
  • Decide which type of API to use
  • App name and service description
  • Redirect URI
  • Privacy policy URL
  • Production domain or test domain

5. OAuth Token Issuance Flow

1) Generate the user authentication URL with the Client ID 2) The user logs in with their Yahoo! account 3) Receive the authorization code at the Redirect URI 4) Request an Access Token with the authorization code + Client Secret 5) Call the API with the Access Token 6) Refresh with the Refresh Token when needed
