EMS (Korea Post)
API Key Issuance Manual

Korea Post International Express (EMS) is a contract-based service that relies on a contract (customer) code. This manual outlines the procedure for registering the Korea Post API key and customer (contract) code you receive after signing an EMS contract in the GenieGo Integration Hub.

1. Before You Begin

EMS is not a self-service console issuance; it is a contract-based service that relies on a contract (customer) code.

Note: Korea Post International Express (EMS) is based on a contract (customer) code. Rather than issuing credentials yourself, you sign an EMS contract and then receive a customer (contract) code and a Korea Post API key.
Requirement

Korea Post EMS contract (customer code)

You need a Korea Post International Express (EMS) contract (customer) code. It is issued only through a contract.

Credentials received

Korea Post API key · Customer (contract) code

After the contract is completed, you receive a customer (contract) code and a Korea Post API key.

2. Issuance Application Procedure

Since this is a contract-based service, sign the contract, receive your credentials, and register them in the order below.

1

Sign the EMS contract

Sign an EMS contract through Internet Post Office or a contract counter.

2

Receive the customer code and API key

After the contract is completed, you receive a customer (contract) code and a Korea Post API key.

3

Register in GenieGo

Enter the credentials you received into GenieGo [Register] and save.

Because this is a contract-based service, it may take some time to receive your credentials depending on the contract processing status.

3. Information to Register in GenieGo

FieldDescription
api_keyThe issued Korea Post API key
cust_codeCustomer (contract) code
When you enter and save them in the GenieGo Integration Hub [Register], credentials are encrypted at rest with AES-256-GCM and masked on read.
