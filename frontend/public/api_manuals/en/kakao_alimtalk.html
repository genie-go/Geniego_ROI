1. Kakao Alimtalk Concept

Alimtalk is a business-message service that sends informational messages such as orders, reservations, shipping, and payments via Kakao Talk. You cannot send directly with a regular Kakao developer app key alone; business channel verification and a sending agency/platform setup are required.

Important: Alimtalk is centered on informational messages rather than promotional ones, and templates require prior review.

2. What You Need

  • Kakao Business account
  • Kakao Talk channel
  • Business verification documents such as a business registration certificate
  • Alimtalk template wording to send
  • A sending agency or cloud messaging service account
  • A policy for collecting recipient phone numbers and managing consent

3. Preparing the Kakao Talk Channel

1

Access Kakao Business

Log in to the Kakao Business admin center.

business.kakao.com
2

Create a Kakao Talk channel

Create a channel under your brand or service name.

Kakao Talk Channel Admin Center
3

Complete business verification

Submit your business information and supporting documents to verify the business channel.

Business Verification
4

Check the channel search ID

Check the channel ID or search ID required for Alimtalk sending settings.

Channel Profile → Search ID
Business verification must be completed before you can register an Alimtalk sender profile.

4. Registering the Sender Profile

1

Choose a sending platform

Select an Alimtalk provider such as NHN Cloud, Naver Cloud SENS, Infobank, or Bizgo.

Messaging Provider
2

Create a project

Create an Alimtalk project on the platform you selected.

Project / Service
3

Connect the Kakao Talk channel

Register the details of your business-verified Kakao Talk channel.

Sender Profile
4

Sender profile review

Complete channel ownership confirmation and sender profile review.

Profile Review
5

Confirm API credentials

Check the App Key, Secret Key, API Key, and other values provided by the platform.

API Key / Secret

5. Registering and Approving Templates

1

Write the template

Write informational wording such as order confirmations and shipping notices.

Template Message
2

Set variables

Set dynamically changing values such as name, order number, and tracking number as variables.

#{variable}
3

Request review

Save the template and submit it for Kakao review.

Request Approval
4

Confirm approval

Confirm the approved template code or ID.

Template Code
If the wording before review differs from the actually sent wording, sending may fail or sanctions may occur.

6. Example API Sending Flow

1) Call the Alimtalk API Endpoint from your server
2) Authenticate with the API Key or Secret
3) Pass the recipient phone number, template code, and variable values
4) Receive the sending result
5) Handle whether to send an SMS fallback on failure

7. Final Checklist

  • Created a Kakao Talk channel.
  • Completed business verification.
  • Selected an Alimtalk sending platform.
  • Registered a Sender Profile.
  • Confirmed the API Key or Secret.
  • Registered and got the template approved.
  • Prepared a sending-failure and fallback-SMS policy.
