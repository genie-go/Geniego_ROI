API Issuance Manual

This guide walks through creating a Provider and a Messaging API Channel in the LINE Developers Console and configuring the Channel Access Token and Webhook URL.

1. LINE API Issuance Concept

LINE integration works by creating a Provider and a Channel in the LINE Developers Console, then issuing the Channel ID, Channel Secret, and Channel Access Token for the Messaging API or LINE Login.

Distinction: Use a Messaging API channel to send messages, and a LINE Login channel for login integration.

2. Prerequisites

  • A LINE account or business account
  • Access permission to the LINE Developers Console
  • A LINE Official Account
  • Service name and description
  • Webhook URL
  • Privacy policy URL and terms of service URL

3. Create a Provider

1

Access LINE Developers

Sign in to the LINE Developers Console.

developers.line.biz
2

Create a Provider

Create a Provider using your company name or service name.

Create Provider
3

Enter basic information

Enter the service operator, contact details, and terms information.

Provider Settings

4. Create a Messaging API Channel

1

Create a new channel

Click Create a new channel inside the Provider.

Create a new channel
2

Select Messaging API

Choose Messaging API as the channel type.

Messaging API
3

Enter channel information

Enter the channel name, description, category, email, and so on.

Channel Information
4

Link a LINE Official Account

Link an existing official account to send messages from, or create a new one.

LINE Official Account
5

Finish channel creation

After creation, check the Channel ID and Channel Secret.

Channel ID / Channel Secret

5. Issue the Channel Access Token

1

Go to the Messaging API tab

On the channel screen you created, move to the Messaging API tab.

Messaging API tab
2

Find the Channel access token item

Locate the Channel access token area at the bottom.

Channel access token
3

Click Issue

Click the Issue or Reissue button to issue the token.

Issue
4

Store it safely on the server

Store the issued token in a server environment variable or Secret Manager.

LINE_CHANNEL_ACCESS_TOKEN
The Channel Access Token is a secret key with permission to send messages. It must not be exposed in HTML, the frontend, or public repositories.

6. Configure the Webhook

1

Enter the Webhook URL

Enter the server URL that will receive user message events.

Webhook URL
2

Enable Use webhook

Change the Use webhook option to Enabled.

Use webhook: Enabled
3

Run the Verify test

Send a test request from LINE to your server to confirm the connection.

Verify
4

Check the response code

Confirm that the server returns 200 OK.

HTTP 200 OK

7. Message Sending Example

POST https://api.line.me/v2/bot/message/push Authorization: Bearer {CHANNEL_ACCESS_TOKEN} Content-Type: application/json { "to": "USER_ID", "messages": [ {"type": "text", "text": "Hello. This is a LINE message test."} ] }

8. Final Checklist

  • Created a Provider.
  • Created a Messaging API channel.
  • Linked a LINE Official Account.
  • Checked the Channel ID and Channel Secret.
  • Issued the Channel Access Token.
  • Registered the Webhook URL and passed Verify.
  • Stored the token safely in a server environment variable.
