API Key Issuance Manual

A beginner-friendly, step-by-step guide to issuing a Consumer Key and Consumer Secret from the REST API settings in your WooCommerce admin and registering them in the GenieGo Integration Hub.

Est. timeAbout 5–10 min
Account neededWordPress/WooCommerce admin privileges
Keys issuedConsumer Key · Consumer Secret
ResultREST API credentials

1. Before You Begin

REST API keys can only be issued with admin privileges.

Required

WooCommerce admin privileges

You need WordPress/WooCommerce admin privileges. REST API keys cannot be issued with regular user privileges.

Keys issued

Consumer Key · Consumer Secret

Adding a key in the REST API settings issues a Consumer Key and a Consumer Secret.

2. Issuance Steps

Issue in the order below and register in GenieGo.

1

Go to the REST API settings

In WooCommerce, go to [Settings] > [Advanced] > [REST API].

WooCommerce → Settings → Advanced → REST API
2

Add a key

Click [Add key], enter a description, select the permissions (Read/Write), and generate.

Add key
3

Copy the keys

Copy the issued Consumer Key and Consumer Secret.

4

Confirm the information

Confirm the copied Consumer Key, Consumer Secret, and your own store URL.

If the Consumer Key and Consumer Secret are shown on the issuance screen, the keys have been issued correctly.
Caution: The Consumer Secret is often shown in full only right after issuance, so be sure to copy it safely at that moment.

3. Information to Register in GenieGo

FieldDescription
site_urlStore URL
consumer_keyThe issued Consumer Key
consumer_secretThe issued Consumer Secret (handle with care)
Enter site_url·consumer_key·consumer_secret in the GenieGo Integration Hub [Register] and save. Credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
No REST API menuInsufficient admin privilegesSign in with an account that has admin privileges and check Settings > Advanced > REST API.
Write operations failRead-only permission selected when creating the keyRe-issue with Read/Write permissions when you [Add key].

Final Checklist

  • Signed in with WordPress/WooCommerce admin privileges.
  • Added a key under Settings > Advanced > REST API.
  • Selected the permissions (Read/Write) and copied the Consumer Key·Consumer Secret.
  • Entered site_url·consumer_key·consumer_secret in GenieGo [Register] and saved.
