KG Inicis
API Key Issuance Manual

A guide that walks you through issuing your INIAPI Key and Web Standard sign key (SignKey) in the KG Inicis merchant admin and registering them together with your store ID (MID) in the GenieGo Integration Hub.

Est. timeAbout 5–15 min
Account neededKG Inicis merchant contract · Merchant admin account
Keys issuedMID · API key · Sign key (SignKey)
ResultPayment integration credentials

1. Before You Begin

KG Inicis can only issue authentication keys after the merchant contract is complete.

Required

Merchant contract · Merchant admin account

You must have a signed KG Inicis merchant contract and be able to log in with a merchant admin account.

Keys issued

INIAPI Key · Web Standard sign key

Create an INIAPI Key in the merchant admin, and issue the Web Standard sign key together via Web Payment signkey generation.

Note: Registration requires three items: store ID (MID), sign key (SignKey), and API key.

2. Issuance Steps

Issue in the order below and register in GenieGo.

1

Log in to the merchant admin

Log in to the Inicis merchant admin.

https://iniweb.inicis.com
2

Go to the Additional Info menu

Go to [Store Info] > [Contract Info] > [Additional Info].

Store Info → Contract Info → Additional Info
3

Create an INIAPI Key

Create an INIAPI Key.

4

Create the Web Standard sign key

Click [Web Payment signkey generation] to issue the Web Standard sign key (SignKey).

Web Payment signkey generation
Once both the INIAPI Key and the Web Standard sign key are created, issuance is complete.

3. Information to Register in GenieGo

FieldDescription
MIDStore ID (MID)
sign_keyThe issued Web Standard sign key (SignKey, handle with care)
api_keyThe issued INIAPI Key
Enter and save in the GenieGo Integration Hub [Register] and the credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
The Additional Info menu is not visibleMerchant contract not completed, or an account without permissionsLog in with a merchant admin account that has a completed merchant contract.
Payment authentication failsSign key or API key entered incorrectlyRe-check the issued sign key and INIAPI Key in Store Info > Contract Info > Additional Info.

Final Checklist

  • Completed the KG Inicis merchant contract and logged in to the merchant admin.
  • Created an INIAPI Key in Store Info > Contract Info > Additional Info.
  • Issued the Web Standard sign key via [Web Payment signkey generation].
  • Entered MID·sign_key·api_key in GenieGo [Register] and saved.
