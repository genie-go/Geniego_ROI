Google Ads
API Key Issuance Manual

A guide that walks you through issuing a developer token in the API Center of your Google Ads manager (MCC) account, creating an OAuth client in Google Cloud to obtain a refresh token, and registering everything in the GenieGo Integration Hub.

Est. timeAbout 20–40 min (developer token review separate)
Account neededGoogle Ads manager (MCC) account · Google account
Keys issuedDeveloper token · client_id/secret · refresh_token · customer_id
ResultGoogle Ads API credentials

1. Before You Begin

Google Ads integration requires both a manager (MCC) account and OAuth setup in Google Cloud.

Developer token

MCC API Center

Apply for a developer token in the API Center of your Google Ads manager (MCC) account. Basic access is approved after a Google review.

OAuth

Google Cloud client

On console.cloud.google.com, create an OAuth client (client_id·client_secret) and authenticate to obtain a refresh token.

Note: Basic access for the developer token is approved after a Google review, and before approval only test accounts can be called. If you don't have a manager (MCC) account, create one first at ads.google.com.

2. Issuance Steps

Issue in the order below and register in GenieGo.

1

Sign in to the manager account · Go to the API Center

Sign in to your Google Ads manager account, then go to [Tools & Settings] > [Setup] > [API Center] at the top.

Tools & Settings → Setup → API Center
2

Apply for a developer token

Apply for a [Developer token] in the API Center. Basic access is approved after a Google review, and before approval only test accounts can be called.

3

Set up a Cloud project · OAuth consent screen

Create a project and configure [APIs & Services] > [OAuth consent screen].

console.cloud.google.com → APIs & Services → OAuth consent screen
4

Create an OAuth client ID

Create [Credentials] > [Create credentials] > [OAuth client ID] to obtain a client_id and client_secret (scope: adwords).

Credentials → Create credentials → OAuth client ID
5

Issue a refresh token

Obtain a refresh token through the OAuth authentication process (use Google OAuth Playground or your own authentication).

6

Find the customer ID

Find the 10-digit customer ID (customer_id) of the ad account you will call.

Once the developer token is approved and client_id·client_secret·refresh_token·customer_id are all ready, issuance is complete.

3. Information to Register in GenieGo

FieldDescription
developer_tokenThe developer token issued in the API Center
client_idOAuth client ID
client_secretOAuth client secret (handle with care)
refresh_tokenThe refresh token issued via OAuth authentication (enter it in the [Access token] field)
customer_idThe 10-digit customer ID of the target ad account
Enter the developer token, client_id, client_secret, refresh_token, and customer_id in GenieGo [Register] and save. Credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
Only test accounts can be calledDeveloper token Basic access review not approvedWait until the Google review of the developer token you applied for in the API Center is approved, then call production accounts.
Authentication failsMissing scope (adwords) or incorrect refresh_tokenAuthenticate again with the adwords scope included in the OAuth client, and recheck the refresh_token.
Customer ID format errorIncorrect 10-digit customer_idRecheck the 10-digit customer ID of the target ad account and enter it.

Final Checklist

  • Prepared a Google Ads manager (MCC) account and signed in.
  • Applied for a developer token in the API Center (Basic access goes through a Google review).
  • Created an OAuth consent screen and OAuth client ID/Secret on console.cloud.google.com (scope: adwords).
  • Obtained a refresh token via OAuth authentication.
  • Found the 10-digit customer_id of the target account.
  • Entered the developer token·client_id/secret·refresh_token·customer_id in GenieGo [Register] and saved.
