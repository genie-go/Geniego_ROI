1. What is the Amazon SP-API?

The Amazon Selling Partner API is an API for programmatically integrating seller data such as orders, inventory, products, shipping, and settlements.

Important: The SP-API is not a simple API Key scheme. It generally requires Amazon Developer registration, app registration, Login with Amazon(LWA), and OAuth authorization together.
★GenieGo integration requires 5 values: refresh_token, client_id, client_secret, marketplace_id, seller_id. (The current SP-API is called with the LWA token alone, and AWS IAM/SigV4 signing is handled internally by GenieGo, so you do not enter it in the registration fields.)

2. Prerequisites

  • An Amazon Seller Central or Vendor Central Professional seller account
  • Company/contact information to use for developer registration
  • App name, app description, and intended use
  • Privacy policy URL
  • OAuth Redirect URL

3. Developer Registration

First, register your SP-API developer profile in Seller Central.

1

Sign in to Seller Central

Sign in to Seller Central with your Amazon seller account.

Seller Central → Apps & Services
2

Go to the Develop Apps menu

From the Apps & Services menu, go to the developer or app development menu.

Apps & Services → Develop Apps
3

Fill in the Developer Profile

Enter your company information, contact information, API use purpose, and data protection plan.

Developer Profile
4

Submit for review

Submit your developer registration information for Amazon's review.

Submit for review
Only after approval is complete can you register a production SP-API app and use its permissions.

4. Register an SP-API App → Client ID / Client Secret

1

Create a new application

Create a new app in Develop Apps.

Add new app
2

Choose the app type

If you connect only your own account it is a Private App; if you provide it to external sellers it is a Public App.

Private / Public App
3

Select API roles

Select the roles you need, such as Orders, Catalog, and Inventory.

API Roles
4

Enter the OAuth Redirect URL

Enter the server URL to return to after authorization is complete.

Redirect URI
5

Save and check the Client info

After creating the app, check the client_id (LWA Client ID) and client_secret (Client Secret).

client_id / client_secret

5. OAuth Authorization → Issue a Refresh Token

1) Access the seller OAuth authorization URL 2) Receive the authorization code at the Redirect URL 3) Pass the authorization code to the LWA Token Endpoint 4) Receive the refresh_token / access_token
The refresh_token is used for long-term calls after seller authorization. GenieGo automatically refreshes the access_token using this refresh_token to call the SP-API, so you only need to register the refresh_token without any separate AWS signing setup.

6. Check the Marketplace ID · Seller ID

Check the remaining two identifiers needed for GenieGo registration.

marketplace_id

Target marketplace ID

The Marketplace ID of the country you are connecting. US: ATVPDKIKX0DER · JP: A1VC38T7YXB528 · DE: A1PA6795UKMFR9

seller_id

Seller ID / Merchant Token

Needed for product registration and order lookup. Check it in your Seller Central account information. Seller Central → Settings → Account Info → Merchant Token

7. Register in the GenieGo Integration Hub

1

Prepare the 5 values

Prepare refresh_token · client_id · client_secret · marketplace_id · seller_id.

5 credentials
2

Integration Hub → Amazon SP-API → [Register]

Enter each value in its field and save (blank fields keep the existing values).

refresh_token · client_id · client_secret · marketplace_id · seller_id
The registered values are encrypted at rest with AES-256-GCM and masked on read. Automatic sync starts immediately after saving.
The client_secret·refresh_token must never be exposed in frontend code, HTML, or public repositories.

8. Final Checklist

  • Submitted the Developer Profile and got it approved.
  • Created the SP-API app and checked the client_id·client_secret.
  • Issued a refresh_token via OAuth authorization.
  • Checked the marketplace_id to connect.
  • Checked the seller_id (Merchant Token) in Seller Central.
  • Entered the 5 values (refresh_token·client_id·client_secret·marketplace_id·seller_id) in GenieGo [Register] and saved.
