KakaoPay
API Key Issuance Manual

A guide that walks you through creating an application in the KakaoPay Developer Center (developers.kakaopay.com) to issue a merchant code (CID) and Secret Key, then registering them in the GenieGo Integration Hub.

Est. timeAbout 5–15 min
Account neededKakaoPay merchant contract · Developer Center account
Keys issuedMerchant code (CID) · Secret Key
ResultPayment integration credentials

1. Before You Begin

KakaoPay issues credentials in the Developer Center after a merchant contract is in place.

Required

Merchant contract · Developer Center account

Your KakaoPay merchant contract must be complete, and you must be able to sign up for and log in to the Developer Center (developers.kakaopay.com).

Keys issued

CID · Secret Key

When you create an application, a merchant code (CID) and a Secret Key are issued.

Note: When calling the KakaoPay API, the authentication header uses the Authorization: SECRET_KEY format.

2. Issuance Steps

Issue in the order below and register in GenieGo.

1

Sign up for and log in to the Developer Center

Sign up for the KakaoPay Developer Center and log in.

https://developers.kakaopay.com
2

Create an application

Create an application to issue or check the merchant code (CID) and Secret Key.

3

Check the authentication header

The issued Secret Key is used in the authentication header in the Authorization: SECRET_KEY format.

Authorization: SECRET_KEY ${secret_key}
Once the merchant code (CID) and Secret Key are issued, you are properly set up.

3. Information to Register in GenieGo

FieldDescription
cidThe issued merchant code (CID)
secret_keyThe issued Secret Key (handle with care, Authorization: SECRET_KEY)
Enter and save them in the GenieGo Integration Hub [Register], and the credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
Authentication header errorThe Authorization header format is incorrectCheck that the Secret Key is passed in the Authorization: SECRET_KEY format.
Cannot create an applicationMerchant contract not completedComplete the merchant contract, then create the application in the Developer Center.

Final Checklist

  • Completed the KakaoPay merchant contract and signed up for the Developer Center (developers.kakaopay.com).
  • Created an application and issued the merchant code (CID) and Secret Key.
  • Verified the Authorization: SECRET_KEY authentication header format.
  • Entered cid·secret_key in GenieGo [Register] and saved.
