Toss Payments
API Key Issuance Manual

A manual covering how to look up your Client Key and Secret Key in the Developer Center of the Toss Payments merchant admin and register them in the GenieGo Integration Hub.

Est. timeAbout 5–10 min
Account neededToss Payments merchant contract · merchant admin account
Keys issuedClient Key · Secret Key
ResultPayment integration credentials

1. Before You Begin

With Toss Payments, you can look up your keys in the merchant admin after signing a merchant contract.

Required

Merchant contract · merchant admin account

You must have completed a Toss Payments merchant contract and be able to sign in to the merchant admin.

Keys issued

Client Key · Secret Key

Look up the Client Key and Secret Key together as a set in the Developer Center and use them together.

Note: The Developer Center separates test and live, so make sure you use the live keys for the actual integration.

2. Issuance Steps

Look them up in the order below and register them in GenieGo.

1

Sign in to the merchant admin

Sign in to the Toss Payments merchant admin.

https://dashboard.tosspayments.com
2

Go to the Developer Center

Go to [Developer Center] at the bottom left.

Developer Center
3

Look up the keys

Select test/live and look up the Client Key and Secret Key. The two keys are used together as a set.

Once you have confirmed both the Client Key and the Secret Key, you are ready to register.

3. Information to Register in GenieGo

FieldDescription
client_keyThe Client Key found in the Developer Center
secret_keyThe Secret Key found in the Developer Center (handle with care)
Enter them in the GenieGo Integration Hub [Register] and save. Credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
Authentication error occursConfusing the test key with the live keyCheck the test/live distinction in the Developer Center and use the key that matches your environment.
No Developer Center menuMerchant contract not completedSign in with a merchant admin account whose merchant contract is complete.

Final Checklist

  • Completed a Toss Payments merchant contract and signed in to the merchant admin.
  • Went to [Developer Center] at the bottom left.
  • Distinguished test/live and looked up the Client Key and Secret Key.
  • Entered client_key·secret_key in GenieGo [Register] and saved.
