📘 Beginner Guide

Meta Ads
API Key Issuance Manual

A beginner-friendly, step-by-step guide to issuing a system user access token from your Meta business app and registering it together with your ad account ID (act_) in the GenieGo Integration Hub to connect your Meta Ads advertising data.

Est. timeAbout 10–20 min
Account neededFacebook personal account · Meta ad account
Keys issuedAccess token · Ad account ID (act_)
ResultMarketing API credentials

1. Before You Begin

Meta Ads integration requires a business account and an ad account that is actively running ads.

Required

Facebook personal account · Ad account

You need a Facebook personal account and a Meta ad account that is actively running ads.

Keys issued

Access token · Ad account ID (act_)

You will obtain a system user token (access token) and a numeric ad account ID that starts with act_.

Note: Accessing data outside your own business requires App Review. If you only connect ad accounts within your own business, you can proceed without review.

2. Issuance Steps

Issue in the order below and register in GenieGo.

1

Create and verify a business account

Open the site in your browser, sign in, then create a business account via [Create Business] and complete email verification.

business.facebook.com
2

Create an app

Click [My Apps] > [Create App] in the top right, choose the "Business" type, and create the app.

developers.facebook.com → My Apps → Create App
3

Add the Marketing API

On the app dashboard, find the "Marketing API" card and click [Set up] to add it as a product.

4

Create a system user and assign permissions

Go to Business Settings > [Users] > [System Users] > [Add] to create a system user, then grant ad-account permissions under [Assign Assets].

business.facebook.com/settings → Users → System Users
5

Generate an access token

On the same screen, click [Generate Token], select your app, check the permissions (ads_read, ads_management), generate the token, and copy its value.

6

Find your ad account ID

In Ads Manager, find the numeric ad account ID that starts with act_.

adsmanager.facebook.com
Once the token is generated with the ads_read and ads_management permissions checked, it has been issued correctly.

3. Information to Register in GenieGo

FieldDescription
access_tokenThe issued system user access token (handle with care)
ad_account_idAd account ID that starts with act_
In the GenieGo Integration Hub > the channel's [Register], paste your access token and ad account ID (act_) and save. Credentials are encrypted at rest with AES-256-GCM and masked on read.

4. Common Issues

IssueCauseSolution
Data can't be queried due to permission errorsToken permissions (ads_read·ads_management) not checked, or ad account permission not assignedAssign ad-account permission to the system user, and check the ads_read·ads_management permissions when generating the token.
Access to another business's data is deniedApp Review not completedTo access data outside your own business, apply for and complete App Review.
Ad account ID format errorMissing act_ prefix, or numbers onlyIn Ads Manager, find the full ID that starts with act_ and enter it.

Final Checklist

  • Created a business account at business.facebook.com and completed email verification.
  • Created a Business-type app at developers.facebook.com and added the Marketing API.
  • Created a system user and assigned ad-account permission.
  • Generated and copied an access token with the ads_read·ads_management permissions.
  • Found the ad account ID that starts with act_.
  • Entered access_token·ad_account_id in GenieGo [Register] and saved.
